# agents/ 目录使用说明

## 概述

`agents/` 目录用于定义子代理（Subagents），每个子代理有独立的提示和工具集，适合处理特定领域的任务。

## 文件位置

```
项目/.claude/agents/*.md
全局/.claude/agents/*.md
```

## 作用

- 创建专门的子代理处理特定任务
- 每个子代理有独立的系统提示
- 可以限制子代理可用的工具
- 实现任务分离和专业化

## 文件格式

### 基本结构

```markdown
---
description: 代码审查专家
model: claude-opus-4-7
tools: [Read, Bash]
timeout: 300000
---

# 代码审查专家

你是一个专业的代码审查专家，专注于：
- 发现代码问题
- 提供改进建议
- 确保代码质量
```

### Frontmatter 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| description | string | 代理描述 |
| model | string | 使用的模型（可选） |
| tools | string[] | 可用工具列表（可选） |
| timeout | number | 超时时间毫秒（可选） |
| disable | string[] | 禁用的工具（可选） |

## 可用工具

| 工具 | 说明 |
|------|------|
| Read | 读取文件 |
| Write | 写入文件 |
| Edit | 编辑文件 |
| Bash | 执行命令 |
| Agent | 调用其他代理 |
| WebSearch | 网页搜索 |
| AskUserQuestion | 询问用户 |

## 代理示例

### 代码审查代理

```markdown
---
description: 专业的代码审查专家
model: claude-opus-4-7
tools: [Read, Edit, Bash]
disable: [Write]
---

# 代码审查专家

## 你的职责

你是一个专业的代码审查专家，专注于发现代码问题和提供改进建议。

## 审查重点

1. **正确性**
   - 逻辑错误
   - 边界条件
   - 错误处理

2. **安全性**
   - 注入漏洞
   - 敏感信息泄露
   - 权限问题

3. **性能**
   - 算法效率
   - 资源使用
   - 缓存策略

4. **可维护性**
   - 代码清晰度
   - 命名规范
   - 文档完整

## 输出格式

### ✅ 优秀
- 亮点 1
- 亮点 2

### ⚠️ 建议
- 建议 1
- 建议 2

### ❌ 问题
- 问题 1
- 问题 2
```

### 文档专家代理

```markdown
---
description: 技术文档编写专家
model: claude-sonnet-4-6
tools: [Read, Write]
---

# 文档专家

## 你的职责

你是一个技术文档专家，负责编写清晰、完整、易懂的技术文档。

## 文档原则

1. **清晰易懂**
   - 使用简单语言
   - 避免术语堆砌
   - 提供示例说明

2. **结构完整**
   - 概览章节
   - 详细说明
   - 示例代码
   - 常见问题

3. **保持更新**
   - 与代码同步
   - 标注版本信息
   - 记录变更历史

## 输出格式

使用 Markdown 格式，包含：
- 目录
- 标题层级
- 代码块
- 表格
- 链接
```

### 测试专家代理

```markdown
---
description: 测试策略和实现专家
model: claude-opus-4-7
tools: [Read, Write, Edit, Bash]
---

# 测试专家

## 你的职责

你是一个测试专家，负责设计测试策略和编写测试代码。

## 测试类型

1. **单元测试**
   - 测试单个函数
   - 覆盖边界条件
   - Mock 外部依赖

2. **集成测试**
   - 测试模块交互
   - 测试 API 接口
   - 测试数据库操作

3. **端到端测试**
   - 测试用户流程
   - 测试关键路径
   - 验证业务逻辑

## 测试原则

- 测试应该独立运行
- 测试应该快速执行
- 测试应该易于维护
- 测试名称应该描述清楚

## 输出格式

```typescript
describe('功能名称', () => {
  it('应该 xxx', () => {
    // 测试代码
  });
});
```
```

   - 找出低效操作
   - 确定优化方向

3. **优化方案**
   - 提供优化建议
   - 评估优化效果
   - 权衡成本收益

## 输出格式

### 性能分析
- 当前指标: xxx

- 差距: xxx

### 优化建议
1. 建议 1
   - 预期提升: xxx

├── code-reviewer.md    # 代码审查专家
├── doc-writer.md       # 文档专家
├── test-expert.md      # 测试专家
├── security-auditor.md # 安全专家
└── performance.md      # 性能专家
```

## 最佳实践

1. **职责单一**：每个代理专注于一个领域
2. **工具限制**：根据需要限制可用工具
3. **清晰提示**：明确代理的职责和输出格式
4. **合理模型**：根据任务复杂度选择模型
5. **适当超时**：设置合理的执行超时

## 与 Skills 的区别

| 特性 | agents/ | skills/ |
|------|---------|---------|
| 独立性 | 完全独立的代理 | 共享主会话 |
| 工具 | 可以限制工具 | 继承主会话工具 |
| 提示 | 独立系统提示 | 任务提示 |
| 用途 | 专业化任务 | 可重用流程 |

## 注意事项

- 代理定义应该提交到代码库
- 复杂代理考虑独立内存
- 定期更新代理提示
- 测试代理输出质量
- 团队共享有效配置
