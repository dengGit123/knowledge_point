# output-styles/ 目录使用说明

## 概述

`output-styles/` 目录用于存放自定义系统提示部分，控制 Claude Code 的输出风格和格式。

## 文件位置

```
项目/.claude/output-styles/*.md
全局/.claude/output-styles/*.md
```

## 作用

- 自定义 Claude 的响应风格
- 控制输出格式和结构
- 调整响应的详细程度
- 定义特定场景的输出规范

## 文件格式

### 基本结构

```markdown
---
description: 简洁输出风格
---

请使用简洁的风格回应：
- 直接给出答案
- 避免冗余说明
- 使用列表形式
- 省略背景信息
```

### Frontmatter 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| description | string | 输出风格描述 |
| trigger | string[] | 触发关键词（可选） |

## 使用方式

### 1. 自动应用

根据文件名自动加载对应的输出风格。

### 2. 触发词应用

如果设置了 trigger，包含关键词时会应用该风格。

## 输出风格示例

### 简洁风格

```markdown
---
description: 极简输出风格
---

# 输出规范

## 原则
- 只输出结果，不解释
- 使用最少的文字
- 省略所有客套话

## 格式
```
结果: xxx
状态: xxx
```
```

### 详细风格

```markdown
---
description: 详细输出风格
---

# 输出规范

## 原则
- 详细解释每个步骤
- 提供背景信息
- 说明决策原因
- 包含相关资源

## 格式
1. 背景说明
2. 分析过程
3. 执行步骤
4. 结果验证
5. 后续建议
```

## 目录示例

```
.claude/output-styles/
├── concise.md       # 简洁风格
├── detailed.md      # 详细风格
├── teaching.md      # 教学风格
├── audit.md         # 审计风格
├── report.md        # 报告风格
└── debug.md         # 调试风格
```

## 最佳实践

1. **明确场景**：每个风格对应特定使用场景
2. **一致格式**：同类输出使用统一格式
3. **清晰结构**：输出有清晰的组织结构
4. **适当详细**：根据需要调整详细程度
5. **易于解析**：考虑机器可读性

## 应用建议

### 场景映射

| 场景 | 推荐风格 |
|------|----------|
| 快速问答 | concise |
| 代码审查 | audit |
| 教学指导 | teaching |
| 问题排查 | debug |
| 方案设计 | detailed |
| 总结报告 | report |

## 注意事项

- 输出风格会叠加到系统提示
- 多个风格文件可能冲突
- 建议针对不同场景创建不同风格
- 风格文件应该提交到代码库
- 团队应统一输出风格
