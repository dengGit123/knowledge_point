# themes/ 目录使用说明

## 什么是 themes/

`themes/` 目录用于存放**自定义主题文件**，控制 Claude Code 界面的外观样式。

> **通俗理解**：就像给 Claude Code 换"皮肤"，可以自定义界面的颜色方案，让它看起来更符合你的审美习惯，或者与其他工具（如 VS Code、终端）保持一致的视觉风格。

## 文件位置

```
全局/.claude/themes/*.json
```

## 目录结构

```
.claude/themes/
├── dark.json              # 内置深色主题
├── light.json             # 内置浅色主题
├── dracula.json           # Dracula 主题
├── nord.json              # Nord 主题
├── solarized-dark.json    # Solarized Dark 主题
├── monokai.json           # Monokai 主题
├── github-dark.json       # GitHub Dark 主题
└── custom.json            # 自定义主题
```

## 为什么需要自定义主题

### 问题场景

默认主题可能不适合所有人：
- 长时间工作需要柔和的颜色
- 希望与其他开发工具风格统一
- 特定光线条件下需要特定的配色
- 个人审美偏好

### 解决方案

通过自定义主题：
- 选择适合的配色方案
- 与其他工具保持一致
- 提高长时间工作的舒适度
- 体现个人风格

## 主题文件格式

### 基本结构

```json
{
  "name": "主题名称",
  "colors": {
    "background": "#1e1e1e",
    "foreground": "#d4d4d4",
    "primary": "#007acc",
    "secondary": "#4ec9b0"
  }
}
```

### 必需字段

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| name | string | ✅ | 主题显示名称 |
| colors | object | ✅ | 颜色配置对象 |

### 颜色字段详解

| 颜色字段 | 说明 | 推荐用途 | 示例值 |
|----------|------|----------|--------|
| background | 背景色 | 界面主背景 | `#1e1e1e` |
| foreground | 前景色/文字色 | 主要文字颜色 | `#d4d4d4` |
| primary | 主色调 | 按钮、链接、强调元素 | `#007acc` |
| secondary | 次要色调 | 辅助元素、次要信息 | `#4ec9b0` |
| accent | 强调色 | 高亮、特殊标记 | `#ce9178` |
| error | 错误色 | 错误提示、失败状态 | `#f14c4c` |
| warning | 警告色 | 警告提示、注意信息 | `#cca700` |
| success | 成功色 | 成功提示、完成状态 | `#4ec9b0` |
| border | 边框色 | 分隔线、边框 | `#3c3c3c` |
| selection | 选中色 | 文本选中、行高亮 | `#264f78` |

## 如何应用主题

### 方法 1：通过配置文件

在 `~/.claude.json` 中设置：

```json
{
  "uiPreferences": {
    "theme": "Dracula"
  }
}
```

### 方法 2：通过命令

在 Claude Code 中执行：

```bash
/config
```

然后按照提示选择主题。

### 方法 3：环境变量

```bash
export CLAUDE_THEME="Dracula"
```

## 创建自定义主题

### 步骤流程

```
1. 选择基础主题作为参考
   ↓
2. 调整各个颜色值
   ↓
3. 保存为新的 JSON 文件
   ↓
4. 测试主题效果
   ↓
5. 根据需要微调
```

### 示例：基于 Dark 创建自定义主题

```json
{
  "name": "My Custom Dark",
  "colors": {
    "background": "#232323",    // 稍微调亮背景
    "foreground": "#e0e0e0",     // 更亮的文字
    "primary": "#4fc3f7",       // 青色主调
    "secondary": "#81c784",     // 柔和的绿色
    "accent": "#ffb74d",        // 橙色强调

    "warning": "#ffa726",       // 橙色警告

    "border": "#424242",        // 边框色
    "selection": "#2e3c4f"      // 选中色
  }
}
```

## 主题设计指南

### 1. 确保足够的对比度

前景色和背景色需要足够的对比度，确保文字可读：

```
✅ 好的对比度
background: #1e1e1e
foreground: #d4d4d4

❌ 对比度不足
background: #2a2a2a
foreground: #303030
```

### 2. 语义明确

错误、警告、成功色应该有明显区分：

```
✅ 清晰的语义
error:   红色系 #f14c4c
warning: 黄色系 #cca700
success: 绿色系 #4ec9b0

❌ 模糊的语义
error:   红色系 #f14c4c
warning: 橙色系 #f05050
success: 深红系 #d03030
```

### 3. 配色和谐

使用相近的色相或互补色，避免颜色冲突：

```
✅ 和谐的配色
使用色轮上相邻的颜色
或使用互补色搭配

❌ 冲突的配色
使用过多不相关的颜色
```

### 4. 保持一致

主色调和辅助色应该与整体风格保持一致：

```
✅ 一致的风格
蓝色主题: primary-blue, secondary-light-blue, accent-blue-purple

❌ 不一致的风格
主色调蓝色，辅助色红色，强调色黄色
```

## 最佳实践

### 1. 命名清晰

主题名称要能描述其风格：

```
✅ 好的命名
"Dracula"
"Nord"
"GitHub Dark"

❌ 不好的命名
"Theme1"
"MyTheme"
"Custom"
```

### 2. 配色和谐

确保颜色搭配协调，避免刺眼或不舒适的组合。

### 3. 对比度足够

保证文字可读性，特别是前景色和背景色。

### 4. 语义明确

错误、警告、成功色有明显区分，便于快速识别。

### 5. 持续维护

根据使用体验调整颜色，保持舒适度。

## 注意事项

1. **颜色格式**：颜色值使用十六进制格式 `#rrggbb`
2. **文件命名**：文件名与主题名称不需要一致
3. **主题位置**：主题文件放在全局 `~/.claude/themes/` 目录
4. **重启生效**：修改主题后可能需要重启 Claude Code
5. **颜色空间**：支持的颜色空间为 sRGB

## 常见问题

### Q：如何预览主题效果？

**A**：
1. 创建主题文件
2. 在配置中应用主题
3. 重启 Claude Code 查看效果
4. 根据需要调整颜色

### Q：主题支持透明度吗？

**A**：不支持。颜色必须是完整的六位十六进制值，不支持 alpha 通道。

### Q：可以只修改部分颜色吗？

**A**：可以复制现有主题，只修改需要的颜色字段，其他使用默认值。

### Q：如何与其他团队成员共享主题？

**A**：
1. 将主题文件提交到代码库
2. 在 README 或文档中说明主题用途
3. 团队成员复制主题到本地

### Q：修改主题后不生效怎么办？

**A**：
1. 检查主题文件格式是否正确（有效的 JSON）
2. 确认主题文件在正确位置
3. 重启 Claude Code
4. 检查配置文件中的主题名称拼写
