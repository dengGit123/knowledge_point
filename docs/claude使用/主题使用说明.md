# themes/ 目录使用说明

## 概述

`themes/` 目录用于存放自定义主题文件，控制 Claude Code 界面的外观样式。

## 文件位置

```
全局/.claude/themes/*.json
```

## 主题文件格式

### 基本结构

```json
{
  "name": "主题名称",
  "colors": {
    "background": "#1e1e1e",
    "foreground": "#d4d4d4",
    "primary": "#007acc",
    "secondary": "#4ec9b0"
  }
}
```

### 必需字段

| 字段 | 类型 | 说明 |
|------|------|------|
| name | string | 主题名称 |
| colors | object | 颜色配置 |

### 颜色字段

| 颜色 | 说明 | 示例 |
|------|------|------|
| background | 背景色 | `#1e1e1e` |
| foreground | 前景色/文字色 | `#d4d4d4` |
| primary | 主色调 | `#007acc` |
| secondary | 次要色调 | `#4ec9b0` |
| accent | 强调色 | `#ce9178` |
| error | 错误色 | `#f14c4c` |
| warning | 警告色 | `#cca700` |
| success | 成功色 | `#4ec9b0` |
| border | 边框色 | `#3c3c3c` |
| selection | 选中色 | `#264f78` |

## 应用主题

### 通过配置文件

在 `~/.claude.json` 中设置：

```json
{
  "uiPreferences": {
    "theme": "Dracula"
  }
}
```

### 通过命令

```
/config
```

然后选择主题。

## 最佳实践

1. **命名清晰**：主题名称要能描述其风格
2. **配色和谐**：确保颜色搭配协调
3. **对比度足够**：保证文字可读性
4. **语义明确**：错误、警告、成功色有明显区分
5. **持续维护**：根据需要调整颜色

## 创建主题步骤

1. 选择基础主题作为参考
2. 调整各个颜色值
3. 保存为新的 JSON 文件
4. 测试主题效果
5. 根据需要微调

## 注意事项

- 颜色值使用十六进制格式 `#rrggbb`
- 文件名与主题名称不需要一致
- 主题文件放在全局目录
- 修改主题后可能需要重启
- 支持的颜色空间：sRGB
