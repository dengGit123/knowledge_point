# CLAUDE.md 使用说明

## 概述

`CLAUDE.md` 是项目级的配置文件，用于定义项目特定的约定、指令和文档。Claude Code 在每次会话开始时自动读取该文件。

## 文件位置

```
项目根目录/.claude/CLAUDE.md
```

## 主要用途

### 1. 项目约定

```markdown
# 项目约定

## 技术栈
- 前端：React + TypeScript
- 状态管理：Zustand
- 样式：Tailwind CSS

## 代码风格
- 使用函数式组件
- 状态放在组件顶部
- Props 使用 TypeScript 接口定义
```

### 2. 目录结构说明

```markdown
## 目录结构

```
src/
├── components/    # 可复用组件
├── pages/        # 页面组件
├── hooks/        # 自定义 Hooks
├── utils/        # 工具函数
└── types/        # 类型定义
```
```

### 3. 工作流程指南

```markdown
## 开发流程

1. 创建功能分支：`feature/功能名称`
2. 开发完成后运行测试：`npm test`
3. 代码必须通过 ESLint 检查
4. 提交前使用 Conventional Commits 格式
```

### 4. 常见命令

```markdown
## 常用命令

| 命令 | 说明 |
|------|------|
| npm dev | 启动开发服务器 |
| npm test | 运行测试 |
| npm build | 构建生产版本 |
| npm lint | 代码检查 |
```

### 5. 注意事项

```markdown
## 注意事项

- 不要直接修改 src/core/ 中的文件
- 新增 API 需要更新文档
- 数据库变更需要写迁移脚本
```

## 最佳实践

1. **保持简洁**：只包含真正需要 Claude 知道的信息
2. **分类清晰**：使用标题组织内容
3. **及时更新**：项目变更时同步更新
4. **避免冗余**：不要重复 README 的内容
