### 一、概述

蓝牙（Bluetooth）是一种短距离无线通信技术标准，工作在 2.4 GHz ISM 频段，用于在手机、电脑、智能设备等之间传输数据。对于前端开发者而言，蓝牙技术已成为连接硬件设备（打印机、手环、传感器等）的重要桥梁，微信小程序、uni-app 和 H5 均提供了蓝牙开发 API。

> 📖 **参考文档：**
> - [Bluetooth SIG 官方规范](https://www.bluetooth.com/specifications/specs/)
> - [微信小程序 - 蓝牙低功耗 (BLE) API](https://developers.weixin.qq.com/miniprogram/dev/framework/device/bluetooth-ble.html)
> - [微信小程序 - 蓝牙信标 (Beacon)](https://developers.weixin.qq.com/miniprogram/dev/framework/device/beacon.html)
> - [uni-app - 蓝牙低功耗 API](https://uniapp.dcloud.net.cn/api/system/ble.html)
> - [Web Bluetooth API - MDN](https://developer.mozilla.org/en-US/docs/Web/API/Web_Bluetooth_API)

### 二、蓝牙版本演进

#### 1. 版本发展历程

| 版本 | 发布年份 | 最大速率 | 最大范围 | 核心特性 |
|---|---|---|---|---|
| **1.0 / 1.0B** | 1999 | 721 Kbps | ~10 m | 初代标准，互操作性问题多 |
| **1.1 / 1.2** | 2003 | 721 Kbps | ~10 m | 修复互操作性问题，增加自适应跳频（AFH） |
| **2.0 + EDR** | 2004 | 3 Mbps | ~10 m | **EDR（增强数据速率）**，音频质量显著提升 |
| **3.0 + HS** | 2009 | 24 Mbps | ~10 m | 通过 Wi-Fi 协议实现高速传输（借助 802.11） |
| **4.0** | 2010 | 25 Mbps | ~50 m | ⭐ **引入 BLE（低功耗蓝牙）**，里程碑版本 |
| **4.1** | 2013 | 25 Mbps | ~50 m | 改善与 LTE 共存，支持多设备同时连接 |
| **4.2** | 2014 | 25 Mbps | ~50 m | 增加 IPv6/6LoWPAN，数据包容量增大 10 倍 |
| **5.0** | 2016 | 50 Mbps | ~200 m | ⭐ **4× 范围、2× 速度、8× 广播容量** |
| **5.1** | 2019 | 50 Mbps | ~200 m | **方向寻址（AoA/AoD）**，厘米级定位精度 |
| **5.2** | 2020 | 50 Mbps | ~200 m | **LE Audio（LC3 编码）**，支持多流音频 |
| **5.3** | 2021 | 50 Mbps | ~200 m | 优化周期性广播，增强加密密钥长度控制 |
| **5.4** | 2023 | 50 Mbps | ~200 m | PAwR（带响应的周期性广播），电子货架标签场景 |

#### 2. 蓝牙 5.0 核心改进

蓝牙 5.0 是继 4.0 引入 BLE 之后的又一次重大升级：

```
                    蓝牙 5.0 vs 4.2 对比

  ┌─────────────────────────────────────────────┐
  │                                              │
  │   传输速度    1 Mbps  ───────→  2 Mbps       │
  │              (×2 提升)                       │
  │                                              │
  │   传输距离    ~50 m   ───────→  ~200 m       │
  │              (×4 提升)                       │
  │                                              │
  │   广播容量    31 字节 ───────→  255 字节      │
  │              (×8 提升)                       │
  │                                              │
  └─────────────────────────────────────────────┘
```

> 💡 **提示：** 前端开发者通常不需要关心蓝牙底层版本细节，但了解版本差异有助于理解设备的兼容性和能力边界。例如：只有蓝牙 4.0+ 的设备才支持 BLE，蓝牙 5.0+ 的设备才能享受更远的通信距离。

### 三、经典蓝牙 vs BLE

#### 1. 核心对比

| 对比项 | 经典蓝牙 (Classic) | BLE (低功耗蓝牙) |
|---|---|---|
| **协议版本** | Bluetooth 2.1/3.0/4.0+ | Bluetooth 4.0+ |
| **通信模型** | SPP（串口协议） | GATT（通用属性协议） |
| **传输速率** | 较高（1~3 Mbps） | 较低（125 Kbps~2 Mbps） |
| **功耗** | 较高（数十 mA） | 极低（微安级待机） |
| **连接时间** | 数秒 | 数毫秒 |
| **数据传输方式** | 流式（类似串口） | 基于 Service/Characteristic 读写 |
| **小程序支持** | 已逐步弃用 | ✅ 完全支持（主流方案） |
| **Web Bluetooth** | ❌ 不支持 | ✅ 支持（仅 Chromium 内核） |
| **典型设备** | 蓝牙耳机、音箱 | 手环、传感器、打印机、Beacon |

#### 2. 如何选择

```js
// ✅ 选择判断流程
function chooseBluetoothType(device) {
  if (device.needHighSpeed || device.needAudioStream) {
    return 'Classic'   // 音频流传输、大数据量 → 经典蓝牙
  }
  if (device.needLowPower || device.needMobileConnect) {
    return 'BLE'       // IoT、穿戴、打印机、小程序 → BLE
  }
  return 'BLE'         // 前端开发默认选 BLE
}
```

### 四、BLE GATT 协议深入

GATT（Generic Attribute Profile，通用属性配置文件）是 BLE 的核心数据通信协议，定义了设备之间如何组织和交换数据。

#### 1. GATT 层级结构

```
┌──────────────────────────────────────────────────────────┐
│                    GATT Server（从设备）                    │
│                                                           │
│  ┌─── Service A（服务） ────────────────────────────┐     │
│  │  UUID: 0000180A-...  (Device Information)        │     │
│  │                                                   │     │
│  │  ┌── Characteristic 1（特征值）────────────┐     │     │
│  │  │  UUID: 00002A29-...                     │     │     │
│  │  │  Properties: read                       │     │     │
│  │  │  Value: "Manufacturer Name"             │     │     │
│  │  │                                          │     │     │
│  │  │  ┌── Descriptor（描述符）────────┐       │     │     │
│  │  │  │  UUID: 0x2901                │       │     │     │
│  │  │  │  Value: "厂家名称"            │       │     │     │
│  │  │  └──────────────────────────────┘       │     │     │
│  │  └─────────────────────────────────────────┘     │     │
│  └───────────────────────────────────────────────────┘     │
│                                                           │
│  ┌─── Service B（服务） ────────────────────────────┐     │
│  │  UUID: 0000FF00-...  (自定义服务)                │     │
│  │                                                   │     │
│  │  ┌── Characteristic（写）─────────────────┐      │     │
│  │  │  UUID: 0000FF02-...                    │      │     │
│  │  │  Properties: write, writeWithoutResponse│      │     │
│  │  └────────────────────────────────────────┘      │     │
│  │                                                   │     │
│  │  ┌── Characteristic（通知）───────────────┐      │     │
│  │  │  UUID: 0000FF01-...                    │      │     │
│  │  │  Properties: notify, indicate          │      │     │
│  │  └────────────────────────────────────────┘      │     │
│  └───────────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────────┘
```

#### 2. 四个核心概念

| 概念 | 作用 | 类比 |
|---|---|---|
| **Profile（配置文件）** | 由多个 Service 组成，定义设备的完整功能 | 一本产品说明书 |
| **Service（服务）** | 由多个 Characteristic 组成，代表一个功能模块 | 说明书中的一章 |
| **Characteristic（特征值）** | 承载具体数据，是读写操作的基本单元 | 章节中的一个参数项 |
| **Descriptor（描述符）** | 描述 Characteristic 的元信息（如单位、说明） | 参数项的备注说明 |

#### 3. UUID 详解

UUID（Universally Unique Identifier）用于唯一标识 Service 和 Characteristic。

| 类型 | 格式 | 说明 | 示例 |
|---|---|---|---|
| **16-bit UUID** | `XXXX` | Bluetooth SIG 官方定义的标准服务/特征值 | `180A`（设备信息服务）、`2A29`（厂商名称） |
| **128-bit UUID** | `XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX` | 厂商自定义的服务/特征值 | `0000FF00-0000-1000-8000-00805F9B34FB` |

```js
// 16-bit UUID 转 128-bit UUID 的规则
// Bluetooth SIG 基础 UUID: 0000XXXX-0000-1000-8000-00805F9B34FB
function uuid16To128(uuid16) {
  const base = '0000-1000-8000-00805F9B34FB'
  return `0000${uuid16}-${base}`
}

// 示例: 0x180A → 0000180A-0000-1000-8000-00805F9B34FB
console.log(uuid16To128('180A'))
```

> 💡 **提示：** Bluetooth SIG 定义了标准 UUID（可在 [官方 UUID 列表](https://www.bluetooth.com/specifications/assigned-numbers/) 查看）。实际开发中，打印机等外设通常使用厂商自定义的 128-bit UUID，需要通过 API 动态枚举。

#### 4. Characteristic 属性（Properties）

| 属性 | 说明 | 典型用途 |
|---|---|---|
| `read` | 允许读取特征值 | 读取传感器数据、设备状态 |
| `write` | 允许写入特征值（需响应） | 发送控制指令、打印数据 |
| `writeWithoutResponse` | 允许写入（无需响应，速度更快） | 高频数据发送 |
| `notify` | 允许订阅通知（设备主动推送） | 接收实时数据（心率、温度等） |
| `indicate` | 类似 notify，但需客户端确认 | 重要状态变更通知 |
| `broadcast` | 允许广播 | Beacon 广播 |

#### 5. GATT 通信流程

```
  手机（GATT Client）                      设备（GATT Server）
       │                                       │
       │  ① connect                            │
       │──────────────────────────────────────→│
       │                                       │
       │  ② getPrimaryServices()               │
       │──────────────────────────────────────→│  返回服务列表
       │←──────────────────────────────────────│
       │                                       │
       │  ③ getCharacteristic(uuid)            │
       │──────────────────────────────────────→│  返回特征值
       │←──────────────────────────────────────│
       │                                       │
       │  ④a readValue()                       │
       │──────────────────────────────────────→│  返回数据
       │←──────────────────────────────────────│
       │                                       │
       │  ④b writeValue(data)                  │
       │──────────────────────────────────────→│  写入成功
       │←──────────────────────────────────────│
       │                                       │
       │  ④c startNotifications()              │
       │──────────────────────────────────────→│
       │         ... 持续接收通知 ...            │
       │←─ ←─ ←─ ←─ ←─ ←─ ←─ ←─ ←─ ←─ ←─ ←─│
```

### 五、前端蓝牙开发场景

#### 1. 三大平台对比

| 平台 | API 前缀 | 蓝牙类型 | 安全要求 | 适用场景 |
|---|---|---|---|---|
| **微信小程序** | `wx.` | BLE + Beacon | 小程序内即可 | 打印机、IoT、信标 |
| **uni-app** | `uni.` | BLE + Beacon | 各平台适配 | 跨平台蓝牙应用 |
| **H5 (Web Bluetooth)** | `navigator.bluetooth` | BLE | HTTPS + 用户手势 | PWA、Web 应用 |

#### 2. 微信小程序 BLE API 速查

```js
// ========================================
// 微信小程序 BLE 完整 API 速查
// ========================================

// ---- 适配器管理 ----
wx.openBluetoothAdapter()            // 初始化蓝牙适配器（必须首先调用）
wx.closeBluetoothAdapter()           // 关闭蓝牙适配器
wx.getBluetoothAdapterState()        // 获取蓝牙适配器状态
wx.onBluetoothAdapterStateChange()   // 监听蓝牙适配器状态变化

// ---- 设备搜索（经典蓝牙 + BLE） ----
wx.startBluetoothDevicesDiscovery()  // 开始搜索蓝牙设备
wx.stopBluetoothDevicesDiscovery()   // 停止搜索
wx.getBluetoothDevices()             // 获取已发现的设备
wx.onBluetoothDeviceFound()          // 监听发现新设备

// ---- BLE 连接 ----
wx.createBLEConnection()             // 建立 BLE 连接
wx.closeBLEConnection()              // 断开 BLE 连接
wx.onBLEConnectionStateChange()      // 监听连接状态变化

// ---- BLE 服务与特征值 ----
wx.getBLEDeviceServices()            // 获取设备所有服务
wx.getBLEDeviceCharacteristics()     // 获取服务下所有特征值

// ---- BLE 数据读写 ----
wx.readBLECharacteristicValue()      // 读取特征值数据
wx.writeBLECharacteristicValue()     // 写入特征值数据
wx.notifyBLECharacteristicValueChange() // 订阅/取消订阅特征值通知
wx.onBLECharacteristicValueChange()  // 监听特征值变化

// ---- MTU 协商 ----
wx.setBLEMTU()                       // 设置最大传输单元
```

#### 3. 小程序 BLE 错误码

| errCode | 含义 | 常见原因 |
|---|---|---|
| `10000` | 未初始化蓝牙适配器 | 未先调用 `openBluetoothAdapter` |
| `10001` | 蓝牙适配器不可用 | 蓝牙未开启或设备不支持 |
| `10002` | 没有找到指定设备 | deviceId 错误或设备不可达 |
| `10003` | 连接失败 | 设备离线、信号弱或被其他设备占用 |
| `10004` | 没有找到指定服务 | serviceId 不正确 |
| `10005` | 没有找到指定特征值 | characteristicId 不正确 |
| `10006` | 当前连接已断开 | 设备距离过远或电量耗尽 |
| `10007` | 特征值不支持此操作 | 对只读特征值执行写操作等 |
| `10008` | 系统上报异常 | 其余所有系统错误 |
| `10009` | 系统版本不支持 BLE | Android 4.3 以下 |
| `10012` | 连接超时 | 超出 timeout 设置 |
| `10013` | deviceId 格式不正确 | deviceId 为空或格式错误 |

#### 4. Web Bluetooth API

```js
// ========================================
// Web Bluetooth API 使用示例
// 仅 Chromium 内核浏览器支持，需 HTTPS 环境
// ========================================

// 检测浏览器是否支持
if (!navigator.bluetooth) {
  console.error('当前浏览器不支持 Web Bluetooth API')
}

// 请求设备（必须由用户手势触发，如按钮点击）
async function connectBLE() {
  // 第 1 步：请求用户选择设备
  const device = await navigator.bluetooth.requestDevice({
    filters: [
      { services: ['heart_rate'] },                    // 按标准服务过滤
      // 或按名称过滤
      // { namePrefix: 'Printer' }
    ],
    // 如果无法用 filters，可用 optionalServices + acceptAllDevices
    optionalServices: ['battery_service', '0000ff00-0000-1000-8000-00805f9b34fb']
  })

  // 第 2 步：连接 GATT 服务器
  const server = await device.gatt.connect()

  // 第 3 步：获取服务
  const service = await server.getPrimaryService('heart_rate')

  // 第 4 步：获取特征值
  const characteristic = await service.getCharacteristic('heart_rate_measurement')

  // 第 5 步：订阅通知（接收实时心率数据）
  await characteristic.startNotifications()
  characteristic.addEventListener('characteristicvaluechanged', (event) => {
    const value = event.target.value  // DataView 类型
    const heartRate = value.getUint8(1)
    console.log('心率:', heartRate)
  })

  // 或者：读取数据
  // const value = await characteristic.readValue()

  // 或者：写入数据
  // await characteristic.writeValueWithResponse(new Uint8Array([0x01, 0x02]))
}
```

|---|---|
| Chrome（桌面 / Android） | ✅ 支持 |
| Edge（Chromium） | ✅ 支持 |

#### 5. uni-app 跨平台适配

// ========================================

const ble = {

        success: resolve,

  write(deviceId, serviceId, charId, buffer) {
    return new Promise((resolve, reject) => {
      uni.writeBLECharacteristicValue({
        deviceId,
        serviceId,
        characteristicId: charId,
        value: buffer,
        success: resolve,
        fail: reject
      })
    })

### 六、蓝牙打印机集成

#### 1. 连接流程

蓝牙打印机是前端蓝牙开发最常见的场景之一，完整流程：

```
初始化适配器 → 搜索打印机 → 建立连接 → 获取服务和特征值 → 写入打印数据
```

```js
// ========================================
// 蓝牙打印机连接工具类
// ========================================

class BluetoothPrinter {
  constructor() {
    this.deviceId = ''
    this.serviceId = ''
    this.writeCharId = ''
    this.connected = false
  }

  /** 初始化蓝牙适配器 */
  async openAdapter() {
    try {
      await wx.openBluetoothAdapter()
    } catch (err) {
      if (err.errCode === 10001) {
        throw new Error('请检查蓝牙是否已开启')
      }
      throw err
    }
  }

  /** 搜索蓝牙设备 */
  async searchDevices(onDeviceFound) {
    wx.onBluetoothDeviceFound((res) => {
      res.devices.forEach(device => {
        if (device.name && device.name !== '未知设备') {
          onDeviceFound?.(device)
        }
      })
    })
    await wx.startBluetoothDevicesDiscovery({
      allowDuplicatesKey: false,
      interval: 0
    })
  }

  /** 连接打印机 */
  async connect(deviceId) {
    await wx.stopBluetoothDevicesDiscovery()
    await wx.createBLEConnection({ deviceId, timeout: 10000 })

    this.deviceId = deviceId
    this.connected = true

    // 监听断开
    wx.onBLEConnectionStateChange((res) => {
      if (!res.connected) this.connected = false
    })

    // 尝试协商 MTU
    try {
      await wx.setBLEMTU({ deviceId, mtu: 512 })
    } catch (e) { /* 部分设备不支持 */ }

    // 自动发现写入通道
    await this._discoverWriteChannel()
  }

  /** 自动发现支持 write 的特征值 */
  async _discoverWriteChannel() {
    const { services } = await wx.getBLEDeviceServices({ deviceId: this.deviceId })

    for (const service of services) {
      const { characteristics } = await wx.getBLEDeviceCharacteristics({
        deviceId: this.deviceId,
        serviceId: service.uuid
      })

      for (const char of characteristics) {
        if (char.properties.write) {
          this.serviceId = service.uuid
          this.writeCharId = char.uuid
        }

        // 同时订阅通知特征值（接收打印机状态）
        if (char.properties.notify || char.properties.indicate) {
          await wx.notifyBLECharacteristicValueChange({
            deviceId: this.deviceId,
            serviceId: service.uuid,
            characteristicId: char.uuid,
            state: true
          })
        }
      }

      if (this.writeCharId) break
    }

    if (!this.writeCharId) {
      throw new Error('未找到可写入的特征值')
    }
  }

  /** 写入数据 */
  async writeData(buffer) {
    if (!this.connected) throw new Error('蓝牙未连接')
    await wx.writeBLECharacteristicValue({
      deviceId: this.deviceId,
      serviceId: this.serviceId,
      characteristicId: this.writeCharId,
      value: buffer,
      writeType: 'write'
    })
  }

  /** 断开连接 */
  async disconnect() {
    if (this.deviceId) {
      await wx.closeBLEConnection({ deviceId: this.deviceId })
    }
    await wx.closeBluetoothAdapter()
    this.connected = false
    this.deviceId = ''
  }
}
```

#### 2. ESC/POS 打印指令

蓝牙打印机普遍采用 **ESC/POS** 指令集（Epson 定义的热敏打印机控制标准），所有内容需编码为二进制字节流发送。

**常用指令速查：**

| 功能 | 指令 | 十六进制 | 说明 |
|---|---|---|---|
| 初始化打印机 | `ESC @` | `1B 40` | 恢复默认设置，每次打印前调用 |
| 打印并换行 | `LF` | `0A` | 走纸一行 |
| 走纸 n 行 | `ESC d n` | `1B 44 n` | 向下走纸 n 行 |
| 加粗（开/关） | `ESC E n` | `1B 45 01/00` | n=1 加粗，n=0 取消 |
| 下划线 | `ESC - n` | `1B 2D n` | n=0 关, n=1 细线, n=2 粗线 |
| 对齐方式 | `ESC a n` | `1B 61 n` | n=0 左, n=1 居中, n=2 右 |
| 字符大小 | `GS ! n` | `1D 21 n` | 高 4 位=高度倍数, 低 4 位=宽度倍数 |
| 行间距 | `ESC 3 n` | `1B 33 n` | n 点（1 点≈0.125mm） |
| 打印条形码 | `GS k` | `1D 6B ...` | 支持多种条码格式 |
| 打印二维码 | `GS ( k` | `1D 28 6B ...` | 多步命令：设置大小→纠错→数据→打印 |
| 打印图片 | `GS v 0` | `1D 76 30 ...` | 光栅位图打印 |
| 切纸 | `GS V n` | `1D 56 n` | n=0 全切, n=1 半切 |

**指令工具类：**

```js
// ========================================
// ESC/POS 指令生成器
// ========================================

class ESCPOSBuilder {
  constructor() {
    this.buffer = []
  }

  _add(...bytes) {
    this.buffer.push(...bytes)
    return this
  }

  _text(str) {
    this.buffer.push(...new TextEncoder().encode(str))
    return this
  }

  // 基础指令
  init() { return this._add(0x1B, 0x40) }
  newLine() { return this._add(0x0A) }
  feedLines(n) { return this._add(0x1B, 0x64, n) }
  cut(partial = false) { return this._add(0x1D, 0x56, partial ? 1 : 0) }

  // 文字样式
  setAlign(n) { return this._add(0x1B, 0x61, n) }       // 0:左 1:中 2:右
  setBold(on) { return this._add(0x1B, 0x45, on ? 1 : 0) }
  setUnderline(n) { return this._add(0x1B, 0x2D, n) }    // 0:关 1:细 2:粗
  setTextSize(w, h) {
    return this._add(0x1D, 0x21, ((h & 7) << 4) | (w & 7))
  }
  setLineSpacing(n) { return this._add(0x1B, 0x33, n) }

  // 打印文字
  textLine(str) { this._text(str); return this.newLine() }
  text(str) { return this._text(str) }

  // 打印二维码
  qrcode(data, size = 6, ecLevel = 1) {
    const d = [...new TextEncoder().encode(data)]
    const len = d.length + 3
    this._add(0x1D, 0x28, 0x6B, 3, 0, 0x31, 0x43, size)              // 设置大小
    this._add(0x1D, 0x28, 0x6B, 3, 0, 0x31, 0x45, 48 + ecLevel)      // 设置纠错等级
    this._add(0x1D, 0x28, 0x6B, len % 256, ~~(len / 256), 0x31, 0x50, 0x30, ...d)  // 数据
    this._add(0x1D, 0x28, 0x6B, 3, 0, 0x31, 0x51, 0x30)              // 打印
    return this
  }

  // 打印条形码
  barcode(data, type = 73, width = 3, height = 80, showText = true) {
    this._add(0x1D, 0x48, showText ? 2 : 0)  // HRI 位置
    this._add(0x1D, 0x77, width)               // 模块宽度
    this._add(0x1D, 0x68, height)              // 条码高度
    const d = [...data].map(c => c.charCodeAt(0))
    this._add(0x1D, 0x6B, type, data.length, ...d)
    return this
  }

  // 打印图片（Canvas → 单色位图）
  printImage(canvas, maxWidth = 384) {
    const ctx = canvas.getContext('2d')
    const { data, width, height } = ctx.getImageData(0, 0, canvas.width, canvas.height)
    const ratio = maxWidth / width
    const th = Math.floor(height * ratio)
    const bpr = Math.ceil(maxWidth / 8)
    const bitmap = []

    for (let y = 0; y < th; y++) {
      for (let x = 0; x < bpr; x++) {
        let byte = 0
        for (let bit = 0; bit < 8; bit++) {
          const px = x * 8 + bit
          if (px < maxWidth) {
            const i = (Math.floor(y / ratio) * width + Math.floor(px / ratio)) * 4
            const gray = data[i] * 0.299 + data[i + 1] * 0.587 + data[i + 2] * 0.114
            if (gray < 128) byte |= (0x80 >> bit)
          }
        }
        bitmap.push(byte)
      }
    }

    this._add(0x1D, 0x76, 0x30, 0x00,
      bpr % 256, ~~(bpr / 256),
      th % 256, ~~(th / 256),
      ...bitmap)
    return this
  }

  // 输出
  toBuffer() { return new Uint8Array(this.buffer).buffer }
}
```

#### 3. 数据分包发送

BLE 单次写入数据量有限（通常 20~512 字节），必须分包发送：

```js
/**
 * 分包发送打印数据
 * @param {BluetoothPrinter} printer - 打印机实例
 * @param {ArrayBuffer} buffer - 完整打印数据
 * @param {number} chunkSize - 每包字节数（默认 20，根据 MTU 调整）
 * @param {number} delay - 包间延迟（ms，防止缓冲区溢出）
 */
async function sendInChunks(printer, buffer, chunkSize = 20, delay = 30) {
  const data = new Uint8Array(buffer)
  for (let i = 0; i < data.length; i += chunkSize) {
    const chunk = data.slice(i, Math.min(i + chunkSize, data.length)).buffer
    await printer.writeData(chunk)
    if (delay > 0 && i + chunkSize < data.length) {
      await new Promise(r => setTimeout(r, delay))
    }
  }
}
```

### 七、蓝牙信标（Beacon）

#### 1. Beacon 工作原理

Beacon 是一种基于 BLE 广播的定位技术。Beacon 设备不断向外发送广播包，手机接收后可根据信号强度（RSSI）估算距离，实现**室内定位、消息推送、签到打卡**等功能。

```
┌──────────┐    广播包（UUID + Major + Minor + RSSI）    ┌──────────┐
│  Beacon  │ ─────────────────────────────────────────→  │   手机   │
│  设备 A  │        不需要配对，单向广播                    │  小程序   │
└──────────┘                                           └──────────┘

广播包数据结构：
┌──────────┬──────────┬──────────┬──────────┐
│  UUID    │  Major   │  Minor   │   TX     │
│ (16字节) │ (2字节)  │ (2字节)  │  Power   │
└──────────┴──────────┴──────────┴──────────┘
```

- **UUID**：标识一组 Beacon（如某商场的所有 Beacon）
- **Major**：标识子组（如某楼层）
- **Minor**：标识具体设备（如某个展位）
- **TX Power**：发射功率，用于距离估算

### 九、常见问题与注意事项

#### 1. 连接问题

| 问题 | 原因 | 解决方案 |
|---|---|---|
| 初始化失败（10001） | 蓝牙未开启或不支持 | 引导用户开启蓝牙 |
| 搜索不到设备 | 设备未开机/距离太远/被占用 | 确认打印机开机且未连接其他手机 |
| 连接后立即断开 | 部分设备需先系统配对 | 先在系统蓝牙设置中配对 |
| iOS/Android 行为不同 | 平台限制差异 | iOS 不能按 serviceId 搜索，需按名称过滤 |

#### 2. 数据传输问题

| 问题 | 原因 | 解决方案 |
|---|---|---|
| 写入报错 `property not support` | 特征值不支持 write 或写入过快 | 确认 write 属性，控制写入频率 |
| 打印乱码 | 编码不匹配 | 确保使用 UTF-8；用 `ESC t n` 切换字符代码表 |
| 打印内容截断 | 分包丢包 | 增大包间延迟（50~100ms），确认每包成功 |
| 图片不完整 | 数据量太大 | 降低分辨率，增加延时 |

#### 3. 平台差异

| 差异点 | iOS | Android |
|---|---|---|
| 默认 MTU | 185 字节 | 23 字节（有效 20） |
| 搜索方式 | 不支持指定 serviceId 搜索 | 支持指定 serviceId |
| deviceId 格式 | UUID 格式 | MAC 地址格式 |
| 后台蓝牙 | 受限，需配置 background modes | 相对自由 |

### 十、总结

| 要点 | 说明 |
|---|---|
| **蓝牙版本** | 4.0 引入 BLE 是分水岭，5.0 显著提升范围和速度 |
| **经典蓝牙 vs BLE** | 前端开发**优先选 BLE**，功耗低、小程序/Web 均支持 |
| **GATT 协议** | BLE 通信核心 —— Service → Characteristic → Descriptor 层级结构 |
| **UUID** | 标准 16-bit（SIG 定义）或自定义 128-bit，需动态枚举 |
| **开发平台** | 微信小程序（最成熟）、uni-app（跨平台）、Web Bluetooth（H5，兼容性有限） |
| **蓝牙打印机** | ESC/POS 指令 + BLE 分包发送，注意纸宽适配和编码问题 |
| **Beacon 信标** | 基于 BLE 广播，用于室内定位、签到、消息推送 |
| **平台差异** | iOS/Android 的 MTU、搜索方式、deviceId 格式均不同，需分别测试 |
