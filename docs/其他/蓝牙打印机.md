### 一、概述

蓝牙打印机（特别是蓝牙热敏打印机）是餐饮收银、物流快递、票据打印等场景中非常常见的硬件设备。这类打印机通常使用 **ESC/POS 命令集** 来控制打印内容，通过 **蓝牙** 与手机或电脑进行无线通信。

开发蓝牙打印功能，需要掌握三个核心知识：
1. **蓝牙通信协议** —— 手机怎么和打印机建立连接、怎么发送数据
2. **ESC/POS 命令集** —— 发什么数据能让打印机打出文字、条码、二维码、图片
3. **各平台蓝牙 API** —— 微信小程序、uni-app、Web 端各自的调用方式

---

### 二、蓝牙协议：SPP vs BLE

蓝牙打印机主要使用两种蓝牙通信方式：**经典蓝牙 SPP** 和 **低功耗蓝牙 BLE (GATT)**。两者的通信模型完全不同，开发方式也不同。

#### 1. Classic Bluetooth — SPP（串口协议）

**SPP（Serial Port Profile）** 是经典蓝牙的串口协议，它模拟了一个"无线串口线"。你可以把它想象成：手机和打印机之间连了一根看不见的串口线，数据就像水流一样连续不断地传输过去。

- **原理**：基于 RFCOMM 协议，模拟 RS-232 串口通信
- **数据传输**：流式传输，无需分包，想发多少发多少
- **连接方式**：需要先配对，然后建立 RFCOMM 通道
- **平台支持**：
  - Android：支持（使用 `BluetoothSocket`）
  - iOS：**不支持**（iOS 不开放 SPP 接口）
  - 微信小程序：不支持
  - Web：不支持

#### 2. BLE — GATT（低功耗蓝牙）

**BLE（Bluetooth Low Energy）** 使用 **GATT（Generic Attribute Profile）** 协议进行通信。你可以把它想象成：打印机上有很多个"小盒子"（特征值），每个盒子有自己的编号（UUID），你往指定的盒子里放数据，打印机就从那个盒子里取数据去打印。

- **原理**：基于 GATT 协议，数据通过"服务 → 特征值"的层级结构进行读写
- **数据传输**：每次写入有长度限制（通常 20 字节，MTU 协商后可达 512 字节），大数据需要分包
- **连接方式**：无需配对（也可配对），直接连接 GATT 服务器
- **平台支持**：
  - Android：支持
  - iOS：支持（iOS **只支持** BLE 方式）
  - 微信小程序：支持
  - Web（Chrome）：支持

#### 3. SPP vs BLE 对比

| 对比项 | SPP（经典蓝牙） | BLE（低功耗蓝牙） |
|--------|-----------------|-------------------|
| 功耗 | 较高 | 极低 |
| 数据吞吐量 | 较高（~300KB/s） | 较低（~10-50KB/s） |
| Android 支持 | 支持 | 支持 |
| iOS 支持 | **不支持** | **支持** |
| 微信小程序 | 不支持 | 支持 |
| Web 浏览器 | 不支持 | Chrome 支持 |
| 连接模型 | 流式（串口仿真） | 服务/特征值（属性读写） |
| 数据分包 | 不需要 | 需要（默认 20 字节/包） |
| 典型用途 | 传统蓝牙打印机 | 便携式/电池供电打印机 |

> 💡 **提示：** 由于 iOS 只支持 BLE，现在市面上的便携蓝牙热敏打印机（58mm、80mm）绝大多数都采用 BLE 方式。如果你的产品需要同时支持 Android 和 iOS，**必须使用 BLE**。

### 三、ESC/POS 命令集

**ESC/POS** 是 Epson 制定的打印机控制命令标准（Epson Standard Code for Point Of Sale），几乎所有热敏打印机都兼容这套命令。它的核心思路是：把控制命令和数据混合在一起，通过字节流发送给打印机。

命令格式中的关键符号：
- `ESC` = `0x1B`（十进制 27）—— 转义字符，大部分命令以它开头
- `GS` = `0x1D`（十进制 29）—— Group Separator，高级命令以此开头
- `LF` = `0x0A`（十进制 10）—— 换行
- `NUL` = `0x00`（十进制 0）—— 空字符

> 💡 **提示：** 以下所有字节序列均以十六进制表示。在实际代码中，你需要将这些字节序列构造成 `ArrayBuffer`（微信小程序/uni-app）或 `Uint8Array`（Web）后发送。

#### 1. 打印机初始化

在开始打印之前，通常需要先初始化打印机，清除之前的格式设置。

| 命令 | 名称 | 字节序列（Hex） | 说明 |
|------|------|----------------|------|
| `ESC @` | 初始化打印机 | `1B 40` | 清除打印缓冲区，恢复默认设置 |

```js
// 初始化打印机
const initCmd = new Uint8Array([0x1B, 0x40])
```

#### 2. 文本格式化

##### 2.1 加粗 — `ESC E n`

| 字节序列 | 说明 |
|---------|------|
| `1B 45 01` | 开启加粗 |
| `1B 45 00` | 关闭加粗 |

```js
// 开启加粗
const boldOn = new Uint8Array([0x1B, 0x45, 0x01])
// 关闭加粗
const boldOff = new Uint8Array([0x1B, 0x45, 0x00])
```

##### 2.2 下划线 — `ESC - n`

| 字节序列 | 说明 |
|---------|------|
| `1B 2D 00` | 关闭下划线 |
| `1B 2D 01` | 1 点宽下划线 |
| `1B 2D 02` | 2 点宽下划线 |

##### 2.3 对齐 — `ESC a n`

| 字节序列 | 说明 |
|---------|------|
| `1B 61 00` | 左对齐 |
| `1B 61 01` | 居中对齐 |
| `1B 61 02` | 右对齐 |

```js
// 居中打印标题
const centerAlign = new Uint8Array([0x1B, 0x61, 0x01])
// 左对齐（正文）
const leftAlign = new Uint8Array([0x1B, 0x61, 0x00])
```

##### 2.4 综合打印模式 — `ESC ! n`

通过一个字节 `n` 的位域同时设置多种格式：

| 位 | 功能 | 0 | 1 |
|----|------|---|---|
| Bit 0 (0x01) | 字体选择 | Font A (12×24) | Font B (9×17) |
| Bit 3 (0x08) | 加粗 | 关闭 | 开启 |
| Bit 4 (0x10) | 双高 | 关闭 | 开启 |
| Bit 5 (0x20) | 双宽 | 关闭 | 开启 |
| Bit 7 (0x80) | 下划线 | 关闭 | 开启 |

```js
// 常用组合示例
const normal = new Uint8Array([0x1B, 0x21, 0x00])       // 正常
const boldLarge = new Uint8Array([0x1B, 0x21, 0x38])    // 加粗 + 双高 + 双宽 (0x08 | 0x10 | 0x20)
const underlineBold = new Uint8Array([0x1B, 0x21, 0x88]) // 加粗 + 下划线 (0x08 | 0x80)
```

##### 2.5 字体选择 — `ESC M n`

| 字节序列 | 说明 |
|---------|------|
| `1B 4D 00` | Font A（标准字体，12×24 点） |
| `1B 4D 01` | Font B（紧凑字体，9×17 点） |

##### 2.6 字符大小 — `GS ! n`

通过 `n` 的位域分别控制水平和垂直方向的放大倍数：

- **高 4 位**（Bit 4-7）：垂直方向放大倍数（0-7 表示 1-8 倍）
- **低 4 位**（Bit 0-3）：水平方向放大倍数（0-7 表示 1-8 倍）

```js
// n = (vertical - 1) << 4 | (horizontal - 1)
const doubleWidth = new Uint8Array([0x1D, 0x21, 0x10]) // 水平 2 倍，垂直 1 倍
const doubleHeight = new Uint8Array([0x1D, 0x21, 0x01]) // 水平 1 倍，垂直 2 倍
const doubleBoth = new Uint8Array([0x1D, 0x21, 0x11])   // 水平 2 倍，垂直 2 倍
const tripleBoth = new Uint8Array([0x1D, 0x21, 0x22])   // 水平 3 倍，垂直 3 倍
```

#### 3. 打印和走纸

| 命令 | 字节序列 | 说明 |
|------|---------|------|
| `LF` | `0A` | 打印并走纸一行 |
| `ESC d n` | `1B 64 n` | 打印并走纸 n 行（n = 0-255） |
| `ESC J n` | `1B 4A n` | 打印并走纸 n 个点行（n = 0-255），不回车换行 |

```js
// 走纸 3 行
const feedLines = new Uint8Array([0x1B, 0x64, 0x03])
// 走纸 30 个点行（用于精确间距控制）
const feedDots = new Uint8Array([0x1B, 0x4A, 0x1E])
```

#### 4. 条码打印

打印条码前需要先设置条码参数（宽度、高度、HRI 文字位置），然后发送条码数据。

##### 4.1 条码参数设置

| 命令 | 字节序列 | 说明 |
|------|---------|------|
| `GS w n` | `1D 77 n` | 设置条码宽度（n = 2-6，单位：模块宽度） |
| `GS h n` | `1D 68 n` | 设置条码高度（n = 1-255，单位：点） |
| `GS H n` | `1D 48 n` | 设置 HRI 文字位置：0=不打印，1=上方，2=下方，3=上下都打印 |

##### 4.2 条码类型（m 值对应表）

| m 值 | 条码类型 | 数据长度限制 |
|------|---------|-------------|
| 65 (0x41) | UPC-A | 11-12 位数字 |
| 66 (0x42) | UPC-E | 11-12 位数字 |
| 67 (0x43) | EAN-13 | 12-13 位数字 |
| 68 (0x44) | EAN-8 | 7-8 位数字 |
| 69 (0x45) | CODE39 | 1 位以上 |
| 70 (0x46) | ITF | 偶数位数字 |
| 71 (0x47) | CODABAR | 1 位以上 |
| 73 (0x49) | CODE128 | 1 位以上 |

##### 4.3 打印条码 — `GS k`

**Format A**（NUL 终止，适用于早期打印机）：

```
1D 6B m d1 d2 ... dk 00
```

**Format B**（长度指定，推荐）：

```
1D 6B m n d1 d2 ... dn
```

其中 `n` 是数据字节数。

```js
// 示例：打印 EAN-13 条码 "6901234567892"
// 使用 Format B
function printBarcode(data) {
  const barcodeData = encode(data) // 将字符串转为字节数组
  const cmd = [
    0x1D, 0x77, 0x02,   // GS w 2 — 条码宽度 2
    0x1D, 0x68, 0x64,   // GS h 100 — 条码高度 100 点
    0x1D, 0x48, 0x02,   // GS H 2 — HRI 文字在下方
    0x1D, 0x6B, 0x43,   // GS k 67 — EAN-13
    barcodeData.length,  // 数据长度
    ...barcodeData       // 条码数据
  ]
  return new Uint8Array(cmd)
}
```

#### 5. 二维码打印 — `GS ( k`

打印二维码需要按顺序执行 5 个步骤。每个步骤都是一条独立的 `GS ( k` 命令。

##### 步骤 1：选择 QR 模型

```
1D 28 6B 04 00 31 41 n1 00
```

| n1 值 | 模型 |
|-------|------|
| 49 (0x31) | QR Code 模型 1 |
| 50 (0x32) | QR Code 模型 2（**推荐**，最常用） |
| 51 (0x33) | Micro QR Code |

##### 步骤 2：设置模块大小

```
1D 28 6B 03 00 31 43 n
```

`n` 的取值范围：1-16（默认通常为 3 或 4）。值越大，二维码越大。

##### 步骤 3：设置纠错级别

```
1D 28 6B 03 00 31 45 n
```

| n 值 | 纠错级别 | 纠错能力 | 适用场景 |
|------|---------|---------|---------|
| 48 (0x30) | L | 7% | 环境干净，追求小码 |
| 49 (0x31) | M | 15% | **一般推荐** |
| 50 (0x32) | Q | 25% | 有一定遮挡风险 |
| 51 (0x33) | H | 30% | 工业环境，可靠性优先 |

##### 步骤 4：存储 QR 数据

```
1D 28 6B pL pH 31 50 30 d1 d2 ... dk
```

其中：
- `pL + pH × 256` = 数据长度 + 3（即 `k + 3`）
- `d1...dk` 是实际的二维码内容数据

##### 步骤 5：打印 QR 码

```
1D 28 6B 03 00 31 51 30
```

##### 完整示例

```js
/**
 * 生成打印二维码的 ESC/POS 命令
 * @param {string} text - 二维码内容
 * @param {number} size - 模块大小（1-16，默认 6）
 * @returns {Uint8Array} 完整的命令字节序列
 */
function generateQrCodeCommands(text, size = 6) {
  const textBytes = []
  // 将字符串编码为字节
  for (let i = 0; i < text.length; i++) {
    textBytes.push(text.charCodeAt(i))
  }

  const storeLen = textBytes.length + 3
  const pL = storeLen % 256
  const pH = Math.floor(storeLen / 256)

  const cmd = [
    // 步骤 1：选择 QR 模型 2
    0x1D, 0x28, 0x6B, 0x04, 0x00, 0x31, 0x41, 0x32, 0x00,
    // 步骤 2：设置模块大小
    0x1D, 0x28, 0x6B, 0x03, 0x00, 0x31, 0x43, size,
    // 步骤 3：设置纠错级别 M (15%)
    0x1D, 0x28, 0x6B, 0x03, 0x00, 0x31, 0x45, 0x31,
    // 步骤 4：存储 QR 数据
    0x1D, 0x28, 0x6B, pL, pH, 0x31, 0x50, 0x30,
    ...textBytes,
    // 步骤 5：打印 QR 码
    0x1D, 0x28, 0x6B, 0x03, 0x00, 0x31, 0x51, 0x30,
  ]

  return new Uint8Array(cmd)
}

// 使用示例：打印一个包含网址的二维码
const qrCmd = generateQrCodeCommands('https://example.com', 6)
```

#### 6. 图片/光栅位图打印

打印图片有两种命令：`GS v 0`（光栅位图）和 `ESC *`（点阵位图）。推荐使用 `GS v 0`，它更适合打印完整图片。

##### 6.1 光栅位图 — `GS v 0`

```
1D 76 30 m xL xH yL yH d1 d2 ... dk
```

| 字段 | 说明 |
|------|------|
| `m` | 打印模式：0=正常，1=双倍宽，2=双倍高，3=双倍宽高 |
| `xL + xH × 256` | 水平方向字节数（每个字节代表 8 个水平像素点） |
| `yL + yH × 256` | 垂直方向点数（像素行数） |
| `d1...dk` | 位图数据（1=打印黑色，0=白色），总字节数 = xL+xH×256 × yL+yH×256 |

对于 58mm 打印机（纸宽 384 点），每行 384 / 8 = 48 字节。
对于 80mm 打印机（纸宽 576 点），每行 576 / 8 = 72 字节。

```js
/**
 * 将图片数据转换为 GS v 0 命令
 * @param {number[][]} imageData - 二维数组，0=白色 1=黑色，每行宽度必须是 8 的倍数
 * @param {number} width - 图片宽度（像素）
 */
function generateRasterImageCommand(imageData, width) {
  const bytesPerRow = Math.ceil(width / 8)  // 每行字节数
  const height = imageData.length            // 总行数（垂直像素数）

  const xL = bytesPerRow % 256
  const xH = Math.floor(bytesPerRow / 256)
  const yL = height % 256
  const yH = Math.floor(height / 256)

  const header = [0x1D, 0x76, 0x30, 0x00, xL, xH, yL, yH]
  const data = []

  for (let row = 0; row < height; row++) {
    for (let col = 0; col < bytesPerRow; col++) {
      let byte = 0
      for (let bit = 0; bit < 8; bit++) {
        const pixelIndex = col * 8 + bit
        if (pixelIndex < width && imageData[row][pixelIndex] === 1) {
          byte |= (0x80 >> bit) // MSB 在前，最高位对应最左边的像素
        }
      }
      data.push(byte)
    }
  }

  return new Uint8Array([...header, ...data])
}
```

##### 6.2 点阵位图 — `ESC *`

```
1B 2A m nL nH d1 d2 ... dk
```

| m 值 | 模式 | 水平点数 | 纵向点数 |
|------|------|---------|---------|
| 0 | 8 点单密度 | nL+nH×256 | 8 |
| 1 | 8 点双密度 | nL+nH×256 | 8 |
| 32 (0x20) | 24 点单密度 | nL+nH×256 | 24 |
| 33 (0x21) | 24 点双密度 | nL+nH×256 | 24 |

```js
// 示例：使用 8 点单密度模式打印一行 48 字节（384 点）的图片数据
const m = 0x00         // 8 点单密度
const nL = 48 % 256    // 低字节
const nH = Math.floor(48 / 256) // 高字节
const imageData = [/* 48 字节的图片数据 */]

const cmd = new Uint8Array([0x1B, 0x2A, m, nL, nH, ...imageData, 0x0A])
```

#### 8. 字符编码

不同语言的内容需要选择正确的字符编码表，否则会出现乱码。

| 命令 | 字节序列 | 说明 |
|------|---------|------|
| `ESC t n` | `1B 74 n` | 选择字符代码表 |

常用代码表编号：

| n 值 | 代码表 | 适用语言 |
|------|-------|---------|

| 3 | CP860 | 葡萄牙语 |
| 16 | WPC1252 | 西欧语言 |
| 255 | UTF-8 | Unicode（**部分打印机支持**，推荐先测试） |

```js
// 尝试设置 UTF-8 编码（需打印机支持）
const setUtf8 = new Uint8Array([0x1B, 0x74, 0xFF])
```

> ⚠️ **注意：** 中文字符的打印需要打印机支持中文字库。部分打印机通过 `FS &` 命令切换到中文模式。如果打印机没有中文字库，中文内容需要作为图片打印。不同品牌打印机的中文支持方案有差异，请参考具体打印机的文档。

### 四、微信小程序蓝牙打印

微信小程序提供了完整的 BLE（低功耗蓝牙）API，可以用于连接和控制蓝牙热敏打印机。

#### 1. 完整工作流程

```
初始化蓝牙 → 搜索设备 → 连接设备 → 获取服务 → 获取特征值 → 写入数据（打印） → 断开连接
```

#### 2. API 详解

##### 2.1 初始化蓝牙模块 — `wx.openBluetoothAdapter`

在使用任何蓝牙 API 之前，必须先调用此方法初始化蓝牙模块。

```js
wx.openBluetoothAdapter({
  success(res) {
    console.log('蓝牙初始化成功', res)
  },
  fail(err) {
    console.error('蓝牙初始化失败', err)
    // err.errCode:
    //   10001 — 当前蓝牙适配器不可用（蓝牙未开启）
    //   10000 — 未初始化
  }
})
```

##### 2.2 开始搜索蓝牙设备 — `wx.startBluetoothDevicesDiscovery`


```js
// ===== 工具函数 =====

// 字符串转 ArrayBuffer
function str2ab(str) {
  const buffer = new ArrayBuffer(str.length)
  const view = new Uint8Array(buffer)
  for (let i = 0; i < str.length; i++) {
    view[i] = str.charCodeAt(i)
  }
  return buffer
}

// 合并多个 ArrayBuffer
function concatArrayBuffers(...buffers) {
  const totalLength = buffers.reduce((sum, buf) => sum + buf.byteLength, 0)
  const result = new Uint8Array(totalLength)
  let offset = 0
  for (const buf of buffers) {
    result.set(new Uint8Array(buf), offset)
    offset += buf.byteLength
  }
  return result.buffer
}

// Uint8Array 转 ArrayBuffer
function u8toab(u8) {
  return u8.buffer
}

// ===== 打印机命令 =====

const ESC = 0x1B
const GS = 0x1D

function initPrinter() {
  return u8toab(new Uint8Array([ESC, 0x40]))
}

function printText(text) {
  return str2ab(text + '\n')
}

function setAlign(mode) { // 0=左 1=中 2=右
  return u8toab(new Uint8Array([ESC, 0x61, mode]))
}

function setBold(on) {
  return u8toab(new Uint8Array([ESC, 0x45, on ? 0x01 : 0x00]))
}

function feedLines(n) {
  return u8toab(new Uint8Array([ESC, 0x64, n]))
}

function cutPaper() {
  return u8toab(new Uint8Array([GS, 0x56, 0x00]))
}

// ===== 蓝牙连接与打印 =====

const SERVICE_UUID = '0000FF00-0000-1000-8000-00805F9B34FB'
const WRITE_CHAR_UUID = '0000FF02-0000-1000-8000-00805F9B34FB'
let deviceId = ''
let writeCharId = WRITE_CHAR_UUID

// 分包写入（BLE 单次写入有限制）
function writeData(buffer, chunkSize = 20) {
  const data = new Uint8Array(buffer)
  const totalPackets = Math.ceil(data.length / chunkSize)

  return new Promise((resolve, reject) => {
    let offset = 0

    function writeNext() {
      if (offset >= data.length) {
        resolve()
        return
      }

      const end = Math.min(offset + chunkSize, data.length)
      const chunk = data.slice(offset, end)
      const chunkBuffer = new ArrayBuffer(chunk.length)
      new Uint8Array(chunkBuffer).set(chunk)

      wx.writeBLECharacteristicValue({
        deviceId,
        serviceId: SERVICE_UUID,
        characteristicId: writeCharId,
        value: chunkBuffer,
        success() {
          offset = end
          writeNext()
        },
        fail(err) {
          reject(err)
        }
      })
    }

    writeNext()
  })
}

// 完整打印流程
async function printReceipt() {
  try {
    // 1. 初始化蓝牙
    await new Promise((resolve, reject) => {
      wx.openBluetoothAdapter({ success: resolve, fail: reject })
    })

    // 2. 搜索设备（实际项目中应让用户选择设备）
    await new Promise((resolve, reject) => {
      wx.startBluetoothDevicesDiscovery({ success: resolve, fail: reject })
    })

    // 3. 等待发现目标设备（这里简化处理）
    const targetDevice = await new Promise((resolve) => {
      wx.onBluetoothDeviceFound(res => {
        const found = res.devices.find(d => d.name.includes('Printer'))
        if (found) {
          deviceId = found.deviceId
          resolve(found)
        }
      })
    })

    // 4. 停止搜索
    await new Promise((resolve) => {
      wx.stopBluetoothDevicesDiscovery({ success: resolve })
    })

    // 5. 连接设备
    await new Promise((resolve, reject) => {
      wx.createBLEConnection({
        deviceId,
        success: resolve,
        fail: reject
      })
    })

    // 6. 获取服务和特征值（如果 UUID 已知可跳过发现步骤）
    // 实际项目中建议先 getBLEDeviceServices → getBLEDeviceCharacteristics 确认可写入的特征值

    // 7. 构造打印数据
    const printData = concatArrayBuffers(
      initPrinter(),                     // 初始化
      setAlign(1),                       // 居中
      setBold(true),                     // 加粗
      printText('MY STORE'),             // 店名
      setBold(false),
      printText('================'),
      setAlign(0),                       // 左对齐
      printText('Item 1        $10.00'),
      printText('Item 2        $20.00'),
      printText('----------------'),
      setBold(true),
      printText('Total         $30.00'),
      setBold(false),
      feedLines(3),                      // 走纸 3 行
      cutPaper()                         // 切纸
    )

    // 8. 发送打印数据
    await writeData(printData)

    console.log('打印完成')

    // 9. 断开连接
    await new Promise((resolve) => {
      wx.closeBLEConnection({ deviceId, success: resolve })
    })
  } catch (err) {
    console.error('打印失败：', err)
  } finally {
    // 10. 关闭蓝牙模块
    wx.closeBluetoothAdapter({})
  }
}
```

### 五、uni-app 蓝牙打印

uni-app 的蓝牙 API 与微信小程序高度一致，只需将 `wx.` 替换为 `uni.` 即可。这意味着你可以用几乎相同的代码实现跨平台（iOS、Android、微信小程序、H5）的蓝牙打印功能。

#### 1. API 对照表

| 微信小程序 | uni-app | 说明 |
|-----------|---------|------|
| `wx.openBluetoothAdapter` | `uni.openBluetoothAdapter` | 初始化蓝牙模块 |
| `wx.startBluetoothDevicesDiscovery` | `uni.startBluetoothDevicesDiscovery` | 开始搜索 |
| `wx.onBluetoothDeviceFound` | `uni.onBluetoothDeviceFound` | 监听发现设备 |
| `wx.stopBluetoothDevicesDiscovery` | `uni.stopBluetoothDevicesDiscovery` | 停止搜索 |
| `wx.createBLEConnection` | `uni.createBLEConnection` | 连接 BLE 设备 |
| `wx.getBLEDeviceServices` | `uni.getBLEDeviceServices` | 获取服务列表 |
| `wx.getBLEDeviceCharacteristics` | `uni.getBLEDeviceCharacteristics` | 获取特征值列表 |
| `wx.writeBLECharacteristicValue` | `uni.writeBLECharacteristicValue` | 写入数据 |
| `wx.notifyBLECharacteristicValueChange` | `uni.notifyBLECharacteristicValueChange` | 订阅通知 |
| `uni.onBLECharacteristicValueChange` | `uni.onBLECharacteristicValueChange` | 监听数据变化 |
| `wx.closeBLEConnection` | `uni.closeBLEConnection` | 断开连接 |
| `wx.closeBluetoothAdapter` | `uni.closeBluetoothAdapter` | 关闭蓝牙模块 |
| `wx.setBLEMTU` | `uni.setBLEMTU` | 设置 MTU（仅 Android） |

#### 2. 跨平台注意事项

| 平台 | 注意事项 |
|------|---------|
| Android | 支持 BLE 和经典蓝牙 SPP；`uni.setBLEMTU` 可协商更大的 MTU |
| iOS | 仅支持 BLE；重复调用 `createBLEConnection` 可能导致连接异常 |
| 微信小程序 | 仅支持 BLE；需要在 `app.json` 中声明蓝牙权限 |

> ⚠️ **注意：** uni-app 中 Android 端多次调用 `createBLEConnection` 可能导致系统持有同一设备多个连接实例，调用 `closeBLEConnection` 时不能真正断开。建议在全局维护连接状态，确保成对调用 `createBLEConnection` 和 `closeBLEConnection`。

#### 3. uni-app 打印示例（Promise 封装）

```js
// bluetooth.js — 蓝牙工具模块

const SERVICE_UUID = '0000FF00-0000-1000-8000-00805F9B34FB'
const WRITE_CHAR_UUID = '0000FF02-0000-1000-8000-00805F9B34FB'

// Promise 化 uni API
function promisify(fn) {
  return (options = {}) =>
    new Promise((resolve, reject) => {
      fn({
        ...options,
        success: resolve,
        fail: reject
      })
    })
}

// 初始化并搜索设备
export async function connectPrinter() {
  const openAdapter = promisify(uni.openBluetoothAdapter)
  const startDiscovery = promisify(uni.startBluetoothDevicesDiscovery)

  await openAdapter()

  // 搜索指定 service 的设备
  await startDiscovery({
    services: [SERVICE_UUID.substring(0, 8).replace(/^0000/, '')],
    allowDuplicatesKey: false
  })

  // 等待发现目标设备
  const device = await new Promise((resolve, reject) => {
    const timer = setTimeout(() => {
      reject(new Error('搜索超时'))
    }, 10000)

    uni.onBluetoothDeviceFound(res => {
      const found = res.devices.find(d => d.name && d.name.includes('Printer'))
      if (found) {
        clearTimeout(timer)
        resolve(found)
      }
    })
  })

  // 停止搜索
  await promisify(uni.stopBluetoothDevicesDiscovery)()

  // 连接设备
  await promisify(uni.createBLEConnection)({ deviceId: device.deviceId })

  return device
}

// 分包写入打印数据
export async function sendData(deviceId, data, chunkSize = 20) {
  const write = promisify(uni.writeBLECharacteristicValue)
  const bytes = new Uint8Array(data)

  for (let i = 0; i < bytes.length; i += chunkSize) {
    const end = Math.min(i + chunkSize, bytes.length)
    const chunk = new Uint8Array(end - i)
    chunk.set(bytes.slice(i, end))

    await write({
      deviceId,
      serviceId: SERVICE_UUID,
      characteristicId: WRITE_CHAR_UUID,
      value: chunk.buffer
    })
  }
}

// 断开连接
export async function disconnectPrinter(deviceId) {
  await promisify(uni.closeBLEConnection)({ deviceId })
  await promisify(uni.closeBluetoothAdapter)()
}
```

### 六、Web Bluetooth API

Web Bluetooth API 允许网页直接与 BLE 设备通信，适用于 Chrome 浏览器环境（如 POS 系统的 Web 端、PWA 应用等）。

> ⚠️ **注意：** Web Bluetooth API 仅在**安全上下文**（HTTPS 或 localhost）中可用，且需要用户主动触发（如点击按钮）。

#### 1. 完整连接与打印流程

```js

      { services: [PRINTER_SERVICE_UUID] },

    ],

  return device

// 步骤 2：连接 GATT 服务器
async function connectDevice() {

  console.log('GATT 服务器已连接')
  return server
}

// 步骤 3：获取服务和特征值
async function getWriteCharacteristic() {
  const service = await server.getPrimaryService(PRINTER_SERVICE_UUID)
  writeCharacteristic = await service.getCharacteristic(WRITE_CHAR_UUID)
  console.log('写入特征值已获取')
  return writeCharacteristic
}

// 步骤 4：写入数据（打印）
async function printData(uint8Array) {
  if (!writeCharacteristic) {
    throw new Error('未连接到打印机')
  }

  // 方式一：写入并等待响应（可靠）
  await writeCharacteristic.writeValueWithResponse(uint8Array)

  // 方式二：写入不等待响应（更快，但不保证送达）
  // await writeCharacteristic.writeValueWithoutResponse(uint8Array)
}

// 步骤 5：断开连接
function disconnect() {
  if (device && device.gatt.connected) {
    device.gatt.disconnect()
  }
```

#### 3. 完整 Web 打印示例

```js
class WebBluetoothPrinter {
  constructor(serviceUUID, writeCharUUID) {
    this.serviceUUID = serviceUUID
    this.writeCharUUID = writeCharUUID
    this.device = null
    this.characteristic = null
  }

  async connect() {
    // 请求用户选择设备
    this.device = await navigator.bluetooth.requestDevice({
      filters: [{ services: [this.serviceUUID] }]
    })

    // 监听断连
    this.device.addEventListener('gattserverdisconnected', () => {
      console.log('打印机已断开')
      this.characteristic = null
    })

    // 连接 GATT
    const server = await this.device.gatt.connect()
    const service = await server.getPrimaryService(this.serviceUUID)
    this.characteristic = await service.getCharacteristic(this.writeCharUUID)
  }

  async write(data) {
    if (!this.characteristic) throw new Error('未连接')

    // BLE 单次写入有长度限制，需要分包
    const MTU = 20  // 默认 MTU
    for (let i = 0; i < data.length; i += MTU) {
      const chunk = data.slice(i, i + MTU)
      await this.characteristic.writeValueWithoutResponse(new Uint8Array(chunk))
      // 小延时，避免数据丢失
      await new Promise(r => setTimeout(r, 10))
    }
  }

  disconnect() {
    if (this.device?.gatt?.connected) {
      this.device.gatt.disconnect()
    }
  }

  // 便捷方法：打印文本
  async printText(text) {
    const encoder = new TextEncoder()
    const data = [...encoder.encode(text), 0x0A]  // 加换行
    await this.write(data)
  }
}

// 使用示例（需要用户点击触发）
document.getElementById('print-btn').addEventListener('click', async () => {
  const printer = new WebBluetoothPrinter(
    '0000ff00-0000-1000-8000-00805f9b34fb',
    '0000ff02-0000-1000-8000-00805f9b34fb'
  )

  await printer.connect()

  // 初始化
  await printer.write([0x1B, 0x40])
  // 打印内容
  await printer.printText('Hello World!')
  // 断开
  printer.disconnect()
})
```

### 七、常见蓝牙打印机 UUID

蓝牙打印机没有统一的 UUID 标准，不同厂商使用不同的 UUID。以下是最常见的几种。

#### 1. 经典蓝牙 SPP UUID

| UUID | 说明 |
|------|------|
| `00001101-0000-1000-8000-00805F9B34FB` | SPP（串口协议）标准 UUID |

> 适用于经典蓝牙连接方式。Android 可通过 `BluetoothSocket` 使用此 UUID 建立 RFCOMM 连接。

#### 2. BLE 打印机常见 Service UUID

| Service UUID | 常见品牌/型号 |
|-------------|-------------|
| `0000FF00-0000-1000-8000-00805F9B34FB` | 佳博（Gainscha）、芯烨（Xprinter）、汉印（HOIN）等国产打印机（最常见） |
| `E7810A71-73AE-499D-8C15-FAA9AEF0C3F2` | Goojprt PT-210、NIIMBOT 标签打印机系列 |
| `000018F0-0000-1000-8000-00805F9B34FB` | 部分打印机广播此 Service UUID |

#### 3. BLE 打印机常见特征值 UUID

| 特征值 UUID | 类型 | 说明 |
|------------|------|------|
| `0000FF02-0000-1000-8000-00805F9B34FB` | Write | **最常用的写入特征值**，用于发送 ESC/POS 打印数据 |
| `0000FF01-0000-1000-8000-00805F9B34FB` | Notify / Read | 通常用于读取打印机状态（如缺纸、电量等） |

#### 4. 如何确定你的打印机的 UUID

```js
// 通用的 UUID 发现方法（适用于微信小程序 / uni-app）
async function discoverPrinterUUIDs(deviceId) {
  // 获取所有服务
  const servicesRes = await new Promise((resolve, reject) => {
    wx.getBLEDeviceServices({
      deviceId,
      success: resolve,
      fail: reject
    })
  })

  for (const service of servicesRes.services) {
    console.log('Service UUID:', service.uuid)

    // 获取每个服务下的特征值
    const charsRes = await new Promise((resolve, reject) => {
      wx.getBLEDeviceCharacteristics({
        deviceId,
        serviceId: service.uuid,
        success: resolve,
        fail: reject
      })
    })

    for (const char of charsRes.characteristics) {
      console.log('  Characteristic UUID:', char.uuid)
      console.log('    Properties:', {
        read: char.properties.read,
        write: char.properties.write,
        notify: char.properties.notify,
        indicate: char.properties.indicate
      })
    }
  }
}
```

> ⚠️ **注意：** 不同厂商甚至不同型号的打印机 UUID 可能不同。在实际项目中，建议先通过上述方法发现你的打印机的实际 UUID，或者查阅打印机的技术文档。

### 八、常见问题与注意事项

#### 1. BLE 数据分包

BLE 默认 MTU 为 20 字节（可用数据），而打印数据通常远大于 20 字节。分包策略至关重要：

```js
/**
 * 通用的分包写入函数
 * @param {Object} writeApi - 平台写入 API（wx/uni/Web Bluetooth）
 * @param {Uint8Array} data - 要发送的数据
 * @param {number} chunkSize - 每包大小（默认 20）
 * @param {number} delay - 包间延时（毫秒，默认 20）
 */
async function sendInChunks(writeFn, data, chunkSize = 20, delay = 20) {
  for (let i = 0; i < data.length; i += chunkSize) {
    const chunk = data.slice(i, Math.min(i + chunkSize, data.length))
    await writeFn(chunk)
    if (delay > 0) {
      await new Promise(r => setTimeout(r, delay))
    }
  }
}
```

> 💡 **提示：** Android 上可以通过 `wx.setBLEMTU` / `uni.setBLEMTU` 协商更大的 MTU（最高 512 字节），显著减少分包次数，提高打印速度。iOS 端 MTU 由系统自动协商。

#### 2. 连接稳定性

- 蓝牙连接随时可能断开，务必监听 `onBLEConnectionStateChange` / `gattserverdisconnected` 事件
- 建议实现自动重连机制，最多重试 3 次
- 搜索设备后及时停止搜索（`stopBluetoothDevicesDiscovery`），否则会持续消耗电量和影响连接稳定性

#### 3. 中文打印问题

- 热敏打印机通常使用内置字库打印中文，需要打印机支持中文字库
- 如果打印出来的中文是乱码，可能原因：
  - 字符编码不匹配（尝试 `ESC t n` 切换编码表）
  - 打印机不支持中文字库（需要将中文渲染为图片后用位图命令打印）
  - 数据在分包时被截断（确保 UTF-8 中文字符（3 字节）不被分包切断）

#### 4. iOS 和 Android 差异

| 差异点 | Android | iOS |
|-------|---------|-----|
| `deviceId` 格式 | MAC 地址（如 `AA:BB:CC:DD:EE:FF`） | UUID（如 `XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX`） |
| MTU 协商 | 可手动设置（`setBLEMTU`） | 系统自动协商 |
| SPP 支持 | 支持 | 不支持 |
| 后台蓝牙 | 支持但有限制 | 需要特殊权限 |
| 配对 | 可通过代码触发 | 系统弹窗处理 |

### 九、总结

| 知识点 | 关键要点 |
|-------|---------|
| 蓝牙协议 | 优先使用 BLE（GATT），兼容 iOS 和 Android |
| ESC/POS 命令 | 所有命令以 `ESC`（0x1B）或 `GS`（0x1D）开头，数据以字节流发送 |
| 文本格式化 | `ESC E` 加粗、`ESC a` 对齐、`GS !` 放大 |
| 条码打印 | 先设参数（`GS w/h/H`），再用 `GS k` 发送数据 |
| 二维码打印 | 按 5 步顺序执行 `GS ( k` 命令（选模型 → 设大小 → 设纠错 → 存数据 → 打印） |
| 图片打印 | 推荐用 `GS v 0` 光栅位图命令，将图片转为单色位图数据 |
| 微信小程序 | 完整 BLE API 支持，注意分包写入（默认 MTU 20 字节） |
| uni-app | API 与微信小程序一致（`uni.` 替代 `wx.`），可跨平台 |
| Web Bluetooth | 仅 Chrome 支持，需要 HTTPS 和用户手势触发 |
| UUID | 因厂商而异，最常见的写入特征值 UUID 为 `0000FF02` |
