# Module Federation 模块联邦

## 一、什么是模块联邦

### 1.1 基本定义

**模块联邦（Module Federation）** 是 Webpack 5 引入的一项革命性功能，允许多个独立的构建（应用）在**运行时**动态共享代码依赖，而不需要通过 npm 发布或集中构建。

简单来说，它让 JavaScript 应用能够**动态加载**另一个应用的代码，就像加载本地模块一样简单。

### 1.2 核心概念

```
┌─────────────────────────────────────────────────────────────────┐
│                    传统代码共享方式                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  方式一：npm 包发布                                              │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │ 1. 开发组件库                                            │     │
│  │ 2. 执行 npm publish 发布到 npm                            │     │
│  │ 3. 消费方执行 npm install 安装                            │     │
│  │ 4. 重新构建消费者应用                                     │     │
│  │ 5. 部署                                                   │     │
│  │                                                         │     │
│  ❌ 更新组件库需要重新发布、重新安装、重新构建                 │     │
│  └─────────────────────────────────────────────────────────┘     │
│                                                                 │
│  方式二：Monorepo 共享构建                                      │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │ 1. 所有代码在同一个仓库                                    │     │
│  │ 2. 构建时打包所有代码                                      │     │
│  │ 3. 部署单体应用                                           │     │
│  │                                                         │     │
│  ❌ 应用耦合，无法独立部署                                    │     │
│  └─────────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    模块联邦方式                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │ App A（提供者）          App B（消费者）                   │     │
│  │                         │                                │     │
│  │  ┌─────────────────┐   │                                │     │
│  │  │  Shared Module  │───│── 运行时动态加载                 │     │
│  │  └─────────────────┘   │                                │     │
│  │                         │                                │     │
│  │  独立构建、独立部署        │  独立构建、独立部署             │     │
│  └─────────────────────────────────────────────────────────┘     │
│                                                                 │
│  ✅ 无需 npm 发布                                              │
│  ✅ 运行时动态加载                                              │
│  ✅ 各应用独立开发、独立部署                                    │
│  ✅ 代码实时同步                                                │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 核心术语

```
┌─────────────────────────────────────────────────────────────────┐
│  模块联邦核心术语                                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Host（宿主）                                                   │
│  • 消费其他应用模块的应用                                        │
│  • 可以同时是 Host 和 Remote                                     │
│                                                                 │
│  Remote（远程）                                                 │
│  • 提供模块给其他应用使用的应用                                  │
│  • 可以同时是 Remote 和 Host                                    │
│                                                                 │
│  Exposes（暴露）                                                │
│  • Remote 声明要暴露给其他应用的模块                            │
│  • 指定模块的入口文件                                          │
│                                                                 │
│  Remotes（远程引用）                                            │
│  • Host 声明要使用的 Remote 应用                                │
│  • 指定 Remote 的访问地址                                      │
│                                                                 │
│  Shared（共享依赖）                                             │
│  • 多个应用共享的依赖库                                         │
│  • 避免重复加载，减少体积                                       │
└─────────────────────────────────────────────────────────────────┘
```

### 1.4 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    模块联邦工作流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 构建时阶段                                                  │
│     ┌───────────────────────────────────────────────────────┐   │
│     │ Remote App 构建时：                                   │   │
│     │ • 分析暴露的模块                                       │   │
│     │ • 生成远程入口文件（remote entry）                    │   │
│     │ • 将模块代码分离成独立的 chunk                         │   │
│     │ • 生成 manifest 描述文件                               │   │
│     └───────────────────────────────────────────────────────┘   │
│                                                                 │
│  2. 运行时阶段                                                  │
│     ┌───────────────────────────────────────────────────────┐   │
│     │ Host App 运行时：                                      │   │
│     │ 1. 加载 Remote App 的 remote entry                    │   │
│     │ 2. 解析 manifest，获取可用模块列表                      │   │
│     │ 3. 按需加载需要的模块 chunk                            │   │
│     │ 4. 执行模块代码                                        │   │
│     └───────────────────────────────────────────────────────┘   │
│                                                                 │
│  3. 依赖共享                                                    │
│     ┌───────────────────────────────────────────────────────┐   │
│     │ • 如果 Host 和 Remote 有相同的依赖（如 React）         │   │
│     │ • 只加载一次，共享使用                                  │   │
│     │ • 通过 shared 配置实现                                  │   │
│     └───────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## 二、模块联邦的作用/优势

### 2.1 独立开发与部署

```
│  └── 发布周期长，风险高                                         │
```

## 三、使用场景

### 3.1 ✅ 适合使用模块联邦的场景

| 场景 | 说明 | 典型案例 |
|------|------|----------|
| **微前端架构** | 多个子应用组合成一个主应用 | 电商前台 + 管理后台 |
| **组件库开发** | 组件库与应用分离开发，实时预览 | Ant Design、Material-UI |
| **系统渐进升级** | 老系统逐步迁移到新技术栈 | React 15 → React 18 |
| **多团队协作** | 不同团队独立开发、独立部署 | 跨部门项目 |
| **插件化架构** | 主应用 + 动态插件系统 | CMS 系统、SaaS 平台 |
| **SSR 架构** | 同构渲染应用中分离客户端/服务端代码 | Next.js 扩展 |
| **多页面应用** | 共享组件和逻辑的多个独立页面 | 营销活动页面 |

### 3.2 ❌ 不适合使用模块联邦的场景

| 场景 | 原因 |
|------|------|
| **小型项目** | 增加复杂度，收益不大 |
| **简单应用** | 不需要动态加载 |
| **首屏性能极度敏感** | 额外的网络请求可能影响性能 |
| **团队不熟悉 Webpack** | 学习成本较高 |
| **不支持 Webpack 5** | 需要升级构建工具 |

### 4.2 Webpack 5 配置详解

```javascript
// webpack.config.js（Remote 应用配置）
const ModuleFederationPlugin = requirewebpack/lib/container/ModuleFederationPlugin);

module.exports = {
  entry: './src/index',
  mode: 'development',
  devServer: {
    port: 3001
  },
  plugins: [
    new ModuleFederationPlugin({
      // 1. 当前应用名称（必须唯一）
      name: 'remote_app',

      // 2. 声明要暴露的模块
      exposes: {
        './Button': './src/components/Button',
        './Header': './src/components/Header',
        // 或者暴露整个模块
        './utils': './src/utils'
      },

      // 3. 声明要使用的远程模块
      remotes: {
        // 如果当前应用也使用其他远程模块
        // app1: 'app1@http://localhost:3002/remoteEntry.js'
      },

      // 4. 共享的依赖
      shared: {
        react: {
          singleton: true,      // 只加载一个实例
          requiredVersion: '^18.0.0',
          eager: false          // 异步加载
        },
        'react-dom': {
          singleton: true,
          requiredVersion: '^18.0.0'
        }
      }
    })
  ],

  // 5. 构建优化
  optimization: {
    runtimeChunk: false  // 禁用运行时 chunk
  },

  // 6. 解决开发环境热更新问题
  resolve: {
    extensions: ['.jsx', '.js', '.json']
  }
};
```

```javascript
// webpack.config.js（Host 应用配置）
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');

module.exports = {
```

### 4.4 React 18 配置示例

```javascript
// webpack.config.js
const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
const deps = require('./package.json').dependencies;

module.exports = {
  entry: './src/index',

  plugins: [
    new ModuleFederationPlugin({
      name: 'remote_react',
      filename: 'remoteEntry.js',

      exposes: {
        './Button': './src/components/Button',
        './useAuth': './src/hooks/useAuth'
      },

      shared: {
        ...deps,
        react: {
          singleton: true,
          requiredVersion: deps.react
        },
        'react-dom': {
          singleton: true,
          requiredVersion: deps['react-dom']
        }
      }
    }),

    new HtmlWebpackPlugin({
      template: './public/index.html'
    })
  ],

  // 解决 React 热更新问题
  resolve: {
    alias: {
      react: require.resolve('react')
    }
  }
};
```

## 五、完整实现步骤

### 5.1 场景设定

我们要实现一个电商系统，包含：
- **主应用（Host）**：电商平台主站
- **远程应用（Remote）**：购物车组件

### 5.2 初始化项目

```bash
# 1. 创建项目目录
mkdir module-federation-demo
cd module-federation-demo

# 2. 初始化项目
pnpm init

# 3. 安装依赖
pnpm add -D webpack webpack-cli webpack-dev-server html-webpack-plugin
pnpm add react react-dom

# 4. 创建目录结构
mkdir -p host/src remote/src
```

### 5.3 配置 Remote 应用

```javascript
// remote/webpack.config.js
const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');

module.exports = {
  mode: 'development',

  entry: './src/index.js',

  output: {
    publicPath: 'http://localhost:3001/',  // 重要：指定远程地址
  },

  plugins: [
    new ModuleFederationPlugin({
      name: 'shopping_cart',
      filename: 'remoteEntry.js',

      // 暴露的模块
      exposes: {
        './Cart': './src/Cart',
        './CartIcon': './src/CartIcon'
      },

      // 共享依赖
      shared: {
        react: {
          singleton: true,
          requiredVersion: '^18.0.0'
        },
        'react-dom': {
          singleton: true
        }
      }
    }),

    new HtmlWebpackPlugin({
      template: './public/index.html'
    })
  ],

  devServer: {
    port: 3001
```

### 6.3 错误处理

```javascript
// src/App.js
import React, { Suspense, useState, useEffect } from 'react';

function App() {
  const [loadError, setLoadError] = useState(null);
  const [retryCount, setRetryCount] = useState(0);

  // 错误边界
  function ErrorFallback({ error, retry }) {
    return (
      <div>
        <h2>组件加载失败</h2>
        <p>{error.message}</p>
        <button onClick={retry}>重试</button>
      </div>
    );
  }

  // 远程组件加载包装
  function RemoteComponentWrapper({ component, fallback }) {
    const [Component, setComponent] = useState(null);

    useEffect(() => {
      import(`shopping_cart/${component}`)
        .then(module => setComponent(() => module.default))
        .catch(error => {
          console.error('Failed to load remote component:', error);
          setLoadError(error);
        });
    }, [component, retryCount]);

    if (loadError) {
      return <ErrorFallback error={loadError} retry={() => setRetryCount(c => c + 1)} />;
    }

    return (
      <Suspense fallback={fallback}>
        {Component && <Component />}
      </Suspense>
    );
  }

  return (
    <RemoteComponentWrapper
      component="Cart"
      fallback={<div>购物车加载中...</div>}
    />
  );
}
```
