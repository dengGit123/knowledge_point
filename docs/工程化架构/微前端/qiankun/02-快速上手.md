# qiankun 快速上手

> 📖 官方文档：[快速上手](https://qiankun.umijs.org/zh/guide/getting-started) | [vite-plugin-qiankun](https://www.npmjs.com/package/vite-plugin-qiankun)

本文用一个最小可运行示例，带你把一个 Vue3 主应用 + 两个子应用（Vue3+Vite、Vue2+Webpack）跑起来。深度配置放在后续文档展开。

## 一、整体流程

```text
1. 主应用安装 qiankun，注册子应用并 start()
2. 子应用导出生命周期（bootstrap / mount / unmount）
3. 子应用配置打包（umd / vite-plugin-qiankun）+ CORS
4. 浏览器访问子应用路由前缀 → qiankun 自动加载并渲染子应用
```

## 二、主应用搭建（Vue3 + Vite）

### 2.1 安装 qiankun

```bash
npm i qiankun
```

### 2.2 主应用入口

```ts
// main-app/src/main.ts
import { createApp } from 'vue'
import App from './App.vue'
import { registerMicroApps, start } from 'qiankun'

createApp(App).mount('#app')

// 注册子应用
registerMicroApps([
  {
    name: 'sub-vue',                  // 子应用名，必须全局唯一，需与子应用打包名一致
    entry: '//localhost:7105',        // 子应用访问地址（HTML Entry）
    container: '#subapp-viewport',    // 子应用挂载的容器选择器
    activeRule: '/sub-vue',           // 路由前缀：访问 /sub-vue 时激活该子应用
    props: { routerBase: '/sub-vue' } // 传给子应用的数据
  },
  {
    name: 'sub-vue2',
    entry: '//localhost:7106',
    container: '#subapp-viewport',
    activeRule: '/sub-vue2',
    props: { routerBase: '/sub-vue2' }
  }
])

// 启动 qiankun
start()
```

### 2.3 主应用容器

主应用需要提供一个挂载点 `#subapp-viewport`，子应用被激活后会渲染到这里：

```vue
<!-- main-app/src/App.vue -->
<template>
  <div class="layout">
    <header class="nav">
      <button @click="go('/sub-vue')">Vue3 子应用</button>
      <button @click="go('/sub-vue2')">Vue2 子应用</button>
    </header>
    <!-- ⭐ qiankun 子应用挂载点 -->
    <main id="subapp-viewport"></main>
  </div>
</template>

<script setup lang="ts">
function go(path: string) {
  // qiankun 已劫持 history.pushState，调用后会触发 activeRule 重新匹配
  window.history.pushState({}, '', path)
}
</script>
```

> 💡 **提示：** `start()` 必须在容器 DOM 存在后调用。这里主应用 `createApp(App).mount('#app')` 同步渲染出了 `#subapp-viewport`，所以紧接着 `start()` 没问题；若容器在异步路由页面里，需在该页面 `mounted` 后再 `start()`（见 [11-常见问题](./11-常见问题与注意事项.md)）。

## 三、子应用一：Vue3 + Vite

Vite 默认输出 ESM，与 qiankun 默认的 JS 沙箱机制不兼容，需要借助 [`vite-plugin-qiankun`](https://www.npmjs.com/package/vite-plugin-qiankun) 接入。

### 3.1 安装插件

```bash
cd sub-vue
npm i vite-plugin-qiankun
```

### 3.2 配置 vite.config.ts

```ts
// sub-vue/vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import qiankun from 'vite-plugin-qiankun'

export default defineConfig({
  plugins: [
    vue(),
    qiankun('sub-vue', {       // 第一个参数是子应用名，需与主应用注册的 name 一致
      useDevMode: true         // true：关闭热更新，可作为子应用加载
    })
  ],
  server: {
    port: 7105,
    cors: true,                // ⭐ 主应用会跨域 fetch 子应用资源，必须开启
    origin: 'http://localhost:7105'  // 让资源地址带完整域名，避免相对路径 404
  }
})
```

> ⚠️ **注意 `useDevMode`**：
> - `true`：关闭 Vite 热更新，子应用可正常被主应用加载（**开发时用这个**）
> - `false`：保留热更新，但**无法作为子应用加载**，只能独立运行调试
>
> 二者不可兼得，这是 ESM 加载方式与热更新插件冲突导致的。

### 3.3 改造入口 main.ts

```ts
// sub-vue/src/main.ts
import { createApp, type App as VueApp } from 'vue'
import { createRouter, createWebHistory } from 'vue-router'
import {
  renderWithQiankun,
  qiankunWindow
} from 'vite-plugin-qiankun/dist/helper'
import App from './App.vue'
import routes from './router'

let app: VueApp | null = null

function render(props: Record<string, any> = {}) {
  const { container, routerBase } = props

  const router = createRouter({
    // 作为子应用时用主应用传入的前缀，独立运行时用根路径
    history: createWebHistory(
      qiankunWindow.__POWERED_BY_QIANKUN__ ? routerBase : '/'
    ),
    routes
  })

  app = createApp(App)
  app.use(router)
  // 子应用：挂载到主应用容器内的 #app；独立运行：挂载到 document 的 #app
  const root = container
    ? container.querySelector('#app')
    : document.getElementById('app')
  app.mount(root!)
}

// ⭐ 通过插件注册生命周期（替代手动 export）
renderWithQiankun({
  bootstrap() {
    console.log('[sub-vue] bootstrap')
  },
  mount(props) {
    render(props)
    // 接收主应用全局状态（见 06-应用间通信）
    props.onGlobalStateChange?.((state: any) => {
      console.log('[sub-vue] 收到全局状态:', state)
    }, true)
  },
  unmount() {
    app?.unmount()
    app = null
  },
  update(props) {
    console.log('[sub-vue] update', props)
  }
})

// 独立运行（不通过主应用也能单独跑起来调试）
if (!qiankunWindow.__POWERED_BY_QIANKUN__) {
  render()
}
```

> ⚠️ **注意：** Vite ESM 子应用**不运行在 qiankun 的 JS 沙箱中**。需要操作 `window` 时，务必用 `qiankunWindow.xxx` 而不是 `window.xxx`，否则可能污染其他子应用。详见 [07-JS沙箱](./07-JS沙箱.md)。

## 四、子应用二：Vue2 + Webpack

Webpack 子应用走的是标准 qiankun 接入方式（运行在 JS 沙箱内）。

### 4.1 配置打包（vue.config.js）

```js
// sub-vue2/vue.config.js
const { name } = require('./package.json')

module.exports = {
  devServer: {
    port: 7106,
    headers: {
      'Access-Control-Allow-Origin': '*'   // ⭐ 允许主应用跨域 fetch
    }
  },
  configureWebpack: {
    output: {
      library: `${name}-[name]`,            // 导出为全局变量，供主应用识别生命周期
      libraryTarget: 'umd',                 // UMD 格式
      chunkLoadingGlobal: `webpackJsonp_${name}`  // webpack5 写法；webpack4 用 jsonpFunction
    }
  }
}
```

> 💡 **提示：** `package.json` 的 `name` 字段必须与主应用注册的 `name` 一致，且全局唯一。

### 4.2 改造入口 main.js

```js
// sub-vue2/src/main.js
import Vue from 'vue'
import VueRouter from 'vue-router'
import App from './App.vue'
import routes from './router'

// ⭐ 运行时 publicPath：解决子应用动态加载资源路径 404（见 11-常见问题）
if (window.__POWERED_BY_QIANKUN__) {
  // eslint-disable-next-line no-undef
  __webpack_public_path__ = window.__INJECTED_PUBLIC_PATH_BY_QIANKUN__
}

Vue.use(VueRouter)

let instance = null
let router = null

function render(props = {}) {
  const { container, routerBase } = props

  router = new VueRouter({
    base: window.__POWERED_BY_QIANKUN__ ? routerBase : '/',
    mode: 'history',
    routes
  })

  instance = new Vue({
    router,
    render: h => h(App)
  }).$mount(container ? container.querySelector('#app') : '#app')
}

// ⭐ Webpack 子应用需手动导出生命周期
export async function bootstrap() {
  console.log('[sub-vue2] bootstrap')
}

export async function mount(props) {
  render(props)
  props.onGlobalStateChange?.((state) => {
    console.log('[sub-vue2] 收到全局状态:', state)
  }, true)
}

export async function unmount() {
  instance?.$destroy()
  instance.$el.innerHTML = ''   // 清空容器内容，避免残留
  instance = null
  router = null
}

// 独立运行
if (!window.__POWERED_BY_QIANKUN__) {
  render()
}
```

## 五、独立运行 vs 集成运行

每个子应用都应支持两种模式，方便单独开发调试：

| 模式 | 判断条件 | 行为 |
| --- | --- | --- |
| **集成运行**（被主应用加载） | `__POWERED_BY_QIANKUN__` 为 `true` | 用主应用传入的 `routerBase`、`container` |
| **独立运行**（单独 `npm run dev`） | `__POWERED_BY_QIANKUN__` 为 `false` | 路由用根路径 `/`，挂载到 `document` 的 `#app` |

```ts
// 判断当前是否在 qiankun 环境下
const isQiankun = window.__POWERED_BY_QIANKUN__   // webpack 子应用
const isQiankun2 = qiankunWindow.__POWERED_BY_QIANKUN__  // vite 子应用
```

## 六、运行验证

1. 分别启动三个应用：
   ```bash
   # 主应用
   cd main-app && npm run dev      # 默认 http://localhost:5173
   # 子应用
   cd sub-vue  && npm run dev      # http://localhost:7105
   cd sub-vue2 && npm run serve    # http://localhost:7106
   ```
2. 打开主应用 `http://localhost:5173`，点击「Vue3 子应用」按钮
3. 观察：
   - 浏览器 URL 变为 `/sub-vue`
   - `#subapp-viewport` 内出现子应用内容
   - 控制台依次打印 `bootstrap` → `mount`
4. 点击「Vue2 子应用」→ URL 变 `/sub-vue2`，旧子应用 `unmount`，新子应用加载

> ⚠️ **排查清单：** 若子应用没出来，按顺序检查：
> 1. 子应用是否已启动且能独立访问（浏览器直接打开 `http://localhost:7105`）
> 2. 子应用是否配了 CORS（控制台是否有跨域报错）
> 3. `name` 是否与主应用注册一致、全局唯一
> 4. `activeRule` 是否与实际访问 URL 前缀匹配
> 5. 控制台是否有 `You need to export the functional lifecycles` 报错（见 [11-常见问题](./11-常见问题与注意事项.md)）

## 七、总结

- 主应用三步：`registerMicroApps` 注册 → 提供容器 → `start()` 启动
- Vite 子应用用 `vite-plugin-qiankun`，通过 `renderWithQiankun` 注册生命周期
- Webpack 子应用手动 `export` 生命周期 + 配置 `library/umd`
- 每个子应用都要支持独立运行，用 `__POWERED_BY_QIANKUN__` 区分环境
- 子应用资源**必须支持跨域**
