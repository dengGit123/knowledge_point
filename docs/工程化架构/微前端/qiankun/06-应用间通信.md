# qiankun 应用间通信

> 📖 官方文档：[initGlobalState](https://qiankun.umijs.org/zh/api#initglobalstatestate)

qiankun 的通信分为：**主应用 ↔ 子应用**、**子应用 ↔ 子应用**。本文覆盖官方 `initGlobalState` 方案与几种常见的子应用间通信方案。

## 一、通信方案概览

| 方案 | 方向 | 特点 | 适用场景 |
| --- | --- | --- | --- |
| **initGlobalState** | 主 ↔ 子、子 ↔ 子 | 官方推荐，响应式广播 | 共享用户信息、主题等全局状态 |
| **props 传递** | 主 → 子 | 单向，初始化时传 | 传配置、回调函数 |
| **CustomEvent** | 子 ↔ 子 | 走真实 DOM 事件，不受沙箱影响 | 一次性事件通知 |
| **共享 EventBus** | 子 ↔ 子 | 主应用创建实例通过 props 共享 | 复杂事件通信 |
| **localStorage** | 跨标签页 | 持久化，但同步阻塞 | 持久化少量数据 |

## 二、initGlobalState（官方推荐）

qiankun 内置的全局状态方案。主应用初始化状态，子应用通过 `props` 自动获取通信方法。

### 2.1 API 说明

```ts
import { initGlobalState } from 'qiankun'

const actions = initGlobalState(state)  // 返回 MicroAppStateActions
```

`MicroAppStateActions` 三个方法：

| 方法 | 说明 |
| --- | --- |
| `onGlobalStateChange(callback, fireImmediately?)` | 监听全局状态变化；`fireImmediately=true` 立即触发一次当前状态 |
| `setGlobalState(state)` | 按一级属性设置全局状态；**微应用只能修改已存在的一级属性** |
| `offGlobalStateChange()` | 移除当前应用的状态监听（微应用 `unmount` 时默认调用） |

> ⚠️ **关键限制：** 子应用 `setGlobalState` **只能修改主应用已定义的一级属性**，不能新增一级属性。所以主应用 `initGlobalState` 时要预定义好所有一级属性。

### 2.2 主应用初始化

```ts
// main-app/src/main.ts
import { registerMicroApps, start, initGlobalState } from 'qiankun'

// 1. 初始化全局状态（预定义所有一级属性）
const actions = initGlobalState({
  user: { name: '游客', id: 0 },
  theme: 'light',
  token: ''
})

// 2. 主应用自己也能监听和修改
actions.onGlobalStateChange((state, prev) => {
  console.log('[主应用] 状态变化', state, prev)
})

// 登录后更新全局状态
function login(user) {
  actions.setGlobalState({ user, token: 'xxx' })
}

registerMicroApps([
  { name: 'sub-a', entry: '//localhost:7105', container: '#subapp', activeRule: '/sub-a' },
  { name: 'sub-b', entry: '//localhost:7106', container: '#subapp', activeRule: '/sub-b' }
])
start()
```

### 2.3 子应用接收

qiankun 会把 `onGlobalStateChange` / `setGlobalState` **自动注入**到子应用 `mount(props)` 的 `props` 中：

```ts
// sub-a/src/main.ts
import { ref, type Ref } from 'vue'

let globalState: Ref<any> = ref({})

export async function mount(props) {
  // ⭐ props 自动注入了通信方法
  props.onGlobalStateChange((state, prev) => {
    console.log('[sub-a] 收到状态', state)
    globalState.value = state
  }, true)  // true: 立即触发一次当前状态

  const app = createApp(App)
  app.provide('globalState', globalState)  // 提供给组件使用
  app.mount(props.container.querySelector('#app'))
}

export async function unmount() {
  // offGlobalStateChange 会被 qiankun 自动调用，无需手动
}
```

子应用修改全局状态（会广播给主应用和其他子应用）：

```ts
// sub-a 某组件中
props.setGlobalState({ theme: 'dark' })  // 修改已存在的一级属性 theme
```

### 2.4 子应用间通过全局状态通信

由于 `setGlobalState` 会广播给**所有监听者**（主应用 + 其他子应用），子应用间天然能通过全局状态通信：

```text
子应用 A 调用 setGlobalState({ user: {...} })
        │
        ▼
   qiankun 广播给所有监听者
        │
   ┌────┴────┐
   ▼         ▼
 主应用    子应用 B（onGlobalStateChange 触发）
```

```ts
// 子应用 A：发布
props.setGlobalState({ user: { name: '张三', id: 1 } })

// 子应用 B：订阅（在 mount 中）
props.onGlobalStateChange((state) => {
  console.log('A 更新了用户:', state.user)
})
```

> 💡 **提示：** 这种方式要求主应用预定义 `user` 一级属性。适合「共享状态」类通信，不适合「一次性事件」。

## 三、props 传递（主 → 子）

主应用注册时通过 `props` 传数据/函数给子应用，单向、初始化时传递：

```ts
registerMicroApps([{
  name: 'sub-a',
  entry: '//localhost:7105',
  container: '#subapp',
  activeRule: '/sub-a',
  props: {
    routerBase: '/sub-a',
    config: { apiUrl: '/api' },          // 静态配置
    getToken: () => localStorage.getItem('token'),  // 回调函数
    apiService: {                        // 共享服务
      request: (url) => fetch(url).then(r => r.json())
    }
  }
}])
```

子应用 `mount(props)` 中接收。适合传「不常变」的配置和工具函数。

## 四、子应用间通信方案

当子应用间需要**直接通信**（不经主应用全局状态）时，可用以下方案。

### 4.1 方案一：CustomEvent（事件通知）

走真实 DOM 事件，不受 JS 沙箱影响，适合一次性事件通知：

```ts
// ===== 子应用 A：发送事件 =====
window.dispatchEvent(new CustomEvent('sub-a:data-ready', {
  detail: { msg: '你好，B', time: Date.now() }
}))

// ===== 子应用 B：监听事件 =====
let handler
export async function mount(props) {
  handler = (e) => {
    console.log('[sub-b] 收到 A 的事件:', e.detail)
  }
  window.addEventListener('sub-a:data-ready', handler)
}

export async function unmount() {
  // ⚠️ 必须移除监听，否则重复绑定
  window.removeEventListener('sub-a:data-ready', handler)
}
```

> ⚠️ **注意：** `unmount` 时务必 `removeEventListener`，否则每次 mount 都会重复绑定，导致回调被多次触发。

### 4.2 方案二：共享 EventBus（主应用创建）

主应用创建一个 EventBus 实例，通过 `props` 传给所有子应用，确保共享同一实例：

```ts
// ===== 主应用：创建并传递 EventBus =====
class EventBus {
  private events: Record<string, Function[]> = {}
  on(event: string, cb: Function) {
    (this.events[event] ||= []).push(cb)
  }
  off(event: string, cb: Function) {
    this.events[event] = this.events[event]?.filter(fn => fn !== cb) || []
  }
  emit(event: string, data?: any) {
    this.events[event]?.forEach(cb => cb(data))
  }
}

const bus = new EventBus()

registerMicroApps([
  { name: 'sub-a', ..., props: { bus } },
  { name: 'sub-b', ..., props: { bus } }
])
```

```ts
// ===== 子应用 A：发布 =====
export async function mount(props) {
  props.bus.emit('user:updated', { name: '李四' })
}

// ===== 子应用 B：订阅 =====
let handler
export async function mount(props) {
  handler = (data) => console.log('[sub-b] 收到:', data)
  props.bus.on('user:updated', handler)
}

export async function unmount(props) {
  props.bus.off('user:updated', handler)
}
```

> 💡 **为什么不用 `window.bus`？** 沙箱环境下子应用的 `window` 是代理对象，子应用往 `window` 写的属性不会共享给其他子应用。所以 EventBus 要由主应用创建（真实对象）并通过 `props` 传递。

### 4.3 方案三：localStorage（持久化）

适合跨标签页、需持久化的少量数据：

```ts
// 写入
localStorage.setItem('shared-data', JSON.stringify({ user: '王五' }))

// 其他标签页监听变化
window.addEventListener('storage', (e) => {
  if (e.key === 'shared-data') {
    console.log('数据变化:', JSON.parse(e.newValue))
  }
})
```

> ⚠️ **注意：** `localStorage` 是同步操作，且容量有限（约 5MB），不适合频繁、大量数据。

## 五、通信方案对比

| 方案 | 方向 | 实时性 | 持久化 | 复杂度 | 推荐度 |
| --- | --- | --- | --- | --- | --- |
| `initGlobalState` | 主↔子、子↔子 | 实时 | 否 | 低 | ⭐⭐⭐⭐⭐ |
| `props` | 主→子 | 初始化 | 否 | 低 | ⭐⭐⭐⭐ |
| `CustomEvent` | 子↔子 | 实时 | 否 | 低 | ⭐⭐⭐⭐ |
| 共享 `EventBus` | 子↔子 | 实时 | 否 | 中 | ⭐⭐⭐ |
| `localStorage` | 跨标签页 | storage 事件 | 是 | 低 | ⭐⭐ |

## 六、实战示例：用户登录状态共享

完整链路：主应用登录 → 广播给所有子应用 → 子应用响应式更新 UI。

```ts
// ===== 主应用 =====
import { initGlobalState } from 'qiankun'

const actions = initGlobalState({
  user: { name: '游客', id: 0 },
  isLogin: false
})

// 登录成功后广播
async function handleLogin() {
  const user = await loginApi()
  actions.setGlobalState({ user, isLogin: true })
}
```

```ts
// ===== 子应用（Vue3 响应式） =====
import { ref, type Ref } from 'vue'

const user = ref({ name: '游客' })
const isLogin = ref(false)

export async function mount(props) {
  props.onGlobalStateChange((state) => {
    user.value = state.user
    isLogin.value = state.isLogin
  }, true)

  const app = createApp(App)
  app.provide('user', user)
  app.provide('isLogin', isLogin)
  app.mount(props.container.querySelector('#app'))
}
```

```vue
<!-- 子应用组件 -->
<template>
  <div v-if="isLogin">欢迎，{{ user.name }}</div>
  <div v-else>请登录</div>
</template>

<script setup lang="ts">
import { inject, type Ref } from 'vue'

const user = inject<Ref<{ name: string }>>('user')!
const isLogin = inject<Ref<boolean>>('isLogin')!
</script>
```

## 七、注意事项

### 7.1 子应用只能修改已存在的一级属性

```ts
// 主应用预定义：{ user, theme }
initGlobalState({ user: {...}, theme: 'light' })

// 子应用
props.setGlobalState({ theme: 'dark' })      // ✅ theme 已存在
props.setGlobalState({ newProp: 'x' })       // ❌ newProp 不存在，无效
props.setGlobalState({ user: { name: 'x' } }) // ✅ 修改 user 整体
```

### 7.2 监听器必须在 unmount 清理

`onGlobalStateChange` 注册的监听，qiankun 会在 `unmount` 时**自动** `offGlobalStateChange`。但 `CustomEvent`、`EventBus` 的监听需**手动**移除，否则重复绑定。

### 7.3 fireImmediately 的作用

`onGlobalStateChange(cb, true)` 的第二个参数为 `true` 时，注册后立即用当前状态触发一次 `cb`。子应用 mount 后想立刻拿到当前全局状态，要传 `true`。

### 7.4 不要用全局变量通信

沙箱环境下，子应用往 `window` 写的变量不会共享。跨应用通信必须用上述官方/事件方案。

## 八、总结

- 主↔子通信首选 `initGlobalState`，子应用通过 `props.onGlobalStateChange/setGlobalState` 收发
- 主→子传配置用 `props`
- 子↔子通信：全局状态中转、`CustomEvent`、共享 `EventBus`
- 子应用 `setGlobalState` 只能改已存在的一级属性，主应用要预定义
- 所有事件监听在 `unmount` 时清理，防止重复绑定
