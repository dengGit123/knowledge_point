# qiankun 加载与预加载策略

> 📖 官方文档：[prefetch 配置](https://qiankun.umijs.org/zh/api#startopts) | [prefetchApps](https://qiankun.umijs.org/zh/api#prefetchappsapps-importentryopts)

qiankun 加载子应用有「按需加载」和「预加载」两种策略。合理配置预加载能显著提升二次访问速度。本文讲解加载流程、`prefetch` 配置与性能优化。

## 一、子应用加载流程

当一个子应用被 `activeRule` 激活时，qiankun（通过 `import-html-entry`）的加载过程：

```text
① beforeLoad（主应用钩子）
        │
        ▼
② fetch 子应用 HTML
        │
        ▼
③ 解析 HTML，提取 <script> / <style> / <link>
        │
        ▼
④ fetch 这些 JS / CSS 资源
        │
        ▼
⑤ 在 JS 沙箱内按顺序执行脚本
        │
        ▼
⑥ 识别子应用导出的 bootstrap / mount / unmount
        │
        ▼
⑦ bootstrap() → beforeMount → mount() → afterMount
```

**按需加载**：只有访问到对应路由，子应用资源才被请求。首次访问会有网络 + 解析 + 执行的开销。

**预加载**：在浏览器空闲时，提前把「即将可能访问」的子应用资源下载并缓存，用户真正访问时秒开。

## 二、prefetch 预加载配置

`start({ prefetch })` 支持四种配置：

### 2.1 四种配置

```ts
// ① 默认 true：第一个子应用 mount 完成后，预加载其他子应用资源
start({ prefetch: true })

// ② 'all'：start 后立即预加载所有子应用资源
start({ prefetch: 'all' })

// ③ string[]：第一个子应用 mounted 后，只预加载指定子应用
start({ prefetch: ['sub-vue', 'sub-vue2'] })

// ④ function：完全自定义首屏 / 次屏应用
start({
  prefetch: (apps) => ({
    criticalAppNames: ['sub-vue'],   // 首屏应用：立即加载
    minorAppsName: ['sub-vue2']      // 次屏应用：浏览器空闲时加载
  })
})
```

| 配置 | 预加载时机 | 范围 | 适用场景 |
| --- | --- | --- | --- |
| `true` | 第一个子应用 mount 后 | 其他所有子应用 | 通用，平衡速度与流量 |
| `'all'` | `start()` 后立即 | 所有子应用 | 子应用少、资源小、要求首屏快 |
| `string[]` | 第一个 mount 后 | 指定子应用 | 只预加载高频访问的子应用 |
| `function` | 自定义 | 自定义首屏/次屏 | 精细控制加载优先级 |
| `false` | 不预加载 | 无 | 流量敏感、子应用大 |

### 2.2 预加载时机示意

```text
配置 prefetch: true（默认）
─────────────────────────────────────
start()                用户访问 /sub-vue
  │                          │
  │                          ▼
  │                     sub-vue 加载 + mount
  │                          │
  │                          ▼ mount 完成
  │                     浏览器空闲时预加载
  │                     sub-vue2 / sub-vue3 资源
  │                          │
  │                          ▼
  │                     用户访问 /sub-vue2 → 秒开（资源已缓存）


配置 prefetch: 'all'
─────────────────────────────────────
start()
  │
  ▼ 立即预加载所有子应用资源
  │
  ▼ 用户访问任意子应用 → 秒开
```

> 💡 **提示：** 预加载只下载资源（JS/CSS），**不执行**脚本、不渲染。真正激活时才执行 `bootstrap`/`mount`，但省去了网络等待。

## 三、prefetchApps —— 手动预加载

仅 `loadMicroApp` 手动加载场景使用；基于路由自动激活的场景直接配 `start({ prefetch })` 即可。

```ts
import { prefetchApps } from 'qiankun'

// 手动预加载指定子应用的静态资源
prefetchApps([
  { name: 'app1', entry: '//localhost:7001' },
  { name: 'app2', entry: '//localhost:7002' }
])
```

## 四、loadMicroApp —— 手动加载

适用于不与路由关联、需手动控制加载/卸载的场景（如页面内嵌组件）：

```ts
import { loadMicroApp } from 'qiankun'

const microApp = loadMicroApp({
  name: 'sub-vue',
  entry: '//localhost:7105',
  container: '#panel',
  props: { brand: 'qiankun' }
}, {
  singular: false   // 多实例：允许多个子应用同时存在
})

// 手动卸载
microApp.unmount()

// 更新 props（需子应用导出 update 钩子）
microApp.update({ brand: 'new' })
```

> 💡 **提示：** `loadMicroApp` 的 `singular` 默认 `false`（多实例），与 `start` 的默认 `true` 不同。详见 [03-主应用配置](./03-主应用配置.md#五、loadmicroapp-——-手动加载子应用)。

## 五、singular 单实例 / 多实例

`singular` 控制同一时刻能渲染几个子应用，影响加载行为：

### 5.1 单实例（默认 `singular: true`）

```ts
start({ singular: true })
```

- 同一时刻只渲染一个子应用
- 路由切换：旧子应用 `unmount` → 新子应用 `mount`
- 沙箱用 LegacySandbox（有 Proxy）或 SnapshotSandbox（降级）
- **适合**：后台管理系统，一个页面一个子应用

### 5.2 多实例（`singular: false`）

```ts
start({ singular: false })
```

- 多个子应用可同时渲染在不同容器
- 沙箱必须用 ProxySandbox（每个子应用独立 fakeWindow）
- **注意**：多实例样式隔离需额外配置，否则子应用样式互相覆盖
- **适合**：仪表盘、多面板同时展示

```text
单实例：                多实例：
┌──────────────┐      ┌──────┬──────┐
│              │      │ 子A  │ 子B  │
│   子应用 A    │      │      │      │
│   （唯一）    │      ├──────┴──────┤
│              │      │    子应用 C   │
└──────────────┘      └───────────────┘
```

## 六、性能优化建议

### 6.1 选择合适的 prefetch 策略

| 场景 | 推荐配置 |
| --- | --- |
| 子应用少（≤3个）、资源小 | `'all'` |
| 子应用多、流量敏感 | `true` 或 `string[]`（只预加载高频应用） |
| 需精细控制 | `function` 区分首屏/次屏 |

### 6.2 资源上 CDN

子应用的 JS/CSS/图片/字体部署到 CDN，提升 fetch 速度，也便于跨域配置。

### 6.3 共享依赖（externals）

多个子应用都用 Vue 时，可将 Vue 设为 external，由主应用统一提供，避免重复加载：

```js
// 子应用 webpack 配置
externals: {
  vue: 'Vue',
  'vue-router': 'VueRouter'
}
```

> ⚠️ **注意：** 共享依赖要求主子应用 Vue 版本一致，否则可能不兼容。这是把双刃剑，需谨慎。

### 6.4 自定义 fetch 加缓存

```ts
start({
  fetch: (url, ...args) => {
    return window.fetch(url, {
      ...args,
      cache: 'force-cache'   // 强制缓存
    })
  }
})
```

### 6.5 子应用按需拆包

子应用自身用路由懒加载，避免单个 chunk 过大：

```ts
// 子应用路由
const routes = [
  { path: '/home', component: () => import('./views/Home.vue') },
  { path: '/list', component: () => import('./views/List.vue') }
]
```

## 七、总结

- 加载流程：fetch HTML → 解析 → 执行脚本 → 识别生命周期 → bootstrap → mount
- `prefetch` 四种配置：`true`（默认）/`'all'`/`string[]`/`function`，按场景选择
- `prefetchApps` 仅手动加载场景用
- `loadMicroApp` 手动加载，`singular` 默认 `false`（多实例）
- 性能优化：合理 prefetch、CDN、共享依赖、缓存、子应用懒加载
