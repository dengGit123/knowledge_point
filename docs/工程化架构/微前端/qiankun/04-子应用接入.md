# qiankun 子应用接入

> 📖 官方文档：[微应用接入](https://qiankun.umijs.org/zh/guide/getting-started#%E5%BE%AE%E5%BA%94%E7%94%A8) | [常见问题](https://qiankun.umijs.org/zh/faq)

子应用接入 qiankun 的核心是两件事：**导出生命周期钩子** + **配置打包工具**。本文覆盖 Webpack（v4/v5）与 Vite 两种构建场景。

## 一、接入总览

```text
子应用需要做的：
├── 1. 入口 js 导出 bootstrap / mount / unmount（+ 可选 update）
├── 2. 打包配置（umd 导出 / vite-plugin-qiankun）
├── 3. publicPath 配置（解决资源 404）
├── 4. dev server 开启 CORS
└── 5. 支持独立运行
```

## 二、导出生命周期钩子

子应用必须在自己的入口 JS（webpack 的 entry js）导出三个生命周期钩子，供主应用在适当时机调用：

```ts
/**
 * 初始化（只执行一次）
 * 应用首次加载时调用，用于初始化全局配置
 */
export async function bootstrap() {
  console.log('[子应用] bootstrap')
}

/**
 * 挂载（每次激活时执行）
 * 应用进入激活状态时调用，props 由主应用传入
 */
export async function mount(props) {
  const { container, routerBase } = props
  render({ container, routerBase })
}

/**
 * 卸载（每次失活时执行）
 * 应用离开激活状态时调用，必须清理干净
 */
export async function unmount(props) {
  // 销毁实例、清空容器、移除副作用
}

/**
 * 更新（可选）
 * 仅 loadMicroApp 手动加载场景需要
 */
export async function update(props) {
  console.log('[子应用] update', props)
}
```

> ⚠️ **注意：** 这四个函数名是 qiankun 约定的，**不能改名**，且必须用 `export` 导出。导出失败会报 `You need to export the functional lifecycles in xxx entry`。

## 三、Webpack 子应用打包配置

为了让主应用能识别子应用导出的生命周期，子应用的 `output` 需配置为 UMD 格式。

### 3.1 webpack v5

```js
// vue.config.js 或 webpack.config.js
const packageName = require('./package.json').name

module.exports = {
  configureWebpack: {
    output: {
      library: `${packageName}-[name]`,        // 导出库名
      libraryTarget: 'umd',                    // UMD 格式
      chunkLoadingGlobal: `webpackJsonp_${packageName}`  // 隔离 chunk 加载
    }
  }
}
```

### 3.2 webpack v4

```js
const packageName = require('./package.json').name

module.exports = {
  configureWebpack: {
    output: {
      library: `${packageName}-[name]`,
      libraryTarget: 'umd',
      jsonpFunction: `webpackJsonp_${packageName}`  // v4 用 jsonpFunction
    }
  }
}
```

### 3.3 关键检查项（来自官方 FAQ）

打包配置若不对，会出现 `LOADING_SOURCE_CODE` 错误。逐条排查：

| 检查项 | 要求 |
| --- | --- |
| 生命周期导出 | 入口 js 必须导出 `bootstrap`/`mount`/`unmount` |
| `output.globalObject` | 若配置了，值必须是 `'window'`，或直接移除用默认值 |
| `package.json` 的 `name` | 全局唯一，且与主应用注册的 `name` 一致 |
| 入口 js 加载顺序 | 必须是 HTML 中**最后一个**加载的脚本（或手动标记 `entry`） |
| 模块化格式 | 不能是 AMD/CommonJS，否则需将 `libraryTarget` 改为 `'window'` |

### 3.4 AMD/CommonJS 兼容问题

若主应用和子应用用了 AMD 或 CommonJS 模块化，`libraryTarget: 'umd'` 会失效。两种解决办法：

```js
// 办法一：libraryTarget 改为 'window'
output: {
  library: `${packageName}-[name]`,
  libraryTarget: 'window',   // 而非 'umd'
  chunkLoadingGlobal: `webpackJsonp_${packageName}`
}
```

```js
// 办法二：不用打包，直接把生命周期挂到 window 上（非 webpack 构建的应用）
window['sub-vue'] = {
  bootstrap() { /* ... */ },
  mount(props) { /* ... */ },
  unmount(props) { /* ... */ }
}
```

## 四、Vite 子应用打包配置

Vite 输出 ESM，qiankun 默认沙箱无法直接处理，需用 [`vite-plugin-qiankun`](https://www.npmjs.com/package/vite-plugin-qiankun)。

```ts
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import qiankun from 'vite-plugin-qiankun'

export default defineConfig({
  plugins: [
    vue(),
    qiankun('sub-vue', { useDevMode: true })  // 名称与主应用注册一致
  ],
  base: 'http://localhost:7105/',   // 生产环境需改为线上域名
  server: {
    port: 7105,
    cors: true,
    origin: 'http://localhost:7105'
  }
})
```

入口用 `renderWithQiankun` 注册生命周期（替代手动 `export`）：

```ts
import { renderWithQiankun, qiankunWindow } from 'vite-plugin-qiankun/dist/helper'

renderWithQiankun({
  bootstrap() { /* ... */ },
  mount(props) { render(props) },
  unmount() { app?.unmount() },
  update(props) { /* ... */ }
})

if (!qiankunWindow.__POWERED_BY_QIANKUN__) {
  render()  // 独立运行
}
```

> ⚠️ **注意：**
> - `useDevMode: true` 会关闭热更新（与热更新插件冲突），开发时如此配置才能作为子应用加载
> - Vite ESM 子应用**不运行在 JS 沙箱中**，操作 `window` 要用 `qiankunWindow`
> - 完整示例见 [02-快速上手](./02-快速上手.md#三、子应用一：vue3--vite)

## 五、publicPath 配置（解决资源 404）

子应用被主应用加载后，其动态资源（脚本、样式、图片）的加载路径容易出错导致 404。两种解决方式：

### 5.1 运行时 publicPath（推荐）

qiankun 会在子应用 `bootstrap` 前注入一个运行时变量。在**入口 js 最顶部**添加：

```js
// 必须放在所有动态 import 之前
if (window.__POWERED_BY_QIANKUN__) {
  // eslint-disable-next-line no-undef
  __webpack_public_path__ = window.__INJECTED_PUBLIC_PATH_BY_QIANKUN__
}
```

它解决子应用**动态载入**的脚本、样式、图片地址不正确的问题。

### 5.2 静态 publicPath

直接把 `publicPath` 配成绝对地址：

```js
output: {
  publicPath: `//localhost:7105`   // 生产环境改为线上域名
}
```

> 💡 **提示：** 运行时方案更灵活（一份代码适配多个环境），静态方案更简单。生产环境推荐静态配置线上 CDN 域名。

## 六、CORS 跨域配置

主应用通过 `fetch` 跨域获取子应用资源，**子应用静态资源必须支持跨域**。

### 6.1 Webpack dev server

```js
devServer: {
  headers: {
    'Access-Control-Allow-Origin': '*'
  }
}
```

### 6.2 Vite dev server

```ts
server: {
  cors: true
}
```

### 6.3 生产环境 Nginx

```nginx
location / {
  add_header Access-Control-Allow-Origin *;
  add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS';
  add_header Access-Control-Allow-Headers 'Content-Type';
}
```

> ⚠️ **注意：** 第三方脚本若无法添加跨域头，可下载到本地由自己的服务器 serve，再配置跨域。

## 七、支持独立运行

子应用应能脱离主应用独立运行，方便单独开发调试。核心是判断 `__POWERED_BY_QIANKUN__`：

```ts
// Webpack 子应用
if (!window.__POWERED_BY_QIANKUN__) {
  render()  // 独立运行时手动挂载
}

// Vite 子应用
if (!qiankunWindow.__POWERED_BY_QIANKUN__) {
  render()
}
```

`render` 函数内部根据是否在 qiankun 环境决定路由 base 和挂载容器（见 [02-快速上手](./02-快速上手.md#五、独立运行-vs-集成运行)）。

## 八、子应用改造清单

接入一个现有应用，按此清单逐项检查：

```text
□ 1. package.json 的 name 全局唯一，与主应用注册名一致
□ 2. 入口导出 bootstrap / mount / unmount
□ 3. 打包配置 output.library + libraryTarget: 'umd'
       webpack5: chunkLoadingGlobal
       webpack4: jsonpFunction
□ 4. 入口顶部加运行时 publicPath（webpack 子应用）
□ 5. dev server 配置 CORS
□ 6. mount 中用 container.querySelector('#app') 挂载
□ 7. unmount 中销毁实例 + 清空容器 innerHTML
□ 8. 路由 base 用 props.routerBase（qiankun 环境）
□ 9. 支持独立运行（!__POWERED_BY_QIANKUN__ 时手动 render）
□ 10. 字体/图片资源处理（CDN 或 base64，见 11-常见问题）
```

## 九、总结

- 子应用接入核心：**导出生命周期** + **配置打包**（umd / vite-plugin-qiankun）
- Webpack 子应用配 `library/umd`，Vite 子应用用 `vite-plugin-qiankun`
- `publicPath` 用运行时注入或静态绝对地址，解决资源 404
- 静态资源**必须支持跨域**
- 每个子应用都要支持独立运行，用 `__POWERED_BY_QIANKUN__` 区分
