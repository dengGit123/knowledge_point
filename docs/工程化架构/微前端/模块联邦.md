### 1.2 模块联邦在微前端中的定位

**模块联邦（Module Federation）** 是一种**编译时+运行时混合**的模块共享方案，允许开发者将应用程序拆分为多个独立的构建块，每个构建块可以独立开发、部署和运行。

| 特性 | qiankun/single-spa | 模块联邦 | iframe |
|------|-------------------|----------|--------|
| **集成时机** | 纯运行时 | 编译时配置 + 运行时加载 | 纯运行时 |
| **技术栈限制** | 需遵循生命周期协议 | 完全无限制 | 完全无限制 |
| **依赖共享** | 需要外部 CDN | 原生支持依赖共享 | 依赖重复加载 |
| **开发复杂度** | 中等 | 较低 | 低 |
| **类型支持** | 较弱 | 强（TypeScript 友好） | 无 |
| **性能开销** | 中等 | 低 | 高 |
| **适用场景** | 完整的微前端架构 | 模块/组件级共享 | 跨系统集成 |

### 1.3 模块联邦的核心优势

**核心优势：**

1. **细粒度共享** - 可以共享组件、函数、常量等任何模块
2. **依赖去重** - 自动识别并共享相同的依赖版本
3. **类型安全** - 完整的 TypeScript 支持
4. **独立部署** - 每个应用可独立开发、测试、部署
5. **热更新** - 开发环境支持跨应用的 HMR

## 二、模块联邦核心概念

### 2.1 基本术语

| 术语 | 说明 | 示例 |
|------|------|------|
| **Host（宿主）** | 消费其他应用模块的应用 | 主应用加载远程组件 |
| **Remote（远程）** | 提供模块给其他应用的应用 | 组件库应用导出组件 |
| **Shared（共享）** | 多个应用间共享的依赖 | Vue、Pinia、lodash |
| **Exposes（暴露）** | Remote 提供给外部使用的模块映射 | `./Button: './src/Button.vue'` |
| **Remotes（远程引用）** | Host 引用的 Remote 应用配置 | `libA: 'lib_a@http://...'` |

### 2.2 工作原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                      模块联邦加载流程                                │
└─────────────────────────────────────────────────────────────────────┘

1. 构建时 (Build Time)
═══════════════════════════════════════════════════════════════════════

   Remote 应用                          Host 应用
   ┌─────────────────┐                ┌─────────────────┐
   │                 │                │                 │
   │  构建产物:       │                │  配置:          │
   │  ┌───────────┐  │                │  ┌───────────┐  │
   │  │remoteEntry│  │────────────────┼─>│remotes:   │  │
   │  │   .js     │  │                │  │ libA: ... │  │
   │  │           │  │                │  └───────────┘  │
   │  │ src_*.js  │  │                │                 │
   │  │ (chunks)  │  │                │                 │
   │  └───────────┘  │                │                 │
   └─────────────────┘                └─────────────────┘
            │                                  │
            │                                  │
            └────────────┬─────────────────────┘
                         │
                         ▼

2. 运行时 (Runtime)
═══════════════════════════════════════════════════════════════════════

   ① 浏览器加载 Host 应用
                          ↓
   ② 解析 remotes 配置，获取 remoteEntry URL
                          ↓
   ③ 加载 remoteEntry.js (获取 manifest)
      ┌─────────────────────────────────────┐
      │ {                                   │
      │   "./Button": {                     │
      │     "name": "./Button",             │
      │     "type": "namespace",            │
      │     "entry": "src_Button_js.js"     │
      │   },                                │
      │   "./Card": { ... }                 │
      │ }                                   │
      └─────────────────────────────────────┘
                          ↓
   ④ 代码执行到 import('libA/Button')
                          ↓
   ⑤ 触发容器初始化，进行版本协商
      ┌─────────────────────────────────────┐
      │ 共享依赖版本协商:                     │
      │ - vue: 3.3.4 (Host)                 │
      │ - vue: 3.3.4 (Remote) ✓ 一致        │
      │ - 使用单例模式共享                  │
      └─────────────────────────────────────┘
                          ↓
   ⑥ 加载实际模块 chunk (src_Button_js.js)
                          ↓
   ⑦ 返回模块导出内容，完成加载
```

### 2.3 依赖共享机制

```typescript
// 版本协商算法详解
shared: {
  "vue": {
    requiredVersion: "^3.3.0",  // 要求的版本范围
    singleton: true,              // 全局只加载一个版本
    strictVersion: false,         // 是否严格版本匹配
    eager: false,                 // 是否立即加载（同步）
    version: "3.3.4"            // 当前应用的版本
  }
}

// 协商流程:
// ┌─────────────────────────────────────────────────────────────┐
// │ 1. 收集所有应用对 vue 的版本要求和实际版本                    │
// │    - Host: 3.3.4, 要求 ^3.3.0                               │
// │    - RemoteA: 3.3.4, 要求 ^3.3.0                            │
// │    - RemoteB: 3.3.0, 要求 ^3.0.0                            │
// │                                                              │
// │ 2. 找到满足所有要求的最高版本                                  │
// │    - 3.3.4 ✓ 满足所有要求                                     │
// │                                                              │
// │ 3. 所有应用共享这个版本                                        │
// │    - 只加载一次 3.3.4                                        │
// │    - 其他应用复用已加载的版本                                  │
// └─────────────────────────────────────────────────────────────┘
```

```typescript
// packages/remote-ui-lib/vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import federation from '@originjs/vite-plugin-federation'

export default defineConfig({
  plugins: [
    vue(),
    federation({
      // 应用唯一标识
      name: 'uiLib',

      // 入口文件名
      filename: 'remoteEntry.js',

      // 暴露给外部使用的模块
      exposes: {
        './Button': './src/components/Button.vue',
        './Card': './src/components/Card.vue',
        './Table': './src/components/Table.vue',
        './Modal': './src/components/Modal.vue',
        './composables': './src/composables/index.ts',
        './utils': './src/utils/index.ts'
      },

      // 共享依赖
      shared: {
        vue: {
          requiredVersion: '^3.3.0',
          singleton: true
        }
      }
    })
  ],

  // 构建配置
  build: {
    target: 'esnext',
    minify: 'esbuild',
    cssCodeSplit: true,
    rollupOptions: {
      output: {
        // 手动分块
        manualChunks: {
          'vue-vendor': ['vue']
        }
      }
    }
  },

  // 开发服务器配置
  server: {
    port: 5001,
    strictPort: true,
    headers: {
      // 允许跨域访问
      'Access-Control-Allow-Origin': '*',
      'Access-Control-Allow-Methods': 'GET, POST, OPTIONS',
      'Access-Control-Allow-Headers': 'Content-Type'
    }
  }
})
```

```vue
  width?: string

  columns: Column[]
  data: T[]
  loading?: boolean
  bordered?: boolean
  rowKey?: string
  pagination?: boolean
  pageSize?: number
}

const props = withDefaults(defineProps<Props>(), {
  loading: false,
  bordered: false,
  rowKey: 'id',
  pagination: false,
  pageSize: 10
})

const sortKey = ref<string>('')
const sortOrder = ref<'asc' | 'desc' | ''>('')
const currentPage = ref(1)

const sortedData = computed(() => {
  let result = [...props.data]

  // 排序
  if (sortKey.value) {
    result.sort((a, b) => {
      const aVal = a[sortKey.value]
      const bVal = b[sortKey.value]

      if (aVal === bVal) return 0

      const result = aVal > bVal ? 1 : -1
      return sortOrder.value === 'asc' ? result : -result
    })
  }

  // 分页
  if (props.pagination) {
    const start = (currentPage.value - 1) * props.pageSize
    const end = start + props.pageSize
    result = result.slice(start, end)
  }

  return result
})

const totalPages = computed(() => {
  return Math.ceil(props.data.length / props.pageSize)
})

const getSortIcon = (key: string) => {
  if (sortKey.value !== key) return '↕'
  if (sortOrder.value === 'asc') return '↑'
  if (sortOrder.value === 'desc') return '↓'
  return '↕'
}

const handleSort = (key: string) => {
  if (sortKey.value === key) {
    if (sortOrder.value === 'asc') {
      sortOrder.value = 'desc'
    } else if (sortOrder.value === 'desc') {
      sortOrder.value = ''
      sortKey.value = ''
    } else {
      sortOrder.value = 'asc'
    }
  } else {
    sortKey.value = key
    sortOrder.value = 'asc'
  }
}

watch(currentPage, () => {
  // 滚动到顶部
  window.scrollTo({ top: 0, behavior: 'smooth' })
})
```

```typescript
/**
 * 计数器组合式函数
 */
export function useCounter(initialValue = 0) {
  const count = ref(initialValue)

  const increment = (step = 1) => {
    count.value += step
  }

  const decrement = (step = 1) => {
    count.value -= step
  }

  const reset = () => {
    count.value = initialValue
  }

  return {
    count,
    increment,
    decrement,
    reset
  }
}

/**
 * 异步状态管理
 */
export interface AsyncState<T> {
  data: Ref<T | null>
  loading: Ref<boolean>
  error: Ref<Error | null>
}

export function useAsync<T>(
  asyncFn: () => Promise<T>,
  immediate = false
): AsyncState<T> & { execute: () => Promise<void> } {
  const data = ref<T | null>(null) as Ref<T | null>
  const loading = ref(false)
  const error = ref<Error | null>(null)

  const execute = async () => {
    loading.value = true
    error.value = null

    try {
      data.value = await asyncFn()
    } catch (err) {
      error.value = err as Error
    } finally {
      loading.value = false
    }
  }

  if (immediate) {
    execute()
  }

  return { data, loading, error, execute }
}

/**
 * 防抖
 */
export function useDebounce<T extends (...args: any[]) => any>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timeoutId: ReturnType<typeof setTimeout>

  return (...args: Parameters<T>) => {
    clearTimeout(timeoutId)
    timeoutId = setTimeout(() => fn(...args), delay)
  }
}

/**
 * 节流
 */
export function useThrottle<T extends (...args: any[]) => any>(
  fn: T,
  limit: number
): (...args: Parameters<T>) => void {
  let inThrottle: boolean

  return (...args: Parameters<T>) => {
    if (!inThrottle) {
      fn(...args)
      inThrottle = true
      setTimeout(() => (inThrottle = false), limit)
    }
  }
}
```

```typescript
/**

export const formatDate = (
  date: Date | string | number,
  format = 'YYYY-MM-DD HH:mm:ss'
): string => {
  const d = typeof date === 'string' ? new Date(date) : new Date(date)

  const year = d.getFullYear()
  const month = String(d.getMonth() + 1).padStart(2, '0')
  const day = String(d.getDate()).padStart(2, '0')
  const hours = String(d.getHours()).padStart(2, '0')
  const minutes = String(d.getMinutes()).padStart(2, '0')
  const seconds = String(d.getSeconds()).padStart(2, '0')

  return format
    .replace('YYYY', String(year))
    .replace('MM', month)
    .replace('DD', day)
    .replace('HH', hours)
    .replace('mm', minutes)
    .replace('ss', seconds)
}

/**
 * 深拷贝
 */
export const deepClone = <T>(obj: T): T => {
  if (obj === null || typeof obj !== 'object') return obj
  if (obj instanceof Date) return new Date(obj.getTime()) as any
  if (obj instanceof Array) return obj.map(item => deepClone(item)) as any

  const clonedObj = {} as T
  for (const key in obj) {
    if (Object.prototype.hasOwnProperty.call(obj, key)) {
      clonedObj[key] = deepClone(obj[key])
    }
  }
  return clonedObj
}

    const v = c === 'x' ? r : (r & 0x3) | 0x8

  })
```

### 6.1 动态 Remote 配置

```typescript
// packages/host-app/src/utils/dynamicRemote.ts
/**
 * 运行时动态加载 Remote 应用
 */

interface RemoteConfig {
  scope: string
  module: string
  url: string
}

// 记录已加载的 Remote
const loadedRemotes = new Set<string>()

export async function loadRemote<T = any>(
  config: RemoteConfig
): Promise<T> {
  const { scope, module, url } = config
  const key = `${scope}@${url}`

  // 避免重复加载
  if (loadedRemotes.has(key)) {
    return getRemoteModule<T>(scope, module)
  }

  try {
    // 1. 动态加载 remoteEntry.js
    await loadScript(url)

    // 2. 初始化共享作用域
    await __webpack_init_sharing__('default')

    // 3. 获取容器
    const container = (window as any)[scope]
    if (!container) {
      throw new Error(`Remote container "${scope}" not found`)
    }

    // 4. 初始化容器
    await container.init(__webpack_share_scopes__.default)

    // 5. 标记已加载
    loadedRemotes.add(key)

    // 6. 获取模块
    return getRemoteModule<T>(scope, module)
  } catch (error) {
    console.error(`Failed to load remote: ${scope}`, error)
    throw error
  }
}

function loadScript(url: string): Promise<void> {
  return new Promise((resolve, reject) => {
    const element = document.querySelector(`script[src="${url}"]`)
    if (element) {
      resolve()
      return
    }

    const script = document.createElement('script')
    script.src = url
    script.type = 'text/javascript'
    script.async = true

    script.onload = () => resolve()
    script.onerror = () => reject(new Error(`Failed to load script: ${url}`))

    document.head.appendChild(script)
  })
}

async function getRemoteModule<T>(scope: string, module: string): Promise<T> {
  const container = (window as any)[scope]
  const factory = await container.get(module)
  const Module = factory()
  return Module as T
}

// 使用示例
const loadRemoteComponent = async () => {
  try {
    const Button = await loadRemote({
      scope: 'uiLib',
      module: './Button',
      url: 'https://cdn.example.com/remoteEntry.js'
    })
    return Button.default
  } catch (error) {
    // 返回降级组件
    return FallbackButton
  }
}
```

### 7.4 监控与降级

```typescript
// packages/host-app/src/utils/remoteMonitor.ts
interface RemoteHealth {
  name: string
  url: string
  status: 'loading' | 'success' | 'error'
  lastCheck: number
  loadTime?: number
  errorCount: number
}

class RemoteMonitor {
  private healthMap = new Map<string, RemoteHealth>()
  private checkInterval: NodeJS.Timeout | null = null

  /**
   * 检查 Remote 健康状态
   */
  async checkRemote(name: string, url: string): Promise<boolean> {
    const startTime = performance.now()
    const existing = this.healthMap.get(name)

    try {
      const response = await fetch(url, { method: 'HEAD' })
      const loadTime = performance.now() - startTime

      this.healthMap.set(name, {
        name,
        url,
        status: response.ok ? 'success' : 'error',
        lastCheck: Date.now(),
        loadTime,
        errorCount: response.ok ? 0 : (existing?.errorCount || 0) + 1
      })

      return response.ok
    } catch (error) {
      this.healthMap.set(name, {
        name,
        url,
        status: 'error',
        lastCheck: Date.now(),
        errorCount: (existing?.errorCount || 0) + 1
      })

      this.reportError(name, error)

      // 错误次数超过阈值，触发降级
      if ((existing?.errorCount || 0) >= 3) {
        this.triggerFallback(name)
      }

      return false
    }
  }

  /**
   * 上报错误到监控平台
   */
  private reportError(name: string, error: any) {
    console.error(`[Remote Error] ${name}:`, error)

    // 发送到 Sentry
    // Sentry.captureException(error, {
    //   tags: { remote: name }
    // })
  }

  /**
   * 触发降级策略
   */
  private triggerFallback(name: string) {
    console.warn(`[Remote Fallback] Triggering fallback for ${name}`)

    // 使用本地缓存版本
    const cached = localStorage.getItem(`remote_cache_${name}`)
    if (cached) {
      window.postMessage({
        type: 'REMOTE_FALLBACK',
        data: { name, cached }
      }, '*')
    }
  }

  /**
   * 启动定时检查
   */
  startHealthCheck(interval: number = 60000) {
    if (this.checkInterval) {
      clearInterval(this.checkInterval)
    }

    this.checkInterval = setInterval(() => {
      this.healthMap.forEach((health, name) => {
        this.checkRemote(name, health.url)
      })
    }, interval)
  }

  /**
   * 获取所有健康状态
   */
  getHealthStatus(): RemoteHealth[] {
    return Array.from(this.healthMap.values())
  }
}

export const remoteMonitor = new RemoteMonitor()
```

## 八、常见问题与解决方案

### 8.1 开发环境跨域问题

**问题：** 本地开发时 Host 无法加载 Remote 的 remoteEntry.js

**解决方案：**

```typescript
// 方案一：Remote 应用配置 CORS
export default defineConfig({
  server: {
    headers: {
      'Access-Control-Allow-Origin': '*',
      'Access-Control-Allow-Methods': 'GET, POST, OPTIONS',
      'Access-Control-Allow-Headers': 'Content-Type'
    }
  }
})

// 方案二：使用代理
export default defineConfig({
  server: {
    proxy: {
      '/remote': {
        target: 'http://localhost:5001',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/remote/, '')
      }
    }
  }
})
```

### 8.2 共享依赖版本冲突

**问题：** 不同的 Remote 使用不同版本的依赖，导致运行时错误

**解决方案：**

```typescript
// 使用严格的版本管理
shared: {
  vue: {
    singleton: true,              // 只加载一个版本
    requiredVersion: '^3.3.0',    // 要求的版本
    strictVersion: true,          // 严格版本检查
    version: '3.3.4'             // 当前版本
  }
}

// 或者在 package.json 中锁定版本
{
  "dependencies": {
    "vue": "3.3.4"  // 使用精确版本
  }
}
```

### 8.3 类型推断失败

**问题：** 无法获得 Remote 组件的 TypeScript 类型

**解决方案：**

```typescript
// 方案一：创建类型声明文件
// types/remotes.d.ts
declare module 'uiLib/*' {
  import { DefineComponent } from 'vue'
  const component: DefineComponent<any>
  export default component
}

// 方案二：使用共享类型包
// packages/shared-types/src/button.ts
export interface ButtonProps {
  type?: 'primary' | 'secondary'
  onClick?: () => void
}

// Remote 应用
export { type ButtonProps }
export default defineComponent<ButtonProps>(...)

// Host 应用
import type { ButtonProps } from '@shared/types/button'
import RemoteButton from 'uiLib/Button'
```

### 8.5 共享状态管理

**问题：** 不同应用间需要共享状态

**解决方案：**

```typescript
// packages/shared/src/store.ts
import { createGlobalState } from '@vueuse/core'

export const useGlobalUserState = createGlobalState(() => {
  return reactive({
    user: null as User | null,
    token: '',
    login(user: User, token: string) {
      this.user = user
      this.token = token
      localStorage.setItem('token', token)
    },
    logout() {
      this.user = null
      this.token = ''
      localStorage.removeItem('token')
    }
  })
})

// Remote 应用中使用
import { useGlobalUserState } from '@shared/store'

const userState = useGlobalUserState()
// userState 是跨应用共享的单例
```

### 8.7 样式冲突

**问题：** Remote 组件的样式影响主应用

**解决方案：**

```vue
<!-- 方案一：使用 scoped 样式 -->
<style scoped>
/* 样式只作用于当前组件 */
.button {
  /* ... */
}
</style>

<!-- 方案二：使用 CSS Modules -->
<template>
  <button :class="$style.button">点击</button>
</template>

<style module>
.button {
  /* 生成的类名会自动添加 hash */
}
</style>

<!-- 方案三：使用命名空间 -->
<template>
  <div class="ui-lib-button">按钮</div>
</template>

<style>
.ui-lib-button {
  /* 所有样式都带前缀 */
}
</style>
```
