## 一、white-space —— 空白符和换行控制

### 1.1 属性值详解

`white-space` 属性控制如何处理元素内的**空白符**（空格、制表符等）和**换行符**。

| 值 | 空白符合并 | 换行符处理 | 自动换行 | 典型应用 |
|---|:---:|:---:|:---:|---|
| `normal` | ✅ 合并 | ❌ 忽略 | ✅ 允许 | 默认文本 |
| `nowrap` | ✅ 合并 | ❌ 忽略 | ❌ 禁止 | 单行标题 |
| `pre` | ❌ 保留 | ✅ 保留 | ❌ 禁止 | 代码块 |
| `pre-wrap` | ❌ 保留 | ✅ 保留 | ✅ 允许 | 格式化文本 |
| `pre-line` | ✅ 合并 | ✅ 保留 | ✅ 允许 | 诗歌/地址 |

### 1.2 代码示例

```css
/* HTML 源代码：
   这是   多个   空格
   这是第二行
*/

/* normal：默认行为 */
.normal { white-space: normal; }
/* 输出：这是 多个 空格 这是第二行 */

/* nowrap：不换行 */
.nowrap { white-space: nowrap; }
/* 输出：这是 多个 空格 这是第二行（强制一行） */

/* pre：保留原始格式 */
.pre { white-space: pre; }
/* 输出：这是   多个   空格
         这是第二行 */

/* pre-wrap：保留格式但自动换行 */
.pre-wrap { white-space: pre-wrap; }
/* 输出：这是   多个   空格
         这是第二行（超出宽度时自动换行） */

/* pre-line：合并空格但保留换行 */
.pre-line { white-space: pre-line; }
/* 输出：这是 多个 空格
         这是第二行 */
```

### 1.3 实际应用

```css
/* 应用 1：省略号效果 */
.ellipsis {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

/* 应用 2：代码块展示 */
code {
  white-space: pre;
  font-family: monospace;
  background: #f4f4f4;
  padding: 2px 6px;
}

/* 应用 3：诗歌排版 */
.poem {
  white-space: pre-line;
  font-style: italic;
}
```

## 二、word-break —— 换行策略控制

### 2.1 属性值详解

`word-break` 主要控制**非 CJK 文本**（中文/日文/韩文）的换行行为，决定是否允许在单词内部断行。

| 值 | CJK 文本 | 非 CJK 文本 | 说明 |
|---|----------|-------------|------|
| `normal` | 字之间换行 | 单词之间换行 | 默认值 |
| `break-all` | 任意位置换行 | 任意位置换行 | 强制断行，不保留单词完整性 |
| `keep-all` | 不换行（同英文） | 单词之间换行 | CJK 文本像单词一样处理 |
| `break-word` | - | - | 已弃用，等同于 `overflow-wrap: break-word` |

### 2.2 代码示例

```css
/* 测试文本：This is a verylongwordthatisunbreakable */

/* normal：默认行为 */
.normal { word-break: normal; }
/* 长单词会溢出容器或不换行 */

/* break-all：强制断行 */
.break-all { word-break: break-all; }
/* 在任意字符处换行：
   This is a verylongword
   thatisunbreakable */

/* keep-all：保持单词完整 */
.keep-all { word-break: keep-all; }
/* 中文：中文文本不会在字符间断开
   英文：normal 行为 */
```

### 2.3 CJK 文本的特殊性

```css
/* 中文默认可以在任意字符间换行 */
.zh-text {
  /* 这是一段很长
     的中文文本 */  /* 默认会在任意位置换行 */
}

/* 使用 keep-all 让中文像英文一样换行 */
.zh-text-keep {
  word-break: keep-all;
  /* 只在标点或空格处换行 */
}
```

### 2.4 实际应用

```css
/* 应用 1：防止内容溢出 */
.safe-container {
  word-break: break-all;
  /* 确保长 URL 或连续字符不会撑破布局 */
}

/* 应用 2：中文排版优化 */
.chinese-article {
  word-break: normal;
  /* 中文默认正常换行即可 */
}

/* 应用 3：日文排版 */
.japanese-text {
  word-break: keep-all;
  /* 日文通常希望在语义边界处换行 */
}
```

## 三、overflow-wrap —— 长单词处理

### 3.1 属性详解

`overflow-wrap`（原名 `word-wrap`）专门用于处理**过长单词或 URL** 的换行问题。

<div style="background-color: #fff3cd; border-left: 4px solid #ffc107; padding: 10px; margin: 10px 0;">
<strong>⚠️ 命名历史：</strong>该属性最初名为 <code>word-wrap</code>（IE 私有），后来在 CSS3 中标准化为 <code>overflow-wrap</code>。为了兼容性，建议同时声明两个属性。
</div>

| 值 | 行为 | 使用场景 |
|---|------|----------|
| `normal` | 只在正常断点处换行 | 默认值 |
| `break-word` | 必要时在单词内换行 | 长单词/URL 溢出时 |
| `anywhere` | 在任意字符处换行，影响文本对齐 | 紧急情况，极少使用 |

### 3.2 与 word-break 的区别

```css
/* 测试文本：Supercalifragilisticexpialidocious */

/* overflow-wrap: break-word */
.ow-break {
  overflow-wrap: break-word;
}
/* 行为：先尝试正常换行，实在不行才断开单词 */

/* word-break: break-all */
.wb-break {
  word-break: break-all;
}
/* 行为：直接在任意字符处换行，不考虑单词完整性 */
```

### 3.3 代码示例

```css
/* 兼容性写法 */
.url-wrapper {
  word-wrap: break-word;      /* 旧版浏览器 */
  overflow-wrap: break-word;  /* 标准属性 */
}

/* 实际效果 */
.container {
  width: 200px;
  overflow-wrap: break-word;
}

/* URL: https://example.com/very/long/path/that/exceeds/container/width */
/* 结果：
  https://example.com/very/long/path/
  that/exceeds/container/width
*/
```

### 3.4 实际应用

```css
/* 应用 1：处理用户生成内容 */
.user-content {
  overflow-wrap: break-word;
  /* 防止用户输入长 URL 撑破布局 */
}

/* 应用 2：移动端适配 */
.mobile-content {
  overflow-wrap: break-word;
  max-width: 100vw;
}

/* 应用 3：数据表格 */
.table-cell {
  overflow-wrap: break-word;
  min-width: 100px;
}
```

## 四、其他相关属性

### 4.1 line-break

控制标点符号的换行行为，主要针对 CJK 文本。

```css
line-break: auto | loose | normal | strict | anywhere;
```

| 值 | 说明 |
|---|------|
| `auto` | 默认值，使用规则断行 |
| `loose` | 宽松断行，最小行尾限制 |
| `normal` | 常规断行 |
| `strict` | 严格断行，避免在标点后换行 |
| `anywhere` | 类似 `word-break: anywhere` |

```css
/* 中文排版示例 */
.chinese-text {
  line-break: strict;
  /* 避免"。"、"，"等标点出现在行首 */
}
```

### 4.2 hyphens

控制连字符的使用（需要语言支持）。

```css
hyphens: none | manual | auto;
```

```css
.hyphenated {
  hyphens: auto;
  lang: en;  /* 需要指定语言 */
  /* 可能结果：exam-
              ple */
}
```

### 4.3 text-align-last

控制最后一行或强制换行行的对齐方式。

```css
.justified-text {
  text-align: justify;
  text-align-last: center;  /* 最后一行居中 */
}
```

## 五、属性对比总结

### 5.1 核心区别

| 属性 | 核心关注点 | 典型使用场景 | 是否影响空白符 |
|------|-----------|-------------|--------------|
| `white-space` | 空白符和换行符的处理 | 代码展示、诗歌、省略号 | ✅ 是 |
| `word-break` | 换行策略，是否允许断词 | 中文排版、防止溢出 | ❌ 否 |
| `overflow-wrap` | 长单词的溢出处理 | URL、长单词 | ❌ 否 |

### 5.2 选择流程图

```
开始
  │
  ├─→ 需要保留原始格式（空格/换行）？
  │     ├─ 是 → white-space: pre（代码块）
  │     └─ 否 → white-space: pre-wrap（格式化+自动换行）
  │
  ├─→ 需要单行显示（省略号）？
  │     ├─ 是 → white-space: nowrap + overflow: hidden
  │     └─ 否 → 继续判断
  │
  ├─→ 有长 URL 或连续字符？
  │     ├─ 是 → overflow-wrap: break-word
  │     └─ 否 → 继续判断
  │
  └─→ 中文排版需要特殊处理？
        ├─ 是 → word-break: keep-all（像英文一样）
        └─ 否 → 使用默认值即可
```

## 六、常见问题和解决方案

### 6.1 URL 溢出容器

```css
/* ❌ 问题 */
.url-container {
  width: 200px;
}
/* URL 会溢出容器 */

/* ✅ 解决方案 1：推荐 */
.url-container {
  overflow-wrap: break-word;
}

/* ✅ 解决方案 2：更强力 */
.url-container {
  word-break: break-all;
}

/* ✅ 解决方案 3：滚动 */
.url-container {
  overflow-x: auto;
  white-space: nowrap;
}
```

### 6.2 中文换行位置不当

```css
/* ❌ 问题：中文在字符间任意换行 */
.chinese {
  /* 这是一段文
     本，换行位
     置不当 */
}

/* ✅ 解决方案 */
.chinese {
  word-break: keep-all;
  /* 这是一段文本，换行位置更合理 */
}
```

### 6.3 代码块显示混乱

```css
/* ❌ 问题 */
code {
  /* 缩进和换行都丢失了 */
}

/* ✅ 解决方案 */
code {
  white-space: pre;
  /* 保留所有空格和换行 */
}

/* 或者带自动换行 */
code {
  white-space: pre-wrap;
  word-break: break-all;  /* 防止长代码溢出 */
}
```

### 6.4 文本省略号失效

```css
/* ❌ 问题：省略号不显示 */
.ellipsis {
  overflow: hidden;
  text-overflow: ellipsis;
  /* 没有设置 white-space */
}

/* ✅ 完整写法 */
.ellipsis {
  white-space: nowrap;      /* 必需 */
  overflow: hidden;          /* 必需 */
  text-overflow: ellipsis;   /* 必需 */
}
```

## 七、实战案例

### 7.1 响应式卡片

```css
.card {
  width: 300px;
  padding: 16px;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
}

.card-title {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;

  font-weight: bold;

  overflow-wrap: break-word;
  line-height: 1.6;

```

```css
.code-editor {
  white-space: pre;
  font-family: 'Consolas', 'Monaco', monospace;
  font-size: 14px;
  line-height: 1.5;
  tab-size: 4;
  overflow-x: auto;
  background: #1e1e1e;

```

### 7.3 多语言文本容器

```css
.multilingual-text {
  /* 通用设置 */
  overflow-wrap: break-word;
  line-break: auto;

  /* 中文特定优化 */
  &:lang(zh), &:lang(ja), &:lang(ko) {
    word-break: normal;
  }

  /* 英文特定优化 */
  &:lang(en) {
    word-break: keep-all;
  }
}
```
