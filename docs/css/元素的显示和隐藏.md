# CSS 元素的显示和隐藏

> 控制元素的可见性是 CSS 中的基础技能，不同方法有不同的使用场景和渲染行为。

## 一、方法对比总览

| 属性 | 是否占据空间 | 是否响应事件 | 是否可动画 | 典型场景 |
|------|--------------|--------------|------------|----------|
| `display: none` | ❌ | ❌ | ❌ | 完全隐藏、条件渲染 |
| `visibility: hidden` | ✅ | ❌ | ✅ | 临时隐藏、保留布局 |
| `opacity: 0` | ✅ | ✅ | ✅ | 渐变动画 |
| `position: absolute` + 偏移 | ✅ | ❌ | ✅ | 移出视口 |
| `clip-path` | ✅ | ❌ | ✅ | 裁剪隐藏 |
| `height: 0` / `width: 0` | 视情况 | ❌ | ✅ | 折叠动画 |

## 二、display: none

### 2.1 基本用法

```css
.element {
  display: none;
}
```

### 2.2 特性详解

**渲染行为**：

```html
<!-- DOM 结构 -->
<div class="container">
  <div class="box-a">A</div>
  <div class="box-b" style="display: none;">B</div>
  <div class="box-c">C</div>
</div>

/* B 元素不占据空间，A 和 C 相邻 */
```

```
渲染结果：
┌─────────┐ ┌─────────┐
│    A    │ │    C    │
└─────────┘ └─────────┘
```

**关键特性**：

| 特性 | 说明 |
|------|------|
| 空间占据 | 元素完全不占据页面空间 |
| 事件响应 | 不响应任何鼠标/键盘事件 |
| 可访问性 | 屏幕阅读器会忽略内容 |
| 过渡动画 | 无法直接过渡，需配合其他技巧 |
| 后代元素 | 所有后代元素同时隐藏 |
| 重排重绘 | 触发重排（reflow） |

### 2.3 实际应用

```css
/* 响应式隐藏 */
@media (max-width: 768px) {
  .sidebar {
    display: none;
  }
}

/* Tab 切换 */
.tab-content {
  display: none;
}

.tab-content.active {
  display: block;
}

/* 条件渲染 */
[hidden] {
  display: none;
}
```

```html
<!-- 使用 HTML5 hidden 属性 -->
<div hidden>此内容默认隐藏</div>
```

### 2.4 过渡技巧

`display` 无法直接过渡，可通过技巧实现：

```css
.fade-element {
  opacity: 0;
  visibility: hidden;
  transition: opacity 0.3s, visibility 0.3s;
}

.fade-element.visible {
  opacity: 1;
  visibility: visible;
}

/* 需要完全移除时 */
.fade-element.hidden {
  display: none;
}
```

```javascript
// 先做过渡动画，再移除 display
function hideElement(el) {
  el.classList.remove('visible');
  setTimeout(() => {
    el.classList.add('hidden');
  }, 300);
}
```

## 三、visibility: hidden

### 3.1 基本用法

```css
.element {
  visibility: hidden;
}

.element {
  visibility: visible;  /* 默认值 */
}

.element {
  visibility: collapse;  /* 表格相关 */
}
```

### 3.2 特性详解

**渲染行为**：

```html
<div class="container">
  <div class="box-a">A</div>
  <div class="box-b" style="visibility: hidden;">B</div>
  <div class="box-c">C</div>
</div>
```

```
渲染结果（B 占据空间但不可见）：
┌─────────┐ ┌─────────┐ ┌─────────┐
│    A    │ │ (空格)  │ │    C    │
└─────────┘ └─────────┘ └─────────┘
```

**与 display: none 对比**：

```css
/* 左图：display: none */
/* 右图：visibility: hidden */

.container {
  display: flex;
}

.item {
  width: 100px;
  height: 100px;
  background: lightblue;
  margin: 10px;
}

/* display: none - 其他元素会占据其空间 */
.item-hidden-1 {
  display: none;  /* 元素完全消失，flex 布局重新计算 */
}

/* visibility: hidden - 保留空间 */
.item-hidden-2 {
  visibility: hidden;  /* 元素不可见但占据原位置 */
}
```

**关键特性**：

| 特性 | 说明 |
|------|------|
| 空间占据 | 元素仍占据原有空间 |
| 事件响应 | 不响应事件 |
| 可访问性 | 屏幕阅读器默认读取 |
| 过渡动画 | 支持（visible ↔ hidden） |
| 后代元素 | 可通过 `visibility: visible` 覆盖 |
| 重排重绘 | 只触发重绘 |

### 3.3 实际应用

```css
/* 加载状态占位 */
.loading-placeholder {
  visibility: hidden;
}

/* 可访问性优化 */
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
  visibility: hidden; /* 确保视觉隐藏 */
}

/* 焦点时显示 */
.button:focus .tooltip {
  visibility: visible;
}

.tooltip {
  visibility: hidden;
  opacity: 0;
  transition: opacity 0.2s;
}
```

### 3.4 collapse 值

`collapse` 主要用于表格元素：

```css
/* 表格行列隐藏 */
table {
  border-collapse: collapse;
}

tr.hidden {
  visibility: collapse;  /* 行不显示，也不占据空间（类似 display:none） */
}

td.hidden {
  visibility: collapse;  /* 单元格不显示，同行其他单元格扩展 */
}
```

| 元素类型 | `visibility: collapse` 效果 |
|----------|----------------------------|
| 表格行/列 | 不显示且不占据空间（类似 display: none） |
| 其他元素 | 效果等同于 hidden |

## 四、opacity: 0

### 4.1 基本用法

```css
.element {
  opacity: 0;      /* 完全透明 */
  opacity: 0.5;    /* 半透明 */
  opacity: 1;      /* 完全不透明（默认） */
}
```

### 4.2 特性详解

**渲染行为**：

```html
<div class="container">
  <button>按钮 A</button>
  <button style="opacity: 0;">按钮 B（透明但可点击）</button>
  <button>按钮 C</button>
</div>
```

**关键特性**：

| 特性 | 说明 |
|------|------|
| 空间占据 | 元素占据原有空间 |
| 事件响应 | 响应事件（可点击） |
| 可访问性 | 屏幕阅读器正常读取 |
| 过渡动画 | 完美支持 |
| 后代元素 | 所有后代元素同时透明 |
| 重排重绘 | 只触发重绘 |

### 4.3 实际应用

```css
/* 淡入淡出动画 */
.fade-in {
  animation: fadeIn 0.5s ease-in;
}

@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

/* 悬停效果 */
.card {
  opacity: 0.8;
  transition: opacity 0.3s;
}

.card:hover {
  opacity: 1;
}

/* 禁用状态 */
.button:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}

/* 加载遮罩 */
.overlay {
  position: fixed;
  inset: 0;
  background: rgba(0, 0, 0, 0.5);
  opacity: 0;
  pointer-events: none;
  transition: opacity 0.3s;
}

.overlay.active {
  opacity: 1;
  pointer-events: auto;
}
```

### 4.4 与 pointer-events 结合

```css
/* 透明且不响应事件 */
.invisible {
  opacity: 0;
  pointer-events: none;
}

/* 仅视觉隐藏，保留可访问性 */
.visually-hidden {
  opacity: 0;
  position: absolute;
  pointer-events: none;
}
```

## 五、position 方案

### 5.1 移出视口

```css
/* 移到屏幕外 */
.off-screen {
  position: absolute;
  left: -9999px;
}

/* 移到视口上方 */
.above-viewport {
  position: absolute;
  top: -100vh;
}

/* 移到视口外 */
.out-of-flow {
  position: absolute;
  left: 100%;
}
```

### 5.2 特性详解

| 特性 | 说明 |
|------|------|
| 空间占据 | 脱离文档流，不占据原空间 |
| 事件响应 | 视口外不响应 |
| 可访问性 | 屏幕阅读器可读取 |
| 过渡动画 | 支持 transform 过渡 |
| 重排重绘 | 触发重排 |

### 5.3 实际应用

```css
/* 抽屉菜单 */
.drawer {
  position: fixed;
  top: 0;
  left: 0;
  height: 100%;
  width: 300px;
  transform: translateX(-100%);
  transition: transform 0.3s ease;
}

.drawer.open {
  transform: translateX(0);
}

/* 滑入通知 */
.notification {
  position: fixed;
  top: -100px;
  left: 50%;
  transform: translateX(-50%);
  transition: top 0.3s ease;
}

.notification.show {
  top: 20px;
}
```

## 六、clip-path 方案

### 6.1 基本用法

```css
.element {
  clip-path: circle(0%);  /* 完全裁剪 */
}

.element {
  clip-path: inset(50%);  /* 从四边向内裁剪 50% */
}

.element {
  clip-path: polygon(0 0, 0 0, 0 0);  /* 无区域 */
}
```

### 6.2 特性详解

| 特性 | 说明 |
|------|------|
| 空间占据 | 占据原有空间 |
| 事件响应 | 裁剪区域不响应 |
| 可访问性 | 可读取 |
| 过渡动画 | 部分支持（需同形状函数） |

### 6.3 实际应用

```css
/* 揭示动画 */
.reveal {
  clip-path: inset(0 100% 0 0);
  transition: clip-path 0.6s ease;
}

.reveal.active {
  clip-path: inset(0 0 0 0);
}

/* 圆形展开 */
.circle-expand {
  clip-path: circle(0% at 50% 50%);
  transition: clip-path 0.5s ease;
}

.circle-expand.active {
  clip-path: circle(100% at 50% 50%);
}
```

## 七、尺寸归零方案

### 7.1 基本用法

```css
/* 高度归零 */
.collapse-vertical {
  height: 0;
  overflow: hidden;
}

/* 宽度归零 */
.collapse-horizontal {
  width: 0;
  overflow: hidden;
  white-space: nowrap;
}

/* 完全折叠 */
.collapse-all {
  width: 0;
  height: 0;
  overflow: hidden;
  padding: 0;
  border: 0;
}
```

### 7.2 特性详解

| 特性 | 说明 |
|------|------|
| 空间占据 | 尺寸归零，但仍可能占 margin/border 空间 |
| 事件响应 | 不响应 |
| 过渡动画 | 支持 height/width 过渡 |
| 重排重绘 | 触发重排 |

### 7.3 实际应用

```css
/* 手风琴效果 */
.accordion-content {
  height: 0;
  overflow: hidden;
  transition: height 0.3s ease;
}

.accordion-item.active .accordion-content {
  height: var(--content-height);
}
```

```javascript
// 需要动态计算高度
const content = item.querySelector('.accordion-content');
content.style.setProperty('--content-height', content.scrollHeight + 'px');
```

## 八、可访问性隐藏

### 8.1 屏幕阅读器专用

```css
/* 视觉隐藏，但屏幕阅读器可读 */
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

/* 焦点时可见 */
.sr-only-focusable:focus {
  position: static;
  width: auto;
  height: auto;
  overflow: visible;
  clip: auto;
  white-space: normal;
}
```

### 8.2 实际应用

```html
<!-- 为图标按钮添加文字说明 -->
<button>
  <svg>...</svg>
  <span class="sr-only">关闭对话框</span>
</button>

<!-- 跳转到主内容 -->
<a href="#main" class="sr-only sr-only-focusable">跳转到主要内容</a>
```

## 九、场景选择指南

### 9.1 决策树

```
需要隐藏元素
    │
    ├─ 需要保留布局空间？
    │   ├─ 是 ──► visibility: hidden / opacity: 0
    │   └─ 否 ──► display: none / position 方案
    │
    ├─ 需要响应事件？
    │   ├─ 是 ──► opacity: 0
    │   └─ 否 ──► 其他方案
    │
    ├─ 需要过渡动画？
    │   ├─ 是 ──► opacity / transform / clip-path
    │   └─ 否 ──► display: none
    │
    └─ 需要屏幕阅读器读取？
        ├─ 是 ──► sr-only / opacity: 0
        └─ 否 ──► display: none / visibility: hidden
```

### 9.2 常见场景

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| Tab 切换 | `display: none` | 完全隐藏，不占空间 |
| 模态框 | `visibility: hidden` + `opacity` | 支持过渡动画 |
| 加载占位 | `visibility: hidden` | 保留空间，内容加载后显示 |
| 悬停提示 | `opacity: 0` + `:hover` | 平滑过渡 |
| 抽屉菜单 | `transform: translateX` | GPU 加速，性能好 |
| 手风琴 | `height: 0` + 过渡 | 折叠动画 |
| 可访问性标签 | `.sr-only` | 视觉隐藏，可读 |

## 十、性能考虑

### 10.1 重排 vs 重绘

```css
/* 触发重排（Reflow）- 性能开销大 */
.element {
  display: none;
  width: 0;
  height: 0;
  position: absolute;
}

/* 仅重绘（Repaint）- 性能开销小 */
.element {
  visibility: hidden;
  opacity: 0;
}

/* 合成层（Composite）- 性能最优 */
.element {
  transform: translateX(-100%);
  opacity: 0;
}
```

### 10.2 性能优化建议

```css
/* 动画优先使用 transform 和 opacity */
.animated-element {
  will-change: transform, opacity;
  transform: translateZ(0); /* 开启 GPU 加速 */
}

/* 避免频繁触发布局的操作 */
/* ❌ 不好的做法 */
element.style.width = '100px';
element.offsetHeight; /* 强制同步布局 */
element.style.height = '100px';

/* ✅ 好的做法 */
element.style.cssText = 'width: 100px; height: 100px';
```

## 十一、组合技巧

### 11.1 淡入淡出 + 移除

```css
.fade-enter {
  opacity: 0;
  visibility: visible;
}

.fade-enter-active {
  opacity: 1;
  transition: opacity 0.3s;
}

.fade-exit {
  opacity: 1;
  visibility: visible;
}

.fade-exit-active {
  opacity: 0;
  transition: opacity 0.3s;
}

.fade-exit-done {
  display: none;
}
```

### 11.2 无障碍组件

```css
/* 可展开的组件 */
.disclosure {
  display: none;
}

.disclosure[aria-expanded="true"] {
  display: block;
  animation: slideDown 0.3s ease;
}

@keyframes slideDown {
  from {
    opacity: 0;
    transform: translateY(-10px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

```html
<button aria-expanded="false" aria-controls="content">
  显示更多
</button>
<div id="content" class="disclosure">
  隐藏的内容
</div>
```

## 十二、浏览器兼容性

| 方案 | 兼容性 | 备注 |
|------|--------|------|
| `display: none` | IE6+ | 完全兼容 |
| `visibility: hidden` | IE6+ | 完全兼容 |
| `opacity` | IE9+ | IE8 使用 `filter: alpha()` |
| `clip-path` | IE 不支持 | 需要降级方案 |
| `transform` | IE10+ | IE9 使用 `-ms-` 前缀 |

```css
/* opacity 降级 */
.element {
  opacity: 0.5;
  filter: alpha(opacity=50); /* IE8 */
}
```
