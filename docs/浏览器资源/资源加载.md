# 浏览器资源加载机制

## 一、浏览器渲染流程

```
HTML 解析 → DOM 树
              ↓
CSS 解析 → CSSOM 树
              ↓
    DOM + CSSOM → 渲染树 (Render Tree)
              ↓
          布局 (Layout)
              ↓
          绘制 (Paint)
              ↓
          合成 (Composite)
```

## 二、关键资源 vs 非关键资源

| 资源类型 | 是否阻塞渲染 | 优先级 |
|---------|-------------|--------|
| HTML | ✅ 阻塞 | 最高 |
| CSS (同步) | ✅ 阻塞 | 高 |
| JS (同步) | ✅ 阻塞 | 高 |
| 图片 | ❌ 不阻塞 | 中 |
| 字体 | ⚠️ 可能阻塞 FOIT | 中 |
| async/defer JS | ❌ 不阻塞 | 中/低 |

## 三、CSS 加载

### 3.1 默认行为

```html
<!-- 同步加载，阻塞渲染 -->
<link href="style.css" rel="stylesheet">
```

**执行顺序：**
1. 遇到 `<link>` 开始下载 CSS
2. 继续解析 HTML
3. CSS 下载完成后解析成 CSSOM
4. **CSSOM 构建完成前，页面不会渲染**

### 3.2 CSS 不阻塞 HTML 解析，但阻塞渲染

```html
<!DOCTYPE html>
<html>
<head>
    <link href="large.css" rel="stylesheet">  <!-- 下载中... -->
</head>
<body>
    <h1>内容</h1>  <!-- HTML 继续解析 -->
    <!-- 但不会渲染，直到 CSS 下载完成 -->
</body>
</html>
```

### 3.3 媒体查询优化

```html
<!-- 打印时才加载 -->
<link href="print.css" rel="stylesheet" media="print">

<!-- 仅在大屏幕加载 -->
<link href="large.css" rel="stylesheet" media="(min-width: 1024px)">

<!-- 条件不匹配时不阻塞渲染 -->
```

## 四、JavaScript 加载

### 4.1 三种加载方式对比

| 方式 | HTML 解析 | 执行时机 | 顺序保证 | DOMContentLoaded |
|-----|----------|---------|---------|------------------|
| `<script>` | 阻塞 | 立即执行 | ✅ 按顺序 | 等待执行完成 |
| `defer` | 不阻塞 | HTML 解析后 | ✅ 按顺序 | 等待执行完成 |
| `async` | 不阻塞 | 下载完后 ASAP | ❌ 无序 | 不等待 |

### 4.2 默认行为（阻塞）

```html
<script src="app.js"></script>
<!-- HTML 解析暂停，等待脚本下载并执行 -->
```

### 4.3 defer - 推荐

```html
<script defer src="app1.js"></script>
<script defer src="app2.js"></script>

<script defer src="analytics.js"></script>

<!-- 特点：
     1. 不阻塞 HTML 解析
     2. 在 DOMContentLoaded 前执行
     3. 按顺序执行
-->
```

### 4.4 async - 独立脚本

```html
<script async src="analytics.js"></script>
<script async src="ads.js"></script>

<!-- 特点：
     1. 不阻塞 HTML 解析
     2. 下载完立即执行
     3. 不保证顺序
     4. 不依赖 DOM 的脚本使用
-->
```

### 4.5 使用场景

```html
<!-- ✅ defer: 依赖 DOM，需要按顺序 -->
<script defer src="jquery.js"></script>
<script defer src="app.js"></script>

<!-- ✅ async: 独立脚本，不依赖 DOM 和其他脚本 -->
<script async src="google-analytics.js"></script>
<script async src="hotjar.js"></script>

<!-- ✅ 内联脚本：需要立即执行的代码 -->
<script>
  // 关键 CSS 内联，防止 FOUC
  document.documentElement.style.visibility = 'hidden';
</script>
```

## 五、图片加载

### 5.1 默认行为

```html
<img src="photo.jpg" loading="lazy">
```

**特点：**
- 不阻塞 HTML 解析
- 不阻塞页面渲染
- 异步加载，并行下载

### 5.2 懒加载

```html
<!-- 浏览器原生懒加载 -->
<img src="photo.jpg" loading="lazy">

<!-- 等价于：图片进入视口附近时才下载 -->
```

### 5.3 响应式图片

```html
<img
  src="small.jpg"
  srcset="small.jpg 320w, medium.jpg 640w, large.jpg 1024w"
  sizes="(max-width: 640px) 100vw, 50vw"
  alt="响应式图片">
```

## 六、字体加载

### 6.1 默认行为 (FOIT)

```css
@font-face {
  font-family: 'CustomFont';
  src: url('font.woff2');
}

/* 默认：字体加载前文本不可见 (Flash of Invisible Text) */
```

### 6.2 font-display 优化

```css
@font-face {
  font-family: 'CustomFont';
  src: url('font.woff2');
  font-display: swap;  /* 立即显示回退字体，加载后切换 */
}

/* 值：
   auto    - 浏览器默认
   block   - 短暂隐藏，然后阻塞（约3秒）
   swap    - 立即显示回退，加载后替换 ✅ 推荐
   fallback - 极短隐藏，短暂回退，然后使用回退
   optional - 极短隐藏，允许使用回退到底
*/
```

## 七、资源加载优先级

### 7.1 浏览器优先级分类

```
Highest (最高)
  ├── 关键 CSS
  ├── 同步 JS
  └── 字体（已渲染的文本）
High (高)
  ├── preload 预加载资源
  └── 当前视口的图片
Medium (中)
  ├── defer/async JS
  ├── 非视口图片
  └── 音视频
Low (低)
  ├── prefetch 预取资源
  └── 预连接
```

### 7.2 手动控制优先级

```html
<!-- 提升优先级：关键资源 -->
<link rel="preload" href="critical.css" as="style">
<link rel="preload" href="critical.js" as="script">
<link rel="preload" href="font.woff2" as="font" crossorigin>

<!-- 降低优先级：未来可能用到的资源 -->
<link rel="prefetch" href="next-page.js">
<link rel="prefetch" href="lazy-image.jpg">

<!-- 预连接：提前建立连接 -->
<link rel="preconnect" href="https://api.example.com">
<link rel="dns-prefetch" href="https://cdn.example.com">
```

## 八、资源加载顺序最佳实践

### 8.1 head 中的顺序

```html
<head>
  <!-- 1. Meta 标签 -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width">

  <!-- 2. 预连接 -->
  <link rel="preconnect" href="https://cdn.example.com">

  <!-- 3. 关键 CSS 内联（首屏） -->
  <style>
    /* 首屏关键样式 */
  </style>

  <!-- 4. 预加载关键资源 -->
  <link rel="preload" href="main.css" as="style">
  <link rel="preload" href="main.js" as="script">

  <!-- 5. 非关键 CSS -->
  <link href="main.css" rel="stylesheet">

  <!-- 6. defer JS（放在 head 或 body 结尾都可以） -->
  <script defer src="main.js"></script>
</head>
```

### 8.2 body 中的顺序

```html
<body>
  <!-- 内容 -->
  <h1>Title</h1>

  <!-- 图片懒加载 -->
  <img src="image.jpg" loading="lazy">

  <!-- 独立脚本 async -->
  <script async src="analytics.js"></script>

  <!-- body 结尾：传统做法（现在用 defer 更好） -->
  <!-- <script src="main.js"></script> -->
</body>
```

## 九、常见问题

### 9.1 CSS 会阻塞 JS 执行吗？

```html
<link href="style.css" rel="stylesheet">
<script src="app.js"></script>

<!-- ✅ 是的：JS 执行需要等待 CSS 加载完成 -->
<!-- 原因：JS 可能操作 DOM 样式，需要 CSSOM 就绪 -->
```

### 9.2 defer 脚本能访问 DOM 吗？

```html
<script defer>
  console.log(document.querySelector('h1')); // ✅ 可以
  console.log(document.readyState); // "interactive" 或更高
</script>

<!-- defer 脚本在 DOM 构建完成后执行 -->
```

### 9.3 async 和 defer 可以混用吗？

```html
<script async src="analytics.js"></script>
<script defer src="app.js"></script>

<!-- ✅ 可以，但要注意：
     - async 脚本可能先于 defer 执行
     - 不要让它们之间有依赖关系
-->
```

## 十、性能优化检查清单

- [ ] 关键 CSS 内联到 HTML
- [ ] 非关键 CSS 异步加载
- [ ] script 使用 `defer`（有 DOM 依赖）
- [ ] script 使用 `async`（独立脚本）
- [ ] 图片使用 `loading="lazy"`
- [ ] 字体使用 `font-display: swap`
- [ ] 关键资源使用 `preload`
- [ ] 第三方域名使用 `preconnect`
- [ ] 响应式图片使用 `srcset`
- [ ] 避免重复加载相同资源
