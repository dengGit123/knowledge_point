### 一、概述

路由传参是将数据从 URL 中传递给组件的方式。Vue Router 提供了多种传参途径：**路径参数（params）**、**查询参数（query）**、**hash**，以及通过 **props** 解耦传参。

> 📖 [Vue Router - 路由传参](https://router.vuejs.org/zh/guide/essentials/passing-props.html)

### 二、路径参数（params）

通过 URL 路径中的动态段传递：

```js
// 路由配置
{ path: '/user/:id', name: 'user', component: UserView }
```

```js
// 传递
router.push({ name: 'user', params: { id: 123 } })
// URL：/user/123
```

```vue
<!-- 接收 -->
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
console.log(route.params.id) // '123'（字符串）
</script>
```

> ⚠️ **注意：** `params` 只能配合 `name` 使用，不能配合 `path`。

```js
// ❌ params 会被忽略
router.push({ path: '/user', params: { id: 123 } })

// ✅ 使用 name
router.push({ name: 'user', params: { id: 123 } })

// ✅ 或直接写在 path 里
router.push('/user/123')
```

### 三、查询参数（query）

通过 URL 中 `?` 后面的部分传递：

```js
// 传递
router.push({ path: '/search', query: { q: 'vue', page: 1 } })
// URL：/search?q=vue&page=1
```

```vue
<!-- 接收 -->
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
console.log(route.query.q)    // 'vue'
console.log(route.query.page) // '1'（字符串）
</script>
```

```vue
<!-- RouterLink 传递 query -->
<RouterLink :to="{ path: '/search', query: { q: 'vue' } }">搜索</RouterLink>
```

### 四、Hash 传参

通过 URL 中 `#` 后面的锚点传递：

```js
router.push({ path: '/docs', hash: '#section-1' })
// URL：/docs#section-1
```

```vue
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
console.log(route.hash) // '#section-1'
</script>
```

### 五、Props 传参（推荐）

默认通过 `route.params` 获取参数，组件与路由耦合。启用 `props: true` 后，参数会作为组件的 `props` 传入，组件更易测试和复用。

#### 1. 布尔模式

```js
// 路由配置
{ path: '/user/:id', component: UserView, props: true }
```

```vue
<!-- UserView.vue -->
<script setup>
// ✅ 通过 props 接收，不依赖 useRoute
defineProps({
  id: String,
})
</script>

<template>
  <p>用户 ID：{{ id }}</p>
</template>
```

#### 2. 函数模式

灵活转换 query 或其他数据为 props：

```js
{
  path: '/search',
  component: SearchView,
  props: (route) => ({
    keyword: route.query.q,
    page: Number(route.query.page) || 1,
  }),
}
```

#### 3. 对象模式

传入固定的额外 props：

```js
{
  path: '/promo',
  component: PromoView,
  props: { showBanner: true, title: '限时优惠' },
}
```

### 六、三种传参方式对比

| 方式 | URL 表现 | 获取方式 | 适用场景 |
| :--: | :--: | :--: | :--: |
| **params** | `/user/:id` | `route.params.id` | 标识资源（用户 ID、文章 ID） |
| **query** | `/search?q=vue` | `route.query.q` | 过滤、搜索、分页 |
| **hash** | `/docs#section` | `route.hash` | 锚点定位 |
| **props** | 同 params | 组件 `defineProps` | 解耦组件与路由 ⭐ |

### 七、注意事项

```js
// ⚠️ 所有参数值都是字符串
route.params.id    // '123'
route.query.page   // '2'

// 需要手动转换类型
const id = Number(route.params.id)
const page = parseInt(route.query.page) || 1
```
