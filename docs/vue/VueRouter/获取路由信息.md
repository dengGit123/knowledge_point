### 一、概述

在组件中可以通过 `useRoute()` 获取**当前路由信息**，包括路径、参数、查询字符串、元信息等。这是组件感知当前 URL 状态的主要方式。

> 📖 [Vue Router - useRoute](https://router.vuejs.org/zh/api/#useroute)

### 二、基本用法

```vue
<script setup>
import { useRoute } from 'vue-router'

const route = useRoute()

console.log(route.path)     // '/user/123'
console.log(route.fullPath) // '/user/123?tab=posts#section'
console.log(route.name)     // 'user'
console.log(route.params)   // { id: '123' }
console.log(route.query)    // { tab: 'posts' }
console.log(route.hash)     // '#section'
console.log(route.meta)     // { requiresAuth: true }
</script>
```

### 三、route 对象属性

| 属性 | 类型 | 说明 | 示例 |
| :--: | :--: | :--: | :--: |
| `path` | `string` | 当前路径（不含 query 和 hash） | `/user/123` |
| `fullPath` | `string` | 完整路径（含 query 和 hash） | `/user/123?tab=posts#top` |
| `name` | `string/undefined` | 路由名称 | `'user'` |
| `params` | `object` | 动态路径参数 | `{ id: '123' }` |
| `query` | `object` | 查询参数 | `{ tab: 'posts' }` |
| `hash` | `string` | URL 哈希值 | `'#top'` |
| `meta` | `object` | 路由元信息（自定义数据） | `{ requiresAuth: true }` |
| `matched` | `array` | 匹配的路由记录数组 | 嵌套路由时有多个 |
| `redirectedFrom` | `string/undefined` | 重定向前的路径 | `'/home'` |

### 四、监听路由变化

从 `/user/1` 导航到 `/user/2` 时，组件被复用不会重建，需要用 `watch` 监听变化：

```vue
<script setup>
import { useRoute } from 'vue-router'
import { watch, ref } from 'vue'

const route = useRoute()
const user = ref(null)

// ✅ 监听参数变化，重新加载数据

  },
  { immediate: true }
)

// 监听 query 变化
watch(
  () => route.query.page,
  (newPage) => {
    console.log('翻页:', newPage)
  }
)
</script>
```

### 五、注意事项

```js
// ⚠️ params 和 query 的值始终是字符串
route.params.id     // '123'（不是数字 123）
route.query.page    // '2'（不是数字 2）

// 需要手动转换
const id = Number(route.params.id)
const page = parseInt(route.query.page) || 1
```

```js
// ⚠️ route 是只读的，不能直接修改
route.params.id = '456'  // ❌ 无效

// ✅ 通过 router.push 改变路由
router.push({ name: 'user', params: { id: '456' } })
```
