### 一、概述

导航守卫（Navigation Guards）是路由跳转的**拦截机制**，可以在跳转前、后、失败时执行自定义逻辑。常用于权限校验、数据预加载、页面标题设置、防止数据丢失等。

简单理解：**导航守卫就是路由跳转的"安检站"——你可以决定放行、拦截、或重定向。**

> 📖 [Vue Router - 导航守卫](https://router.vuejs.org/zh/guide/advanced/navigation-guards.html)

### 二、三层守卫分类

```
┌─────────────────────────────────────────────┐
│  全局守卫（对所有路由跳转生效）                  │
│  beforeEach / afterEach / beforeResolve       │
├─────────────────────────────────────────────┤
│  路由独享守卫（只对该条路由生效）                │
│  beforeEnter                                  │
├─────────────────────────────────────────────┤
│  组件内守卫（只对该组件生效）                    │
│  onBeforeRouteUpdate / onBeforeRouteLeave     │
└─────────────────────────────────────────────┘
```

### 三、守卫参数与返回值

#### 参数

```js
(to, from) => {
  // to：目标路由对象
  //   to.path / to.name / to.params / to.query / to.meta / to.matched

  // from：当前路由对象（即将离开的路由）
}
```

#### 返回值

| 返回值 | 效果 |
| :--: | :--: |
| `undefined` / `true` | ✅ 放行 |
| `false` | ❌ 取消导航 |
| `'/login'` 或 `{ name: 'login' }` | 🔄 重定向 |
| `new Error()` | ❌ 取消导航，错误被 `router.onError` 捕获 |

### 四、全局守卫

#### 1. beforeEach —— 全局前置守卫（最常用）

每次导航**之前**执行，适合做权限校验：

```js
router.beforeEach((to, from) => {
  if (to.meta.requiresAuth && !isLoggedIn()) {
    return { name: 'login', query: { redirect: to.fullPath } }
  }
})
```

#### 2. afterEach —— 全局后置钩子

导航**完成之后**执行，无法修改导航结果，适合设置标题、打日志：

```js
router.afterEach((to, from) => {
  document.title = to.meta.title || '默认标题'
})
```

#### 3. beforeResolve —— 全局解析守卫

所有组件内守卫和异步路由组件**解析完成后**、导航**确认之前**执行：

```js
router.beforeResolve(async (to) => {
  for (const record of to.matched) {
    if (record.meta.loadData) {
      await record.meta.loadData()
    }
  }
})
```

### 五、路由独享守卫（beforeEnter）

写在**路由配置**中，只对该条路由生效：

```js
const routes = [
  {
    path: '/admin',
    component: AdminView,
    beforeEnter: (to, from) => {
      if (!isAdmin()) return { name: 'forbidden' }
    },
  },
  // 支持数组形式（按顺序执行）
  {
    path: '/article/:id',
    component: ArticleView,
    beforeEnter: [checkAuth, checkArticleExists],
  },
]
```

### 六、组件内守卫

直接写在组件中，与组件状态相关：

```vue
<script setup>
import { onBeforeRouteLeave, onBeforeRouteUpdate } from 'vue-router'

// 离开组件时触发
onBeforeRouteLeave((to, from) => {
  if (hasUnsavedChanges.value) {
    const answer = window.confirm('有未保存的更改，确定离开吗？')
    if (!answer) return false
  }
})

// 同路由参数变化时触发（如 /user/1 → /user/2）
onBeforeRouteUpdate(async (to, from) => {
  userData.value = await fetchUser(to.params.id)
})
</script>
```

| 守卫 | 触发时机 | 常见用途 |
| :--: | :--: | :--: |
| `onBeforeRouteLeave` | 离开组件前 | 防止未保存数据丢失 |
| `onBeforeRouteUpdate` | 同路由参数变化 | 根据新参数重载数据 |

### 七、完整导航解析流程

```
 1. 导航被触发
 2. 调用失活组件的 onBeforeRouteLeave
 3. 调用全局 beforeEach
 4. 调用重用组件的 onBeforeRouteUpdate
 5. 调用路由配置中的 beforeEnter
 6. 解析异步路由组件
 7. 调用被激活组件的 onBeforeRouteEnter（组合式 API 中不可用）
 8. 调用全局 beforeResolve
 9. 导航确认
10. 调用全局 afterEach
11. DOM 更新
```

### 八、常见问题

#### 1. 避免无限重定向

```js
// ❌ 如果 /login 也标记了 requiresAuth → 死循环
router.beforeEach((to) => {
  if (to.meta.requiresAuth && !isLoggedIn()) {
    return '/login'
  }
})

// ✅ 排除登录页
router.beforeEach((to) => {
  if (to.meta.requiresAuth && !isLoggedIn() && to.name !== 'login') {
    return { name: 'login' }
  }
})
```

#### 2. 守卫可以是异步的

```js
router.beforeEach(async (to) => {
  const userStore = useUserStore()
  if (!userStore.userInfo) {
    await userStore.fetchUser()
  }
  if (to.meta.requiresAuth && !userStore.isLoggedIn) {
    return '/login'
  }
})
```

### 九、总结

| 守卫 | 写在哪里 | 作用范围 | 最常用场景 |
| :--: | :--: | :--: | :--: |
| `beforeEach` | router 文件 | 全局 | 权限校验 |
| `afterEach` | router 文件 | 全局 | 页面标题 |
| `beforeResolve` | router 文件 | 全局 | 数据预加载确认 |
| `beforeEnter` | 路由配置 | 单条路由 | 特定页面检查 |
| `onBeforeRouteUpdate` | 组件内 | 当前组件 | 参数变化重载 |
| `onBeforeRouteLeave` | 组件内 | 当前组件 | 防止数据丢失 |
