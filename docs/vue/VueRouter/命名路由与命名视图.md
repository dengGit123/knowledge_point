### 一、概述

**命名路由**给路由起名字，通过名字导航而不是硬编码路径。**命名视图**允许一个路由同时渲染多个 `<RouterView>`，适合复杂布局场景。

> 📖 [Vue Router - 命名路由](https://router.vuejs.org/zh/guide/essentials/named-routes.html)

### 二、命名路由

#### 1. 定义

```js
const routes = [
  { path: '/user/:id', name: 'user', component: UserView },
  { path: '/settings', name: 'settings', component: SettingsView },
]
```

#### 2. 通过名称导航

```js
// 编程式导航
router.push({ name: 'user', params: { id: 1 } })

// 声明式导航
```

```vue
<RouterLink :to="{ name: 'user', params: { id: 1 } }">用户</RouterLink>
```

> 💡 **提示：** 推荐使用命名路由。如果以后改了 `path`，所有通过 `name` 导航的代码都不需要修改。

### 三、命名视图

一个路由同时渲染多个组件到不同的 `<RouterView>` 插槽中：

#### 1. 路由配置（用 components 复数形式）

```js
const routes = [
  {
    path: '/layout',
    components: {
      default: MainContent,    // 默认插槽 <RouterView />
      sidebar: SideBar,        // <RouterView name="sidebar" />
      footer: FooterContent,   // <RouterView name="footer" />
    },
  },
]
```

#### 2. 模板中使用

```vue
<template>
  <div>
    <RouterView />                   <!-- 渲染 MainContent -->
    <RouterView name="sidebar" />    <!-- 渲染 SideBar -->
    <RouterView name="footer" />     <!-- 渲染 FooterContent -->
  </div>
</template>
```

### 四、命名视图 + 嵌套路由

```js
const routes = [
  {
    path: '/dashboard',
    components: {
      default: DashboardMain,
      sidebar: DashboardSidebar,
    },
    children: [
      { path: '', component: DashboardHome },
      { path: 'analytics', component: DashboardAnalytics },
    ],
  },
]
```

```vue
<!-- DashboardLayout.vue -->
<template>
  <div>
    <RouterView name="sidebar" />  <!-- 侧边栏（固定） -->
    <main>
      <RouterView />               <!-- 子路由内容（切换） -->
    </main>
  </div>
</template>
```

### 五、对比

| 概念 | 配置字段 | 用途 |
| :--: | :--: | :--: |
| 命名路由 | `name: 'xxx'` | 通过名字导航，避免硬编码路径 |
| 命名视图 | `components: { default, xxx }` | 一个路由渲染多个组件到不同位置 |
