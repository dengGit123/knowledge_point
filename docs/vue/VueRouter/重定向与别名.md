### 一、概述

**重定向（redirect）** 让用户访问某个路径时自动跳转到另一个路径，URL 会变。**别名（alias）** 给路由一个"外号"，通过别名也能访问同一组件，URL 不变。

> 📖 [Vue Router - 重定向和别名](https://router.vuejs.org/zh/guide/essentials/redirect-and-alias.html)

### 二、重定向（redirect）

访问旧路径时自动跳转到新路径，**URL 会改变**：

```js
const routes = [
  // 方式 1：重定向到路径
  { path: '/home', redirect: '/' },

  // 方式 2：重定向到命名路由
  { path: '/home', redirect: { name: 'home' } },

  // 方式 3：函数形式（动态重定向）
  {
    path: '/search/:category',
    redirect: (to) => {
      return { path: '/products', query: { cat: to.params.category } }
    },
  },
]
```

> ⚠️ **注意：** 导航守卫不会应用在重定向路由上，而是应用在最终的目标路由上。

### 三、别名（alias）

给路由一个"外号"，通过别名访问时**URL 不变**，直接渲染对应组件：

```js
const routes = [
  // 访问 / 或 /home 都渲染 HomeView，URL 保持用户输入的
  { path: '/', component: HomeView, alias: '/home' },

  // 多个别名
  { path: '/user/:id', component: UserView, alias: ['/u/:id', '/profile/:id'] },
]
```

### 四、对比

| 对比项 | `redirect` | `alias` |
| :--: | :--: | :--: |
| URL 是否变化 | ✅ 变为目标 URL | ❌ 保持输入的 URL |
| 是否产生新导航记录 | ✅ | ❌ |
| 适用场景 | 旧路径迁移到新路径 | 同一页面的多个入口路径 |

### 五、常见用法

```js
const routes = [
  // 旧路径迁移：用户收藏的旧链接不会失效
  { path: '/old-dashboard', redirect: '/dashboard' },

  // 默认子路由重定向
  {
    path: '/dashboard',
    component: DashboardLayout,
    children: [
      { path: '', redirect: 'overview' },
      { path: 'overview', component: OverviewView },
    ],
  },
]
```
