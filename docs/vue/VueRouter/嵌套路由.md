### 一、概述

嵌套路由允许在组件内部再嵌套 `<RouterView>`，实现**页面中的局部区域切换**。常用于后台管理系统的布局结构：侧边栏固定，右侧内容区随路由变化。

> 📖 [Vue Router - 嵌套路由](https://router.vuejs.org/zh/guide/essentials/nested-routes.html)

### 二、基本用法

#### 1. 路由配置

使用 `children` 定义子路由，子路由的 `path` 会**自动拼接父路由的 path**：

```js
const routes = [
  {
    path: '/dashboard',
    component: DashboardLayout,
    children: [
      // /dashboard（空路径表示父路径本身）
      { path: '', name: 'dashboard-home', component: DashboardHome },
      // /dashboard/settings
      { path: 'settings', name: 'dashboard-settings', component: SettingsView },
      // /dashboard/profile
      { path: 'profile', name: 'dashboard-profile', component: ProfileView },
    ],
  },
]
```

> ⚠️ **注意：** 子路由的 `path` **不要**以 `/` 开头。`'settings'` 会拼接成 `/dashboard/settings`，而 `'/settings'` 会变成绝对路径 `/settings`。

#### 2. 父组件提供布局

```vue
<!-- DashboardLayout.vue -->
<template>
  <div class="dashboard">
    <aside class="sidebar">
      <RouterLink to="/dashboard">概览</RouterLink>
      <RouterLink to="/dashboard/settings">设置</RouterLink>
      <RouterLink to="/dashboard/profile">个人资料</RouterLink>
    </aside>
    <main class="content">
      <!-- 子路由组件渲染在这里 -->
      <RouterView />
    </main>
  </div>
</template>
```

### 三、多层嵌套

路由可以无限层级嵌套：

```js
const routes = [
  {
    path: '/dashboard',
    component: DashboardLayout,
    children: [
      { path: '', component: DashboardHome },
      {
        path: 'settings',
        component: SettingsLayout,
        children: [
          // /dashboard/settings/general
          { path: 'general', component: GeneralSettings },
          // /dashboard/settings/security
          { path: 'security', component: SecuritySettings },
        ],
      },
    ],
  },
]
```

```
/dashboard
  └── DashboardLayout（第一层 RouterView）
        ├── DashboardHome          (/dashboard)
        └── SettingsLayout（第二层 RouterView）
              ├── GeneralSettings  (/dashboard/settings/general)
              └── SecuritySettings (/dashboard/settings/security)
```

### 四、子路由访问父级参数

子路由组件可以直接访问父路由的动态参数：

```js
const routes = [
  {
    path: '/user/:id',
    component: UserLayout,
    children: [
      { path: 'posts', component: UserPosts },    // /user/123/posts
      { path: 'friends', component: UserFriends }, // /user/123/friends
    ],
  },
]
```

```vue
<!-- UserPosts.vue -->
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()

// 直接获取父路由的参数
console.log(route.params.id) // '123'
</script>
```

### 五、空子路由的用途

`path: ''` 表示匹配父路径本身，用于给父路由一个默认内容：

```js
{
  path: '/dashboard',
  component: DashboardLayout,
  children: [
    // 访问 /dashboard 时，RouterView 默认渲染 DashboardHome
    { path: '', component: DashboardHome },
    { path: 'settings', component: SettingsView },
  ],
}
```

### 六、注意事项

```js
// ❌ 子路由 path 不要加 /
children: [
  { path: '/settings', component: SettingsView }, // 变成绝对路径 /settings
]

// ✅ 子路由 path 不加 /
children: [
  { path: 'settings', component: SettingsView }, // 拼接为 /dashboard/settings
]
```

```js
// ⚠️ 如果没有匹配的子路由，RouterView 什么都不渲染
// 可以用空路径子路由提供默认内容
children: [
  { path: '', component: DefaultContent },
]
```
