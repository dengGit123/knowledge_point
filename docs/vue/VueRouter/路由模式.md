### 一、概述

Vue Router 支持三种路由模式，决定了 URL 的**外观形式**和**工作原理**。选择合适的模式会影响 URL 是否美观、是否需要服务端配置。

> 📖 [Vue Router - 不同历史模式](https://router.vuejs.org/zh/guide/essentials/history-mode.html)

### 二、三种模式对比

| 模式 | URL 样式 | 创建方式 | 需要服务端配置 |
| :--: | :--: | :--: | :--: |
| **History** | `/user/profile` | `createWebHistory()` | ✅ 需要 |
| **Hash** | `/#/user/profile` | `createWebHashHistory()` | ❌ 不需要 |
| **Memory** | URL 不变 | `createMemoryHistory()` | — |

### 三、History 模式（推荐）

URL 干净无 `#`，是最推荐的模式：

```js
import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(),
  routes,
})
```

**原理：** 使用 HTML5 的 `history.pushState()` API 来管理 URL，不会触发页面刷新。

**服务端配置：** 因为 SPA 只有一个 `index.html`，用户直接访问 `/user/profile` 或刷新页面时，服务端需要**将所有请求回退到 `index.html`**，否则会 404。

```js
// vite.config.js（开发环境）
export default defineConfig({
  server: {
    historyApiFallback: true,
  },
})
```

```nginx
# Nginx 配置（生产环境）
location / {
  try_files $uri $uri/ /index.html;
}
```

### 四、Hash 模式

URL 中带 `#`，兼容性最好，不需要服务端配置：

```js
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes,
})
```

**原理：** 使用 URL 的 hash（`#` 后面的部分）来模拟完整路径。hash 变化不会触发页面刷新，服务端也不会收到 `#` 后面的内容。

### 五、Memory 模式

URL 完全不变，路由状态保存在内存中，主要用于 **SSR（服务端渲染）**场景：

```js
import { createRouter, createMemoryHistory } from 'vue-router'

const router = createRouter({
  history: createMemoryHistory(),
  routes,
})
```

### 六、如何选择

| 场景 | 推荐模式 |
| :--: | :--: |
| 正常 Web 项目 | **History**（URL 干净） |
| 没有服务端配置权限 | **Hash**（无需配置） |
| SSR / 测试环境 | **Memory** |
