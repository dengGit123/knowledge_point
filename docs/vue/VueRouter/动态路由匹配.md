### 一、概述

动态路由匹配允许在 URL 路径中使用**参数变量**，让一条路由规则匹配多种 URL。比如 `/user/1` 和 `/user/2` 都匹配 `/user/:id`，其中 `:id` 是动态参数。

> 📖 [Vue Router - 动态路由匹配](https://router.vuejs.org/zh/guide/essentials/dynamic-matching.html)

### 二、基本参数匹配

#### 1. 必需参数（`:param`）

```js
const routes = [
  // 匹配 /user/1、/user/abc、/user/anything
  { path: '/user/:id', component: UserView },

  // 多个参数
  { path: '/user/:userId/post/:postId', component: PostView },
]
```

```vue
<!-- UserView.vue -->
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
console.log(route.params.id) // '123'（始终是字符串）
</script>
```

#### 2. 可选参数（`:param?`）

```js
const routes = [
  // 匹配 /user 和 /user/1
  { path: '/user/:id?', component: UserView },

  // 多个可选参数
  { path: '/category/:type?/:sub?', component: CategoryView },
]
```

#### 3. 可重复参数（`:param+` 和 `:param*`）

```js
const routes = [
  // :chapters+（必需，匹配 1 个或多个段）
  // /book/a         → { chapters: ['a'] }
  // /book/a/b/c     → { chapters: ['a', 'b', 'c'] }
  { path: '/book/:chapters+', component: BookView },

  // :chapters*（可选，匹配 0 个或多个段）
  // /book           → { chapters: [] }
  // /book/a         → { chapters: ['a'] }
  // /book/a/b       → { chapters: ['a', 'b'] }
  { path: '/book/:chapters*', component: BookView },
]
```

### 三、参数正则校验

通过括号内写正则表达式来限制参数格式：

```js
const routes = [
  // 只匹配数字：/user/1 ✅  /user/abc ❌
  { path: '/user/:id(\\d+)', component: UserView },

  // 只匹配特定值
  { path: '/:lang(en|zh|ja)', component: HomeView },

  // 可重复 + 正则
  // 匹配 /files/jpg、/files/jpg/png、/files/jpg/png/gif
  { path: '/files/:ext(\\w+)+', component: FilesView },
]
```

### 四、参数语法汇总

| 语法 | 匹配规则 | 示例 |
| :--: | :--: | :--: |
| `:param` | 必需，匹配 1 段 | `/user/:id` → `/user/123` |
| `:param?` | 可选，匹配 0-1 段 | `/user/:id?` → `/user`、`/user/123` |
| `:param+` | 必需，匹配 1+ 段 | `/book/:ch+` → `/book/a`、`/book/a/b` |
| `:param*` | 可选，匹配 0+ 段 | `/book/:ch*` → `/book`、`/book/a` |
| `:param(\\d+)` | 正则校验 | `/user/:id(\\d+)` → `/user/1` |

### 五、捕获 404（未匹配路由）

```js
const routes = [
  { path: '/', component: HomeView },
  { path: '/user/:id', component: UserView },

  // ⚠️ 必须放在最后，匹配所有未命中的路径
  { path: '/:pathMatch(.*)*', name: 'not-found', component: NotFoundView },
]
```

> 💡 **提示：** `/:pathMatch(.*)*` 中的 `(.*)` 匹配任意字符，`*` 表示可重复，组合起来匹配任何路径。

### 六、组件复用问题

从 `/user/1` 导航到 `/user/2` 时，因为渲染的是同一个组件，Vue 会**复用组件实例**而不会销毁重建：

```vue
<script setup>
import { useRoute } from 'vue-router'
import { watch, ref } from 'vue'

const route = useRoute()
const user = ref(null)

// ❌ 只在组件首次创建时执行，参数变化不会重新执行
fetchUser(route.params.id)

// ✅ 用 watch 监听参数变化
watch(
  () => route.params.id,
  (newId) => {
    if (newId) fetchUser(newId)
  },
  { immediate: true }
)
</script>
```

### 七、优先级规则

路由按**声明顺序**匹配，先声明的优先：

```js
const routes = [
  // ✅ 精确匹配优先
  { path: '/user/profile', component: ProfileView },
  // 动态匹配在后
  { path: '/user/:id', component: UserView },
  // 404 兜底放在最后
  { path: '/:pathMatch(.*)*', component: NotFoundView },
]
```
