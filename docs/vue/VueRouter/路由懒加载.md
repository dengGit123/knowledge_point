### 一、概述

路由懒加载让组件在**首次被访问时才下载**，而不是在应用启动时就全部加载。这样可以减小首屏 JS 体积，加快页面首次打开速度。

> 📖 [Vue Router - 路由懒加载](https://router.vuejs.org/zh/guide/advanced/lazy-loading.html)

### 二、静态导入 vs 懒加载

```js
// ❌ 静态导入：所有组件打包到主 bundle，首屏加载所有代码
import HomeView from '@/views/HomeView.vue'
import AboutView from '@/views/AboutView.vue'

const routes = [
  { path: '/', component: HomeView },
  { path: '/about', component: AboutView },
]
```

```js
// ✅ 懒加载：每个组件生成单独的 chunk 文件，访问时才下载
const routes = [
  { path: '/', component: () => import('@/views/HomeView.vue') },
  { path: '/about', component: () => import('@/views/AboutView.vue') },
  { path: '/user/:id', component: () => import('@/views/UserView.vue') },
]
```

### 三、工作原理

`() => import(...)` 是 ES 模块的**动态导入**语法：

```
构建时：
  main.js          → main.[hash].js（包含 Router、Pinia 等核心代码）
  HomeView.vue     → HomeView.[hash].js（独立 chunk）
  AboutView.vue    → AboutView.[hash].js（独立 chunk）
  UserView.vue     → UserView.[hash].js（独立 chunk）

用户访问 /about 时：
  浏览器下载 main.js + AboutView.js
  访问 /user/1 时才下载 UserView.js
```

### 四、分组打包

把多个路由组件打包到同一个 chunk 文件中：

```js
// Webpack 的魔法注释（Vite 也支持）
const routes = [
  {
    path: '/admin',
    component: () => import(/* webpackChunkName: "admin" */ '@/views/AdminView.vue'),
  },
  {
    path: '/admin/users',
    component: () => import(/* webpackChunkName: "admin" */ '@/views/AdminUsers.vue'),
  },
  {
    path: '/admin/settings',
    component: () => import(/* webpackChunkName: "admin" */ '@/views/AdminSettings.vue'),
  },
]
// 以上三个组件会打包到同一个 admin.[hash].js 文件中
```

### 五、推荐做法

```js
// 首页等核心页面也可以懒加载（Vite 构建优化通常足够好）
const routes = [
  { path: '/', component: () => import('@/views/HomeView.vue') },
  { path: '/about', component: () => import('@/views/AboutView.vue') },
  { path: '/user/:id', component: () => import('@/views/UserView.vue') },

  // 嵌套路由也可以懒加载
  {
    path: '/dashboard',
    component: () => import('@/views/DashboardLayout.vue'),
    children: [
      { path: '', component: () => import('@/views/DashboardHome.vue') },
      { path: 'settings', component: () => import('@/views/SettingsView.vue') },
    ],
  },
]
```

### 六、注意事项

```js
// ⚠️ 懒加载的组件在首次访问时会有短暂加载时间
// 可以配合 Suspense 或加载动画优化体验

// ⚠️ 不要过度拆分：每个 chunk 都有额外的网络请求开销
// 对于经常一起访问的页面，用分组打包合并
```
