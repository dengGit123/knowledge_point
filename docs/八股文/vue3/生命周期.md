## 一、生命周期概览

Vue 组件从创建到销毁的完整生命周期流程：

```
                    创建阶段
                       │
         ┌─────────────▼─────────────┐
         │    beforeCreate（Vue 2）   │
         │    setup()（Vue 3）        │
         └─────────────┬─────────────┘
                       │
         ┌─────────────▼─────────────┐
         │    created（Vue 2）        │
         │    （Vue 3 由 setup 替代） │
         └─────────────┬─────────────┘
                       │
              挂载阶段 │
         ┌─────────────▼─────────────┐
         │    onBeforeMount           │
         └─────────────┬─────────────┘
                       │
         ┌─────────────▼─────────────┐
         │    onMounted ✅            │ ← DOM 可用
         └─────────────┬─────────────┘
                       │
              更新阶段 │
         ┌─────────────▼─────────────┐
         │    onBeforeUpdate         │
         └─────────────┬─────────────┘
                       │
         ┌─────────────▼─────────────┐
         │    onUpdated              │
         └─────────────┬─────────────┘
                       │
              卸载阶段 │
         ┌─────────────▼─────────────┐
         │    onBeforeUnmount        │
         └─────────────┬─────────────┘
                       │
         ┌─────────────▼─────────────┐
         │    onUnmounted ✅          │ ← 清理工作
         └─────────────┘
```

## 二、Vue 2 vs Vue 3 生命周期对照

| Vue 2（Options API） | Vue 3（Options API） | Vue 3（Composition API） | 说明 |
|---------------------|---------------------|------------------------|------|
| `beforeCreate` | `beforeCreate` | ❌ 不需要（`setup` 自身替代） | 实例初始化之后，数据观测之前 |
| `created` | `created` | ❌ 不需要（`setup` 自身替代） | 实例创建完成，可访问数据 |
| `beforeMount` | `beforeMount` | `onBeforeMount()` | 挂载开始之前 |
| `mounted` | `mounted` | `onMounted()` | DOM 挂载完成 |
| `beforeUpdate` | `beforeUpdate` | `onBeforeUpdate()` | 数据变化，DOM 更新前 |
| `updated` | `updated` | `onUpdated()` | DOM 重新渲染完成 |
| `beforeDestroy` | **`beforeUnmount`** | `onBeforeUnmount()` | 组件卸载前 |
| `destroyed` | **`unmounted`** | `onUnmounted()` | 组件卸载完成 |
| `activated` | `activated` | `onActivated()` | 被 keep-alive 缓存的组件激活 |
| `deactivated` | `deactivated` | `onDeactivated()` | 被 keep-alive 缓存的组件停用 |
| `errorCaptured` | `errorCaptured` | `onErrorCaptured()` | 捕获后代组件错误 |

> **命名变化**：Vue 3 中 `destroyed` → `unmounted`，`beforeDestroy` → `beforeUnmount`，更语义化。

## 三、Composition API 中的使用

### 基本用法

```vue
<script setup>
import {
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted
} from 'vue'

onBeforeMount(() => {
  console.log('即将挂载，模板编译完成，但 DOM 未渲染')
})

onMounted(() => {
  console.log('DOM 已挂载，可以操作 DOM、发请求')
  // ✅ 访问 DOM 元素
  // ✅ 发送 API 请求
  // ✅ 添加事件监听
  // ✅ 初始化第三方库
})

onBeforeUpdate(() => {
  console.log('数据已变化，DOM 即将更新')
})

onUpdated(() => {
  console.log('DOM 已更新完成')
  // ⚠️ 不要在这里修改响应式状态，可能导致无限循环
})

onBeforeUnmount(() => {
  console.log('组件即将卸载，实例仍然可用')
})

onUnmounted(() => {
  console.log('组件已卸载')
  // ✅ 清除定时器
  // ✅ 移除事件监听
  // ✅ 取消网络请求
  // ✅ 释放资源
})
</script>
```

### 配合 Composable 的模式

```javascript
// composables/useEventListener.js
import { onMounted, onUnmounted } from 'vue'

export function useEventListener(target, event, handler) {
  onMounted(() => target.addEventListener(event, handler))
  onUnmounted(() => target.removeEventListener(event, handler))
}

// 组件中使用
import { useEventListener } from '@/composables/useEventListener'

useEventListener(window, 'resize', handleResize)
useEventListener(document, 'keydown', handleKeydown)
```

> **要点**：生命周期钩子可以**多次调用**，按注册顺序依次执行。

## 四、各钩子的典型使用场景

| 钩子 | 典型场景 |
|------|---------|
| `setup()` / `<script setup>` | 初始化响应式数据、computed、watch |
| `onBeforeMount` | 很少使用，可以在挂载前修改数据 |
| `onMounted` | DOM 操作、API 请求、事件监听、第三方库初始化 |
| `onBeforeUpdate` | 获取更新前的 DOM 状态 |
| `onUpdated` | 访问更新后的 DOM（需避免修改状态） |
| `onBeforeUnmount` | 在组件仍完整时做清理准备 |
| `onUnmounted` | 清除定时器、移除事件监听、取消请求 |
| `onActivated` | keep-alive 组件激活时恢复状态 |
| `onDeactivated` | keep-alive 组件停用时保存状态 |
| `onErrorCaptured` | 捕获子组件错误、显示错误提示 |

## 五、服务端渲染（SSR）中的生命周期

| 钩子 | SSR 中是否执行 |
|------|---------------|
| `setup()` | ✅ 执行 |
| `onBeforeMount` | ❌ 不执行 |
| `onMounted` | ❌ 不执行 |
| `onBeforeUpdate` | ❌ 不执行 |
| `onUpdated` | ❌ 不执行 |
| `onBeforeUnmount` | ❌ 不执行 |
| `onUnmounted` | ❌ 不执行 |

> **结论**：在 SSR 环境中，只有 `setup()` 会执行。所有 DOM 操作和副作用（API 请求、事件监听等）应放在 `onMounted` 中。

## 六、父子组件生命周期顺序

### 挂载阶段

```
父 setup
  → 父 onBeforeMount
    → 子 setup
      → 子 onBeforeMount
        → 子 onMounted ✅
  → 父 onMounted ✅
```

### 更新阶段

```
父 onBeforeUpdate
  → 子 onBeforeUpdate
    → 子 onUpdated
→ 父 onUpdated
```

### 卸载阶段

```
父 onBeforeUnmount
  → 子 onBeforeUnmount
    → 子 onUnmounted
→ 父 onUnmounted
```

> **记忆口诀**：挂载时"由内到外"（先子后父），卸载时也是"由内到外"（先子后父）。

## 七、新增的调试钩子

Vue 3 新增了用于调试的生命周期钩子：

```vue
<script setup>
import { onRenderTracked, onRenderTriggered } from 'vue'

// 组件渲染时追踪到响应式依赖时调用
onRenderTracked((event) => {
  console.log('追踪到依赖:', event)
})

// 响应式依赖变化触发组件重新渲染时调用
onRenderTriggered((event) => {
  console.log('触发重新渲染:', event)
  // event 包含：effect、target、key、type、newValue、oldValue
})
</script>
```

## 八、面试常见问题

### Q1：Vue 3 的 setup 替代了哪些生命周期？

`setup()` 替代了 `beforeCreate` 和 `created`。因为 `setup` 本身就是在组件实例创建之初、数据初始化之前执行的，相当于 `beforeCreate`；而 `setup` 中的代码（同步部分）在数据响应式初始化完成后执行，相当于 `created`。

### Q2：为什么建议在 onMounted 而不是 setup 中发送 API 请求？

- 如果使用 SSR，`setup` 在服务端和客户端都会执行，而 `onMounted` 只在客户端执行
- 在 `onMounted` 中可以确保 DOM 已经渲染，便于请求失败时展示错误 UI
- 但如果需要 **SSR 数据预获取**，应在 `setup` 中使用 `async/await` 配合 Suspense

### Q3：onUpdated 中可以修改响应式数据吗？

**不建议**。在 `onUpdated` 中修改响应式数据可能导致无限更新循环。如果确实需要在状态变化后执行逻辑，推荐使用 `watch` 或 `watchEffect`，它们可以精确控制执行时机。

### Q4：keep-alive 组件的生命周期是怎样的？

被 `<KeepAlive>` 包裹的组件切换时不会销毁，而是触发 `onDeactivated`；重新激活时触发 `onActivated`。这两个钩子在常规生命周期之外，用于管理缓存组件的状态恢复。
