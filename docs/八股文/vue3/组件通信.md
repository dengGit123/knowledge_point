## 一、组件通信方式概览

```
┌──────────────────────────────────────────────────────────────┐
│                    Vue 3 组件通信方式                          │
├──────────────┬──────────────────────┬────────────────────────┤
│    方式      │      数据方向        │      适用场景          │
├──────────────┼──────────────────────┼────────────────────────┤
│ props/emit   │ 父 ↔ 子             │ 最基础的父子通信        │
│ v-model      │ 父 ↔ 子（双向绑定）  │ 表单、弹窗控制         │
│ provide/inject│ 祖先 → 后代         │ 跨层级传递（主题、配置）│
│ ref + expose │ 父 → 子（命令式）    │ 父直接调用子组件方法    │
│ 事件总线     │ 任意组件             │ 简单跨组件通信          │
│ 状态管理     │ 任意组件             │ 复杂全局状态（Pinia）   │
│ $attrs       │ 父 → 子（透传）      │ 属性透传（封装组件）    │
└──────────────┴──────────────────────┴────────────────────────┘
```

## 二、props / emit — 父子通信

### props（父 → 子）

```vue
<!-- Parent.vue -->
<script setup>
import Child from './Child.vue'
import { ref } from 'vue'

const message = ref('来自父组件的消息')
</script>

<template>
  <Child :message="message" :count="42" />
</template>

<!-- Child.vue -->
<script setup>
// 方式一：运行时声明
const props = defineProps({
  message: {
    type: String,
    required: true
  },
  count: {
    type: Number,
    default: 0
  }
})

// 方式二：TypeScript 类型声明
// const props = defineProps<{
//   message: string
//   count?: number
// }>()

// 方式三：带默认值的 TypeScript 声明
// const props = withDefaults(defineProps<{
//   message: string
//   count?: number
// }>(), {
//   count: 0
// })

console.log(props.message)
</script>
```

**props 的特性**：

| 特性 | 说明 |
|------|------|
| **单向数据流** | 父 → 子方向，子组件不应直接修改 props |
| **只读** | 直接修改 props 会触发警告 |
| **响应式** | 父组件数据变化时自动更新子组件 |
| **类型验证** | 支持类型检查和自定义验证函数 |

### emit（子 → 父）

```vue
<!-- Child.vue -->
<script setup>
const emit = defineEmits<{
  update: [value: string]
  delete: [id: number]
}>()

function handleInput(e) {
  emit('update', e.target.value)
}
</script>

<template>
  <input @input="handleInput" />
  <button @click="emit('delete', 1)">删除</button>
</template>

<!-- Parent.vue -->
<script setup>
import Child from './Child.vue'

function onUpdate(value) {
  console.log('收到更新:', value)
}

function onDelete(id) {
  console.log('收到删除:', id)
}
</script>

<template>
  <Child @update="onUpdate" @delete="onDelete" />
</template>
```

## 三、v-model — 双向绑定

`v-model` 本质是 `props + emit` 的语法糖，Vue 3 中做了增强。

### 基本用法

```vue
<!-- 等价于 -->
<Child v-model="value" />
<!-- 以下写法 -->
<Child :modelValue="value" @update:modelValue="value = $event" />
```

```vue
<!-- Child.vue -->
<script setup>
const props = defineProps<{ modelValue: string }>()
const emit = defineEmits<{ 'update:modelValue': [value: string] }>()

function onInput(e) {
  emit('update:modelValue', e.target.value)
}
</script>

<template>
  <input :value="modelValue" @input="onInput" />
</template>
```

### 多个 v-model

Vue 3 支持在一个组件上使用**多个 v-model**：

```vue
<!-- Parent.vue -->
<UserForm v-model:first-name="firstName" v-model:last-name="lastName" />

<!-- UserForm.vue -->
<script setup>
const firstName = defineModel('firstName')
const lastName = defineModel('lastName')
</script>

<!-- 或者使用传统方式 -->
<script setup>
const props = defineProps({
  firstName: String,
  lastName: String
})
const emit = defineEmits(['update:firstName', 'update:lastName'])
</script>
```

### `defineModel()` — Vue 3.4+ 简化语法

```vue
<!-- Child.vue（Vue 3.4+） -->
<script setup>
// defineModel() 返回一个可读写的 ref
const modelValue = defineModel()
modelValue.value = 'new value'  // 自动触发 emit

// 带选项
const count = defineModel({ type: Number, default: 0 })

// 命名 model
const title = defineModel('title', { type: String })
</script>

<template>
  <input v-model="modelValue" />
</template>
```

### v-model 修饰符

```vue
<!-- 使用内置修饰符 -->
<Child v-model.trim="text" />
<Child v-model.number="age" />

<!-- 自定义修饰符 -->
<Child v-model.capitalize="name" />

<!-- Child.vue -->
<script setup>
const [modelValue, modelModifiers] = defineModel()

// modelModifiers.capitalize 为 true 时表示使用了 capitalize 修饰符

function handleInput(e) {
  let value = e.target.value
  if (modelModifiers.capitalize) {
    value = value.charAt(0).toUpperCase() + value.slice(1)
  }
  modelValue.value = value
}
</script>
```

## 四、provide / inject — 跨层级通信

`provide / inject` 允许祖先组件向所有后代组件**注入数据**，无论层级有多深。

```
┌─ 祖先组件 ────────────────────────┐
│  provide('theme', { color: red }) │
│                                    │
│  ┌─ 子组件 ──────────────────┐    │
│  │                            │    │
│  │  ┌─ 孙子组件 ──────────┐  │    │
│  │  │ inject('theme')     │  │    │
│  │  │ → { color: 'red' }  │  │    │
│  │  └──────────────────────┘  │    │
│  └────────────────────────────┘    │
└────────────────────────────────────┘
```

### 基本用法

```vue
<!-- 祖先组件 -->
<script setup>
import { provide, ref } from 'vue'

// 提供静态值
provide('theme', { color: 'red' })

// 提供响应式数据
const user = ref({ name: 'Tom' })
provide('user', user)

// 提供带默认值的数据
provide('config', { api: '/api/v1' })
</script>

<!-- 后代组件（任意层级） -->
<script setup>
import { inject } from 'vue'

// 接收注入
const theme = inject('theme')
const user = inject('user')

// 带默认值
const config = inject('config', { api: '/default' })

// 使用工厂函数作为默认值
const data = inject('data', () => createDefaultData(), true)
</script>
```

### 响应式 provide / inject 最佳实践

```vue
<!-- 祖先组件：提供响应式数据 + 修改方法 -->
<script setup>
import { provide, ref } from 'vue'

const count = ref(0)
const increment = () => count.value++

// 将数据和修改方法一起提供，避免后代直接修改
provide('counter', {
  count: readonly(count),  // 只读数据
  increment                // 修改方法
})
</script>

<!-- 后代组件 -->
<script setup>
import { inject } from 'vue'

const { count, increment } = inject('counter')
</script>

<template>
  <p>{{ count }}</p>
  <button @click="increment">+1</button>
</template>
```

> **最佳实践**：provide 响应式数据时，建议搭配 `readonly()` 和修改方法，保持数据流向清晰。

### Symbol 作为 key

```javascript
// keys.js
export const ThemeKey = Symbol('theme')
export const UserKey = Symbol('user')

// 祖先组件
import { ThemeKey } from './keys'
provide(ThemeKey, { color: 'red' })

// 后代组件
import { ThemeKey } from './keys'
const theme = inject(ThemeKey)
```

> **推荐使用 Symbol 作为 key**，避免与第三方库的 provide 产生命名冲突。

## 五、ref + expose — 父直接操作子

```vue
<!-- Child.vue -->
<script setup>
import { ref } from 'vue'

const count = ref(0)
const privateData = ref('secret')

function increment() {
  count.value++
}

function reset() {
  count.value = 0
}

// 只暴露必要的方法和数据
defineExpose({ count, increment, reset })
</script>

<!-- Parent.vue -->
<script setup>
import { ref } from 'vue'
import Child from './Child.vue'

const childRef = ref(null)

function handleIncrement() {
  childRef.value.increment()   // ✅ 调用子组件方法
  console.log(childRef.value.count)  // ✅ 读取子组件数据
}
</script>

<template>
  <Child ref="childRef" />
  <button @click="handleIncrement">操作子组件</button>
</template>
```

## 六、事件总线（mitt）

Vue 3 移除了 `$on`、`$off`、`$once` 等事件 API，需要使用第三方库（如 [mitt](https://github.com/developit/mitt)）实现事件总线：

```javascript
// eventBus.js
import mitt from 'mitt'
export const emitter = mitt()

// 类型安全的事件总线（TypeScript）
type Events = {
  search: string
  notify: { type: 'success' | 'error'; message: string }
}
export const emitter = mitt<Events>()
```

```vue
<!-- 组件 A：发送事件 -->
<script setup>
import { emitter } from '@/eventBus'

function handleSearch(keyword) {
  emitter.emit('search', keyword)
}
</script>

<!-- 组件 B：接收事件 -->
<script setup>
import { onMounted, onUnmounted } from 'vue'
import { emitter } from '@/eventBus'

function onSearch(keyword) {
  console.log('搜索:', keyword)
}

onMounted(() => emitter.on('search', onSearch))
onUnmounted(() => emitter.off('search', onSearch))
</script>
```

> **注意**：事件总线适合简单场景，复杂项目建议使用 Pinia 状态管理。

## 七、Pinia 状态管理

Pinia 是 Vue 3 官方推荐的状态管理库，替代 Vuex。

```javascript
// stores/useCounter.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useCounterStore = defineStore('counter', () => {
  // state
  const count = ref(0)

  // getters
  const doubled = computed(() => count.value * 2)

  // actions
  function increment() {
    count.value++
  }

  return { count, doubled, increment }
})
```

```vue
<!-- 任意组件中使用 -->
<script setup>
import { useCounterStore } from '@/stores/useCounter'

const counter = useCounterStore()

// 读取 state
console.log(counter.count)

// 读取 getter
console.log(counter.doubled)

// 调用 action
counter.increment()

// 直接修改 state
counter.count = 10

// $patch 批量修改
counter.$patch({ count: 20 })

// 解构（保持响应式）
const { count, doubled } = storeToRefs(counter)
</script>
```

## 八、$attrs — 属性透传

```vue
<!-- Parent.vue -->
<Child class="custom" id="main" data-testid="child" title="hello" />

<!-- Child.vue -->
<script setup>
// defineProps 不声明 title，则 title 会通过 $attrs 透传
defineProps({
  // 只声明了 class
})

// 可以手动绑定 $attrs 到内部元素
</script>

<template>
  <!-- $attrs 包含 { id, title, data-testid, class } -->
  <!-- 禁用自动透传：defineOptions({ inheritAttrs: false }) -->
  <div v-bind="$attrs">
    <!-- 手动绑定透传属性 -->
  </div>
</template>
```

## 九、通信方式选择决策表

| 场景 | 推荐方式 | 原因 |
|------|---------|------|
| 父传子数据 | `props` | 单向数据流，清晰可控 |
| 子传父事件 | `emit` | 符合 Vue 数据流规范 |
| 父子双向绑定 | `v-model` | `props + emit` 的语法糖 |
| 跨多级传递 | `provide / inject` | 无需中间层逐层传递 |
| 父调子方法 | `ref + defineExpose` | 命令式操作 |
| 兄弟组件通信 | Pinia | 集中管理，可追踪 |
| 全局状态 | Pinia | 完善的 DevTools 支持 |
| 临时跨组件通信 | mitt 事件总线 | 轻量，但不宜多用 |
| 封装组件传递属性 | `$attrs` | 简化属性透传 |

## 十、面试常见问题

### Q1：Vue 3 的 v-model 和 Vue 2 有什么区别？

| 维度 | Vue 2 | Vue 3 |
|------|-------|-------|
| 默认 prop 名 | `value` | `modelValue` |
| 默认事件名 | `@input` | `@update:modelValue` |
| 多个 v-model | ❌ 需要 `.sync` 修饰符 | ✅ 原生支持 `v-model:xxx` |
| 修饰符 | 固定内置 | 支持自定义修饰符 |

### Q2：provide/inject 的数据是响应式的吗？

**不一定**。`provide` 传入什么，`inject` 就收到什么：
- 传入 `ref` / `reactive` → 响应式 ✅
- 传入普通对象 → 非响应式 ❌

建议始终提供响应式数据，并用 `readonly()` 保护，防止后代意外修改。

### Q3：为什么不推荐用事件总线（mitt）做大型项目？

1. **事件来源不可追踪**：难以知道事件从哪里发出、谁在监听
2. **调试困难**：没有 DevTools 支持，状态变化无法追踪
3. **内存泄漏风险**：忘记取消监听会导致组件卸载后仍然响应
4. **类型安全差**：事件名和参数难以约束

大型项目应使用 Pinia，事件总线仅用于少量、简单的跨组件通知。
