## 一、自定义渲染器（Custom Renderer）

### 概念

Vue 3 的运行时被拆分为**平台无关的核心**（`@vue/runtime-core`）和**平台特定的实现**（如 `@vue/runtime-dom`）。自定义渲染器允许你将 Vue 的响应式系统和组件模型映射到**任意渲染目标**。

```
┌──────────────────────────────────────────────────┐
│                Vue 3 运行时架构                    │
│                                                   │
│  @vue/reactivity    ← 响应式系统（平台无关）       │
│         ↓                                         │
│  @vue/runtime-core  ← 组件系统 + 调度器（平台无关）│
│         ↓                                         │
│  @vue/runtime-dom   ← DOM 渲染器（浏览器）         │
│  自定义 renderer    ← Canvas / Native / WebGL ... │
└──────────────────────────────────────────────────┘
```

### `createRenderer()` API

```javascript
import { createRenderer } from '@vue/runtime-core'

const { render, createApp } = createRenderer({
  // 创建元素
  createElement(type) {
    // 例如 Canvas：创建对应的图形对象
  },

  // 设置元素属性
  patchProp(el, key, prevValue, nextValue) {
    // 处理属性更新
  },

  // 插入元素
  insert(el, parent, anchor) {
    // 将元素插入到父容器中
  },

  // 删除元素
  remove(el) {
    // 从父容器中移除元素
  },

  // 创建文本节点
  createText(text) {},

  // 设置文本内容
  setText(node, text) {},

  // 设置元素文本
  setElementText(el, text) {},

  // 获取父节点
  parentNode(node) {},

  // 获取兄弟节点
  nextSibling(node) {},
})
```

### 示例：Canvas 渲染器

```javascript
// canvasRenderer.js
import { createRenderer } from '@vue/runtime-core'

function drawRect(ctx, { x, y, width, height, fill }) {
  ctx.fillStyle = fill
  ctx.fillRect(x, y, width, height)
}

const { render, createApp } = createRenderer({
  createElement(type) {
    return { type, children: [], props: {} }
  },
  patchProp(el, key, prevValue, nextValue) {
    el.props[key] = nextValue
  },
  insert(el, parent) {
    parent.children.push(el)
  },
  remove(el) {
    const index = el.parent.children.indexOf(el)
    if (index > -1) el.parent.children.splice(index, 1)
  },
  setElementText(el, text) {
    el.text = text
  },
  // ... 其他节点操作
})

export function createCanvasApp(rootComponent, canvas) {
  const ctx = canvas.getContext('2d')
  const app = createApp(rootComponent)
  // 自定义挂载逻辑
  return app
}
```

```vue
<!-- 使用 Canvas 渲染器 -->
<script>
import { ref } from '@vue/reactivity'
import { createCanvasApp } from './canvasRenderer'

const App = {
  setup() {
    const x = ref(10)
    return { x }
  },
  template: `<Rect :x="x" :y="10" :width="100" :height="50" fill="red" />`
}

createCanvasApp(App, document.querySelector('canvas'))
</script>
```

## 二、自定义指令（Custom Directives）

### 概念

自定义指令用于**封装底层 DOM 操作**，适用于需要直接操作 DOM 的场景。

### Vue 3 指令钩子

| 钩子 | 说明 | 对应 Vue 2 |
|------|------|-----------|
| `created` | 元素创建后，属性/事件之前 | — |
| `beforeMount` | 元素挂载到 DOM 前 | `bind` |
| `mounted` | 元素挂载到 DOM 后 | `inserted` |
| `beforeUpdate` | 组件更新前 | — |
| `updated` | 组件更新后 | `update` + `componentUpdated` |
| `beforeUnmount` | 元素卸载前 | — |
| `unmounted` | 元素卸载后 | `unbind` |

### 基本用法

```vue
<script setup>
// 局部指令（以 v 开头的变量名自动识别为指令）
const vFocus = {
  mounted(el) {
    el.focus()
  }
}
</script>

<template>
  <input v-focus />
</template>
```

### 全局注册

```javascript
// main.js
import { createApp } from 'vue'

const app = createApp(App)

app.directive('focus', {
  mounted(el) {
    el.focus()
  }
})

// 简写（mounted + updated）
app.directive('color', (el, binding) => {
  el.style.color = binding.value
})
```

### 常见指令示例

```javascript
// 1. v-debounce —— 防抖点击
app.directive('debounce', {
  mounted(el, binding) {
    let timer = null
    el.addEventListener('click', () => {
      if (timer) clearTimeout(timer)
      timer = setTimeout(() => {
        binding.value()
      }, binding.arg ? parseInt(binding.arg) : 300)
    })
  }
})
// 使用: <button v-debounce:500="handleClick">点击</button>

// 2. v-permission —— 权限控制
app.directive('permission', {
  mounted(el, binding) {
    const userPermissions = getUserPermissions()
    if (!userPermissions.includes(binding.value)) {
      el.parentNode?.removeChild(el)
    }
  }
})
// 使用: <button v-permission="'admin'">管理员操作</button>

// 3. v-loading —— 加载状态
app.directive('loading', {
  mounted(el, binding) {
    const mask = document.createElement('div')
    mask.className = 'loading-mask'
    mask.innerHTML = '<div class="spinner"></div>'
    el.style.position = 'relative'
    el.instance = mask
    if (binding.value) {
      el.appendChild(mask)
    }
  },
  updated(el, binding) {
    if (binding.value) {
      el.appendChild(el.instance)
    } else {
      el.instance.parentNode?.removeChild(el.instance)
    }
  }
})
// 使用: <div v-loading="isLoading">内容</div>

// 4. v-copy —— 一键复制
app.directive('copy', {
  mounted(el, binding) {
    el.addEventListener('click', () => {
      navigator.clipboard.writeText(binding.value)
        .then(() => alert('复制成功'))
        .catch(() => alert('复制失败'))
    })
  }
})
// 使用: <button v-copy="textToCopy">复制</button>
```

### 指令钩子参数

```javascript
app.directive('demo', {
  mounted(el, binding, vnode) {
    // el —— 指令绑定的元素（可直接操作 DOM）
    // binding —— 包含指令信息的对象
    console.log(binding.value)      // 指令的绑定值: v-demo="1+1" → 2
    console.log(binding.oldValue)   // 前一个值（仅在 updated 中可用）
    console.log(binding.arg)        // 指令参数: v-demo:foo → 'foo'
    console.log(binding.modifiers)  // 修饰符: v-demo.bar.baz → { bar: true, baz: true }
    console.log(binding.instance)   // 组件实例
    // vnode —— Vue 编译生成的虚拟节点
  }
})
```

## 三、插件（Plugins）

### 概念

插件是**为 Vue 添加全局功能**的代码，常用于：
- 注册全局组件、指令
- 添加全局方法或属性
- 提供 inject/provide 的全局服务
- 添加配置

### 插件结构

```javascript
// 方式一：对象形式（带 install 方法）
const myPlugin = {
  install(app, options) {
    // app —— Vue 应用实例
    // options —— 用户传入的配置
    app.config.globalProperties.$translate = (key) => {
      return options.messages[key]
    }
  }
}

// 方式二：函数形式（函数本身作为 install）
const myPlugin = (app, options) => {
  app.provide('config', options)
}
```

### 注册插件

```javascript
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import pinia from './stores'

const app = createApp(App)

// 使用插件
app.use(router)             // Vue Router
app.use(pinia)              // Pinia
app.use(myPlugin, {         // 自定义插件 + 配置
  messages: { hello: '你好' }
})

app.mount('#app')
```

### 实用插件示例

```javascript
// 1. 全局配置插件
const configPlugin = {
  install(app, options) {
    // 全局属性
    app.config.globalProperties.$config = options

    // 全局 provide
    app.provide('appConfig', options)

    // 注册全局组件
    app.component('GlobalButton', GlobalButtonComponent)

    // 注册全局指令
    app.directive('focus', {
      mounted(el) { el.focus() }
    })
  }
}

// 2. HTTP 请求插件
const httpPlugin = {
  install(app, options) {
    const http = {
      async get(url) {
        const response = await fetch(options.baseURL + url, {
          headers: options.headers
        })
        return response.json()
      },
      async post(url, data) {
        const response = await fetch(options.baseURL + url, {
          method: 'POST',
          headers: { 'Content-Type': 'application/json', ...options.headers },
          body: JSON.stringify(data)
        })
        return response.json()
      }
    }

    app.config.globalProperties.$http = http
    app.provide('http', http)
  }
}

// 3. 事件追踪插件（利用组合式 API）
const trackingPlugin = {
  install(app) {
    app.mixin({
      onMounted() {
        // 不推荐 mixin，仅演示插件能力
      }
    })
  }
}
```

## 四、渲染函数（Render Function）

### h() 函数

`h()` 是 `createElement` 的缩写，用于编程式创建 VNode：

```vue
<script setup>
import { h, ref } from 'vue'

const count = ref(0)

// h(type, props, children)
const vnode = h('div', { class: 'container' }, [
  h('h1', null, '标题'),
  h('p', null, `计数: ${count.value}`),
  h('button', { onClick: () => count.value++ }, '点击')
])
</script>
```

### 函数式组件

```javascript
// 函数式组件：无状态、简单高效
const FunctionalButton = (props, { slots }) => {
  return h('button', { class: props.type }, slots.default())
}

FunctionalButton.props = ['type']
```

## 五、面试常见问题

### Q1：自定义渲染器有什么实际应用？

1. **跨平台开发**：将 Vue 组件渲染到原生移动端（类似 NativeScript-Vue）
2. **Canvas / WebGL**：在 Canvas 中使用 Vue 组件化开发游戏或可视化
3. **终端渲染**：在命令行终端中渲染 Vue 组件
4. **PDF 生成**：将 Vue 组件树映射为 PDF 文档
5. **自定义 DOM**：如 Shadow DOM 环境的渲染

### Q2：Vue 3 的自定义指令和 Vue 2 有什么区别？

| 维度 | Vue 2 | Vue 3 |
|------|-------|-------|
| 钩子命名 | `bind`、`inserted`、`update`、`componentUpdated`、`unbind` | `created`、`beforeMount`、`mounted`、`beforeUpdate`、`updated`、`beforeUnmount`、`unmounted` |
| 钩子命名风格 | 独立命名 | **与组件生命周期对齐** |
| 钩子数量 | 5 个 | 7 个（新增 `created`、`beforeMount`、`beforeUnmount`） |

### Q3：什么场景用自定义指令，什么场景用组件？

- **自定义指令**：纯 DOM 操作，不需要自己的模板和响应式状态（如 focus、复制、拖拽）
- **组件**：需要渲染 UI、管理状态、响应用户交互（如弹窗、表单、列表）

> 如果逻辑涉及渲染内容，优先用组件；如果只是增强已有元素的行为，用指令。

### Q4：插件和 Composable 怎么选？

| 维度 | 插件 | Composable |
|------|------|-----------|
| 作用范围 | **全局**（app 级别） | **组件级**（按需引入） |
| 注册方式 | `app.use()` | `import` + `use` |
| 典型场景 | 路由、状态管理、全局配置 | 数据获取、表单、动画逻辑 |
| Tree-shaking | 全局注册，难以 tree-shake | ✅ 按需导入，天然 tree-shake |
| 推荐程度 | 必要时使用 | ✅ 优先选择 |

> **原则**：能用 Composable 解决的，就不用插件。插件只用于真正需要全局注册的场景。
