# 响应式原理

> 官方文档：[深入响应式系统](https://cn.vuejs.org/guide/extras/reactivity-in-depth.html)

> 官方文档：[响应式 API：核心](https://cn.vuejs.org/api/reactivity-core.html)

---

## 一、核心概念

Vue 的响应式系统本质上是一个**依赖收集 + 派发更新**的机制：

```
┌─────────────────────────────────────────────────────────┐
│                     响应式流程                           │
│                                                         │
│   数据变化 ──→ Proxy 拦截 set ──→ 触发依赖 ──→ 更新视图  │
│                                                         │
│   读取数据 ──→ Proxy 拦截 get ──→ 收集依赖（副作用函数） │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

> **通俗理解**：响应式就像一个"订阅-通知"系统。当你读取数据时（订阅），系统记住"谁在关注这个数据"；当数据变化时（通知），系统通知所有关注者去更新。

## 二、Proxy vs Object.defineProperty

### Vue 2 的 `Object.defineProperty`

```javascript
// Vue 2 的响应式实现（简化版）
function defineReactive(obj, key, val) {
  Object.defineProperty(obj, key, {
    get() {
      // 收集依赖
      dep.depend()
      return val
    },
    set(newVal) {
      val = newVal
      // 触发更新
      dep.notify()
    }
  })
}
```

**局限性**：

| 问题 | 说明 |
|------|------|
| 无法检测属性新增/删除 | 需要使用 `Vue.set()` / `Vue.delete()` |
| 无法检测数组索引赋值 | `arr[0] = newValue` 不会触发更新 |
| 无法检测数组长度修改 | `arr.length = 0` 不会触发更新 |
| 初始化时全量递归 | 深层嵌套对象性能开销大 |
| 不支持 Map / Set | `Object.defineProperty` 只能劫持普通对象 |

### Vue 3 的 `Proxy`

```javascript
// Vue 3 的响应式实现（简化版）
function reactive(target) {
  return new Proxy(target, {
    get(target, key, receiver) {
      // 收集依赖
      track(target, key)
      const result = Reflect.get(target, key, receiver)
      // 惰性深层代理：只有访问到时才代理嵌套对象
      if (typeof result === 'object' && result !== null) {
        return reactive(result)
      }
      return result
    },
    set(target, key, value, receiver) {
      const oldValue = target[key]
      const result = Reflect.set(target, key, value, receiver)
      if (oldValue !== value) {
        // 触发更新
        trigger(target, key)
      }
      return result
    }
  })
}
```

**Proxy 的优势**：

| 特性 | 说明 |
|------|------|
| 拦截范围广 | 可以拦截属性的**读取、设置、删除、枚举、has** 等操作 |
| 属性新增/删除 | 自动检测 ✅ |
| 数组操作 | 全面支持 ✅ |
| Map / Set / WeakMap / WeakSet | 通过专门的集合处理器支持 ✅ |
| 惰性代理 | 嵌套对象在访问时才被代理，初始化性能更优 |
| 代理整个对象 | 不需要逐个属性劫持 |

## 三、核心 API

### 1. `reactive()`

返回对象的**响应式代理**，适用于对象类型（Object、Array、Map、Set 等）。

```vue
<script setup>
import { reactive } from 'vue'

const state = reactive({
  count: 0,
  nested: {
    value: 'hello'
  },
  list: [1, 2, 3]
})

// 所有操作自动触发响应式
state.count++                    // ✅
state.nested.value = 'world'     // ✅ 深层也是响应式
state.list.push(4)               // ✅ 数组操作
delete state.count               // ✅ 删除属性
</script>
```

**注意事项**：

| 陷阱 | 说明 | 解决方案 |
|------|------|---------|
| 有限的值类型 | 只适用于对象类型 | 基本类型用 `ref()` |
| 不能替换整个对象 | 替换后会丢失响应式 | 使用 `Object.assign()` 或重新赋值属性 |
| 解构丢失响应式 | 解构后变为普通值 | 使用 `toRefs()` |

```vue
<script setup>
import { reactive, toRefs } from 'vue'

const state = reactive({ x: 0, y: 0 })

// ❌ 解构后丢失响应式
let { x, y } = state
x++  // 不会触发更新

// ✅ 使用 toRefs 保持响应式
const { x, y } = toRefs(state)
x.value++  // 触发更新
</script>
```

### 2. `ref()`

接收任意类型的值，返回一个**响应式引用对象** `{ value: T }`。是声明响应式状态的**推荐方式**。

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)           // Ref<number>
const name = ref('hello')     // Ref<string>
const list = ref([1, 2, 3])   // Ref<number[]>
const obj = ref({ a: 1 })     // Ref<{ a: number }>

// 读取和修改都需要 .value
console.log(count.value)  // 0
count.value++
console.log(count.value)  // 1
</script>

<template>
  <!-- 模板中自动解包，不需要 .value -->
  <p>{{ count }}</p>
</template>
```

**ref 的内部机制**：

```javascript
// ref 内部实现（简化版）
function ref(value) {
  return new RefImpl(value)
}

class RefImpl {
  constructor(value) {
    // 如果值是对象，内部会用 reactive 包装
    this._value = isObject(value) ? reactive(value) : value
    this.__v_isRef = true
  }

  get value() {
    // 收集依赖
    track(this, 'value')
    return this._value
  }

  set value(newVal) {
    this._value = isObject(newVal) ? reactive(newVal) : newVal
    // 触发更新
    trigger(this, 'value')
  }
}
```

### 3. `reactive` vs `ref` 选择指南

| 维度 | `reactive()` | `ref()` |
|------|-------------|---------|
| 适用类型 | 仅对象类型 | 任意类型（含基本类型） |
| 访问方式 | 直接访问属性 `state.count` | 需要 `.value`（模板中自动解包） |
| 解构 | 丢失响应式（需 `toRefs`） | 解构后仍保持响应式 |
| 替换整个对象 | 不可以 | 可以（直接赋新值给 `.value`） |
| 官方推荐 | 偏好 | ✅ **推荐首选** |

> **最佳实践**：优先使用 `ref()`，它适用于所有类型；只在需要使用集合类型（Map、Set）或明确知道对象结构不会替换时使用 `reactive()`。

## 四、依赖收集与触发

### 核心数据结构

```
targetMap（WeakMap）
  │
  ├── target（原始对象） → depsMap（Map）
  │                          │
  │                          ├── key（属性名） → dep（Set<ReactiveEffect>）
  │                          │                    ├── effect1
  │                          │                    ├── effect2
  │                          │                    └── ...
  │                          │
  │                          └── ...
  │
  └── ...
```

- **`targetMap`**：全局 WeakMap，存储所有响应式对象的依赖关系
- **`depsMap`**：每个响应式对象对应一个 Map，键是属性名
- **`dep`**：每个属性对应一个 Set，存放所有依赖该属性的副作用函数

### 工作流程

```
1. 创建响应式对象（reactive / ref）
         │
         ▼
2. 副作用函数执行（组件渲染、watch、computed 等）
         │
         ▼
3. 读取响应式数据 → Proxy get 拦截 → track() 收集当前副作用函数
         │
         ▼
4. 数据变化 → Proxy set 拦截 → trigger() 找到所有依赖并执行
         │
         ▼
5. 副作用函数重新执行 → 组件重新渲染
```

```javascript
// 依赖收集（简化版）
function track(target, key) {
  // 获取当前正在运行的副作用函数
  const activeEffect = getCurrentEffect()
  if (!activeEffect) return

  let depsMap = targetMap.get(target)
  if (!depsMap) {
    targetMap.set(target, (depsMap = new Map()))
  }

  let dep = depsMap.get(key)
  if (!dep) {
    depsMap.set(key, (dep = new Set()))
  }

  // 将当前副作用函数加入依赖集合
  dep.add(activeEffect)
}

// 触发更新（简化版）
function trigger(target, key) {
  const depsMap = targetMap.get(target)
  if (!depsMap) return

  const dep = depsMap.get(key)
  if (!dep) return

  // 执行所有依赖该属性的副作用函数
  dep.forEach(effect => {
    // 避免无限循环
    if (effect !== getCurrentEffect()) {
      effect.scheduler ? effect.scheduler() : effect.run()
    }
  })
}
```

## 五、深层响应式

### 惰性代理机制

Vue 3 采用**惰性深层代理**，只有实际访问到的嵌套对象才会被代理：

```javascript
const state = reactive({
  level1: {
    level2: {
      level3: 'deep value'
    }
  }
})

// 只有访问 state.level1 时，level1 才会被代理为响应式
// 只有访问 state.level1.level2 时，level2 才会被代理
// 不会在初始化时递归代理整个对象树
```

**性能优势**：
- 初始化更快：不需要递归遍历整个对象
- 内存更省：未访问的深层对象不会被代理
- 按需响应：只有实际使用的数据才有响应式开销

## 六、只读与浅层 API

### `readonly()` — 只读代理

```vue
<script setup>
import { reactive, readonly } from 'vue'

const original = reactive({ count: 0 })
const copy = readonly(original)

copy.count++  // ⚠️ 警告：Set operation on key "count" failed
</script>
```

### `shallowReactive()` — 浅层响应式

```vue
<script setup>
import { shallowReactive } from 'vue'

const state = shallowReactive({
  foo: 1,
  nested: { bar: 2 }
})

state.foo++          // ✅ 触发更新（顶层属性）
state.nested.bar++   // ❌ 不触发更新（嵌套对象不是响应式的）
</script>
```

### `shallowRef()` — 浅层 ref

```vue
<script setup>
import { shallowRef, triggerRef } from 'vue'

const state = shallowRef({ count: 0 })

state.value.count++         // ❌ 不触发更新（.value 没变）
state.value = { count: 1 }  // ✅ 触发更新（替换了 .value）

// 如果修改了内部值又想触发更新，可以手动调用
state.value.count++
triggerRef(state)            // ✅ 强制触发依赖
</script>
```

### API 对比表

| API | 顶层响应式 | 嵌套响应式 | 适用场景 |
|-----|----------|----------|---------|
| `reactive()` | ✅ | ✅ 递归 | 一般对象 |
| `shallowReactive()` | ✅ | ❌ | 大型对象，只关注顶层变化 |
| `ref()` | ✅（.value） | ✅ 自动包装 reactive | 任意类型 |
| `shallowRef()` | ✅（.value 替换） | ❌ | 大型数据结构 |
| `readonly()` | 只读 | 只读（递归） | 配置对象、props |
| `shallowReadonly()` | 只读 | 不限制 | props 的顶层保护 |

## 七、原始值转换

### `toRaw()` 和 `markRaw()`

```vue
<script setup>
import { reactive, toRaw, markRaw } from 'vue'

const state = reactive({ list: [] })

// toRaw：获取响应式对象的原始版本（不触发响应式）
const rawState = toRaw(state)
rawState.count = 999  // 不会触发更新

// markRaw：标记对象永远不转为响应式
const staticData = markRaw({ bigArray: new Array(10000) })
state.data = staticData  // data 属性不会被代理
</script>
```

### `toRef()` 和 `toRefs()`

```vue
<script setup>
import { reactive, toRef, toRefs } from 'vue'

const state = reactive({ x: 0, y: 0 })

// toRef：为某个属性创建 ref，保持与原对象的连接
const xRef = toRef(state, 'x')
xRef.value++          // state.x 也变成 1
state.x++             // xRef.value 也变成 2

// toRefs：为所有属性创建 ref
const { x, y } = toRefs(state)
x.value++             // 联动更新
</script>
```

## 八、Vue 3 响应式系统的可独立使用

Vue 3 的 `@vue/reactivity` 包可以**脱离 Vue 独立使用**：

```javascript
import { reactive, effect, computed } from '@vue/reactivity'

const state = reactive({
  count: 0
})

const doubled = computed(() => state.count * 2)

effect(() => {
  console.log(`count: ${state.count}, doubled: ${doubled.value}`)
})
// 立即输出: count: 0, doubled: 0

state.count++
// 自动输出: count: 1, doubled: 2
```

## 九、面试常见问题

### Q1：为什么 Vue 3 选择 Proxy 而不是继续用 defineProperty？

1. **功能完整**：Proxy 可以拦截对象上的所有操作（13 种），不再需要各种 hack
2. **性能优化**：惰性代理 + 不需要初始化时全量递归
3. **未来兼容**：Proxy 是 JS 标准的未来方向，浏览器支持度已足够

### Q2：ref 和 reactive 底层有什么区别？

- `reactive()` 内部使用 `Proxy` 代理整个对象
- `ref()` 内部使用类（`RefImpl`）的 `getter/setter` 实现对 `.value` 的追踪
- `ref()` 包装对象值时，内部会对 `.value` 调用 `reactive()`
- `ref()` 是更通用的方案，是官方推荐首选

### Q3：为什么 Vue 3 的响应式可以独立使用？

Vue 3 将响应式系统拆分为独立的 `@vue/reactivity` 包，它不依赖任何 Vue 运行时代码。这种模块化设计使得响应式可以在任何 JS 项目中使用。

### Q4：shallowReactive 和 reactive 的性能差异在什么场景下明显？

当对象**非常深**或**非常大**时（如大型表格数据、复杂配置树），`shallowReactive` 避免了对嵌套对象的递归代理，初始化和更新性能会显著优于 `reactive`。对于简单对象，差异可以忽略不计。
