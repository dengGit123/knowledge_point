# 编译优化

> 官方文档：[渲染机制](https://cn.vuejs.org/guide/extras/rendering-mechanism.html)

> 官方文档：[编译器优化](https://cn.vuejs.org/guide/extras/rendering-mechanism.html#static-optimization)

---

## 一、编译优化的核心思想

Vue 3 的编译器在编译模板时，会进行大量**静态分析**，生成经过优化的渲染函数代码，从而减少运行时的开销。

```
模板（Template）  ──→  编译器（Compiler）  ──→  优化的渲染函数（Render Function）
                          │
                          ├── 静态提升
                          ├── PatchFlag 标记
                          ├── Block Tree 追踪
                          ├── Tree-shaking
                          └── 静态字符串化
```

> **通俗理解**：编译器就像一个聪明的翻译官，在翻译时就把"哪些内容是固定的、哪些是变化的"标注好了，运行时只需要关注变化的部分。

---

## 二、静态提升（Static Hoisting）

### 概念

将模板中的**静态节点**提升到渲染函数之外，只创建一次，后续渲染直接复用。

```vue
<template>
  <div>
    <h1>固定标题</h1>
    <p>{{ message }}</p>
    <span>固定文本</span>
  </div>
</template>
```

### 编译结果对比

```javascript
// ❌ 没有静态提升（每次渲染都重新创建）
export function render() {
  return h('div', [
    h('h1', '固定标题'),       // 每次都创建新 VNode
    h('p', this.message),      // 动态
    h('span', '固定文本')       // 每次都创建新 VNode
  ])
}

// ✅ 有静态提升（静态节点只创建一次）
const _hoisted_1 = h('h1', '固定标题')      // 提升到外部
const _hoisted_2 = h('span', '固定文本')     // 提升到外部

export function render() {
  return h('div', [
    _hoisted_1,                 // 直接复用
    h('p', this.message),       // 动态
    _hoisted_2                  // 直接复用
  ])
}
```

### 提升条件

| 情况 | 是否提升 |
|------|---------|
| 纯文本节点 | ✅ 提升 |
| 只包含静态内容的元素 | ✅ 提升 |
| 包含动态绑定的元素 | ❌ 不提升 |
| 含有动态子节点的元素 | ❌ 不提升 |

### 静态属性提升

不仅仅是整个节点，**静态属性**也会被单独提升：

```javascript
// <div class="container" :id="dynamicId">{{ text }}</div>
const _hoisted_1 = { class: 'container' }

export function render() {
  return h('div', _hoisted_1, this.text)
  // class 属性被提升，每次渲染只重新计算 id
}
```

## 三、PatchFlag（补丁标记）

### 概念

编译器为动态节点添加**数字标记**，标识该节点哪些部分是动态的，Diff 时只检查被标记的部分。

```vue
<template>
  <div :class="activeClass" :id="staticId">
    <span>{{ message }}</span>
    <p static-text>不变化</p>
  </div>
</template>
```

### PatchFlag 类型

```javascript
export const enum PatchFlags {
  TEXT            = 1,        // 动态文本内容
  CLASS           = 1 << 1,   // 动态 class 绑定
  STYLE           = 1 << 2,   // 动态 style 绑定
  PROPS           = 1 << 3,   // 非 class/style 的动态属性
  FULL_PROPS      = 1 << 4,   // 有动态 key 的属性（需要完整 diff）
  HYDRATE_EVENTS  = 1 << 5,   // 有事件监听器
  STABLE_FRAGMENT = 1 << 6,   // 子节点顺序不变的 fragment
  KEYED_FRAGMENT  = 1 << 7,   // 有 key 的子节点 fragment
  UNKEYED_FRAGMENT = 1 << 8,  // 无 key 的子节点 fragment
  NEED_PATCH      = 1 << 9,   // 需要进行非 props 比较（ref 等）
  DYNAMIC_SLOTS   = 1 << 10,  // 动态插槽
  HOISTED         = -1,       // 静态提升的节点（跳过 diff）
  BAIL            = -2,       // 退出优化模式（无法确定时）
}
```

### 实际编译示例

```javascript
// <div :class="active">{{ text }}</div>
// 编译结果
_createVNode("div",
  { class: _ctx.active },   // 动态 class
  _ctx.text,                 // 动态文本
  PatchFlags.TEXT | PatchFlags.CLASS  // 标记：class + 文本是动态的
)

// Diff 时只检查 class 和 textContent，跳过其他属性
```

### 优化效果

| 优化 | 说明 |
|------|------|
| **跳过静态节点** | PatchFlag = HOISTED 的节点直接跳过 diff |
| **精确比较** | 只检查被标记的动态属性 |
| **减少遍历** | 不需要遍历所有 props 进行比较 |

## 四、Block Tree（块级树）

### 概念

Vue 3 引入了 **Block** 的概念。每个 Block 维护一个 **dynamicChildren** 数组，收集内部所有动态后代节点。更新时只需遍历这个数组，而不是整棵虚拟 DOM 树。

```
普通 VNode 树遍历                    Block 优化遍历
┌──────────────────────┐           ┌──────────────────────┐
│        div           │           │        div (Block)    │
│       /   \          │           │     dynamicChildren:  │
│    span    p         │           │     [span, p]         │
│   (静态) (动态)      │           │                        │
│                       │           │     只遍历这两个节点 ✅ │
│  遍历所有节点 ❌       │           │                        │
└──────────────────────┘           └──────────────────────┘
```

### Block 的收集过程

```javascript
// 模板
// <div>
//   <h1>静态标题</h1>
//   <p>{{ dynamicText }}</p>
//   <span class="static">静态内容</span>
// </div>

// 编译后的 Block 结构
const block = {
  type: 'div',
  dynamicChildren: [
    // 只包含动态节点（跳过了 h1 和 span）
    { type: 'p', children: dynamicText, patchFlag: PatchFlags.TEXT }
  ]
}

// 更新时：只遍历 dynamicChildren，跳过所有静态节点
```

### 什么时候会创建新的 Block？

| 情况 | 是否创建新 Block |
|------|----------------|
| 普通元素作为根节点 | ✅ 根节点自动成为 Block |
| `v-if` | ✅ 每个分支创建新的 Block |
| `v-for` | ✅ 每个循环项创建新的 Block |
| `v-slot`（动态插槽） | ✅ 创建新的 Block |
| 纯静态模板 | ❌ 无动态节点，不需要 Block |

## 五、静态字符串化（Static Stringification）

### 概念

当模板中连续的静态内容超过一定阈值（默认 20 个节点）时，编译器会将这些静态节点直接**序列化为 HTML 字符串**。

```javascript
// 普通静态提升：仍然创建 VNode
const _hoisted_1 = h('div', [
  h('h1', '标题'),
  h('p', '段落1'),
  h('p', '段落2'),
  // ...很多静态节点
])

// 静态字符串化：直接生成 HTML 字符串
const _hoisted_1 = /* @__PURE__ */ createStaticVNode(
  '<div><h1>标题</h1><p>段落1</p><p>段落2</p>...</div>',
  20  // 节点数量
)
```

**优势**：

| 对比 | 静态提升 | 静态字符串化 |
|------|---------|------------|
| 内存占用 | 每个节点一个 VNode 对象 | 一个字符串 |
| 初始化速度 | 逐个调用 `h()` 创建 VNode | 直接用字符串 |
| SSR 优化 | 需要 VNode → HTML | 直接输出字符串 |

## 六、Tree-shaking

### 概念

Vue 3 将所有全局 API 改为**具名导出**，配合构建工具实现 Tree-shaking，未使用的代码不会打包。

```javascript
// Vue 2：全量引入，无法 tree-shake
import Vue from 'vue'
// 即使只用了 nextTick，整个 Vue 都会被打包

// Vue 3：按需引入，自动 tree-shake
import { nextTick, ref, computed } from 'vue'
// 只打包这三个 API 的实现代码
// watchEffect、watch、onMounted 等未使用的不会打包
```

### 可 Tree-shake 的 API

| 类别 | API |
|------|-----|
| 响应式 | `ref`、`reactive`、`computed`、`watch`、`watchEffect` |
| 生命周期 | `onMounted`、`onUpdated`、`onUnmounted` ... |
| 依赖注入 | `provide`、`inject` |
| 工具函数 | `nextTick`、`toRef`、`toRaw` |
| 内置组件 | `Transition`、`KeepAlive`、`Teleport`、`Suspense` |
| 指令 | `v-model`、`v-if`、`v-for`（只有使用到的才会编译进去） |

### 包体积对比

| 版本 | 运行时大小（gzip） | 全量大小（gzip） |
|------|-------------------|-----------------|
| Vue 2 | — | ~23KB |
| Vue 3（完整） | — | ~33KB |
| Vue 3（按需） | ~13KB | 按使用量增长 |

> **结论**：虽然 Vue 3 全量包比 Vue 2 大，但通过 Tree-shaking，实际项目中 Vue 3 的打包体积通常**更小**。

## 七、SSR 编译优化

Vue 3 在 SSR 场景下有额外的编译优化：

### 静态内容直接输出字符串

```javascript
// CSR 编译
export function render() {
  return h('div', [
    h('h1', '标题'),         // 创建 VNode
    h('p', _ctx.message)     // 创建 VNode
  ])
}

// SSR 编译
export function ssrRender(_ctx, _push) {
  _push('<div><h1>标题</h1>')        // 静态部分直接拼接字符串
  _push(`<p>${_ctx.message}</p>`)    // 动态部分插值
  _push('</div>')
}
```

> **优势**：SSR 时跳过虚拟 DOM 创建和 Diff，静态内容直接输出 HTML 字符串，速度提升数倍。

## 八、编译优化总结

| 优化策略 | 作用 | 适用阶段 |
|---------|------|---------|
| **静态提升** | 静态节点只创建一次 | CSR + SSR |
| **PatchFlag** | 精确标记动态部分，Diff 时只比较标记部分 | CSR |
| **Block Tree** | 收集动态节点，靶向更新 | CSR |
| **静态字符串化** | 大量静态内容序列化为字符串 | CSR + SSR |
| **Tree-shaking** | 移除未使用的 API | 构建 |
| **SSR 字符串输出** | 静态内容直接拼接为 HTML | SSR |

## 九、面试常见问题

### Q1：Vue 3 的编译优化有哪些？

核心优化有五项：
1. **静态提升**：静态节点/属性提升到渲染函数外部，只创建一次
2. **PatchFlag**：为动态节点标记变化的类型，Diff 时精确比较
3. **Block Tree**：收集动态节点到数组，更新时只遍历动态节点
4. **静态字符串化**：大量静态内容直接序列化为 HTML 字符串
5. **Tree-shaking**：具名导出 API，未使用的代码不打包

### Q2：什么是 Block？为什么需要 Block？

Block 是 Vue 3 中带有 **dynamicChildren** 数组的特殊 VNode。普通 Diff 需要遍历整棵 VNode 树，而 Block 只遍历 `dynamicChildren` 数组中的动态节点。这就像是在说"只有这几个地方会变，其他都不用看"。

### Q3：PatchFlag 具体是怎么工作的？

编译器在编译模板时，分析每个节点的动态部分，用位运算的组合标记。比如 `TEXT | CLASS = 3` 表示文本内容和 class 绑定是动态的。运行时 Diff 时，只对被标记的属性进行比较，跳过所有静态属性。

### Q4：Vue 3 的 Tree-shaking 为什么比 Vue 2 好？

Vue 2 将所有 API 挂载在 `Vue` 构造函数上，打包工具无法分析哪些被使用了。Vue 3 改为具名导出（`import { ref } from 'vue'`），构建工具可以精确分析依赖关系，移除未使用的代码。这意味着如果你不用 Transition，它的代码就不会出现在你的打包产物中。
