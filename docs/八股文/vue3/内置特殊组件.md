## 一、Teleport — 传送门

> 官方文档：[Teleport](https://cn.vuejs.org/guide/built-ins/teleport.html)

### 概念

Teleport 允许将组件模板的一部分**传送到 DOM 中的其他位置**，但逻辑上仍然属于当前组件。

```
组件结构：                          实际 DOM 结构：
┌─ ModalComponent ──────┐         ┌─ body ─────────────────┐
│  <Teleport to="body">  │         │  ├── ...               │
│    <div class="modal">  │ ───→    │  └── div.modal ✅      │
│      弹窗内容           │         │                        │
│    </div>               │         └────────────────────────┘
│  </Teleport>            │
│  <button>打开</button>  │         模态框出现在 body 下
└─────────────────────────┘         不受父元素 overflow 影响
```

### 使用场景

| 场景 | 为什么需要 Teleport |
|------|-------------------|
| 模态框/弹窗 | 避免被父元素 `overflow: hidden` 裁切 |
| 全局通知/Toast | 渲染到固定位置，不受组件层级影响 |
| 全屏遮罩 | 确保遮罩覆盖整个页面 |
| 下拉菜单 | 防止被容器裁切 |

### 基本用法

```vue
<template>
  <button @click="showModal = true">打开弹窗</button>

  <Teleport to="body">
    <div v-if="showModal" class="modal-overlay">
      <div class="modal-content">
        <h2>弹窗标题</h2>
        <p>弹窗内容</p>
        <button @click="showModal = false">关闭</button>
      </div>
    </div>
  </Teleport>
</template>

<script setup>
import { ref } from 'vue'
const showModal = ref(false)
</script>
```

### 高级用法

```vue
<!-- 传送目标也可以是选择器 -->
<Teleport to="#modal-container">...</Teleport>
<Teleport to=".modals">...</Teleport>

<!-- disabled 控制是否传送 -->
<Teleport to="body" :disabled="isMobile">
  <!-- 移动端不传送，桌面端传送到 body -->
  <ModalContent />
</Teleport>

<!-- 多个 Teleport 传送到同一目标，按顺序追加 -->
<Teleport to="#modals">
  <p>A</p>  <!-- 先渲染 -->
</Teleport>
<Teleport to="#modals">
  <p>B</p>  <!-- 后追加 -->
</Teleport>
<!-- 结果: <div id="modals"><p>A</p><p>B</p></div> -->
```

## 二、Suspense — 异步协调

> 官方文档：[Suspense](https://cn.vuejs.org/guide/built-ins/suspense.html)

### 概念

Suspense 用于协调组件树中的**异步依赖**，在异步内容加载完成前显示 fallback 内容。

```
┌─ Suspense ──────────────────────────┐
│                                      │
│  ┌─ default slot ────────────────┐  │
│  │  <AsyncComponent />            │  │
│  │  （正在加载...）                │  │
│  └────────────────────────────────┘  │
│          │ 加载中                     │
│          ▼                           │
│  ┌─ fallback slot ───────────────┐  │
│  │  <LoadingSpinner />            │  │
│  └────────────────────────────────┘  │
│                                      │
│  加载完成后：显示 default，隐藏 fallback
└──────────────────────────────────────┘
```

### 使用场景

1. **异步组件加载**：`defineAsyncComponent`
2. **异步 setup**：`<script setup>` 中使用 `await`
3. **异步数据获取**：组件内 `async setup()`

### 基本用法 — 异步组件

```vue
<template>
  <Suspense>
    <template #default>
      <AsyncPage />
    </template>
    <template #fallback>
      <div class="loading">加载中...</div>
    </template>
  </Suspense>
</template>

<script setup>
import { defineAsyncComponent } from 'vue'

const AsyncPage = defineAsyncComponent(() =>
  import('./HeavyPage.vue')
)
</script>
```

### 异步 setup

```vue
<!-- AsyncComponent.vue -->
<script setup>
// <script setup> 中的顶层 await 会让组件变为异步组件
const data = await fetch('/api/data').then(r => r.json())
</script>

<template>
  <div>{{ data.title }}</div>
</template>

<!-- Parent.vue -->
<template>
  <Suspense>
    <template #default>
      <AsyncComponent />
    </template>
    <template #fallback>
      <p>数据加载中...</p>
    </template>
  </Suspense>
</template>
```

### 事件监听

```vue
<template>
  <Suspense @resolve="onResolve" @pending="onPending" @fallback="onFallback">
    <template #default>
      <AsyncComponent />
    </template>
    <template #fallback>
      <Loading />
    </template>
  </Suspense>
</template>

<script setup>
function onResolve() {
  console.log('异步内容加载完成')
}
function onPending() {
  console.log('出现新的异步依赖')
}
function onFallback() {
  console.log('显示 fallback 内容')
}
</script>
```

## 三、KeepAlive — 组件缓存

> 官方文档：[KeepAlive](https://cn.vuejs.org/guide/built-ins/keep-alive.html)

### 概念

`<KeepAlive>` 包裹动态组件时，会**缓存不活跃的组件实例**，避免反复重新创建和销毁。

```
不使用 KeepAlive：                 使用 KeepAlive：
切换 Tab → 销毁旧组件              切换 Tab → 停用旧组件（缓存）
       → 创建新组件                       → 激活新组件（从缓存中取）
       → 重新发请求                       → 保留之前的状态 ✅
       → 丢失所有状态 ❌
```

### 基本用法

```vue
<template>
  <div class="tabs">
    <button
      v-for="tab in tabs"
      :key="tab"
      @click="currentTab = tab"
    >
      {{ tab }}
    </button>
  </div>

  <KeepAlive>
    <component :is="currentTabComponent" />
  </KeepAlive>
</template>

<script setup>
import { ref, computed, shallowRef } from 'vue'
import TabHome from './TabHome.vue'
import TabProfile from './TabProfile.vue'
import TabSettings from './TabSettings.vue'

const currentTab = ref('Home')
const tabs = { Home: TabHome, Profile: TabProfile, Settings: TabSettings }
const currentTabComponent = computed(() => tabs[currentTab.value])
</script>
```

### Props 配置

```vue
<!-- include：只缓存匹配的组件 -->
<KeepAlive :include="['TabHome', 'TabProfile']">
  <component :is="current" />
</KeepAlive>

<!-- exclude：不缓存匹配的组件 -->
<KeepAlive :exclude="['TabSettings']">
  <component :is="current" />
</KeepAlive>

<!-- max：最大缓存实例数（LRU 策略） -->
<KeepAlive :max="10">
  <component :is="current" />
</KeepAlive>
```

| Prop | 类型 | 说明 |
|------|------|------|
| `include` | `string \| RegExp \| string[]` | 只缓存匹配的组件 |
| `exclude` | `string \| RegExp \| string[]` | 不缓存匹配的组件 |
| `max` | `number` | 最大缓存数量，超出时销毁最久未使用的 |

### 匹配规则

```vue
<!-- 字符串匹配组件 name -->
<KeepAlive include="TabHome">

<!-- 正则匹配 -->
<KeepAlive :include="/^Tab/">

<!-- 数组匹配 -->
<KeepAlive :include="['TabHome', 'TabProfile']">
```

> **注意**：匹配的是组件的 `name` 选项。`<script setup>` 组件默认使用文件名作为 name，也可以用 `defineOptions({ name: 'TabHome' })` 显式指定。

### 配合生命周期

```vue
<script setup>
import { onActivated, onDeactivated } from 'vue'

onActivated(() => {
  console.log('组件被激活（从缓存中恢复）')
  // 恢复定时器、重新获取数据等
})

onDeactivated(() => {
  console.log('组件被停用（进入缓存）')
  // 暂停定时器、保存滚动位置等
})
</script>
```

## 四、Transition — 过渡动画

> 官方文档：[Transition](https://cn.vuejs.org/guide/built-ins/transition.html)

### 基本用法

```vue
<template>
  <button @click="show = !show">切换</button>

  <Transition name="fade">
    <p v-if="show">Hello Vue 3</p>
  </Transition>
</template>

<script setup>
import { ref } from 'vue'
const show = ref(true)
</script>

<style>
/* 进入和离开的过渡 */
.fade-enter-active,
.fade-leave-active {
  transition: opacity 0.3s ease;
}

.fade-enter-from,
.fade-leave-to {
  opacity: 0;
}
</style>
```

### 过渡类名

```
时间线：
──────────────────────────────────────────→
                持续时间
┌─────────────────────────────────────┐
│  v-enter-from ──→ v-enter-to        │  进入
│  (v-enter-active)                    │
├─────────────────────────────────────┤
│  v-leave-from ──→ v-leave-to        │  离开
│  (v-leave-active)                    │
└─────────────────────────────────────┘
```

| 类名 | 说明 |
|------|------|
| `v-enter-from` | 进入动画的起始状态 |
| `v-enter-active` | 进入动画的生效状态（定义 transition） |
| `v-enter-to` | 进入动画的结束状态 |
| `v-leave-from` | 离开动画的起始状态 |
| `v-leave-active` | 离开动画的生效状态 |
| `v-leave-to` | 离开动画的结束状态 |

### 自定义过渡类名

```vue
<!-- 使用 CSS 库（如 Animate.css） -->
<Transition
  enter-active-class="animate__animated animate__fadeIn"
  leave-active-class="animate__animated animate__fadeOut"
>
  <p v-if="show">Hello</p>
</Transition>
```

### JavaScript 钩子

```vue
<Transition
  @before-enter="onBeforeEnter"
  @enter="onEnter"
  @after-enter="onAfterEnter"
  @before-leave="onBeforeLeave"
  @leave="onLeave"
  @after-leave="onAfterLeave"
>
  <div v-if="show">Content</div>
</Transition>

<script setup>
function onBeforeEnter(el) {
  el.style.opacity = 0
}

function onEnter(el, done) {
  // 使用 GSAP 或其他动画库
  gsap.to(el, {
    opacity: 1,
    duration: 0.5,
    onComplete: done  // 必须调用 done 回调
  })
}

function onLeave(el, done) {
  gsap.to(el, {
    opacity: 0,
    duration: 0.5,
    onComplete: done
  })
}
</script>
```

### TransitionGroup

用于列表的过渡动画：

```vue
<template>
  <TransitionGroup name="list" tag="ul">
    <li v-for="item in items" :key="item.id">
      {{ item.text }}
    </li>
  </TransitionGroup>
</template>

<style>
.list-enter-active,
.list-leave-active {
  transition: all 0.5s ease;
}
.list-enter-from,
.list-leave-to {
  opacity: 0;
  transform: translateX(30px);
}
/* 移动动画 */
.list-move {
  transition: transform 0.5s ease;
}
</style>
```

## 五、Transition vs TransitionGroup

| 维度 | `<Transition>` | `<TransitionGroup>` |
|------|---------------|-------------------|
| 适用场景 | 单个元素/组件的显示隐藏 | 列表项的增删移动 |
| key 要求 | 不需要 | ✅ **必须**设置唯一 key |
| 根元素 | 默认不渲染根元素 | 通过 `tag` 渲染一个根元素 |
| 移动动画 | 不支持 | ✅ 支持 `.v-move` 类 |
| 模式控制 | `mode="out-in"` / `"in-out"` | 不支持（同时进行） |

## 六、面试常见问题

### Q1：Teleport 的 to 属性支持哪些值？

支持任何有效的 **CSS 选择器字符串**或 **DOM 元素**：
- `"body"` → `document.body`
- `"#modal-root"` → `document.getElementById('modal-root')`
- `".modals"` → `document.querySelector('.modals')`
- 也可以绑定到 ref 引用的 DOM 元素

目标元素必须**已存在于 DOM 中**。

### Q2：KeepAlive 的 max 是怎么实现的？

使用 **LRU（最近最少使用）** 缓存策略：
- 超过 `max` 时，销毁**最久没有被访问**的缓存组件
- 每次激活组件时，将其移到缓存链表末尾
- 缓存键为组件的 `type + key`（v-for 的 key 也参与区分）

### Q3：Suspense 目前是稳定功能吗？

Vue 3 中 Suspense 是一个**实验性功能**。它主要用于异步数据获取和异步组件加载场景。官方建议：
- 简单的异步组件加载 → 可以使用
- 复杂的嵌套异步场景 → 谨慎使用，可能 API 会变化
- 服务端渲染中的异步数据获取 → 配合 `async setup()` 使用

### Q4：Transition 的 mode 有什么作用？

`mode` 控制进入和离开动画的时序：
- **`out-in`**（推荐）：当前元素先离开，新元素再进入
- **`in-out`**：新元素先进入，当前元素再离开
- **不设置**：进入和离开同时进行

`out-in` 是最常用的模式，避免两个元素同时出现造成的布局抖动。
