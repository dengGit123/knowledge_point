# 装饰器（Decorators）

> TypeScript 装饰器是一种特殊声明，可以附加到类、方法、属性、访问器或参数上，以修改其行为。
>
> 官方文档：[TypeScript Decorators](https://www.typescriptlang.org/docs/handbook/decorators.html)

---

## 一、什么是装饰器

### 1.1 通俗解释

想象你出门前穿衣服的过程：

```
┌──────────────────────────────────────────────────┐
│  你（原始函数/类）                                  │
│    ↓                                              │
│  穿上外套（第一个装饰器：添加日志功能）               │
│    ↓                                              │
│  戴上帽子（第二个装饰器：添加权限校验）               │
│    ↓                                              │
│  穿上鞋子（第三个装饰器：添加性能监控）               │
│    ↓                                              │
│  最终出门的你 = 拥有所有附加能力的你                  │
└──────────────────────────────────────────────────┘
```

**装饰器本质上就是一个函数**，它"包裹"在目标（类、方法、属性等）外面，在不修改原始代码的前提下，为它添加额外功能。

### 1.2 装饰器模式概念

装饰器模式（Decorator Pattern）是一种结构型设计模式：

```
┌────────────┐     ┌────────────┐     ┌────────────┐
│  Component │◄────│ Decorator  │◄────│ConcreteDec │
│  (接口)     │     │  (装饰基类) │     │ (具体装饰器)│
└────────────┘     └────────────┘     └────────────┘
      ↑                                     │
      │              ┌────────────┐         │
      └──────────────│ConcreteComp│◄────────┘
                     │ (被装饰对象)│
                     └────────────┘
```

核心思想：**动态地给对象添加额外的职责**，而不改变其本身。

### 1.3 装饰器的发展历程

| 阶段 | 名称 | 说明 |
|------|------|------|
| TypeScript 早期 | **Legacy 装饰器**（`experimentalDecorators`） | 基于 TC39 Stage 1/2 提案，TypeScript 独有实现 |
| TC39 Stage 3 | **标准装饰器**（Stage 3 Decorators） | 2022-2023 年推进，语法和行为有较大变化 |
| TypeScript 5.0+ | 原生支持 Stage 3 | 通过 `compilerOptions` 区分两种模式 |

**tsconfig.json 配置对比：**

```jsonc
// Legacy 装饰器（TS 5.0 之前 / 仍可使用）
{
  "compilerOptions": {
    "experimentalDecorators": true,        // 启用 Legacy 装饰器
    "emitDecoratorMetadata": true          // 启用元数据发射
  }
}

// Stage 3 装饰器（TS 5.0+）
{
  "compilerOptions": {
    // 不设置 experimentalDecorators 即使用 Stage 3
    "target": "ES2022",
    "lib": ["ES2022", "DOM"]
  }
}
```

> 注意：本文前半部分（第二章到第八章）主要讲解 **Legacy 装饰器**，因为它是目前生态中使用最广泛的版本。第十章专门介绍 Stage 3 新版装饰器。

## 二、装饰器工厂

### 2.1 什么是装饰器工厂

装饰器工厂是一个**返回装饰器函数的函数**。它的作用是让装饰器可以接收参数，实现可配置化。

```typescript
// 普通装饰器：直接是装饰器函数
function log(target: any, key: string, descriptor: PropertyDescriptor) {
  // ...
}

// 装饰器工厂：返回装饰器函数
function logWithOptions(message: string) {
  // ← 这是工厂函数，接收配置参数
  return function (target: any, key: string, descriptor: PropertyDescriptor) {
    // ← 这是真正的装饰器
    console.log(`[${message}] ${key} 被调用`);
  };
}
```

### 2.2 使用对比

```typescript
class UserService {
  // 普通装饰器 —— 没有参数
  @log
  getUser() {}

  // 装饰器工厂 —— 可以传入配置
  @logWithOptions("DEBUG")
  deleteUser() {}
}
```

**执行流程：**

```
@logWithOptions("DEBUG")
        │
        ▼
logWithOptions("DEBUG")    ← 第1步：调用工厂函数，传入参数
        │
        ▼
  返回装饰器函数            ← 第2步：工厂返回真正的装饰器
        │
        ▼
  装饰器被应用到 deleteUser  ← 第3步：装饰器装饰目标方法
```

### 2.3 完整示例：可配置的颜色装饰器

```typescript
function color(colorName: string) {
  return function (target: any, key: string, descriptor: PropertyDescriptor) {
    const originalMethod = descriptor.value;
    descriptor.value = function (...args: any[]) {
      const result = originalMethod.apply(this, args);
      console.log(`%c${key} 执行完毕，结果: ${result}`, `color: ${colorName}`);
      return result;
    };
  };
}

class Calculator {
  @color("red")
  add(a: number, b: number) {
    return a + b;
  }

  @color("blue")
  multiply(a: number, b: number) {
    return a * b;
  }
}
```

## 三、类装饰器

### 3.1 语法和参数

类装饰器在类声明之前声明，应用于**类构造函数**。

```typescript
function classDecorator(constructor: Function) {
  // constructor 就是类的构造函数
  console.log("被装饰的类：", constructor.name);
}

@classDecorator
class MyClass {}
```

**参数说明：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `constructor` | `Function` | 类的构造函数 |

### 3.2 修改类定义

通过类装饰器，可以给类添加属性或方法：

```typescript
function addTimestamp<T extends { new (...args: any[]): {} }>(constructor: T) {
  return class extends constructor {
    createdAt = new Date().toISOString();

    getCreationInfo() {
      return `${constructor.name} 创建于 ${this.createdAt}`;
    }
  };
}

@addTimestamp
class User {
  constructor(public name: string) {}
}

const user = new User("张三");
console.log((user as any).createdAt);      // "2026-06-05T..."
console.log((user as any).getCreationInfo()); // "User 创建于 2026-06-05T..."
```

### 3.3 替换构造函数

类装饰器可以返回一个新的构造函数，**完全替换**原来的类：

```typescript
function sealed(constructor: Function) {
  Object.seal(constructor);       // 密封构造函数，不可添加属性
  Object.seal(constructor.prototype); // 密封原型，不可添加方法
}

@sealed
class ConfigManager {
  static version = "1.0.0";

  getConfig() {
    return { env: "production" };
  }
}

// 以下操作会静默失败（非严格模式）或抛出错误（严格模式）
// (ConfigManager as any).newProp = "test";
```

### 3.4 类装饰器完整示例：单例模式

```typescript
function Singleton<T extends { new (...args: any[]): {} }>(constructor: T) {
  let instance: any = null;

  return class extends constructor {
    constructor(...args: any[]) {
      if (instance) {
        return instance;
      }
      super(...args);
      instance = this;
    }
  };
}

@Singleton
class DatabaseConnection {
  constructor(public connectionString: string) {
    console.log(`连接数据库: ${connectionString}`);
  }

  query(sql: string) {
    return `执行查询: ${sql}`;
  }
}

const db1 = new DatabaseConnection("mysql://localhost:3306/db1");
const db2 = new DatabaseConnection("mysql://localhost:3306/db2");

console.log(db1 === db2); // true —— 始终返回同一个实例
```

## 四、方法装饰器

### 4.1 语法和参数

方法装饰器声明在方法声明之前，应用于方法的 **Property Descriptor**（属性描述符）。

```typescript
function methodDecorator(
  target: Object,             // 对于静态方法是类的构造函数，对于实例方法是原型对象
  propertyKey: string | symbol, // 方法名
  descriptor: PropertyDescriptor // 属性描述符
) {
  // ...
}
```

**参数详解：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `target` | `Object` | 静态方法 → 类的构造函数；实例方法 → 类的原型 `prototype` |
| `propertyKey` | `string \| symbol` | 方法的名称 |
| `descriptor` | `PropertyDescriptor` | 方法的属性描述符（包含 `value`、`writable`、`configurable` 等） |

### 4.2 Property Descriptor 结构

```
PropertyDescriptor
├── value          → 方法的实际函数体
├── writable       → 是否可修改（默认 true）
├── enumerable     → 是否可枚举（默认 false）
├── configurable   → 是否可配置/删除（默认 true）
├── get            → getter 函数
└── set            → setter 函数
```

### 4.3 修改方法行为 —— 日志装饰器

```typescript
function log(target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
  const originalMethod = descriptor.value;

  descriptor.value = function (...args: any[]) {
    console.log(`📞 调用方法: ${String(propertyKey)}`);
    console.log(`   参数列表:`, args);
    const result = originalMethod.apply(this, args);
    console.log(`   返回结果:`, result);
    return result;
  };

  return descriptor;
}

class OrderService {
  @log
  createOrder(productName: string, quantity: number) {
    return { productName, quantity, orderId: Date.now() };
  }
}

const service = new OrderService();
service.createOrder("TypeScript 实战", 2);
// 输出：
// 📞 调用方法: createOrder
//    参数列表: ["TypeScript 实战", 2]
//    返回结果: { productName: "TypeScript 实战", quantity: 2, orderId: 1749... }
```

### 4.4 实际应用 —— 权限校验

```typescript
interface UserContext {
  role: "admin" | "user" | "guest";
  name: string;
}

// 当前用户上下文（实际项目中可能来自 Vue 的 Pinia/Vuex）
let currentUser: UserContext = { role: "admin", name: "管理员" };

function requireRole(...roles: string[]) {
  return function (target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
    const originalMethod = descriptor.value;

    descriptor.value = function (...args: any[]) {
      if (!roles.includes(currentUser.role)) {
        throw new Error(`⛔ 权限不足！需要 [${roles.join(", ")}]，当前角色: ${currentUser.role}`);
      }
      return originalMethod.apply(this, args);
    };
  };
}

class AdminPanel {
  @requireRole("admin")
  deleteUser(userId: string) {
    console.log(`用户 ${userId} 已被删除`);
  }

  @requireRole("admin", "user")
  viewDashboard() {
    console.log("查看仪表盘");
  }
}

const panel = new AdminPanel();
panel.deleteUser("u123"); // 正常执行（当前是 admin）

currentUser = { role: "guest", name: "游客" };
panel.deleteUser("u123"); // Error: ⛔ 权限不足！
```

### 4.5 实际应用 —— 缓存装饰器（Memoization）

```typescript
function memoize(target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
  const originalMethod = descriptor.value;
  const cache = new Map<string, any>();

  descriptor.value = function (...args: any[]) {
    const key = JSON.stringify(args);

    if (cache.has(key)) {
      console.log(`💾 缓存命中: ${String(propertyKey)}(${key})`);
      return cache.get(key);
    }

    const result = originalMethod.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

class MathService {
  @memoize
  fibonacci(n: number): number {
    if (n <= 1) return n;
    return this.fibonacci(n - 1) + this.fibonacci(n - 2);
  }

  @memoize
  expensiveCalculation(x: number, y: number) {
    console.log("执行耗时计算...");
    return x * y + Math.sqrt(x + y);
  }
}

const math = new MathService();
math.expensiveCalculation(10, 20); // 输出: "执行耗时计算..." → 结果
math.expensiveCalculation(10, 20); // 输出: "💾 缓存命中..." → 直接返回结果
```

### 4.6 实际应用 —— 性能监控（防抖 / 节流）

```typescript
function debounce(delay: number) {
  return function (target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
    const originalMethod = descriptor.value;
    let timer: ReturnType<typeof setTimeout> | null = null;

    descriptor.value = function (...args: any[]) {
      if (timer) clearTimeout(timer);
      timer = setTimeout(() => {
        originalMethod.apply(this, args);
      }, delay);
    };
  };
}

class SearchComponent {
  @debounce(300)
  handleSearch(keyword: string) {
    console.log(`搜索关键词: ${keyword}`);
    // 实际项目中这里会调用 API
  }
}

const search = new SearchComponent();
// 连续快速输入，只有最后一次会在 300ms 后触发
search.handleSearch("T");
search.handleSearch("Ty");
search.handleSearch("Typ");
search.handleSearch("TypeScript");
// 300ms 后输出: "搜索关键词: TypeScript"
```

## 五、属性装饰器

### 5.1 语法和参数

```typescript
function propertyDecorator(
  target: Object,               // 静态属性 → 构造函数；实例属性 → 原型
  propertyKey: string | symbol  // 属性名
) {
  console.log(`属性 ${String(propertyKey)} 被装饰`);
}
```

**注意：** 属性装饰器**只有两个参数**，没有 `descriptor`，因为在类声明阶段实例属性还没有被创建。

### 5.2 基本使用

```typescript
function validateType(expectedType: string) {
  return function (target: Object, propertyKey: string) {
    // 在原型上存储元信息
    const existingKeys = Reflect.getMetadata("design:validate", target) || [];
    existingKeys.push({ propertyKey, expectedType });
    Reflect.defineMetadata("design:validate", existingKeys, target);
  };
}

class UserModel {
  @validateType("string")
  name!: string;

  @validateType("number")
  age!: number;
}
```

### 5.3 元数据注入

属性装饰器常配合 `reflect-metadata` 使用，实现元数据注入：

```typescript
// 安装：npm install reflect-metadata
import "reflect-metadata";

const formatMetadataKey = Symbol("format");

function format(formatString: string) {
  return Reflect.metadata(formatMetadataKey, formatString);
}

function getFormat(target: Object, propertyKey: string) {
  return Reflect.getMetadata(formatMetadataKey, target, propertyKey);
}

class Employee {
  @format("Hello, %s")
  name!: string;

  @format("Age: %d")
  age!: number;
}

const emp = new Employee();
console.log(getFormat(emp, "name")); // "Hello, %s"
console.log(getFormat(emp, "age"));  // "Age: %d"
```

**元数据流程图：**

```
@format("Hello, %s")
        │
        ▼
Reflect.metadata(key, "Hello, %s")
        │
        ▼
将元数据存储到 target 的内部 Map 中
        │
        ▼
getFormat() 通过 Reflect.getMetadata 读取出来
```

## 六、访问器装饰器

### 6.1 语法

访问器装饰器应用于 getter 或 setter，参数与方法装饰器相同。

```typescript
function accessorDecorator(
  target: Object,
  propertyKey: string | symbol,
  descriptor: PropertyDescriptor
) {
  // 可以修改 descriptor.get 或 descriptor.set
}
```

### 6.2 实际示例：属性值验证

```typescript
function validateRange(min: number, max: number) {
  return function (target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
    const originalSetter = descriptor.set!;

    descriptor.set = function (value: number) {
      if (value < min || value > max) {
        throw new RangeError(`${String(propertyKey)} 的值必须在 ${min} ~ ${max} 之间，收到: ${value}`);
      }
      originalSetter.call(this, value);
    };
  };
}

class Temperature {
  private _celsius = 0;

  @validateRange(-273.15, 1000)
  get celsius() {
    return this._celsius;
  }
  set celsius(value: number) {
    this._celsius = value;
  }
}

const temp = new Temperature();
temp.celsius = 25;       // 正常
temp.celsius = -300;     // Error: RangeError: celsius 的值必须在 -273.15 ~ 1000 之间
```

### 6.3 自动格式化

```typescript
function trim(target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
  const originalSetter = descriptor.set!;

  descriptor.set = function (value: string) {
    const trimmed = value.trim();
    originalSetter.call(this, trimmed);
  };
}

class UserProfile {
  private _username = "";

  @trim
  get username() {
    return this._username;
  }
  set username(value: string) {
    this._username = value;
  }
}

const profile = new UserProfile();
profile.username = "  张三  ";
console.log(profile.username); // "张三" —— 前后空格被自动去除
```

## 七、参数装饰器

### 7.1 语法和参数

```typescript
function parameterDecorator(
  target: Object,                // 静态方法 → 构造函数；实例方法 → 原型
  propertyKey: string | symbol,  // 方法名
  parameterIndex: number         // 参数在参数列表中的位置（从0开始）
) {
  console.log(`方法 ${String(propertyKey)} 的第 ${parameterIndex} 个参数被装饰`);
}
```

### 7.2 基本使用

```typescript
function logParam(target: Object, propertyKey: string, parameterIndex: number) {
  console.log(`参数装饰器触发：${propertyKey} 的第 ${parameterIndex} 个参数`);
}

class ApiService {
  fetchData(@logParam url: string, @logParam options?: RequestInit) {
    // ...
  }
}
// 输出：
// 参数装饰器触发：fetchData 的第 1 个参数（options）
// 参数装饰器触发：fetchData 的第 0 个参数（url）
// 注意：参数装饰器从后往前执行！
```

### 7.3 实际应用：参数标记 + 方法装饰器配合

```typescript
import "reflect-metadata";

const requiredMetadataKey = Symbol("required");

// 参数装饰器：标记哪些参数是必填的
function required(target: Object, propertyKey: string | symbol, parameterIndex: number) {
  const existingRequired: number[] =
    Reflect.getOwnMetadata(requiredMetadataKey, target, propertyKey) || [];
  existingRequired.push(parameterIndex);
  Reflect.defineMetadata(requiredMetadataKey, existingRequired, target, propertyKey);
}

// 方法装饰器：读取标记，执行校验
function validateParams(target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
  const originalMethod = descriptor.value;

  descriptor.value = function (...args: any[]) {
    const requiredParams: number[] =
      Reflect.getOwnMetadata(requiredMetadataKey, target, propertyKey) || [];

    requiredParams.forEach((index) => {
      if (args[index] === undefined || args[index] === null) {
        throw new Error(`参数校验失败：${String(propertyKey)} 的第 ${index} 个参数不能为空`);
      }
    });

    return originalMethod.apply(this, args);
  };
}

class UserService {
  @validateParams
  createUser(@required name: string, @required email: string, age?: number) {
    console.log(`创建用户: ${name}, ${email}, ${age}`);
  }
}

const userService = new UserService();
userService.createUser("张三", "zhangsan@example.com", 25); // 正常
userService.createUser("张三", undefined as any);           // Error: 参数校验失败
```

**协作流程：**

```
┌──────────────────────────────────────────────────────┐
│  @required 在参数上打标签                               │
│     ↓                                                 │
│  将参数索引存入 Reflect.metadata                        │
│     ↓                                                 │
│  @validateParams 在方法上读取标签                        │
│     ↓                                                 │
│  在方法调用时检查标记的参数是否为空                        │
│     ↓                                                 │
│  为空 → 抛出错误 | 不为空 → 执行原方法                    │
└──────────────────────────────────────────────────────┘
```

## 八、装饰器执行顺序

### 8.1 多装饰器的执行顺序（同一个目标上）

当同一个目标上有多个装饰器时，执行顺序类似**洋葱模型**：

```typescript
function first() {
  console.log("first(): 工厂执行");
  return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    console.log("first(): 装饰器执行");
  };
}

function second() {
  console.log("second(): 工厂执行");
  return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    console.log("second(): 装饰器执行");
  };
}

class Example {
  @first()
  @second()
  method() {}
}

// 输出：
// second(): 工厂执行
// first(): 工厂执行
// second(): 装饰器执行    ← 从下往上（从靠近方法的开始）
// first(): 装饰器执行     ← 然后是外层的
```

**图解：**

```
@first()      ← 先声明（外层）
@second()     ← 后声明（内层）
method()

工厂执行顺序：second() → first()        （从下往上）
装饰器执行顺序：second() → first()       （从下往上，从内到外）

         ╭─── first() 工厂 ───╮
         │  ╭ second() 工厂 ╮ │
         │  │                │ │
         │  │  method()      │ │
         │  │                │ │
         │  ╰→ second 装饰器 ╯ │
         ╰──→ first 装饰器 ────╯
```

### 8.2 不同类型装饰器的执行顺序

对于同一个类中不同类型的装饰器，执行顺序如下：

```
执行阶段：类声明时（不是实例化时）

1️⃣ 实例属性装饰器         ← 按代码顺序
2️⃣ 实例方法/访问器 参数装饰器
3️⃣ 实例方法/访问器 装饰器   ← 从下往上
4️⃣ 静态属性装饰器
5️⃣ 静态方法/访问器 参数装饰器
6️⃣ 静态方法/访问器 装饰器
```

```typescript
import { Vue, Component, Prop } from "vue-property-decorator";

export default class UserProfile extends Vue {

  userName!: string;

  // 等价于 methods
  greet() {
    console.log(`你好，我是 ${this.userName}`);
  }

  // 等价于 mounted 生命周期
  mounted() {
    this.greet();
  }
}
```

```typescript
function trace(target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
  const original = descriptor.value;

  descriptor.value = function (...args: any[]) {
    const className = target.constructor.name;
    const timestamp = new Date().toISOString();

    }
  };
}
```

### 9.3 权限校验（完整版）

```typescript
import type { Ref } from "vue";
import { ref } from "vue";

// 权限上下文
const currentUserRole: Ref<string> = ref("guest");

function setRole(role: string) {
  currentUserRole.value = role;
}

// 装饰器工厂
function authorize(...allowedRoles: string[]) {
  return function (target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
    const original = descriptor.value;

    descriptor.value = function (...args: any[]) {
      if (!allowedRoles.includes(currentUserRole.value)) {
        console.warn(
          `[权限拦截] ${String(propertyKey)} 需要 [${allowedRoles}], 当前: ${currentUserRole.value}`
        );
        return null;
      }
      return original.apply(this, args);
    };
  };
}

class Dashboard {
  @authorize("admin", "editor")
  editArticle(id: string, content: string) {
    console.log(`文章 ${id} 已编辑`);
    return true;
  }

  @authorize("admin")
  deleteArticle(id: string) {
    console.log(`文章 ${id} 已删除`);
    return true;
  }

  @authorize("admin", "editor", "viewer")
  viewArticle(id: string) {
    console.log(`查看文章 ${id}`);
    return true;
  }
}

// 使用
const dashboard = new Dashboard();

setRole("viewer");
dashboard.viewArticle("001");   // ✅ 成功
dashboard.editArticle("001", "内容"); // ⚠️ 权限拦截

setRole("admin");
dashboard.deleteArticle("001"); // ✅ 成功
```

```typescript
function measure(target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
  const original = descriptor.value;

  descriptor.value = function (...args: any[]) {
    const start = performance.now();
    const result = original.apply(this, args);
    const end = performance.now();
    const duration = (end - start).toFixed(2);

    console.log(`⏱ ${String(propertyKey)} 耗时: ${duration}ms`);

    // 支持 async 函数
    if (result instanceof Promise) {
      return result.then((res) => {
        const asyncEnd = performance.now();
        console.log(`⏱ ${String(propertyKey)} (async) 总耗时: ${(asyncEnd - start).toFixed(2)}ms`);
        return res;
      });
    }

    return result;
  };
}

class DataService {

  syncProcess() {
    let sum = 0;
    for (let i = 0; i < 1_000_000; i++) sum += i;
    return sum;

    const response = await fetch("https://api.example.com/data");

}
```

### 9.5 缓存（进阶版：带 TTL）

```typescript
function cacheWithTTL(ttlMs: number) {
  return function (target: Object, propertyKey: string, descriptor: PropertyDescriptor) {
    const original = descriptor.value;
    const cache = new Map<string, { value: any; expiry: number }>();

    descriptor.value = function (...args: any[]) {
      const key = JSON.stringify(args);
      const cached = cache.get(key);

      if (cached && Date.now() < cached.expiry) {
        console.log(`💾 缓存命中: ${String(propertyKey)}`);
        return cached.value;
      }

      const result = original.apply(this, args);
      cache.set(key, { value: result, expiry: Date.now() + ttlMs });
      return result;
    };
  };
}

class WeatherService {
  @cacheWithTTL(5 * 60 * 1000) // 缓存 5 分钟
  getWeather(city: string) {
    console.log(`获取 ${city} 的天气...（模拟 API 调用）`);
    return { city, temperature: 26, condition: "晴" };
  }
}
```

## 十、Stage 3 装饰器（新版）

### 10.1 与 Legacy 装饰器的区别

| 特性 | Legacy 装饰器 | Stage 3 装饰器 |
|------|-------------|---------------|
| 配置项 | `experimentalDecorators: true` | 无需额外配置（TS 5.0+） |
| 类装饰器参数 | `(constructor: Function)` | `(value: Class, context: ClassDecoratorContext)` |
| 方法装饰器参数 | `(target, key, descriptor)` | `(value: Function, context: ClassMethodDecoratorContext)` |
| 能否修改 descriptor | ✅ 直接修改 | ❌ 返回新函数替换 |
| 元数据 | 依赖 `reflect-metadata` | 内置 `context.metadata` |
| 属性装饰器 | 只有 `target` 和 `key` | 有 `context`，可返回初始化函数 |
| 装饰器工厂 | 手动返回函数 | 同样支持 |
| ECMAScript 标准 | TypeScript 特有 | **TC39 标准提案** |

### 10.2 新语法概览

Stage 3 装饰器的核心变化：**增加了 `context` 对象**，提供了丰富的上下文信息。

#### context 对象通用属性

```
ClassMethodDecoratorContext / ClassFieldDecoratorContext
├── kind          → "method" | "field" | "class" | "getter" | "setter" | "accessor"
├── name          → 装饰目标的名称（string | symbol）
├── static        → 是否为静态成员
├── private       → 是否为私有成员
├── access        → { get(obj), set(obj, value) } 访问器对象
├── addInitializer(fn) → 添加初始化钩子
└── metadata      → 可读写的元数据对象（类级别共享）
```

#### 类装饰器

```typescript
// Stage 3 类装饰器
function logged(value: Function, context: ClassDecoratorContext) {
  // value: 类构造函数
  // context.kind === "class"
  console.log(`类 ${context.name} 被装饰`);

  return class extends (value as any) {
    constructor(...args: any[]) {
      super(...args);
      console.log(`${String(context.name)} 实例已创建`);
    }
  };
}

@logged
class MyService {}
```

#### 方法装饰器

```typescript
// Stage 3 方法装饰器
function trace(
  value: (this: any, ...args: any[]) => any,
  context: ClassMethodDecoratorContext
) {
  const name = String(context.name);

  return function (this: any, ...args: any[]) {
    console.log(`→ ${name} 被调用，参数:`, args);
    const result = value.call(this, ...args);
    console.log(`← ${name} 返回:`, result);
    return result;
  };
}

class Calculator {
  @trace
  add(a: number, b: number) {
    return a + b;
  }
}
```

#### 属性装饰器

```typescript
// Stage 3 属性装饰器 —— 可以返回初始化函数
function upperCase(
  value: undefined,
  context: ClassFieldDecoratorContext
) {
  return function (initialValue: any) {
    if (typeof initialValue === "string") {
      return initialValue.toUpperCase();
    }
    return initialValue;
  };
}

class User {
  @upperCase
  name = "zhangsan";
}

const user = new User();
console.log(user.name); // "ZHANGSAN"
```

#### 内置元数据

```typescript
// Stage 3 内置 metadata —— 不再需要 reflect-metadata
function collectMetadata(value: any, context: ClassFieldDecoratorContext) {
  context.metadata[String(context.name)] = typeof value;
}

class Product {
  @collectMetadata
  title = "默认标题";

  @collectMetadata
  price = 0;
}

// metadata 在类级别共享
console.log(Product); // 类上可以访问到收集到的元数据
```

## 十一、面试常见问题

### Q1：装饰器本质上是什么？

装饰器本质上就是一个**函数**。`@decorator` 语法只是函数调用的语法糖。`@log` 等价于在类定义后执行 `log(MyClass)`。

### Q2：装饰器能改变被装饰者的类型吗？

Legacy 装饰器不会改变 TypeScript 的类型推断。但类装饰器如果返回了一个新类，运行时行为会改变，类型系统不会自动感知，需要手动声明。

```typescript
function addVersion<T extends { new (...args: any[]): {} }>(constructor: T) {
  return class extends constructor {
    version = "1.0.0";
  };
}

// TypeScript 不知道 version 的存在，需要断言
@addVersion
class AppConfig {}

const config = new AppConfig();
console.log((config as any).version); // 需要 as any
```

### Q3：装饰器的执行时机是什么？

**装饰器在类定义时执行，而不是实例化时。** 这意味着装饰器逻辑只运行一次（类加载时），而不是每次 `new` 时。

```
类定义（代码解析时）
  ↓
装饰器执行（仅一次，收集信息 / 修改定义）
  ↓
new MyClass()（实例化时）
  ↓
被装饰后的方法被调用
```

### Q4：Legacy 装饰器和 Stage 3 装饰器最大的区别是什么？

| 维度 | Legacy | Stage 3 |
|------|--------|---------|
| 描述符操作 | 直接修改 descriptor | 返回新值替换 |
| 元数据 | 需要第三方库 reflect-metadata | 内置 context.metadata |
| 标准 | TypeScript 特有 | ECMAScript 标准提案 |
| 属性装饰器 | 无法拦截赋值 | 可返回初始化函数 |
| 兼容性 | 生态成熟 | 逐步推广中 |

### Q5：装饰器和高阶函数有什么区别？

```
装饰器                          高阶函数
─────────────                  ─────────
编译时绑定到类/方法上            运行时动态包装
通过 @ 语法声明                  通过函数调用包装
只能用于类相关的声明             可用于任何函数
TypeScript / JavaScript 特性    所有编程语言通用概念

// 装饰器
class A {
  @log              ← 声明式，绑定在方法上
  method() {}
}

// 高阶函数
const method = log(function method() {});
//                   ↑ 命令式，手动包装
```

### Q6：如何在 Vue 3 项目中合理使用装饰器？

Vue 3 官方推荐 Composition API（`<script setup>`），不推荐 Class 风格。但在以下场景装饰器仍有用武之地：

1. **工具类 / Service 类**：非组件的 TypeScript 类可以用装饰器（日志、缓存、校验）
2. **Pinia Store**：如果 Store 用 class 风格写，可用装饰器
3. **类型校验**：class-validator 风格的数据校验

```typescript
// 在 Vue 3 + Vite 项目中，Service 类使用装饰器
import { cacheWithTTL } from "./decorators";

export class WeatherService {
  @cacheWithTTL(5 * 60 * 1000)
  async getWeather(city: string) {
    const res = await fetch(`/api/weather?city=${city}`);
    return res.json();
  }
}

// 在 Vue 组件中使用（Composition API）
import { defineComponent, ref } from "vue";
import { WeatherService } from "./WeatherService";

export default defineComponent({
  setup() {
    const weather = ref(null);
    const service = new WeatherService();

    async function loadWeather() {
      weather.value = await service.getWeather("北京");
    }

    return { weather, loadWeather };
  },
});
```

### Q7：装饰器能装饰箭头函数吗？

**不能。** 装饰器只能用于类和类的成员（方法、属性、访问器、构造函数参数）。普通函数和箭头函数不能使用装饰器语法。

```typescript
// ❌ 不合法
@log
const myFunc = () => {};

// ✅ 合法 —— 类方法
class MyClass {
  @log
  myMethod() {}
}
```

### Q8：装饰器的洋葱模型是什么？

当多个装饰器叠加在同一个方法上时，调用过程形成洋葱模型 —— 外层装饰器先开始、后结束，内层装饰器后开始、先结束。

```
调用方向 →→→→→→→→→→→→→→→→→→→→→→
  ╭─── @decoratorA（外层）────────╮
  │  ╭─── @decoratorB（内层）───╮ │
  │  │     原始方法执行          │ │
  │  ╰──────────────────────────╯ │
  ╰───────────────────────────────╯
返回方向 ←←←←←←←←←←←←←←←←←←←←←←
```

```typescript
function wrap(label: string) {
  return function (target: any, key: string, descriptor: PropertyDescriptor) {
    const original = descriptor.value;
    descriptor.value = function (...args: any[]) {
      console.log(`[${label}] 开始`);
      const result = original.apply(this, args);
      console.log(`[${label}] 结束`);
      return result;
    };
  };
}

class Example {
  @wrap("A")     // 外层
  @wrap("B")     // 内层
  run() {
    console.log("  执行 run");
  }
}

new Example().run();
// [B] 开始
// [A] 开始
//   执行 run
// [A] 结束
// [B] 结束
```
