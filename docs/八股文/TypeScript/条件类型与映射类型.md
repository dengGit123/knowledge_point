## 一、条件类型（Conditional Types）

### 1.1 基本语法

```typescript
type Result = T extends U ? X : Y
```

```
         T extends U ?
        ┌─────┴─────┐
       yes          no
        │            │
        X            Y
```

**通俗解释**：条件类型就是 **类型层面的三元表达式**。

| 对比维度 | JS 三元表达式 | TS 条件类型 |
|---------|-------------|------------|
| 操作对象 | 值 (value) | 类型 (type) |
| 判断条件 | 布尔值 | `extends` 关系 |
| 返回结果 | 值 | 类型 |
| 执行时机 | 运行时 | 编译时 |

```typescript
// JS 三元表达式 —— 对值做判断
const value = condition ? 'yes' : 'no'

// TS 条件类型 —— 对类型做判断
type IsString<T> = T extends string ? 'yes' : 'no'

type A = IsString<string>  // 'yes'
type B = IsString<number>  // 'no'
```

类比：假设你是一个海关检查员（TypeScript 编译器），每个入境的人（类型 T）都要检查护照——
- 如果护照 nationality 是 U，走「免签通道」（X）
- 否则，走「签证通道」（Y）

### 1.2 实际使用场景

```typescript
// 判断是否为 string 类型
type IsString<T> = T extends string ? true : false
type T1 = IsString<'hello'>  // true
type T2 = IsString<42>       // false

// 根据输入类型返回不同类型
type TypeName<T> = T extends string
  ? 'string'
  : T extends number
  ? 'number'
  : T extends boolean
  ? 'boolean'
  : T extends undefined
  ? 'undefined'
  : T extends Function
  ? 'function'
  : 'object'

type N1 = TypeName<string>    // 'string'
type N2 = TypeName<() => {}>  // 'function'
type N3 = TypeName<number[]>  // 'object'
```

### 1.3 分布式条件类型（Distributive Conditional Types）

**核心规则**：当条件类型作用于 **裸类型参数（naked type parameter）** 时，会自动把联合类型 **拆开** 逐一判断，再把结果 **合并** 回去。

```
输入：T = string | number

  string | number extends string ? X : Y

         拆开 ↓ 分布式

  ┌─────────────────┬──────────────────┐
  string extends    │ number extends    │
  string ? X : Y   │ string ? X : Y   │
  = X              │ = Y              │
  └─────────────────┴──────────────────┘
         合并 ↓
         X | Y
```

```typescript
type ToArray<T> = T extends unknown ? T[] : never

// 分布式：先拆开 string | number，再各自包成数组，最后合并
type Result = ToArray<string | number>
// 等价于：ToArray<string> | ToArray<number>
// 等价于：string[] | number[]
```

**更直观的例子**：

```typescript
type WrapToArray<T> = T extends any ? T[] : never

type R1 = WrapToArray<string>           // string[]
type R2 = WrapToArray<string | number>  // string[] | number[]  ← 分布式！
type R3 = WrapToArray<boolean | null>   // boolean[] | null[]
```

> **关键点**：`string[] | number[]` 和 `(string | number)[]` 是不同的类型！

```typescript
// 对比
type Distributed = WrapToArray<string | number>   // string[] | number[]
type NonDistributed = (string | number)[]         // (string | number)[]

// 区别：
const a: Distributed = ['only strings']      // OK
const b: Distributed = [1, 2, 3]             // OK
const c: Distributed = ['mix', 1]            // Error! 不能混

const d: NonDistributed = ['mix', 1]         // OK!
```

### 1.4 非分布式条件类型

如果你 **不想** 分布式拆开，可以用 **元组 `[T]`** 包裹类型参数：

```typescript
// 分布式
type Distributed<T> = T extends string ? 'is-string' : 'not-string'
type D1 = Distributed<string | number>  // 'is-string' | 'not-string'

// 非分布式 —— 用 [T] 包裹
type NonDistributed<T> = [T] extends [string] ? 'is-string' : 'not-string'
type D2 = NonDistributed<string | number>  // 'not-string'  ← 整体判断！
```

**原理**：`[T]` 不再是裸类型参数，所以 TypeScript 不会进行分布式拆解。

```
分布式 (裸类型参数):
  (string | number) extends string ? ...
  → string extends string ? ... | number extends string ? ...
  → 'is-string' | 'not-string'

非分布式 ([T] 包裹):
  [string | number] extends [string] ? ...
  → 整体判断：[string | number] 能赋值给 [string] 吗？不能。
  → 'not-string'
```

**实际应用**——判断是否为 never：

```typescript
// 错误写法：分布式导致 never 被跳过，结果直接是 never
type IsNeverWrong<T> = T extends never ? true : false
type W = IsNeverWrong<never>  // never  ← 不是我们期望的 true！

// 正确写法：用 [T] 包裹避免分布式
type IsNever<T> = [T] extends [never] ? true : false
type R = IsNever<never>     // true  ✓
type R2 = IsNever<string>   // false ✓
```

## 二、infer 关键字

### 2.1 基本概念

`infer` 只能在 `extends` 条件类型的右侧使用，用于 **在类型匹配过程中声明一个「待推断的类型变量」**。

类比：`infer` 就像正则表达式中的 **捕获组** `()` —— 先匹配模式，再提取你关心的部分。

```typescript
type Match<T> = T extends ???? ? 对应的类型 : 其他处理
//                        ^^^^
//                  这里用 infer 声明一个「占位符」
```

### 2.2 提取函数返回类型

```typescript
// 手动实现 ReturnType
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never

type Fn = (x: number, y: string) => boolean
type R = MyReturnType<Fn>  // boolean

// TypeScript 内置的 ReturnType 工具类型就是这个原理
type R2 = ReturnType<Fn>   // boolean
```

```
函数类型签名：
  (x: number, y: string) => boolean
  ─────────────────────    ────────
        参数部分           返回类型
                        infer R 捕获 ↑
```

**Vue 中的实际应用**——提取 composable 的返回类型：

```typescript
// 一个 Vue composable
function useUser() {
  const name = ref('张三')
  const age = ref(25)
  const updateName = (n: string) => { name.value = n }
  return { name, age, updateName }
}

// 自动推断 composable 返回类型
type UseUserReturn = ReturnType<typeof useUser>
// { name: Ref<string>; age: Ref<number>; updateName: (n: string) => void }
```

### 2.3 提取 Promise 内部类型

```typescript
// 手动实现 Awaited（类似 TypeScript 内置）
type MyAwaited<T> = T extends Promise<infer U> ? MyAwaited<U> : T

type P1 = MyAwaited<Promise<string>>           // string
type P2 = MyAwaited<Promise<Promise<number>>>  // number（递归解包）
type P3 = MyAwaited<string>                    // string（非 Promise 直接返回）
```

```
Promise<Promise<Promise<string>>>
   ↓ 第1次 infer：Promise<Promise<string>>
   ↓ 第2次 infer：Promise<string>
   ↓ 第3次 infer：string   ← 最终结果
```

**在 async 函数中的应用**：

```typescript
// 一个 async 函数
async function fetchUser(id: number) {
  const res = await fetch(`/api/user/${id}`)
  return res.json() as Promise<{ name: string; age: number }>
}

// 提取异步函数的返回值类型（自动解包 Promise）
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T
type UserData = UnwrapPromise<ReturnType<typeof fetchUser>>
// { name: string; age: number }
```

### 2.4 提取数组元素类型

```typescript
// 提取数组元素的类型
type ElementOf<T> = T extends (infer E)[] ? E : never

type T1 = ElementOf<string[]>   // string
type T2 = ElementOf<number[]>   // number
type T3 = ElementOf<boolean>    // never（不是数组）

// 提取元组第一个元素的类型
type First<T extends any[]> = T extends [infer F, ...any[]] ? F : never

type F1 = First<[string, number, boolean]>  // string
type F2 = First<[42]>                       // 42
type F3 = First<[]>                         // never

// 提取元组最后一个元素的类型
type Last<T extends any[]> = T extends [...any[], infer L] ? L : never

type L1 = Last<[string, number, boolean]>  // boolean
type L2 = Last<[42]>                       // 42
```

### 2.5 提取构造函数实例类型

```typescript
// 获取构造函数创建的实例类型
type InstanceOf<T> = T extends new (...args: any[]) => infer I ? I : never

class Person {
  constructor(public name: string, public age: number) {}
  greet() { return `Hi, I'm ${this.name}` }
}

type PersonInstance = InstanceOf<typeof Person>
// Person  ← 拿到了完整的实例类型

// TypeScript 内置的 InstanceType 就是这个原理
type PersonInstance2 = InstanceType<typeof Person>
```

### 2.6 多位置 infer

`infer` 可以在 **多个位置** 同时使用，一次推断多个类型变量：

```typescript
// 同时提取函数的参数和返回值
type FnAnalysis<T> = T extends (...args: infer A) => infer R
  ? { params: A; return: R }
  : never

type Analyzed = FnAnalysis<(x: number, y: string) => boolean>
// { params: [number, string]; return: boolean }

// 交换函数的参数为元组，返回值为返回类型
type SplitFn<T> = T extends (...args: infer Args) => infer Return
  ? { args: Args; ret: Return }
  : never

// 提取 Promise 构造函数中 resolve 的类型
type ResolveType<T> = T extends Promise<infer V>
  ? V
  : T extends Thenable<infer V>
  ? V
  : T
```

**字符串模式匹配**（多 infer 实战）：

```typescript
// 提取字符串中 "前缀_后缀" 模式的两部分
type SplitByUnderscore<T extends string> =
  T extends `${infer Left}_${infer Right}`
    ? { left: Left; right: Right }
    : never

type S1 = SplitByUnderscore<'hello_world'>  // { left: 'hello'; right: 'world' }
type S2 = SplitByUnderscore<'foo_bar_baz'>  // { left: 'foo'; right: 'bar_baz' }

// 提取括号内的内容
type InsideParens<T extends string> =
  T extends `${infer Before}(${infer Inside})${infer After}`
    ? { before: Before; inside: Inside; after: After }
    : never

type P = InsideParens<'hello(world)end'>
// { before: 'hello'; inside: 'world'; after: 'end' }
```

## 三、映射类型（Mapped Types）

### 3.1 基本语法

```typescript
type NewType = {
  [K in keyof T]: NewValueType
}
```

类比：映射类型就像一条 **工厂流水线** —— 把旧类型拆成一个个属性（原料），对每个属性做同样的加工（处理），最后组装成新类型（成品）。

```
旧类型 T
  │
  ├── name: string    ──→  加工  ──→  name: 新类型
  ├── age: number     ──→  加工  ──→  age: 新类型
  └── email: string   ──→  加工  ──→  email: 新类型
  │
  ▼
新类型 { name: 新类型; age: 新类型; email: 新类型 }
```

```typescript
// 基础示例：把所有属性变成 string
type Stringify<T> = {
  [K in keyof T]: string
}

interface User {
  name: string
  age: number
  active: boolean
}

type StringUser = Stringify<User>
// { name: string; age: string; active: string }
```

### 3.2 遍历联合类型创建新类型

```typescript
// 从联合类型创建对象类型
type Flags = 'read' | 'write' | 'execute'

type Permission = {
  [K in Flags]: boolean
}
// { read: boolean; write: boolean; execute: boolean }

// 在 Vue i18n 中使用
type Locale = 'zh' | 'en' | 'ja'
type Messages = {
  [L in Locale]: string
}
// { zh: string; en: string; ja: string }
```

### 3.3 修饰符：readonly、?

映射类型支持对属性修饰符进行 **添加（+）** 或 **移除（-）** 操作。

| 修饰符 | 添加 | 移除 |
|-------|------|------|
| `readonly` | `+readonly`（可省略 +） | `-readonly` |
| 可选 `?` | `+?`（可省略 +） | `-?` |

```
原始类型:
  { name: string; readonly age: number; active?: boolean }

  +readonly          +?            -readonly       -?
  ↓                  ↓             ↓                ↓
所有属性变为:    所有属性变为:    所有readonly    所有可选属性
readonly         可选            被移除           变为必填
```

```typescript
// ---- 所有属性变为可选 ----
type MyPartial<T> = {
  [K in keyof T]?: T[K]
}

// ---- 所有属性变为必填 ----
type MyRequired<T> = {
  [K in keyof T]-?: T[K]
}

// ---- 所有属性变为只读 ----
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K]
}

// ---- 移除所有只读 ----
type Mutable<T> = {
  -readonly [K in keyof T]: T[K]
}

// 实际使用
interface Config {
  readonly host: string
  readonly port: number
  debug?: boolean
}

type MutableConfig = Mutable<Config>
// { host: string; port: number; debug?: boolean }  ← readonly 移除了

type RequiredConfig = MyRequired<Config>
// { readonly host: string; readonly port: number; debug: boolean }  ← debug 必填了
```

### 3.4 key 重映射（Key Remapping via `as`）

TypeScript 4.1 引入了 `as` 子句，可以在映射时 **重命名 key**：

```typescript
type NewType = {
  [K in keyof T as NewKey]: T[K]
}
```

```typescript
// 把所有属性名加上 "get" 前缀
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K]
}

interface Person {
  name: string
  age: number
}

type PersonGetters = Getters<Person>
// {
//   getName: () => string
//   getAge: () => number
// }
```

**Vue 中提取 emit 事件类型**：

```typescript
// 给事件名加上 "on" 前缀
type EventHandlers<T> = {
  [K in keyof T as `on${Capitalize<string & K>}`]: (payload: T[K]) => void
}

interface Events {
  click: MouseEvent
  change: string
  submit: FormData
}

type Handlers = EventHandlers<Events>
// {
//   onClick: (payload: MouseEvent) => void
//   onChange: (payload: string) => void
//   onSubmit: (payload: FormData) => void
// }
```

### 3.5 过滤属性（用 as + never）

当 `as` 后面的类型解析为 `never` 时，该属性会被 **直接移除**：

```typescript
// 只保留 string 类型的属性
type OnlyStrings<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K]
}

interface Mixed {
  name: string
  age: number
  email: string
  active: boolean
}

type OnlyStringFields = OnlyStrings<Mixed>
// { name: string; email: string }  ← age 和 active 被过滤掉了
```

```
Mixed 接口:
  name: string   ──→ string extends string ? 'name' : never ──→ 'name'  ✓ 保留
  age: number    ──→ number extends string ? 'age' : never  ──→ never  ✗ 移除
  email: string  ──→ string extends string ? 'email' : never ──→ 'email' ✓ 保留
  active: boolean ──→ boolean extends string ? 'active' : never ──→ never  ✗ 移除
```

**更多过滤示例**：

```typescript
// 移除函数类型的属性（提取数据属性）
type DataFields<T> = {
  [K in keyof T as T[K] extends Function ? never : K]: T[K]
}

interface ViewModel {
  name: string
  count: number
  increment: () => void
  reset: () => void
}

type DataOnly = DataFields<ViewModel>
// { name: string; count: number }

// 只保留以特定前缀开头的属性
type FilterByPrefix<T, Prefix extends string> = {
  [K in keyof T as K extends `${Prefix}${string}` ? K : never]: T[K]
}

interface ApiConfig {
  API_BASE_URL: string
  API_TIMEOUT: number
  DEBUG: boolean
  VERSION: string
}

type ApiOnly = FilterByPrefix<ApiConfig, 'API'>
// { API_BASE_URL: string; API_TIMEOUT: number }
```

## 四、模板字面量类型（Template Literal Types）

### 4.1 基本语法

模板字面量类型使用反引号 `` ` `` 包裹，语法与 JS 模板字符串一致，但操作的是 **类型**：

```typescript
type World = 'world'
type Greeting = `hello ${World}`  // 'hello world'

// 联合类型会自动展开（笛卡尔积）
type Color = 'red' | 'blue'
type Size = 'small' | 'large'
type Shirt = `${Size}-${Color}`
// 'small-red' | 'small-blue' | 'large-red' | 'large-blue'
```

```
         'small'              'large'
        ┌──────┐            ┌──────┐
    'red'  'blue'       'red'  'blue'
     │       │           │       │
     ▼       ▼           ▼       ▼
 'small-  'small-    'large-  'large-
  red'     blue'      red'     blue'
```

### 4.2 内置的工具字符串类型

TypeScript 提供了 4 个内置的字符串操作类型：

| 工具类型 | 作用 | 示例 |
|---------|------|------|
| `Uppercase<S>` | 全大写 | `Uppercase<'hello'>` → `'HELLO'` |
| `Lowercase<S>` | 全小写 | `Lowercase<'HELLO'>` → `'hello'` |
| `Capitalize<S>` | 首字母大写 | `Capitalize<'hello'>` → `'Hello'` |
| `Uncapitalize<S>` | 首字母小写 | `Uncapitalize<'Hello'>` → `'hello'` |

### 4.3 与映射类型结合

```typescript
// CSS 属性名转换：将 camelCase 转为 kebab-case 的简单版本
type CSSProperty = 'fontSize' | 'backgroundColor' | 'marginBottom'

// 自动生成 CSS-in-JS 风格的类型
type CSSKebab<T extends string> = T extends `${infer First}${infer Rest}`
  ? `${Lowercase<First>}${Rest}`
  : T

// 生成 v-model 相关的事件类型
type ModelEventName<T extends string> = `update:${T}`

type PropName = 'name' | 'age' | 'email'
type ModelEvents = ModelEventName<PropName>
// 'update:name' | 'update:age' | 'update:email'
```

### 4.4 实际应用

**1) Vue 组件 emit 事件类型安全**：

```typescript
// 根据 props 自动生成 v-model 事件类型
type VModelEvents<T extends string> = {
  [K in T as `update:${K}`]: (value: any) => void
}

type Emits = VModelEvents<'name' | 'count'>
// {
//   'update:name': (value: any) => void
//   'update:count': (value: any) => void
// }
```

**2) CSS 类型安全**：

```typescript
type CSSUnit = 'px' | 'rem' | 'em' | 'vh' | 'vw' | '%'
type CSSValue = `${number}${CSSUnit}`

// 合法值
const width: CSSValue = '100px'    // OK
const height: CSSValue = '50vh'    // OK
const margin: CSSValue = '1.5rem'  // OK

// 非法值
// const bad: CSSValue = 'abc'     // Error!
// const bad2: CSSValue = '100'    // Error! 缺少单位
```

**3) 路由路径类型安全**：

```typescript
type Route = '/' | '/user' | '/user/:id' | '/post/:id'

// 提取动态路由参数名
type ExtractParams<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? Param | ExtractParams<Rest>
    : T extends `${string}:${infer Param}`
    ? Param
    : never

type Params = ExtractParams<'/user/:userId/post/:postId'>
// 'userId' | 'postId'
```

## 五、keyof 操作符

### 5.1 基本用法

`keyof` 用于获取一个类型的 **所有属性名的联合类型**：

```typescript
interface User {
  name: string
  age: number
  email: string
}

type UserKeys = keyof User  // 'name' | 'age' | 'email'

// 等价手写
type UserKeysManual = 'name' | 'age' | 'email'
```

```
interface User {          keyof User
  name: string     ──→
  age: number      ──→   'name' | 'age' | 'email'
  email: string    ──→
}
```

### 5.2 与索引签名结合

```typescript
// 数组的 keyof
type ArrayKeys = keyof string[]       // number | 'length' | 'push' | 'pop' | ...
type TupleKeys = keyof [string, number]  // '0' | '1' | 'length' | ...

// Record 的 keyof
type RecordKeys = keyof Record<string, any>  // string
```

### 5.3 与映射类型配合

`keyof` 和映射类型是 **最佳搭档**，几乎总是形影不离：

```typescript
// 1) 遍历接口的所有键，创建新类型
type Optional<T> = {
  [K in keyof T]?: T[K]
}

// 2) 创建类型安全的 getter 函数
type GettersMap<T> = {
  [K in keyof T]: () => T[K]
}

// 3) 创建类型安全的 pick 函数
function pick<T, K extends keyof T>(obj: T, ...keys: K[]): Pick<T, K> {
  const result = {} as Pick<T, K>
  keys.forEach(key => {
    result[key] = obj[key]
  })
  return result
}

const user = { name: '张三', age: 25, email: 'zhang@test.com' }
const picked = pick(user, 'name', 'age')
// { name: '张三'; age: 25 }  ← 类型安全！
// pick(user, 'xxx')          // Error! 'xxx' 不在 keyof 中
```

### 5.4 约束泛型参数

```typescript
// 确保 key 存在于对象类型中
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]
}

const person = { name: '张三', age: 25 }
getProperty(person, 'name')  // OK，返回类型 string
getProperty(person, 'age')   // OK，返回类型 number
// getProperty(person, 'job') // Error! 'job' 不存在于类型中
```

## 六、高级类型体操示例

### 6.1 DeepPartial —— 递归可选

内置的 `Partial` 只处理一层，`DeepPartial` 递归处理所有嵌套：

```typescript
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object
    ? DeepPartial<T[K]>
    : T[K]
}

// 使用示例
interface AppState {
  user: {
    profile: {
      name: string
      avatar: string
    }
    settings: {
      theme: 'light' | 'dark'
      language: string
    }
  }
  notifications: {
    items: Array<{ id: number; read: boolean }>
  }
}

type PartialState = DeepPartial<AppState>
// 所有嵌套层级都是可选的
const state: PartialState = {
  user: {
    profile: {
      // name 和 avatar 都可以不写
    }
  }
}
```

### 6.2 DeepReadonly —— 递归只读

```typescript
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends Function
    ? T[K]                           // 函数类型不需要 readonly
    : T[K] extends object
    ? DeepReadonly<T[K]>
    : T[K]
}

// 使用示例
interface Config {
  database: {
    host: string
    port: number
    credentials: {
      username: string
      password: string
    }
  }
  features: string[]
}

type ReadonlyConfig = DeepReadonly<Config>

const config: ReadonlyConfig = {
  database: {
    host: 'localhost',
    port: 3306,
    credentials: { username: 'root', password: '123456' }
  },
  features: ['auth']
}

// config.database.host = '127.0.0.1'         // Error! readonly
// config.database.credentials.username = 'admin'  // Error! 深层 readonly
// config.features.push('new')                    // Error! readonly
```

### 6.3 Mutable —— 去除 readonly

```typescript
type Mutable<T> = {
  -readonly [K in keyof T]: T[K]
}

// 使用示例
interface FrozenUser {
  readonly id: number
  readonly name: string
  readonly tags: readonly string[]
}

type EditableUser = Mutable<FrozenUser>
// { id: number; name: string; tags: readonly string[] }
// 注意：只去掉了第一层的 readonly，内层 readonly string[] 不受影响

// 深度 Mutable
type DeepMutable<T> = {
  -readonly [K in keyof T]: T[K] extends Function
    ? T[K]
    : T[K] extends object
    ? DeepMutable<T[K]>
    : T[K]
}

type FullyEditable = DeepMutable<FrozenUser>
// { id: number; name: string; tags: string[] }
```

### 6.4 RequiredKeys —— 获取必填键

```typescript
// 获取类型中所有必填属性的键的联合类型
type RequiredKeys<T> = {
  [K in keyof T]-?: {} extends Pick<T, K> ? never : K
}[keyof T]

interface Form {
  name: string         // 必填
  age: number          // 必填
  nickname?: string    // 可选
  avatar?: string      // 可选
}

type Required = RequiredKeys<Form>  // 'name' | 'age'
```

**原理解析**：

```
对于每个属性 K：
  {} extends Pick<Form, K> ?
  意思是：空对象 {} 能满足这个属性的要求吗？

  name: string   → {} extends { name: string }   → false → 'name'   （必填）
  age: number    → {} extends { age: number }    → false → 'age'    （必填）
  nickname?: string → {} extends { nickname?: string } → true → never（可选）
  avatar?: string → {} extends { avatar?: string }   → true → never（可选）

最后 {[keyof T]} 取值：
  { name: 'name'; age: 'age'; nickname: never; avatar: never }['name' | 'age' | 'nickname' | 'avatar']
  = 'name' | 'age'
```

### 6.5 Getters —— 自动生成 getter 类型

```typescript
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K]
}

interface Product {
  id: number
  title: string
  price: number
  inStock: boolean
}

type ProductGetters = Getters<Product>
// {
//   getId: () => number
//   getTitle: () => string
//   getPrice: () => number
//   getInStock: () => boolean
// }

// 运行时实现（Vue composable）
function createGetters<T extends object>(obj: T): Getters<T> {
  return new Proxy({} as Getters<T>, {
    get(_, prop: string) {
      const key = prop.charAt(3).toLowerCase() + prop.slice(4) as keyof T
      return () => obj[key]
    }
  })
}

const product = { id: 1, title: '手机', price: 3999, inStock: true }
const getters = createGetters(product)
getters.getTitle()  // '手机'
getters.getPrice()  // 3999
```

## 七、面试常见问题

### Q1：条件类型什么时候会分布式展开？怎么避免？

**答**：当条件类型左侧是 **裸类型参数**（没有被包裹在其他结构中），且传入的是 **联合类型** 时，会自动分布式展开。

```typescript
// 裸类型参数 T → 会分布式
type D<T> = T extends string ? 'yes' : 'no'
type R1 = D<string | number>  // 'yes' | 'no'

// 包裹在元组 [T] 中 → 不分布式
type ND<T> = [T] extends [string] ? 'yes' : 'no'
type R2 = ND<string | number>  // 'no'
```

### Q2：`infer` 只能在哪里使用？能声明多个吗？

**答**：`infer` 只能在 `extends` 条件类型的 **右侧** 使用，可以同时声明多个 `infer` 变量。但不能在普通类型别名、接口等地方使用。

```typescript
// 正确：在条件类型右侧
type R = T extends (arg: infer A) => infer R ? [A, R] : never

// 错误：不能在条件类型左侧
// type Bad = infer T extends string ? T : never  // Error!
```

### Q3：映射类型的 `as` 子句有什么用？

**答**：TypeScript 4.1 引入的 `as` 子句有两个核心用途：
1. **重命名 key** —— 如把 `name` 变成 `getName`
2. **过滤属性** —— 让 `as` 解析为 `never` 即可移除该属性

### Q4：`keyof any` 是什么类型？

**答**：`keyof any` 等价于 `string | number | symbol`，这是 JavaScript 对象键的所有可能类型。在映射类型中经常用 `K extends keyof any` 来约束泛型。

```typescript
type Keys = keyof any  // string | number | symbol
```

### Q5：内置工具类型 `Partial`、`Required`、`Readonly`、`Pick`、`Omit`、`Record` 的底层实现？

**答**：它们都是基于映射类型实现的：

```typescript
type Partial<T> = { [K in keyof T]?: T[K] }
type Required<T> = { [K in keyof T]-?: T[K] }
type Readonly<T> = { readonly [K in keyof T]: T[K] }
type Pick<T, K extends keyof T> = { [P in K]: T[P] }
type Omit<T, K extends keyof T> = Pick<T, Exclude<keyof T, K>>
type Record<K extends keyof any, T> = { [P in K]: T }
```

### Q6：如何在 Vue 项目中实际应用条件类型和映射类型？

**答**：常见场景包括：

```typescript
// 1) 类型安全的 Props
type PropType<T> = T extends BooleanConstructor
  ? Boolean
  : T extends Function
  ? T
  : T

// 2) 根据 API 响应类型自动生成 loading/mutation 类型
type MutationKey<T extends string> = `use${Capitalize<T>}Mutation`
type QueryKey<T extends string> = `use${Capitalize<T>}Query`

type ApiMethods = 'getUser' | 'updateUser' | 'deleteUser'
type Queries = QueryKey<ApiMethods>
// 'useGetUserQuery' | 'useUpdateUserQuery' | 'useDeleteUserQuery'

// 3) 自动生成表单验证类型
type FormErrors<T> = {
  [K in keyof T]?: string[]
}

interface LoginForm {
  username: string
  password: string
}

type LoginFormErrors = FormErrors<LoginForm>
// { username?: string[]; password?: string[] }
```

### Q7：条件类型和映射类型能嵌套/组合使用吗？

**答**：完全可以，而且实际开发中经常组合使用：

```typescript
// 组合示例：递归地把所有函数属性提取出来
type ExtractMethods<T> = {
  [K in keyof T as T[K] extends Function ? K : never]: T[K]
}

// 组合示例：条件类型 + 映射类型 + 模板字面量
type ActionCreators<T> = {
  [K in keyof T as T[K] extends Function ? `do${Capitalize<string & K>}` : never]: T[K]
}

interface Service {
  fetch: () => Promise<void>
  submit: (data: FormData) => Promise<boolean>
  name: string  // 不是函数，会被过滤
}

type Actions = ActionCreators<Service>
// {
//   doFetch: () => Promise<void>
//   doSubmit: (data: FormData) => Promise<boolean>
// }
```
