## 🔥 高频面试题速查 Top 10

| # | 面试题 | 一句话答案 | 详见 |
|---|--------|-----------|------|
| 1 | `interface` 和 `type` 的区别？ | `interface` 支持声明合并和 extends 继承；`type` 支持联合/交叉/条件类型等更灵活的组合；日常开发推荐 interface 优先 | [接口与类型别名](./接口与类型别名.md) |
| 2 | `any` 和 `unknown` 的区别？ | `any` 放弃所有类型检查，`unknown` 是类型安全的顶级类型，必须先做类型收窄才能使用 | [基础类型](./基础类型.md) |
| 3 | 什么是泛型？ | 泛型是类型的"参数"，让函数/类/接口在定义时不绑定具体类型，使用时再指定，实现类型复用 | [泛型](./泛型.md) |
| 4 | `never` 和 `void` 的区别？ | `void` 表示没有返回值（返回 undefined），`never` 表示永远不会到达终点（抛异常/死循环） | [基础类型](./基础类型.md) |
| 5 | TypeScript 的类型推断是怎样的？ | 在不给定类型注解时，TS 会根据初始值、返回值、上下文自动推断类型；推荐"能推断就不写" | [类型推断](./类型推断.md) |
| 6 | `keyof` 和 `typeof` 的区别？ | `keyof T` 获取类型 T 的所有属性名联合类型，`typeof obj` 获取变量的类型，常搭配使用 | [条件类型与映射类型](./条件类型与映射类型.md) |
| 7 | `Partial<T>` 和 `Required<T>` 做了什么？ | `Partial` 把 T 所有属性变为可选（`?`），`Required` 把所有属性变为必填，底层通过映射类型实现 | [内置工具类型](./内置工具类型.md) |
| 8 | 枚举的底层原理？ | 数字枚举编译成双向映射对象（键→值 + 值→键），字符串枚举编译成单向映射，`const enum` 内联编译 | [枚举](./枚举.md) |
| 9 | 如何为第三方库添加类型？ | 安装 `@types/xxx`（DefinitelyTyped）、编写本地 `.d.ts` 声明文件、或使用 `declare module` 扩展 | [声明文件](./声明文件.md) |
| 10 | `strict` 模式开启了哪些检查？ | 包含 strictNullChecks、strictFunctionTypes、noImplicitAny、strictBindCallApply 等 8 项严格检查 | [tsconfig配置](./tsconfig配置.md) |
