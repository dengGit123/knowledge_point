# TypeScript 内置工具类型（Utility Types）

> 官方文档：[TypeScript Utility Types](https://www.typescriptlang.org/docs/handbook/utility-types.html)

---

## 一、什么是工具类型

### 通俗解释

**工具类型（Utility Types）** 就是 TypeScript 自带的一套"类型转换函数"。你可以把它想象成 **类型界的瑞士军刀** —— 它不处理具体的值，而是处理"类型本身"。

```
类比：
┌─────────────────────────────────────────────────┐
│  JavaScript 世界                                │
│  ┌──────────────┐    ┌──────────────┐           │
│  │  值 a = 1    │──▶ │ 函数 f(a)    │──▶ 新值   │
│  └──────────────┘    └──────────────┘           │
│                                                   │
│  TypeScript 世界（类型层面）                      │
│  ┌──────────────┐    ┌──────────────┐           │
│  │ 类型 T       │──▶ │ 工具类型 F<T>│──▶ 新类型  │
│  └──────────────┘    └──────────────┘           │
└─────────────────────────────────────────────────┘
```

简单说：**你给它一个类型，它返回一个变换后的新类型**。

### 底层实现原理

工具类型的底层依赖两大核心机制：

```
┌──────────────────────────────────────────────────────────┐
│                   工具类型的基石                          │
│                                                          │
│   1. 映射类型（Mapped Types）                             │
│      ── 遍历对象的每个属性，逐一变换                       │
│      type NewType = { [K in keyof T]: ... }              │
│                                                          │
│   2. 条件类型（Conditional Types）                        │
│      ── 根据类型关系做分支判断                            │
│      type Result = T extends U ? X : Y                   │
│                                                          │
│   3. keyof / in / extends 等关键字配合                    │
│      ── 实现类型的"编程"能力                              │
└──────────────────────────────────────────────────────────┘
```

一个最简单的映射类型示例：

```typescript
// 把 T 的每个属性都变成 string
type Stringify<T> = {
  [K in keyof T]: string
}

interface User {
  name: string
  age: number
}

type StringUser = Stringify<User>
// 等价于 { name: string; age: string }
```

## 二、属性修饰类

这类工具类型的作用是 **改变对象属性的修饰状态**（可选、必填、只读）。

```
┌─────────────────────────────────────────────────┐
│              属性修饰类 · 全家福                  │
│                                                   │
│   Partial<T>    所有属性 → 可选（加 ?）           │
│   Required<T>   所有属性 → 必填（去 ?）           │
│   Readonly<T>   所有属性 → 只读（加 readonly）    │
│   Record<K,V>   全新的键值对类型                  │
│                                                   │
│   记忆口诀：                                      │
│   "Part 可选 Req 必填，Readonly 加锁 Record 填"  │
└─────────────────────────────────────────────────┘
```

---

### Partial\<T> — 全部变可选

**作用**：将类型 `T` 的所有属性变为可选（加 `?`）。

**使用场景**：更新对象时，只传需要修改的字段。

#### 原理实现

```typescript
type Partial<T> = {
  [P in keyof T]?: T[P]
}
```

拆解：

```
[ P  in  keyof T ]  ? :  T[P]
  │   │    │       │      │
  │   │    │       │      └── 保持原有类型不变
  │   │    │       └── 加上 ? 修饰符，变成可选
  │   │    └── 遍历 T 的所有属性名
  │   └── 对每个属性进行映射
  └── 属性名变量
```

#### 代码示例

```typescript
interface UserInfo {
  name: string
  age: number
  email: string
}

// 全部变成可选
type PartialUser = Partial<UserInfo>
// 等价于：
// {
//   name?: string
//   age?: number
//   email?: string
// }

// 实际使用：更新用户信息，只传需要改的字段
function updateUser(id: number, info: Partial<UserInfo>) {
  // ...
}

// 可以只传一个字段
updateUser(1, { name: '张三' })
// 也可以传多个
updateUser(2, { name: '李四', age: 28 })
```

**Vue 中的典型场景**：

```typescript
// Vue 3 props 定义
interface Props {
  title: string
  count: number
  disabled: boolean
}

// 只想覆盖部分 props 的默认值
const defaults: Partial<Props> = {
  disabled: false,
}

// 组件中使用
import { defineComponent } from 'vue'

export default defineComponent({
  props: {
    title: { type: String, required: true },
    count: { type: Number, default: 0 },
    disabled: { type: Boolean, default: false },
  },
  setup(props: Props) {
    // props.title 一定是 string
  },
})
```

### Required\<T> — 全部变必填

**作用**：将类型 `T` 的所有属性变为必填（去掉 `?`）。

**使用场景**：一个类型部分属性可选，但某个函数需要全部字段都存在时。

#### 原理实现

```typescript
type Required<T> = {
  [P in keyof T]-?: T[P]  // -? 表示移除可选修饰符
}
```

注意 `-?` 这个语法，减号代表 **移除** `?` 修饰符。

#### 代码示例

```typescript
interface Config {
  host?: string
  port?: number
  debug?: boolean
}

// 全部变成必填
type RequiredConfig = Required<Config>
// 等价于：
// {
//   host: string
//   port: number
//   debug: boolean
// }

function createServer(config: RequiredConfig) {
  console.log(`Starting ${config.host}:${config.port}`)
}

// 报错！缺少 port 和 debug
// createServer({ host: 'localhost' })

// 必须全部传入
createServer({ host: 'localhost', port: 3000, debug: false })
```

#### Partial 与 Required 的对比

| 特性 | `Partial<T>` | `Required<T>` |
|------|-------------|--------------|
| 功能 | 所有属性变可选 | 所有属性变必填 |
| 修饰符 | 加 `?` | 去掉 `?`（`-?`） |
| 互补关系 | 互为逆操作 | 互为逆操作 |
| 典型场景 | 更新数据（局部修改） | 校验数据（完整性检查） |

```typescript
interface Todo {
  title: string
  description?: string
}

type A = Partial<Todo>   // { title?: string; description?: string }
type B = Required<Todo>  // { title: string; description: string }

// 注意：Required 对已经必填的属性没有副作用
//       Partial 对已经可选的属性也没有副作用
```

### Readonly\<T> — 全部变只读

**作用**：将类型 `T` 的所有属性变为只读（加 `readonly`）。

**使用场景**：防止对象被意外修改。

#### 原理实现

```typescript
type Readonly<T> = {
  readonly [P in keyof T]: T[P]
}
```

#### 代码示例

```typescript
interface Point {
  x: number
  y: number
}

type ReadonlyPoint = Readonly<Point>
// 等价于：
// {
//   readonly x: number
//   readonly y: number
// }

const point: ReadonlyPoint = { x: 1, y: 2 }
// point.x = 3  // 报错！Cannot assign to 'x' because it is a read-only property
```

**Vue/Vite 中的实际场景**：

```typescript
// Vite 配置中，某些选项应该是不可变的
import type { Readonly } from 'typescript'

interface ViteConfig {
  root: string
  mode: string
  plugins: any[]
}

function freezeConfig(config: Readonly<ViteConfig>) {
  // config.root = '/other'  // 报错！不允许修改
  return Object.freeze(config)
}

// Vue 3 provide/inject 场景
import { inject, provide, type InjectionKey, type Readonly } from 'vue'

// 注入一个只读的状态，子组件不能修改
const stateKey: InjectionKey<Readonly<Ref<number>>> = Symbol()

// 父组件
const count = ref(0)
provide(stateKey, readonly(count))  // vue 的 readonly

// 子组件拿到的就是只读的
const readOnlyCount = inject(stateKey)!
// readOnlyCount.value++  // 报错！
```

### Record\<K, V> — 构造键值对类型

**作用**：快速构造一个"键为 K 类型、值为 V 类型"的对象类型。

**使用场景**：字典、映射表、枚举对象等。

#### 原理实现

```typescript
type Record<K extends keyof any, V> = {
  [P in K]: V
}
```

`K extends keyof any` 意味着 K 只能是 `string | number | symbol` 的子类型。

#### 代码示例

```typescript
// 基本用法：定义一个角色权限映射
type Role = 'admin' | 'editor' | 'viewer'
type Permission = string[]

const rolePermissions: Record<Role, Permission> = {
  admin: ['read', 'write', 'delete', 'manage'],
  editor: ['read', 'write'],
  viewer: ['read'],
}
// 如果漏写一个角色，TypeScript 会报错！
```

**更实用的场景**：

```typescript
// Vue Router 路由配置
type PageName = 'home' | 'about' | 'contact'

interface RouteMeta {
  title: string
  requiresAuth: boolean
}

const routeMeta: Record<PageName, RouteMeta> = {
  home: { title: '首页', requiresAuth: false },
  about: { title: '关于我们', requiresAuth: false },
  contact: { title: '联系我们', requiresAuth: true },
}

// API 响应状态码映射
type StatusCode = 200 | 400 | 401 | 403 | 404 | 500

const statusMessages: Record<StatusCode, string> = {
  200: '请求成功',
  400: '请求参数错误',
  401: '未授权',
  403: '禁止访问',
  404: '资源不存在',
  500: '服务器内部错误',
}
```

#### Record 与 Index Signature 的对比

| 特性 | `Record<K, V>` | `{ [key: string]: V }` |
|------|---------------|----------------------|
| 键的类型约束 | 精确到指定的联合类型 | 宽泛的 string |
| 类型安全 | 高（少写一个键会报错） | 低（任意 string 键都行） |
| 适用场景 | 键已知的字典 | 键未知的动态对象 |
| 可读性 | 语义更明确 | 更通用但更宽松 |

## 三、类型筛选类

这类工具类型用于 **从一个类型中提取或排除部分内容**。

```
┌──────────────────────────────────────────────────────────┐
│                  类型筛选类 · 全家福                      │
│                                                          │
│  对象属性级别：                                           │
│    Pick<T, K>     从 T 中"挑选"部分属性                   │
│    Omit<T, K>     从 T 中"排除"部分属性                   │
│                                                          │
│  联合类型级别：                                           │
│    Extract<T, U>  从联合类型 T 中"提取"匹配 U 的成员      │
│    Exclude<T, U>  从联合类型 T 中"排除"匹配 U 的成员      │
│    NonNullable<T> 从 T 中排除 null 和 undefined           │
│                                                          │
│  记忆：Pick/Extract 是"留"，Omit/Exclude 是"扔"          │
└──────────────────────────────────────────────────────────┘
```

---

### Pick\<T, K> — 选取部分属性

**作用**：从类型 `T` 中选取一组属性 `K`，组成新类型。

#### 原理实现

```typescript
type Pick<T, K extends keyof T> = {
  [P in K]: T[P]
}
```

#### 代码示例

```typescript
interface User {
  id: number
  name: string
  email: string
  avatar: string
  role: string
  createdAt: Date
}

// 只需要展示卡片信息
type UserCard = Pick<User, 'id' | 'name' | 'avatar'>
// 等价于：{ id: number; name: string; avatar: string }

// Vue 组件 props
import { defineComponent, type PropType } from 'vue'

export default defineComponent({
  props: {
    user: {
      type: Object as PropType<Pick<User, 'id' | 'name' | 'avatar'>>,
      required: true,
    },
  },
  template: `
    <div class="user-card">
      <img :src="user.avatar" />
      <span>{{ user.name }}</span>
    </div>
  `,
})
```

---

### Omit\<T, K> — 排除部分属性

**作用**：从类型 `T` 中排除一组属性 `K`，剩余的组成新类型。

#### 原理实现

```typescript
type Omit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>
```

拆解：

```
Omit<User, 'email'>
    │
    ▼
Pick<User, Exclude<'id'|'name'|'email'|'avatar', 'email'>>
    │
    ▼
Pick<User, 'id'|'name'|'avatar'>
    │
    ▼
{ id: number; name: string; avatar: string }
```

#### 代码示例

```typescript
interface Article {
  id: number
  title: string
  content: string
  author: string
  createdAt: Date
  updatedAt: Date
}

// 创建文章时不需要 id 和时间戳（由后端生成）
type CreateArticleDTO = Omit<Article, 'id' | 'createdAt' | 'updatedAt'>
// { title: string; content: string; author: string }

function createArticle(data: CreateArticleDTO) {
  return fetch('/api/articles', {
    method: 'POST',
    body: JSON.stringify(data),
  })
}

createArticle({
  title: 'TypeScript 工具类型',
  content: '...',
  author: '张三',
})
```

#### Pick 与 Omit 的对比

| 特性 | `Pick<T, K>` | `Omit<T, K>` |
|------|-------------|--------------|
| 思路 | 白名单（留什么） | 黑名单（排除什么） |
| K 的含义 | 要 **保留** 的属性名 | 要 **排除** 的属性名 |
| 属性多时 | 繁琐（要列很多） | 方便（只写排除的） |
| 属性少时 | 方便（只写保留的） | 繁琐（要排除很多） |
| 底层关系 | 基础操作 | `Pick` + `Exclude` 组合 |

```typescript
// 两者可以互相替代
interface Foo { a: string; b: number; c: boolean }

type KeepAB1 = Pick<Foo, 'a' | 'b'>     // { a: string; b: number }
type KeepAB2 = Omit<Foo, 'c'>           // { a: string; b: number }
// 结果一样！
```

---

### Exclude\<T, U> — 从联合类型中排除

**作用**：从联合类型 `T` 中排除那些可以赋值给 `U` 的成员。

#### 原理实现

```typescript
type Exclude<T, U> = T extends U ? never : T
```

原理图：

```
T = 'a' | 'b' | 'c'
U = 'a'

         'a' extends 'a' ? never : 'a'  → never
         'b' extends 'a' ? never : 'b'  → 'b'
         'c' extends 'a' ? never : 'c'  → 'c'

结果：'b' | 'c'   （never 会被自动忽略）
```

#### 代码示例

```typescript
type AllTypes = 'click' | 'hover' | 'focus' | 'blur'
type MouseEvents = Exclude<AllTypes, 'focus' | 'blur'>
// 'click' | 'hover'

// 实际场景：从权限列表中排除某些
type Permission = 'read' | 'write' | 'delete' | 'admin'
type UserPermission = Exclude<Permission, 'admin'>
// 'read' | 'write' | 'delete'

// 在 Vue 组件中排除不需要的事件
type AllEmits = 'change' | 'input' | 'blur' | 'focus'
type ValidEmits = Exclude<AllEmits, 'blur'>
```

---

### Extract\<T, U> — 从联合类型中提取

**作用**：从联合类型 `T` 中提取那些可以赋值给 `U` 的成员。

#### 原理实现

```typescript
type Extract<T, U> = T extends U ? T : never
```

与 Exclude 恰好相反：

```
Exclude: T extends U ? never : T    不匹配的留下
Extract: T extends U ? T   : never  匹配的留下
```

#### 代码示例

```typescript
type AllTypes = string | number | boolean | null | undefined
type StringOrNumber = Extract<AllTypes, string | number>
// string | number

// 提取函数类型
type Mixed = string | (() => void) | number | (() => string)
type Functions = Extract<Mixed, Function>
// (() => void) | (() => string)

// 实际场景：从 CSS 属性中提取长度相关
type CSSValue = 'auto' | '100px' | '200px' | 'inherit' | '50%'
type PixelValues = Extract<CSSValue, `${number}px`>
// '100px' | '200px'
```

---

### NonNullable\<T> — 排除 null 和 undefined

**作用**：从类型 `T` 中排除 `null` 和 `undefined`。

#### 原理实现

```typescript
type NonNullable<T> = T extends null | undefined ? never : T
```

#### 代码示例

```typescript
type MaybeString = string | null | undefined
type SureString = NonNullable<MaybeString>
// string

// 实际场景：API 返回值可能为空，但使用时确定不为空
interface ApiResponse {
  data: User | null
  error: string | null
}

function handleResponse(response: ApiResponse) {
  // response.data 可能是 null
  const user: NonNullable<typeof response.data> = response.data!
  //                                        注意这里的非空断言 ↑

  // 更好的做法：结合类型守卫
  if (response.data !== null) {
    // 这里 TypeScript 自动推断为 User 类型
    console.log(response.data.name)
  }
}
```

#### 三者的关系对比

```
原始联合类型：  'a' | 'b' | 'c'
                    │
        ┌───────────┼───────────┐
        ▼           ▼           ▼
   Extract      Exclude    NonNullable
   (提取匹配)   (排除匹配)  (排除空值)

   对字符串联合：  对字符串联合：   对含 null 的类型：
   Extract<      Exclude<        NonNullable<
     T, 'a'        T, 'a'          string|null
   >             >               >
   = 'a'         = 'b' | 'c'     = string
```

## 四、函数相关类

这类工具类型用于 **从函数类型中提取参数或返回值的类型信息**。

```
┌──────────────────────────────────────────────────────────┐
│                函数相关类 · 全家福                        │
│                                                          │
│   Parameters<T>              获取函数参数 → 元组类型       │
│   ReturnType<T>              获取函数返回值类型            │
│   ConstructorParameters<T>   获取构造函数参数 → 元组类型   │
│   InstanceType<T>            获取构造函数实例类型          │
│                                                          │
│   类比：                                                   │
│   函数 ──→ Parameters 拿"入参"                            │
│         ──→ ReturnType 拿"出参"                           │
│   类   ──→ ConstructorParameters 拿"构造参数"             │
│         ──→ InstanceType 拿"实例"                         │
└──────────────────────────────────────────────────────────┘
```

---

### Parameters\<T> — 获取函数参数类型

**作用**：获取函数类型 `T` 的参数列表，返回一个元组类型。

#### 原理实现

```typescript
type Parameters<T extends (...args: any) => any> =
  T extends (...args: infer P) => any
    ? P
    : never
```

关键：`infer P` 的作用是 **推断** 出参数的类型并绑定到 `P`。

```
Parameters<(a: string, b: number) => void>

过程：
  T = (a: string, b: number) => void
  匹配 (...args: infer P) => any
  推断 P = [a: string, b: number]

结果：[string, number]
```

#### 代码示例

```typescript
function createUser(name: string, age: number, role: string) {
  return { name, age, role }
}

type CreateUserParams = Parameters<typeof createUser>
// [string, number, string]

// 实际使用：包装一个函数，保持参数类型
function wrapper(fn: typeof createUser, ...args: Parameters<typeof createUser>) {
  console.log('调用前')
  const result = fn(...args)
  console.log('调用后')
  return result
}

// Vue 3 场景：对 emit 事件的参数做类型约束
import { defineComponent } from 'vue'

export default defineComponent({
  emits: {
    change: (value: string, index: number) => true,
    delete: (id: number) => true,
  },
  setup(props, { emit }) {
    // 获取 'change' 事件的参数类型
    type ChangeArgs = Parameters<{
      (value: string, index: number): any
    }>
    // [string, number]
  },
})
```

### ReturnType\<T> — 获取函数返回类型

**作用**：获取函数类型 `T` 的返回值类型。

#### 原理实现

```typescript
type ReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R
    ? R
    : never
```

#### 代码示例

```typescript
function fetchUser() {
  return { id: 1, name: '张三', email: 'zhangsan@example.com' }
}

type UserResponse = ReturnType<typeof fetchUser>
// { id: number; name: string; email: string }

// 实际场景：API 请求函数的返回值类型自动推导
async function getArticleList() {
  const res = await fetch('/api/articles')
  const data: { id: number; title: string; content: string }[] = await res.json()
  return data
}

type ArticleList = ReturnType<typeof getArticleList>
// Promise<{ id: number; title: string; content: string }[]>
// 注意：async 函数返回的是 Promise 包裹的类型

// 如果想拿到解包后的类型，配合 Awaited
type ArticleListUnwrapped = Awaited<ReturnType<typeof getArticleList>>
// { id: number; title: string; content: string }[]
```

**Vite 配置中的场景**：

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

// defineConfig 的返回值类型
type ViteConfig = ReturnType<typeof defineConfig>

// 自定义配置合并
function createConfig(overrides: Partial<ViteConfig>) {
  return defineConfig({
    plugins: [vue()],
    ...overrides,
  })
}
```

### ConstructorParameters\<T> — 获取构造函数参数类型

**作用**：获取构造函数类型 `T` 的参数列表，返回一个元组类型。

#### 原理实现

```typescript
type ConstructorParameters<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: infer P) => any
    ? P
    : never
```

#### 代码示例

```typescript
class Person {
  constructor(public name: string, public age: number) {}
}

type PersonParams = ConstructorParameters<typeof Person>
// [string, number]

// 使用场景：动态创建实例
function createInstance<T extends new (...args: any) => any>(
  Cls: T,
  ...args: ConstructorParameters<T>
): InstanceType<T> {
  return new Cls(...args)
}

const person = createInstance(Person, '张三', 25)
// person 的类型自动推导为 Person
```

### InstanceType\<T> — 获取构造函数实例类型

**作用**：获取构造函数类型 `T` 的实例类型。

#### 原理实现

```typescript
type InstanceType<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: any) => infer R
    ? R
    : never
```

#### 代码示例

```typescript
class DatePicker {
  value: Date = new Date()
  open() {}
  close() {}
}

type DatePickerInstance = InstanceType<typeof DatePicker>
// DatePicker

// 实际场景：Vue 3 中 ref 引用组件实例
import { ref } from 'vue'

// 不需要导入组件类型，用 InstanceType 也能推导
const datePickerRef = ref<InstanceType<typeof DatePicker> | null>(null)

function openPicker() {
  datePickerRef.value?.open()
}
```

## 五、字符串操作类

这类工具类型用于 **对字符串字面量类型进行大小写转换**。

```
┌──────────────────────────────────────────────────────────┐
│                字符串操作类 · 全家福                      │
│                                                          │
│  Uppercase<S>     全部大写   'hello' → 'HELLO'           │
│  Lowercase<S>     全部小写   'HELLO' → 'hello'           │
│  Capitalize<S>    首字母大写 'hello' → 'Hello'           │
│  Uncapitalize<S>  首字母小写 'Hello' → 'hello'           │
│                                                          │
│  记忆：                                                   │
│  Upper 全大写 / Lower 全小写                              │
│  Capital 首字母 ↑ / UnCapital 首字母 ↓                   │
└──────────────────────────────────────────────────────────┘
```

> 这四个类型的底层实现是 TypeScript 编译器内置的，不是纯类型层面能表达的。

### Uppercase\<S> — 全部大写

```typescript
type Upper = Uppercase<'hello world'>
// 'HELLO WORLD'

// 实际场景：常量命名
type EventName = 'click' | 'hover' | 'focus'
type ConstantName = Uppercase<EventName>
// 'CLICK' | 'HOVER' | 'FOCUS'
```

### Lowercase\<S> — 全部小写

```typescript
type Lower = Lowercase<'HELLO WORLD'>
// 'hello world'

// 实际场景：CSS 属性名规范化
type CSSProp = 'Font-Size' | 'Background-Color'
type NormalizedCSS = Lowercase<CSSProp>
// 'font-size' | 'background-color'
```

### Capitalize\<S> — 首字母大写

```typescript
type Cap = Capitalize<'hello'>
// 'Hello'

// 实际场景：组件名 PascalCase 化
type ComponentName = Capitalize<'button'> | Capitalize<'input'>
// 'Button' | 'Input'

// 实际场景：事件处理函数名
type EventHandler<T extends string> = `on${Capitalize<T>}`
type ClickHandler = EventHandler<'click'>   // 'onClick'
type FocusHandler = EventHandler<'focus'>   // 'onFocus'
```

### Uncapitalize\<S> — 首字母小写

```typescript
type Uncap = Uncapitalize<'HelloWorld'>
// 'helloWorld'

// 实际场景：PascalCase → camelCase
type PascalCase = 'UserService' | 'OrderManager' | 'PaymentHandler'
type CamelCase = Uncapitalize<PascalCase>
// 'userService' | 'orderManager' | 'paymentHandler'
```

### 字符串操作类组合使用

```typescript
// 实现 camelCase → snake_case 的转换（简化版）
type Kebbabize<S extends string> =
  S extends `${infer First}${infer Rest}`
    ? First extends Uppercase<First>
      ? `-${Lowercase<First>}${Kebbabize<Rest>}`
      : `${First}${Kebbabize<Rest>}`
    : S

// Vue 组件 props 的命名转换
// template 中使用 kebab-case，JS 中使用 camelCase
type PropName = 'userName' | 'userAge' | 'isActive'
// 想要：'user-name' | 'user-age' | 'is-active'
```

## 六、工具类型组合技巧

工具类型真正的威力在于 **组合使用**，就像乐高积木一样。

### Partial + Pick 组合 — 部分属性可选

```typescript
interface FormState {
  username: string
  password: string
  rememberMe: boolean
  captcha: string
}

// 只有 captcha 是可选的，其余必填
type LoginForm = Omit<FormState, 'captcha'> & Partial<Pick<FormState, 'captcha'>>
// 等价于：
// {
//   username: string
//   password: string
//   rememberMe: boolean
//   captcha?: string       ← 只有这个可选
// }

function submitForm(data: LoginForm) {
  // data.username 一定是 string
  // data.captcha 可能是 undefined
}

submitForm({
  username: 'admin',
  password: '123456',
  rememberMe: false,
  // captcha 可省略
})
```

### Omit + Extract 组合 — 精确排除

```typescript
interface APIResponse {
  code: number
  message: string
  data: unknown
  timestamp: number
  requestId: string
}

// 只排除元数据字段，保留业务字段
type MetadataKeys = 'code' | 'message' | 'timestamp' | 'requestId'
type BusinessData = Omit<APIResponse, MetadataKeys>
// { data: unknown }

// 等价于
type BusinessData2 = Pick<APIResponse, 'data'>
```

### 深层 Partial — 递归实现

标准的 `Partial` 只处理第一层属性。如果需要所有嵌套属性都变成可选，需要递归实现：

```typescript
// 标准的 Partial 只做浅层
interface NestedConfig {
  server: {
    host: string
    port: number
  }
  database: {
    url: string
    name: string
  }
}

type ShallowPartial = Partial<NestedConfig>
// {
//   server?: { host: string; port: number }   ← 内层仍然是必填！
//   database?: { url: string; name: string }
// }

// 递归的 DeepPartial
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object
    ? T[P] extends Function
      ? T[P]              // 函数不做递归
      : DeepPartial<T[P]> // 对象继续递归
    : T[P]                // 基本类型直接可选
}

type DeepPartialConfig = DeepPartial<NestedConfig>
// {
//   server?: { host?: string; port?: number }   ← 内层也变可选了！
//   database?: { url?: string; name?: string }
// }

// 使用
const config: DeepPartialConfig = {
  server: {
    // port 可以不传
    host: 'localhost',
  },
  // database 整个可以不传
}
```

### 深层 Readonly

```typescript
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object
    ? T[P] extends Function
      ? T[P]
      : DeepReadonly<T[P]>
    : T[P]
}

interface AppState {
  user: {
    profile: {
      name: string
      avatar: string
    }
    settings: {
      theme: 'light' | 'dark'
      language: string
    }
  }
}

type FrozenState = DeepReadonly<AppState>
const state: FrozenState = {
  user: {
    profile: { name: '张三', avatar: 'a.png' },
    settings: { theme: 'dark', language: 'zh-CN' },
  },
}

// state.user.profile.name = '李四'  // 报错！深层只读
```

## 七、面试常见问题

### 面试题 1：Partial 的底层实现原理是什么？

**答**：

`Partial<T>` 使用了 **映射类型（Mapped Types）**，通过 `keyof T` 获取 `T` 的所有属性名，然后用 `[P in keyof T]` 遍历每个属性，加上 `?` 修饰符使其变为可选：

```typescript
type Partial<T> = {
  [P in keyof T]?: T[P]
}
```

关键是 `?` 修饰符。TypeScript 还支持 `-?` 移除可选（即 `Required` 的实现）、`+readonly` 添加只读（即 `Readonly` 的实现）、`-readonly` 移除只读。

---

### 面试题 2：Pick 和 Omit 有什么区别？它们之间有什么关系？

**答**：

| 方面 | Pick\<T, K> | Omit\<T, K> |
|------|------------|------------|
| 策略 | 白名单 | 黑名单 |
| K 的含义 | 要保留的属性 | 要排除的属性 |
| 底层实现 | 基础操作 | `Pick` + `Exclude` 组合 |

`Omit` 的本质就是 `Pick<T, Exclude<keyof T, K>>`。在属性很多、只排除少数属性时用 `Omit` 更方便；在属性很多、只保留少数属性时用 `Pick` 更方便。两者可以互相替代。

---

### 面试题 3：如何实现一个 DeepPartial，让嵌套对象的所有属性都变为可选？

**答**：

```typescript
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object
    ? T[P] extends Function
      ? T[P]
      : DeepPartial<T[P]>
    : T[P]
}
```

要点：
1. 使用 `?:` 让当前层级属性可选
2. 用 `T[P] extends object` 判断是否为对象类型
3. 排除 `Function` 类型，避免函数被错误递归
4. 对对象类型递归调用 `DeepPartial`
5. 基本类型直接用 `T[P]`

---

### 面试题 4：Exclude 和 Omit 的区别是什么？

**答**：

它们操作的对象层级不同：

```
Exclude：操作 联合类型 的成员
Omit：  操作 对象类型 的属性

// Exclude — 从联合类型中排除
type T1 = Exclude<'a' | 'b' | 'c', 'a'>  // 'b' | 'c'

// Omit — 从对象类型中排除属性
type T2 = Omit<{ a: 1; b: 2; c: 3 }, 'a'>  // { b: 2; c: 3 }

// Omit 的底层就是 Pick + Exclude
type Omit<T, K> = Pick<T, Exclude<keyof T, K>>
//                        ^^^^^^ 这里用的是 Exclude！
```

简单记忆：**Exclude 处理联合类型，Omit 处理对象类型**。`Omit` 内部用 `Exclude` 来过滤属性名。

---

### 面试题 5：infer 关键字在工具类型中起到什么作用？举几个例子。

**答**：

`infer` 用于在 **条件类型中推断（推导）出一个类型变量**。它只能在 `extends` 条件分支的右侧使用。

```typescript
// Parameters 中推断参数类型
type Parameters<T> = T extends (...args: infer P) => any ? P : never
//                                                    ^^^^^ 推断参数类型

// ReturnType 中推断返回值类型
type ReturnType<T> = T extends (...args: any) => infer R ? R : never
//                                                     ^^^^^ 推断返回类型

// InstanceType 中推断实例类型
type InstanceType<T> = T extends new (...args: any) => infer I ? I : never
//                                                        ^^^^^ 推断实例类型
```

类比理解：`infer` 就像是一个 **占位符**，TypeScript 在做类型匹配时，会自动"填入"匹配到的类型。

```
函数签名：  (a: string, b: number) => boolean
模式：      (...args: infer P) => infer R

匹配过程：
  ...args 对应 (a: string, b: number) → P 被推断为 [string, number]
  返回值  对应 boolean               → R 被推断为 boolean
```
