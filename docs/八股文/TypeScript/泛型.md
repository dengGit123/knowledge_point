> [TypeScript 官方文档 - Generics](https://www.typescriptlang.org/docs/handbook/2/generics.html)

## 一、什么是泛型

### 通俗解释

你可以把泛型理解为 **"类型的参数"**。

- 普通函数的参数接收的是 **值**（value）
- 泛型的参数接收的是 **类型**（type）

类比：

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   普通函数：                                                      │
│   function add(a: number, b: number)  ← 参数是「值」               │
│                                                                  │
│   泛型函数：                                                      │
│   function identity<T>(value: T): T    ← T 是「类型的参数」         │
│                                                                  │
│   ┌─────────────────┐        ┌─────────────────────────┐         │
│   │  函数参数 = 传值  │   vs   │  泛型参数 = 传类型       │         │
│   │  fn(42)          │        │  fn<number>(42)          │         │
│   └─────────────────┘        └─────────────────────────┘         │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

简单来说：**泛型让你写一份代码，适配多种类型，同时保留类型安全。**

### 为什么需要泛型

来看一个真实的痛点场景 —— 封装一个通用的 `localStorage` 读取函数：

```typescript
// ❌ 没有泛型：要么丢失类型信息，要么要写大量重复代码

// 方案 1：用 any —— 完全丧失类型检查
function getStorage(key: string): any {
  const value = localStorage.getItem(key)
  return value ? JSON.parse(value) : null
}

const user = getStorage('user') // 类型是 any，调用任何属性都不报错
user.name       // 不报错，但运行时可能是 undefined
user.whatever   // 不报错，但运行时一定是 undefined


// 方案 2：为每种类型写一个函数 —— 大量重复
function getUser(key: string): User | null { /* ... */ }
function getToken(key: string): string | null { /* ... */ }
function getConfig(key: string): Config | null { /* ... */ }
```

```typescript
// ✅ 使用泛型：一份代码，多种类型，类型安全

function getStorage<T>(key: string): T | null {
  const value = localStorage.getItem(key)
  return value ? JSON.parse(value) as T : null
}

const user = getStorage<User>('user')   // 类型是 User | null
const token = getStorage<string>('token') // 类型是 string | null

user.name    // ✅ 有类型提示，有自动补全
user.age     // ✅ 有类型提示
// user.whatever // ❌ 编译时报错！类型安全
```

### 没有泛型的痛点总结

| 痛点 | 说明 |
|------|------|
| 类型丢失 | 使用 `any` 后，编译器无法帮你检查类型错误 |
| 代码重复 | 为每种类型写一个几乎相同的函数 |
| 不安全 | `any` 允许任意操作，隐患在运行时才暴露 |
| 维护困难 | 同一逻辑散落在多个函数中，改一处忘改另一处 |

## 二、泛型函数

### 基本语法 `<T>`

```typescript
// <T> 声明一个类型参数，可以在参数列表、返回值、函数体中使用
function identity<T>(value: T): T {
  return value
}

// 调用时显式指定类型参数
const result1 = identity<string>('hello')   // 返回类型: string
const result2 = identity<number>(42)        // 返回类型: number

// 调用时省略类型参数 —— TypeScript 自动推断（Type Inference）
const result3 = identity('hello')  // TypeScript 推断 T = string
const result4 = identity(42)       // TypeScript 推断 T = number
```

**泛型函数的执行流程：**

```
identity<number>(42)

  │
  ▼
┌─────────────────────────────────────────────┐
│  1. 传入类型参数 T = number                   │
│  2. 函数签名变为 (value: number): number      │
│  3. 执行函数体，返回 number 类型的值           │
│  4. 返回值自动获得 number 类型                │
└─────────────────────────────────────────────┘
```

### 多个类型参数

一个函数可以有多个泛型参数，用不同的字母命名：

```typescript
// 常见命名惯例：
// T — Type（类型）
// K — Key（键）
// V — Value（值）
// E — Element（元素）
// R — Result（返回值）

function merge<T, U>(obj1: T, obj2: U): T & U {
  return { ...obj1, ...obj2 }
}

const merged = merge(
  { name: 'Alice', age: 25 },
  { email: 'alice@example.com' }
)
// merged 的类型: { name: string; age: number } & { email: string }
// 即: { name: string; age: number; email: string }
```

```typescript
// 实际场景：Vue 3 Composable 中的多泛型
function useRefState<T, K extends keyof T>(
  obj: T,
  key: K
): Ref<T[K]> {
  return ref(obj[key]) as Ref<T[K]>
}

const person = { name: 'Alice', age: 25 }
const nameRef = useRefState(person, 'name')  // Ref<string>
const ageRef = useRefState(person, 'age')    // Ref<number>
```

### 泛型与普通参数的对比

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   function fn<T>(a: T, b: number): T                            │
│              ─┬─  ──┬──  ────┬──── ─┬─                           │
│               │      │        │      │                           │
│          类型参数  使用T   普通参数  返回T                        │
│          (编译时)  作为   (运行时)  作为                          │
│                   类型           类型                            │
│                                                                  │
│   调用：fn<string>("hi", 42)                                     │
│         ────┬────  ─┬─  ─┬─                                      │
│         传入类型   传入值  传入值                                   │
│         参数       参数    参数                                    │
│                                                                  │
│   ═══ 关键区别 ═══                                                │
│   类型参数 <T>：编译时确定，运行时被擦除（Type Erasure）            │
│   普通参数    ：运行时存在，占据内存                                │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

```typescript
// 编译前（TypeScript）
function first<T>(arr: T[]): T | undefined {
  return arr[0]
}
const n = first([1, 2, 3]) // number

// 编译后（JavaScript）—— 泛型被擦除
function first(arr) {
  return arr[0]
}
const n = first([1, 2, 3]) // 运行时没有任何泛型信息
```

## 四、泛型类

### 类中使用泛型

```typescript
// 通用的缓存类
class Cache<T> {
  private store = new Map<string, { value: T; expireAt: number }>()

  set(key: string, value: T, ttl: number = 60000): void {
    this.store.set(key, {
      value,
      expireAt: Date.now() + ttl,
    })
  }

  get(key: string): T | undefined {
    const entry = this.store.get(key)
    if (!entry) return undefined
    if (Date.now() > entry.expireAt) {
      this.store.delete(key)
      return undefined
    }
    return entry.value
  }

  has(key: string): boolean {
    return this.get(key) !== undefined
  }

  clear(): void {
    this.store.clear()
  }
}

// 使用 —— 每个实例有独立的类型
const userCache = new Cache<User>()       // Cache<User>
const tokenCache = new Cache<string>()    // Cache<string>

userCache.set('current', { id: 1, name: 'Alice', email: 'a@test.com' })
tokenCache.set('auth', 'eyJhbGciOiJIUzI1NiJ9...', 3600000)

const currentUser = userCache.get('current')  // User | undefined
const authToken = tokenCache.get('auth')      // string | undefined
```

```typescript
// Vue/Vite 实战场景：通用的事件总线类
type EventHandler<T = any> = (payload: T) => void

class EventBus<Events extends Record<string, any> = {}> {
  private listeners = new Map<keyof Events, Set<EventHandler<any>>>()

  on<K extends keyof Events>(event: K, handler: EventHandler<Events[K]>): () => void {
    if (!this.listeners.has(event)) {
      this.listeners.set(event, new Set())
    }
    this.listeners.get(event)!.add(handler)

    // 返回取消订阅的函数
    return () => {
      this.listeners.get(event)?.delete(handler)
    }
  }

  emit<K extends keyof Events>(event: K, payload: Events[K]): void {
    this.listeners.get(event)?.forEach(handler => handler(payload))
  }

  off<K extends keyof Events>(event: K, handler: EventHandler<Events[K]>): void {
    this.listeners.get(event)?.delete(handler)
  }
}

// 定义事件类型映射
interface AppEvents {
  'user:login': { userId: number; name: string }
  'user:logout': undefined
  'notification': { title: string; message: string }
  'theme:change': 'light' | 'dark'
}

// 创建类型安全的事件总线
const bus = new EventBus<AppEvents>()

// ✅ 正确使用 —— 类型安全
bus.on('user:login', (payload) => {
  console.log(payload.userId, payload.name) // 自动推断出类型
})

bus.emit('notification', { title: '提示', message: '操作成功' })

// ❌ 错误使用 —— 编译时报错
// bus.emit('user:login', { wrong: true })    // 报错：缺少 userId
// bus.on('notification', (p: string) => {})  // 报错：参数类型不匹配
```

### 静态成员的限制

**重要规则：泛型类的静态成员不能引用类的类型参数。**

```typescript
class Box<T> {
  contents: T

  constructor(contents: T) {
    this.contents = contents
  }

  // ❌ 编译错误！静态成员不能使用类的泛型参数 T
  // static defaultValue: T               // Error!
  // static create(): T { ... }           // Error!

  // ✅ 静态方法可以定义自己的泛型，与类的 T 无关
  static identity<U>(value: U): U {
    return value
  }

  // ✅ 静态方法使用非泛型类型也没问题
  static isBox(obj: unknown): boolean {
    return obj instanceof Box
  }
}
```

原因解释：

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   class Box<T> { ... }                                          │
│                                                                  │
│   const stringBox = new Box<string>("hello")                     │
│   const numberBox = new Box<number>(42)                          │
│                                                                  │
│   问题：Box<T> 是一个类型，但 static 成员只有「一份」               │
│                                                                  │
│   如果允许 static defaultValue: T                                │
│                                                                  │
│   那么 Box<string>.defaultValue 应该是 string                    │
│        Box<number>.defaultValue 应该是 number                    │
│                                                                  │
│   但 static 成员在内存中只有一份！无法同时是 string 和 number      │
│                                                                  │
│   所以 TypeScript 禁止在 static 成员中引用类的类型参数             │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 五、泛型约束（extends）

### 限制泛型参数的范围

默认情况下，泛型参数 `T` 可以是任何类型。但有时我们需要限制它的范围，这就需要 `extends` 关键字。

```typescript
// 需求：获取某个属性的值
// 问题：不是所有类型都有 .length 属性
function getLength<T>(value: T): number {
  return value.length // ❌ 编译错误：T 不保证有 length 属性
}

// 解决：用 extends 约束 T 必须有 length 属性
interface HasLength {
  length: number
}

function getLength<T extends HasLength>(value: T): number {
  return value.length // ✅ T 保证有 length 属性
}

getLength('hello')        // ✅ string 有 length
getLength([1, 2, 3])      // ✅ array 有 length
getLength({ length: 10 }) // ✅ 对象有 length 属性也可以
// getLength(123)          // ❌ number 没有 length，编译错误
// getLength(true)         // ❌ boolean 没有 length，编译错误
```

```typescript
// 实战场景：确保泛型参数是对象类型
function getProperty<T extends object, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]
}

const person = { name: 'Alice', age: 25, email: 'a@test.com' }

getProperty(person, 'name')   // 返回类型: string
getProperty(person, 'age')    // 返回类型: number
// getProperty(person, 'phone') // ❌ 编译错误：'phone' 不是 keyof typeof person
```

### keyof 约束

`keyof` 是 TypeScript 中极其常用的约束手段，它将泛型参数约束为某个类型的 **属性名联合类型**：

```typescript
// 类型安全的对象遍历工具
function pluck<T, K extends keyof T>(array: T[], key: K): T[K][] {
  return array.map(item => item[key])
}

interface Student {
  name: string
  age: number
  score: number
}

const students: Student[] = [
  { name: 'Alice', age: 20, score: 95 },
  { name: 'Bob', age: 22, score: 88 },
  { name: 'Charlie', age: 21, score: 92 },
]

const names = pluck(students, 'name')   // string[]
const ages = pluck(students, 'age')     // number[]
const scores = pluck(students, 'score') // number[]
// pluck(students, 'grade')             // ❌ 编译错误：'grade' 不是合法 key
```

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   K extends keyof T  的含义                                       │
│                                                                  │
│   T = Student                                                    │
│   keyof T = "name" | "age" | "score"                            │
│                                                                  │
│   K extends "name" | "age" | "score"                             │
│                                                                  │
│   所以 K 只能是这三个字符串之一                                     │
│                                                                  │
│   ┌─────────────────────────────────────────┐                    │
│   │  Student                                │                    │
│   │  ├── name  ──→ string    ← K = 'name'   │                    │
│   │  ├── age   ──→ number    ← K = 'age'    │                    │
│   │  └── score ──→ number    ← K = 'score'  │                    │
│   └─────────────────────────────────────────┘                    │
│                                                                  │
│   T[K] 表示：用 K 去索引 T 得到的属性值类型                          │
│   T['name'] = string                                             │
│   T['age']  = number                                             │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 多重约束

使用交叉类型 `&` 实现多重约束：

```typescript
interface HasId {
  id: number
}

interface HasTimestamp {
  createdAt: Date
  updatedAt: Date
}

// T 必须同时满足 HasId 和 HasTimestamp
function updateEntity<T extends HasId & HasTimestamp>(
  entity: T,
  updates: Partial<Omit<T, 'id' | 'createdAt'>>
): T {
  return {
    ...entity,
    ...updates,
    updatedAt: new Date(),
  }
}

interface Article extends HasId, HasTimestamp {
  id: number
  title: string
  content: string
  createdAt: Date
  updatedAt: Date
}

const article: Article = {
  id: 1,
  title: 'TypeScript 泛型',
  content: '泛型是...',
  createdAt: new Date('2024-01-01'),
  updatedAt: new Date('2024-01-01'),
}

const updated = updateEntity(article, { title: '深入 TypeScript 泛型' })
// updated.title = '深入 TypeScript 泛型'
// updated.updatedAt = new Date() （自动更新）
```

```typescript
// 另一种写法：使用嵌套 extends
function process<T extends HasId>(entity: T): T & HasTimestamp
function process<T extends HasId & HasTimestamp>(entity: T): T
function process<T extends HasId>(entity: T): T & HasTimestamp {
  return {
    ...entity,
    createdAt: new Date(),
    updatedAt: new Date(),
  }
}
```

## 六、默认类型参数

### 语法和用法

泛型参数可以设置默认类型，就像函数参数可以有默认值一样：

```typescript
// T 默认为 string
function createArray<T = string>(length: number, fill: T): T[] {
  return Array(length).fill(fill)
}

const strings = createArray(3, 'hi')         // string[] —— T 推断为 string
const numbers = createArray<number>(3, 42)   // number[] —— 显式指定 T = number
const defaults = createArray(3, 'default')   // string[] —— T 推断为 string
```

```typescript
// Vue 3 实战：带默认类型的 API 响应
interface ApiResponse<T = unknown, M = string> {
  code: number
  data: T
  message: M
  timestamp: number
}

// 不指定泛型 —— 使用默认类型
const defaultResponse: ApiResponse = {
  code: 200,
  data: { anything: 'goes' },  // unknown
  message: 'success',           // string
  timestamp: Date.now(),
}

// 指定 data 类型，message 使用默认
const userResponse: ApiResponse<User> = {
  code: 200,
  data: { id: 1, name: 'Alice', email: 'a@test.com' },
  message: 'success',  // string（默认）
  timestamp: Date.now(),
}

// 两个都指定
const customResponse: ApiResponse<User, string | null> = {
  code: 200,
  data: { id: 1, name: 'Alice', email: 'a@test.com' },
  message: null,  // string | null（自定义）
  timestamp: Date.now(),
}
```

### 默认值的规则

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   默认类型参数的规则：                                             │
│                                                                  │
│   1. 有默认值的类型参数可以省略不传                                 │
│   2. 有默认值的参数必须放在没有默认值的参数后面                       │
│   3. 默认类型可以引用前面的类型参数                                  │
│                                                                  │
│   ✅ 正确写法：                                                   │
│   interface Foo<T, U = string> { ... }                           │
│   interface Foo<T, U = string, V = number> { ... }               │
│   interface Foo<T, U = T> { ... }                                │
│                                                                  │
│   ❌ 错误写法：                                                   │
│   interface Foo<T = string, U> { ... }                           │
│   //              有默认值 ↑   ↑ 无默认值，但在后面，不允许          │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

```typescript
// 规则 3：默认类型引用前面的类型参数
interface PaginatedResult<T, Item = T> {
  items: Item[]
  total: number
  page: number
  hasNext: boolean
}

// 等价写法，但上面的方式在某些场景更灵活
interface PaginatedResult2<
  T,
  Meta = { page: number; total: number }
> {
  items: T[]
  meta: Meta
}
```

## 七、常见泛型模式

### 工厂函数

```typescript
// 通用的工厂函数
function createInstance<T>(
  Constructor: new (...args: any[]) => T,
  ...args: any[]
): T {
  return new Constructor(...args)
}

class Animal {
  constructor(public name: string, public sound: string) {}
  speak() { console.log(`${this.name}: ${this.sound}!`) }
}

const cat = createInstance(Animal, 'Kitty', 'Meow')
cat.speak() // "Kitty: Meow!"
// cat 的类型自动推断为 Animal
```

  return res.data
}

### 事件系统

```typescript
// 类型安全的事件发射器
type EventMap = Record<string, any>

class TypedEmitter<Events extends EventMap> {
  private events = new Map<keyof Events, Set<Function>>()

  on<E extends keyof Events>(
    event: E,
    listener: (payload: Events[E]) => void
  ): () => void {
    if (!this.events.has(event)) {
      this.events.set(event, new Set())
    }
    this.events.get(event)!.add(listener)
    return () => this.events.get(event)?.delete(listener)
  }

  once<E extends keyof Events>(
    event: E,
    listener: (payload: Events[E]) => void
  ): () => void {
    const wrapped = (payload: Events[E]) => {
      listener(payload)
      this.off(event, wrapped)
    }
    return this.on(event, wrapped)
  }

  emit<E extends keyof Events>(event: E, payload: Events[E]): void {
    this.events.get(event)?.forEach(fn => fn(payload))
  }

  off<E extends keyof Events>(
    event: E,
    listener: (payload: Events[E]) => void
  ): void {
    this.events.get(event)?.delete(listener)
  }
}

// 定义事件类型
interface AppEvents {
  'route:change': { from: string; to: string }
  'user:login': { id: number; name: string }
  'user:logout': never  // 无 payload
  'toast': { type: 'success' | 'error' | 'info'; message: string }
}

// 创建实例
const emitter = new TypedEmitter<AppEvents>()

// 使用 —— 完全类型安全
emitter.on('route:change', (payload) => {
  // payload 自动推断为 { from: string; to: string }
  console.log(`从 ${payload.from} 跳转到 ${payload.to}`)
})

emitter.on('toast', ({ type, message }) => {
  // type 自动推断为 'success' | 'error' | 'info'
  console.log(`[${type}] ${message}`)
})

emitter.emit('user:login', { id: 1, name: 'Alice' })

// ❌ 编译时错误
// emitter.emit('user:login', { wrong: true })  // 类型不匹配
// emitter.on('nonexistent', () => {})           // 不存在的事件名
```

// 泛型栈（Stack）—— 后进先出
class Stack<T> {
  private items: T[] = []

  push(item: T): void {
    this.items.push(item)
  }

  pop(): T | undefined {
    return this.items.pop()
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1]
  }

  get size(): number {
    return this.items.length
  }

  get isEmpty(): boolean {
    return this.items.length === 0
  }

  toArray(): T[] {
    return [...this.items]
  }
}

const numberStack = new Stack<number>()
numberStack.push(1)
numberStack.push(2)
numberStack.push(3)
console.log(numberStack.pop())  // 3
console.log(numberStack.peek()) // 2
console.log(numberStack.size)   // 2

// 泛型队列（Queue）—— 先进先出
class Queue<T> {
  private items: T[] = []

  enqueue(item: T): void {
    this.items.push(item)
  }

  dequeue(): T | undefined {
    return this.items.shift()
  }

  front(): T | undefined {
    return this.items[0]
  }

  get size(): number {
    return this.items.length
  }

  get isEmpty(): boolean {
    return this.items.length === 0
  }
}

// 任务队列 —— Vue/Vite 项目中的实际应用
interface Task {
  id: string
  name: string
  priority: number
}

const taskQueue = new Queue<Task>()

taskQueue.enqueue({ id: '2', name: '测试', priority: 2 })

const nextTask = taskQueue.dequeue() // Task | undefined

## 八、泛型与 any 的区别

### 对比表格

| 特性 | 泛型 `<T>` | `any` |
|:---|:---|:---|
| **类型安全** | 保留类型信息，编译时检查 | 完全放弃类型检查 |
| **类型推断** | TypeScript 自动推断具体类型 | 永远推断为 `any` |
| **智能提示** | 完整的属性/方法提示 | 无任何提示 |
| **编译检查** | 不匹配时编译报错 | 不报错，隐患留到运行时 |
| **代码可读性** | 类型意图清晰，自文档化 | 类型意图模糊，需要猜 |
| **重构安全** | 修改类型后，所有引用处报错 | 无任何保护，改漏不报错 |
| **复用性** | 一份代码适配所有类型 | 一份代码，但无类型约束 |
| **运行时影响** | 无（泛型在编译后擦除） | 无（any 在编译后也不存在） |

### 为什么泛型更好

```typescript
// ─── 场景：一个通用的数组去重函数 ───

// 用 any 实现 —— 丢失类型信息
function uniqueAny(arr: any[]): any[] {
  return [...new Set(arr)]
}

const nums = uniqueAny([1, 2, 2, 3])        // any[] —— 无法提示 .map, .toFixed 等
const strs = uniqueAny(['a', 'b', 'b', 'c']) // any[] —— 丧失了 string 类型信息

nums.forEach(n => {
  n.toFixed()   // 不报错，但如果传入的是字符串，运行时 crash
  n.toUpperCase() // 不报错，但数字没有 toUpperCase
})


// 用泛型实现 —— 完整保留类型信息
function unique<T>(arr: T[]): T[] {
  return [...new Set(arr)]
}

const nums2 = unique([1, 2, 2, 3])          // number[] —— 完整的 number 方法提示
const strs2 = unique(['a', 'b', 'b', 'c'])  // string[] —— 完整的 string 方法提示

nums2.forEach(n => {
  n.toFixed()    // ✅ 有提示，且安全
  // n.toUpperCase() // ❌ 编译错误：number 上不存在 toUpperCase
})
```

**一句话总结：`any` 是「我放弃类型检查」，泛型是「我暂时不确定类型，但使用时会确定」。**

## 九、面试常见问题

### 问题 1：什么是泛型？泛型解决了什么问题？

**参考答案：**

泛型（Generics）是一种在定义函数、接口或类时**不预先指定具体类型，而在使用时再指定类型**的机制。泛型的核心价值在于：

1. **类型安全**：在编译时捕获类型错误，而不是运行时才发现
2. **代码复用**：一份代码适用于多种类型，避免为每种类型写重复代码
3. **更好的开发体验**：IDE 能提供精准的自动补全和类型提示

本质上是将「类型」也作为一种参数来传递，实现了类型层面的参数化。

---

### 问题 2：`any` 和泛型有什么区别？什么时候用哪个？

**参考答案：**

| 维度 | `any` | 泛型 |
|:---|:---|:---|
| 本质 | 放弃类型检查 | 延迟类型决定 |
| 类型信息 | 丢失 | 保留 |
| 适用场景 | 确实无法确定类型（如第三方库迁移期） | 需要复用但保留类型安全 |

**选择原则：**

- 需要类型安全和复用 -> 泛型
- 确实无法确定类型、且不关心类型 -> `any`（但应尽量避免）
- 想要「不知道类型但不允许随意操作」 -> `unknown`（比 `any` 更安全的替代）

---

### 问题 3：什么是泛型约束（Generic Constraints）？如何使用 `extends` 和 `keyof`？

**参考答案：**

泛型约束通过 `extends` 关键字限制泛型参数的范围：

```typescript
// extends 约束：T 必须有 length 属性
function logLength<T extends { length: number }>(value: T): void {
  console.log(value.length)
}

// keyof 约束：K 必须是 T 的属性名
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]
}
```

- `extends` 的本质是：要求泛型参数满足某个接口/类型的结构
- `keyof` 的本质是：将一个类型的所有属性名提取为联合类型，常用于约束属性名参数
- 多重约束通过交叉类型 `T extends A & B` 实现

### 问题 4：下面代码输出什么类型？解释原因。

```typescript
function first<T>(arr: T[]): T | undefined {
  return arr[0]
}

const a = first([1, 2, 3])              // ?
const b = first(['hello'])              // ?
const c = first<boolean>([true, false]) // ?
const d = first([])                     // ?
```

**参考答案：**

```typescript
const a = first([1, 2, 3])              // number | undefined
// TypeScript 根据 [1, 2, 3] 推断 T = number，返回 T | undefined

const b = first(['hello'])              // string | undefined
// 推断 T = string，返回 string | undefined

const c = first<boolean>([true, false]) // boolean | undefined
// 显式指定 T = boolean，返回 boolean | undefined

const d = first([])                     // never | undefined
// 空数组无法推断元素类型，TypeScript 推断 T = never
// never 是所有类型的子类型，表示「永远不会出现的值」
```

### 问题 5：请实现一个类型安全的 `deepClone` 函数签名（不需要实现具体逻辑）。

**参考答案：**

```typescript
// 方案 1：基础版
function deepClone<T>(obj: T): T {
  // 实现省略
  return JSON.parse(JSON.stringify(obj))
}

// 方案 2：约束版 —— 限制为可序列化的类型
type Primitive = string | number | boolean | null | undefined
type Serializable = Primitive | Serializable[] | { [key: string]: Serializable }

function deepClone<T extends Serializable>(obj: T): T {
  return JSON.parse(JSON.stringify(obj))
}

// 方案 3：Vue/Vite 项目中的实用版 —— 用 structuredClone（现代浏览器支持）
function deepClone<T>(obj: T): T {
  if (typeof structuredClone === 'function') {
    return structuredClone(obj)
  }
  return JSON.parse(JSON.stringify(obj))
}
```
