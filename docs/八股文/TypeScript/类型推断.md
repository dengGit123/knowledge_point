# 类型推断（Type Inference）

> 官方文档：[Type Inference - TypeScript Handbook](https://www.typescriptlang.org/docs/handbook/type-inference.html)

---

## 一、什么是类型推断

### 通俗解释

想象你去超市购物，把商品放到自助结账机上：

```
+--------------------------------------------------+
|              超市自助结账（类比 TS 类型推断）        |
+--------------------------------------------------+
|                                                  |
|  你放入: 苹果   →  机器自动识别: 🍎 水果类          |
|  你放入: 牛奶   →  机器自动识别: 🥛 乳制品类        |
|  你放入: 面包   →  机器自动识别: 🍞 烘焙类          |
|                                                  |
|  你不需要手动告诉机器"这是水果"                      |
|  机器通过扫描（分析值）就能自动分类                   |
|                                                  |
|  TypeScript 也是如此：                             |
|  你写了 let x = 42 → TS 自动知道 x 是 number       |
|  你不需要手动写 let x: number = 42                 |
+--------------------------------------------------+
```

**类型推断**是指 TypeScript 编译器在没有显式类型注解的情况下，根据代码上下文自动推导出变量、函数返回值等表达式的类型的能力。这是 TypeScript 最核心的便捷特性之一——大多数情况下你不需要处处写类型注解，编译器帮你搞定。

### 什么时候发生类型推断

```
+------------------------------------------------------------------+
|                       类型推断的触发时机                            |
+------------------------------------------------------------------+
|                                                                  |
|  1. 变量初始化时          let name = 'Vue'    → string           |
|                                                                  |
|  2. 函数返回值时          function add(a, b) { return a + b }    |
|                                                                  |
|  3. 上下文类型推断        [1, 2, 3].forEach(n => ...)            |
|                                                                  |
|  4. 成员赋值时            obj.name = 'Vite'  → 推断属性类型       |
|                                                                  |
|  5. 解构赋值时            const { name } = obj  → 从源类型推断     |
|                                                                  |
+------------------------------------------------------------------+
```

## 二、变量初始化推断

### let 与 const 的区别

这是类型推断中一个非常重要的区别：`let` 会推断为宽泛的基础类型，而 `const` 会推断为精确的字面量类型（literal type）。

```typescript
// ==============================
// let：推断为宽泛的基础类型
// ==============================
let age = 25            // 推断为 number，不是 25
let name = 'Vue'        // 推断为 string，不是 'Vue'
let active = true       // 推断为 boolean，不是 true
let items = [1, 2, 3]   // 推断为 number[]

// 因为是 let，后续可以重新赋值为同类型的其他值
age = 30      // ✅ number 可以赋 number
age = 'hello' // ❌ 不能将 string 赋给 number


// ==============================
// const：推断为精确的字面量类型
// ==============================
const age = 25            // 推断为 25（字面量类型），而非 number
const name = 'Vue'        // 推断为 'Vue'（字面量类型），而非 string
const active = true       // 推断为 true（字面量类型），而非 boolean
const items = [1, 2, 3]   // 推断为 number[]（注意：数组本身是可变的，所以元素不是字面量）

// 因为是 const，无法重新赋值
age = 30    // ❌ 无法分配到 "age" ，因为它是常数
```

### 初始值决定推断结果

```typescript
// 有初始值 → 自动推断
let count = 0            // 推断为 number

// 无初始值 → 推断为 any（严格模式下会报错）
let count                // 推断为 any（noImplicitAny 模式下会警告）

// 特殊值
let value = null         // 推断为 any（严格模式下推断为 null）
let data = undefined     // 推断为 any（严格模式下推断为 undefined）

// 对象推断
const config = {
  host: 'localhost',
  port: 3000,
  secure: false
}
// 推断结果：
// {
//   host: string
//   port: number
//   secure: boolean
// }
```

### 对比表格：let vs const 的推断

| 声明方式 | 代码示例 | 推断类型 | 说明 |
|---------|---------|---------|------|
| `let` | `let x = 'hello'` | `string` | 拓宽为通用类型 |
| `const` | `const x = 'hello'` | `'hello'` | 锁定为字面量类型 |
| `let` | `let n = 42` | `number` | 拓宽为通用类型 |
| `const` | `const n = 42` | `42` | 锁定为字面量类型 |
| `let` | `let b = true` | `boolean` | 拓宽为通用类型 |
| `const` | `const b = true` | `true` | 锁定为字面量类型 |
| `const` | `const arr = [1, 2]` | `number[]` | 数组元素仍被拓宽 |

## 三、函数返回值推断

### 自动根据 return 语句推断

TypeScript 会分析函数体中的所有 `return` 语句，综合推断出函数的返回类型。

```typescript
// 单个 return → 直接推断
function double(n: number) {
  return n * 2
}
// 返回类型推断为 number


// 多个 return → 取联合类型
function formatInput(input: string | number) {
  if (typeof input === 'string') {
    return input.toUpperCase()     // string
  }
  return input.toFixed(2)          // string
}
// 返回类型推断为 string


// 无 return → 推断为 void
function logMessage(msg: string) {
  console.log(msg)
}
// 返回类型推断为 void


// never 的推断
function throwError(msg: string) {
  throw new Error(msg)
}
// 返回类型推断为 never（函数永远不会正常返回）


// Vue 场景：返回一个计算属性值
function getFullName(firstName: string, lastName: string) {
  return `${firstName} ${lastName}`
}
// 返回类型推断为 string
```

### 什么时候需要手动注解返回类型

```
+---------------------------------------------------------------+
|           需要手动注解返回类型的场景                               |
+---------------------------------------------------------------+
|                                                               |
|  1. 函数实现较复杂，推断结果不够直观                              |
|     → 团队协作时，显式注解是文档的一部分                          |
|                                                               |
|  2. 需要比推断更宽泛的返回类型                                   |
|     → 推断太精确，限制了对外的灵活性                              |
|                                                               |
|  3. 递归函数，推断器可能无法正确推导                              |
|     → 需要引导编译器                                           |
|                                                               |
|  4. 返回值类型依赖运行时逻辑，推断结果不符合预期                    |
|     → 特别是涉及联合类型、重载的场景                              |
|                                                               |
|  5. 库/公共 API 的导出函数                                      |
|     → 显式注解防止内部实现变化导致外部类型突变                     |
|                                                               |
+---------------------------------------------------------------+
```

```typescript
// 场景 1：复杂逻辑，显式注解更清晰
function getUserDisplayName(user: { firstName: string; lastName: string | null }): string {
  if (user.lastName) {
    return `${user.firstName} ${user.lastName}`
  }
  return user.firstName
}
// 虽然推断也是 string，但显式注解是良好的实践


// 场景 2：需要更宽泛的返回类型
function getOrCreateUser(id: number): { id: number; name: string } | null {
  // 可能在缓存中找到，也可能返回 null
  const cached = userCache.get(id)
  if (cached) return cached
  return null
}


// 场景 3：递归函数
interface TreeNode {
  value: number
  children?: TreeNode[]
}

function flattenTree(node: TreeNode): number[] {
  const result: number[] = [node.value]
  node.children?.forEach(child => {
    result.push(...flattenTree(child))  // 递归调用
  })
  return result
}


// 场景 4：Vue composable 的返回值
function useCounter(initial = 0) {
  const count = ref(initial)
  const increment = () => count.value++
  const decrement = () => count.value--
  const reset = () => { count.value = initial }

  return { count, increment, decrement, reset }
  // 虽然能推断，但公共 API 建议显式注解
}
```

## 四、上下文类型推断（Contextual Typing）

上下文类型推断是类型推断中最精妙的特性之一。当一个表达式的类型不能从自身推断出来时，TypeScript 会根据它**所在的位置**（上下文）来推断类型。

```
+---------------------------------------------------------------+
|                                                               |
|     正向推断（从值到类型）        上下文推断（从上下文到值）        |
|                                                               |
|     let x = 42                  callback: (n: number) => ...  |
|     x 的类型 ← 42              ... 的类型 ← callback 签名      |
|                                                                 |
|     值 → 类型                    上下文 → 值的类型               |
|                                                                 |
+---------------------------------------------------------------+
```

### 回调函数参数的类型推断

```typescript
// 数组的 forEach 回调 —— 参数 n 被自动推断为 number
const scores = [95, 87, 92, 78]
scores.forEach(n => {
  // n 自动推断为 number
  console.log(n.toFixed(1))  // ✅ number 方法可用
})

// map 回调
const doubled = scores.map(n => n * 2)
// n 推断为 number，返回值推断为 number[]

// filter 回调
const highScores = scores.filter(n => n >= 90)
// n 推断为 number，返回值推断为 number[]

// reduce 回调
const total = scores.reduce((acc, cur) => acc + cur, 0)
// acc 推断为 number，cur 推断为 number
```

### 事件处理中的推断

在 Vue 和浏览器事件处理中，上下文类型推断非常实用。

```typescript
// Vue 3 + TypeScript 场景
import { ref, onMounted } from 'vue'

onMounted(() => {
  // 回调参数（无参）自动推断
})

// DOM 事件处理
const button = document.querySelector('#submit-btn')

button?.addEventListener('click', (event) => {
  // event 自动推断为 MouseEvent
  console.log(event.clientX)   // ✅ MouseEvent 的属性
  console.log(event.clientY)   // ✅
  console.log(event.button)    // ✅
})

// Input 事件
const input = document.querySelector<HTMLInputElement>('#name-input')
input?.addEventListener('input', (event) => {
  // event 自动推断为 InputEvent
  const target = event.target as HTMLInputElement
  console.log(target.value)  // ✅ 输入框的值
})

// Vue 模板中的事件处理（在 <script setup> 中）
// <input @input="handleInput" />
function handleInput(event: Event) {
  const target = event.target as HTMLInputElement
  console.log(target.value)
}
```

### 对象字面量的推断

```typescript
// 窗口配置对象的上下文推断
interface WindowConfig {
  width: number
  height: number
  title: string
}

function createWindow(config: WindowConfig) {
  // ...
}

// 传入对象字面量时，属性值会根据上下文推断类型
createWindow({
  width: 800,       // 上下文要求 number，推断为 number ✅
  height: 600,      // 同上
  title: 'My App'   // 上下文要求 string，推断为 string ✅
})

// 回调函数风格的配置
interface FetchOptions {
  url: string
  method?: 'GET' | 'POST' | 'PUT' | 'DELETE'
  onSuccess?: (data: unknown) => void
  onError?: (error: Error) => void
}

function fetchData(options: FetchOptions) {
  // ...
}

fetchData({
  url: '/api/users',
  method: 'GET',
  onSuccess(data) {
    // data 被推断为 unknown
    console.log(data)
  },
  onError(error) {
    // error 被推断为 Error
    console.log(error.message)  // ✅
  }
})
```

## 五、最佳通用类型（Best Common Type）

当 TypeScript 需要从多个表达式中推断出一个共同类型时（比如数组的多个元素），它会计算出一个能兼容所有值的"最佳通用类型"。

### 数组元素类型推断

```typescript
// 所有元素类型相同 → 直接推断
const numbers = [1, 2, 3, 4, 5]
// 推断为 number[]


// 混合类型 → 推断为联合类型数组
const mixed = [1, 'two', 3, 'four']
// 推断为 (string | number)[]


// 有共同基类 → 推断为基类数组
class Animal {
  name = ''
  move() {}
}
class Dog extends Animal {
  bark() {}
}
class Cat extends Animal {
  meow() {}
}

const pets = [new Dog(), new Cat()]
// 推断为 Animal[] —— Dog 和 Cat 的最佳通用类型是 Animal


// 没有共同基类 → 推断为联合类型
const randomItems = [1, 'hello', true]
// 推断为 (string | number | boolean)[]


// 包含 null / undefined
const maybeNumbers = [1, 2, null, 3]
// 推断为 (number | null)[]
```

### 联合类型的推断

```typescript
// 三元表达式
const value = Math.random() > 0.5 ? 'hello' : 42
// 推断为 string | number

// 条件赋值
let result: string | number
if (someCondition) {
  result = 'success'
} else {
  result = 0
}

// 数组的 map 操作
const items: (string | number)[] = [1, 'a', 2, 'b']
const transformed = items.map(item => {
  if (typeof item === 'string') {
    return item.toUpperCase()   // string
  }
  return item * 2               // number
})
// 推断为 (string | number)[]
```

### 推断流程图

```
     多个值需要推断共同类型
              |
              v
     +--------------------+
     | 是否有共同的基类？    |
     +--------------------+
        |            |
       Yes           No
        |            |
        v            v
  取最近的共同     取所有类型的
  祖先类型        联合类型(Union)
        |            |
        v            v
   Animal[]      (A | B)[]
```

## 六、as const 断言（Const Assertion）

`as const` 是 TypeScript 中控制类型推断精度的一把利器。它告诉编译器："把推断的类型锁到最精确的字面量级别，并且标记为只读。"

### 字面量类型的锁定

```typescript
// 不用 as const —— 类型被拓宽
let str = 'hello'           // string
let num = 42                // number
let bool = true             // boolean

// 使用 as const —— 类型被锁定为字面量
let str = 'hello' as const  // 'hello'
let num = 42 as const       // 42
let bool = true as const    // true
```

### 与 readonly 的关系

`as const` 会将所有属性递归地标记为 `readonly`，这是它和 `as` 类型断言的本质区别。

```typescript
// 普通对象 → 属性可变，类型宽泛
const config = {
  host: 'localhost',
  port: 3000,
}
// 推断类型:
// {
//   host: string
//   port: number
// }
config.host = '127.0.0.1'  // ✅ 可以修改


// as const 对象 → 属性只读，类型精确
const config = {
  host: 'localhost',
  port: 3000,
} as const
// 推断类型:
// {
//   readonly host: 'localhost'
//   readonly port: 3000
// }
config.host = '127.0.0.1'  // ❌ 无法分配到 "host" ，因为它是只读属性
```

### 在数组和对象中的使用

```typescript
// ==============================
// 数组：as const 产生 readonly 元组
// ==============================

// 普通数组
const arr = [1, 2, 3]
// 推断为 number[]

// as const 数组
const tuple = [1, 2, 3] as const
// 推断为 readonly [1, 2, 3]  ← 精确的只读元组
tuple.push(4)     // ❌ readonly 上不存在 push
tuple[0] = 10     // ❌ 只读


// ==============================
// 实际应用：定义路由或常量
// ==============================

// Vue Router 风格的路由定义
const ROUTES = [
  { path: '/', name: 'home' },
  { path: '/about', name: 'about' },
  { path: '/contact', name: 'contact' },
] as const

// 推断类型为：
// readonly [
//   { readonly path: '/'; readonly name: 'home' },
//   { readonly path: '/about'; readonly name: 'about' },
//   { readonly path: '/contact'; readonly name: 'contact' }
// ]

// 可以利用字面量类型进行精确的类型检查
type RouteName = typeof ROUTES[number]['name']
// 'home' | 'about' | 'contact'


// ==============================
// 实际应用：Vite 配置中的模式
// ==============================

const MODES = ['development', 'staging', 'production'] as const
type Mode = typeof MODES[number]
// 'development' | 'staging' | 'production'

function isMode(value: string): value is Mode {
  return MODES.includes(value as any)
}


// ==============================
// Redux / Pinia 风格的 action 类型
// ==============================

const UserActions = {
  SET_NAME: 'user/SET_NAME',
  SET_AGE: 'user/SET_AGE',
  RESET: 'user/RESET',
} as const

type UserAction = typeof UserActions[keyof typeof UserActions]
// 'user/SET_NAME' | 'user/SET_AGE' | 'user/RESET'
```

### 对比表格：各种断言方式

| 写法 | 推断结果 | 可变性 | 精确度 |
|------|---------|--------|--------|
| `const obj = { x: 1 }` | `{ x: number }` | 可修改属性 | 低（宽泛） |
| `const obj = { x: 1 } as { x: 1 }` | `{ x: 1 }` | 可修改属性 | 高但不只读 |
| `const obj = { x: 1 } as const` | `{ readonly x: 1 }` | 不可修改 | 最高且只读 |

## 七、类型拓宽（Type Widening）

类型拓宽是指 TypeScript 在某些情况下将字面量类型自动"拓宽"为更宽泛的基础类型。理解拓宽对于精确控制类型至关重要。

### 什么情况类型会被拓宽

```typescript
// ==============================
// 1. let 声明 → 拓宽
// ==============================
let str = 'hello'     // 'hello' → string（拓宽）
let num = 42          // 42 → number（拓宽）
let bool = true       // true → boolean（拓宽）

// ==============================
// 2. 对象属性 → 拓宽
// ==============================
const obj = {
  name: 'Vue',        // string（拓宽了）
  version: 3,         // number（拓宽了）
}
obj.name = 'React'    // ✅ 因为类型是 string，不是 'Vue'

// ==============================
// 3. 数组元素 → 拓宽
// ==============================
const arr = [1, 2, 3]
// number[]，元素类型从字面量 1,2,3 拓宽为 number

// ==============================
// 4. 函数参数的默认值 → 拓宽
// ==============================
function greet(name = 'World') {
  // name 被推断为 string，不是 'World'
  name = 'Vue'  // ✅
}

// ==============================
// 5. 类属性初始化 → 拓宽
// ==============================
class Config {
  env = 'development'   // string（拓宽了）
  port = 3000           // number（拓宽了）
}
```

### 拓宽流程图

```
  字面量值
     |
     v
  赋值给 let 变量 / 对象属性 / 数组元素
     |
     v
  +-------------------+
  |  TypeScript 认为   |
  |  这个值可能会变     |
  |  所以拓宽为通用类型  |
  +-------------------+
     |
     v
  'hello' → string
  42      → number
  true    → boolean
```

### 如何避免拓宽

```typescript
// ==============================
// 方法 1：使用 as const 断言
// ==============================
const config = {
  env: 'development' as const,   // 'development'（不拓宽）
  port: 3000 as const,           // 3000（不拓宽）
}
// config.env 的类型是 'development'，不是 string

// ==============================
// 方法 2：显式类型注解
// ==============================
let env: 'development' | 'production' | 'test' = 'development'
// 不会拓宽为 string，保持联合类型

const theme: 'light' | 'dark' = 'light'
// 不会拓宽为 string

// ==============================
// 方法 3：使用 satisfies + as const（TS 4.9+）
// ==============================
type Theme = {
  colors: {
    primary: string
    secondary: string
  }
}

const theme = {
  colors: {
    primary: '#42b883',
    secondary: '#35495e',
  }
} as const satisfies Theme

// theme.colors.primary 的类型是 '#42b883'（精确字面量）
// 同时满足 Theme 的结构要求

// ==============================
// 方法 4：const + 字面量类型
// ==============================
// const 变量的原始值不拓宽
const ENV = 'development'  // 类型是 'development'，不是 string

// 但 const 对象的属性会拓宽（除非用 as const）
const settings = { debug: true }   // { debug: boolean }
const settings2 = { debug: true } as const  // { readonly debug: true }
```

## 八、typeof 操作符

TypeScript 中的 `typeof` 有两种完全不同的使用场景：**运行时**和**编译时**。

### 运行时 vs 编译时的 typeof

```
+---------------------------------------------------------------+
|                 typeof 的双重身份                                |
+---------------------------------------------------------------+
|                                                               |
|  JavaScript（运行时）          TypeScript（编译时）              |
|  typeof value                 type T = typeof value            |
|  返回: "string" | "number"    返回: value 的精确类型            |
|         | "boolean" | ...            |                         |
|         | string                  完整的类型结构                 |
|  在表达式上下文中使用          在类型上下文中使用                   |
|  运行时执行                    编译时静态分析                     |
|                                                               |
+---------------------------------------------------------------+
```

```typescript
// ==============================
// 运行时 typeof —— 返回字符串描述
// ==============================
console.log(typeof 'hello')    // "string"
console.log(typeof 42)         // "number"
console.log(typeof true)       // "boolean"
console.log(typeof undefined)  // "undefined"
console.log(typeof null)       // "object"（JS 的历史 bug）
console.log(typeof [])         // "object"
console.log(typeof {})         // "object"
console.log(typeof (() => {})) // "function"

// 用于类型守卫
function processValue(value: string | number) {
  if (typeof value === 'string') {
    return value.toUpperCase()  // TS 知道这里是 string
  }
  return value.toFixed(2)      // TS 知道这里是 number
}


// ==============================
// 编译时 typeof —— 获取变量的精确类型
// ==============================
const config = {
  host: 'localhost',
  port: 3000,
  secure: false,
}

// 用 typeof 提取 config 的类型
type Config = typeof config
// 等价于:
// {
//   host: string
//   port: number
//   secure: boolean
// }

// 结合 as const 获取精确字面量类型
const routes = ['home', 'about', 'contact'] as const
type Routes = typeof routes
// readonly ['home', 'about', 'contact']

type RouteValue = typeof routes[number]
// 'home' | 'about' | 'contact'
```

## 九、面试常见问题

### 题目 1：let 和 const 的类型推断有什么区别？

**答案：**

`let` 声明的变量，TypeScript 会将字面量值**拓宽**为对应的基础类型（`string`、`number`、`boolean`），因为 `let` 变量后续可能被重新赋值。

`const` 声明的变量，TypeScript 会将类型推断为**精确的字面量类型**（如 `'hello'` 而非 `string`），因为 `const` 变量无法重新赋值，类型可以锁定在最精确的层面。

```typescript
let a = 'hello'    // string
const b = 'hello'  // 'hello'

let c = 42         // number
const d = 42       // 42
```

但注意：`const` 声明的**对象和数组**，其内部属性/元素仍然会被拓宽：

```typescript
const obj = { name: 'Vue' }   // { name: string }，不是 { name: 'Vue' }
const arr = [1, 2, 3]         // number[]，不是 readonly [1, 2, 3]

// 需要配合 as const 才能锁定
const obj2 = { name: 'Vue' } as const   // { readonly name: 'Vue' }
const arr2 = [1, 2, 3] as const          // readonly [1, 2, 3]
```

### 题目 2：什么是上下文类型推断（Contextual Typing）？请举例说明。

**答案：**

上下文类型推断是指 TypeScript 根据表达式**所在的位置**（而非表达式自身的值）来推断其类型。最典型的场景是回调函数的参数类型。

```typescript
// 数组方法回调：参数 n 的类型由数组的元素类型决定
const names = ['Vue', 'Vite', 'Pinia']
names.forEach(n => {
  // n 自动推断为 string（由上下文 [string[]] 决定）
})

// 事件监听器回调：event 的类型由事件名决定
document.addEventListener('click', (event) => {
  // event 自动推断为 MouseEvent
})

// 函数参数中的回调
function fetchData(cb: (data: { id: number; name: string }) => void) {}
fetchData((data) => {
  // data 自动推断为 { id: number; name: string }
})
```

关键点：没有上下文时，参数会被推断为 `any`：

```typescript
const callback = (n) => n.toFixed(1)
//                        ^^^ ❌ 参数 n 隐式具有 any 类型

const arr = [1, 2, 3]
arr.forEach((n) => n.toFixed(1))  // ✅ n 由上下文推断为 number
```

### 题目 3：as const 的作用是什么？它和 readonly 修饰符有什么区别？

**答案：**

`as const` 的作用有两个：
1. 将类型推断锁定为**最精确的字面量类型**
2. 将所有层级递归地标记为 **`readonly`**

与 `readonly` 修饰符的区别：

```typescript
// readonly 只标记第一层为只读，类型仍然是宽泛的
const obj1: readonly number[] = [1, 2, 3]
// readonly number[] —— 类型是 number，不是 1 | 2 | 3

// as const 递归标记只读，类型是精确字面量
const obj2 = [1, 2, 3] as const
// readonly [1, 2, 3] —— 类型是精确的元组

// readonly 修饰符需要逐层添加
interface Obj3 {
  readonly data: {
    readonly value: number  // 需要手动给每一层加 readonly
  }
}

// as const 一次性递归全部标记
const obj4 = {
  data: {
    value: 42
  }
} as const
// { readonly data: { readonly value: 42 } }
```

实际应用场景：定义常量集合、精确的联合类型推导、Vue/Pinia 的 action 类型等。

### 题目 4：TypeScript 中 typeof 在运行时和编译时分别有什么作用？

**答案：**

`typeof` 在 TypeScript 中根据使用位置有两种完全不同的含义：

**运行时（表达式上下文）**：返回一个描述类型的字符串，是 JavaScript 原生运算符。

```typescript
typeof 'hello'  // "string"
typeof 42       // "number"
typeof null     // "object"  （JS 历史遗留问题）
```

**编译时（类型上下文）**：获取变量/表达式的精确 TypeScript 类型，是 TS 扩展的类型查询运算符。

```typescript
const config = { host: 'localhost', port: 3000 }

// 类型上下文中使用 typeof
type Config = typeof config
// { host: string; port: number }

// 结合工具类型使用
type ConfigKeys = keyof typeof config  // 'host' | 'port'
```

区分方法：`typeof` 出现在**类型声明**的位置（`type` 后面、接口中、泛型参数中）就是编译时的；出现在**表达式**的位置就是运行时的。

```typescript
// 运行时
if (typeof x === 'string') { ... }

// 编译时
type T = typeof x
function fn(arg: typeof x) { ... }
```

### 题目 5：什么是类型拓宽（Widening）？如何避免？

**答案：**

类型拓宽是指 TypeScript 在推断类型时，自动将字面量类型"放宽"为更宽泛的基础类型的行为。这种设计是为了适配变量可能被重新赋值的场景。

```typescript
let name = 'Vue'       // 'Vue' 被拓宽为 string
const obj = {
  version: 3           // 3 被拓宽为 number
}
```

避免拓宽的三种方法：

| 方法 | 示例 | 推断结果 |
|------|------|---------|
| `as const` 断言 | `let x = 'hello' as const` | `'hello'` |
| 显式类型注解 | `let x: 'hello' = 'hello'` | `'hello'` |
| `const` 声明 | `const x = 'hello'` | `'hello'` |

```typescript
// 避免拓宽的实际场景
type Env = 'development' | 'staging' | 'production'

// 写法 1：显式注解
const env: Env = 'development'

// 写法 2：as const
const env = 'development' as const

// 写法 3：satisfies + as const（TS 4.9+）
const config = {
  env: 'development',
  port: 3000,
} as const satisfies Record<string, string | number>
// env 类型是 'development'，不会拓宽为 string
```

核心原则：当你需要精确的字面量类型时，主动使用 `as const` 或显式注解来告诉编译器"不要拓宽"。
