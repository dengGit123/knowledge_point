## 一、原始类型（Primitive Types）

JavaScript 有 7 种原始类型，TypeScript 完全继承并支持它们：

```
┌──────────────────────────────────────────────┐
│              TypeScript 原始类型              │
├──────────┬───────────────────────────────────┤
│ boolean  │ 布尔值：true / false              │
│ number   │ 数字：整数、浮点数、NaN、Infinity  │
│ string   │ 字符串：'hello' / "hi" / `world`  │
│ bigint   │ 大整数：100n（ES2020+）            │
│ symbol   │ 唯一标识：Symbol('id')             │
│ null     │ 空值：明确的"没有值"               │
│ undefined│ 未定义：变量声明但未赋值            │
└──────────┴───────────────────────────────────┘
```

> 通俗类比：原始类型就像**单个原子** —— 它们是最小的、不可再分的数据单元。与之相对，对象类型（object）是由多个原子组合而成的**分子**。

### 1.1 boolean — 布尔值

最简单的类型，只有 `true` 和 `false` 两个值。

```typescript
let isDone: boolean = false
let hasPermission: boolean = true

// Vue 组件中的常见用法
const isLoading = ref<boolean>(false)
const isVisible = ref<boolean>(true)
```

```typescript
// ⚠️ 注意：Boolean（大写）是构造函数，boolean（小写）才是类型
let wrong: Boolean = true    // 不推荐，这是包装对象类型
let right: boolean = true    // ✅ 推荐
```

### 1.2 number — 数字

TypeScript 中的 `number` 涵盖了 JavaScript 的所有数字形式：整数、浮点数、二进制、八进制、十六进制。

```typescript
let decimal: number = 6
let hex: number = 0xf00d      // 十六进制
let binary: number = 0b1010   // 二进制
let octal: number = 0o744     // 八进制
let price: number = 99.99
let invalid: number = NaN
let infinite: number = Infinity
```

```typescript
// Vue 组件中的实际场景
interface Product {
  id: number
  price: number
  stock: number
}

const products = ref<Product[]>([
  { id: 1, price: 29.9, stock: 100 },
  { id: 2, price: 49.9, stock: 0 },
])
```

### 1.3 string — 字符串

使用单引号 `'`、双引号 `"` 或模板字符串 `` ` `` 包裹的文本数据。

```typescript
let name: string = '张三'
let greeting: string = `你好，${name}`  // 模板字符串

// Vue 组件中的常见用法
const title = ref<string>('首页')
const user = reactive<{ name: string; email: string }>({
  name: '李四',
  email: 'lisi@example.com',
})
```

### 1.4 bigint — 大整数

ES2020 引入，用于表示大于 `Number.MAX_SAFE_INTEGER`（2^53 - 1）的整数。

```typescript
let big: bigint = 100n          // 字面量后缀 n
let alsoBig: bigint = BigInt(100) // 使用 BigInt() 函数

// 超出安全范围的计算
const maxSafe: number = Number.MAX_SAFE_INTEGER  // 9007199254740991
const bigger: bigint = 9007199254740992n          // ✅ bigint 可以精确表示

// ⚠️ bigint 和 number 不能混合运算
// let result = 100n + 50  // ❌ 报错：不能将 bigint 和 number 混合
let result = 100n + BigInt(50) // ✅ 需要显式转换
```

### 1.5 symbol — 唯一标识

每次调用 `Symbol()` 都会返回一个**全局唯一**的值。

```typescript
let uniqueId: symbol = Symbol('id')
let anotherId: symbol = Symbol('id')

console.log(uniqueId === anotherId) // false — 每个 Symbol 都是唯一的

// 常见用途：作为对象的"私有"属性键
const secretKey: symbol = Symbol('secret')

const obj: Record<symbol, string> = {
  [secretKey]: 'hidden value',
}

// Vue/Vite 中常见：用于插件标识
interface Plugin {
  id: symbol
  install: () => void
}
```

### 1.6 null 和 undefined — 空值双子星

这两个类型既是**类型**也是**值**，是 JavaScript/TypeScript 中最容易混淆的概念。

```
┌─────────────────────────────────────────────────────────────┐
│                    null vs undefined                         │
├──────────────┬──────────────────────┬───────────────────────┤
│     对比项    │       null           │      undefined        │
├──────────────┼──────────────────────┼───────────────────────┤
│ 含义         │ 明确的"空值"          │ "尚未赋值"            │
│ typeof 返回  │ "object"（历史遗留）  │ "undefined"           │
│ 谁产生的     │ 开发者手动赋值        │ JavaScript 引擎默认   │
│ 转为 number  │ 0                    │ NaN                   │
│ 语义         │ "这里应该有值，但现在是空的" │ "这里还没有值"   │
│ JSON.stringify │ 输出 "null"        │ 被忽略（不输出）       │
└──────────────┴──────────────────────┴───────────────────────┘
```

```typescript
// undefined：变量声明了但没赋值
let age: number
console.log(age) // undefined

// null：开发者主动赋值，表示"空"
let selectedUser: { name: string } | null = { name: '张三' }
selectedUser = null // 用户取消了选择

// Vue 组件中的典型场景
const currentUser = ref<{ name: string } | null>(null)

onMounted(async () => {
  // 初始状态是 null（还没加载）
  currentUser.value = null
  // 加载完成后赋值
  currentUser.value = await fetchUser()
})
```

```typescript
// ⚠️ strictNullChecks 的影响
// 当 strictNullChecks: true（推荐开启）时：
let name: string = undefined  // ❌ 不能将 undefined 赋给 string
let age: number = null        // ❌ 不能将 null 赋给 number

// 正确写法：使用联合类型
let name: string | undefined = undefined  // ✅
let age: number | null = null             // ✅
```

## 二、数组与元组（Array & Tuple）

### 2.1 数组（Array）

TypeScript 中数组有两种等价写法：

```
写法一：元素类型[]     → number[]        （推荐，更简洁）
写法二：Array<元素类型> → Array<number>   （泛型写法，更通用）
```

```typescript
// 两种写法完全等价
let scores: number[] = [90, 85, 92, 88]
let names: Array<string> = ['张三', '李四', '王五']

// 推荐使用 T[] 语法，更简洁
const products: { id: number; name: string }[] = [
  { id: 1, name: '手机' },
  { id: 2, name: '电脑' },
]

// 使用泛型写法的场景（当类型复杂时更清晰）
type ResponseData = Array<{
  id: number
  title: string
  createdAt: string
}>
```

```typescript
// Vue 3 中的常见用法
import { ref } from 'vue'

const todoList = ref<string[]>([])
const numbers = ref<number[]>([1, 2, 3])

// 添加元素时自动类型检查
todoList.value.push('学习 TypeScript') // ✅
// todoList.value.push(123)            // ❌ 类型 number 不能赋给 string
```

```typescript
// ⚠️ 注意：不要混淆 number[] 和 [number]
let a: number[] = [1, 2, 3]    // 数组：任意长度的 number 列表
let b: [number] = [1]          // 元组：固定长度为 1，且第一个元素是 number
```

### 2.2 元组（Tuple）

元组是**长度固定、每个位置类型确定**的数组。可以理解为"精确到每个位置的数组"。

```
┌────────────────────────────────────────────────────┐
│                 数组 vs 元组                         │
├────────────┬───────────────────────────────────────┤
│   数组     │ 长度不限，所有元素类型相同               │
│   number[] │ [1, 2, 3, 4, ...] 任意多个 number     │
├────────────┼───────────────────────────────────────┤
│   元组     │ 长度固定，每个位置类型可不同              │
│   [string, number] │ ['张三', 25] 刚好2个元素      │
└────────────┴───────────────────────────────────────┘
```

```typescript
// 基本元组
let user: [string, number] = ['张三', 25]
//            ↑ 类型1     ↑ 类型2
//            索引0       索引1

// 访问元素时，TypeScript 知道每个位置的确切类型
user[0].toUpperCase()  // ✅ TS 知道这是 string
user[1].toFixed(2)     // ✅ TS 知道这是 number
// user[0].toFixed(2)  // ❌ string 上没有 toFixed 方法
```

```typescript
// 实际场景：函数返回多个不同类型的值
function useState<T>(initial: T): [T, (val: T) => void] {
  let state = initial
  const setState = (val: T) => {
    state = val
  }
  return [state, setState]
}

const [count, setCount] = useState(0)       // count: number, setCount: (val: number) => void
const [name, setName] = useState('张三')    // name: string, setName: (val: string) => void

// 这正是 Vue 3 的 ref 和 React 的 useState 的底层原理！
// Vue 3 示例：
import { ref } from 'vue'
const count = ref(0)  // count.value 是 number 类型
```

```typescript
// 实际场景：API 返回的键值对
const entries: [string, number][] = [
  ['语文', 90],
  ['数学', 95],
  ['英语', 88],
]

// 类似 Object.entries() 的返回类型
const obj = { name: '张三', age: 25 }
// Object.entries(obj) 的类型是 [string, string | number][]
```

### 2.3 可选元素与 rest 元素

```typescript
// 可选元素：用 ? 标记，放在最后面
let info: [string, number?, boolean?] = ['张三']
info    // ✅ 长度可以是 1、2 或 3

// rest 元素：用 ... 表示"剩余部分"
type StringNumberBooleans = [string, number, ...boolean[]]
//                          固定两个   ^^^^ 后面可以有任意多个 boolean

const a: StringNumberBooleans = ['hello', 1]                   // ✅
const b: StringNumberBooleans = ['hello', 1, true]             // ✅
const c: StringNumberBooleans = ['hello', 1, true, false, true] // ✅

// rest 元素放在前面
type BooleansStringNumber = [...boolean[], string, number]
const d: BooleansStringNumber = ['hello', 1]                        // ✅
const e: BooleansStringNumber = [true, false, 'hello', 1]           // ✅

// 实际场景：函数参数的类型安全约束
function logMessage(level: 'info' | 'warn' | 'error', ...messages: string[]): void {
  console.log(`[${level}]`, ...messages)
}
```

### 2.4 只读元组

```typescript
// 使用 readonly 修饰元组，防止修改
const point: readonly [number, number] = [0, 0]
// point[0] = 1  // ❌ 无法赋值，因为它是只读的
// point.push(3) // ❌ 属性 push 在类型 readonly [number, number] 上不存在

// as const 也能产生 readonly 元组
const tuple = [1, 'hello'] as const
// tuple 的类型是 readonly [1, "hello"]（字面量类型 + 只读）
```

## 三、特殊类型

TypeScript 在 JavaScript 原始类型的基础上，新增了几个**仅存在于类型系统**中的特殊类型：

```
┌──────────────────────────────────────────────────────────────────┐
│                     TypeScript 特殊类型总览                        │
├──────────┬───────────────────────────────────────────────────────┤
│  void    │ 函数没有返回值时使用                                    │
│  never   │ 永远不会存在的值（不可能到达的终点）                      │
│  any     │ 完全放弃类型检查（类型系统的"逃生舱"）                    │
│  unknown │ 类型安全的顶级类型（any 的安全替代）                      │
└──────────┴───────────────────────────────────────────────────────┘
```

### 3.1 void — 空的

`void` 表示**函数没有返回值**。它只在函数返回类型中有意义，**不应该用来声明变量**。

```typescript
// void 作为函数返回类型
function logMessage(msg: string): void {
  console.log(msg)
  // 没有返回值，等价于 return undefined
}

// Vue 3 组件中的常见用法
function handleClick(): void {
  console.log('按钮被点击了')
}

// 回调函数
function fetchData(callback: (data: string) => void): void {
  callback('数据加载完成')
}
```

```typescript
// ⚠️ void 变量没有实际用途
let unusable: void = undefined // 技术上可以，但毫无意义
// unusable = null             // strictNullChecks 开启时报错
```

### 3.2 never — 永远不会到达

`never` 表示**永远不会存在的值**。它是所有类型的子类型，但没有任何类型是它的子类型。

```
never 的三种典型场景：

1. 函数抛出异常        → throw new Error()
2. 函数永远不会返回     → while(true) {} 死循环
3. 类型收窄到不可能的分支 → else 分支的类型保护
```

```typescript
// 场景一：抛出异常
function throwError(message: string): never {
  throw new Error(message)
}

// 场景二：死循环
function infiniteLoop(): never {
  while (true) {
    // 永远不会退出
  }
}

// 场景三：类型收窄的"兜底"
type Shape = 'circle' | 'square'

function getArea(shape: Shape): number {
  switch (shape) {
    case 'circle':
      return Math.PI * 10
    case 'square':
      return 10 * 10
    default:
      // 如果所有 case 都处理了，shape 在这里是 never
      const _exhaustiveCheck: never = shape
      return _exhaustiveCheck
  }
}

// 如果将来新增 type Shape = 'circle' | 'square' | 'triangle'
// 但忘记在 switch 中处理，default 分支会报错 → 帮你发现遗漏！
```

```typescript
// 实际应用：在 Vue 组件中使用 never 做穷尽检查
type Status = 'idle' | 'loading' | 'success' | 'error'

function getStatusText(status: Status): string {
  switch (status) {
    case 'idle': return '空闲'
    case 'loading': return '加载中'
    case 'success': return '成功'
    case 'error': return '错误'
    default:
      const exhaustive: never = status
      return exhaustive
  }
}
```

### 3.3 any — 放弃类型检查

`any` 是 TypeScript 的"逃生舱"，它会**完全关闭类型检查**。使用 `any` 等于退回到了 JavaScript。

```
┌──────────────────────────────────────────────────────────┐
│              any：类型系统的"免死金牌"                      │
│                                                          │
│   一旦给变量标上 any：                                     │
│   ✅ 可以赋任何值                                         │
│   ✅ 可以调用任何方法                                     │
│   ✅ 可以当作任何类型使用                                  │
│   ❌ 但也失去了所有类型保护                                │
└──────────────────────────────────────────────────────────┘
```

```typescript
let data: any = 'hello'
data = 42           // ✅ 可以赋值 number
data = true         // ✅ 可以赋值 boolean
data.foo()          // ✅ 可以调用不存在的方法（运行时报错！）
data.bar.baz        // ✅ 可以访问不存在的属性（运行时报错！）

// ⚠️ any 会"污染"其他变量
let a: any = 'hello'
let b: number = a   // ✅ 编译通过！但运行时 b 其实是 string
b.toFixed()         // 运行时报错：b.toFixed is not a function
```

```typescript
// 什么时候不得不用 any？
// 1. 第三方库没有类型声明，且来不及写 .d.ts
// 2. 快速原型开发，后续再补类型
// 3. 类型转换的中间步骤

// 但大多数情况下，应该用 unknown 替代 any
```

### 3.4 unknown — 类型安全的 any

`unknown` 是 TypeScript 3.0 引入的，是 `any` 的**安全替代品**。它是**顶级类型**（所有类型都可以赋给它），但**使用前必须收窄类型**。

```
┌──────────────────────────────────────────────────────────────┐
│                 any vs unknown 对比                            │
├───────────────┬──────────────────────┬───────────────────────┤
│     对比项     │        any           │      unknown          │
├───────────────┼──────────────────────┼───────────────────────┤
│ 接收任意值    │ ✅                    │ ✅                     │
│ 赋给其他变量  │ ✅ 直接赋             │ ❌ 必须先收窄类型      │
│ 调用方法      │ ✅ 随便调             │ ❌ 必须先收窄类型      │
│ 类型安全      │ ❌ 完全不安全          │ ✅ 强制类型检查        │
│ 推荐程度      │ ⚠️ 尽量避免           │ ✅ 优先使用            │
└───────────────┴──────────────────────┴───────────────────────┘
```

```typescript
// unknown：可以接收任意值
let value: unknown = 'hello'
value = 42        // ✅
value = { a: 1 }  // ✅

// 但使用前必须"收窄"类型
let val: unknown = 'hello'

// val.toUpperCase()  // ❌ 对象的类型为 unknown

// ✅ 通过类型收窄后使用
if (typeof val === 'string') {
  console.log(val.toUpperCase()) // ✅ TS 知道这里 val 是 string
}
```

```typescript
// 实际场景：API 响应数据
async function fetchData(): Promise<unknown> {
  const res = await fetch('/api/data')
  return res.json()
}

// 使用时必须先校验类型
const data: unknown = await fetchData()

// 方式一：typeof 收窄
if (typeof data === 'string') {
  console.log(data.toUpperCase())
}

// 方式二：类型谓词（自定义类型守卫）
function isUser(data: unknown): data is { name: string; age: number } {
  return (
    typeof data === 'object' &&
    data !== null &&
    'name' in data &&
    'age' in data
  )
}

if (isUser(data)) {
  console.log(data.name) // ✅ TS 知道 data 有 name 属性
}
```

### 3.5 null / undefined 作为类型的特殊说明

```typescript
// TypeScript 中 null 和 undefined 是独立的类型
let u: undefined = undefined
let n: null = null

// 它们也是所有类型的子类型（strictNullChecks 关闭时）
// strictNullChecks: false（不推荐）
let name2: string = null      // ✅（危险！）
let age2: number = undefined  // ✅（危险！）

// strictNullChecks: true（推荐，Vite 项目默认开启）
// let name3: string = null      // ❌
// let age3: number = undefined  // ❌

// 正确做法：使用联合类型
let maybeName: string | null = null          // ✅
let maybeAge: number | undefined = undefined // ✅

// Vue 中的常见模式
const selectedId = ref<number | null>(null)

// 使用非空断言（!）告诉 TS "我确定它不是 null"
function processId(id: number | null): string {
  return id!.toString() // ⚠️ 确保运行时 id 确实不是 null
}

// 更安全的方式：类型守卫
function processIdSafe(id: number | null): string {
  if (id === null) {
    return '无'
  }
  return id.toString() // ✅ TS 自动收窄为 number
}
```

```
                    unknown（顶级类型，所有值的父类型）
                      │
          ┌───────────┼───────────┐
          │           │           │
        number      string    boolean ...（原始类型）
          │           │           │
          └───────────┼───────────┘

                    never（底部类型，所有类型的子类型）
                      │
                      │

```

## 四、对象类型（Object Types）

### 4.1 object vs Object — 一字之差，天壤之别

```
┌─────────────────────────────────────────────────────────────────────┐
│                   三种"对象"类型的区别                                 │
├──────────┬──────────────────────────────────────────────────────────┤
│ object   │ 非原始类型。包括对象、数组、函数、Map、Set 等所有非原始值   │
│          │ 排除：string, number, boolean, symbol, bigint, null, undefined │
├──────────┼──────────────────────────────────────────────────────────┤
│ Object   │ JavaScript 的 Object 构造函数实例类型                     │
│          │ 几乎所有值都是 Object（包括原始类型的包装对象）              │
│          │ 实际开发中几乎不使用                                       │
├──────────┼──────────────────────────────────────────────────────────┤
│ {}       │ 空对象类型（等价于 Object）                                │
│          │ 任何有属性的对象都可以赋值给它                               │
└──────────┴──────────────────────────────────────────────────────────┘
```

```typescript
// object：排除所有原始类型
let obj: object = { name: '张三' }     // ✅
obj = [1, 2, 3]                        // ✅ 数组
obj = () => {}                         // ✅ 函数
obj = new Map()                        // ✅ Map
// obj = 'hello'                       // ❌ string 是原始类型
// obj = 42                            // ❌ number 是原始类型
// obj = null                          // ❌ null 是原始类型

// Object（大写 O）：几乎所有值都兼容
let bigObj: Object = { name: '张三' }   // ✅
bigObj = 'hello'                       // ✅ string 也是 Object（包装对象）
bigObj = 42                            // ✅ number 也是 Object（包装对象）

// {} 空对象类型：和 Object 几乎等价
let empty: {} = { name: '张三' }       // ✅
empty = 'hello'                        // ✅
```

> **最佳实践**：优先使用具体的对象类型（如 `{ name: string; age: number }`），避免使用 `object`、`Object` 或 `{}`。

### 4.2 匿名对象类型

TypeScript 中定义对象类型有两种方式：**内联（匿名）** 和 **type/interface 命名**。

```typescript
// 方式一：匿名对象类型（适合简单、一次性使用）
function printUser(user: { name: string; age: number }): void {
  console.log(`${user.name}, ${user.age}岁`)
}

printUser({ name: '张三', age: 25 }) // ✅
// printUser({ name: '张三' })       // ❌ 缺少 age 属性
```

```typescript
// 方式二：type 别名（适合复用）
type User = {
  name: string
  age: number
}

function printUser(user: User): void {
  console.log(`${user.name}, ${user.age}岁`)
}

// 方式三：interface（适合复用 + 继承）
interface IUser {
  name: string
  age: number
}
```

```typescript
// Vue 组件中推荐使用 interface
interface Props {
  title: string
  count?: number
  items: string[]
}

// 使用 defineProps
const props = defineProps<Props>()
```

### 4.3 可选属性（?）

用 `?` 标记的属性表示**该属性可以不存在**。

```typescript
interface Config {
  host: string        // 必填
  port: number        // 必填
  timeout?: number    // 可选，可以不传
  retry?: boolean     // 可选
}

const config1: Config = { host: 'localhost', port: 3000 }           // ✅
const config2: Config = { host: 'localhost', port: 3000, timeout: 5000 } // ✅

// 使用可选属性前需要检查
function connect(config: Config): void {
  const timeout = config.timeout ?? 3000 // 使用 ?? 提供默认值
  console.log(`连接 ${config.host}:${config.port}，超时 ${timeout}ms`)
}
```

```typescript
// Vue 组件 defineProps 中的可选属性
interface Props {
  title: string      // 必填
  subtitle?: string  // 可选
  theme?: 'light' | 'dark'
}

const props = withDefaults(defineProps<Props>(), {
  subtitle: '',
  theme: 'light',
})
```

### 4.4 只读属性（readonly）

用 `readonly` 修饰的属性**只能在创建时赋值，之后不可修改**。

```typescript
interface Point {
  readonly x: number
  readonly y: number
}

const point: Point = { x: 10, y: 20 }
// point.x = 30  // ❌ 无法赋值，因为是只读属性

// ⚠️ readonly 只是浅层只读
interface UserInfo {
  readonly name: string
  readonly scores: number[]  // 数组本身是 readonly，但数组内容不是
}

const user: UserInfo = { name: '张三', scores: [90, 85] }
// user.name = '李四'       // ❌ 无法赋值
user.scores.push(95)         // ✅ 可以修改数组内容！readonly 是浅层的
```

```typescript
// 深层只读：使用 Readonly 或手动嵌套
interface DeepReadonlyUser {
  readonly name: string
  readonly scores: readonly number[]  // 深层只读
}

const deepUser: DeepReadonlyUser = { name: '张三', scores: [90, 85] }
// deepUser.scores.push(95)  // ❌ 报错，数组也是只读的

// Vue 3 中的 readonly()
import { readonly, reactive } from 'vue'

const state = reactive({ count: 0 })
const readOnlyState = readonly(state)
// readOnlyState.count = 1  // ❌ Vue 也会阻止修改
```

### 4.5 索引签名

当你不确定对象有哪些属性名，但知道属性值的类型时，使用索引签名。

```typescript
// 索引签名：键是 string，值是 number
interface ScoreMap {
  [subject: string]: number
}

const scores: ScoreMap = {
  语文: 90,
  数学: 95,
  英语: 88,
}

// 也支持数字键
interface NumberMap {
  [key: number]: string
}

const names: NumberMap = ['张三', '李四'] // ✅ 数组的数字索引

// Vue/Vite 环境变量的常见类型
interface ImportMetaEnv {
  [key: `VITE_${string}`]: string
}
```

## 五、类型注解 vs 类型推断

### 5.1 核心概念

```
┌──────────────────────────────────────────────────────────────┐
│               类型注解 vs 类型推断                              │
├──────────────┬───────────────────────────────────────────────┤
│  类型注解     │ 开发者手动指定类型                              │
│              │ let name: string = '张三'                     │
│              │ "我告诉 TS 这是什么类型"                        │
├──────────────┼───────────────────────────────────────────────┤
│  类型推断     │ TypeScript 自动推导出类型                      │
│              │ let name = '张三'  // TS 推断为 string         │
│              │ "TS 自己猜出来是什么类型"                       │
└──────────────┴───────────────────────────────────────────────┘
```

> 通俗类比：类型注解就像在快递上写清楚"易碎品"，类型推断就像快递员通过包裹大小自己判断。**如果一目了然，就让 TS 自己判断；如果存在歧义，就手动标注。**

### 5.2 类型推断的常见场景

```typescript
// 1. 变量初始化时自动推断
let name = '张三'       // 推断为 string
let age = 25            // 推断为 number
let isActive = true     // 推断为 boolean
let list = [1, 2, 3]   // 推断为 number[]

// 2. 函数返回值自动推断
function add(a: number, b: number) {
  return a + b // TS 自动推断返回值类型为 number
}

// 3. 对象属性自动推断
const user = {
  name: '张三',     // string
  age: 25,          // number
  hobbies: ['读书'], // string[]
}
// user 的类型被自动推断为 { name: string; age: number; hobbies: string[] }
```

### 5.3 什么时候需要手动注解

```
需要手动注解的情况：
┌────────────────────────────────────────────────────────────┐
│ 1. 变量声明但不立即初始化                                     │
│ 2. 函数参数（TS 无法推断参数类型）                             │
│ 3. 类型推断结果不够精确（需要更宽或更窄的类型）                  │
│ 4. 联合类型场景（值可能是多种类型）                             │
│ 5. "意外推断" 导致类型不对                                    │
└────────────────────────────────────────────────────────────┘
```

```typescript
// 1. 变量声明但不初始化 → 必须注解
let userName: string  // 如果不注解，TS 推断为 any（strict 模式下报错）

// 2. 函数参数 → 必须注解
function greet(name: string, age: number): string {
  return `你好，${name}，${age}岁`
}

// 3. 需要更精确的类型
let id: string | number = 'abc'  // 如果不注解，TS 会推断为 string
id = 123                          // ✅

// 4. 联合类型
let data: string | null = null
data = '加载完成'

// 5. 避免意外推断
let items = []           // 推断为 any[]（危险！）
let items2: string[] = [] // ✅ 明确指定为 string[]
```

### 5.4 最佳实践总结

| 场景 | 做法 | 示例 |
|------|------|------|
| 变量声明且立即赋值 | 省略注解，让 TS 推断 | `let name = '张三'` |
| 变量声明但延迟赋值 | 手动注解 | `let name: string` |
| 函数参数 | 手动注解 | `function fn(a: number)` |
| 函数返回值 | 通常省略，复杂时注解 | `function fn(): string` |
| 对象/数组 | 简单场景省略，复杂时注解 | `const list: number[] = []` |
| 可能是多种类型 | 手动注解联合类型 | `let id: string \| number` |

## 六、类型兼容性

### 6.1 结构化类型系统（Structural Type System）

TypeScript 采用的是**结构化类型系统**（也叫"鸭子类型"——Duck Typing）。

> 通俗类比：**"如果它走起来像鸭子，叫起来像鸭子，那它就是鸭子。"**
>
> TypeScript 不关心变量叫什么名字（名义类型），只关心它**长得什么样**（结构）。

```
┌─────────────────────────────────────────────────────────────────┐
│               名义类型 vs 结构化类型                              │
├──────────────┬──────────────────────────────────────────────────┤
│  名义类型    │ Java/C# 等语言：类名必须匹配                       │
│  （Nominal） │ class Dog {} class Cat {}                        │
│              │ Dog 不能赋给 Cat，即使结构完全相同                  │
├──────────────┼──────────────────────────────────────────────────┤
│  结构化类型  │ TypeScript：只看结构是否匹配                        │
│（Structural）│ type Dog = { name: string }                     │
│              │ type Cat = { name: string }                      │
│              │ Dog 可以赋给 Cat，因为结构相同 ✅                  │
└──────────────┴──────────────────────────────────────────────────┘
```

```typescript
// 结构化类型的核心：只要形状匹配，就兼容
interface Point2D {
  x: number
  y: number
}

interface Point3D {
  x: number
  y: number
  z: number
}

const point3d: Point3D = { x: 1, y: 2, z: 3 }
const point2d: Point2D = point3d // ✅ Point3D 包含 Point2D 的所有属性

// 通俗理解：Point3D 是 Point2D 的"超集"
// "有 x 和 y 的都能当 Point2D 用"
```

### 6.2 赋值兼容规则

**核心原则：目标类型需要的属性，源类型必须都有（多余没关系，少了不行）。**

```
兼容性检查规则：
┌───────────────────────────────────────────────┐
│                                               │
│   源类型（实际值）  ──赋值给──▶  目标类型（变量）  │
│                                               │
│   ✅ 源类型的属性 ≥ 目标类型需要的属性            │
│   ❌ 源类型的属性 < 目标类型需要的属性            │
│                                               │
│   "多的可以，少的不行"                          │
│                                               │
└───────────────────────────────────────────────┘
```

```typescript
// ✅ 多的属性可以
interface Animal {
  name: string
}

interface Dog extends Animal {
  breed: string
}

const dog: Dog = { name: '旺财', breed: '柯基' }
const animal: Animal = dog // ✅ Dog 有 Animal 需要的 name 属性

// ❌ 少的属性不行
const cat = { name: '喵喵' }
// const dog2: Dog = cat  // ❌ 缺少 breed 属性
```

```typescript
// 函数参数的兼容性（协变与逆变）

// 协变：函数参数类型可以接收子类型
function logName(animal: { name: string }): void {
  console.log(animal.name)
}

const dog = { name: '旺财', breed: '柯基' }
logName(dog) // ✅ dog 包含 name 属性

// 函数返回值的兼容性
function createAnimal(): { name: string; age: number } {
  return { name: '小猫', age: 1 }
}

const animal: { name: string } = createAnimal() // ✅ 返回值包含 name

// 实际场景：Vue 组件 Props 兼容性
interface BaseButtonProps {
  text: string
  onClick: () => void
}

interface IconButtonProps extends BaseButtonProps {
  icon: string
}

// IconButton 的 Props 可以传给只需要 BaseButton Props 的组件
function renderButton(props: BaseButtonProps) {
  // ...
}

const iconProps: IconButtonProps = {
  text: '保存',
  onClick: () => {},
  icon: 'save',
}
renderButton(iconProps) // ✅ 兼容
```

### 6.3 原始类型的兼容性

```typescript
// number 兼容 number 枚举（反过来不行）
enum Direction { Up, Down, Left, Right }
let dir: Direction = Direction.Up
let num: number = dir           // ✅ 枚举可以赋给 number
// let dir2: Direction = 0      // ❌ number 不能赋给枚举（需要类型断言）

// null 和 undefined 的兼容性
// strictNullChecks: true 时
let str: string = 'hello'
// str = null      // ❌
// str = undefined  // ❌
let maybeStr: string | null = null // ✅
```

### 6.4 类型兼容性速查表

| 源类型 | 目标类型 | 兼容？ | 说明 |
|--------|---------|--------|------|
| `{ a, b, c }` | `{ a, b }` | ✅ | 多余属性可以 |
| `{ a }` | `{ a, b }` | ❌ | 缺少属性不行 |
| `string` | `string \| number` | ✅ | 联合类型包含成员 |
| `any` | 任意类型 | ✅ | any 可以赋给任何类型 |
| `unknown` | 任意类型 | ❌ | unknown 不能直接赋值（需收窄） |
| `never` | 任意类型 | ✅ | never 可以赋给任何类型 |
| 子接口 | 父接口 | ✅ | 子集兼容超集 |

## 七、面试常见问题

### Q1：TypeScript 中 type 和 interface 的区别？

**答案：**

两者大多数场景可以互换，但有细微区别：

```
┌──────────────────┬──────────────────────┬───────────────────────┐
│      对比项       │      interface       │        type           │
├──────────────────┼──────────────────────┼───────────────────────┤
│  扩展方式        │ extends 继承          │ & 交叉类型             │
│  声明合并        │ ✅ 支持（同名自动合并）│ ❌ 不支持              │
│  描述联合类型    │ ❌ 不能               │ ✅ type A = B \| C    │
│  描述基本类型    │ ❌ 不能               │ ✅ type ID = string   │
│  描述元组        │ ❌ 不方便             │ ✅ type T = [string]  │
│  计算属性        │ ❌ 不能               │ ✅ 支持映射类型        │
└──────────────────┴──────────────────────┴───────────────────────┘
```

```typescript
// interface：支持声明合并（第三方库类型扩展常用）
interface Window {
  myCustomProp: string
}
// 编译时会自动与原有的 Window 合并

// type：支持联合类型、交叉类型、映射类型等高级操作
type Status = 'active' | 'inactive'
type ID = string | number
type Readonly<T> = { readonly [K in keyof T]: T[K] }
```

**选择建议**：定义对象结构用 `interface`，其他场景用 `type`。

### Q2：any、unknown、never 的区别和应用场景？

**答案：**

| 类型 | 含义 | 能接收任意值？ | 能赋给其他类型？ | 使用场景 |
|------|------|:---:|:---:|------|
| `any` | 放弃类型检查 | ✅ | ✅ | 临时绕过类型（不推荐长期使用） |
| `unknown` | 安全的顶级类型 | ✅ | ❌（需收窄） | API 返回值、第三方数据 |
| `never` | 不可能的值 | ❌ | ✅ | 穷尽检查、不可能的分支 |

```typescript
// unknown 必须收窄后才能使用
function process(data: unknown) {
  // data.toFixed()  // ❌
  if (typeof data === 'number') {
    data.toFixed()   // ✅
  }
}

// never 用于穷尽检查
type Action = 'create' | 'update' | 'delete'
function handle(action: Action) {
  switch (action) {
    case 'create': return 1
    case 'update': return 2
    case 'delete': return 3
    default:
      const _: never = action  // 如果新增了 Action 但忘了处理，这里会报错
  }
}
```

### Q3：什么是结构化类型系统？和名义类型有什么区别？

**答案：**

**结构化类型**（TypeScript）：只要两个类型的"形状"相同就兼容，不看名字。

**名义类型**（Java/C#）：即使结构相同，类名不同也不兼容。

```typescript
// TypeScript：结构化类型
type UserA = { name: string; age: number }
type UserB = { name: string; age: number }

const a: UserA = { name: '张三', age: 25 }
const b: UserB = a // ✅ 结构完全相同，兼容
```

```java
// Java：名义类型（类名必须匹配）
class UserA { String name; int age; }
class UserB { String name; int age; }

UserA a = new UserA();
UserB b = a; // ❌ 编译错误，类名不同
```

**影响**：在 TypeScript 中，只要对象包含了要求的属性，就可以赋值，不需要显式 `implements`。

### Q4：null 和 undefined 的区别？什么是 strictNullChecks？

**答案：**

- `undefined`：变量已声明但**未赋值**时的默认值，表示"尚未定义"
- `null`：开发者**主动赋值**，表示"这里应该有值但现在是空的"

`strictNullChecks`（推荐开启）的影响：
- **关闭**时：`null` 和 `undefined` 可以赋给任何类型（危险）
- **开启**时：必须显式声明 `T | null` 或 `T | undefined`（安全）

```typescript
// strictNullChecks: true
let name: string = null       // ❌
let name: string | null = null // ✅

// 安全的空值检查
function getLength(str: string | null): number {
  if (str === null) return 0  // ✅ 先排除 null
  return str.length           // TS 自动收窄为 string
}

// 可选链和空值合并
const user: { address?: { city: string } } = {}
user.address?.city            // undefined（不会报错）
user.address?.city ?? '未知'   // '未知'
```

### Q5：readonly 和 const 有什么区别？

**答案：**

```
┌──────────────┬──────────────────────────────┬──────────────────────────────┐
│    对比项     │          const               │         readonly             │
├──────────────┼──────────────────────────────┼──────────────────────────────┤
│  作用层面    │ 变量（变量不能重新赋值）        │ 属性（属性不能被修改）         │
│  所在位置    │ JavaScript 层面               │ TypeScript 类型层面           │
│  运行时效果  │ 有效（运行时也会阻止修改）      │ 无效（编译后消失，运行时不检查）│
│  深层保护    │ ❌ 不保护对象内部属性           │ ❌ 不保护嵌套属性（浅层）      │
└──────────────┴──────────────────────────────┴──────────────────────────────┘
```

```typescript
// const：变量本身不能重新赋值，但对象内部可以修改
const arr = [1, 2, 3]
// arr = [4, 5, 6]   // ❌ 不能重新赋值
arr.push(4)          // ✅ 但可以修改内容

// readonly：属性不能被修改（仅编译时）
interface Config {
  readonly host: string
  readonly ports: number[]
}

const config: Config = { host: 'localhost', ports: [3000] }
// config.host = '127.0.0.1'  // ❌ 编译时报错
config.ports.push(4000)       // ✅ 编译通过（readonly 是浅层的）

// 两者配合使用效果最佳
const config2: Readonly<Config> = Object.freeze({
  host: 'localhost',
  ports: [3000],
})
```
