## 一、什么是枚举

### 通俗解释

枚举（Enum）就是**给一组相关的常量起一个集合名字**。

> 通俗类比：想象你开了一家奶茶店，菜单上的杯子有三种规格——小杯、中杯、大杯。你可以用 `1`、`2`、`3` 来表示，但时间久了你看到 `2` 就要想一下："2 是中杯还是大杯来着？" 而用枚举 `CupSize.Small`、`CupSize.Medium`、`CupSize.Large`，一眼就能看懂，代码自己就能"说话"。

```
┌─────────────────────────────────────────────────────┐
│                    枚举 的本质                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│   把 "散落的魔法值"  ──→  "有名字、有组织" 的集合    │
│                                                     │
│   status = 1         ──→  Status.Active             │
│   status = 0         ──→  Status.Inactive           │
│   status = -1        ──→  Status.Banned             │
│                                                     │
│   可读性 ✅   可维护性 ✅   类型安全 ✅               │
└─────────────────────────────────────────────────────┘
```

### 为什么需要枚举 —— vs 魔法数字/字符串

**魔法数字（Magic Number）的问题：**

```typescript
// ❌ 魔法数字：维护噩梦
function updateUserStatus(userId: string, status: number) {
  // status = 1 是什么意思？激活？禁用？需要翻文档
  if (status === 1) {
    // ...
  } else if (status === 0) {
    // ...
  } else if (status === -1) {
    // ...
  }
}
```

**魔法字符串的问题：**

```typescript
// ❌ 魔法字符串：容易拼错，没有自动补全
function setTheme(theme: string) {
  if (theme === 'dark') {
    // ...
  } else if (theme === 'lite') {
    // 拼写错误！light 写成了 lite，运行时才报错
  }
}
```

**用枚举解决：**

```typescript
// ✅ 枚举：语义清晰、类型安全、IDE 自动补全
enum UserStatus {
  Active = 1,
  Inactive = 0,
  Banned = -1,
}

function updateUserStatus(userId: string, status: UserStatus) {
  if (status === UserStatus.Active) {
    // 一眼就知道是"激活"状态
  }
}

// 调用处也有自动补全和类型检查
updateUserStatus('u_001', UserStatus.Active)  // ✅
updateUserStatus('u_001', 999)                 // ❌ 类型错误
```

### 使用枚举 vs 常量对象的动机

```typescript
// 方案 A：常量对象
const STATUS = {
  Active: 1,
  Inactive: 0,
  Banned: -1,
} as const

// 方案 B：枚举
enum Status {
  Active = 1,
  Inactive = 0,
  Banned = -1,
}

// 枚举天然是类型 —— 可以直接当作类型使用
function setStatus(s: Status) { /* ... */ }

// 常量对象需要额外操作才能当类型用
function setStatus2(s: typeof STATUS[keyof typeof STATUS]) { /* ... */ }
```

> 后面第七节会做更详细的对比分析。

## 二、数字枚举（Numeric Enums）

### 基本语法和自动递增

数字枚举是最常见的枚举形式。如果没有显式赋值，从 `0` 开始自动递增。

```typescript
enum Direction {
  Up,      // 0
  Down,    // 1
  Left,    // 2
  Right,   // 3
}

console.log(Direction.Up)     // 0
console.log(Direction.Down)   // 1
console.log(Direction.Right)  // 3
```

**也可以手动指定起始值：**

```typescript
enum Direction {
  Up = 1,    // 1
  Down,      // 2（自动递增）
  Left,      // 3
  Right,     // 4
}
```

**甚至可以在中间打断自动递增：**

```typescript
enum Mixed {
  A = 1,    // 1
  B,        // 2（自动递增）
  C = 10,   // 10（手动赋值）
  D,        // 11（从上一个值继续递增）
  E,        // 12
}
```

```
自动递增规则：
┌─────────────────────────────────────────┐
│  有初始值？──── Yes ──→ 使用该值        │
│       │                                  │
│      No                                  │
│       │                                  │
│  上一个值 + 1                            │
│  (第一个默认为 0)                        │
└─────────────────────────────────────────┘
```

### 反向映射（Reverse Mapping）

数字枚举有一个独特特性：**反向映射** —— 既可以通过键取值，也可以通过值取键。

```typescript
enum Direction {
  Up,      // 0
  Down,    // 1
  Left,    // 2
  Right,   // 3
}

// 正向映射：键 → 值
console.log(Direction.Up)       // 0

// 反向映射：值 → 键
console.log(Direction[0])       // "Up"
console.log(Direction[Direction.Up])  // "Up"
```

> 这是数字枚举独有的特性，字符串枚举和 `const enum` 都没有。

### 编译产物分析

TypeScript 编译器为数字枚举生成的 JavaScript 代码：

**TypeScript 源码：**

```typescript
enum Direction {
  Up,
  Down,
  Left,
  Right,
}
```

**编译后的 JavaScript：**

```javascript
"use strict";
var Direction;
(function (Direction) {
  Direction[Direction["Up"] = 0] = "Up";       // 正向 + 反向映射
  Direction[Direction["Down"] = 1] = "Down";
  Direction[Direction["Left"] = 2] = "Left";
  Direction[Direction["Right"] = 3] = "Right";
})(Direction || (Direction = {}));
```

**逐行拆解编译产物：**

```
Direction["Up"] = 0        // 正向映射：Direction.Up = 0
Direction[0] = "Up"        // 反向映射：Direction[0] = "Up"

完整执行后 Direction 对象：
┌─────────────────────────────┐
│  Direction = {              │
│    "0": "Up",    ← 反向     │
│    "1": "Down",  ← 反向     │
│    "2": "Left",  ← 反向     │
│    "3": "Right", ← 反向     │
│    "Up": 0,      ← 正向     │
│    "Down": 1,    ← 正向     │
│    "Left": 2,    ← 正向     │
│    "Right": 3,   ← 正向     │
│  }                          │
└─────────────────────────────┘
```

> 这就是反向映射的原理：编译时生成了一个双向键值对对象。

## 三、字符串枚举（String Enums）

### 语法

字符串枚举的每个成员都必须显式赋值为字符串字面量，或引用另一个字符串枚举成员。**没有自动递增。**

```typescript
enum Theme {
  Dark = 'DARK',
  Light = 'LIGHT',
  System = 'SYSTEM',
}

console.log(Theme.Dark)   // "DARK"
console.log(Theme.Light)  // "LIGHT"
```

```typescript
// 在 Vue 组件中使用
<script setup lang="ts">
import { ref, watch } from 'vue'

enum Theme {
  Dark = 'DARK',
  Light = 'LIGHT',
  System = 'SYSTEM',
}

const currentTheme = ref<Theme>(Theme.System)

watch(currentTheme, (newTheme) => {
  document.documentElement.setAttribute('data-theme', newTheme)
})
</script>
```

### 没有反向映射

字符串枚举**不生成反向映射**，只能键 → 值，不能值 → 键。

```typescript
enum Theme {
  Dark = 'DARK',
  Light = 'LIGHT',
}

console.log(Theme.Dark)      // "DARK" ✅ 正向映射正常
console.log(Theme['DARK'])   // undefined ❌ 没有反向映射！
```

### 编译产物对比

**TypeScript 源码：**

```typescript
enum Theme {
  Dark = 'DARK',
  Light = 'LIGHT',
}
```

**编译后的 JavaScript：**

```javascript
"use strict";
var Theme;
(function (Theme) {
  Theme["Dark"] = "DARK";     // 只有正向映射
  Theme["Light"] = "LIGHT";
})(Theme || (Theme = {}));
```

**对比数字枚举的编译产物：**

```
数字枚举编译产物：               字符串枚举编译产物：
┌────────────────────────┐     ┌────────────────────────┐
│ Direction["Up"] = 0    │     │ Theme["Dark"] = "DARK" │
│ Direction[0] = "Up"    │     │ （没有反向映射那行）    │
│                        │     │                        │
│ 双向键值对对象          │     │ 单向键值对对象          │
└────────────────────────┘     └────────────────────────┘

为什么字符串枚举没有反向映射？
→ 因为字符串值本身就是有意义的，反向映射的必要性不大
→ 而且运行时对象属性的键只能是 string | symbol，值→键的反向映射
   在数字枚举中可以通过 number 键实现，在字符串枚举中则无法区分
```

## 四、异构枚举（Heterogeneous Enums）

### 混合数字和字符串

TypeScript 允许在同一个枚举中混合数字和字符串成员，这就是"异构枚举"。

```typescript
enum MixedEnum {
  No = 0,           // 数字
  Yes = 'YES',      // 字符串
}

console.log(MixedEnum.No)    // 0
console.log(MixedEnum.Yes)   // "YES"
```

### 不推荐使用，原因如下

> **TypeScript 官方文档明确不推荐使用异构枚举。**

```
为什么不推荐异构枚举？
┌───────────────────────────────────────────────────────────┐
│                                                           │
│  1. 语义混乱 —— 数字和字符串混在一起，含义不统一           │
│     枚举的初衷是表达"同一类"事物的不同状态，               │
│     混合类型破坏了这个语义                                 │
│                                                           │
│  2. 反向映射不一致 —— 数字成员有反向映射，字符串成员没有    │
│     同一个枚举中行为不一致，容易踩坑                       │
│                                                           │
│  3. 类型推断复杂 —— TypeScript 对异构枚举的类型推断       │
│     更加复杂，容易产生意外的类型                           │
│                                                           │
│  4. 可读性差 —— 阅读代码的人需要时刻注意                   │
│     "这个成员到底是数字还是字符串？"                       │
│                                                           │
└───────────────────────────────────────────────────────────┘
```

**正确做法 —— 拆分成多个枚举或使用其他方案：**

```typescript
// ❌ 异构枚举
enum Response {
  No = 0,
  Yes = 'YES',
}

// ✅ 拆分为两个语义明确的枚举
enum StatusCode {
  Success = 200,
  NotFound = 404,
  ServerError = 500,
}

enum ResponseType {
  Json = 'application/json',
  Html = 'text/html',
  Xml = 'application/xml',
}
```

## 五、const enum

### 内联编译机制

`const enum` 在编译时会被**直接内联替换**为对应的值，不会生成任何 JavaScript 运行时代码。

```typescript
// TypeScript 源码
const enum Color {
  Red = 0,
  Green = 1,
  Blue = 2,
}

let myColor = Color.Red
```

**编译后的 JavaScript：**

```javascript
"use strict";
let myColor = 0 /* Color.Red */;   // 直接替换成 0，没有生成任何枚举对象
```

### 与普通枚举的编译对比

```typescript
// 普通枚举
enum Direction {
  Up = 0,
  Down = 1,
}

// const enum
const enum Color {
  Red = 0,
  Green = 1,
}

const d = Direction.Up
const c = Color.Red
```

**编译结果对比：**

```javascript
// 普通枚举 —— 生成了运行时代码
var Direction;
(function (Direction) {
  Direction[Direction["Up"] = 0] = "Up";
  Direction[Direction["Down"] = 1] = "Down";
})(Direction || (Direction = {}));
const d = Direction.Up;           // 运行时通过对象访问

// const enum —— 直接内联，零运行时开销
const c = 0 /* Color.Red */;     // 编译时替换为常量
```

```
编译产物对比：
┌──────────────────────┬──────────────────────┐
│      普通枚举         │     const enum       │
├──────────────────────┼──────────────────────┤
│ 生成运行时对象        │ 不生成任何对象        │
│ 通过对象属性访问      │ 直接内联为常量值      │
│ 有反向映射（数字）    │ 无反向映射            │
│ 产物体积较大          │ 产物体积最小          │
│ 可以遍历              │ 不能遍历              │
│ 运行时存在            │ 编译时消除            │
└──────────────────────┴──────────────────────┘
```

### 使用场景和注意事项

**适合用 `const enum` 的场景：**

```typescript
// 性能敏感、不需要运行时枚举对象的场景
const enum HttpStatus {
  OK = 200,
  NotFound = 404,
  InternalError = 500,
}

function handleError(status: HttpStatus) {
  switch (status) {
    case HttpStatus.OK:
      return 'success'
    case HttpStatus.NotFound:
      return 'not found'
    default:
      return 'unknown'
  }
}
```

**`const enum` 的限制：**

```typescript
const enum Season {
  Spring = 'SPRING',
  Summer = 'SUMMER',
  Autumn = 'AUTUMN',
  Winter = 'WINTER',
}

// ❌ 不能遍历 const enum（编译后不存在这个对象）
// Object.values(Season)    // 编译错误

// ❌ 不能用作计算属性名
// const key = Season.Spring  // 这个值在编译时虽然能拿到，但有些场景受限

// ❌ 不能在动态字符串模板中使用
// `season_${Season.Spring}`  // 某些编译配置下可能出问题
```

**`isolatedModules` 下的限制：**

> 当 `tsconfig.json` 中启用 `"isolatedModules": true` 时（Vite 默认启用），**不能跨文件使用 `const enum`**。

```json
// tsconfig.json
{
  "compilerOptions": {
    "isolatedModules": true   // Vite 项目默认开启
  }
}
```

```typescript
// fileA.ts
export const enum Status {   // 在 isolatedModules 下跨文件导出
  Active = 'ACTIVE',
}

// fileB.ts
import { Status } from './fileA'
const s = Status.Active  // ❌ isolatedModules 下报错：
                         // Const enum member can only be accessed
                         // using a string literal
```

```
isolatedModules 的原因：
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  isolatedModules 要求每个文件可以独立编译，               │
│  不依赖其他文件的类型信息。                               │
│                                                          │
│  const enum 的内联需要知道具体的值，                     │
│  跨文件时就打破了"独立编译"的约束。                      │
│                                                          │
│  Vite 使用 esbuild 做转换，esbuild 按文件独立处理，      │
│  无法在编译 fileB.ts 时去读取 fileA.ts 中 const enum    │
│  的值来做内联替换。                                       │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

> **实际项目建议**：在 Vite 等使用 `isolatedModules` 的项目中，尽量避免使用 `const enum`，改用普通枚举或 `as const` 对象。

## 六、枚举成员类型

### 计算成员 vs 常量成员

枚举成员分为两类：**常量成员（constant member）** 和 **计算成员（computed member）**。

```
枚举成员分类：
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  常量成员（Constant Member）                                  │
│  ├── 没有初始值，自动递增                                     │
│  ├── 初始值是数字字面量                                       │
│  ├── 初始值是字符串字面量                                     │
│  ├── 初始值是对前一个常量成员的引用                            │
│  └── 初始值是括号包裹的常量表达式  (1 + 2)                    │
│                                                              │
│  计算成员（Computed Member）                                  │
│  └── 初始值是运行时表达式，如函数调用、对象属性引用            │
│     例如：Math.random()，someString.length                   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

```typescript
// 常量成员示例
const enum Flags {
  None = 0,               // 数字字面量 → 常量
  Read = 1 << 0,          // 位运算（编译时求值）→ 常量
  Write = 1 << 1,         // 常量
  ReadWrite = Read | Write, // 引用常量成员 → 常量
}

// 计算成员示例
enum RandomEnum {
  A = Math.random(),       // 计算成员 —— 运行时求值
  B = 'hello'.length,      // 计算成员
  C = 123,                 // 常量成员
  // D,                    // ❌ 计算成员之后的成员必须有初始值
}

// ⚠️ 计算成员后面不能放没有初始值的成员
// 因为自动递增依赖前一个常量值，而计算值在编译时不知道
```

```typescript
// 位运算枚举 —— 经典用法：权限组合
enum Permission {
  Read    = 1 << 0,  // 0001 = 1
  Write   = 1 << 1,  // 0010 = 2
  Execute = 1 << 2,  // 0100 = 4
  Admin   = Read | Write | Execute,  // 0111 = 7
}

// 组合权限
const userPermission: Permission = Permission.Read | Permission.Write  // 3

// 检查权限
function hasPermission(perm: Permission, flag: Permission): boolean {
  return (perm & flag) === flag
}

hasPermission(userPermission, Permission.Read)    // true
hasPermission(userPermission, Permission.Execute) // false
```

### 枚举作为类型使用

枚举本身就是一个类型，可以直接在类型注解中使用。

```typescript
enum UserRole {
  Admin = 'ADMIN',
  Editor = 'EDITOR',
  Viewer = 'VIEWER',
}

// 枚举作为类型
interface User {
  id: string
  name: string
  role: UserRole   // 使用枚举类型
}

const admin: User = {
  id: '001',
  name: '张三',
  role: UserRole.Admin,
}

// ❌ 类型错误：不能赋值为非枚举值
// const badUser: User = {
//   ...
//   role: 'SUPER_ADMIN',  // Error!
// }
```

```typescript
// 在 Vue 组件中使用枚举类型
<script setup lang="ts">
import { ref, computed } from 'vue'

enum LoadingState {
  Idle = 'IDLE',
  Loading = 'LOADING',
  Success = 'SUCCESS',
  Error = 'ERROR',
}

const state = ref<LoadingState>(LoadingState.Idle)
const isLoading = computed(() => state.value === LoadingState.Loading)

async function fetchData() {
  state.value = LoadingState.Loading
  try {
    // const res = await fetch('/api/data')
    state.value = LoadingState.Success
  } catch {
    state.value = LoadingState.Error
  }
}
</script>

<template>
  <div v-if="isLoading">加载中...</div>
  <div v-else-if="state === LoadingState.Error">出错了</div>
  <div v-else>内容</div>
</template>
```

## 七、枚举 vs 其他方案对比

### vs 联合字面量类型

```typescript
// 方案 A：枚举
enum Status {
  Active = 'ACTIVE',
  Inactive = 'INACTIVE',
}

// 方案 B：联合字面量类型
type StatusType = 'ACTIVE' | 'INACTIVE'

function setStatus(status: StatusType) { /* ... */ }
setStatus('ACTIVE')       // ✅ 有自动补全
setStatus('ACTIVEE')      // ❌ 类型错误
```

**区别：**

```
枚举 vs 联合字面量类型：
┌────────────────────┬──────────────────────┬──────────────────────┐
│                    │       枚举           │   联合字面量类型      │
├────────────────────┼──────────────────────┼──────────────────────┤
│ 定义方式           │ enum { A = 'a' }     │ type = 'a' | 'b'     │
│ 运行时存在         │ ✅ 是（普通枚举）     │ ❌ 否（纯类型）       │
│ 自动补全           │ ✅                    │ ✅                    │
│ 可遍历             │ ✅ Object.values()    │ ❌ 纯类型无法遍历     │
│ 反向映射           │ ✅（数字枚举）        │ ❌                    │
│ 需要引入           │ 需要 import           │ 不需要                │
│ 可读性             │ Status.Active ✅      │ 'ACTIVE' 🤔          │
│ 值和类型耦合       │ 是                    │ 否                    │
└────────────────────┴──────────────────────┴──────────────────────┘
```

### vs `as const` 对象

```typescript
// 方案 A：枚举
enum Theme {
  Dark = 'DARK',
  Light = 'LIGHT',
}

// 方案 B：as const 对象
const ThemeConst = {
  Dark: 'DARK',
  Light: 'LIGHT',
} as const

// 使用枚举类型
function setTheme(t: Theme) { /* ... */ }

// 使用 as const 对象的类型
type ThemeType = typeof ThemeConst[keyof typeof ThemeConst]
function setTheme2(t: ThemeType) { /* ... */ }

setTheme2('DARK')        // ✅
setTheme2(ThemeConst.Dark)  // ✅
```

### 大表格综合对比

## 八、最佳实践

### 什么时候用枚举

```typescript
// ✅ 场景 1：有限的、固定的状态集合
enum HttpStatusCode {
  OK = 200,
  Created = 201,
  BadRequest = 400,
  Unauthorized = 401,
  NotFound = 404,
  InternalServerError = 500,
}

// ✅ 场景 2：需要运行时遍历和类型安全同时存在
enum Weekday {
  Monday = 'MON',
  Tuesday = 'TUE',
  Wednesday = 'WED',
  Thursday = 'THU',
  Friday = 'FRI',
  Saturday = 'SAT',
  Sunday = 'SUN',
}

// 可以遍历
const workdays = Object.values(Weekday).filter(
  day => !['SAT', 'SUN'].includes(day)
)

// 也有类型安全
function getSchedule(day: Weekday): string[] {
  // ...
  return []
}
```

### 什么时候用替代方案

```typescript
// ✅ 用联合字面量：简单、值少、不需要运行时遍历
type Size = 'small' | 'medium' | 'large'
type Method = 'GET' | 'POST' | 'PUT' | 'DELETE'

// ✅ 用 as const 对象：需要遍历，同时需要 tree-shaking 友好
// （Vite 项目推荐这种方式）
const PAGE_SIZE = {
  Small: 10,
  Medium: 20,
  Large: 50,
} as const

type PageSize = typeof PAGE_SIZE[keyof typeof PAGE_SIZE]

// 可以遍历
const sizes = Object.values(PAGE_SIZE)  // [10, 20, 50]

// 也有类型安全
function setPageSize(size: PageSize) { /* ... */ }
```

### 推荐做法总结

```
推荐做法一览：
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  1. 优先使用字符串枚举（而非数字枚举）                       │
│     → 值在调试时更可读                                      │
│     → 避免反向映射带来的困惑                                │
│                                                            │
│  2. 在 Vite 项目中避免使用 const enum                       │
│     → isolatedModules 限制了跨文件使用                      │
│     → 改用普通枚举或 as const 对象                          │
│                                                            │
│  3. 始终给枚举成员显式赋值                                   │
│     → 避免依赖自动递增（删除/插入成员会导致值变化）          │
│                                                            │
│  4. 不要使用异构枚举                                        │
│     → 拆分为多个单一类型枚举                                │
│                                                            │
│  5. 枚举名用单数，不用复数                                  │
│     → enum Status ✅  enum Statuses ❌                     │
│                                                            │
│  6. 在 Vue/Vite 项目中，优先考虑 as const 对象              │
│     → 更好的 tree-shaking                                  │
│     → 无 isolatedModules 问题                              │
│     → 符合 JavaScript 社区主流趋势                         │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

```typescript
// Vite 项目中的推荐模式
<script setup lang="ts">
import { ref, computed } from 'vue'

// 推荐：as const 对象 + 类型提取
const ORDER_STATUS = {
  Pending: 'PENDING',
  Processing: 'PROCESSING',
  Shipped: 'SHIPPED',
  Delivered: 'DELIVERED',
  Cancelled: 'CANCELLED',
} as const

// 提取类型
type OrderStatus = typeof ORDER_STATUS[keyof typeof ORDER_STATUS]

// 使用
const status = ref<OrderStatus>(ORDER_STATUS.Pending)

  label,
  value,
}))

  const map: Record<OrderStatus, string> = {

    PROCESSING: '处理中',
    SHIPPED: '已发货',
    DELIVERED: '已送达',
    CANCELLED: '已取消',
  }
  return map[status.value]
})
</script>
```

## 九、面试常见问题

### Q1：TypeScript 中数字枚举的反向映射是怎么实现的？

**答：** 数字枚举在编译时会生成一个双向键值对的 JavaScript 对象。编译产物中通过 `obj[obj["Key"] = value] = "Key"` 这种自赋值模式同时设置正向映射（键→值）和反向映射（值→键）。因此 `Direction.Up` 得到数字值 `0`，`Direction[0]` 得到字符串键 `"Up"`。

```typescript
// TypeScript
enum Direction { Up, Down, Left, Right }

// 编译产物中的关键代码
Direction[Direction["Up"] = 0] = "Up"
// 等价于：
// Direction["Up"] = 0;     正向
// Direction[0] = "Up";     反向
```

> 注意：字符串枚举和 `const enum` 没有反向映射。

---

### Q2：`enum` 和 `const enum` 有什么区别？为什么 Vite 项目不建议用 `const enum`？

**答：** 核心区别在于编译产物。

- `enum` 编译后生成一个运行时对象，通过对象属性访问值。
- `const enum` 编译后不生成任何运行时代码，所有引用处被直接替换为常量值。

在 Vite 项目中不建议使用 `const enum` 的原因是：Vite 使用 esbuild 做文件级转换，`tsconfig.json` 默认启用 `isolatedModules: true`，要求每个文件能独立编译。但 `const enum` 的内联替换需要跨文件读取枚举定义，这违反了独立编译的约束。跨文件导入 `const enum` 会导致编译错误。

```typescript
// a.ts
export const enum Color { Red = 0, Blue = 1 }

// b.ts（isolatedModules 下报错）
import { Color } from './a'
const c = Color.Red  // Error: const enum member can only be accessed
                     // using a string literal
```

---

### Q3：TypeScript 中枚举和 `as const` 对象有什么区别？各有什么优劣？

**答：**

| 维度 | 枚举 | `as const` 对象 |
|---|---|---|
| 本质 | 既有类型又有运行时值 | 运行时对象 + 通过 `typeof` 提取类型 |
| 类型提取 | 直接使用 `EnumName` | 需要 `typeof Obj[keyof typeof Obj]` |
| 运行时 | 生成额外对象代码 | 原生 JS 对象，tree-shaking 友好 |
| 兼容性 | 普通枚举无限制，`const enum` 有 isolatedModules 问题 | 完全无限制 |
| 社区趋势 | TS 特有语法，非 JS 标准 | 更贴近 JavaScript 本身 |

**总结：** 在 Vite 等现代前端项目中，`as const` 对象是更推荐的做法，因为它避免了 `const enum` 的限制，产物更友好，同时仍然能获得完整的类型安全。

### Q4：下面的代码输出什么？为什么？

```typescript
enum E {
  A = 1,
  B,
  C = 'C',
  // D,  // 如果取消注释会怎样？
}

console.log(E.A)
console.log(E[1])
console.log(E.C)
console.log(E['C'])
```

**答：**

```
console.log(E.A)     // → 1        （正向映射：键→值）
console.log(E[1])    // → "A"      （反向映射：值→键，数字枚举特有）
console.log(E.C)     // → "C"      （正向映射：键→值）
console.log(E['C'])  // → undefined（字符串枚举没有反向映射）
```

如果取消 `D` 的注释，会**编译报错**。因为 `C = 'C'` 是一个字符串枚举成员（计算值），它后面的成员 `D` 没有显式初始值，TypeScript 无法对字符串做自动递增。只有当前一个成员是数字常量时才能自动递增。

---

### Q5：实际项目中你会如何选择枚举方案？

**答：** 根据场景选择：

1. **简单的类型约束，不需要运行时值** → 联合字面量类型 `type T = 'A' | 'B' | 'C'`
2. **需要运行时遍历 + 类型安全（Vite 项目）** → `as const` 对象 + `typeof` 提取类型
3. **需要运行时遍历 + 类型安全（非 isolatedModules 项目）** → 字符串枚举
4. **性能敏感 + 仅编译时使用** → `const enum`（确保不跨文件使用）
5. **权限位运算等特殊场景** → 数字枚举 + 位运算

**核心原则：** 能用 JavaScript 原生方案解决的，就不引入 TypeScript 独有语法。`as const` 对象在 Vite 项目中是最稳妥的选择。
