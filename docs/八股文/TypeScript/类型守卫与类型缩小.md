# 类型守卫与类型缩小

> 官方文档：[Narrowing](https://www.typescriptlang.org/docs/handbook/2/narrowing.html)

---

## 一、什么是类型缩小（Narrowing）

### 1.1 概念解释

**类型缩小（Type Narrowing）** 是 TypeScript 在编译阶段通过分析代码的控制流，将一个较宽泛的类型（如 `string | number`）逐步"收窄"为更具体类型（如 `string`）的过程。

通俗理解：就像 funnel（漏斗）一样，宽口进、窄口出 —— 变量的类型在经过条件判断后会变得更加精确。

```
         ┌──────────────────┐
         │   string | number │  ← 宽类型（联合类型）
         │    （漏斗入口）     │
         └────────┬─────────┘
                  │
         ┌────────▼─────────┐
         │   if (typeof x   │  ← 类型守卫（过滤条件）
         │   === 'string')  │
         └────────┬─────────┘
                  │
        ┌─────────┴──────────┐
        ▼                    ▼
  ┌───────────┐       ┌───────────┐
  │   string  │       │   number  │  ← 窄类型（收窄后）
  │ （确定！） │       │ （确定！） │
  └───────────┘       └───────────┘
```

### 1.2 为什么需要类型缩小？

```typescript
// 不收窄 → TS 不敢让你随便操作
function printValue(value: string | number) {
  // ❌ 报错：Property 'toUpperCase' does not exist on type 'number'
  console.log(value.toUpperCase())

  // ❌ 报错：Property 'toFixed' does not exist on type 'string'
  console.log(value.toFixed(2))
}

// 收窄后 → TS 知道类型是确定的，允许操作
function printValue(value: string | number) {
  if (typeof value === 'string') {
    // ✅ 这里 TS 知道 value 是 string
    console.log(value.toUpperCase())
  } else {
    // ✅ 这里 TS 知道 value 是 number
    console.log(value.toFixed(2))
  }
}
```

> **一句话总结**：类型缩小就是让 TypeScript 从"我不确定"变成"我现在确定了"的过程。

## 二、typeof 守卫

### 2.1 基本用法

`typeof` 是最常用的类型守卫方式，用于检测 JavaScript 的原始类型。

```typescript
function padLeft(value: string, padding: string | number): string {
  if (typeof padding === 'number') {
    // 此处 padding 被收窄为 number
    return ' '.repeat(padding) + value
  }
  // 此处 padding 被收窄为 string
  return padding + value
}
```

### 2.2 typeof 能检测的类型

| `typeof` 返回值    | 对应 TS 类型        | 说明           |
|-------------------|--------------------|---------------|
| `'string'`        | `string`           | 字符串          |
| `'number'`        | `number`           | 数字           |
| `'boolean'`       | `boolean`          | 布尔值         |
| `'symbol'`        | `symbol`           | Symbol        |
| `'bigint'`        | `bigint`           | BigInt        |
| `'undefined'`     | `undefined`        | 未定义         |
| `'object'`        | `object` / `null`  | **包含 null！** |
| `'function'`      | Function           | 函数           |

### 2.3 局限性

```typescript
// ⚠️ typeof null === 'object'（JavaScript 历史 Bug）
function check(value: string | null) {
  if (typeof value === 'object') {
    // ❌ 这里 value 是 null，不是 object！
    // typeof null === 'object' 是 JS 的一个著名历史遗留问题
    console.log(value) // null
  }
}

// ✅ 正确的 null 检查方式
function check(value: string | null) {
  if (value === null) {
    console.log('是 null')
  } else {
    // 这里 value 被收窄为 string
    console.log(value.toUpperCase()) // ✅
  }

  // 或者用 truthy 检查（null 是 falsy）
  if (value) {
    console.log(value.toUpperCase()) // ✅ 收窄为 string
  }
}
```

### 2.4 typeof 实战：表单值处理

```typescript
// Vue + TypeScript 场景：处理表单输入值
interface FormField {
  value: string | number | boolean
  type: 'text' | 'number' | 'checkbox'
}

function formatFieldValue(field: FormField): string {
  const { value, type } = field

  if (typeof value === 'string') {
    return value.trim()
  }

  if (typeof value === 'number') {
    return value.toFixed(2)
  }

  if (typeof value === 'boolean') {
    return value ? '是' : '否'
  }

  return String(value)
}
```

## 三、instanceof 守卫

### 3.1 基本用法

`instanceof` 用于检测某个值是否是某个类的实例，适用于所有使用 `new` 创建的对象。

```typescript
function logValue(x: Date | string) {
  if (x instanceof Date) {
    // x 被收窄为 Date
    console.log(x.toUTCString())
  } else {
    // x 被收窄为 string
    console.log(x.toUpperCase())
  }
}
```

### 3.2 工作原理

```
  instanceof 检查的是：对象的原型链上是否存在构造函数的 prototype

       x instanceof Date
            │
            ▼
  ┌───────────────────────┐
  │  x.__proto__.__proto__ │
  │      ...               │
  │  === Date.prototype ?  │
  └───────────┬───────────┘
              │
       ┌──────┴──────┐
       ▼             ▼
     true          false
   收窄为 Date    收窄为 string
```

### 3.3 适用场景

```typescript
// 场景一：错误处理 —— 区分不同类型的错误
async function fetchUserInfo(id: string): Promise<User> {
  try {
    const response = await fetch(`/api/users/${id}`)
    return await response.json()
  } catch (error: unknown) {
    if (error instanceof TypeError) {
      // 网络错误或请求格式错误
      console.error('请求失败:', error.message)
    } else if (error instanceof SyntaxError) {
      // JSON 解析失败
      console.error('响应解析失败:', error.message)
    } else if (error instanceof Error) {
      // 其他已知错误
      console.error('未知错误:', error.message)
    } else {
      // 完全不知道是什么
      console.error('发生了意料之外的情况')
    }
    throw error
  }
}

// 场景二：区分内置对象类型
function processData(data: ArrayBuffer | string | number[]) {
  if (data instanceof ArrayBuffer) {
    // data 是 ArrayBuffer
    const view = new DataView(data)
    return view.getInt32(0)
  } else if (data instanceof Array) {
    // data 是 number[]
    return data.reduce((sum, n) => sum + n, 0)
  } else {
    // data 是 string
    return data.length
  }
}
```

### 3.4 注意事项

```typescript
// ⚠️ instanceof 对跨 iframe / 跨 realm 失效
// 不同 iframe 中的 Array 构造函数不同
// const iframeArray = iframe.contentWindow.Array
// [] instanceof iframeArray  → false

// ⚠️ instanceof 不能用于原始类型
'hello' instanceof String   // false（原始类型不是实例对象）
new String('hello') instanceof String  // true（包装对象才是）
```

## 四、in 操作符守卫

### 4.1 基本用法

`in` 操作符检测对象上是否存在某个属性，TypeScript 会据此收窄类型。

```typescript
interface Fish {
  swim: () => void
  name: string
}

interface Bird {
  fly: () => void
  name: string
}

function move(animal: Fish | Bird) {
  if ('swim' in animal) {
    // animal 被收窄为 Fish
    animal.swim()
  } else {
    // animal 被收窄为 Bird
    animal.fly()
  }
}
```

### 4.2 区分不同接口

```typescript
// Vue 项目中常见的场景：区分不同类型的 API 响应
interface SuccessResponse<T> {
  code: number
  data: T
  message: string
}

interface ErrorResponse {
  code: number
  error: string
  message: string
}

type ApiResponse<T> = SuccessResponse<T> | ErrorResponse

function handleResponse<T>(response: ApiResponse<T>) {
  if ('data' in response) {
    // 收窄为 SuccessResponse<T>
    console.log('请求成功:', response.data)
    return response.data
  } else {
    // 收窄为 ErrorResponse
    console.error('请求失败:', response.error)
    throw new Error(response.error)
  }
}

// 另一个实战例子：区分不同类型的组件 Props
interface TextProps {
  type: 'text'
  content: string
  fontSize?: number
}

interface ImageProps {
  type: 'image'
  src: string
  alt: string
  width?: number
  height?: number
}

type MediaProps = TextProps | ImageProps

function renderMedia(props: MediaProps) {
  if ('src' in props) {
    // ImageProps
    return `<img src="${props.src}" alt="${props.alt}" />`
  } else {
    // TextProps
    return `<span style="font-size:${props.fontSize || 14}px">${props.content}</span>`
  }
}
```

### 4.3 注意事项

```typescript
// ⚠️ in 检测的是属性名是否存在（包括原型链），不区分 undefined 值
interface A {
  prop: string | undefined
}

interface B {
  prop: string
}

type Union = A | B

function test(obj: Union) {
  if ('prop' in obj) {
    // ⚠️ 两个接口都有 prop，所以这里不会收窄
    // in 只检测"属性是否存在"，不检测"属性值是什么"
  }
}
```

## 五、字面量类型守卫

### 5.1 switch / case 方式

通过检查字面量字段（如 `type`、`kind` 等判别属性）来收窄类型，这种模式也叫 **可辨识联合（Discriminated Union）**。

```typescript
interface Circle {
  kind: 'circle'
  radius: number
}

interface Rectangle {
  kind: 'rectangle'
  width: number
  height: number
}

interface Triangle {
  kind: 'triangle'
  base: number
  height: number
}

type Shape = Circle | Rectangle | Triangle

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':
      // shape 收窄为 Circle
      return Math.PI * shape.radius ** 2
    case 'rectangle':
      // shape 收窄为 Rectangle
      return shape.width * shape.height
    case 'triangle':
      // shape 收窄为 Triangle
      return (shape.base * shape.height) / 2
    default:
      // 确保所有情况都被处理（穷尽检查）
      const _exhaustiveCheck: never = shape
      return _exhaustiveCheck
  }
}
```

### 5.2 === 比较方式

```typescript
type ResponseStatus = 'loading' | 'success' | 'error'

interface LoadingState {
  status: 'loading'
}

interface SuccessState {
  status: 'success'
  data: string[]
}

interface ErrorState {
  status: 'error'
  message: string
}

type State = LoadingState | SuccessState | ErrorState

function renderState(state: State): string {
  if (state.status === 'loading') {
    // state 收窄为 LoadingState
    return '加载中...'
  }

  if (state.status === 'success') {
    // state 收窄为 SuccessState
    return `数据：${state.data.join(', ')}`
  }

  if (state.status === 'error') {
    // state 收窄为 ErrorState
    return `错误：${state.message}`
  }

  return ''
}
```

### 5.3 Vue 中的实战：通知组件

```typescript
// 定义通知类型（可辨识联合）
type NotificationType = 'info' | 'warning' | 'success' | 'error'

interface BaseNotification {
  id: string
  title: string
  timestamp: number
}

interface InfoNotification extends BaseNotification {
  type: 'info'
}

interface SuccessNotification extends BaseNotification {
  type: 'success'
  duration?: number
}

interface ErrorNotification extends BaseNotification {
  type: 'error'
  retryable: boolean        // 是否可重试
  retryAction?: () => void
}

type Notification = InfoNotification | SuccessNotification | ErrorNotification

// 根据 type 字段收窄，渲染不同样式
function getNotificationIcon(notification: Notification): string {
  switch (notification.type) {
    case 'info':
      return 'ℹ️'
    case 'success':
      return '✅'
    case 'error':
      // notification 被收窄为 ErrorNotification
      if (notification.retryable) {
        return '🔄'
      }
      return '❌'
    default:
      return '💬'
  }
}
```

## 六、自定义类型守卫（Type Predicate）

### 6.1 is 关键字

当内置的类型守卫不够用时，可以用 **类型谓词（Type Predicate）** 自定义守卫逻辑。

```typescript
// 语法格式
function isTypeName(value: any): value is TargetType {
  return /* 布尔表达式 */
}
```

```
  ┌───────────────────────────────────────┐
  │  function isFish(animal): animal is Fish │
  │                    │          │         │
  │                    │          └── 目标类型 │
  │                    └── 参数名            │
  │                                        │
  │  返回 true  → TS 认为参数是 Fish 类型     │
  │  返回 false → TS 认为参数不是 Fish 类型   │
  └───────────────────────────────────────┘
```

### 6.2 基本示例

```typescript
interface Fish {
  swim: () => void
  name: string
}

interface Bird {
  fly: () => void
  name: string
}

// 自定义类型守卫
function isFish(animal: Fish | Bird): animal is Fish {
  return (animal as Fish).swim !== undefined
}

// 使用
function move(animal: Fish | Bird) {
  if (isFish(animal)) {
    // ✅ animal 被收窄为 Fish
    animal.swim()
  } else {
    // ✅ animal 被收窄为 Bird
    animal.fly()
  }
}
```

### 6.3 实际应用场景

#### 场景一：过滤数组中的非空值

```typescript
// 经典用法：过滤 null / undefined
function isNonNull<T>(value: T | null | undefined): value is T {
  return value !== null && value !== undefined
}

const items: (string | null | undefined)[] = ['a', null, 'b', undefined, 'c']

// ❌ 不用类型守卫 —— TS 不知道过滤后的类型
const filtered1 = items.filter((item) => item !== null && item !== undefined)
// filtered1 的类型仍然是 (string | null | undefined)[]

// ✅ 使用类型守卫 —— TS 正确推断类型
const filtered2 = items.filter(isNonNull)
// filtered2 的类型是 string[] ✅
```

#### 场景二：验证 API 响应数据

```typescript
interface User {
  id: number
  name: string
  email: string
}

// 运行时检查 + 编译时收窄，一举两得
function isUser(data: unknown): data is User {
  return (
    typeof data === 'object' &&
    data !== null &&
    'id' in data &&
    typeof (data as User).id === 'number' &&
    'name' in data &&
    typeof (data as User).name === 'string' &&
    'email' in data &&
    typeof (data as User).email === 'string'
  )
}

// 在 Vue 项目中使用
async function loadUser(id: number): Promise<User> {
  const response = await fetch(`/api/users/${id}`)
  const data: unknown = await response.json()

  if (isUser(data)) {
    // ✅ data 被收窄为 User
    return data
  }

  throw new Error('Invalid user data from API')
}
```

#### 场景三：Axios 拦截器中校验响应

```typescript
interface ApiSuccessResponse<T> {
  code: 0
  data: T
  message: string
}

interface ApiErrorResponse {
  code: number
  message: string
  errors?: Record<string, string[]>
}

function isSuccessResponse<T>(
  response: ApiSuccessResponse<T> | ApiErrorResponse
): response is ApiSuccessResponse<T> {
  return response.code === 0 && 'data' in response
}

// 在请求封装中使用
function handleApiResult<T>(
  result: ApiSuccessResponse<T> | ApiErrorResponse
): T {
  if (isSuccessResponse(result)) {
    return result.data
  }

  // result 被收窄为 ApiErrorResponse
  const messages = Object.values(result.errors ?? {}).flat()
  throw new Error(messages.join('; ') || result.message)
}
```

### 6.4 Assertion Functions（断言函数）

与 `is` 类似，`asserts` 关键字表示"如果不满足条件就抛错"。

```typescript
// asserts 语法
function assertIsString(value: unknown): asserts value is string {
  if (typeof value !== 'string') {
    throw new TypeError(`Expected string, got ${typeof value}`)
  }
}

// 使用
function processInput(input: unknown) {
  assertIsString(input)
  // ✅ 断言通过后，input 被收窄为 string
  console.log(input.toUpperCase())
}

// asserts value is NonNullable —— 断言非空
function assertDefined<T>(
  value: T,
  message?: string
): asserts value is NonNullable<T> {
  if (value === null || value === undefined) {
    throw new Error(message ?? 'Value must be defined')
  }
}
```

## 七、可赋值性守卫（Assignment Narrowing）

### 7.1 变量赋值自动收窄

TypeScript 会根据赋给变量的值自动收窄其类型。

```typescript
let value: string | number | boolean

value = 'hello'         // TS 自动将 value 收窄为 string
value.toUpperCase()     // ✅ 允许

value = 42              // TS 自动将 value 收窄为 number
value.toFixed(2)        // ✅ 允许

value = true            // TS 自动将 value 收窄为 boolean
value.valueOf()         // ✅ 允许
```

### 7.2 赋值收窄的边界

```typescript
let value: string | number

// 赋值后自动收窄
value = 'hello'
value.toUpperCase() // ✅

// ⚠️ 但是在函数内部，赋值收窄可能被函数调用打断
// 因为函数可能改变变量的值
let x: string | number = Math.random() > 0.5 ? 'hello' : 42

if (typeof x === 'string') {
  // x 是 string ✅
  someFunction() // 如果 someFunction 修改了 x，TS 不一定能感知

  // TS 默认仍然认为 x 是 string（假设 x 不是 let 模块级变量被闭包捕获）
  console.log(x.toUpperCase()) // ✅
}
```

### 7.3 解构赋值中的收窄

```typescript
interface Point2D {
  x: number
  y: number
}

interface Point3D {
  x: number
  y: number
  z: number
}

function processPoint(point: Point2D | Point3D) {
  const { x, y, z } = point
  // ⚠️ 解构后，z 的类型是 number | undefined（不是收窄）
  // 因为 Point2D 没有 z 属性
}
```

## 八、控制流分析

### 8.1 TypeScript 如何跟踪代码路径

TypeScript 编译器会沿着所有可能的执行路径（控制流），逐步收窄变量的类型。

```
                    ┌─────────────┐
                    │  value:     │
                    │  string |   │
                    │  number |   │
                    │  boolean    │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │ typeof 检查   │
                    └──────┬──────┘
                           │
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
    ┌───────────┐   ┌───────────┐   ┌───────────┐
    │  string   │   │  number   │   │  boolean  │
    │ if 分支   │   │ else if   │   │ else 分支 │
    └─────┬─────┘   └─────┬─────┘   └─────┬─────┘
          │               │               │
          └───────────────┴───────────────┘
                          │
                   ┌──────▼──────┐
                   │  合并后回到   │
                   │  string |    │
                   │  number |    │
                   │  boolean     │
                   └─────────────┘
```

### 8.2 if / else 中的收窄

```typescript
function example(value: string | number | boolean) {
  // value: string | number | boolean

  if (typeof value === 'string') {
    // value: string ✅
    return value.toUpperCase()
  }

  // value: number | boolean（已排除 string）

  if (typeof value === 'number') {
    // value: number ✅
    return value.toFixed(2)
  }

  // value: boolean（已排除 string 和 number）
  return value ? '真' : '假'
}
```

### 8.3 switch 中的收窄

```typescript
type Status = 'pending' | 'active' | 'inactive' | 'deleted'

function getStatusLabel(status: Status): string {
  switch (status) {
    case 'pending':
      // status: 'pending'
      return '待审核'
    case 'active':
      // status: 'active'
      return '已激活'
    case 'inactive':
      // status: 'inactive'
      return '未激活'
    case 'deleted':
      // status: 'deleted'
      return '已删除'
  }
}
```

### 8.4 循环中的收窄

```typescript
// while 循环中的收窄
function processValues(values: (string | null)[]): string[] {
  const result: string[] = []

  for (const item of values) {
    if (item !== null) {
      // item 被收窄为 string
      result.push(item.toUpperCase())
    }
  }

  return result
}

// while 循环
function findFirstString(values: (string | number)[]): string | undefined {
  let index = 0
  while (index < values.length) {
    const item = values[index]
    if (typeof item === 'string') {
      // item 被收窄为 string
      return item.toUpperCase()
    }
    index++
  }
  return undefined
}
```

### 8.5 try / catch 中的收窄

```typescript
// catch 中的类型：默认为 unknown（TS 4.4+）
async function safeFetch(url: string): Promise<string> {
  try {
    const response = await fetch(url)
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`)
    }
    return await response.text()
  } catch (error) {
    // error 的类型是 unknown
    if (error instanceof Error) {
      // error 收窄为 Error
      console.error('请求失败:', error.message)
    } else if (typeof error === 'string') {
      // error 收窄为 string
      console.error('请求失败:', error)
    } else {
      console.error('未知错误')
    }
    throw error
  }
}
```

### 8.6 三元表达式中的收窄

```typescript
type Result = { success: true; data: string } | { success: false; error: string }

function getResultMessage(result: Result): string {
  // 三元表达式中也能收窄
  return result.success ? result.data : result.error
  //     result: success=true  ↑         ↑ result: success=false
  //     收窄为 { success: true; data }     收窄为 { success: false; error }
}
```

### 8.7 逻辑运算符中的收窄

```typescript
// && （与运算）收窄
function processValue(value?: string | null) {
  // value && ... ：如果 value 为 truthy 才执行
  value && console.log(value.toUpperCase()) // ✅ 收窄为 string

  // || （或运算）收窄
  const name = value || 'default'
  // name: string（因为 'default' 是 string）
}

// ?. 和 ?? 结合
function getLength(value?: string | null): number {
  // ?. 安全访问
  return value?.length ?? 0
}

// ! 非空断言（不是收窄，是断言 —— 谨慎使用）
function forceProcess(value?: string) {
  // ⚠️ 告诉 TS "我保证 value 不是 null/undefined"
  // 但运行时可能是 undefined，会报错
  console.log(value!.toUpperCase())
}
```

## 九、类型断言（as）vs 类型守卫

### 9.1 对比

| 特性            | 类型守卫（Type Guard）             | 类型断言（Type Assertion / `as`）    |
|----------------|----------------------------------|-------------------------------------|
| **运行时行为**    | 有实际的检查代码，安全              | 没有任何检查，直接"骗"编译器            |
| **编译时效果**    | TS 自动收窄类型                    | 手动指定类型                          |
| **安全性**       | 高（双重保障）                     | 低（可能隐藏运行时错误）               |
| **适用场景**     | 不确定类型时                       | 你比 TS 更了解类型时                  |
| **可维护性**     | 好                                | 差（代码变更后可能不再正确）             |
| **调试友好**     | 是                                | 否（出错时难以定位）                   |

### 9.2 代码对比

```typescript
interface Dog {
  bark(): void
  type: 'dog'
}

interface Cat {
  meow(): void
  type: 'cat'
}

type Pet = Dog | Cat

// ❌ 类型断言 —— 不安全
function makeSound_Unsafe(pet: Pet) {
  // 强制告诉 TS "这是 Dog"，但如果实际是 Cat，运行时 bark 不存在
  ;(pet as Dog).bark()
}

// ✅ 类型守卫 —— 安全
function makeSound_Safe(pet: Pet) {
  if (pet.type === 'dog') {
    // TS 自动收窄为 Dog
    pet.bark()
  } else {
    // TS 自动收窄为 Cat
    pet.meow()
  }
}
```

### 9.3 什么时候可以用 as？

```typescript
// 1. DOM 操作（TS 无法推断 DOM 元素的具体类型）
const input = document.getElementById('myInput') as HTMLInputElement
input.value = 'hello'

// 2. 你确信的类型（比如从后端获取的已知格式的数据）
const data = JSON.parse(response) as User[]

// 3. 测试代码中的 mock 数据
const mockUser = { name: 'test' } as User

// ⚠️ 但更好的做法是：用自定义类型守卫做运行时校验
// 只有在你 100% 确定时才用 as
```

### 9.4 as const —— 另一种断言

```typescript
// as const：将值收窄为字面量类型
const config = {
  method: 'GET',     // 类型是 string
  timeout: 5000,     // 类型是 number
} as const

// 等价于：
// const config: {
//   readonly method: 'GET'
//   readonly timeout: 5000
// }

config.method = 'POST' // ❌ Cannot assign to 'method' because it is a read-only property
```

## 十、面试常见问题

### Q1：typeof null 的结果是什么？如何正确判断 null？

```typescript
// typeof null === 'object'（JS 历史遗留 Bug）
// 正确判断方式：

// 方式一：严格相等
if (value === null) { /* ... */ }

// 方式二：truthy / falsy（如果不需要区分 null 和 undefined）
if (value) { /* value 不是 null/undefined/0/''/false/NaN */ }

// 方式三：== null（同时检查 null 和 undefined）
if (value == null) { /* value 是 null 或 undefined */ }
```

### Q2：什么是可辨识联合（Discriminated Union）？

可辨识联合是指联合类型中的每个成员都有一个共同的属性（通常叫 `type`、`kind`、`status` 等），但值不同，TypeScript 据此进行收窄。

```typescript
// 关键三要素：
// 1. 联合类型（type Shape = Circle | Rectangle | Triangle）
// 2. 共同的判别属性（kind）
// 3. 每个成员的判别属性值不同（'circle' | 'rectangle' | 'triangle'）

type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'rectangle'; width: number; height: number }
  | { kind: 'triangle'; base: number; height: number }

function area(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':    return Math.PI * shape.radius ** 2
    case 'rectangle': return shape.width * shape.height
    case 'triangle':  return (shape.base * shape.height) / 2
  }
}
```

### Q3：is 和 asserts 有什么区别？

```typescript
// is（类型谓词）—— 返回 boolean，不抛错
function isString(value: unknown): value is string {
  return typeof value === 'string'
}

if (isString(value)) {
  // value 被收窄为 string
}

// asserts（断言函数）—— 不返回值，失败时抛错
function assertString(value: unknown): asserts value is string {
  if (typeof value !== 'string') {
    throw new TypeError('Expected string')
  }
}

assertString(value)
// 断言通过后，value 被收窄为 string（不需要 if 包裹）

// 区别总结：
// is     → 配合 if 使用，返回 boolean，安全检查
// asserts → 直接调用，抛异常，用在"不满足就报错"的场景
```

### Q4：为什么优先使用类型守卫而不是类型断言（as）？

```
  类型守卫                          类型断言（as）
  ──────────                       ──────────────
  ✅ 运行时有实际检查               ❌ 运行时无任何检查
  ✅ 重构时编译器能报错              ❌ 重构后可能隐藏错误
  ✅ 代码自文档化                    ❌ 读者需要信任开发者的判断
  ✅ TypeScript 官方推荐            ⚠️ 仅在你比 TS 更了解类型时使用

  通俗类比：
  类型守卫 = 安检门（真的会检查你的包）
  类型断言 = 假工作证（只是"声称"你有权限，没人验证）
```

### Q5：never 类型在类型守卫中有什么用？

`never` 类型可用于 **穷尽检查（Exhaustiveness Check）**，确保所有联合成员都被处理。

```typescript
type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'rectangle'; width: number; height: number }

function area(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':    return Math.PI * shape.radius ** 2
    case 'rectangle': return shape.width * shape.height
    default:
      // 如果 Shape 新增了成员但这里没处理，
      // shape 的类型就不是 never，会产生编译错误
      const _exhaustiveCheck: never = shape
      return _exhaustiveCheck
  }
}

// 新增 triangle 后：
// type Shape = ... | { kind: 'triangle'; base: number; height: number }
// default 分支中 shape 的类型是 { kind: 'triangle'; ... }
// 不能赋值给 never → 编译报错，提醒你补全 case
```

### Q6：如何在运行时安全地校验外部数据（如 API 响应）？

```typescript
// 推荐方案：自定义类型守卫 + 运行时校验

interface User {
  id: number
  name: string
  email: string
}

function isUser(data: unknown): data is User {
  if (typeof data !== 'object' || data === null) return false
  const obj = data as Record<string, unknown>
  return (
    typeof obj.id === 'number' &&
    typeof obj.name === 'string' &&
    typeof obj.email === 'string'
  )
}
```

### Q7：如何收窄 unknown 类型？

```typescript
function processUnknown(value: unknown) {
  // 逐步收窄
  if (typeof value === 'string') {
    // string
    console.log(value.toUpperCase())
  } else if (typeof value === 'number') {
    // number
    console.log(value.toFixed(2))
  } else if (Array.isArray(value)) {
    // any[]
    console.log(value.length)
  } else if (typeof value === 'object' && value !== null) {
    // object
    console.log(Object.keys(value))
  } else {
    // 剩余：boolean | symbol | bigint | undefined | null | function
    console.log('其他类型')
  }
}
```

## 总结：类型守卫速查表

| 守卫方式                   | 语法                            | 适用场景                         |
|--------------------------|--------------------------------|--------------------------------|
| `typeof`                 | `typeof x === 'string'`        | 原始类型：string / number / boolean / symbol / bigint / undefined |
| `instanceof`             | `x instanceof Date`            | 类实例：Date / Error / Array / 自定义类    |
| `in`                     | `'swim' in animal`             | 检测对象是否有某个属性                   |
| 字面量比较                  | `x.kind === 'circle'`          | 可辨识联合（Discriminated Union）       |
| `switch / case`          | `switch (x.kind)`              | 多分支可辨识联合                     |
| Truthy 检查               | `if (x)`                       | 排除 null / undefined / false / 0 / ''  |
| 相等比较                   | `x === null` / `x !== undefined` | 精确排除 null 或 undefined          |
| 自定义守卫（`is`）          | `function isX(v): v is X`      | 复杂类型校验、数组过滤                  |
| 断言函数（`asserts`）       | `function assert(v): asserts v is X` | 不满足就抛错的场景                  |
| 赋值收窄                   | `x = 'hello'`                  | 赋值后自动收窄                      |
