# 联合类型与交叉类型

> TypeScript 类型系统中两个最核心的组合类型工具：**联合类型（Union Types）** 取"或"的关系，**交叉类型（Intersection Types）** 取"且"的关系。
>
> 官方文档：[Union Types](https://www.typescriptlang.org/docs/handbook/2/everyday-types.html#union-types) | [Intersection Types](https://www.typescriptlang.org/docs/handbook/2/objects.html#intersection-types) | [Discriminated Unions](https://www.typescriptlang.org/docs/handbook/2/narrowing.html#discriminated-unions)

---

## 一、联合类型（Union Types）

### 1.1 基本概念

联合类型使用 `|` 符号连接多个类型，表示变量可以是这些类型中的**任意一种**。

```
类比：一个 USB-C 接口 —— 可以插充电线、传输数据线、投屏线，
      接口不关心具体是哪根线，只要"形状匹配"就行。
      联合类型就是这个"万能接口"的类型声明。
```

```
         ┌─────────────────────────────────────────┐
         │           Union Type  A | B              │
         │                                         │
         │    取值可以是 A，也可以是 B               │
         │    ┌───┐          ┌───┐                  │
         │    │ A │    OR    │ B │                  │
         │    └───┘          └───┘                  │
         │                                         │
         │    只能访问 A 和 B 的【公共成员】          │
         └─────────────────────────────────────────┘
```

```typescript
// 基本用法：变量可以是多种类型之一
let id: string | number

id = 'abc-123'  // OK
id = 42         // OK
id = true       // Error: Type 'boolean' is not assignable to type 'string | number'
```

```typescript
// 函数参数使用联合类型
function formatValue(value: string | number): string {
  // 此处只能调用 string 和 number 的公共方法
  return value.toString() // OK — toString() 是公共方法
}

// 常见场景：函数接受多种输入
function padLeft(value: string, padding: string | number): string {
  if (typeof padding === 'number') {
    return ' '.repeat(padding) + value
  }
  return padding + value
}

padLeft('hello', 4)      // "    hello"
padLeft('hello', '>>>')  // ">>>hello"
```

**核心规则**：当 TypeScript 不确定变量到底是联合类型中的哪一个时，**只允许访问所有类型的公共成员**。

```typescript
interface Bird {
  fly(): void
  layEggs(): void
}

interface Fish {
  swim(): void
  layEggs(): void
}

type Pet = Bird | Fish

function handlePet(pet: Pet) {
  pet.layEggs()  // OK — layEggs 是 Bird 和 Fish 都有的
  pet.fly()      // Error — Fish 没有 fly，不能保证安全
}
```

### 1.2 字面量联合类型

联合类型不仅限于类型名，还可以把**具体的字面量值**作为类型成员，常用于限制变量取值范围。

```typescript
// 字符串字面量联合 — 类似枚举但更轻量
type Theme = 'light' | 'dark' | 'system'
type HttpMethod = 'GET' | 'POST' | 'PUT' | 'DELETE' | 'PATCH'

// 数字字面量联合
type HttpStatus = 200 | 301 | 400 | 404 | 500
type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6

// 布尔字面量联合（等价于 boolean，因为 boolean 就是 true | false 的语法糖）
type YesNo = true | false
```

```typescript
// 在 Vue 组件中约束 Props 的取值
// Child.vue
<script setup lang="ts">
interface Props {
  size: 'small' | 'medium' | 'large'
  type: 'primary' | 'success' | 'warning' | 'danger'
}

const props = defineProps<Props>()
</script>

<template>
  <button :class="[`btn-${props.size}`, `btn-${props.type}`]">
    <slot />
  </button>
</template>
```

```typescript
// Vite 配置中常见字面量联合
// vite.config.ts
import { defineConfig } from 'vite'

export default defineConfig({
  mode: 'development' as 'development' | 'production' | 'test',
  build: {
    target: 'es2015' as 'es2015' | 'es2020' | 'esnext',
    minify: 'terser' as 'terser' | 'esbuild' | false,
  },
})
```

**字面量联合 vs 枚举**：

| 特性 | 字面量联合类型 | enum |
|------|---------------|------|
| 编译产物 | 无额外代码 | 生成 JS 代码 |
| Tree-shaking | 完美支持 | 可能有问题 |
| 类型提示 | 编辑器自动补全 | 编辑器自动补全 |
| 运行时安全 | 无（纯类型层） | 有（运行时对象） |
| 反向映射 | 不支持 | `const enum` 不支持，普通 `enum` 支持 |
| 推荐程度 | 优先推荐 | 特定场景使用 |

### 1.3 类型缩小（收窄联合类型）

联合类型的变量在使用前，需要通过**类型守卫（Type Guard）** 将类型范围"缩小"到具体的一个类型。

```
    收窄前：string | number
          │
    ┌─────┴──────┐
    │ typeof 判断 │
    └─────┬──────┘
          │
   ┌──────┴──────┐
   ▼             ▼
 string        number
 (可以调        (可以调
  string 方法)   number 方法)
```

```typescript
function processValue(value: string | number | boolean): string {
  // typeof 收窄
  if (typeof value === 'string') {
    // 此分支 value: string
    return value.toUpperCase()
  }

  if (typeof value === 'number') {
    // 此分支 value: number
    return value.toFixed(2)
  }

  // 此分支 value: boolean（TypeScript 自动推断）
  return value ? '真' : '假'
}
```

```typescript
// switch 语句也能收窄
function getIcon(type: 'folder' | 'file' | 'link'): string {
  switch (type) {
    case 'folder':
      return '📁' // type: 'folder'
    case 'file':
      return '📄' // type: 'file'
    case 'link':
      return '🔗' // type: 'link'
  }
}
```

## 二、交叉类型（Intersection Types）

### 2.1 基本概念

交叉类型使用 `&` 符号连接多个类型，表示变量必须**同时满足所有类型**的要求。

```
类比：交叉类型就像多继承或 mixin。
      想象一个人同时持有"身份证"和"护照"，
      他必须同时满足两种证件的信息要求：
      姓名 + 身份证号 + 国籍 + 护照号。
```

```
         ┌─────────────────────────────────────────┐
         │       Intersection Type  A & B           │
         │                                         │
         │    必须同时满足 A 和 B 的所有要求          │
         │    ┌───┐          ┌───┐                  │
         │    │ A │   AND    │ B │                  │
         │    └─┬─┘          └─┬─┘                  │
         │      └──────┬───────┘                    │
         │             ▼                            │
         │     A 的所有属性 + B 的所有属性            │
         └─────────────────────────────────────────┘
```

```typescript
interface Timestamped {
  createdAt: Date
  updatedAt: Date
}

interface Identified {
  id: string
}

// 交叉类型：同时拥有 id、createdAt、updatedAt
type Entity = Identified & Timestamped

const user: Entity = {
  id: 'u-001',
  createdAt: new Date(),
  updatedAt: new Date(),
}
```

---

### 2.2 多个类型合并

交叉类型可以合并任意数量的类型，非常适合组合功能。

```typescript
// 基础能力接口
interface Loggable {
  log(message: string): void
}

interface Serializable {
  serialize(): string
}

interface Cloneable<T> {
  clone(): T
}

// 组合多个能力
type SmartObject = Loggable & Serializable & Cloneable<SmartObject>

// 实现组合后的类型
const smartObj: SmartObject = {
  log(message: string) {
    console.log(`[SmartObject] ${message}`)
  },
  serialize() {
    return JSON.stringify(this)
  },
  clone() {
    return { ...this }
  },
}
```

**在 Vue 组合式函数（Composable）中的实际应用**：

```typescript
// useLoading.ts — 提供 loading 状态
interface WithLoading {
  loading: boolean
  setLoading(val: boolean): void
}

function useLoading(): WithLoading {
  const loading = ref(false)
  const setLoading = (val: boolean) => { loading.value = val }
  return { loading, setLoading }
}

// useError.ts — 提供 error 状态
interface WithError {
  error: Ref<string | null>
  setError(msg: string): void
  clearError(): void
}

function useError(): WithError {
  const error = ref<string | null>(null)
  const setError = (msg: string) => { error.value = msg }
  const clearError = () => { error.value = null }
  return { error, setError, clearError }
}

// 组合多个 composable 的返回类型
type AsyncState<T> = {
  data: Ref<T | null>
  execute: () => Promise<void>
} & WithLoading & WithError

function useAsync<T>(fn: () => Promise<T>): AsyncState<T> {
  const data = ref<T | null>(null) as Ref<T | null>
  const { loading, setLoading } = useLoading()
  const { error, setError, clearError } = useError()

  const execute = async () => {
    setLoading(true)
    clearError()
    try {
      data.value = await fn()
    } catch (e) {
      setError((e as Error).message)
    } finally {
      setLoading(false)
    }
  }

  return { data, execute, loading, setLoading, error, setError, clearError }
}
```

### 2.3 属性冲突处理

当交叉类型的多个成员中存在**同名属性**时，TypeScript 的处理规则：

#### 同名属性类型兼容 — 自动合并

```typescript
interface A {
  name: string
  id: string | number
}

interface B {
  age: number
  id: string | number
}

type AB = A & B

// OK：id 同时满足两个接口中 string | number 的要求
const obj: AB = {
  name: '张三',
  age: 25,
  id: 'abc',
}
```

#### 同名属性类型不兼容 — 变成 never

```typescript
interface X {
  value: string
}

interface Y {
  value: number
}

type XY = X & Y
// 等价于：{ value: string & number }
// string & number = never —— 没有任何值能同时是 string 和 number

// 实际效果：这个类型无法被实例化
const impossible: XY = {
  value: '' as never, // 只能用 as never 强行绕过，但不安全
}
```

```
  属性冲突规则总结：
  ┌──────────────────┬──────────────────────────┬─────────────┐
  │   冲突类型        │   结果                    │   能否赋值    │
  ├──────────────────┼──────────────────────────┼─────────────┤
  │ 相同类型          │ 保持不变                  │ 可以         │
  │ 联合类型 & 具体类型│ 交叉后收窄                │ 可以（收窄后）│
  │ 互不兼容类型      │ 变为 never                │ 不可以       │
  └──────────────────┴──────────────────────────┴─────────────┘
```

```typescript
// 收窄示例：联合 & 具体 → 收窄为具体
interface WithId {
  id: string | number
}

interface StrictId {
  id: string
}

type Result = WithId & StrictId
// id 的类型: (string | number) & string = string
// 交叉后 id 被收窄为 string

const item: Result = {
  id: 'abc-123', // OK
  // id: 42,     // Error: number 不能赋给 string
}
```

## 三、可辨识联合（Discriminated Union）

### 3.1 概念和核心思想

可辨识联合是联合类型的高级模式，通过在每个类型分支中添加一个**相同的字面量字段（tag/discriminant）**，让 TypeScript 可以根据这个字段自动收窄类型。

```
类比：快递分拣中心
      每个包裹上都有一个"标签"（tag），写着 "加急"、"普通"、"国际"
      分拣员看到标签就知道走哪条流水线。
      TypeScript 编译器就是那个"分拣员"，tag 字段就是那个"标签"。
```

```
    ┌──────────────────────────────────────┐
    │       可辨识联合的三要素                │
    │                                      │
    │  1. 共同的 tag 字段（字面量类型不同）    │
    │  2. 用联合类型把各分支组合起来           │
    │  3. 用 switch / if 检查 tag 进行收窄   │
    └──────────────────────────────────────┘

         Shape = Circle | Rectangle | Triangle
                   │
           ┌───────┼───────┐
           ▼       ▼       ▼
        Circle  Rectangle  Triangle
        kind:   kind:      kind:
       "circle" "rect"    "triangle"
```

---

### 3.2 使用 tag 字段进行区分

```typescript
// 定义可辨识联合 — 每个分支都有 kind 字段，但字面量值不同
interface Circle {
  kind: 'circle'
  radius: number
}

interface Rectangle {
  kind: 'rectangle'
  width: number
  height: number
}

interface Triangle {
  kind: 'triangle'
  base: number
  height: number
}

type Shape = Circle | Rectangle | Triangle

// 使用 tag 字段进行区分
function getArea(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':
      // shape 被收窄为 Circle
      return Math.PI * shape.radius ** 2
    case 'rectangle':
      // shape 被收窄为 Rectangle
      return shape.width * shape.height
    case 'triangle':
      // shape 被收窄为 Triangle
      return (shape.base * shape.height) / 2
  }
}
```

### 3.3 结合 switch / if 进行类型守卫

**switch 语句（推荐）**：

```typescript
type NetworkState =
  | { status: 'idle' }
  | { status: 'loading'; startedAt: number }
  | { status: 'success'; data: string }
  | { status: 'error'; error: Error }

function handleNetworkState(state: NetworkState): string {
  switch (state.status) {
    case 'idle':
      return '等待请求...'
    case 'loading':
      return `请求中，开始时间: ${new Date(state.startedAt).toLocaleString()}`
    case 'success':
      return `请求成功: ${state.data}`
    case 'error':
      return `请求失败: ${state.error.message}`
  }
}
```

**if 语句**：

```typescript
type ApiResponse<T> =
  | { success: true; data: T }
  | { success: false; error: string }

function handleResponse<T>(response: ApiResponse<T>) {
  if (response.success) {
    // response 收窄为 { success: true; data: T }
    console.log('数据:', response.data)
  } else {
    // response 收窄为 { success: false; error: string }
    console.error('错误:', response.error)
  }
}
```

**穷尽检查（Exhaustive Check）**——确保所有分支都被处理：

```typescript
// 利用 never 实现穷尽检查
function assertNever(x: never): never {
  throw new Error(`未处理的分支: ${JSON.stringify(x)}`)
}

type Action =
  | { type: 'INCREMENT'; payload: number }
  | { type: 'DECREMENT'; payload: number }
  | { type: 'RESET' }

function reducer(state: number, action: Action): number {
  switch (action.type) {
    case 'INCREMENT':
      return state + action.payload
    case 'DECREMENT':
      return state - action.payload
    case 'RESET':
      return 0
    default:
      // 如果 Action 新增了分支但这里没处理，编译时会报错
      return assertNever(action)
  }
}
```

## 四、联合类型 vs 交叉类型对比

### 4.1 大表格对比

| 维度 | 联合类型 `A \| B` | 交叉类型 `A & B` |
|------|-------------------|-------------------|
| **符号** | `\|`（竖线） | `&`（与号） |
| **语义** | "或" —— 取值属于 A **或** B | "且" —— 取值同时满足 A **和** B |
| **类比** | 一个接口，多种实现 | 多继承 / mixin |
| **结果类型的成员** | 只能访问各类型的**公共成员** | 拥有所有类型的**全部成员** |
| **取值范围** | 并集（变大） | 交集（变小） |
| **赋值难度** | 容易（满足其中一个即可） | 困难（必须全部满足） |
| **类型收窄** | 需要收窄后才能安全使用 | 不需要收窄，直接使用全部成员 |
| **常见用途** | 多态参数、状态标记、可辨识联合 | mixin、功能组合、类型扩展 |
| **同属性冲突** | 取类型的联合（放宽） | 取类型的交叉（收窄，可能变 never） |
| **运行时表现** | 值是其中一种类型 | 值是组合后的单一对象 |

### 4.2 取交集 vs 取并集 — 直观图解

```
  联合类型 A | B（取并集，范围变大）：

        ┌──────────────────────┐
        │     A | B 的取值范围    │
        │  ┌────────┐          │
        │  │   A    │          │
        │  │    ┌───┼────┐     │
        │  │    │交集│    │     │
        │  └────┼───┘    │     │
        │       │   B    │     │
        │       └────────┘     │
        └──────────────────────┘
        属于 A 或属于 B 的值都可以赋值


  交叉类型 A & B（取交集，范围变小）：

                  ┌────────┐
                  │  A & B │
                  │ 的取值  │
                  │ 范围    │
                  └────────┘
        必须同时满足 A 和 B 的值才能赋值
```

### 4.3 各自适用场景

```typescript
// ──────────────────────────────
// 联合类型适用：一个东西有多种形态
// ──────────────────────────────

// 场景 1：函数参数多态
function greet(name: string | string[]): string {
  if (Array.isArray(name)) {
    return name.join(', ') + ' 你们好！'
  }
  return name + ' 你好！'
}

// 场景 2：可辨识联合 / 状态
type LoadingState =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'success'; data: unknown }
  | { status: 'error'; message: string }

// 场景 3：允许 null / undefined
function findUser(id: string): User | null {
  // ...
}

// ──────────────────────────────
// 交叉类型适用：一个东西组合多种能力
// ──────────────────────────────

// 场景 1：mixin 组合
type Timestamped = { createdAt: Date; updatedAt: Date }
type SoftDelete = { deletedAt: Date | null; deleted: boolean }
type BaseEntity = { id: string } & Timestamped & SoftDelete

// 场景 2：Composable 返回类型组合
type UseFetchReturn<T> = {
  data: Ref<T | null>
  error: Ref<Error | null>
} & UseFetchOptions

// 场景 3：高阶组件 / 增强功能
type WithPagination = {
  page: number
  pageSize: number
  total: number
}
type WithSearch = {
  keyword: string
  onSearch: (keyword: string) => void
}
type DataTableProps = WithPagination & WithSearch & {
  columns: Column[]
  data: Row[]
}
```

## 五、类型收窄技巧

### 5.1 typeof

`typeof` 适合判断 JavaScript 原始类型。

```typescript
function print(value: string | number | boolean | bigint) {
  if (typeof value === 'string') {
    console.log(value.toUpperCase()) // value: string
  } else if (typeof value === 'number') {
    console.log(value.toFixed(2)) // value: number
  } else if (typeof value === 'boolean') {
    console.log(value ? 'TRUE' : 'FALSE') // value: boolean
  } else {
    console.log(value.toString()) // value: bigint
  }
}
```

typeof 能识别的类型：`'string'`、`'number'`、`'bigint'`、`'boolean'`、`'symbol'`、`'undefined'`、`'object'`、`'function'`

> 注意：`typeof null === 'object'`（JavaScript 历史遗留 bug），typeof 无法区分 `object` 的子类型。

### 5.4 字面量类型守卫

直接比较字面量值即可收窄，这是可辨识联合的核心。

```typescript
    console.log('未找到:', res.message) // res.message 存在

  }
}
```

### 5.5 自定义类型谓词（Type Predicate）

当内置的类型收窄不够用时，可以用 `parameterName is Type` 语法定义自定义类型守卫。

```typescript
// 基本语法
function isString(value: unknown): value is string {
  return typeof value === 'string'
}

function isNumber(value: unknown): value is number {
  return typeof value === 'number'
}

// 使用
function processArray(arr: (string | number)[]) {
  const strings = arr.filter(isString)   // strings: string[]
  const numbers = arr.filter(isNumber)   // numbers: number[]
}
```

```typescript
// 实际应用：API 响应类型验证
interface User {
  id: string
  name: string
  email: string
}

function isUser(data: unknown): data is User {
  return (
    typeof data === 'object'
    && data !== null
    && 'id' in data
    && 'name' in data
    && 'email' in data
    && typeof (data as User).id === 'string'
    && typeof (data as User).name === 'string'
    && typeof (data as User).email === 'string'
  )
}

// 在 async 函数中使用
async function fetchUser(id: string): Promise<User> {
  const response = await fetch(`/api/users/${id}`)
  const data: unknown = await response.json()

  if (isUser(data)) {
    return data // data 已被收窄为 User
  }

  throw new Error('Invalid user data')
}
```

### 6.1 表单状态管理

```typescript
// 表单的多种状态
type FormState<T> =
  | { status: 'idle' }                                    // 初始状态
  | { status: 'editing'; values: T; dirty: boolean }      // 编辑中
  | { status: 'submitting'; values: T }                   // 提交中
  | { status: 'success'; values: T; submittedAt: Date }   // 成功
  | { status: 'error'; values: T; error: string }         // 失败

interface LoginForm {
  username: string
  password: string
}

// Vue 组合式 API 使用
import { ref, computed, type Ref } from 'vue'

function useForm<T>(initialValues: T) {
  const state: Ref<FormState<T>> = ref({ status: 'idle' })

  const isSubmitting = computed(() => state.value.status === 'submitting')
  const canSubmit = computed(() => {
    const s = state.value
    return s.status === 'editing' && s.dirty
  })

  function startEditing() {
    state.value = {
      status: 'editing',
      values: { ...initialValues },
      dirty: false,
    }
  }

  function updateValues(values: T) {
    if (state.value.status === 'editing') {
      state.value = {
        status: 'editing',
        values,
        dirty: true,
      }
    }
  }

  async function submit(fn: (values: T) => Promise<void>) {
    if (state.value.status !== 'editing') return

    const values = state.value.values
    state.value = { status: 'submitting', values }

    try {
      await fn(values)
      state.value = { status: 'success', values, submittedAt: new Date() }
    } catch (e) {
      state.value = {
        status: 'error',
        values,
        error: (e as Error).message,
      }
    }
  }

  return { state, isSubmitting, canSubmit, startEditing, updateValues, submit }
}
```

### 6.2 API 响应处理

```typescript
// 通用 API 响应类型
type ApiResult<T> =
  | { ok: true; data: T; message?: string }
  | { ok: false; error: string; code: number }

// 封装请求函数
async function api<T>(
  url: string,
  options?: RequestInit
): Promise<ApiResult<T>> {
  try {
    const response = await fetch(url, {
      headers: { 'Content-Type': 'application/json' },
      ...options,
    })

    if (!response.ok) {
      return {
        ok: false,
        error: `HTTP Error: ${response.statusText}`,
        code: response.status,
      }
    }

    const data: T = await response.json()
    return { ok: true, data }
  } catch (e) {
    return {
      ok: false,
      error: (e as Error).message,
      code: 0,
    }
  }
}

// 在 Vue 组件中使用
async function loadUserInfo() {
  const result = await api<{ name: string; age: number }>('/api/user')

  if (result.ok) {
    // result.data 可以安全访问
    console.log(`用户: ${result.data.name}, 年龄: ${result.data.age}`)
  } else {
    // result.error 可以安全访问
    console.error(`请求失败 [${result.code}]: ${result.error}`)
  }
}
```

## 七、面试常见问题

### Q1：联合类型和交叉类型的区别是什么？

**答**：
- 联合类型 `A | B` 表示"或"的关系，值只需要满足其中一个类型。只能访问公共成员，使用时需要类型收窄。取值范围是并集（变大）。
- 交叉类型 `A & B` 表示"且"的关系，值必须同时满足所有类型。可以直接访问所有成员，无需收窄。取值范围是交集（变小）。

---

### Q2：什么是可辨识联合（Discriminated Union）？

**答**：可辨识联合是一种设计模式，在每个类型分支上添加一个**相同名称但不同字面量值的字段**（tag/discriminant），TypeScript 可以通过检查这个字段自动进行类型收窄。三要素：
1. 有一个共同的 tag 字段
2. 各分支的 tag 使用不同的字面量类型
3. 通过 switch/if 检查 tag 实现类型守卫

---

### Q3：交叉类型中属性冲突会怎样？

**答**：
- 同名属性类型相同或兼容：保持类型不变或交叉收窄
- 同名属性类型不兼容（如 `string & number`）：该属性类型变为 `never`，无法赋值

---

### Q4：TypeScript 有哪些类型收窄的方式？

**答**：
1. `typeof` —— 判断原始类型
2. `instanceof` —— 判断类实例
3. `in` —— 判断对象是否有某属性
4. 字面量类型守卫（`===` 比较）
5. 自定义类型谓词（`x is Type`）
6. `Array.isArray()` —— 判断数组
7. 赋值收窄
8. 控制流分析（if/else/switch）

---

### Q5：如何确保 switch 语句处理了联合类型的所有分支？

**答**：使用**穷尽检查（Exhaustive Check）**。定义一个 `assertNever` 函数，在 switch 的 default 分支调用它。如果后续新增了联合类型的成员但没有在 switch 中处理，TypeScript 会在编译时报错：

```typescript
function assertNever(x: never): never {
  throw new Error(`Unhandled case: ${x}`)
}

type Status = 'active' | 'inactive' | 'pending'

function handleStatus(s: Status) {
  switch (s) {
    case 'active': return '激活'
    case 'inactive': return '未激活'
    case 'pending': return '待审核'
    default: return assertNever(s) // 如果新增了成员这里会报错
  }
}
```

---

### Q6：联合类型为什么只能访问公共成员？

**答**：因为 TypeScript 是静态类型系统，在不进行类型收窄的情况下，编译器不知道运行时变量具体是联合类型中的哪一个。如果允许访问非公共成员，当运行时恰好是不包含该成员的类型时就会报错。所以 TypeScript 保守地只允许访问所有类型都有的公共成员，保证类型安全。

---

### Q7：字面量联合类型和枚举（enum）怎么选？

**答**：
- **优先用字面量联合类型**：编译产物更少（零运行时开销）、Tree-shaking 友好、类型推断更自然
- **用 enum 的场景**：需要运行时对象（如反向映射）、需要从值集合中迭代
- 如果两者都不需要运行时，可以用 `as const` 对象 + `typeof` 模拟枚举：

```typescript
// 推荐方案：as const 对象模拟枚举
const STATUS = {
  ACTIVE: 'active',
  INACTIVE: 'inactive',
  PENDING: 'pending',
} as const

type Status = (typeof STATUS)[keyof typeof STATUS]
// 等价于 'active' | 'inactive' | 'pending'
```

---

### Q8：如何在运行时验证联合类型的值？

**答**：联合类型只在编译时存在，运行时需要手动验证。推荐方案：
1. **自定义类型谓词**（`is` 函数）—— 简单场景
2. **zod 等运行时验证库** —— 复杂场景，可以自动推导类型
3. **JSON Schema** —— 需要前后端共享验证规则

---

### Q9：交叉类型和 extends 继承有什么区别？

**答**：

| 维度 | 交叉类型 `A & B` | `interface C extends A, B` |
|------|-------------------|---------------------------|
| 语法 | 类型别名 | 接口声明 |
| 冲突处理 | 同名不同类型属性变 `never` | 直接报错 |
| 灵活性 | 可以交叉任何类型 | 只能 extends 接口或对象类型 |
| 可读性 | 内联使用 | 命名引用，更清晰 |
| 声明合并 | 不支持 | interface 支持声明合并 |

优先使用 `extends` 获得更好的错误提示，需要灵活组合时用交叉类型。
