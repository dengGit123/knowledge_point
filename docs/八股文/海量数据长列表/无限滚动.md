# 无限滚动（Infinite Scroll）

## 一、什么是无限滚动？

无限滚动是一种**数据按需加载**的交互模式——用户向下滚动列表时，当接近底部（或触顶）时自动发起请求加载更多数据，并将新数据追加到已有列表中，用户感知上列表是"无限"的。

> 核心思想：**"不到底不加载数据，滚动到底再请求下一页"**

### 无限滚动 vs 虚拟列表

| 维度 | 无限滚动 | 虚拟列表 |
|------|---------|---------|
| **核心目标** | 减少**网络请求和数据传输** | 减少 **DOM 节点数量** |
| **数据量** | 总量未知（服务端分页返回） | 总量已知（全量数据在内存） |
| **DOM 数量** | 随滚动不断增长 | 始终恒定 |
| **实现重心** | 触发加载的时机 + 分页管理 | 可视区域计算 + 渲染优化 |
| **典型场景** | 社交动态、商品瀑布流、新闻列表 | 本地大数据表格、聊天记录 |

> 两者常常**组合使用**：无限滚动负责按需加载数据到内存，虚拟列表负责高效渲染内存中的数据。

## 二、核心原理

### 2.1 工作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      无限滚动工作流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ① 页面初始化，加载第一页数据（如 page=1, pageSize=20）           │
│     ↓                                                           │
│  ② 将数据渲染到列表 DOM                                          │
│     ↓                                                           │
│  ③ 用户向下滚动...                                               │
│     ↓                                                           │
│  ④ 检测：滚动位置是否接近底部？                                    │
│     ├── 否 → 什么都不做，继续监听                                 │
│     └── 是 → 判断是否正在加载 / 是否已全部加载                     │
│              ├── 正在加载 → 跳过（防重复请求）                     │
│              ├── 已全部加载 → 显示"没有更多了"                     │
│              └── 可以加载 → 发起请求加载下一页                     │
│     ↓                                                           │
│  ⑤ 显示加载状态（loading spinner）                               │
│     ↓                                                           │
│  ⑥ 请求返回，将新数据追加到列表                                    │
│     ↓                                                           │
│  ⑦ 隐藏 loading，回到步骤 ③                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 触发加载的时机

"接近底部"的判定是无限滚动的关键，有三种常见方案：

```
┌──────────────────────── 容器（可视区域） ────────────────────────┐
│                                                                 │
│  已渲染的列表项                                                  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  item 1                                                   │  │
│  │  item 2                                                   │  │
│  │  item 3                                                   │  │
│  │  ...                                                      │  │
│  │  item N-2                                                 │  │
│  │  item N-1                                                 │  │
│  │  item N                                                   │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ─ ─ ─ ─ ─ ─ ─ ─ 容器底部边界 ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  🔄 loading spinner（还在可视区域内）                       │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ─ ─ ─ ─ ─ ─ ─ ─ 距离底部 threshold（如 200px） ─ ─ ─ ─ ─ ─ ─  │
│                                                                 │
│  当滚动位置 >= 内容高度 - 容器高度 - threshold 时，触发加载        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**触发公式：**

```javascript
// 关键判别公式
const distanceToBottom = scrollHeight - scrollTop - clientHeight

if (distanceToBottom <= threshold) {
  loadMore()
}
```

| 变量 | 含义 |
|------|------|
| `scrollHeight` | 内容总高度（包括溢出不可见的部分） |
| `scrollTop` | 当前滚动偏移量 |
| `clientHeight` | 容器可视区域高度 |
| `threshold` | 触发阈值（如 200px） |

### 2.3 触发方案对比

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **scroll 事件** | 监听 scroll，计算距离底部的差值 | 兼容性好，控制精确 | 高频触发需节流 |
| **IntersectionObserver** | 底部哨兵元素进入视口时触发 | 性能最优，无节流负担 | IE 不支持 |
| **scroll 事件 + requestAnimationFrame** | scroll 回调中用 rAF 节流 | 平衡性能与兼容性 | 代码略复杂 |

### 3.3 Vue 3 Composition API 实现（composable 模式）

**核心 composable：`useInfiniteScroll.js`**

```javascript
// composables/useInfiniteScroll.js
import { ref, computed, onMounted, onUnmounted, watch, nextTick } from 'vue'

/**
 * 无限滚动 composable
 * @param {Function} fetchFn - 数据获取函数 (page, pageSize) => Promise<{ data, total?, hasMore }>
 * @param {Object} options - 配置项
 */
export function useInfiniteScroll(fetchFn, options = {}) {
  const {
    pageSize = 20,
    threshold = 200,            // scroll 方案阈值
    rootMargin = '0px 0px 300px 0px',  // IntersectionObserver 提前量
    immediate = true,           // 是否立即加载第一页
  } = options

  // ==================== 状态 ====================
  const data = ref([])            // 已加载的全部数据
  const page = ref(0)             // 当前页码
  const loading = ref(false)      // 是否正在加载
  const error = ref(null)         // 错误信息
  const hasMore = ref(true)       // 是否还有更多
  const containerRef = ref(null)  // 滚动容器 ref

  // ==================== 计算属性 ====================
  const isEmpty = computed(() => data.value.length === 0 && !loading.value)
  const totalLoaded = computed(() => data.value.length)

  // ==================== 加载更多 ====================
  async function loadMore() {
    if (loading.value || !hasMore.value) return

    loading.value = true
    error.value = null
    page.value++

    try {
      const result = await fetchFn(page.value, pageSize)

      // 追加数据
      data.value = [...data.value, ...result.data]
      hasMore.value = result.hasMore ?? (data.value.length < (result.total ?? Infinity))
    } catch (err) {
      error.value = err
      page.value--  // 回退页码，允许重试
    } finally {
      loading.value = false
    }
  }

  // ==================== 重置 ====================
  function reset() {
    data.value = []
    page.value = 0
    loading.value = false
    error.value = null
    hasMore.value = true
  }

  // ==================== 刷新（重置并重新加载） ====================
  async function refresh() {
    reset()
    await loadMore()
  }

  return {
    data,
    page,
    loading,
    error,
    hasMore,
    isEmpty,
    totalLoaded,
    containerRef,
    loadMore,
    reset,
    refresh,
  }
}
```

**IntersectionObserver 指令：`vInfiniteScroll.js`**

```javascript
// directives/vInfiniteScroll.js

/**
 * 自定义指令：v-infinite-scroll
 *
 * 用法：
 *   <div v-infinite-scroll="loadMore" :infinite-scroll-disabled="loading">
 *     ...
 *   </div>
 */
export const vInfiniteScroll = {
  mounted(el, binding) {
    const callback = binding.value
    const disabled = binding.modifiers.disabled

    // 默认 rootMargin，提前 300px 触发
    const rootMargin = el.getAttribute('infinite-scroll-distance')
      ? `0px 0px ${el.getAttribute('infinite-scroll-distance')}px 0px`
      : '0px 0px 300px 0px'

    // 创建哨兵
    const sentinel = document.createElement('div')
    sentinel.setAttribute('data-infinite-sentinel', '')
    sentinel.style.height = '1px'
    el.appendChild(sentinel)

    const observer = new IntersectionObserver(
      (entries) => {
        if (entries[0].isIntersecting) {
          const isDisabled = el.getAttribute('infinite-scroll-disabled') === 'true'
          if (!isDisabled) {
            callback()
          }
        }
      },
      { root: el, rootMargin, threshold: 0 }
    )

    observer.observe(sentinel)

    // 存储引用，方便卸载时清理
    el._infiniteObserver = observer
    el._infiniteSentinel = sentinel
  },

  unmounted(el) {
    if (el._infiniteObserver) {
      el._infiniteObserver.disconnect()
      delete el._infiniteObserver
      delete el._infiniteSentinel
    }
  },
}
```

### 3.4 全页面滚动 + 无限加载

最常见的场景是**整个页面（window）作为滚动容器**：

```javascript
// composable: useWindowInfiniteScroll.js
import { ref, onMounted, onUnmounted } from 'vue'

export function useWindowInfiniteScroll(loadMore, options = {}) {
  const { threshold = 300 } = options
  const loading = ref(false)
  const hasMore = ref(true)

  let ticking = false

  function handleScroll() {
    if (ticking) return

    requestAnimationFrame(() => {
      const { scrollTop, scrollHeight, clientHeight } =
        document.documentElement

      const distanceToBottom = scrollHeight - scrollTop - clientHeight

      if (distanceToBottom <= threshold && !loading.value && hasMore.value) {
        loading.value = true
        loadMore().finally(() => {
          loading.value = false
        })
      }

      ticking = false
    })

    ticking = true
  }

  onMounted(() => window.addEventListener('scroll', handleScroll))
  onUnmounted(() => window.removeEventListener('scroll', handleScroll))

  return { loading, hasMore }
}
```

**使用示例：**

## 四、关键技术细节

### 4.1 防抖 vs 节流 vs rAF

| 方案 | 原理 | 适用场景 |
|------|------|---------|
| **debounce** | 停止滚动 N ms 后才触发 | ❌ 不适合——用户持续滚动时不会触发加载 |
| **throttle** | 每 N ms 最多触发一次 | ✅ 可用，但需选好间隔 |
| **requestAnimationFrame** | 与浏览器刷新率同步（~16ms） | ✅ 最佳——自动对齐帧率，无需手动设间隔 |

```javascript
// ✅ 推荐：rAF + flag（最简洁高效）
let ticking = false

container.addEventListener('scroll', () => {
  if (ticking) return
  ticking = true
  requestAnimationFrame(() => {
    checkAndLoad()
    ticking = false
  })
})

// ✅ 也可：throttle（更容易控制频率）
import { throttle } from 'lodash-es'

container.addEventListener('scroll', throttle(() => {
  checkAndLoad()
}, 200))
```

### 4.2 分页策略

| 策略 | 参数格式 | 特点 |
|------|---------|------|
| **页码分页** | `page=1&pageSize=20` | 最常见，后端实现简单 |
| **游标分页** | `after=abc123&limit=20` | 性能最优，适合大数据量 |
| **偏移分页** | `offset=40&limit=20` | 灵活但大数据量偏移慢 |

```javascript
// 页码分页
async function fetchByPage(page, pageSize) {
  return fetch(`/api/items?page=${page}&pageSize=${pageSize}`)
}

// 游标分页（推荐：性能更好，数据一致性更强）
let cursor = null
async function fetchByCursor(limit) {
  const url = cursor
    ? `/api/items?limit=${limit}&after=${cursor}`
    : `/api/items?limit=${limit}`
  const res = await fetch(url)
  const json = await res.json()
  cursor = json.nextCursor  // 后端返回下一页的游标
  return json
}
```

> **游标分页的优势**：页码分页在高并发场景下，如果新增/删除了数据，翻页时可能出现**重复或遗漏**。游标分页基于唯一标识定位，不会受新增/删除影响。

### 4.3 threshold（触发阈值）的选择

```
threshold 过小（如 0px）：
  → 用户滚到底才加载，需要等待请求，体验卡顿

threshold 过大（如 2000px）：
  → 用户还没滚到底就加载了，浪费请求，提前消耗数据

推荐值：
  → 200px ~ 500px（约 1~2 屏的距离）
  → 结合接口响应时间调整：接口越慢，threshold 应越大
```

### 4.4 IntersectionObserver 的 rootMargin 详解

```javascript
const observer = new IntersectionObserver(callback, {
  // rootMargin 格式同 CSS margin: "top right bottom left"
  // 正值 = 扩大检测范围（提前触发）
  // 负值 = 缩小检测范围（延后触发）
  rootMargin: '0px 0px 300px 0px',
  //                       ↑ 底部扩展 300px
  // 意味着哨兵元素距离可视区域底部还有 300px 时就触发
})
```

```
┌────────────────────── 实际视口 ──────────────────────┐

│                                                       │
├────────────────────── 视口底部 ───────────────────────┤

│                                                       │
│  ┌─────┐                                              │
```

### 5.1 双向无限滚动（聊天场景）

聊天记录需要**向上滚动加载历史消息**，同时保持当前阅读位置不变：

```javascript
// composable: useBidirectionalScroll.js

export function useBidirectionalScroll(fetchFn, options = {}) {
  const { pageSize = 20, threshold = 100 } = options

  const data = ref([])
  const loadingUp = ref(false)
  const loadingDown = ref(false)
  const hasMoreUp = ref(true)
  const hasMoreDown = ref(true)

  let oldestCursor = null
  let newestCursor = null

  // 向上加载（历史消息）
  async function loadOlder() {
    if (loadingUp.value || !hasMoreUp.value) return

    loadingUp.value = true
    const container = containerRef.value
    const prevScrollHeight = container.scrollHeight  // 记录旧高度

    try {
      const result = await fetchFn({
        direction: 'up',
        before: oldestCursor,
        limit: pageSize,
      })

      // 前置插入
      data.value = [...result.data, ...data.value]
      oldestCursor = result.data[0]?.cursor

      if (result.data.length < pageSize) {
        hasMoreUp.value = false
      }

      // 关键：修正滚动位置，保持视觉不跳动
      await nextTick()
      const newScrollHeight = container.scrollHeight
      container.scrollTop += (newScrollHeight - prevScrollHeight)

    } finally {
      loadingUp.value = false
    }
  }

  // 向下加载（新消息）
  async function loadNewer() {
    if (loadingDown.value || !hasMoreDown.value) return

    loadingDown.value = true

    try {
      const result = await fetchFn({
        direction: 'down',
        after: newestCursor,
        limit: pageSize,
      })

      // 后置追加
      data.value = [...data.value, ...result.data]
      newestCursor = result.data[result.data.length - 1]?.cursor

      if (result.data.length < pageSize) {
        hasMoreDown.value = false
      }
    } finally {
      loadingDown.value = false
    }
  }

  return {
    data,
    loadingUp,
    loadingDown,
    hasMoreUp,
    hasMoreDown,
    loadOlder,
    loadNewer,
  }
}
```

### 5.4 下拉刷新 + 无限滚动

移动端常见模式：顶部下拉刷新，底部无限加载：

```javascript
// composable: usePullRefresh.js

export function usePullRefresh(refreshFn, options = {}) {
  const { threshold = 80 } = options
  const isRefreshing = ref(false)
  const pullDistance = ref(0)
  const isPulling = ref(false)

  let startY = 0

  function onTouchStart(e) {
    // 只有在顶部才能下拉
    if (document.documentElement.scrollTop > 0) return
    startY = e.touches[0].clientY
    isPulling.value = true
  }

  function onTouchMove(e) {
    if (!isPulling.value) return
    const currentY = e.touches[0].clientY
    pullDistance.value = Math.max(0, currentY - startY)

    // 阻止浏览器默认下拉行为
    if (pullDistance.value > 0) {
      e.preventDefault()
    }
  }

  async function onTouchEnd() {
    if (pullDistance.value >= threshold && !isRefreshing.value) {
      isRefreshing.value = true
      try {
        await refreshFn()
      } finally {
        isRefreshing.value = false
      }
    }
    pullDistance.value = 0
    isPulling.value = false
  }

  return {
    isRefreshing,
    pullDistance,
    onTouchStart,
    onTouchMove,
    onTouchEnd,
  }
}
```

## 六、注意事项与常见问题

### 6.1 请求竞态问题

快速滚动时可能同时触发多次请求，导致数据重复或乱序：

```javascript
// ❌ 没有防重复，可能同时发起多个请求
async function loadMore() {
  const res = await fetch(`/api?page=${page}`)
  // 如果上一次请求还没回来，这里会导致数据重复追加
}

// ✅ 方案一：loading flag 防重复
async function loadMore() {
  if (loading.value) return  // 正在加载中，跳过
  loading.value = true
  // ...
  loading.value = false
}

// ✅ 方案二：AbortController 取消上一次请求
let abortController = null

async function loadMore() {
  // 取消上一次未完成的请求
  if (abortController) {
    abortController.abort()
  }

  abortController = new AbortController()

  try {
    const res = await fetch(`/api?page=${page}`, {
      signal: abortController.signal,
    })
    // ...
  } catch (err) {
    if (err.name === 'AbortError') return  // 被取消，忽略
    throw err
  } finally {
    abortController = null
  }
}
```

### 6.2 数据去重

服务端数据在多次请求间可能发生变化（新增、删除），导致加载的数据出现重复：

```javascript
// ✅ 使用 Set 去重
const loadedIds = new Set()

function appendData(newData) {
  const unique = newData.filter(item => {
    if (loadedIds.has(item.id)) return false
    loadedIds.add(item.id)
    return true
  })
  data.value = [...data.value, ...unique]
}
```

### 6.3 内存泄漏

滚动监听和 Observer 如果不在组件卸载时清理，会导致内存泄漏：

```javascript
// ✅ 组件卸载时清理

  if (observer) {
    observer.disconnect()
  }
  window.removeEventListener('scroll', handleScroll)
})
```

### 6.7 性能问题：DOM 无限增长

无限滚动的本质是**不断追加 DOM 节点**。当加载数据量达到数千条时，滚动会明显卡顿：

```
加载 20 条   → DOM 节点 ~20 个   → 流畅 ✅
加载 100 条  → DOM 节点 ~100 个  → 流畅 ✅
加载 500 条  → DOM 节点 ~500 个  → 开始卡顿 ⚠️
加载 2000 条 → DOM 节点 ~2000 个 → 严重卡顿 ❌
```

**解决方案：**

```
方案一：结合虚拟列表（推荐）
  → 始终只渲染可视区域内的 DOM 节点
  → 详见 5.2 节

方案二：卸载远离视口的 DOM
  → 将远离可视区域的列表项替换为占位符
  → 类似"窗口化"技术

方案三：限制最大加载数
  → 设置最大加载上限（如最多 500 条）
  → 超过后提示用户使用搜索/筛选缩小范围
```

## 七、完整方案选型对比

| 方案 | 适用场景 | 性能 | 实现复杂度 | 兼容性 |
|------|---------|------|----------|--------|
| **scroll + rAF** | 通用 | ★★★ | ★☆☆ | ★★★ |
| **IntersectionObserver** | 现代 Web 应用 | ★★★ | ★☆☆ | ★★☆ |
| **scroll 事件 + throttle** | 兼容性要求高 | ★★☆ | ★☆☆ | ★★★ |
| **成熟 UI 库组件** | 快速开发 | ★★★ | ★☆☆ | ★★★ |
| **scroll + 虚拟列表** | 海量数据 | ★★★ | ★★★ | ★★★ |

## 八、总结

无限滚动的核心实现要点：

```
1. 检测滚动位置是否接近底部（scroll 事件 或 IntersectionObserver）
2. 接近底部时判断是否可以加载（非加载中 + 还有更多数据）
3. 发起请求获取下一页数据
4. 将新数据追加到已有列表（push / concat）
5. 更新状态（loading、hasMore、page）
6. 组件卸载时清理监听器和 Observer
```

> **一句话概括**：无限滚动 = 滚动位置检测 + 分页请求 + 数据追加，核心是**在合适的时机加载合适的数据**。
