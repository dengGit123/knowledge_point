# 不定高虚拟列表（Dynamic-Height Virtual List）

## 一、什么是不定高虚拟列表？

不定高虚拟列表是指列表项高度**不固定、不可预知**的虚拟滚动方案。每项高度由其内容自然撑开——可能是一行文字，也可能是多段图文混排。典型场景如社交动态流、商品卡片列表、评论列表等。

> 核心难点：**无法通过 `index × itemHeight` 直接算出任意项的偏移位置，必须依赖额外的定位策略。**

### 与定高虚拟列表的核心区别

| 维度 | 定高 | 不定高 |
|------|------|--------|
| 项高 | 固定已知 | 动态未知 |
| 定位公式 | `index × height` | ❌ 不可用，需要位置缓存 |
| 总高度 | `total × height` | ❌ 不可用，需累加每项实际高度 |
| 滚动条精度 | 100% 精确 | 需要动态校正 |
| 实现复杂度 | ★☆☆ | ★★★ |

---

## 二、核心原理

不定高虚拟列表的本质问题可以归结为**三个关键问题**：

```
① 已知 scrollTop，如何快速找到对应的起始索引 startIndex？
② 如何知道每一项的实际高度，从而精确计算偏移位置？
③ 总内容高度（撑滚动条）怎么算？
```

### 2.1 解决思路：预估高度 + 实际测量 + 位置缓存

整体采用**"先预估、后修正"**的策略：

```
┌─────────────────────────────────────────────────────────────┐
│                    不定高虚拟列表工作流程                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 为每项设定一个「预估高度 estimatedHeight」                  │
│     ↓                                                       │
│  2. 用预估高度构建初始位置缓存 positions[]                      │
│     positions[i] = { top, bottom, height }                  │
│     ↓                                                       │
│  3. 根据 scrollTop 和 positions[] 计算 startIndex / endIndex │
│     ↓                                                       │
│  4. 渲染可见区域的列表项到 DOM                                  │
│     ↓                                                       │
│  5. 渲染后，测量每项的实际高度（getBoundingClientRect）          │
│     ↓                                                       │
│  6. 将实际高度回写到 positions[]，修正后续项的 top/bottom        │
│     ↓                                                       │
│  7. 滚动时重复步骤 3 ~ 6                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 位置缓存（positions 数组）

这是整个不定高方案的**核心数据结构**——一个有序数组，记录每一项的 `top`、`bottom` 和 `height`：

```javascript
// positions 数组结构
const positions = [
  { top: 0,    bottom: 80,  height: 80  },  // item 0
  { top: 80,   bottom: 150, height: 70  },  // item 1
  { top: 150,  bottom: 250, height: 100 },  // item 2
  { top: 250,  bottom: 310, height: 60  },  // item 3
  // ... 每一项都有精确的位置信息
]

// 初始化时用预估高度填充
function initPositions(data, estimatedHeight) {
  return data.map((_, index) => {
    const top = index * estimatedHeight
    return {
      top,
      bottom: top + estimatedHeight,
      height: estimatedHeight,
    }
  })
}
```

### 2.3 二分查找快速定位

因为 `positions` 数组中的 `bottom` 值是**单调递增**的，所以可以用**二分查找**快速定位 `scrollTop` 对应的起始索引，时间复杂度 `O(log n)`：

```javascript
/**
 * 在 positions 数组中二分查找 scrollTop 对应的起始索引
 * 找到最后一个 bottom <= scrollTop 的项，其下一项就是起始项
 */
function findStartIndex(positions, scrollTop) {
  let left = 0
  let right = positions.length - 1
  let result = 0

  while (left <= right) {
    const mid = Math.floor((left + right) / 2)
    const midBottom = positions[mid].bottom

    if (midBottom <= scrollTop) {
      // 当前项的底部还在 scrollTop 上方，继续往下找
      result = mid + 1
      left = mid + 1
    } else {
      // 当前项的底部已超过 scrollTop，往上找
      right = mid - 1
    }
  }

  return result
}
```

**图解二分查找过程：**

```
scrollTop = 200

positions:
  index 0: top=0   bottom=80    ← bottom(80) < 200，继续找
  index 1: top=80  bottom=150   ← bottom(150) < 200，继续找
  index 2: top=150 bottom=250   ← bottom(250) > 200，往上找
  index 3: top=250 bottom=310   ← 不用看了

结果: startIndex = 2（item 2 的 top=150, bottom=250，包含 scrollTop=200）
```

### 2.4 实际高度测量与缓存修正

渲染完成后，用 DOM API 测量每项真实高度，并回写 `positions` 数组：

```javascript
/**
 * 渲染后测量实际高度，修正 positions 缓存
 * @param {HTMLElement} contentEl - 内容层容器
 * @param {number} startIndex - 当前渲染的起始索引
 */
function updatePositions(contentEl, startIndex) {
  const children = contentEl.children

  for (let i = 0; i < children.length; i++) {
    const actualHeight = children[i].getBoundingClientRect().height
    const index = startIndex + i
    const oldHeight = positions[index].height
    const diff = actualHeight - oldHeight

    // 高度有变化才需要修正
    if (diff !== 0) {
      // 修正当前项
      positions[index].height = actualHeight
      positions[index].bottom = positions[index].top + actualHeight

      // 修正后续所有项的 top / bottom（累加偏移差）
      for (let j = index + 1; j < positions.length; j++) {
        positions[j].top = positions[j - 1].bottom
        positions[j].bottom = positions[j].top + positions[j].height
      }
    }
  }
}
```

```html
    // ==================== 配置 ====================
    const ESTIMATED_HEIGHT = 60       // 预估高度（px）
    const LIST_HEIGHT = 500           // 容器可视高度（px）
    const BUFFER_SIZE = 5             // 缓冲项数
    const TOTAL = 10000               // 数据总量

    // ==================== 模拟不定高数据 ====================
    // 随机生成不同长度的文本，模拟不定高内容
    const data = Array.from({ length: TOTAL }, (_, i) => {
      const lineCount = Math.floor(Math.random() * 6) + 1  // 1~6 行
      const text = Array.from({ length: lineCount }, (_, j) =>
        `这是第 ${i + 1} 条数据的第 ${j + 1} 行内容，长度随机变化。`
      ).join('')
      return {
        id: i,
        content: text,
      }
    })

    // ==================== DOM 引用 ====================
    const container = document.getElementById('listContainer')
    const phantom = document.getElementById('listPhantom')
    const content = document.getElementById('listContent')
    const domCountEl = document.getElementById('domCount')
    const totalHeightInfo = document.getElementById('totalHeightInfo')

    // ==================== 位置缓存 ====================
    let positions = []

    /** 初始化位置缓存（用预估高度填充） */
    function initPositions() {
      positions = data.map((_, index) => {
        const top = index * ESTIMATED_HEIGHT
        return {
          top,
          bottom: top + ESTIMATED_HEIGHT,
          height: ESTIMATED_HEIGHT,
        }
      })
    }

    /** 获取总内容高度（取最后一项的 bottom） */
    function getTotalHeight() {
      return positions[positions.length - 1]?.bottom ?? 0
    }

    // ==================== 二分查找 ====================
    /** 根据 scrollTop 查找起始索引 */
    function findStartIndex(scrollTop) {
      let left = 0
      let right = positions.length - 1
      let result = 0

      while (left <= right) {
        const mid = Math.floor((left + right) / 2)
        if (positions[mid].bottom <= scrollTop) {
          result = mid + 1
          left = mid + 1
        } else {
          right = mid - 1
        }
      }

      return result
    }

    /** 根据 scrollTop + listHeight 查找结束索引 */
    function findEndIndex(scrollTop, listHeight) {
      const scrollBottom = scrollTop + listHeight
      let left = 0
      let right = positions.length - 1
      let result = positions.length - 1

      while (left <= right) {
        const mid = Math.floor((left + right) / 2)
        if (positions[mid].top <= scrollBottom) {
          result = mid
          left = mid + 1
        } else {
          right = mid - 1
        }
      }

      return result
    }

    // ==================== 测量 & 修正 ====================
    /** 渲染后测量实际高度，修正 positions 缓存 */
    function updatePositions(startIndex) {
      const children = content.children

      for (let i = 0; i < children.length; i++) {
        const actualHeight = children[i].getBoundingClientRect().height
        const index = startIndex + i
        const oldHeight = positions[index].height
        const diff = actualHeight - oldHeight

        if (diff !== 0) {
          positions[index].height = actualHeight
          positions[index].bottom = positions[index].top + actualHeight

          // 级联修正后续所有项
          for (let j = index + 1; j < positions.length; j++) {
            positions[j].top = positions[j - 1].bottom
            positions[j].bottom = positions[j].top + positions[j].height
          }
        }
      }

      // 修正后更新幽灵层高度和滚动条
      phantom.style.height = `${getTotalHeight()}px`
      totalHeightInfo.textContent = Math.round(getTotalHeight())
    }

    // ==================== 核心渲染 ====================
    function render(scrollTop) {
      // ① 用二分查找定位起止索引
      let startIndex = findStartIndex(scrollTop)
      let endIndex = findEndIndex(scrollTop, LIST_HEIGHT)

      // ② 加入缓冲区
      startIndex = Math.max(0, startIndex - BUFFER_SIZE)
      endIndex = Math.min(TOTAL - 1, endIndex + BUFFER_SIZE)

      // ③ 截取可见数据
      const visibleData = data.slice(startIndex, endIndex + 1)

      // ④ 渲染 DOM
      content.innerHTML = visibleData.map(item => `
        <div class="virtual-list-item">
          <span class="item-id">#${item.id}</span>
          ${item.content}
        </div>
      `).join('')

      // ⑤ 测量实际高度并修正缓存
      updatePositions(startIndex)

      // ⑥ 偏移内容层到正确位置
      content.style.transform = `translateY(${positions[startIndex].top}px)`

      // ⑦ 更新信息
      domCountEl.textContent = visibleData.length
    }

    // ==================== 监听滚动 ====================
    let ticking = false

    container.addEventListener('scroll', () => {
      if (!ticking) {
        requestAnimationFrame(() => {
          render(container.scrollTop)
          ticking = false
        })
        ticking = true
      }
    })

    // ==================== 初始化 ====================
    initPositions()
    phantom.style.height = `${getTotalHeight()}px`
    totalHeightInfo.textContent = Math.round(getTotalHeight())
    render(0)
  </script>

</body>
</html>
```

### 3.2 Vue 3 Composition API 实现

```vue
<template>
  <div class="info">
    <div>
      数据总量: <strong>{{ total }}</strong> 条 ｜
      DOM 节点数: <strong>{{ visibleCount }}</strong> ｜
      总高度: <strong>{{ Math.round(totalHeight) }}</strong>px
    </div>
  </div>

  <!-- 虚拟列表容器 -->
  <div
    class="virtual-list-container"
    ref="containerRef"
    @scroll="handleScroll"
  >
    <!-- 幽灵层：撑出滚动条 -->
    <div
      class="virtual-list-phantom"
      :style="{ height: `${totalHeight}px` }"
    />

    <!-- 内容层 -->
    <div
      class="virtual-list-content"
      :style="{ transform: `translateY(${offsetY}px)` }"
    >
      <div
        v-for="item in visibleData"
        :key="item.id"
        class="virtual-list-item"
      >
        <span class="item-id">#{{ item.id }}</span>
        {{ item.content }}
      </div>
    </div>
  </div>
</template>

<script setup>
import { ref, computed, onMounted, watch, nextTick } from 'vue'

// ==================== Props ====================
const props = defineProps({
  data:             { type: Array,  default: () => [] },
  estimatedHeight:  { type: Number, default: 60 },    // 预估高度
  listHeight:       { type: Number, default: 500 },   // 容器可视高度
  bufferSize:       { type: Number, default: 5 },     // 缓冲项数
})

// ==================== 响应式状态 ====================
const containerRef = ref(null)
const scrollTop = ref(0)
const positions = ref([])   // 位置缓存数组

// ==================== 数据总量 ====================
const total = computed(() => props.data.length)

// ==================== 总高度（取最后一项的 bottom） ====================
const totalHeight = computed(() => {
  if (!positions.value.length) return 0
  return positions.value[positions.value.length - 1].bottom
})

// ==================== 二分查找：起始索引 ====================
function findStartIndex(st) {
  const pos = positions.value
  let left = 0
  let right = pos.length - 1
  let result = 0
  while (left <= right) {
    const mid = Math.floor((left + right) / 2)
    if (pos[mid].bottom <= st) {
      result = mid + 1
      left = mid + 1
    } else {
      right = mid - 1
    }
  }
  return result
}

// ==================== 二分查找：结束索引 ====================
function findEndIndex(st, lh) {
  const pos = positions.value
  const scrollBottom = st + lh
  let left = 0
  let right = pos.length - 1
  let result = pos.length - 1
  while (left <= right) {
    const mid = Math.floor((left + right) / 2)
    if (pos[mid].top <= scrollBottom) {
      result = mid
      left = mid + 1
    } else {
      right = mid - 1
    }
  }
  return result
}

// ==================== 起止索引（含缓冲） ====================
const startIndex = computed(() => {
  if (!positions.value.length) return 0
  const raw = findStartIndex(scrollTop.value)
  return Math.max(0, raw - props.bufferSize)
})

const endIndex = computed(() => {
  if (!positions.value.length) return 0
  const raw = findEndIndex(scrollTop.value, props.listHeight)
  return Math.min(total.value - 1, raw + props.bufferSize)
})

// ==================== 可见数据 ====================
const visibleData = computed(() =>
  props.data.slice(startIndex.value, endIndex.value + 1)
)

// ==================== 可见项数 ====================
const visibleCount = computed(() => visibleData.value.length)

// ==================== 偏移量 ====================
const offsetY = computed(() => {
  if (!positions.value.length) return 0
  return positions.value[startIndex.value]?.top ?? 0
})

// ==================== 初始化位置缓存 ====================
function initPositions() {
  positions.value = props.data.map((_, index) => {
    const top = index * props.estimatedHeight
    return {
      top,
      bottom: top + props.estimatedHeight,
      height: props.estimatedHeight,
    }
  })
}

// ==================== 测量 & 修正位置缓存 ====================
function updatePositions() {
  if (!containerRef.value) return
  const contentEl = containerRef.value.querySelector('.virtual-list-content')
  if (!contentEl) return

  const children = contentEl.children
  let needUpdate = false

  for (let i = 0; i < children.length; i++) {
    const actualHeight = children[i].getBoundingClientRect().height
    const index = startIndex.value + i
    const oldHeight = positions.value[index].height
    const diff = actualHeight - oldHeight

    if (Math.abs(diff) > 1) {  // 容差 1px，避免浮点数抖动
      needUpdate = true
      positions.value[index].height = actualHeight
      positions.value[index].bottom = positions.value[index].top + actualHeight

      // 级联修正后续项
      for (let j = index + 1; j < positions.value.length; j++) {
        positions.value[j].top = positions.value[j - 1].bottom
        positions.value[j].bottom = positions.value[j].top + positions.value[j].height
      }
    }
  }

  return needUpdate
}

// ==================== 滚动处理 ====================
let ticking = false

function handleScroll() {
  if (!containerRef.value) return
  if (!ticking) {
    requestAnimationFrame(() => {
      scrollTop.value = containerRef.value.scrollTop
      ticking = false
    })
    ticking = true
  }
}

// ==================== 数据变化时重新初始化 ====================
watch(() => props.data, () => {
  initPositions()
  scrollTop.value = 0
  if (containerRef.value) {
    containerRef.value.scrollTop = 0
  }
}, { immediate: true })

// ==================== 渲染后测量修正 ====================
watch(visibleData, async () => {
  await nextTick()
  updatePositions()
}, { flush: 'post' })

// ==================== 首次渲染 ====================
onMounted(async () => {
  await nextTick()
  updatePositions()
})
</script>
```

## 四、关键实现细节深度解析

### 4.1 级联更新优化——差量更新

上面的实现中，每次修正一项的高度后，需要**遍历修正后续所有项**的 `top/bottom`，这是 `O(n)` 的复杂度。在数据量极大时有性能瓶颈。

**优化方案：只记录差量，懒计算**

```javascript
// 优化思路：不逐项修正，而是维护一个「累计偏移差」数组
// positions 不再存储绝对 top/bottom，而是仅存 height
// 需要某项的 top 时，通过前缀和（Prefix Sum）计算

// 简化数据结构
const heights = []     // 每项实际高度
const offsets = []     // 累计偏移前缀和

function getItemTop(index) {
  // 前缀和计算（可用 Segment Tree 或 Fenwick Tree 优化）
  let top = 0
  for (let i = 0; i < index; i++) {
    top += heights[i]
  }
  return top
}

// 更优：使用 Fenwick Tree（树状数组）维护前缀和
// 单点更新 + 前缀查询均为 O(log n)
class FenwickTree {
  constructor(size) {
    this.tree = new Array(size + 1).fill(0)
  }

  // 单点更新：位置 index 加上 delta
  update(index, delta) {
    for (let i = index + 1; i < this.tree.length; i += i & -i) {
      this.tree[i] += delta
    }
  }

  // 前缀和查询：[0, index] 的累加值
  query(index) {
    let sum = 0
    for (let i = index + 1; i > 0; i -= i & -i) {
      sum += this.tree[i]
    }
    return sum
  }
}
```

> 在实际项目中，**数据量 < 10 万**时直接遍历级联修正通常就足够了。只有在**极端场景**（百万级数据 + 频繁高度变化）下才需要引入树状数组优化。

### 4.2 预估高度的选择策略

预估高度 `estimatedHeight` 直接影响滚动条的**初始精度**：

```
预估高度 = 实际平均高度  → 滚动条精确
预估高度 > 实际平均高度  → 滚动条偏长，用户滚动到底发现还有内容
预估高度 < 实际平均高度  → 滚动条偏短，用户还没到底就触底了
```

**选取策略：**

| 策略 | 做法 | 适用场景 |
|------|------|---------|
| 固定值 | 写死一个经验值（如 50px） | 已知大概高度范围 |
| 数据推算 | 根据数据特征推算（如文本行数 × 行高） | 有规律的不定高 |
| 首批采样 | 渲染前 N 项，取平均值 | 完全未知高度 |
| 动态均值 | 用已测量高度的均值不断修正 | 高度变化大 |

```javascript
// 动态均值策略
let measuredCount = 0
let measuredTotal = 0

function getEstimatedHeight() {
  if (measuredCount === 0) return 60  // 初始默认值
  return measuredTotal / measuredCount
}

function recordMeasurement(actualHeight) {
  measuredCount++
  measuredTotal += actualHeight
}
```

### 4.3 滚动条跳动问题

**现象**：用户拖拽滚动条时，列表内容突然跳动，滚动条位置不准确。

**原因**：`positions` 缓存中的高度从预估修正为实际值后，总高度发生了变化，导致当前 `scrollTop` 对应的索引发生偏移。

**解决方案：滚动位置校正**

```javascript
let isScrollingFromUser = false  // 区分用户滚动 vs 程序滚动

container.addEventListener('scroll', () => {
  if (isScrollingFromUser) return  // 正在校正中，跳过
  // ... 正常滚动处理
})

function correctScrollPosition(oldTotalHeight, newTotalHeight) {
  if (!containerRef.value || oldTotalHeight === newTotalHeight) return

  const scrollRatio = containerRef.value.scrollTop / oldTotalHeight
  const newScrollTop = scrollRatio * newTotalHeight

  if (Math.abs(newScrollTop - containerRef.value.scrollTop) > 1) {
    isScrollingFromUser = true
    containerRef.value.scrollTop = newScrollTop
    requestAnimationFrame(() => {
      isScrollingFromUser = false
    })
  }
}
```

### 4.4 `getBoundingClientRect` vs `offsetHeight`

测量 DOM 高度有两种常见方式：

```javascript
// 方式一：getBoundingClientRect
const height = element.getBoundingClientRect().height

// 方式二：offsetHeight
const height = element.offsetHeight
```

| 维度 | `getBoundingClientRect` | `offsetHeight` |
|------|------------------------|----------------|
| **触发回流** | ✅ 会触发 | ✅ 会触发 |
| **精度** | 浮点数精度（如 80.5） | 整数（四舍五入） |
| **性能** | 略慢（计算 transform） | 略快 |
| **transform 影响** | 返回变换后的值 | 不受 transform 影响 |

> **建议**：虚拟列表中优先使用 `getBoundingClientRect().height`，因为它返回**精确的浮点高度**，累积误差更小。

### 4.5 级联更新的批量处理

如果一次渲染中有多项高度同时发生变化，可以**先记录所有差量，再一次性级联更新**，避免重复遍历：

```javascript
function updatePositions(startIndex) {
  const children = content.children
  const diffs = []  // 收集所有差量

  for (let i = 0; i < children.length; i++) {
    const actualHeight = children[i].getBoundingClientRect().height
    const index = startIndex + i
    const diff = actualHeight - positions[index].height
    if (Math.abs(diff) > 1) {
      diffs.push({ index, actualHeight, diff })
    }
  }

  if (diffs.length === 0) return

  // 先更新所有变化的项
  for (const { index, actualHeight } of diffs) {
    positions[index].height = actualHeight
  }

  // 再一次性级联修正（只需一轮遍历）
  for (let i = 1; i < positions.length; i++) {
    positions[i].top = positions[i - 1].bottom
    positions[i].bottom = positions[i].top + positions[i].height
  }

  // 更新幽灵层
  phantom.style.height = `${positions[positions.length - 1].bottom}px`
}
```

## 五、进阶优化策略

### 5.1 分段缓存——避免全量级联

将数据分成固定大小的段（如每 100 条一段），每段维护一个段内位置数组和段级偏移量。修正高度时只需更新**当前段**和**后续段的偏移量**，而非每项都修正：

```javascript
const SEGMENT_SIZE = 100

// 段结构
const segments = []

function initSegments(data) {
  for (let i = 0; i < data.length; i += SEGMENT_SIZE) {
    const segment = {
      startIndex: i,
      positions: [],  // 段内位置缓存
      offset: 0,      // 段起始偏移
    }
    // 填充段内 positions...
    segments.push(segment)
  }
}

// 更新时只需重算当前段的 positions 和后续段的 offset
function updateSegment(segmentIndex) {
  // 修正段内 positions（只遍历段内项）
  // 修正后续段的 offset（只遍历段级，而非项级）
}
```

### 5.2 ResizeObserver 监听高度变化

如果列表项高度可能**动态变化**（如展开/折叠、图片懒加载完成后撑高），需要用 `ResizeObserver` 持续监听：

```javascript
const resizeObserver = new ResizeObserver((entries) => {
  for (const entry of entries) {
    const index = Number(entry.target.dataset.index)
    const newHeight = entry.contentRect.height
    const oldHeight = positions[index].height

    if (Math.abs(newHeight - oldHeight) > 1) {
      // 修正缓存并级联更新
      positions[index].height = newHeight
      positions[index].bottom = positions[index].top + newHeight
      for (let j = index + 1; j < positions.length; j++) {
        positions[j].top = positions[j - 1].bottom
        positions[j].bottom = positions[j].top + positions[j].height
      }
      // 更新幽灵层高度
      phantom.style.height = `${positions[positions.length - 1].bottom}px`
    }
  }
})

// 渲染后对每个可见项注册监听
function observeItems(contentEl, startIndex) {
  const children = contentEl.children
  for (let i = 0; i < children.length; i++) {
    const el = children[i]
    el.dataset.index = startIndex + i
    resizeObserver.observe(el)
  }
}

// 元素离开可视区域时取消监听，避免内存泄漏
function unobserveItems(contentEl) {
  const children = contentEl.children
  for (let i = 0; i < children.length; i++) {
    resizeObserver.unobserve(children[i])
  }
}
```

### 5.3 IntersectionObserver 替代 scroll 事件

与定高虚拟列表类似，可用 `IntersectionObserver` 来判断何时需要更新渲染范围，减少 `scroll` 事件的计算频率：

```javascript
// 上下哨兵元素
const sentinelTop = document.createElement('div')
const sentinelBottom = document.createElement('div')

const observer = new IntersectionObserver(
  (entries) => {
    for (const entry of entries) {
      if (entry.isIntersecting) {
        render(container.scrollTop)
      }
    }
  },
  { root: container, rootMargin: '200px 0px', threshold: 0 }
)

observer.observe(sentinelTop)
observer.observe(sentinelBottom)
```

> `rootMargin: '200px 0px'` 表示在可视区域上下各扩展 200px 的检测范围，相当于增强版缓冲区。

## 六、注意事项与常见问题

### 6.1 初始滚动条不准

**现象**：页面刚加载时，滚动条看起来正常；但滚动一段后，发现滚动条位置与实际内容不匹配。

**原因**：初始化时所有项使用预估高度，实际渲染后高度修正导致总高度变化。

**解决方案**：

```javascript
// 首屏尽可能多渲染一些项来快速积累真实高度数据
const INITIAL_RENDER_COUNT = 30  // 比实际可视项数多一些

// 首次渲染后立即测量并修正
onMounted(async () => {
  await nextTick()
  updatePositions()
  // 再次渲染以使用修正后的位置
  await nextTick()
  updatePositions()
})
```

### 6.2 快速滚动时的白屏

不定高列表比定高列表**更容易出现白屏**，因为高度的不确定性意味着缓冲区的大小更难预估。

**解决方案：**

```javascript
// 方案一：增大缓冲区
const BUFFER_SIZE = 15  // 不定高场景建议增大

// 方案二：预估高度取偏大值
const ESTIMATED_HEIGHT = 100  // 宁可偏大，也不要偏小
// 偏大 → 滚动条偏短 → 不会出现"到底了还有内容"
// 偏小 → 滚动条偏长 → 用户以为到底了实际还有

// 方案三：快速滚动时渲染更多项
function getBufferSize(scrollSpeed) {
  // scrollSpeed = |currentScrollTop - lastScrollTop|
  if (scrollSpeed > 1000) return 30   // 快速滚动
  if (scrollSpeed > 500) return 15    // 中速滚动
  return 5                            // 慢速滚动
}
```

### 6.3 图片/异步内容加载

列表项中包含图片时，图片加载完成后会撑高 DOM，但 `positions` 缓存不会自动更新：

```vue
<template>
  <div class="virtual-list-item">
    <!-- ❌ 图片加载后撑高，positions 不会更新 -->
    <img :src="item.imageUrl" />

    <!-- ✅ 方案一：固定图片容器高度 -->
    <div class="img-wrapper">
      <img :src="item.imageUrl" @load="onImageLoad(item.id)" />
    </div>
  </div>
</template>

<script setup>
function onImageLoad(itemId) {
  // 图片加载完成后，触发该项高度重新测量
  const index = props.data.findIndex(d => d.id === itemId)
  if (index >= startIndex.value && index <= endIndex.value) {
    // 重新测量并修正 positions
    nextTick(() => updatePositions())
  }
}
</script>

<style scoped>
.img-wrapper {
  width: 100%;
  height: 200px;   /* 固定容器高度 */
  overflow: hidden;
}
.img-wrapper img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
</style>
```

### 6.4 级联更新的性能陷阱

当已测量的项集中在列表头部，而用户快速滚动到列表尾部时，尾部的 `positions` 全部基于预估高度，误差会**累积放大**：

```
item 0: 预估 60 → 实际 120 → 差 +60
item 1: 预估 60 → 实际 100 → 差 +40
...
item 1000: 预估 60 → 实际 ?? → 误差已累积数百像素
```

**解决方案：预渲染采样**

```javascript
// 在后台（隐藏区域）预渲染均匀分布的采样点，提前获取高度
const SAMPLE_COUNT = 20  // 采样 20 个点

function preSample() {
  const step = Math.floor(TOTAL / SAMPLE_COUNT)
  const sampleIndices = Array.from(
    { length: SAMPLE_COUNT },
    (_, i) => i * step
  )

  // 在隐藏容器中渲染采样项
  const sampler = document.createElement('div')
  sampler.style.visibility = 'hidden'
  sampler.style.position = 'absolute'
  document.body.appendChild(sampler)

  for (const idx of sampleIndices) {
    const el = document.createElement('div')
    el.className = 'virtual-list-item'
    el.innerHTML = renderItem(data[idx])
    sampler.appendChild(el)
  }

  // 测量后回写 positions
  const children = sampler.children
  for (let i = 0; i < children.length; i++) {
    const index = sampleIndices[i]
    positions[index].height = children[i].getBoundingClientRect().height
  }

  // 级联修正
  recalcAllPositions()

  document.body.removeChild(sampler)
}
```

### 6.5 key 的正确设置

与定高虚拟列表相同，**必须使用唯一标识符**作为 `key`：

```vue
<!-- ✅ 正确 -->
<div v-for="item in visibleData" :key="item.id">

<!-- ❌ 错误：index 在不定高场景下危害更大 -->
<div v-for="(item, index) in visibleData" :key="index">
```

不定高场景中，由于可见项数量不固定（不像定高那样稳定为 `ceil(height/itemHeight)+buffer`），`index` 作为 `key` 导致的 DOM 复用错乱更加严重。

### 6.6 `contain` 与 `content-visibility` 的限制

定高列表中推荐的 CSS 优化在不定高场景下有限制：

```css
/* ❌ 不定高列表不能用 contain-intrinsic-height 指定固定值 */
.virtual-list-item {
  content-visibility: auto;
  contain-intrinsic-height: 60px;  /* 这是预估高度，不是实际高度 */
}

/* ✅ 可以使用 contain 限制影响范围 */
.virtual-list-item {
  contain: layout style paint;
}

/* ✅ will-change 仍然可用 */
.virtual-list-content {
  will-change: transform;
}
```

## 七、完整实现方案对比

不定高虚拟列表有多种实现策略，各有取舍：

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **预估 + 测量修正**（本文） | 预估高度初始化，渲染后测量修正 | 实现简单，适用面广 | 滚动条初始不准 |
| **双层渲染** | 先渲染全部但隐藏，测量后切换虚拟 | 滚动条精确 | 首次渲染慢 |
| **累计采样** | 离屏均匀采样，提前获取高度分布 | 精度较高 | 多一次 DOM 渲染 |
| **CSS Grid/Flex 懒加载** | 利用 CSS 的自动布局 + 分块加载 | 不需要手动计算 | 控制粒度粗 |
| **成熟库**（vue-virtual-scroller 等） | 内部集成了多种优化策略 | 开箱即用，功能完善 | 黑盒，定制性差 |

### 成熟库推荐

| 库 | 框架 | 特点 |
|----|------|------|
| [vue-virtual-scroller](https://github.com/Akryum/vue-virtual-scroller) | Vue 2/3 | 支持定高/不定高，Star 最多 |
| [vueuc / VirtualList](https://github.com/07akioni/vueuc) | Vue 3 | 轻量，支持不定高 |
| [@tanstack/vue-virtual](https://github.com/TanStack/virtual) | Vue/框架无关 | TanStack 出品，API 现代 |

## 八、与定高虚拟列表的完整对比

| 维度 | 定高虚拟列表 | 不定高虚拟列表 |
|------|------------|--------------|
| **每项高度** | 固定已知 | 动态，由内容决定 |
| **定位方式** | `index × height` | 位置缓存 + 二分查找 |
| **总高度计算** | `total × height` | 累加所有项高度 |
| **滚动条精度** | ✅ 精确 | ⚠️ 需动态修正 |
| **渲染后修正** | 不需要 | 必须测量 + 级联更新 |
| **实现复杂度** | ★☆☆ | ★★★ |
| **性能开销** | 仅计算 | 计算 + DOM 测量 |
| **缓冲区** | 小即可（2~5） | 建议偏大（5~15） |
| **适用场景** | 日志、表格、简单列表 | 社交动态、商品卡片、评论 |

> **选型建议**：如果列表项高度**可以通过 CSS 控制**为统一值，就选**定高方案**；只有当内容必须自然撑高、且无法预估时，才使用不定高方案。

## 九、总结

不定高虚拟列表的核心流程：

```
1. 用预估高度初始化 positions[] 缓存
2. 二分查找 scrollTop 对应的 startIndex
3. 截取可见数据并渲染到 DOM
4. 渲染后用 getBoundingClientRect 测量实际高度
5. 将实际高度回写 positions[]，级联修正后续项
6. 更新幽灵层高度（修正滚动条）
7. 监听滚动/IntersectionObserver，循环步骤 2~6
```

> **一句话概括**：不定高虚拟列表的本质是在定高方案基础上，增加了一套**"预估 → 测量 → 修正"**的位置缓存机制，用空间换时间来处理高度的不确定性。
