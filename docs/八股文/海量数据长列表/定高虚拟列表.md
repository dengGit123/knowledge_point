# 定高虚拟列表（Fixed-Height Virtual List）

## 一、什么是虚拟列表？

虚拟列表是一种**按需渲染**的技术——不管数据源有多少万条记录，DOM 中始终只保留**用户可视区域内**的少量列表项，从而大幅降低 DOM 节点数量和渲染压力。

> 核心思想：**"可见才渲染，不可见不渲染"**

### 为什么需要虚拟列表？

| 场景 | 直接渲染 | 虚拟列表 |
|------|---------|---------|
| 10 万条数据 | 创建 10 万个 DOM 节点 | 仅创建 ~20 个 DOM 节点 |
| 首屏时间 | 卡顿、白屏 | 秒开 |
| 内存占用 | 随数据量线性增长 | 恒定（与可视区域大小相关） |
| 滚动流畅度 | 掉帧严重 | 接近原生滚动 |

## 二、定高虚拟列表的核心原理

**"定高"** 指的是每一项列表项的高度是**固定且已知**的（例如每项 50px）。这使得我们可以用纯数学计算来定位，而不需要实际测量 DOM。

### 2.1 核心公式

假设：

- `itemHeight` — 每项固定高度（px）
- `listHeight` — 容器可视区域高度（px）
- `scrollTop` — 当前滚动偏移量（px）
- `total` — 数据总条数

```
// ① 可视区域能容纳的最大项数（向上取整，+1 作为缓冲）
visibleCount = Math.ceil(listHeight / itemHeight) + 1

// ② 当前滚动位置对应的起始索引
startIndex = Math.floor(scrollTop / itemHeight)

// ③ 结束索引
endIndex = Math.min(startIndex + visibleCount, total)

// ④ 实际需要渲染的列表项
visibleItems = data.slice(startIndex, endIndex)

// ⑤ 总内容高度（用于撑出滚动条）
totalHeight = total * itemHeight

// ⑥ 列表项的偏移量（将该项定位到正确位置）
offset = index * itemHeight
```

## 三、完整实现

### 3.1 原生 JavaScript 实现

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>定高虚拟列表 - 原生实现</title>
  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }

    /* 虚拟列表容器 */
    .virtual-list-container {
      width: 400px;
      height: 500px;             /* 固定可视区域高度 */
      margin: 50px auto;
      overflow: auto;            /* 关键：允许滚动 */
      border: 2px solid #ccc;
      border-radius: 8px;
      position: relative;
    }

    /* 撑高容器，产生正确的滚动条 */
    .virtual-list-phantom {
      position: absolute;
      left: 0;
      top: 0;
      right: 0;
      z-index: -1;               /* 不遮挡真实内容 */
    }

    /* 实际渲染的列表项容器 */
    .virtual-list-content {
      position: absolute;
      left: 0;
      right: 0;
      top: 0;
    }

    /* 单个列表项 */
    .virtual-list-item {
      height: 50px;              /* 固定高度 */
      line-height: 50px;
      padding: 0 20px;
      border-bottom: 1px solid #eee;
      font-size: 14px;
      color: #333;
    }

    .virtual-list-item:nth-child(odd) {
      background-color: #f9f9f9;
    }

    .virtual-list-item:hover {
      background-color: #e8f4fd;
    }

    /* 信息展示 */
    .info {
      text-align: center;
      margin: 20px;
      font-size: 14px;
      color: #666;
    }
  </style>
</head>
<body>

  <div class="info">
    <span>数据总量: <strong>100,000</strong> 条 ｜ DOM 节点数: <strong id="domCount">0</strong></span>
  </div>

  <!-- 虚拟列表结构 -->
  <div class="virtual-list-container" id="listContainer">
    <!-- 幽灵层：撑出滚动条 -->
    <div class="virtual-list-phantom" id="listPhantom"></div>
    <!-- 内容层：实际渲染的列表项 -->
    <div class="virtual-list-content" id="listContent"></div>
  </div>

  <script>
    // ==================== 配置 ====================
    const ITEM_HEIGHT = 50          // 每项固定高度（px）
    const LIST_HEIGHT = 500         // 容器可视高度（px）
    const BUFFER_SIZE = 5           // 上下缓冲项数（减少快速滚动白屏）
    const TOTAL = 100000            // 数据总量

    // ==================== 模拟数据 ====================
    const data = Array.from({ length: TOTAL }, (_, i) => ({
      id: i,
      content: `这是第 ${i + 1} 条数据 — Content Item ${i + 1}`
    }))

    // ==================== DOM 引用 ====================
    const container = document.getElementById('listContainer')
    const phantom = document.getElementById('listPhantom')
    const content = document.getElementById('listContent')
    const domCountEl = document.getElementById('domCount')

    // ==================== 初始化 ====================
    // 设置幽灵层高度 = 数据总量 × 单项高度，撑出正确的滚动条
    phantom.style.height = `${TOTAL * ITEM_HEIGHT}px`

    // ==================== 核心渲染函数 ====================
    function renderVisibleItems(scrollTop) {
      // ① 计算起始索引（减去上方缓冲）
      const rawStart = Math.floor(scrollTop / ITEM_HEIGHT)
      const startIndex = Math.max(0, rawStart - BUFFER_SIZE)

      // ② 计算可视区域能展示的项数
      const visibleCount = Math.ceil(LIST_HEIGHT / ITEM_HEIGHT)

      // ③ 计算结束索引（加上下方缓冲）
      const endIndex = Math.min(TOTAL, rawStart + visibleCount + BUFFER_SIZE)

      // ④ 截取当前需要渲染的数据
      const visibleData = data.slice(startIndex, endIndex)

      // ⑤ 渲染 DOM
      content.innerHTML = visibleData.map(item => `
        <div class="virtual-list-item" style="height: ${ITEM_HEIGHT}px;">
          ${item.content}
        </div>
      `).join('')

      // ⑥ 用 transform 偏移到正确位置（比 top 性能更好）
      content.style.transform = `translateY(${startIndex * ITEM_HEIGHT}px)`

      // ⑦ 更新 DOM 计数
      domCountEl.textContent = visibleData.length
    }

    // ==================== 监听滚动 ====================
    container.addEventListener('scroll', () => {
      const scrollTop = container.scrollTop
      // 使用 requestAnimationFrame 节流，避免高频重绘
      requestAnimationFrame(() => {
        renderVisibleItems(scrollTop)
      })
    })

    // ==================== 首次渲染 ====================
    renderVisibleItems(0)
  </script>

</body>
</html>
```

### 3.2 Vue 3 Composition API 实现

```vue
<template>
  <div class="info">
    <span>
      数据总量: <strong>{{ total }}</strong> 条 ｜
      DOM 节点数: <strong>{{ visibleData.length }}</strong>
    </span>
  </div>

  <!-- 虚拟列表容器 -->
  <div
    class="virtual-list-container"
    ref="containerRef"
    @scroll="handleScroll"
  >
    <!-- 幽灵层：撑出正确高度的滚动条 -->
    <div
      class="virtual-list-phantom"
      :style="{ height: `${totalHeight}px` }"
    />

    <!-- 内容层：实际渲染的可见列表项 -->
    <div
      class="virtual-list-content"
      :style="{ transform: `translateY(${offsetY}px)` }"
    >
      <div
        v-for="item in visibleData"
        :key="item.id"
        class="virtual-list-item"
        :style="{ height: `${itemHeight}px` }"
      >
        {{ item.content }}
      </div>
    </div>
  </div>
</template>

<script setup>
import { ref, computed, onMounted } from 'vue'

// ==================== Props 定义 ====================
const props = defineProps({
  data:        { type: Array,  default: () => [] },   // 数据源
  itemHeight:  { type: Number, default: 50 },         // 每项固定高度
  listHeight:  { type: Number, default: 500 },        // 容器可视高度
  bufferSize:  { type: Number, default: 5 },          // 缓冲项数
})

// ==================== 响应式状态 ====================
const containerRef = ref(null)
const scrollTop = ref(0)

// ==================== 计算属性 ====================
const total = computed(() => props.data.length)

// 总内容高度（撑出滚动条）
const totalHeight = computed(() => total.value * props.itemHeight)

// 可视区域能容纳的最大项数
const visibleCount = computed(() =>
  Math.ceil(props.listHeight / props.itemHeight)
)

// 起始索引（含上方缓冲）
const startIndex = computed(() => {
  const raw = Math.floor(scrollTop.value / props.itemHeight)
  return Math.max(0, raw - props.bufferSize)
})

// 结束索引（含下方缓冲）
const endIndex = computed(() => {
  const raw = Math.floor(scrollTop.value / props.itemHeight)
  return Math.min(total.value, raw + visibleCount.value + props.bufferSize)
})

// 当前需要渲染的数据切片
const visibleData = computed(() =>
  props.data.slice(startIndex.value, endIndex.value)
)

// 内容层的 Y 轴偏移
const offsetY = computed(() => startIndex.value * props.itemHeight)

// ==================== 滚动处理 ====================
function handleScroll() {
  if (!containerRef.value) return
  // 用 requestAnimationFrame 节流，减少不必要的重绘
  requestAnimationFrame(() => {
    scrollTop.value = containerRef.value.scrollTop
  })
}
</script>

  margin: 0 auto;
  overflow: auto;
  border: 2px solid #ccc;
  border-radius: 8px;
  position: relative;
}

.virtual-list-phantom {
  position: absolute;
  left: 0;
  top: 0;
  right: 0;

.virtual-list-content {

.virtual-list-item {
```

## 四、关键实现细节解析

### 4.1 三层 DOM 结构

```
┌─ container（overflow: auto）  ← 监听滚动事件的可视窗口
│   ├─ phantom（幽灵层）         ← 高度 = total × itemHeight，仅用于撑出滚动条
│   └─ content（内容层）         ← 通过 transform 偏移到正确位置，包含可见列表项
│       ├─ item ...
│       ├─ item ...
│       └─ item ...
```

| 层 | 作用 | 关键样式 |
|----|------|---------|
| **container** | 固定可视区域，提供滚动能力 | `overflow: auto; position: relative;` |
| **phantom** | 撑出与完整列表等高的滚动条 | `position: absolute; height: totalHeight;` |
| **content** | 承载实际 DOM，偏移到正确位置 | `position: absolute; transform: translateY(offset);` |

### 4.2 为什么用 `transform` 而不是 `top`？

```css
/* ✅ 推荐：触发 GPU 合成层，不走主线程布局 */
transform: translateY(500px);

/* ❌ 不推荐：触发 Layout + Paint，性能差 */
top: 500px;
```

`transform` 属于合成属性（Compositing），浏览器会将其提升到 GPU 层处理，**不会触发回流（reflow）**；而修改 `top` 会触发完整的布局计算，在滚动高频触发时性能差距明显。

### 4.3 缓冲区（Buffer）的作用

快速滚动时，`scroll` 事件和 `requestAnimationFrame` 之间可能存在时间差，导致可视区域上下边缘出现短暂的**白屏闪烁**。缓冲区通过**额外多渲染几项**来解决：

```
         ┌───────────────┐
  buffer │  item A       │ ← 虽然不可见，但预先渲染
  buffer │  item B       │
         ├───────────────┤ ← 可视区域上边界
visible │  item C  ★    │
visible │  item D  ★    │
visible │  item E  ★    │
         ├───────────────┤ ← 可视区域下边界
  buffer │  item F       │
  buffer │  item G       │ ← 虽然不可见，但预先渲染
         └───────────────┘
```

**缓冲项数建议：**

- 一般场景：`bufferSize = 2 ~ 5` 即可
- 快速滚动场景：可增大到 `10 ~ 20`
- 缓冲越大，白屏越少，但 DOM 节点越多——需要权衡

## 五、滚动性能优化

### 5.1 `requestAnimationFrame` 节流

滚动事件每秒可能触发几十上百次，每次都执行计算和 DOM 更新会造成性能浪费。用 `requestAnimationFrame` 将更新合并到浏览器的下一次重绘周期：

```javascript
container.addEventListener('scroll', () => {
  requestAnimationFrame(() => {
    renderVisibleItems(container.scrollTop)
  })
})
```

> 比 `throttle` 更适合动画/滚动场景——它会自动与屏幕刷新率（通常 60fps）同步。

### 5.2 使用 `IntersectionObserver` 替代 `scroll` 事件（进阶）

```javascript
// 在容器顶部和底部各放置一个哨兵元素
const sentinelTop = document.createElement('div')
const sentinelBottom = document.createElement('div')
content.prepend(sentinelTop)
content.append(sentinelBottom)

const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        // 哨兵进入视口 → 重新计算渲染范围
        renderVisibleItems(container.scrollTop)
      }
    })
  },
  { root: container, threshold: 0 }
)

observer.observe(sentinelTop)
observer.observe(sentinelBottom)
```

### 5.3 CSS 层面优化

```css
/* 提示浏览器内容层将频繁变化，提前创建合成层 */
.virtual-list-content {
  will-change: transform;
  contain: layout style paint;
}

/* 列表项使用 content-visibility，跳过屏幕外渲染 */
.virtual-list-item {
  content-visibility: auto;
  contain-intrinsic-height: 50px;  /* 告知浏览器固定高度 */
}
```

| 属性 | 作用 |
|------|------|
| `will-change: transform` | 提前告知浏览器 transform 属性会变化，提前创建合成层 |
| `contain: layout style paint` | 限制重绘/重排的影响范围，防止列表项相互影响 |
| `content-visibility: auto` | 跳过不可见元素的渲染（需要浏览器支持） |

## 六、注意事项与常见问题

### 6.1 滚动条高度精度问题

当数据量极大时，`totalHeight = total * itemHeight` 可能超出安全整数范围：

```javascript
// ❌ 超出 Number.MAX_SAFE_INTEGER，精度丢失
const totalHeight = 10000000000 * 50  // 500000000000 — 精确
const totalHeight = 100000000000 * 50 // 5000000000000 — 可能丢失精度

// ✅ 安全检查
if (total * itemHeight > Number.MAX_SAFE_INTEGER) {
  console.warn('总高度超出安全整数范围，滚动定位可能出现偏差')
}
```

### 6.2 快速滚动白屏问题

**现象**：快速拖拽滚动条时，中间区域出现白屏闪烁。

**原因**：`scroll` 事件触发频率有限，中间的计算和渲染跟不上滚动的速度。

**解决方案**：

```javascript
// 方案一：增大缓冲区
const BUFFER_SIZE = 20  // 从 5 增加到 20

// 方案二：滚动结束前显示占位背景（视觉补偿）
.virtual-list-container {
  background: linear-gradient(
    to bottom,
    #f5f5f5 var(--scroll-progress),
    transparent var(--scroll-progress)
  );
}

// 方案三：使用 CSS scroll-snap 或虚拟滚动库的预渲染策略
```

### 6.3 列表项中包含复杂组件

如果列表项不是纯文本，而是包含图片、异步加载内容等，需要注意：

```vue
<!-- ❌ 图片加载后会撑高列表项，破坏定高假设 -->
<div class="virtual-list-item">
  <img :src="item.imageUrl" />  <!-- 高度不确定 -->
</div>

<!-- ✅ 给图片容器固定高度，图片 object-fit 裁剪 -->
<div class="virtual-list-item">
  <div class="item-image-wrapper">
    <img :src="item.imageUrl" style="width: 100%; height: 100%; object-fit: cover;" />
  </div>
</div>

<style>
.item-image-wrapper {
  width: 40px;
  height: 40px;
  flex-shrink: 0;
  overflow: hidden;
}
</style>
```

> **原则**：定高虚拟列表的每一项高度必须严格等于 `itemHeight`，任何内部内容都不能撑破这个高度。

### 6.4 `key` 的正确设置

```vue
<!-- ✅ 使用唯一 id 作为 key -->
<div v-for="item in visibleData" :key="item.id">

<!-- ❌ 使用 index 作为 key — 滚动时 index 在不同数据间复用，导致渲染错乱 -->
<div v-for="(item, index) in visibleData" :key="index">
```

使用 `index` 作为 `key` 时，Vue 的 diff 算法会错误地复用 DOM 节点，导致列表项内容显示错乱。**务必使用数据中的唯一标识符**。

### 6.5 动态数据更新

当数据源发生变化（增删改）时，需要确保虚拟列表正确响应：

```javascript
// ✅ Vue 响应式会自动触发 computed 重算
const listData = ref([])
listData.value = newDataArray  // 整体替换

// ❌ 直接操作数组索引 — Vue 2 中可能不触发更新
// this.items[0] = newItem  // Vue 2 不能检测到
// Vue 3 的 Proxy 响应式可以检测到
```

### 6.6 横向虚拟列表

定高虚拟列表的原理同样适用于**横向滚动**，只需将纵向属性映射为横向：

| 纵向 | 横向 |
|------|------|
| `scrollTop` | `scrollLeft` |
| `height` | `width` |
| `translateY` | `translateX` |
| `itemHeight` | `itemWidth` |

### 6.7 服务端分页/异步加载

虚拟列表本身只是前端渲染优化，**不会减少网络请求**。对于真正的海量数据，需要配合分页加载策略：

```javascript
const PAGE_SIZE = 100
const loadedData = ref([])

async function loadMore() {
  // 当滚动到已加载数据的 80% 时，请求下一页
  const loadThreshold = loadedData.value.length * 0.8
  if (endIndex.value >= loadThreshold) {
    const nextPage = await fetchData(
      loadedData.value.length / PAGE_SIZE
    )
    loadedData.value.push(...nextPage)
  }
}
```

## 七、与不定高虚拟列表的对比

| 维度 | 定高虚拟列表 | 不定高虚拟列表 |
|------|------------|--------------|
| **项高** | 固定已知（如 50px） | 动态未知（由内容撑开） |
| **定位方式** | 纯数学计算 `index × height` | 需要维护位置缓存或测量 DOM |
| **实现复杂度** | ★☆☆ 简单 | ★★★ 复杂 |
| **性能** | ★★★ 最优 | ★★☆ 较好 |
| **适用场景** | 聊天记录、日志、表格行 | 社交动态流、商品列表 |
| **滚动条精度** | 精确 | 需要额外处理 |

> 当列表项高度可预知且统一时，**优先选择定高方案**——实现简单，性能最优。

## 八、总结

定高虚拟列表的核心实现步骤：

```
1. 创建三层 DOM 结构：容器 → 幽灵层 → 内容层
2. 幽灵层高度 = 数据总量 × 单项高度（撑出滚动条）
3. 监听 scroll 事件，计算当前可见的 startIndex / endIndex
4. 截取可见数据切片，渲染到内容层
5. 用 transform: translateY() 偏移内容层到正确位置
6. requestAnimationFrame 节流 + 缓冲区防白屏
```

> **一句话概括**：用数学计算代替 DOM 遍历，只渲染可视区域内的少量节点，通过绝对定位和变换将其放在正确的视觉位置。
