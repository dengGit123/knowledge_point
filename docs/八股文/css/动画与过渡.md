# 动画与过渡

> 官方文档：[MDN - 使用 CSS 动画](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_animations/Using_CSS_animations)

> 深入学习：[过渡 transition 详解](../../css/过渡(transition).md)

---

## 一、核心概念

CSS 提供两种动画机制：
- **transition（过渡）**：状态 A → 状态 B 的平滑变化，需要触发条件
- **animation（动画）**：关键帧驱动的复杂动画，可自动播放

---

## 二、transition（过渡）

### 2.1 语法

```css
/* 完整语法 */
.element {
  transition: property duration timing-function delay;
}

/* 简写 */
.box {
  transition: all 0.3s ease;
}

/* 分别设置 */
.box {
  transition-property: background-color, transform;
  transition-duration: 0.3s;
  transition-timing-function: ease-in-out;
  transition-delay: 0.1s;
}
```

| 属性 | 说明 | 示例 |
|------|------|------|
| `transition-property` | 要过渡的属性 | `all`、`transform`、`opacity` |
| `transition-duration` | 过渡持续时间 | `0.3s`、`300ms` |
| `transition-timing-function` | 缓动函数 | `ease`、`linear`、`ease-in-out` |
| `transition-delay` | 延迟时间 | `0.1s` |

### 2.2 触发条件

```css
.box {
  background: blue;
  transition: background 0.3s ease;
}

/* 通过伪类触发 */
.box:hover { background: red; }
.box:focus { background: green; }
.box:active { background: orange; }

/* 通过 JS 添加 class 触发 */
.box.active { background: red; }
```

### 2.3 缓动函数（timing-function）

```
linear:         ease:           ease-in:        ease-out:
匀速             快开始慢结束     慢开始快结束     快开始慢结束
│ ╱│            │╱│             │  ╱│           │╱
│╱ │            │╱              │ ╱              │╱
│  │            ╱               │╱               ╱│
```

```css
.box {
  /* 预设值 */
  transition-timing-function: linear;
  transition-timing-function: ease;
  transition-timing-function: ease-in;
  transition-timing-function: ease-out;
  transition-timing-function: ease-in-out;

  /* 贝塞尔曲线（可自定义任意曲线） */
  transition-timing-function: cubic-bezier(0.68, -0.55, 0.265, 1.55);

  /* 步进函数（逐帧动画） */
  transition-timing-function: steps(4, end);
}
```

### 2.4 贝塞尔曲线

```css
/* cubic-bezier(x1, y1, x2, y2) */
/* (x1,y1) 和 (x2,y2) 是两个控制点 */

/* 弹性效果 */
transition-timing-function: cubic-bezier(0.68, -0.55, 0.265, 1.55);

/* 回弹效果 */
transition-timing-function: cubic-bezier(0.175, 0.885, 0.32, 1.275);
```

> 工具推荐：[cubic-bezier.com](https://cubic-bezier.com/) 可视化生成贝塞尔曲线参数。

## 三、animation（动画）

### 3.1 语法

```css
/* 定义关键帧 */
@keyframes slideIn {
  from {
    transform: translateX(-100%);
    opacity: 0;
  }
  to {
    transform: translateX(0);
    opacity: 1;
  }
}

/* 或使用百分比 */
@keyframes bounce {
  0%   { transform: translateY(0); }
  50%  { transform: translateY(-20px); }
  100% { transform: translateY(0); }
}

/* 使用动画 */
.box {
  animation: slideIn 0.5s ease forwards;
}
```

### 3.2 animation 属性详解

| 属性 | 说明 | 示例 |
|------|------|------|
| `animation-name` | 关键帧名称 | `slideIn` |
| `animation-duration` | 持续时间 | `0.5s` |
| `animation-timing-function` | 缓动函数 | `ease` |
| `animation-delay` | 延迟 | `0.2s` |
| `animation-iteration-count` | 播放次数 | `1`、`infinite` |
| `animation-direction` | 播放方向 | `normal`、`alternate`、`reverse` |
| `animation-fill-mode` | 填充模式 | `forwards`、`backwards`、`both` |
| `animation-play-state` | 播放状态 | `running`、`paused` |

```css
.box {
  /* 简写 */
  animation: name duration timing-function delay iteration-count direction fill-mode;

  /* 示例 */
  animation: bounce 1s ease 0.2s infinite alternate forwards;
}
```

### 3.3 animation-fill-mode（重要）

```
none:       动画开始前和结束后，回到原始样式
forwards:   动画结束后，保持最后一帧的样式
backwards:  动画开始前（delay 期间），应用第一帧的样式
both:       同时应用 forwards 和 backwards
```

```css
/* ❌ 不加 forwards，动画结束后回到初始状态 */
.box {
  animation: slideIn 0.5s ease;
  /* 动画结束后元素会"弹回"原始位置 */
}

/* ✅ 加 forwards，保持动画结束状态 */
.box {
  animation: slideIn 0.5s ease forwards;
  /* 动画结束后元素停在最终位置 */
}
```

### 3.4 animation-direction

| 值 | 说明 |
|---|------|
| `normal` | 正向播放（默认） |
| `reverse` | 反向播放 |
| `alternate` | 正向 → 反向交替 |
| `alternate-reverse` | 反向 → 正向交替 |

### 3.5 逐帧动画（steps）

```css
/* 精灵图动画 */
@keyframes run {
  to { background-position: -800px 0; }
}

.sprite {
  width: 100px;
  height: 100px;
  background: url(sprite.png) no-repeat;
  animation: run 0.8s steps(8) infinite;
  /* steps(8)：分8帧播放，不做平滑插值 */
}
```

## 四、transition vs animation 对比

| 对比项 | transition | animation |
|--------|-----------|-----------|
| 触发方式 | 需要触发（hover/class） | 可自动播放 |
| 关键帧 | 只有起止两个状态 | 可定义任意多个关键帧 |
| 循环 | ❌ 不能循环 | ✅ 可循环 |
| 逆向 | 触发条件消失自动回退 | 需要设置 `direction: alternate` |
| 控制精细度 | 低（两个状态） | 高（多关键帧） |
| 适用场景 | 简单的状态过渡 | 复杂的持续动画 |

## 五、动画性能

### 5.1 安全属性 vs 昂贵属性

| 分类 | 属性 | 是否触发回流/重绘 | 性能 |
|------|------|-----------------|------|
| ✅ 安全 | `transform`、`opacity` | ❌ 只触发合成 | ⭐⭐⭐ 快 |
| ⚠️ 谨慎 | `box-shadow`、`border-radius` | ✅ 触发重绘 | ⭐⭐ 中 |
| ❌ 避免 | `width`、`height`、`top`、`left`、`margin` | ✅ 触发回流 | ⭐ 慢 |

### 5.2 性能优化建议

```css
/* ✅ 推荐：使用 transform 代替 top/left */
.box {
  transition: transform 0.3s ease;
}
.box:hover {
  transform: translateX(100px); /* GPU 加速 */
}

/* ❌ 避免：直接改变 top/left */
.box {
  transition: left 0.3s ease;
}
.box:hover {
  left: 100px; /* 触发回流，性能差 */
}

/* ✅ 提前告知浏览器将要变化的属性 */
.box {
  will-change: transform, opacity;
}
```

### 5.3 will-change

```css
.box {
  /* 提前告知浏览器要变化的属性，让浏览器优化 */
  will-change: transform;

  /* ⚠️ 不要滥用，太多 will-change 反而影响性能 */
  /* 用完后通过 JS 移除 */
}
```

## 六、实战示例

### 6.1 Loading 旋转

```css
.loader {
  width: 40px;
  height: 40px;
  border: 4px solid #f3f3f3;
  border-top: 4px solid #3498db;
  border-radius: 50%;
  animation: spin 1s linear infinite;
}

@keyframes spin {
  to { transform: rotate(360deg); }
}
```

### 6.2 淡入动画

```css
.fade-in {
  animation: fadeIn 0.5s ease forwards;
}

@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

### 6.3 按钮涟漪效果

```css
.ripple {
  position: relative;
  overflow: hidden;
}

.ripple::after {
  content: '';
  position: absolute;
  width: 100%;
  height: 100%;
  top: 0;
  left: 0;
  background: radial-gradient(circle, rgba(255,255,255,0.3) 10%, transparent 10%);
  transform: scale(10);
  opacity: 0;
  transition: transform 0.5s, opacity 0.5s;
}

.ripple:active::after {
  transform: scale(0);
  opacity: 1;
  transition: 0s;
}
```

## 七、常见面试题

### Q1：transition 和 animation 的区别？

**答：** `transition` 需要触发条件（hover、class 变化），只有起止两个状态，不能循环。`animation` 通过 `@keyframes` 定义多个关键帧，可自动播放、循环、暂停。简单过渡用 `transition`，复杂持续动画用 `animation`。

### Q2：CSS 动画如何优化性能？

**答：** ① 只动画 `transform` 和 `opacity`（不触发回流/重绘，GPU 合成层加速）；② 避免动画 `width`/`height`/`top`/`left`/`margin` 等触发布局重计算的属性；③ 使用 `will-change` 提前告知浏览器要变化的属性；④ 减少 `box-shadow` 的使用（可考虑用 `transform: scale()` + 静态 shadow 替代）。

### Q3：`animation-fill-mode: forwards` 的作用？

**答：** 动画结束后，元素保持**最后一帧**的样式。如果不设置，动画结束后元素会回到动画开始前的原始样式。开发中经常需要配合 `forwards` 保持动画结束状态。

### Q4：如何实现逐帧动画？

**答：** 使用 `steps()` 缓动函数。例如精灵图动画：`animation: run 0.8s steps(8) infinite`，`steps(8)` 将动画分为 8 个离散帧，不做平滑插值，每帧直接跳到下一状态，配合 `background-position` 实现逐帧切换。

## 八、注意事项

1. **transition 不能自动触发**：必须通过 hover、focus、active 或 JS 改变 class 来触发
2. **transform 顺序影响结果**：`translate(50px) rotate(45deg)` 和 `rotate(45deg) translate(50px)` 效果不同
3. **will-change 不要滥用**：每个 `will-change` 都会创建新的合成层，消耗内存
4. **动画结束后移除 will-change**：通过 JS 监听 `animationend`/`transitionend` 事件后移除
