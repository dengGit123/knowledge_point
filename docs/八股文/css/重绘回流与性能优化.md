# 重绘、回流与性能优化

> 官方文档：[MDN - CSS 性能优化](https://developer.mozilla.org/zh-CN/docs/Web/Performance/CSS_animations_performance)

---

## 一、浏览器渲染流程

浏览器将 HTML/CSS/JS 渲染为像素的过程：

```
HTML → DOM Tree
                → Render Tree → Layout → Paint → Composite → 像素
CSS  → CSSOM Tree
```

```
┌─────────┐   ┌──────────┐   ┌──────────────┐   ┌────────┐   ┌─────────┐   ┌──────────┐
│ 解析    │   │          │   │              │   │        │   │         │   │          │
│ HTML    │ → │ DOM Tree │   │ Render Tree  │ → │ Layout │ → │  Paint  │ → │ Composite│
│ CSS     │   │ CSSOM    │   │ (渲染树)      │   │ (布局) │   │ (绘制)  │   │ (合成)   │
│ JS      │   │          │   │              │   │        │   │         │   │          │
└─────────┘   └──────────┘   └──────────────┘   └────────┘   └─────────┘   └──────────┘
```

### 各阶段说明

| 阶段 | 说明 | 产出 |
|------|------|------|
| **DOM Tree** | 解析 HTML 生成文档对象模型 | 节点树 |
| **CSSOM** | 解析 CSS 生成样式模型 | 样式规则 |
| **Render Tree** | DOM + CSSOM 合并，排除不可见元素（`display:none`） | 可见节点+样式 |
| **Layout（布局/回流）** | 计算每个节点的**位置和大小** | 几何信息 |
| **Paint（绘制/重绘）** | 将节点**填充像素**（颜色、文字、图片、阴影等） | 像素 |
| **Composite（合成）** | 将多个图层合并输出到屏幕 | 最终画面 |

## 二、回流（Reflow / Layout）

### 2.1 什么是回流

当页面**布局或几何属性**发生变化时，浏览器重新计算元素的位置和大小——这就是回流。

> 回流 = 重新执行 Layout 阶段

### 2.2 触发回流的操作

| 类别 | 具体操作 |
|------|---------|
| **DOM 操作** | 添加/删除可见元素、改变元素位置 |
| **几何属性变化** | `width`、`height`、`padding`、`margin`、`border`、`top`、`left` |
| **内容变化** | 文本内容改变、图片加载完成 |
| **样式读取**（强制同步布局） | `offsetWidth`、`offsetHeight`、`clientWidth`、`scrollTop`、`getBoundingClientRect()` |
| **全局触发** | 窗口 resize、字体加载、激活 CSS 伪类 |
| **特定属性** | `display`、`position`、`overflow`、`font-size` |

### 2.3 强制同步布局（Forced Synchronous Layout）

> 这是最常见的性能陷阱！

```javascript
// ❌ 强制同步布局：读写交替导致多次回流
const elements = document.querySelectorAll('.box')
elements.forEach((el) => {
  const width = el.offsetWidth      // 读 → 触发回流
  el.style.width = width * 2 + 'px' // 写 → 标记布局脏
  // 下一次循环的读又会触发回流！
})

// ✅ 批量读写分离
const elements = document.querySelectorAll('.box')
const widths = []
// 先批量读
elements.forEach((el) => {
  widths.push(el.offsetWidth) // 只触发一次回流
})
// 再批量写
elements.forEach((el, i) => {
  el.style.width = widths[i] * 2 + 'px'
})
```

## 三、重绘（Repaint）

### 3.1 什么是重绘

当元素的**外观样式**（不影响布局的属性）发生变化时，浏览器重新绘制该元素的像素。

> 重绘 = 重新执行 Paint 阶段（不执行 Layout）

### 3.2 触发重绘的操作

| 属性 | 示例 |
|------|------|
| 颜色 | `color`、`background-color` |
| 阴影 | `box-shadow`、`text-shadow` |
| 边框颜色 | `border-color` |
| 可见性 | `visibility`（注意不是 `display`） |
| 轮廓 | `outline` |

### 3.3 回流与重绘的关系

```
回流 → 必然触发重绘
重绘 → 不一定触发回流
```

**性能排序**（从便宜到昂贵）：

```
Composite（最便宜） < Repaint < Reflow（最昂贵）
```

## 四、合成层（Composite Layer）

### 4.1 什么是合成层

某些元素会被浏览器提升为**独立的合成层**（GPU 加速），其变化只需重新合成，不触发回流和重绘。

### 4.2 什么会创建合成层

| 条件 | 说明 |
|------|------|
| `transform: translateZ(0)` | 经典触发方式 |
| `will-change: transform` | 显式声明 |
| `opacity` 动画 | 透明度变化 |
| `transform` 动画 | 变换动画 |
| `<video>`、`<canvas>` | 特殊元素 |
| CSS `filter` | 滤镜 |

### 4.3 GPU 加速属性

```css
/* ✅ 只触发 Composite，不触发回流和重绘 */
.box {
  transition: transform 0.3s, opacity 0.3s;
}

/* 这些属性的变化由 GPU 直接处理 */
transform: translateX(100px); /* 位移 */
transform: scale(1.5);        /* 缩放 */
transform: rotate(45deg);     /* 旋转 */
opacity: 0.5;                 /* 透明度 */
```

## 五、性能优化策略

### 5.1 减少 DOM 操作

```javascript
// ❌ 频繁操作 DOM
for (let i = 0; i < 100; i++) {
  const li = document.createElement('li')
  li.textContent = `Item ${i}`
  list.appendChild(li) // 每次都触发回流
}

// ✅ 使用 DocumentFragment
const fragment = document.createDocumentFragment()
for (let i = 0; i < 100; i++) {
  const li = document.createElement('li')
  li.textContent = `Item ${i}`
  fragment.appendChild(li)
}
list.appendChild(fragment) // 只触发一次回流
```

### 5.2 使用 transform 代替布局属性

```css
/* ❌ 触发回流 */
.box {
  transition: left 0.3s, top 0.3s;
}
.box.move {
  left: 100px;
  top: 100px;
}

/* ✅ 只触发合成 */
.box {
  transition: transform 0.3s;
}
.box.move {
  transform: translate(100px, 100px);
}
```

### 5.3 使用 opacity 代替 visibility

```css
/* ⚠️ visibility 变化触发重绘 */
.element { transition: visibility 0.3s; }

/* ✅ opacity 只触发合成 */
.element { transition: opacity 0.3s; }
```

### 5.4 批量样式修改

```javascript
// ❌ 多次修改样式
el.style.width = '100px'
el.style.height = '100px'
el.style.margin = '10px'
// 可能触发多次回流

// ✅ 通过 cssText 批量修改
el.style.cssText = 'width: 100px; height: 100px; margin: 10px;'

// ✅ 通过 class 切换
el.className = 'active'
```

### 5.5 脱离文档流进行复杂动画

```css
/* 对频繁变化的元素使用绝对定位 */
.animating-element {
  position: absolute; /* 或 fixed */
  /* 变化时不影响其他元素的布局 */
}
```

### 5.6 will-change 优化

```css
/* 告诉浏览器该元素即将变化，提前优化 */
.card:hover {
  will-change: transform;
}

/* ⚠️ 不要对所有元素都加 will-change */
```

### 5.7 CSS containment

```css
.container {
  /* 限制元素的渲染影响范围 */
  contain: layout;      /* 布局独立 */
  contain: paint;       /* 绘制独立 */
  contain: size;        /* 尺寸独立 */
  contain: content;     /* 等同于 layout + paint */
  contain: strict;      /* 等同于 size + layout + paint */
}
```

## 六、优化策略总结表

| 策略 | 效果 | 实施难度 |
|------|------|---------|
| 使用 `transform`/`opacity` 做动画 | 只触发合成 | ⭐ 低 |
| 读写分离（避免强制同步布局） | 减少回流次数 | ⭐ 低 |
| `DocumentFragment` 批量 DOM | 减少回流 | ⭐ 低 |
| `class` 切换代替逐条修改样式 | 减少回流 | ⭐ 低 |
| `will-change` 提前优化 | 创建合成层 | ⭐ 低 |
| 脱离文档流做动画 | 不影响其他元素 | ⭐⭐ 中 |
| `contain` 属性限制渲染范围 | 隔离回流/重绘 | ⭐⭐ 中 |
| 虚拟滚动（长列表） | 减少 DOM 数量 | ⭐⭐⭐ 高 |

## 七、常见面试题

### Q1：什么是回流和重绘？它们的区别？

**答：**
- **回流（Reflow）**：页面布局或几何属性变化时，浏览器重新计算元素的位置和大小。影响范围可能是局部或全局。
- **重绘（Repaint）**：元素外观样式变化（如颜色、阴影）时，浏览器重新绘制像素，不影响布局。
- **关系**：回流必然触发重绘，重绘不一定触发回流。回流的性能开销远大于重绘。

### Q2：如何减少回流？

**答：**
1. 使用 `transform` 代替 `top/left` 做动画
2. 读写分离：批量读取 DOM 尺寸，再批量修改
3. 使用 `DocumentFragment` 批量插入 DOM
4. 通过 `class` 切换代替逐条修改 `style`
5. 对频繁动画的元素使用 `position: absolute` 脱离文档流
6. 避免频繁查询 `offsetWidth` 等触发布局计算的属性
7. 使用 `will-change` 提前告知浏览器

### Q3：浏览器的渲染流程是什么？

**答：** HTML 解析为 DOM Tree，CSS 解析为 CSSOM，两者合并为 Render Tree（排除不可见元素）。然后经过 Layout（计算位置大小）→ Paint（绘制像素）→ Composite（合成图层输出到屏幕）。其中 `display: none` 的元素不进入 Render Tree，而 `visibility: hidden` 的元素会进入但不可见。

### Q4：为什么 transform 动画性能好？

**答：** `transform` 属性的变化不会触发 Layout 和 Paint，只触发 Composite 阶段。浏览器会为使用 `transform` 的元素创建独立的**合成层**（Compositing Layer），由 GPU 直接处理，不阻塞主线程。而修改 `width`、`top` 等属性会触发完整的 Layout → Paint → Composite 流程。

### Q5：`display: none` 和 `visibility: hidden` 的区别？

**答：**
- `display: none`：元素完全从渲染树中移除，**不占空间**，会触发回流。
- `visibility: hidden`：元素仍在渲染树中，**占空间但不可见**，只触发重绘。

---

## 八、注意事项

1. **读取布局属性会强制回流**：`offsetWidth`、`clientHeight`、`getComputedStyle()` 等
2. **`will-change` 不能滥用**：每个合成层都消耗 GPU 内存，过多会导致性能下降
3. **CSS `contain` 是现代优化利器**：对独立组件使用 `contain: content` 可显著减少回流范围
4. **Chrome DevTools**：Performance 面板可直观看到 Layout/Paint 的耗时，Layers 面板可查看合成层数量
