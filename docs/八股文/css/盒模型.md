# 盒模型（Box Model）

> 官方文档：[MDN - 盒模型](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_box_model)

> 深入学习：[盒模型详解](../../css/盒子模型.md)

---

## 一、核心概念

CSS 盒模型是布局的基础——**每个 HTML 元素都被视为一个矩形盒子**，由四层嵌套组成：

```
┌─────────────────────────────────────────┐
│                 margin                  │
│   ┌─────────────────────────────────┐   │
│   │             border              │   │
│   │   ┌─────────────────────────┐   │   │
│   │   │        padding          │   │   │
│   │   │   ┌─────────────────┐   │   │   │
│   │   │   │     content     │   │   │   │
│   │   │   └─────────────────┘   │   │   │
│   │   └─────────────────────────┘   │   │
│   └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

通俗类比：**快递包裹** —— 商品本身是 content，泡沫填充是 padding，纸箱是 border，纸箱之间的间距是 margin。

## 二、两种盒模型

### 2.1 标准盒模型（content-box）

`box-sizing: content-box`（默认值）

- `width` / `height` 只设置 **content** 的尺寸
- 元素实际占用空间 = content + padding + border + margin

```
实际宽度 = width + padding-left + padding-right + border-left + border-right + margin-left + margin-right
```

```
┌──────────────────────────────────────┐
│             margin: 10px             │
│   ┌──────────────────────────────┐   │
│   │          border: 5px         │   │
│   │   ┌──────────────────────┐   │   │
│   │   │     padding: 20px    │   │   │
│   │   │   ┌──────────────┐   │   │   │
│   │   │   │  width:100px │   │   │   │
│   │   │   └──────────────┘   │   │   │
│   │   └──────────────────────┘   │   │
│   └──────────────────────────────┘   │
└──────────────────────────────────────┘

实际占用宽度 = 10 + 5 + 20 + 100 + 20 + 5 + 10 = 170px
```

### 2.2 IE 盒模型（border-box）

`box-sizing: border-box`（推荐）

- `width` / `height` 设置的是 **content + padding + border** 的总尺寸
- 元素实际占用空间 = width + margin

```
实际宽度 = width + margin-left + margin-right
（width 已经包含了 content + padding + border）
```

```
┌──────────────────────────────────────┐
│             margin: 10px             │
│   ┌──────────────────────────────┐   │
│   │     width: 100px（含border+  │   │
│   │     padding）                 │   │
│   │   ┌──────────────────────┐   │   │
│   │   │     padding: 20px    │   │   │
│   │   │   ┌──────────────┐   │   │   │
│   │   │   │ content:50px │   │   │   │
│   │   │   └──────────────┘   │   │   │
│   │   └──────────────────────┘   │   │
│   └──────────────────────────────┘   │
└──────────────────────────────────────┘

实际占用宽度 = 10 + 100 + 10 = 120px
content 实际宽度 = 100 - 20*2 - 5*2 = 50px
```

### 2.3 对比总结

| 对比项 | content-box（标准） | border-box（IE） |
|--------|--------------------|-----------------|
| width 包含 content | ✅ 仅 content | ✅ content + padding + border |
| 增大 padding 后元素变大 | ✅ 是 | ❌ 否（content 自动缩小） |
| 计算复杂度 | 高（需手动累加） | 低（width 就是实际宽度） |
| 使用场景 | 特殊需求 | **日常开发推荐** |

## 三、全局最佳实践

```css
/* 所有项目都应加上这段全局重置 */
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

**为什么推荐 border-box？**

```css
/* content-box 的痛点 */
.box {
  width: 100%;
  padding: 20px; /* 实际宽度变成 100% + 40px，可能溢出容器！ */
}

/* border-box 的优势 */
.box {
  box-sizing: border-box;
  width: 100%;
  padding: 20px; /* 实际宽度仍然是 100%，content 自动缩减 */
}
```

## 四、Margin 合并（塌陷）

### 4.1 什么是 Margin 合并

两个相邻块的垂直外边距会**合并为一个外边距**，取两者中的较大值。

```
┌─────────────┐
│   元素 A     │
│ margin: 20px │  ─┐
└─────────────┘   │  合并后间距 = max(20, 30) = 30px（不是 50px）
┌─────────────┐   │
│   元素 B     │  ─┘
│ margin: 30px │
└─────────────┘
```

### 4.2 三种发生场景

| 场景 | 示意 | 说明 |
|------|------|------|
| 相邻兄弟 | A 的 margin-bottom 与 B 的 margin-top | 最常见 |
| 父子元素 | 父 margin-top 与子 margin-top | 子的 margin "溢出"到父 |
| 空块元素 | 自身的 margin-top 与 margin-bottom | 自身合并，高度为 0 |

### 4.3 解决方案

```css
/* 方案 1：BFC（推荐） */
.parent {
  overflow: hidden; /* 触发 BFC，阻止 margin 溢出 */
}

/* 方案 2：给父元素加 border */
.parent {
  border-top: 1px solid transparent;
}

/* 方案 3：给父元素加 padding */
.parent {
  padding-top: 1px;
}

/* 方案 4：Flex / Grid 布局（现代方案，不会发生 margin 合并） */
.parent {
  display: flex;
  flex-direction: column;
}
```

### 4.4 注意事项

- **只有垂直方向的 margin 会合并**，水平方向不会（在常规流中）
- **浮动、绝对定位、inline-block 的元素不会发生 margin 合并**
- **Flex 和 Grid 布局的子项不会发生 margin 合并**

## 五、元素的显示类型

| 类型 | 特点 | 典型元素 | 宽高设置 |
|------|------|---------|---------|
| 块级（block） | 独占一行 | `div`、`p`、`h1`、`ul` | ✅ 有效 |
| 行内（inline） | 与其他行内元素同行 | `span`、`a`、`strong` | ❌ 无效 |
| 行内块（inline-block） | 同行但可设宽高 | `img`、`input` | ✅ 有效 |

```css
/* 块级元素 */
div, p, h1, ul, section { display: block; }

/* 行内元素 */
span, a, strong, em { display: inline; }

/* 行内块元素 */
img, input, button { display: inline-block; }
```

## 六、常见面试题

### Q1：什么是 CSS 盒模型？

**答：** CSS 盒模型将每个元素描述为一个由四层组成的矩形盒子：**content（内容）→ padding（内边距）→ border（边框）→ margin（外边距）**。有两种盒模型：标准盒模型（`content-box`）的 width 只包含内容区，IE 盒模型（`border-box`）的 width 包含内容区 + 内边距 + 边框。实际开发推荐使用 `border-box`。

### Q2：content-box 和 border-box 的区别？

**答：**
- `content-box`：`width` = 内容区宽度。元素实际宽度 = width + padding + border。
- `border-box`：`width` = 内容区 + padding + border。元素实际宽度就是 width。
- `border-box` 更直观，设置 `width: 200px` 就真的是 200px，不会被 padding/border 撑大。

### Q3：为什么会出现 margin 合并？怎么解决？

**答：** CSS 规范规定，在常规文档流中，相邻的垂直 margin 会合并取较大值。这是为了排版美观（多个段落间距不会双倍）。解决方案：触发 BFC（`overflow: hidden`）、加 border/padding 隔离、或使用 Flex/Grid 布局（天然不会合并）。

### Q4：如何计算一个元素的实际占用空间？

**答：**
- `content-box`：总宽度 = `width + padding-left + padding-right + border-left + border-right + margin-left + margin-right`
- `border-box`：总宽度 = `width + margin-left + margin-right`
- 可通过 `element.offsetWidth` 和 `element.getBoundingClientRect()` 获取实际尺寸。

### Q5：行内元素能设置宽高和 margin/padding 吗？

**答：**
- 行内元素**不能**设置 width、height、margin-top、margin-bottom。
- 行内元素**可以**设置 padding 和 margin-left、margin-right，但 padding 的垂直方向**不会影响布局**（不占据空间，但视觉上会显示），可能与其他元素重叠。

## 七、注意事项

1. **全局重置 box-sizing**：所有项目都应在 CSS 重置中设置 `*, *::before, *::after { box-sizing: border-box }`
2. **margin 合并只发生在垂直方向**：水平方向的 margin 不会合并
3. **负 margin**：`margin` 可以为负值，会使元素向相反方向移动，常见于布局技巧
4. **百分比 padding/margin**：无论是水平还是垂直，百分比都**相对于父元素的宽度**计算
5. **outline 不占空间**：`outline` 类似 border 但不属于盒模型，不占据布局空间
