# 浮动与清除浮动

> 官方文档：[MDN - float](https://developer.mozilla.org/zh-CN/docs/Web/CSS/float)

---

## 一、核心概念

`float` 最初是为**文字环绕图片**设计的，后来被广泛用于布局（现已不推荐，推荐用 Flex/Grid）。

> 通俗类比：`float` 就像把一张图片贴在文字旁边——文字自动环绕在图片周围流动。

```css
img {
  float: left; /* 图片浮动到左侧，文字环绕在右侧 */
}
```

## 二、浮动特性

| 特性 | 说明 |
|------|------|
| 脱离文档流 | 浮动元素不再占据正常流中的空间 |
| 文字不重叠 | 浮动元素不会覆盖文字内容（文字会环绕） |
| 行内块化 | 浮动后元素表现为类似 `inline-block` |
| 只能左右 | `float: left` 或 `float: right`，不能上下 |

### 浮动前 vs 浮动后

```css
/* 浮动前：块级元素独占一行 */
.box { width: 100px; height: 100px; }
/* 两个 .box 上下排列 */

/* 浮动后：横向排列 */
.box { float: left; width: 100px; height: 100px; }
/* 两个 .box 左右排列 */
```

## 三、高度塌陷问题

**核心问题**：子元素浮动后脱离文档流，父元素如果没有设置高度，高度会变为 0。

```html
<div class="parent" style="border: 2px solid red;">
  <div class="child" style="float: left; width: 100px; height: 100px; background: blue;"></div>
</div>
<!-- parent 高度为 0！只显示红色边框的一条线 -->
```

```
│                     │

```

## 四、清除浮动的五种方法

### 方法 1：额外标签法（不推荐）

```html
<div class="parent">
  <div class="child" style="float: left;"></div>
  <div style="clear: both;"></div> <!-- 额外空标签 -->
</div>
```

❌ 增加无语义的 HTML 标签，违反结构与样式分离原则。

### 方法 2：父元素 overflow（常用）

```css
.parent {
  overflow: hidden; /* 或 overflow: auto */
}
```

✅ 简单有效。⚠️ 但会裁切溢出内容（如 tooltip、下拉菜单）。

### 方法 3：clearfix 伪元素（经典方案）

```css
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}

/* 更完善的版本 */
.clearfix::after {
  content: '';
  display: table;
  clear: both;
}
```

```html
<div class="parent clearfix">
  <div class="child" style="float: left;"></div>
</div>
```

✅ 无额外 HTML 标签，无副作用。⭐ 最经典的方案。

### 方法 4：display: flow-root（现代推荐）

```css
.parent {
  display: flow-root;
}
```

✅ 语义最清晰，专门为创建 BFC 设计，无副作用。

### 方法 5：父元素也浮动（不推荐）

```css
.parent {
  float: left;
  width: 100%; /* 需要设置宽度 */
}
```

❌ 会影响父元素的兄弟元素布局，问题传递。

### 方法对比

| 方法 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| 额外标签 | 简单 | 无语义标签 | ❌ |
| `overflow: hidden` | 简单 | 裁切溢出内容 | ⚠️ |
| **clearfix** | 无副作用 | 代码稍多 | ⭐⭐⭐ |
| **flow-root** | 语义最佳 | IE 不支持 | ⭐⭐⭐⭐ |
| 父元素浮动 | — | 影响外层布局 | ❌ |

## 五、float 实现的经典布局

> ⚠️ 以下布局现代开发推荐使用 Flex/Grid，这里仅用于面试理解。

### 5.1 两栏布局（左固定右自适应）

```css
.left {
  float: left;
  width: 200px;
}

.right {
  margin-left: 200px; /* 或 overflow: hidden 创建 BFC */
}
```

### 5.2 三栏布局（圣杯布局）

```css
.container {
  padding: 0 200px;
}

.left {
  float: left;
  width: 200px;
  margin-left: -100%;
  position: relative;
  left: -200px;
}

.right {
  float: left;
  width: 200px;
  margin-left: -200px;
  position: relative;
  right: -200px;
}

.center {
  float: left;
  width: 100%;
}
```

## 六、常见面试题

### Q1：为什么浮动会导致高度塌陷？

**答：** 浮动元素脱离了文档流（normal flow），不再占据空间。父元素计算高度时只考虑正常文档流中的子元素，浮动子元素不参与计算，所以父元素高度变为 0（或只包含非浮动子元素的高度）。

### Q2：清除浮动有哪些方式？推荐哪种？

**答：** 五种常见方式：① 额外空标签 `clear: both`；② 父元素 `overflow: hidden`；③ clearfix 伪元素（`::after { clear: both }`）；④ `display: flow-root`；⑤ 父元素也浮动。推荐 **clearfix**（兼容性好）或 **`display: flow-root`**（语义最佳，现代浏览器）。

### Q3：`clear: both` 的原理是什么？

**答：** `clear` 属性规定元素的哪一侧不允许有浮动元素。`clear: both` 表示元素左右两侧都不允许有浮动元素，浏览器会在该元素上方添加足够的间距，使其出现在所有浮动元素的下方。

### Q4：现代布局还需要浮动吗？

**答：** 不需要。浮动用于布局是历史遗留做法，现代开发应使用 **Flexbox**（一维布局）或 **Grid**（二维布局）。`float` 唯一合理的现代用途是**文字环绕图片**——这正是它最初被设计出来的目的。

## 七、注意事项

1. **浮动只影响后面的元素**：浮动元素不会影响它前面的兄弟元素
2. **浮动元素不会超过父元素的内容区域**：不会被 margin 挤到外面
3. **行内元素与浮动**：行内元素不会浮动，设置 float 后会自动变为块级
4. **清除浮动本质**：本质是创建新的 BFC 或在布局上隔离浮动元素
