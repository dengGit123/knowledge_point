# 响应式设计与移动端适配

> 官方文档：[MDN - 响应式设计](https://developer.mozilla.org/zh-CN/docs/Learn/CSS/CSS_layout/Responsive_Design)

---

## 一、核心概念

响应式设计（Responsive Design）使网页能够**自适应不同屏幕尺寸**，提供良好的浏览体验。

> 通俗类比：响应式设计就像一件弹性衣服，无论穿它的人高矮胖瘦，都能自动调整贴合身形。

---

## 二、viewport meta 标签

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, user-scalable=no">
```

| 属性 | 值 | 说明 |
|------|---|------|
| `width` | `device-width` | 视口宽度 = 设备宽度 |
| `initial-scale` | `1.0` | 初始缩放比例 |
| `maximum-scale` | `1.0` | 最大缩放比例 |
| `minimum-scale` | `1.0` | 最小缩放比例 |
| `user-scalable` | `no` | 禁止用户缩放 |

> **不加 viewport meta 的后果**：手机浏览器会将页面按桌面宽度（通常 980px）渲染，然后缩小显示，导致文字极小。

## 三、媒体查询（Media Query）

### 3.1 语法

```css
/* 基本语法 */
@media screen and (max-width: 768px) {
  .container {
    flex-direction: column;
  }
}

/* 多条件 */
@media screen and (min-width: 768px) and (max-width: 1024px) {
  .container {
    padding: 20px;
  }
}

/* 横竖屏 */
@media (orientation: landscape) {
  /* 横屏样式 */
}

/* 高分辨率屏幕 */
@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) {
  /* Retina 屏幕样式 */
}
```

### 3.2 常见断点

| 断点 | 名称 | 设备 |
|------|------|------|
| `< 576px` | xs | 手机竖屏 |
| `≥ 576px` | sm | 手机横屏 |
| `≥ 768px` | md | 平板竖屏 |
| `≥ 992px` | lg | 平板横屏 / 小笔记本 |
| `≥ 1200px` | xl | 桌面 |
| `≥ 1400px` | xxl | 大屏桌面 |

### 3.3 Mobile-first vs Desktop-first

```css
/* Mobile-first（推荐）：从小屏开始，逐步扩展 */
.container {
  width: 100%;      /* 默认：手机 */
}
@media (min-width: 768px) {
  .container {
    width: 750px;   /* 平板 */
  }
}
@media (min-width: 1200px) {
  .container {
    width: 1170px;  /* 桌面 */
  }
}

/* Desktop-first：从大屏开始，逐步缩减 */
.container {
  width: 1170px;    /* 默认：桌面 */
}
@media (max-width: 1200px) {
  .container {
    width: 750px;   /* 平板 */
  }
}
@media (max-width: 768px) {
  .container {
    width: 100%;    /* 手机 */
  }
}
```

**推荐 Mobile-first**：CSS 默认就是小屏样式，`min-width` 向上覆盖，更符合渐进增强理念。

## 四、移动端适配方案

### 4.1 rem 适配方案

```javascript
// 动态设置根元素 font-size
function setRemUnit() {
  const designWidth = 375 // 设计稿宽度
  const baseFontSize = 16 // 基准 font-size
  const scale = document.documentElement.clientWidth / designWidth
  document.documentElement.style.fontSize = baseFontSize * scale + 'px'
}

setRemUnit()
window.addEventListener('resize', setRemUnit)
```

```css
/* 设计稿 375px 下 1rem = 16px */
/* 设计稿中某元素宽 100px → 100/16 = 6.25rem */
.element {
  width: 6.25rem;
}
```

### 4.2 vw 适配方案

```css
/* 设计稿 375px，某元素宽 100px */
/* 100 / 375 × 100 = 26.67vw */
.element {
  width: 26.67vw;
}
```

> 使用 PostCSS 插件 `postcss-px-to-viewport` 可自动将 `px` 转换为 `vw`。

### 4.3 rem + vw 混合方案

```css
html {
  /* 纯 CSS 实现，无需 JS */
  /* 375px 设计稿下 font-size = 100px → 方便计算 */
  font-size: calc(100vw / 3.75);
}

/* 设计稿中 100px → 1rem */
.element {
  width: 1rem; /* 等于 100vw/3.75，自动等比缩放 */
}
```

### 方案对比

| 方案 | 优点 | 缺点 | 是否需要 JS |
|------|------|------|------------|
| **rem + JS** | 兼容性好 | 需要 JS，1px 问题 | ✅ |
| **vw** | 纯 CSS，简洁 | 旧浏览器不支持 | ❌ |
| **rem + vw** | 纯 CSS，兼顾兼容 | 计算稍复杂 | ❌ |

## 五、响应式布局技巧

### 5.1 弹性网格

```css
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 24px;
}
/* 自动根据屏幕宽度调整列数，无需媒体查询 */
```

### 5.2 弹性图片

```css
img {
  max-width: 100%;
  height: auto;
}
```

### 5.3 响应式字体

```css
h1 {
  font-size: clamp(24px, 5vw, 48px);
}
```

### 5.4 响应式导航

```css
/* 桌面：水平导航 */
.nav {
  display: flex;
  gap: 20px;
}

/* 手机：汉堡菜单 */
@media (max-width: 768px) {
  .nav {
    display: none; /* 默认隐藏 */
  }
  .nav.open {
    display: flex;
    flex-direction: column;
  }
  .hamburger {
    display: block; /* 显示汉堡按钮 */
  }
}
```

### 5.5 CSS Container Queries（新特性）

```css
/* 基于容器宽度（而非视口宽度）响应 */
.card-container {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    display: flex;
    gap: 16px;
  }
}

@container (max-width: 399px) {
  .card {
    display: block;
  }
}
```

## 六、常见面试题

### Q1：响应式和自适应的区别？

**答：**
- **响应式（Responsive）**：一套代码，通过媒体查询和弹性布局自动适配各种屏幕。页面布局会根据屏幕宽度流畅变化。
- **自适应（Adaptive）**：为特定屏幕尺寸设计固定布局，通常通过媒体查询在不同断点切换完全不同的布局。像是在不同屏幕上加载不同的页面。

### Q2：rem 适配的原理？

**答：** 通过 JS 根据屏幕宽度动态设置 `<html>` 的 `font-size`。由于 `rem` 相对于根元素的 `font-size`，所有使用 `rem` 的属性会等比缩放。例如：设计稿宽 375px，基准 `font-size: 16px`，屏幕宽 750px 时 `font-size` 变为 32px，所有 rem 值自动放大一倍。

### Q3：常见的响应式断点有哪些？

**答：** 通常按手机（<576px）、平板（768px）、笔记本（992px）、桌面（1200px）划分。Mobile-first 策略使用 `min-width` 向上递进，Desktop-first 使用 `max-width` 向下递减。推荐 Mobile-first。

### Q4：如何不用媒体查询实现响应式？

**答：** 使用 `grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))` 实现自动列数调整；使用 `clamp()` 实现响应式字体；使用 `vw` 单位实现等比缩放；使用 CSS Container Queries 基于容器宽度响应。

## 七、注意事项

1. **viewport meta 必须加**：不加的话移动端会按桌面宽度渲染
2. **Mobile-first 性能更好**：小屏设备只需解析少量 CSS
3. **避免固定宽度**：使用 `max-width` 代替 `width`，保证小屏不溢出
4. **触摸目标大小**：移动端可点击区域不小于 44×44px
5. **`user-scalable=no` 无障碍问题**：禁止缩放影响视障用户，需谨慎使用
