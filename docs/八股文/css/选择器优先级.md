# 选择器优先级（Specificity）

> 官方文档：[MDN - 优先级](https://developer.mozilla.org/zh-CN/docs/Web/CSS/Specificity)

---

## 一、核心概念

当多条 CSS 规则同时作用于同一个元素时，浏览器需要决定最终应用哪一条——这就是**优先级（Specificity）**机制。

> 通俗类比：CSS 优先级就像"说话的份量"。普通建议（类选择器）< 领导指示（ID 选择器）< 老板命令（行内样式）< 圣旨（!important）。

---

## 二、优先级计算规则

### 2.1 权重记分法（A, B, C, D）

| 等级 | 选择器类型 | 权重值 | 示例 |
|------|-----------|--------|------|
| **A** | 行内样式 | 1000 | `style="..."` |
| **B** | ID 选择器 | 100 | `#header` |
| **C** | 类 / 伪类 / 属性选择器 | 10 | `.nav`、`:hover`、`[type="text"]` |
| **D** | 元素 / 伪元素选择器 | 1 | `div`、`::before` |
| — | 通配符 / 结合符 | 0 | `*`、`>`、`+`、`~` |
| — | `!important` | 最高 | 覆盖一切 |

> **注意**：权重不是十进制数，而是**逐级比较**。10 个类选择器（100）也不会超过 1 个 ID 选择器（100），因为它是从左到右逐级比较。

### 2.2 计算示例

```css
/* 权重：0, 0, 0, 1 → (0-0-0-1) */
p { color: gray; }

/* 权重：0, 0, 1, 0 → (0-0-1-0) */
.text { color: blue; }

/* 权重：0, 0, 1, 1 → (0-0-1-1) */
p.text { color: green; }

/* 权重：0, 1, 0, 0 → (0-1-0-0) */
#title { color: orange; }

/* 权重：0, 1, 1, 1 → (0-1-1-1) */
#title p.text { color: red; }

/* 权重：1, 0, 0, 0 → (1-0-0-0) */
<p style="color: purple;">文字</p>

/* 权重：!important → 无限大 */
p { color: black !important; }
```

最终优先级排序（从低到高）：

```
p < .text < p.text < #title < #title p.text < 行内样式 < !important
```

### 2.3 特殊选择器权重

| 选择器 | 权重 | 说明 |
|--------|------|------|
| `*` | (0,0,0,0) | 通配符，零权重但不为零优先级 |
| `div p` | (0,0,0,2) | 后代选择器，权重累加 |
| `div > p` | (0,0,0,2) | 子代选择器，`>` 不计入权重 |
| `div + p` | (0,0,0,2) | 相邻兄弟，`+` 不计入权重 |
| `:not(.foo)` | (0,0,1,0) | `:not()` 本身不计，参数计入 |
| `:is(.foo, #bar)` | 取参数最高权重 | 取括号内最高权重 |
| `:where(.foo, #bar)` | (0,0,0,0) | 零权重！专门用于可覆盖 |
| `:nth-child(2)` | (0,0,1,0) | 伪类，权重为 10 |
| `::before` | (0,0,0,1) | 伪元素，权重为 1 |

### 2.4 `:where()` 和 `:is()` 对比

```css
/* :is() — 取参数中最高的权重 */
:is(#header, .nav) p {
  color: red;
  /* 权重 = (0,1,0,1)，因为 #header 权重最高 */
}

/* :where() — 权重始终为 0 */
:where(#header, .nav) p {
  color: red;
  /* 权重 = (0,0,0,1)，只有 p 贡献权重 */
}
```

## 三、优先级比较流程

```
Step 1: 比较 !important
  ↓ 有 !important 的一方胜出
  ↓ 都有 !important → 继续按权重比较
Step 2: 比较行内样式 (A)
  ↓ A 大的胜出
  ↓ 相同 → 比较 B
Step 3: 比较 ID 选择器数量 (B)
  ↓ B 大的胜出
  ↓ 相同 → 比较 C
Step 4: 比较类/伪类/属性选择器数量 (C)
  ↓ C 大的胜出
  ↓ 相同 → 比较 D
Step 5: 比较元素/伪元素选择器数量 (D)
  ↓ D 大的胜出
  ↓ 相同 → 看源码顺序
Step 6: 源码顺序
  ↓ 后声明的覆盖先声明的
```

### 权重相同时的规则

```css
/* 两条规则权重相同：(0,0,1,0) */
.text { color: blue; }
.text { color: red; }   /* ✅ 胜出：后声明的覆盖先声明的 */
```

```css
/* 相同选择器，相同属性，后写的生效 */
.nav { color: blue; }
.nav { color: red; }    /* 最终为 red */

/* 不同文件中的相同样式，后引入的文件生效 */
@import 'a.css';  /* .text { color: blue; } */
@import 'b.css';  /* .text { color: red; }  → 最终为 red */
```

## 四、常见面试题

### Q1：CSS 选择器优先级怎么计算？

**答：** 使用 `(A, B, C, D)` 四级权重：行内样式(1000) > ID(100) > 类/伪类/属性(10) > 元素/伪元素(1)。比较时从左到右逐级比较，高级别大于低级别的任意数量。权重相同时，后声明的覆盖先声明的。

### Q2：10 个类选择器能覆盖 1 个 ID 选择器吗？

**答：** **不能**。权重不是简单的十进制加法，而是逐级比较。ID 选择器在 B 级（第二位），类选择器在 C 级（第三位）。无论 C 级有多大，都永远小于 B 级的 1。所以 `(0,1,0,0)` 永远大于 `(0,0,10,0)`。

### Q3：`!important` 和行内样式谁的优先级高？

**答：** `!important` 更高。优先级排序：`!important` > 行内样式 > ID 选择器 > 类选择器 > 元素选择器。如果多条规则都有 `!important`，则再按权重比较；权重也相同则后声明的胜出。

### Q4：`inherit`、`initial`、`unset`、`revert` 的区别？

**答：**
- `inherit`：强制继承父元素的值
- `initial`：重置为 CSS 规范定义的初始值
- `unset`：如果属性可继承则等同 `inherit`，否则等同 `initial`
- `revert`：回退到浏览器默认样式（而非 CSS 规范初始值）

### Q5：如何提高选择器的可维护性？

**答：**
- 避免使用 `!important`（调试时可临时用，最终要移除）
- 避免嵌套过深（`.a .b .c .d .e` 权重过高，难以覆盖）
- 优先使用类选择器，少用 ID 选择器
- 利用 `:where()` 创建零权重的"基础样式"，方便后续覆盖

## 五、对比总结

| 优先级等级 | 权重 | 是否推荐使用 |
|-----------|------|------------|
| `!important` | 无限大 | ❌ 尽量避免 |
| 行内样式 `style=""` | (1,0,0,0) | ❌ 避免混用 |
| `#id` | (0,1,0,0) | ⚠️ 少用 |
| `.class` / `:hover` / `[attr]` | (0,0,1,0) | ✅ **推荐** |
| `div` / `::before` | (0,0,0,1) | ✅ 适当使用 |
| `*` / `:where()` | (0,0,0,0) | ✅ 重置/基础样式 |

---

## 六、注意事项

1. **不要用 ID 选择器做样式**：ID 权重太高，难以覆盖，且无法复用
2. **BEM 命名法降低嵌套**：`.block__element--modifier` 用单层类选择器替代深层嵌套
3. **`!important` 是最后手段**：它打破了正常的优先级机制，使代码难以维护
4. **`:where()` 零权重**：非常适合用于 CSS Reset 或第三方组件的"默认样式"
5. **CSS 层叠层（@layer）**：CSS 新特性，可以在不同"层"中组织样式，层内的优先级独立于选择器权重
