# 定位（Position）

> 官方文档：[MDN - position](https://developer.mozilla.org/zh-CN/docs/Web/CSS/position)

---

## 一、核心概念

CSS `position` 属性决定元素的**定位方式**——是遵循正常文档流，还是脱离文档流进行自由定位。

> 通俗类比：正常文档流像排队，`relative` 是"原地稍微挪一下"，`absolute` 是"脱离队伍站到指定位置"，`fixed` 是"钉在屏幕上不动"，`sticky` 是"滚动到一定位置就钉住"。

---

## 二、五种定位值对比

| 值 | 是否脱离文档流 | 参照物 | 常用场景 |
|----|-------------|--------|---------|
| `static`（默认） | ❌ 否 | 无（正常文档流） | 重置定位 |
| `relative` | ❌ 否 | **自身原始位置** | 微调位置、为 absolute 提供参照 |
| `absolute` | ✅ 是 | **最近的已定位祖先** | 弹窗、下拉菜单、徽标 |
| `fixed` | ✅ 是 | **视口（viewport）** | 固定导航栏、回到顶部 |
| `sticky` | ❌ 否（滚动到阈值后固定） | **最近的滚动祖先** | 吸顶导航、表头固定 |

---

## 三、逐一详解

### 3.1 static（静态定位）

默认值，元素正常排列，`top`/`right`/`bottom`/`left`/`z-index` **无效**。

```css
.element {
  position: static;
  top: 100px; /* 无效，不会移动 */
}
```

### 3.2 relative（相对定位）

元素仍在文档流中占位，但可以通过 `top`/`left` 等偏移。**原始位置保留，其他元素不会填补空位**。

```css
.box {
  position: relative;
  top: 20px;  /* 从原始位置向下移动 20px */
  left: 30px; /* 从原始位置向右移动 30px */
}
```

```
正常文档流：          relative 偏移后：
┌───┐ ┌───┐         ┌───┐     ┌───┐
│ A │ │ B │         │ A │     │ B │ ← 移动了，但原位置仍占空间
└───┘ └───┘         └───┘     └───┘
```

**核心用途**：作为 `absolute` 子元素的定位参照物。

```css
.parent {
  position: relative; /* 作为参照 */
}

.child {
  position: absolute;
  top: 10px;
  right: 10px; /* 相对于 .parent 定位 */
}
```

### 3.3 absolute（绝对定位）

元素**脱离文档流**，相对于**最近的已定位祖先（非 static）** 定位。如果没有已定位祖先，则相对于 `初始包含块`（通常是 `<html>`）。

```css
.container {
  position: relative; /* 提供定位参照 */
  width: 300px;
  height: 200px;
}

.badge {
  position: absolute;
  top: -5px;
  right: -5px; /* 相对于 .container 定位 */
}
```

```
┌─ .container (relative) ────────────────┐
│                              ┌──────┐  │
│                              │badge │  │ ← 绝对定位，相对 .container
│                              └──────┘  │
│  正常内容...                           │
└────────────────────────────────────────┘
```

### 3.4 fixed（固定定位）

元素**脱离文档流**，相对于**视口**定位，滚动页面时位置不变。

```css
.back-to-top {
  position: fixed;
  bottom: 30px;
  right: 30px; /* 始终固定在屏幕右下角 */
  z-index: 999;
}

.navbar {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%; /* 固定在页面顶部 */
}
```

> **注意**：如果 fixed 元素的祖先设置了 `transform`、`perspective` 或 `filter`，fixed 的参照物会变为该祖先，而非视口！

### 3.5 sticky（粘性定位）

混合定位：在滚动到阈值前表现为 `relative`，到达阈值后表现为 `fixed`。

```css
.header {
  position: sticky;
  top: 0; /* 滚动到顶部时固定 */
  z-index: 100;
}
```

```
正常状态（relative）          滚动后（fixed）
┌──────────────┐             ┌──────────────┐
│   Header     │             │   Header     │ ← 固定在顶部
├──────────────┤             ├──────────────┤
│   Content    │             │   Content    │
│              │             │              │
│              │             │              │
└──────────────┘             └──────────────┘
```

**sticky 失效的常见原因**：

```css
/* ❌ 父元素 overflow 不是 visible */
.parent {
  overflow: hidden; /* sticky 会失效！ */
  overflow: auto;   /* 同样会限制 sticky */
}

/* ❌ 父元素高度等于 sticky 元素高度 */
.parent {
  height: 50px;
}
.header {
  position: sticky;
  top: 0;
  height: 50px; /* 没有空间"粘住" */
}

/* ✅ 正确使用 */
.parent {
  overflow: visible; /* 默认值，不用设置 */
  /* 父元素高度要大于 sticky 元素 */
}
```

## 四、包含块（Containing Block）

定位元素的参照物不是简单的"父元素"，而是**包含块**：

| 定位方式 | 包含块 |
|---------|--------|
| `static` / `relative` | 最近的块级容器祖先 |
| `absolute` | 最近的 `position` 不是 `static` 的祖先 |
| `fixed` | 视口（除非祖先有 `transform` 等属性） |
| `sticky` | 最近的滚动祖先 |

```css
/* 典型包含块查找过程 */
html                          ← 查找已定位祖先
  └── body                    ← 查找已定位祖先
       └── .wrapper           ← 查找已定位祖先
            └── .parent       ← position: relative ✅ 找到！
                 └── .child   ← position: absolute（参照 .parent）
```

## 五、层叠上下文与 z-index

### 5.1 什么是层叠上下文

层叠上下文是一个**三维概念**——当元素重叠时，浏览器通过层叠上下文决定**谁在上面**。

```
                   屏幕方向（用户视角）
                      ↓
    ┌─────────────────────────────────┐
    │       层叠上下文（根）           │  ← z-index 最底层
    │   ┌─────────────────────────┐   │
    │   │   层叠上下文 A           │   │
    │   │   ┌─────────────────┐   │   │
    │   │   │ 层叠上下文 B     │   │   │  ← z-index 最高
    │   │   └─────────────────┘   │   │
    │   └─────────────────────────┘   │
    └─────────────────────────────────┘
```

### 5.2 什么会创建层叠上下文

| 属性 | 创建条件 |
|------|---------|
| `position` | 非 `static` + `z-index` 非 `auto` |
| `display: flex/grid` | 子项 `z-index` 非 `auto` |
| `opacity` | 值小于 1 |
| `transform` | 值不是 `none` |
| `filter` | 值不是 `none` |
| `will-change` | 指定了上述属性 |
| `isolation` | `isolate` |

### 5.3 z-index 的陷阱

```css
/* z-index 只在同一个层叠上下文内比较 */
.parent-a {
  position: relative;
  z-index: 1; /* 整个 parent-a 的层叠上下文 */
}
.parent-b {
  position: relative;
  z-index: 2; /* parent-b 在 parent-a 之上 */
}

/* ❌ 无论 .child-a 的 z-index 多大，都无法覆盖 .parent-b */
.child-a {
  position: absolute;
  z-index: 9999; /* 只在 parent-a 的层叠上下文内有效 */
}
```

```
┌─ parent-a (z-index: 1) ──────┐   ┌─ parent-b (z-index: 2) ──┐
│  ┌─ child-a (z-index:9999)─┐ │   │                           │
│  │ z-index 9999 只在       │ │   │ ← 仍然在 child-a 之上     │
│  │ parent-a 内部有效       │ │   │                           │
│  └─────────────────────────┘ │   └───────────────────────────┘
└──────────────────────────────┘
```

### 5.4 层叠顺序（从底到顶）

```
1. 层叠上下文的背景和边框
2. z-index 为负值的已定位元素
3. 常规文档流中的块级元素
4. 浮动元素
5. 常规文档流中的行内元素
6. z-index: 0 / auto 的已定位元素
7. z-index 为正值的已定位元素
```

## 六、常见面试题

### Q1：absolute 和 relative 的区别？

**答：**
- `relative` 不脱离文档流，相对**自身原始位置**偏移，原位置保留。
- `absolute` 脱离文档流，相对**最近的已定位祖先**定位，原位置不保留。
- `relative` 常作为 `absolute` 的定位参照容器。

### Q2：absolute 一定是相对父元素定位吗？

**答：** 不是。`absolute` 相对的是**最近的 `position` 不是 `static` 的祖先元素**。如果没有，则相对初始包含块（通常是 `<html>`）。所以使用 `absolute` 时，通常要在父元素上设置 `position: relative`。

### Q3：为什么 z-index 有时不生效？

**答：** `z-index` 只对 `position` 不是 `static` 的元素有效。另外，`z-index` 只在**同一个层叠上下文**内比较——如果父元素的 z-index 较低，子元素 z-index 再大也无法超越父元素层叠上下文之外的元素。

### Q4：sticky 定位有什么限制？

**答：**
1. 必须指定 `top`/`bottom`/`left`/`right` 中的至少一个
2. 父元素不能设置 `overflow: hidden/auto/scroll`
3. 父元素高度必须大于 sticky 元素高度，否则没有空间"粘住"
4. sticky 元素只在父元素范围内生效，滚出父元素后会随父元素一起滚走

### Q5：fixed 定位在什么情况下会失效？

**答：** 当 `fixed` 元素的祖先元素设置了 `transform`、`perspective`、`filter` 或 `will-change: transform` 时，`fixed` 的参照物会从视口变为该祖先元素，导致"固定"效果失效。

## 七、注意事项

1. **absolute 元素设置宽高 100%**：是相对于包含块的 content 区域，不包含 padding
2. **fixed 元素的性能**：固定定位的元素会创建独立的合成层，不跟随页面重绘，滚动性能好
3. **移动端 fixed 问题**：iOS 虚拟键盘弹出时，fixed 元素可能错位，需用其他方案替代
4. **sticky 兼容性**：现代浏览器均已支持，如需兼容 IE 则不可用
