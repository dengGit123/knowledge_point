# 核心概念

> 官方文档：[Webpack 核心概念](https://webpack.docschina.org/concepts/)

---

## 一、Webpack 是什么

Webpack 是一个**静态模块打包工具**（Static Module Bundler），它将项目中的各种资源（JS、CSS、图片、字体等）视为模块，根据依赖关系进行打包，生成适合浏览器运行的静态文件。

```
┌─ Webpack 打包流程 ────────────────────────────────────────────┐
│                                                               │
│  源代码（模块）               Webpack               打包产物    │
│                                                               │
│  app.js ──┐                                             dist/ │
│  utils.js──┤                                            ├── main.3a4b.js
│  style.css─┤──→  依赖分析 + 编译转换 + 代码分割 ──→       ├── vendor.c5d6.js
│  logo.png──┤                                            ├── main.1a2b.css
│  data.json─┘                                            └── logo.7e8f.png
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

> **通俗理解**：Webpack 就像一个"快递分拣中心"——把来自不同地方（各种类型文件）的包裹收集起来，按规则分类处理，最终打包成可以直接投递（浏览器运行）的形式。

## 二、核心概念

### 1. Entry（入口）

入口是 Webpack 开始分析依赖的**起点**，告诉 Webpack 从哪个文件开始打包。

```javascript
// webpack.config.js
module.exports = {
  // 单入口
  entry: './src/main.js',

  // 多入口（对象形式）
  entry: {
    app: './src/app.js',
    admin: './src/admin.js',
    vendor: './src/vendor.js'
  }
}
```

| 形式 | 说明 | 适用场景 |
|------|------|---------|
| 字符串 | 单入口 | 单页应用（SPA） |
| 数组 | 多入口合并为一个 chunk | 第三方库前置 |
| 对象 | 多入口多 chunk | 多页应用（MPA） |

### 2. Output（输出）

输出告诉 Webpack 打包后的文件**放在哪里、叫什么名字**。

```javascript
module.exports = {
  output: {
    path: path.resolve(__dirname, 'dist'),     // 输出目录
    filename: '[name].[contenthash:8].js',     // 文件名模板
    chunkFilename: '[name].[contenthash:8].chunk.js', // 异步 chunk 文件名
    clean: true,                                // 打包前清空 dist
    publicPath: '/',                            // 公共路径前缀
    assetModuleFilename: 'assets/[hash][ext][query]' // 静态资源输出路径
  }
}
```

**filename 占位符**：

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `[name]` | 入口名称 | `app` |
| `[hash]` | 项目级 hash | `3a4b5c6d` |
| `[chunkhash]` | chunk 级 hash | `e7f8g9h0` |
| `[contenthash]` | 文件内容 hash（推荐） | `1a2b3c4d` |
| `[ext]` | 文件扩展名 | `.js` |

> **推荐使用 `[contenthash]`**：文件内容不变则 hash 不变，利于浏览器缓存。

### 3. Loader（加载器）

Webpack 默认只理解 **JavaScript 和 JSON** 文件。Loader 让 Webpack 能够处理**其他类型**的文件。

```
文件处理流程：

源文件（.css/.vue/.ts） → Loader 转换 → JavaScript 模块 → Webpack 打包

例如：
style.css → css-loader → style-loader → 注入到 JS 中的样式代码
```

```javascript
module.exports = {
  module: {
    rules: [
      // 处理 CSS
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader', 'postcss-loader']
      },
      // 处理 Less
      {
        test: /\.less$/,
        use: ['style-loader', 'css-loader', 'postcss-loader', 'less-loader']
      },
      // 处理图片
      {
        test: /\.(png|jpe?g|gif|svg)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: {
            maxSize: 8 * 1024  // 8KB 以下转 base64
          }
        }
      },
      // 处理字体
      {
        test: /\.(woff2?|eot|ttf|otf)$/,
        type: 'asset/resource'
      },
      // 处理 JS/JSX
      {
        test: /\.jsx?$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      }
    ]
  }
}
```

### 4. Plugin（插件）

Plugin 用于执行**范围更广**的任务，如打包优化、资源管理、环境变量注入等。

```javascript
const HtmlWebpackPlugin = require('html-webpack-plugin')
const { DefinePlugin } = require('webpack')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')

module.exports = {
  plugins: [
    // 生成 HTML 文件并自动注入打包资源
    new HtmlWebpackPlugin({
      template: './public/index.html',
      title: 'My App',
      minify: true
    }),

    // 定义环境变量
    new DefinePlugin({
      'process.env.NODE_ENV': JSON.stringify('production'),
      __VUE_OPTIONS_API__: 'true',
      __VUE_PROD_DEVTOOLS__: 'false'
    }),

    // 提取 CSS 为独立文件
    new MiniCssExtractPlugin({
      filename: 'css/[name].[contenthash:8].css'
    })
  ]
}
```

### 5. Mode（模式）

```javascript
module.exports = {
  mode: 'development'  // 或 'production' | 'none'
}
```

| 特性 | development | production |
|------|------------|------------|
| `process.env.NODE_ENV` | `'development'` | `'production'` |
| Source Map | 详细的 eval source map | 精简的 source map |
| 代码压缩 | ❌ 不压缩 | ✅ Terser 压缩 |
| Tree Shaking | ❌ 不启用 | ✅ 自动启用 |
| 优化 | 不优化 | 自动优化（Scope Hoisting 等） |
| 调试信息 | 保留 | 移除 |
| 构建速度 | 快 | 较慢 |

### 6. Resolve（模块解析）

```javascript
module.exports = {
  resolve: {
    // 自动补全扩展名
    extensions: ['.js', '.jsx', '.ts', '.vue', '.json'],

    // 路径别名
    alias: {
      '@': path.resolve(__dirname, 'src'),
      '@components': path.resolve(__dirname, 'src/components'),
      '@utils': path.resolve(__dirname, 'src/utils')
    },

    // 模块搜索目录
    modules: ['node_modules', path.resolve(__dirname, 'src')],

    // 包入口字段（优先级从左到右）
    mainFields: ['browser', 'module', 'main']
  }
}
```

## 三、完整配置示例

```javascript
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')
const { VueLoaderPlugin } = require('vue-loader')

module.exports = {
  mode: 'production',

  entry: './src/main.js',

  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'js/[name].[contenthash:8].js',
    chunkFilename: 'js/[name].[contenthash:8].chunk.js',
    clean: true,
    publicPath: '/'
  },

  resolve: {
    extensions: ['.js', '.jsx', '.ts', '.vue', '.json'],
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  },

  module: {
    rules: [
      {
        test: /\.vue$/,
        loader: 'vue-loader'
      },
      {
        test: /\.jsx?$/,
        exclude: /node_modules/,
        use: 'babel-loader'
      },
      {
        test: /\.css$/,
        use: [MiniCssExtractPlugin.loader, 'css-loader', 'postcss-loader']
      },
      {
        test: /\.less$/,
        use: [MiniCssExtractPlugin.loader, 'css-loader', 'postcss-loader', 'less-loader']
      },
      {
        test: /\.(png|jpe?g|gif|svg)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: { maxSize: 8 * 1024 }
        },
        generator: {
          filename: 'img/[hash][ext][query]'
        }
      },
      {
        test: /\.(woff2?|eot|ttf|otf)$/,
        type: 'asset/resource',
        generator: {
          filename: 'fonts/[hash][ext][query]'
        }
      }
    ]
  },

  plugins: [
    new VueLoaderPlugin(),
    new HtmlWebpackPlugin({
      template: './public/index.html',
      title: 'My App'
    }),
    new MiniCssExtractPlugin({
      filename: 'css/[name].[contenthash:8].css'
    })
  ],

  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendor',
          chunks: 'all'
        }
      }
    }
  },

  // 开发服务器
  devServer: {
    hot: true,
    port: 3000,
    open: true,
    historyApiFallback: true,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: path => path.replace(/^\/api/, '')
      }
    }
  },

  // Source Map 配置
  devtool: 'source-map',

  // 性能提示
  performance: {
    hints: 'warning',
    maxAssetSize: 300000,
    maxEntrypointSize: 500000
  }
}
```

## 四、Webpack 5 的重要变化

| 特性 | 说明 |
|------|------|
| **资源模块（Asset Modules）** | 内置静态资源处理，不再需要 `file-loader`、`url-loader` |
| **持久化缓存** | `cache: { type: 'filesystem' }` 大幅提升构建速度 |
| **模块联邦** | 支持跨应用动态加载模块（微前端） |
| **更好的 Tree Shaking** | 支持嵌套的 Tree Shaking 和内部模块 Tree Shaking |
| **Node.js Polyfill 移除** | 不再自动注入 Node.js 核心模块的 polyfill |
| **Top Level Await** | 支持模块顶层 `await` |

```javascript
// Webpack 5 资源模块（替代 file-loader / url-loader）
module.exports = {
  module: {
    rules: [
      {
        test: /\.png$/,
        type: 'asset/inline'       // 替代 url-loader（base64 内联）
        // type: 'asset/resource'  // 替代 file-loader（输出文件）
        // type: 'asset'           // 自动选择（默认 8KB 以下 inline）
        // type: 'asset/source'    // 替代 raw-loader（导出源码）
      }
    ]
  }
}
```

## 五、面试常见问题

### Q1：Webpack 的核心概念有哪些？

五大核心概念：
1. **Entry**：入口，Webpack 开始打包的起点
2. **Output**：输出，打包后文件的存放规则
3. **Loader**：加载器，将非 JS 文件转换为 Webpack 能处理的模块
4. **Plugin**：插件，扩展 Webpack 功能（压缩、HTML 生成、环境变量等）
5. **Mode**：模式，development / production，影响默认优化策略

### Q2：Loader 和 Plugin 的区别？

| 维度 | Loader | Plugin |
|------|--------|--------|
| 本质 | 文件**转换器** | 功能**扩展器** |
| 作用 | 将非 JS 文件转为 JS 模块 | 执行更广泛的构建任务 |
| 工作时机 | **模块加载时**逐个处理 | **整个构建生命周期**中监听钩子 |
| 配置位置 | `module.rules` | `plugins` 数组 |
| 输入输出 | 接收源文件，返回转换结果 | 通过钩子访问 compiler/compilation |

### Q3：hash、chunkhash、contenthash 的区别？

- **`[hash]`**：项目级 hash，任意文件变化都会改变（缓存利用率最低）
- **`[chunkhash]`**：chunk 级 hash，同一 chunk 中任一模块变化都会改变
- **`[contenthash]`**：文件内容级 hash，只有文件自身内容变化才改变（推荐 ✅）

### Q4：Webpack 的构建流程是怎样的？

```
初始化（读取配置、创建 Compiler）
  → 编译（从 Entry 出发，调用 Loader 编译模块，构建依赖图）
    → 输出（根据依赖图组装 chunk，生成最终文件）
```

详细流程见 [构建流程与AST](./构建流程与AST.md)。
