# 模块联邦

> 官方文档：[Module Federation](https://webpack.js.org/concepts/module-federation/)

> 深入理解：[Module Federation 原理](https://module-federation.github.io/)

---

## 一、模块联邦是什么

Module Federation（模块联邦）是 Webpack 5 引入的重要特性，允许多个独立构建的应用在运行时**动态共享模块**。

```
┌─ 模块联邦架构 ────────────────────────────────────────────┐
│                                                            │
│  ┌─ App A（Host）───┐     ┌─ App B（Remote）───┐          │
│  │                    │     │                     │          │
│  │  本地模块          │     │  本地模块           │          │
│  │  ├── Header       │     │  ├── ProductList    │          │
│  │  ├── Footer       │     │  ├── Cart           │          │
│  │  └── ...          │     │  └── ...            │          │
│  │                    │     │                     │          │
│  │  远程加载 ←────────┼─────┤  exposes:           │          │
│  │  ProductList ──────┼─────┤  ├── ProductList    │          │
│  │  Cart ─────────────┼─────┤  └── Cart           │          │
│  │                    │     │                     │          │
│  └────────────────────┘     └────────────────────┘          │
│                                                            │
│  运行时：App A 按需从 App B 动态加载远程模块                  │
│  各应用独立开发、独立部署、独立运行                           │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

> **通俗理解**：模块联邦就像"共享食堂"——每个公司（应用）有自己的厨房（模块），但可以开放某些菜品（exposes）给其他公司享用，也可以去其他公司的食堂点餐（remotes）。

---

## 二、核心概念

| 概念 | 说明 |
|------|------|
| **Host（消费者）** | 引用远程模块的应用 |
| **Remote（提供者）** | 暴露模块供其他应用使用的应用 |
| **Shared** | 应用间共享的依赖（避免重复加载） |
| **exposes** | Remote 声明哪些模块可以被外部使用 |
| **remotes** | Host 声明从哪里加载远程模块 |

> 一个应用可以**同时**是 Host 和 Remote。

---

## 三、配置示例

### Remote（提供模块的应用）

```javascript
// webpack.config.js（App B —— Remote）
const ModuleFederationPlugin = require('webpack').container.ModuleFederationPlugin

module.exports = {
  plugins: [
    new ModuleFederationPlugin({
      name: 'appB',                    // 应用名称（全局唯一）
      filename: 'remoteEntry.js',      // 远程入口文件名

      // 暴露给其他应用使用的模块
      exposes: {
        './ProductList': './src/components/ProductList.vue',
        './Cart': './src/components/Cart.vue',
        './utils': './src/utils/index.js'
      },

      // 共享依赖（避免重复加载）
      shared: {
        vue: { singleton: true, requiredVersion: '^3.0.0' },
        'vue-router': { singleton: true },
        pinia: { singleton: true }
      }
    })
  ]
}
```

### Host（消费模块的应用）

```javascript
// webpack.config.js（App A —— Host）
const ModuleFederationPlugin = require('webpack').container.ModuleFederationPlugin

module.exports = {
  plugins: [
    new ModuleFederationPlugin({
      name: 'appA',

      // 声明远程模块来源
      remotes: {
        appB: 'appB@http://localhost:3001/remoteEntry.js'
        // 格式: '内部名称@远程入口URL'
      },

      // 共享依赖
      shared: {
        vue: { singleton: true, requiredVersion: '^3.0.0' },
        'vue-router': { singleton: true },
        pinia: { singleton: true }
      }
    })
  ]
}
```

### 使用远程模块

```vue
<!-- App A 中使用 App B 暴露的组件 -->
<script setup>
import { defineAsyncComponent } from 'vue'

// 方式一：直接 import
import ProductList from 'appB/ProductList'

// 方式二：异步加载（推荐）
const Cart = defineAsyncComponent(() => import('appB/Cart'))

// 方式三：动态导入
const loadUtils = () => import('appB/utils')
</script>

<template>
  <ProductList />
  <Cart />
</template>
```

## 四、Shared 配置详解

`shared` 用于避免多个应用重复加载同一依赖（如 Vue、Vue Router）。

```javascript
shared: {
  vue: {
    singleton: true,          // 只加载一个实例（避免多版本冲突）
    requiredVersion: '^3.0.0', // 要求的版本范围
    eager: false,             // 是否同步加载（默认异步）
    strictVersion: true       // 版本不匹配时是否报错
  },
  'vue-router': {
    singleton: true
  },
  lodash: {
    singleton: false          // 可以加载多个版本
  }
}

// 简写形式
shared: ['vue', 'vue-router', 'pinia', 'axios']
```

| 选项 | 说明 |
|------|------|
| `singleton` | 只加载一个版本（框架类库必须设为 true） |
| `requiredVersion` | 要求的版本范围 |
| `eager` | 是否在入口文件中同步加载（默认 false） |
| `strictVersion` | 版本不匹配时是否抛出错误 |
| `shareScope` | 共享作用域名称（默认 `default`） |
| `shareConfig` | 共享配置 |

> **注意**：Vue、React 等框架**必须设置 `singleton: true`**，否则会出现多实例问题。

## 五、微前端架构

模块联邦是实现**微前端**的一种方案，多个前端应用可以独立开发、独立部署，运行时集成。

```
┌─ 微前端架构 ──────────────────────────────────────────┐
│                                                        │
│  ┌─ 主应用（Shell）─────────────────────────────┐      │
│  │                                              │      │
│  │  ┌─ 导航栏 ──────────────────────────────┐   │      │
│  │  │  [首页] [商品] [用户] [订单]           │   │      │
│  │  └──────────────────────────────────────┘   │      │
│  │                                              │      │
│  │  ┌─ 内容区 ─────────────────────────────┐   │      │
│  │  │                                       │   │      │
│  │  │  ← 动态加载远程模块                    │   │      │
│  │  │                                       │   │      │
│  │  └──────────────────────────────────────┘   │      │
│  │                                              │      │
│  └──────────────────────────────────────────────┘      │
│                                                        │
│  Remote: 商品应用 → exposes: ProductList, Detail       │
│  Remote: 用户应用 → exposes: Profile, Settings         │
│  Remote: 订单应用 → exposes: OrderList, OrderDetail    │
│                                                        │
│  各应用独立部署在不同域名/端口                           │
│  主应用按需加载对应模块                                  │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### 微前端方案对比

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **Module Federation** | Webpack 5 运行时共享模块 | 原生支持、粒度细、共享依赖 | 依赖 Webpack 5 |
| qiankun | 基于 single-spa，JS 沙箱 | 框架无关、成熟稳定 | 样式隔离有风险 |
| iframe | 嵌入独立页面 | 完全隔离 | 体验差、通信困难 |
| Web Components | 自定义元素 | 标准化 | 兼容性、生态不成熟 |

## 六、加载流程

```
① 浏览器加载主应用 HTML
     │
     ▼
② 加载主应用 JS（包含远程模块映射）
     │
     ▼
③ 遇到 import('appB/ProductList')
     │
     ▼
④ 加载 App B 的 remoteEntry.js（远程入口，很小）
     │
     ▼
⑤ remoteEntry.js 告知 ProductList 的 chunk 文件路径
     │
     ▼
⑥ 加载 ProductList 的 chunk 文件
     │
     ▼
⑦ 检查 shared 依赖（vue 等）
     │
     ├── 已加载 → 复用 ✅
     └── 未加载 → 从 Remote 或 CDN 加载
     │
     ▼
⑧ 执行 ProductList 组件代码
```

## 七、面试常见问题

### Q1：模块联邦解决了什么问题？

1. **代码共享**：多个应用可以共享组件和模块，避免重复开发
2. **独立部署**：各应用独立构建和部署，互不影响
3. **运行时集成**：在运行时动态加载远程模块，不需要重新构建主应用
4. **共享依赖**：通过 `shared` 配置避免重复加载 Vue 等大型依赖
5. **微前端基础**：为微前端架构提供了底层支持

### Q2：模块联邦和 npm 包共享有什么区别？

| 维度 | npm 包 | 模块联邦 |
|------|--------|---------|
| 集成时机 | 编译时 | **运行时** |
| 更新方式 | 修改版本号 → 重新构建 | 远程应用部署即生效 |
| 独立部署 | 不支持 | ✅ 支持 |
| 粒度 | 整个包 | **单个模块/组件** |
| 适用场景 | 通用工具库 | 跨应用共享业务组件 |

### Q3：shared 的 singleton 是什么意思？

`singleton: true` 确保整个页面只加载**一个实例**。对于 Vue、React 等框架，多实例会导致不可预测的错误（如组件无法注册、状态不共享等）。因此框架类依赖必须设置 `singleton: true`。

### Q4：模块联邦的 remoteEntry.js 是什么？

`remoteEntry.js` 是 Remote 应用的**入口清单文件**，它非常小（几 KB），包含了：
- Remote 暴露的模块列表
- 各模块 chunk 文件的映射关系
- 共享依赖的版本信息

Host 先加载 `remoteEntry.js`，再根据其中的映射关系按需加载具体的模块 chunk。
