## 二、构建速度优化

### 1. 缩小构建范围

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        // ✅ 排除 node_modules，只处理自己的代码
        exclude: /node_modules/,
        use: 'babel-loader'
      },
      {
        test: /\.js$/,
        // ✅ 或者用 include 指定范围（更精确）
        include: path.resolve(__dirname, 'src'),
        use: 'babel-loader'
      }
    ]
  },

  resolve: {
    // ✅ 减少文件搜索范围
    modules: [path.resolve(__dirname, 'node_modules')],
    extensions: ['.js', '.vue', '.json'],  // 不要太多
    // ✅ 直接指定文件路径，避免目录查找
    alias: {
      '@': path.resolve(__dirname, 'src')
    },
    // ✅ 减少入口字段查找
    mainFields: ['main'],
    // ✅ 指定 package.json 中的模块入口
    descriptionFiles: ['package.json']
  }
}
```

### 2. 持久化缓存（Webpack 5）

```javascript
module.exports = {
  cache: {
    type: 'filesystem',           // 使用文件系统缓存
    buildDependencies: {
      config: [__filename]        // 配置文件变化时使缓存失效
    },
    cacheDirectory: path.resolve(__dirname, '.webpack_cache'),
    version: '1.0'                // 版本号变化时使缓存失效
  }
}
```

**效果**：二次构建速度可提升 **60%~90%**。

### 3. 多线程编译

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        include: path.resolve(__dirname, 'src'),
        use: [
          // thread-loader 放在最前面
          {
            loader: 'thread-loader',
            options: {
              workers: 4,             // worker 线程数
              poolTimeout: 2000       // 进程池超时
            }
          },
          'babel-loader'
        ]
      }
    ]
  }
}
```

> **注意**：`thread-loader` 有启动开销（约 600ms），适合项目较大时使用。小项目可能反而变慢。

### 4. DLL 预编译（可选）

将不常变化的第三方库提前编译，减少每次构建的处理量：

```javascript
// webpack.dll.js（单独执行一次）
module.exports = {
  entry: {
    vendor: ['vue', 'vue-router', 'pinia', 'axios']
  },
  output: {
    path: path.resolve(__dirname, 'dll'),
    filename: '[name].dll.js',
    library: '[name]_library'
  },
  plugins: [
    new webpack.DllPlugin({
      name: '[name]_library',
      path: path.resolve(__dirname, 'dll/[name]-manifest.json')
    })
  ]
}

// webpack.config.js（主配置引用 DLL）
constDllReferencePlugin = require('webpack').DllReferencePlugin
const AddAssetHtmlPlugin = require('add-asset-html-webpack-plugin')

module.exports = {
  plugins: [
    new DllReferencePlugin({
      manifest: require('./dll/vendor-manifest.json')
    }),
    new AddAssetHtmlPlugin({ filepath: './dll/*.dll.js' })
  ]
}
```

> **注意**：Webpack 5 的持久化缓存已经能很好地解决这个问题，DLL 方案逐渐被淘汰。

### 5. 开发环境专属优化

```javascript
module.exports = {
  // 开发环境使用 eval-cheap-module-source-map（构建最快）
  devtool: 'eval-cheap-module-source-map',

  // 不生成完整 source map
  // 生产环境: 'source-map' 或 'hidden-source-map'

  optimization: {
    // 开发环境不压缩
    minimize: false
  },

  devServer: {
    hot: true,           // 热更新
    compress: true,      // gzip 压缩
    client: {
      overlay: false     // 关闭全屏错误覆盖
    }
  }
}
```

## 三、产物体积优化

### 1. Tree Shaking

Tree Shaking 基于 ES Module 的**静态分析**，移除未被使用的代码。

```javascript
// math.js
export function add(a, b) { return a + b }     // 被使用
export function subtract(a, b) { return a - b } // 未使用 → 被 Tree Shaking 移除
export const PI = 3.14                          // 未使用 → 被移除

// main.js
import { add } from './math.js'
console.log(add(1, 2))

// 打包结果中只包含 add 函数
```

**使用条件**：

| 条件 | 说明 |
|------|------|
| 使用 ES Module | `import/export`（不能是 CommonJS 的 `require`） |
| `mode: 'production'` | 生产模式自动启用 |
| `optimization.usedExports: true` | 标记未使用的导出 |
| `package.json` 中 `sideEffects` | 告诉 Webpack 哪些文件可以安全 Tree Shake |

```javascript
// package.json
{
  "sideEffects": false,  // 所有文件都没有副作用（可以安全 Tree Shake）
  // 或者指定有副作用的文件：
  // "sideEffects": ["*.css", "*.less", "./src/polyfills.js"]
}
```

### 2. 代码分割（Code Splitting）

```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',               // 对所有模块进行分割
      minSize: 20000,              // 最小分割大小（20KB）
      minChunks: 1,                // 最少被引用次数
      maxAsyncRequests: 30,        // 异步加载最大并行请求数
      maxInitialRequests: 30,      // 入口最大并行请求数
      cacheGroups: {
        // 第三方库单独打包
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendor',
          chunks: 'all',
          priority: 10
        },
        // 公共模块提取
        common: {
          minChunks: 2,
          name: 'common',
          chunks: 'all',
          priority: 5,
          reuseExistingChunk: true
        },
        // CSS 单独提取
        styles: {
          test: /\.(css|less)$/,
          name: 'styles',
          chunks: 'all',
          enforce: true
        }
      }
    },

    // 运行时代码单独提取
    runtimeChunk: 'single'
  }
}
```

### 3. 动态导入（懒加载）

```javascript
// 路由懒加载
const routes = [
  {
    path: '/about',
    component: () => import('./views/About.vue')   // 懒加载
  },
  {
    path: '/dashboard',
    component: () => import(/* webpackChunkName: "admin" */ './views/Dashboard.vue')
  }
]

// 条件加载
button.addEventListener('click', async () => {
  const { Chart } = await import(/* webpackChunkName: "chart" */ './chart.js')
  new Chart(canvas, data)
})

// 预加载（空闲时提前加载）
const Admin = () => import(
  /* webpackPrefetch: true */    // 浏览器空闲时预先加载
  /* webpackChunkName: "admin" */
  './views/Admin.vue'
)
```

| 魔法注释 | 说明 |
|---------|------|
| `webpackChunkName` | 自定义 chunk 名称 |
| `webpackPrefetch` | 空闲时预加载（优先级低） |
| `webpackPreload` | 与父 chunk 并行加载（优先级高） |

### 4. 代码压缩

```javascript
module.exports = {
  optimization: {
    minimize: true,
    minimizer: [
      // JS 压缩
      new TerserPlugin({
        parallel: true,
        extractComments: false,
        terserOptions: {
          compress: {
            drop_console: true,
            drop_debugger: true,
            pure_funcs: ['console.log']  // 只移除 console.log
          }
        }
      }),
      // CSS 压缩
      new CssMinimizerPlugin()
    ]
  }
}
```

### 5. Gzip / Brotli 压缩

```javascript
const CompressionPlugin = require('compression-webpack-plugin')

module.exports = {
  plugins: [
    // Gzip
    new CompressionPlugin({
      algorithm: 'gzip',
      test: /\.(js|css|html|svg)$/,
      threshold: 10240,
      minRatio: 0.8
    }),
    // Brotli（压缩率更高）
    new CompressionPlugin({
      algorithm: 'brotliCompress',
      test: /\.(js|css|html|svg)$/,
      threshold: 10240,
      minRatio: 0.8
    })
  ]
}
```

### 7. 图片优化

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.(png|jpe?g|gif)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: { maxSize: 4 * 1024 }  // 4KB 以下 base64
        },
        generator: {
          filename: 'img/[hash:8][ext][query]'
        }
      },
      {
        test: /\.svg$/,
        oneOf: [
          {
            resourceQuery: /inline/,  // ?inline 内联为组件
            use: ['@svgr/webpack']
          },
          {
            type: 'asset/resource'    // 否则作为文件
          }
        ]
      }
    ]
  }
}
```

| 优化手段 | 优化方向 | 预期效果 |
|---------|---------|---------|
| 持久化缓存 | 构建速度 | 二次构建提速 60%~90% |

## 五、面试常见问题

### Q1：如何优化 Webpack 的构建速度？

1. **缩小构建范围**：`include/exclude`、减少 `extensions`、配置 `resolve.modules`
2. **利用缓存**：Webpack 5 `cache: { type: 'filesystem' }`
3. **多线程**：`thread-loader`
4. **开发模式优化**：`eval-cheap-module-source-map`、关闭压缩
5. **DLL**：预编译第三方库（Webpack 5 中逐渐被缓存取代）

### Q2：如何优化 Webpack 的打包体积？

1. **Tree Shaking**：确保 ES Module + `sideEffects` 配置
2. **代码分割**：`splitChunks` + 动态导入
3. **代码压缩**：TerserPlugin + CssMinimizerPlugin
4. **Gzip/Brotli**：服务器端或构建时压缩
5. **CDN externals**：大型库用 CDN 引入
6. **图片优化**：压缩、base64 内联、SVG 组件化

### Q3：Tree Shaking 的原理和使用条件？

Tree Shaking 基于 ES Module 的**静态结构**特性，在编译时就能分析出哪些导出未被使用。使用条件：
1. 使用 ES Module（`import/export`），不能用 CommonJS
2. `mode: 'production'`
3. 配置 `sideEffects`
4. 没有副作用代码（如修改全局变量、原型链等）

### Q4：splitChunks 的默认配置是什么？

```javascript
splitChunks: {
  chunks: 'async',     // 只分割异步加载的模块
  minSize: 20000,      // 最小 20KB
  minChunks: 1,
  maxAsyncRequests: 30,
  maxInitialRequests: 30,
  cacheGroups: {
    defaultVendors: {
      test: /[\\/]node_modules[\\/]/,
      priority: -10,
      reuseExistingChunk: true
    },
    default: {
      minChunks: 2,
      priority: -20,
      reuseExistingChunk: true
    }
  }
}
```

生产环境建议将 `chunks` 改为 `'all'`，同时分割同步和异步模块。
