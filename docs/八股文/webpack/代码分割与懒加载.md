# 代码分割与懒加载

> 官方文档：[Webpack 代码分割](https://webpack.docschina.org/guides/code-splitting/)

> 官方文档：[Lazy Loading](https://webpack.docschina.org/guides/lazy-loading/)

---

## 一、为什么需要代码分割

没有代码分割时，所有代码打包为一个 JS 文件，存在以下问题：

```
❌ 不分割（单文件）：
┌──────────────────────────────────────────────────┐
│  main.js（3MB）                                    │
│  ├── 业务代码（500KB）                              │
│  ├── vue/vuex/vue-router（1MB）                    │
│  ├── 工具库 lodash/axios（800KB）                   │
│  └── 其他页面代码（700KB）                          │
│                                                    │
│  问题：                                             │
│  1. 首屏加载慢（下载整个 3MB）                       │
│  2. 任何改动都要重新下载整个文件                     │
│  3. 无法利用浏览器缓存                              │
└──────────────────────────────────────────────────┘

✅ 代码分割后：
┌──────────────────────────────────────────────────┐
│  main.js（50KB）    ← 业务代码（经常变）            │
│  vendor.js（1MB）   ← 第三方库（很少变，长期缓存）   │
│  common.js（30KB）  ← 公共模块                      │
│  about.js（20KB）   ← About 页面（按需加载）        │
│  admin.js（100KB）  ← Admin 页面（按需加载）        │
│                                                    │
│  优势：                                             │
│  1. 首屏只加载 main + vendor + 当前页面             │
│  2. 业务代码变化不影响 vendor 缓存                  │
│  3. 非首屏页面按需加载                              │
└──────────────────────────────────────────────────┘
```

## 二、代码分割的三种方式

### 1. 入口分割（Entry Splitting）

最简单的方式，通过多入口实现：

```javascript
module.exports = {
  entry: {
    index: './src/pages/index/main.js',
    about: './src/pages/about/main.js',
    admin: './src/pages/admin/main.js'
  },
  output: {
    filename: '[name].[contenthash:8].js'
  }
}
```

**缺点**：如果多个入口引用了相同的模块，会被重复打包。

### 2. 动态导入（Dynamic Import）

使用 `import()` 语法实现**按需加载**：

```javascript
// 路由懒加载
const routes = [
  {
    path: '/home',
    component: () => import('./views/Home.vue')
  },
  {
    path: '/about',
    component: () => import('./views/About.vue')
  }
]

// 条件加载
async function loadEditor() {
  if (needEditor) {
    const { Editor } = await import('./components/Editor.vue')
    return Editor
  }
}

// 事件触发加载
document.getElementById('btn').addEventListener('click', async () => {
  const { default: Chart } = await import('./chart.js')
  new Chart(document.getElementById('canvas'), data)
})
```

**魔法注释**：

```javascript
import(
  /* webpackChunkName: "admin" */      // 自定义 chunk 名称
  /* webpackPrefetch: true */           // 空闲时预加载
  // /* webpackPreload: true */         // 与父 chunk 并行加载
  './views/Admin.vue'
)
```

| 魔法注释 | 说明 |
|---------|------|
| `webpackChunkName: "name"` | 自定义 chunk 文件名 |
| `webpackPrefetch: true` | 浏览器空闲时自动预加载（低优先级） |
| `webpackPreload: true` | 与父 chunk 并行加载（高优先级） |
| `webpackMode: "lazy"` | 懒加载模式 |
| `webpackMode: "eager"` | 不单独打包，包含在父 chunk 中 |

### 3. SplitChunks（自动分割）

通过 `optimization.splitChunks` 自动分析并分割模块：

```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',

      cacheGroups: {
        // 第三方库
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendor',
          chunks: 'all',
          priority: 10
        },

        // 公共业务模块
        common: {
          minChunks: 2,             // 被引用 2 次以上才提取
          name: 'common',
          chunks: 'all',
          priority: 5,
          reuseExistingChunk: true  // 复用已有 chunk
        },

        // 提取 CSS
        styles: {
          test: /\.(css|less)$/,
          name: 'styles',
          chunks: 'all',
          enforce: true
        }
      }
    },

    // 运行时代码单独提取
    runtimeChunk: {
      name: 'runtime'
    }
  }
}
```

### chunks 选项

| 值 | 说明 |
|-----|------|
| `'async'` | 只分割异步导入的模块（默认值） |
| `'initial'` | 只分割同步导入的模块 |
| `'all'` | 同时分割同步和异步模块（推荐 ✅） |

## 三、懒加载策略

### 路由懒加载

```javascript
const routes = [
  {
    path: '/',
    component: () => import(/* webpackChunkName: "home" */ './views/Home.vue')
  },
  {
    path: '/user',
    component: () => import(/* webpackChunkName: "user" */ './views/User.vue'),
    children: [
      {
        path: 'profile',
        component: () => import(/* webpackChunkName: "user-profile" */ './views/UserProfile.vue')
      }
    ]
  }
]
```

### 组件懒加载

```vue
<script setup>
import { defineAsyncComponent } from 'vue'

// 异步组件
const HeavyChart = defineAsyncComponent(() =>
  import(/* webpackChunkName: "chart" */ './components/HeavyChart.vue')
)

// 带选项的异步组件
const AsyncComp = defineAsyncComponent({
  loader: () => import('./components/Editor.vue'),
  loadingComponent: LoadingSpinner,
  errorComponent: ErrorDisplay,
  delay: 200,
  timeout: 10000
})
</script>

<template>
  <HeavyChart v-if="showChart" />
  <AsyncComp v-if="showEditor" />
</template>
```

### Prefetch vs Preload

```
Prefetch（预获取）：
  → 浏览器空闲时下载，优先级最低
  → 不影响当前页面加载
  → 适合：未来可能访问的页面

Preload（预加载）：
  → 与当前页面并行下载，优先级高
  → 当前页面一定会用到的资源
  → 适合：即将使用的关键资源

示例：
  首页 → Prefetch: 关于页、用户页
  首页 → Preload: 首页需要的大图表库
```

```javascript
// Prefetch（空闲时加载）
const About = () => import(
  /* webpackPrefetch: true */
  /* webpackChunkName: "about" */
  './views/About.vue'
)

// Preload（并行加载）
const Chart = () => import(
  /* webpackPreload: true */
  /* webpackChunkName: "chart" */
  './chart.js'
)
```

## 四、代码分割后的加载策略

```
页面加载流程（代码分割后）：

① 加载 HTML
     │
     ▼
② 加载 runtime.js（Webpack 运行时代码，很小）
     │
     ▼
③ 加载 main.js（业务代码）
     │
     ├── vendor.js（第三方库，并行加载）
     │
     ▼
④ 页面渲染完成
     │
     ├── Prefetch: about.js, user.js（空闲时悄悄加载）
     │
     ▼
⑤ 用户点击"关于"页面
     │
     ├── about.js 已缓存 → 直接使用 ✅
     └── about.js 未加载 → 动态加载
```

### 加载性能指标

| 指标 | 说明 | 优化方式 |
|------|------|---------|
| **FCP**（First Contentful Paint） | 首次内容绘制 | 代码分割减少首屏 JS |
| **TTI**（Time to Interactive） | 可交互时间 | 关键路径最小化 |
| **LCP**（Largest Contentful Paint） | 最大内容绘制 | 预加载关键资源 |
| **TBT**（Total Blocking Time） | 总阻塞时间 | 分割长任务 |

## 五、runtimeChunk 的作用

```javascript
module.exports = {
  optimization: {
    // 将 Webpack 运行时代码单独提取
    runtimeChunk: 'single'
    // 或
    runtimeChunk: {
      name: 'runtime'
    }
  }
}
```

**为什么需要**：

Webpack 在 JS 文件中维护模块 ID 和 chunk 映射关系。如果业务代码变化，可能导致模块 ID 变化，进而影响 `vendor` 的 hash，使浏览器缓存失效。将 runtime 单独提取后，业务代码变化不会影响 `vendor` 的 hash。

```
不提取 runtime：
  main.js 变化 → vendor.js hash 变化 → 浏览器重新下载 vendor ❌

提取 runtime：
  main.js 变化 → runtime.js hash 变化 → vendor.js hash 不变 ✅
```

## 六、面试常见问题

### Q1：代码分割有哪几种方式？

1. **入口分割**：多入口配置（适合多页应用）
2. **动态导入**：`import()` 按需加载（适合单页应用的路由和组件）
3. **SplitChunks**：自动分析模块依赖并分割（提取公共模块和第三方库）

### Q2：import() 和 import 有什么区别？

- `import`：静态导入，编译时分析，会被打包到主 bundle 中
- `import()`：动态导入，运行时执行，会生成独立的 chunk 文件

### Q3：Prefetch 和 Preload 的区别？

- **Prefetch**：浏览器**空闲时**下载，优先级低，适合未来可能需要的资源
- **Preload**：与当前页面**并行**下载，优先级高，适合当前页面一定需要的资源

### Q4：runtimeChunk 是什么？为什么要单独提取？

`runtimeChunk` 将 Webpack 的模块映射和加载逻辑单独提取为一个小文件。目的是防止业务代码变化导致第三方库 chunk 的 hash 改变，从而保持 `vendor` 文件的长期缓存。
