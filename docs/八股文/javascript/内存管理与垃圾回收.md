## 一、内存生命周期

无论使用什么编程语言，内存的生命周期都遵循同样的阶段：

```
┌──────────────────────────────────────────────────────┐
│                 内存生命周期                           │
│                                                       │
│  ① 分配内存（Allocate）                               │
│     │  → 为变量、函数、对象等分配所需的内存空间          │
│     ▼                                                 │
│  ② 使用内存（Use）                                    │
│     │  → 对内存进行读写操作                            │
│     ▼                                                 │
│  ③ 释放内存（Release）                                │
│        → 不再需要时释放内存，归还给系统                 │
│                                                       │
└──────────────────────────────────────────────────────┘
```

### JavaScript 中的内存分配

```javascript
// ① 自动分配——基本类型
let num = 42              // 为数字分配内存
let str = 'hello'         // 为字符串分配内存
let bool = true           // 为布尔值分配内存

// ① 自动分配——引用类型
let obj = { name: 'Tom' } // 为对象及其属性分配内存
let arr = [1, 2, 3]       // 为数组及其元素分配内存
function foo() {}         // 为函数对象分配内存

// 显式分配（较少使用）
const buffer = new ArrayBuffer(1024)  // 分配 1KB 内存
```

> JavaScript 使用**自动垃圾回收**机制，开发者通常不需要手动释放内存。

## 二、栈内存与堆内存

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  栈（Stack）                              堆（Heap）          │
│  ┌─────────────────────┐                 ┌───────────────┐  │
│  │ num: 42             │                 │ Object {      │  │
│  │ str: 'hello'        │    ──引用──→    │   name: 'Tom',│  │
│  │ bool: true          │                 │   age: 18     │  │
│  │ obj: 0x001 ─────────┼───────┘        │ }             │  │
│  │ arr: 0x002 ─────────┼──────────────→ │ [1, 2, 3]     │  │
│  └─────────────────────┘                 └───────────────┘  │
│                                                              │
│  存储内容：基本类型的值         存储内容：引用类型的实际数据    │
│  大小固定，自动分配释放         大小不固定，由 GC 管理         │
│  空间小，读写快                 空间大，读写稍慢               │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

| 维度 | 栈内存 | 堆内存 |
|------|--------|--------|
| 存储内容 | 基本类型值、引用地址 | 对象、数组、函数等引用类型 |
| 大小 | 固定（编译时确定） | 动态（运行时分配） |
| 生命周期 | 函数执行结束自动回收 | 由垃圾回收器管理 |
| 大小限制 | 较小（浏览器通常几 MB） | 较大 |
| 访问速度 | 快 | 相对较慢 |

## 三、垃圾回收算法

### 1. 引用计数（Reference Counting）

**原理**：跟踪每个值被引用的次数，引用次数为 0 时回收内存。

```javascript
// 引用计数的工作方式
let obj = { name: 'Tom' }  // { name: 'Tom' } 引用计数 = 1
let copy = obj             // 引用计数 = 2
obj = null                 // 引用计数 = 1
copy = null                // 引用计数 = 0 → 可回收 ✅
```

**致命缺陷——循环引用**：

```javascript
function createCycle() {
  const a = {}
  const b = {}
  a.ref = b    // a 引用 b
  b.ref = a    // b 引用 a
  // 函数执行后，a 和 b 的引用计数都不为 0（互相引用）
  // 但它们已经无法从外部访问 → 内存泄漏！❌
}
```

> **注意**：早期 IE6/7 的 DOM 对象使用引用计数，会导致 DOM 与 JS 对象之间的循环引用泄漏。现代浏览器已不再使用此算法。

### 2. 标记-清除（Mark-Sweep）—— 现代引擎使用

**原理**：从"根对象"出发，递归遍历所有可达对象并标记，未标记的对象视为垃圾并清除。

```
标记-清除过程：

① 标记阶段（Mark）
   从根对象（window / global）出发，遍历所有可达对象并打上标记

   Root
    ├── obj1 ✅ (可达)
    │    └── prop ✅
    ├── obj2 ✅ (可达)
    │    └── ref → obj1 ✅
    └── obj3 ❌ (不可达，无标记)

② 清除阶段（Sweep）
   遍历堆内存，回收所有未标记的对象

   → obj3 被回收 ✅
```

```javascript
// 标记-清除解决循环引用
function createCycle() {
  const a = {}
  const b = {}
  a.ref = b
  b.ref = a
  // 函数结束后，a 和 b 都无法从根对象到达
  // → 标记阶段不会被标记 → 被清除 ✅
}
```

### V8 引擎的分代回收

V8（Chrome/Node.js 的 JS 引擎）使用**分代回收**策略：

```
┌─────────────────────────────────────────────┐
│              V8 堆内存结构                    │
│                                              │
│  ┌─ 新生代（Young Generation）──────┐       │
│  │  存放：新创建的、短生命周期对象    │       │
│  │  大小：~1-8MB                     │       │
│  │  算法：Scavenge（复制算法）        │       │
│  │  频率：频繁回收                    │       │
│  │                                   │       │
│  │  ┌─────────┐  ┌─────────┐       │       │
│  │  │  From   │  │   To    │       │       │
│  │  │(活动区) │→ │(空闲区) │       │       │
│  │  └─────────┘  └─────────┘       │       │
│  └───────────────────────────────────┘       │
│                    │ 经过 2 次 Scavenge       │
│                    ▼ 晋升                     │
│  ┌─ 老生代（Old Generation）──────┐          │
│  │  存放：长期存活的对象             │          │
│  │  大小：较大                      │          │
│  │  算法：标记-清除 + 标记-整理      │          │
│  │  频率：较少回收                   │          │
│  └──────────────────────────────────┘          │
│                                              │
└─────────────────────────────────────────────┘
```

**新生代回收（Scavenge 算法）**：

```
① 将堆分为两个等大的半空间：From（活动区）和 To（空闲区）
② 新对象分配在 From 区
③ GC 时，从根遍历 From 区的可达对象，复制到 To 区
④ 交换 From 和 To 的角色
⑤ 经历两次 Scavenge 仍存活的对象晋升到老生代
```

**老生代回收（Mark-Sweep + Mark-Compact）**：

```
① 标记-清除：标记所有可达对象，清除未标记的
② 标记-整理：清除后将存活对象向一端移动，消除内存碎片
③ 增量标记：将标记任务分为多个小步骤，穿插在 JS 执行中，避免长时间暂停
```

## 四、内存泄漏

内存泄漏是指**不再使用的内存没有被释放**，导致可用内存逐渐减少。

### 常见内存泄漏场景

#### 1. 意外的全局变量

```javascript
// ❌ 忘记声明变量 → 成为全局变量，永远不会被回收
function foo() {
  bar = '意外的全局变量'    // 等同于 window.bar
  this.baz = '也是全局变量'  // 函数中 this 指向 window
}
foo()

// ✅ 使用严格模式
'use strict'
function foo() {
  // bar = 'error'  // ReferenceError
}
```

#### 2. 被遗忘的定时器

```javascript
// ❌ 组件销毁后定时器仍在运行
function startPolling() {
  setInterval(() => {
    fetchData()    // 引用了外部变量，无法被回收
  }, 1000)
}

// ✅ 及时清除
let timer = setInterval(() => { fetchData() }, 1000)
function stopPolling() {
  clearInterval(timer)
  timer = null
}
```

#### 3. 被遗忘的事件监听器

```javascript
// ❌ 添加后未移除
function setup() {
  const handler = () => { /* ... */ }
  window.addEventListener('resize', handler)
  // 组件销毁时未移除
}

// ✅ 及时移除
function setup() {
  const handler = () => { /* ... */ }
  window.addEventListener('resize', handler)
  return () => window.removeEventListener('resize', handler)
}
```

#### 4. 闭包引用

```javascript
// ❌ 闭包意外持有大对象
function createHandler() {
  const hugeData = new Array(1000000).fill('*')

  return function() {
    // 即使不使用 hugeData，它也不会被回收
    console.log('click')
  }
}

// ✅ 使用后释放
function createHandler() {
  let hugeData = new Array(1000000).fill('*')
  const result = process(hugeData)
  hugeData = null   // 不再需要时释放
  return function() {
    console.log(result)
  }
}
```

#### 5. 脱离 DOM 的引用

```javascript
// ❌ DOM 元素从文档中移除，但 JS 仍持有引用
const elements = {
  button: document.getElementById('myButton'),
  image: document.getElementById('myImage')
}

document.body.removeChild(elements.button)
// button 从 DOM 中移除，但 elements.button 仍引用它 → 内存泄漏

// ✅ 移除 DOM 时同时清除引用
document.body.removeChild(elements.button)
elements.button = null
```

#### 6. 未取消的订阅/观察者

```javascript
// ❌ 忘记取消订阅
const subscription = eventBus.on('data', handler)
// 组件销毁时未取消

// ✅ 及时取消
onUnmounted(() => {
  subscription.off()
})
```

## 五、内存优化策略

### 1. 及时释放引用

```javascript
// 大对象使用完毕后设为 null
function process() {
  let bigData = loadHugeData()
  const result = analyze(bigData)
  bigData = null    // 释放引用
  return result
}
```

### 2. 使用弱引用（WeakMap / WeakSet）

```javascript
// WeakMap 的键是弱引用，不影响垃圾回收
const cache = new WeakMap()

function process(obj) {
  if (cache.has(obj)) return cache.get(obj)
  const result = heavyCompute(obj)
  cache.set(obj, result)
  return result
}
// 当 obj 被回收时，对应的缓存也会自动被回收
```

### 3. 避免频繁创建对象

```javascript
// ❌ 循环中频繁创建对象
function animate() {
  for (let i = 0; i < 1000; i++) {
    const point = { x: i, y: i * 2 }  // 每次创建新对象
    draw(point)
  }
}

// ✅ 复用对象
function animate() {
  const point = { x: 0, y: 0 }  // 只创建一次
  for (let i = 0; i < 1000; i++) {
    point.x = i
    point.y = i * 2
    draw(point)
  }
}
```

### 4. 使用对象池

```javascript
class ObjectPool {
  constructor(createFn, resetFn, initialSize = 10) {
    this.pool = []
    this.createFn = createFn
    this.resetFn = resetFn
    for (let i = 0; i < initialSize; i++) {
      this.pool.push(createFn())
    }
  }

  acquire() {
    return this.pool.length > 0 ? this.pool.pop() : this.createFn()
  }

  release(obj) {
    this.resetFn(obj)
    this.pool.push(obj)
  }
}

// 使用
const vecPool = new ObjectPool(
  () => ({ x: 0, y: 0 }),
  (v) => { v.x = 0; v.y = 0 }
)

const v = vecPool.acquire()
v.x = 10; v.y = 20
// 使用完毕
vecPool.release(v)
```

## 七、面试常见问题

### Q1：JavaScript 的垃圾回收机制是什么？

现代 JavaScript 引擎使用**标记-清除**算法：
1. 从根对象（window/global）出发
2. 递归遍历所有可达对象，打上标记
3. 清除所有未标记的对象，释放内存

V8 引擎使用**分代回收**：新生代用 Scavenge（复制算法），老生代用标记-清除 + 标记-整理。

### Q2：什么是内存泄漏？如何避免？

内存泄漏是不再使用的内存没有被释放。常见原因：
1. 意外的全局变量 → 使用严格模式
2. 忘记清除的定时器和事件监听 → 组件销毁时清除
3. 闭包持有不需要的引用 → 及时设为 null
4. 脱离 DOM 的引用 → 移除 DOM 时清除引用

### Q3：WeakMap 和 Map 的区别？

| 维度 | Map | WeakMap |
|------|-----|---------|
| 键类型 | 任意类型 | 只能是对象 |
| 引用类型 | 强引用 | **弱引用** |
| GC 影响 | 键不会被回收 | 键可被 GC 回收 |
| 可遍历 | ✅ | ❌ |
| 用途 | 一般键值存储 | 缓存、关联私有数据 |

### Q4：为什么 V8 要使用分代回收？

因为**大多数对象生命周期很短**（如函数内的临时变量），只有少数对象会长期存活。分代回收针对不同生命周期的对象使用不同策略：
- 新生代：频繁、快速的小规模回收（Scavenge）
- 老生代：低频、完整的大规模回收（Mark-Sweep）

这样整体回收效率远高于单一策略。
