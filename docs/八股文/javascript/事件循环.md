# 事件循环

> 官方文档：[MDN - 事件循环](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Event_loop)

> 深入理解：[HTML Spec - Event Loops](https://html.spec.whatwg.org/multipage/webappapis.html#event-loops)

---

## 一、为什么需要事件循环

JavaScript 是**单线程**语言，但它需要处理：
- 用户交互（点击、滚动）
- 网络请求
- 定时器
- 渲染更新

事件循环（Event Loop）是 JavaScript 实现**非阻塞异步**的核心机制。

```
┌─ 浏览器环境 ──────────────────────────────────────────────┐
│                                                            │
│  ┌───────────────────────┐                                │
│  │    调用栈（Call Stack）│  ← 同步代码逐行执行              │
│  └───────────┬───────────┘                                │
│              │ 执行完毕                                     │
│              ▼                                             │
│  ┌───────────────────────┐    ┌──────────────────────┐    │
│  │   微任务队列           │    │   宏任务队列          │    │
│  │  (Microtask Queue)    │    │  (Macrotask Queue)   │    │
│  │                       │    │                      │    │
│  │  Promise.then/catch   │    │  setTimeout          │    │
│  │  MutationObserver     │    │  setInterval         │    │
│  │  queueMicrotask       │    │  I/O 回调            │    │
│  │  process.nextTick     │    │  UI 渲染             │    │
│  │  (Node.js)            │    │  requestAnimationFrame│    │
│  └───────────────────────┘    └──────────────────────┘    │
│                                                            │
│  事件循环：调用栈清空 → 执行所有微任务 → 取一个宏任务 → 循环   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 二、核心概念

### 1. 调用栈（Call Stack）

JavaScript 引擎维护的**执行上下文栈**，跟踪当前正在执行的代码。

```javascript
function foo() {
  console.log('foo')
  bar()
}

function bar() {
  console.log('bar')
}

foo()

// 调用栈变化：
// 1. [foo]               → 执行 foo
// 2. [foo, console.log]  → 执行 console.log('foo')
// 3. [foo, bar]          → 执行 bar
// 4. [foo, bar, console.log] → 执行 console.log('bar')
// 5. [foo]               → bar 执行完毕
// 6. []                  → foo 执行完毕
```

### 2. 宏任务（Macrotask）

| 来源 | 示例 |
|------|------|
| 定时器 | `setTimeout`、`setInterval` |
| I/O 操作 | 网络请求、文件读取回调 |
| UI 渲染 | DOM 变更后的渲染 |
| 用户交互 | 点击、键盘事件回调 |
| `requestAnimationFrame` | 下一帧渲染前执行 |
| `MessageChannel` | 消息通道回调 |

### 3. 微任务（Microtask）

| 来源 | 示例 |
|------|------|
| Promise | `.then()`、`.catch()`、`.finally()` |
| `queueMicrotask()` | 手动添加微任务 |
| `MutationObserver` | DOM 变动监听回调 |
| `process.nextTick` | Node.js 中的微任务（优先级最高） |

## 三、事件循环执行流程

```
┌────────────────────────────────────────────────────────┐
│                    事件循环流程                          │
│                                                         │
│  ① 执行同步代码（调用栈中的任务）                         │
│         │                                               │
│         ▼                                               │
│  ② 调用栈清空 → 执行所有微任务                           │
│         │  （微任务中产生的微任务也在本轮执行完）           │
│         ▼                                               │
│  ③ 微任务队列清空 → 浏览器渲染（如果需要）                │
│         │                                               │
│         ▼                                               │
│  ④ 取下一个宏任务执行                                    │
│         │                                               │
│         ▼                                               │
│  ⑤ 回到步骤 ②，继续循环                                 │
│                                                         │
└────────────────────────────────────────────────────────┘
```

> **关键**：每次宏任务执行完后，会**清空所有微任务**，然后再执行下一个宏任务。

## 四、经典输出顺序题

### 题目一：基础

```javascript
console.log('1')              // 同步

setTimeout(() => {
  console.log('2')            // 宏任务
}, 0)

Promise.resolve().then(() => {
  console.log('3')            // 微任务
})

console.log('4')              // 同步

// 输出顺序：1 → 4 → 3 → 2
// 解析：
// 同步：1, 4
// 微任务：3
// 宏任务：2
```

### 题目二：嵌套

```javascript
setTimeout(() => {
  console.log('1')
  Promise.resolve().then(() => {
    console.log('2')
  })
}, 0)

new Promise((resolve) => {
  console.log('3')
  resolve()
}).then(() => {
  console.log('4')
  setTimeout(() => {
    console.log('5')
  }, 0)
})

console.log('6')

// 输出顺序：3 → 6 → 4 → 1 → 2 → 5
// 解析：
// 第一轮：同步 3, 6 → 微任务 4（注册 setTimeout(5)）→ 宏任务无
// 第二轮：宏任务 setTimeout(1) → 同步 1 → 微任务 2 → 宏任务无
// 第三轮：宏任务 setTimeout(5) → 同步 5
```

### 题目三：async/await

```javascript
async function async1() {
  console.log('1')
  await async2()
  console.log('2')    // await 后面的代码相当于 .then()，是微任务
}

async function async2() {
  console.log('3')
}

console.log('4')

setTimeout(() => {
  console.log('5')
}, 0)

async1()

new Promise((resolve) => {
  console.log('6')
  resolve()
}).then(() => {
  console.log('7')
})

console.log('8')

// 输出顺序：4 → 1 → 3 → 6 → 8 → 2 → 7 → 5
// 解析：
// 同步：4, 1, 3, 6, 8
// 微任务：2, 7（async1 中 await 后的 console.log('2') 是微任务）
// 宏任务：5
```

### 题目四：微任务中产生微任务

```javascript
Promise.resolve()
  .then(() => {
    console.log('1')
    return Promise.resolve('2')
  })
  .then((val) => {
    console.log(val)
  })

Promise.resolve()
  .then(() => {
    console.log('3')
  })
  .then(() => {
    console.log('4')
  })

// 输出顺序：1 → 3 → 4 → 2
// 解析：return Promise.resolve('2') 会多加一层微任务
```

## 五、Node.js 的事件循环

Node.js 的事件循环分为**多个阶段**：

```
   ┌──────────────────────────┐
───┤     timers（定时器）       │  ← setTimeout / setInterval
   └──────────┬───────────────┘
   ┌──────────▼───────────────┐
───┤   pending callbacks       │  ← 系统级回调（TCP 错误等）
   └──────────┬───────────────┘
   ┌──────────▼───────────────┐
───┤     idle, prepare         │  ← 内部使用
   └──────────┬───────────────┘
   ┌──────────▼───────────────┐
───┤     poll（轮询）          │  ← I/O 回调（文件读取、网络等）
   └──────────┬───────────────┘
   ┌──────────▼───────────────┐
───┤     check（检查）         │  ← setImmediate
   └──────────┬───────────────┘
   ┌──────────▼───────────────┐
───┤     close callbacks       │  ← socket.on('close')
   └──────────────────────────┘
```

### Node.js 微任务优先级

```javascript
// process.nextTick 优先于 Promise.then
process.nextTick(() => console.log('nextTick'))
Promise.resolve().then(() => console.log('promise'))
// 输出：nextTick → promise
```

### setTimeout vs setImmediate

```javascript
// 在非 I/O 上下文中，顺序不确定
setTimeout(() => console.log('timeout'), 0)
setImmediate(() => console.log('immediate'))
// 可能 timeout 先，也可能 immediate 先

// 在 I/O 回调中，setImmediate 一定先于 setTimeout
const fs = require('fs')
fs.readFile('file.txt', () => {
  setTimeout(() => console.log('timeout'), 0)
  setImmediate(() => console.log('immediate'))
})
// 输出：immediate → timeout（check 阶段在 timers 之前）
```

## 六、requestAnimationFrame

`requestAnimationFrame`（rAF）不属于宏任务也不属于微任务，它在**微任务之后、浏览器渲染之前**执行。

```
事件循环的一帧：

同步代码 → 微任务全部执行 → requestAnimationFrame → 浏览器渲染 → 下一个宏任务
```

```javascript
console.log('同步')

setTimeout(() => console.log('setTimeout'), 0)

Promise.resolve().then(() => console.log('Promise'))

requestAnimationFrame(() => console.log('rAF'))

// 输出顺序：同步 → Promise → rAF → setTimeout
// 注意：rAF 的实际执行时机与浏览器渲染帧有关
```

## 七、面试常见问题

### Q1：宏任务和微任务的区别？

| 维度 | 宏任务 | 微任务 |
|------|--------|--------|
| 执行时机 | 每轮事件循环取一个 | 每轮事件循环清空全部 |
| 产生方式 | setTimeout、I/O、事件等 | Promise.then、MutationObserver 等 |
| 优先级 | 低（晚执行） | 高（早执行） |
| 数量控制 | 每次执行一个 | 全部执行完 |

### Q2：为什么微任务优先于宏任务？

微任务的设计目的是**尽快执行**，在当前宏任务结束后立即处理，不需要等待渲染或其他宏任务。这样可以保证 Promise 等异步操作的响应速度，减少延迟。

### Q3：事件循环中浏览器渲染发生在什么时候？

在**微任务清空之后、下一个宏任务之前**。具体流程：
1. 执行一个宏任务
2. 清空所有微任务
3. **浏览器判断是否需要渲染**（通常 60fps，约 16.6ms 一次）
4. 如需渲染：执行 rAF → 布局 → 绘制
5. 取下一个宏任务

### Q4：await 后面的代码是微任务吗？

**是的**。`await` 后面的代码相当于 `Promise.resolve().then()` 的回调，会被放入微任务队列。

```javascript
async function foo() {
  console.log('1')      // 同步
  await bar()
  console.log('2')      // 微任务（等同于 .then 回调）
}
```
