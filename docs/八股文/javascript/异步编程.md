## 一、为什么需要异步

JavaScript 是**单线程**语言，如果所有操作都同步执行，耗时任务（网络请求、文件读取、定时器）会**阻塞**整个程序。异步编程允许在等待耗时操作完成时，继续执行其他代码。

```
同步执行：                           异步执行：
任务1 ████████████                   任务1 ████████████
任务2              ████████████       任务2    ████████████
任务3                           ████  任务3       ████████████
                                     （任务2等待期间执行任务3）
→ 耗时操作阻塞后续所有任务            → 耗时操作不阻塞
```

## 三、回调函数（Callback）

### 基本用法

```javascript
// 定时器回调
setTimeout(() => {
  console.log('1 秒后执行')
}, 1000)

// 事件回调
button.addEventListener('click', () => {
  console.log('按钮被点击')
})

// Node.js 风格回调
fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error(err)
    return
  }
  console.log(data)
})
```

### 回调地狱（Callback Hell）

多个异步操作有依赖关系时，回调函数层层嵌套：

```javascript
// ❌ 回调地狱
getUser(userId, (err, user) => {
  if (err) return handleError(err)
  getOrders(user.id, (err, orders) => {
    if (err) return handleError(err)
    getOrderDetail(orders[0].id, (err, detail) => {
      if (err) return handleError(err)
      getProduct(detail.productId, (err, product) => {
        if (err) return handleError(err)
        console.log(product)
        // 继续嵌套...
      })
    })
  })
})
```

**问题**：代码嵌套深、难以阅读、错误处理复杂、难以维护。

## 四、Promise

### 核心概念

Promise 是一个代表**异步操作最终完成或失败**的对象。

```
Promise 的三种状态：

  pending（待定）──→ fulfilled（已兑现）──→ .then() 处理
                 │
                 └──→ rejected（已拒绝）──→ .catch() 处理

  ⚠️ 状态一旦改变就不可逆（pending → fulfilled 或 pending → rejected）
```

### 基本用法

```javascript
// 创建 Promise
const promise = new Promise((resolve, reject) => {
  // resolve —— 成功时调用
  // reject  —— 失败时调用
  setTimeout(() => {
    const success = true
    if (success) {
      resolve('操作成功')
    } else {
      reject(new Error('操作失败'))
    }
  }, 1000)
})

// 使用 Promise
promise
  .then(result => {
    console.log(result)    // '操作成功'
    return '链式调用'
  })
  .then(result => {
    console.log(result)    // '链式调用'
  })
  .catch(error => {
    console.error(error)
  })
  .finally(() => {
    console.log('无论成功失败都执行')
  })
```

### 链式调用解决回调地狱

```javascript
// ✅ Promise 链式调用
getUser(userId)
  .then(user => getOrders(user.id))
  .then(orders => getOrderDetail(orders[0].id))
  .then(detail => getProduct(detail.productId))
  .then(product => console.log(product))
  .catch(err => console.error('任何一步出错都会到这里:', err))
```

### Promise 静态方法

| 方法 | 说明 |
|------|------|
| `Promise.all(promises)` | 所有都成功才成功，一个失败就失败 |
| `Promise.allSettled(promises)` | 等待所有完成，无论成功失败 |
| `Promise.race(promises)` | 取最先完成的（无论成功失败） |
| `Promise.any(promises)` | 取最先成功的，全部失败才失败 |
| `Promise.resolve(value)` | 返回一个已成功的 Promise |
| `Promise.reject(reason)` | 返回一个已拒绝的 Promise |

```javascript
// Promise.all —— 并行请求，全部成功
Promise.all([
  fetch('/api/users'),
  fetch('/api/orders'),
  fetch('/api/products')
])
  .then(([users, orders, products]) => {
    console.log('全部请求成功')
  })
  .catch(err => {
    console.error('有一个请求失败了')
  })

// Promise.allSettled —— 获取所有结果
Promise.allSettled([
  fetch('/api/users'),
  fetch('/api/orders')
])
  .then(results => {
    results.forEach(result => {
      if (result.status === 'fulfilled') {
        console.log('成功:', result.value)
      } else {
        console.log('失败:', result.reason)
      }
    })
  })

// Promise.race —— 请求超时
Promise.race([
  fetch('/api/data'),
  new Promise((_, reject) =>
    setTimeout(() => reject(new Error('超时')), 5000)
  )
])

// Promise.any —— 取最快成功的
Promise.any([
  fetch('https://cdn1.example.com/data'),
  fetch('https://cdn2.example.com/data'),
  fetch('https://cdn3.example.com/data')
])
  .then(firstSuccess => console.log('最快的成功响应'))
```

## 五、async / await

### 核心概念

`async/await` 是 Promise 的**语法糖**，让异步代码看起来像同步代码。

```javascript
// async 函数自动返回 Promise
async function fetchData() {
  return '数据'   // 等同于 return Promise.resolve('数据')
}

// await 等待 Promise 完成
async function getData() {
  try {
    const result = await fetchData()
    console.log(result)  // '数据'
  } catch (error) {
    console.error(error)
  }
}
```

### 解决回调地狱（终极方案）

```javascript
// ✅ async/await —— 最优雅的方式
async function getProductInfo(userId) {
  try {
    const user = await getUser(userId)
    const orders = await getOrders(user.id)
    const detail = await getOrderDetail(orders[0].id)
    const product = await getProduct(detail.productId)
    return product
  } catch (error) {
    console.error('获取商品信息失败:', error)
    throw error
  }
}
```

### 并行执行

```javascript
// ❌ 串行执行（慢）
async function serial() {
  const users = await fetchUsers()      // 等 1 秒
  const orders = await fetchOrders()    // 再等 1 秒
  // 总共 2 秒
}

// ✅ 并行执行（快）
async function parallel() {
  const [users, orders] = await Promise.all([
    fetchUsers(),
    fetchOrders()
  ])
  // 总共 1 秒
}
```

### 错误处理

```javascript
// 方式一：try/catch（推荐）
async function fetchData() {
  try {
    const response = await fetch('/api/data')
    const data = await response.json()
    return data
  } catch (error) {
    console.error('请求失败:', error)
    return null
  }
}

// 方式二：.catch()
async function fetchData() {
  const response = await fetch('/api/data').catch(err => {
    console.error('请求失败:', err)
    return null
  })
  if (!response) return null
  return response.json()
}

// 方式三：包装函数
function to<T>(promise: Promise<T>): Promise<[null, T] | [Error, null]> {
  return promise
    .then<[null, T]>(data => [null, data])
    .catch<[Error, null]>(err => [err, null])
}

async function fetchData() {
  const [err, data] = await to(fetch('/api/data').then(r => r.json()))
  if (err) {
    console.error('请求失败:', err)
    return null
  }
  return data
}
```

## 六、Generator 函数

### 基本用法

Generator 是一种可以**暂停执行**的函数，通过 `yield` 暂停，通过 `next()` 恢复。

```javascript
function* idGenerator() {
  let id = 1
  while (true) {
    yield id++
  }
}

const gen = idGenerator()
gen.next()  // { value: 1, done: false }
gen.next()  // { value: 2, done: false }
gen.next()  // { value: 3, done: false }
```

### Generator 实现异步流程控制

```javascript
function* fetchData() {
  try {
    const user = yield getUser(userId)
    const orders = yield getOrders(user.id)
    const detail = yield getOrderDetail(orders[0].id)
    return detail
  } catch (error) {
    console.error(error)
  }
}

// 自动执行器
function run(generator) {
  const gen = generator()

  function step(method, arg) {
    const { value, done } = gen[method](arg)
    if (done) return Promise.resolve(value)
    return Promise.resolve(value).then(
      val => step('next', val),
      err => step('throw', err)
    )
  }

  return step('next')
}

run(fetchData).then(result => console.log(result))
```

> async/await 本质上就是 Generator + 自动执行器的语法糖。

## 七、各方案对比

| 维度 | 回调函数 | Promise | Generator | async/await |
|------|---------|---------|-----------|-------------|
| 可读性 | ❌ 嵌套地狱 | ✅ 链式调用 | ⚠️ 需执行器 | ✅✅ 同步风格 |
| 错误处理 | 手动每层传递 | `.catch()` 统一 | try/catch | try/catch |
| 调试 | 困难 | 一般 | 一般 | ✅ 像同步代码 |
| 并行控制 | 手动 | `Promise.all` 等 | 手动 | `Promise.all` + await |
| 兼容性 | 所有环境 | ES6+ | ES6+ | ES2017+ |

## 八、面试常见问题

### Q1：Promise 的状态可以改变几次？

**只能改变一次**。从 `pending` 变为 `fulfilled` 或 `rejected` 后就不可逆。后续的 `resolve`/`reject` 调用会被忽略。

```javascript
const p = new Promise((resolve, reject) => {
  resolve('第一次')
  resolve('第二次')   // 被忽略
  reject('失败')      // 被忽略
})
p.then(val => console.log(val))  // '第一次'
```

### Q2：Promise.all 和 Promise.allSettled 的区别？

- `Promise.all`：**一票否决**——任意一个失败就整体失败，适用于所有请求必须成功的场景
- `Promise.allSettled`：**永远成功**——等待所有 Promise 完成，返回每个 Promise 的状态和结果，适用于需要知道每个请求结果的场景

### Q3：async 函数的错误如何穿透？

```javascript
async function inner() {
  throw new Error('内部错误')
}

async function outer() {
  // 没有 try/catch，错误会向上冒泡
  const result = await inner()
  return result
}

// 错误冒泡到最外层的 catch
outer().catch(err => console.error(err.message))  // '内部错误'
```

### Q4：await 后面一定要跟 Promise 吗？

不一定。`await` 后面如果不是 Promise，会被自动包装为 `Promise.resolve(value)`：

```javascript
async function foo() {
  const a = await 42          // 等同于 await Promise.resolve(42)
  const b = await 'hello'     // 等同于 await Promise.resolve('hello')
  const c = await null        // 等同于 await Promise.resolve(null)
}
```
