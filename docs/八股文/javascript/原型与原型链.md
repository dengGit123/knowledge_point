## 一、核心概念

JavaScript 的面向对象机制基于**原型**，而非类（Class）。每个对象都有一个内部链接指向另一个对象（它的原型），这条链一直延伸到 `null`。

```
┌─ 原型链查找过程 ──────────────────────────────────────────────┐
│                                                               │
│  实例对象  ──→  构造函数.prototype  ──→  Object.prototype      │
│     │              │                         │                 │
│  __proto__     __proto__                 __proto__             │
│                     │                         │                 │
│                     ▼                         ▼                 │
│              原型对象上的方法           Object 上的方法          │
│            (hasOwnProperty等)            → null（终点）        │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

> **通俗理解**：原型链就像"族谱"——你找不到某个属性时，就去问父亲（原型对象），父亲找不到就去问爷爷，一直到祖先（Object.prototype），再往上就是 null（没了）。

## 二、`prototype` 和 `__proto__` 的关系

### 关键规则

```
┌──────────────────────────────────────────────────────┐
│  ① 每个函数都有一个 prototype 属性（原型对象）        │
│  ② 每个对象都有一个 __proto__ 指向其构造函数的 prototype │
│  ③ 实例.__proto__ === 构造函数.prototype               │
│  ④ 构造函数.prototype.constructor === 构造函数          │
└──────────────────────────────────────────────────────┘
```

```javascript
function Person(name) {
  this.name = name
}

Person.prototype.sayHello = function() {
  return `Hello, I'm ${this.name}`
}

const tom = new Person('Tom')

// 核心关系
console.log(tom.__proto__ === Person.prototype)         // true  ①
console.log(Person.prototype.constructor === Person)    // true  ②
console.log(tom.__proto__.__proto__ === Object.prototype) // true ③
console.log(Object.prototype.__proto__ === null)        // true  ④ 终点
```

### 图示

```
                    Person（构造函数）
                    ├── prototype ─────────────────────┐
                    │                                  ▼
                    │              Person.prototype（原型对象）
                    │              ├── constructor → Person
                    │              ├── sayHello() 方法
                    │              └── __proto__ ──────────┐
                    │                                      ▼
new Person('Tom')                    Object.prototype
       │                            ├── hasOwnProperty()
       ▼                            ├── toString()
    tom（实例）                      ├── valueOf()
    ├── name: 'Tom'                  └── __proto__ → null
    └── __proto__ ──→ Person.prototype
```

## 三、属性查找机制

当访问对象的属性时，JavaScript 沿着原型链**逐层向上查找**：

```javascript
function Animal(name) {
  this.name = name
}
Animal.prototype.type = '动物'

function Dog(name, breed) {
  Animal.call(this, name)
  this.breed = breed
}
Dog.prototype = Object.create(Animal.prototype)
Dog.prototype.constructor = Dog
Dog.prototype.bark = function() { return '汪!' }

const dog = new Dog('旺财', '柯基')

// 属性查找过程
console.log(dog.name)     // '旺财'      → 自身属性 ✅
console.log(dog.breed)    // '柯基'      → 自身属性 ✅
console.log(dog.bark())   // '汪!'       → Dog.prototype ✅
console.log(dog.type)     // '动物'      → Animal.prototype ✅
console.log(dog.toString()) // '[object Object]' → Object.prototype ✅
// console.log(dog.xyz)   // undefined   → 找不到，原型链终点为 null
```

### 属性判断方法

```javascript
// hasOwnProperty —— 只检查自身属性
dog.hasOwnProperty('name')    // true
dog.hasOwnProperty('type')    // false（原型链上的）

// in 操作符 —— 检查整个原型链
'name' in dog                 // true
'type' in dog                 // true
'toString' in dog             // true

// for...in —— 遍历可枚举属性（包括原型链）
for (const key in dog) {
  if (dog.hasOwnProperty(key)) {
    console.log('自身:', key)   // name, breed
  }
}
```

## 四、`new` 操作符的实现原理

```javascript
function myNew(Constructor, ...args) {
  // 1. 创建一个新对象，原型指向构造函数的 prototype
  const obj = Object.create(Constructor.prototype)

  // 2. 执行构造函数，this 指向新对象
  const result = Constructor.apply(obj, args)

  // 3. 如果构造函数返回了对象，则返回该对象；否则返回新对象
  return result instanceof Object ? result : obj
}
```

```javascript
function Person(name) {
  this.name = name
}
Person.prototype.sayHi = function() { return `Hi, ${this.name}` }

// 等价关系
const tom1 = new Person('Tom')
const tom2 = myNew(Person, 'Tom')

tom1.sayHi()  // 'Hi, Tom'
tom2.sayHi()  // 'Hi, Tom'
tom1 instanceof Person  // true
tom2 instanceof Person  // true
```

## 五、继承方式

### 1. 原型链继承

```javascript
function Animal(type) {
  this.type = type
  this.colors = ['white', 'black']
}
Animal.prototype.getType = function() { return this.type }

function Dog(name) {
  this.name = name
}
Dog.prototype = new Animal('dog')

const dog1 = new Dog('旺财')
const dog2 = new Dog('富贵')

// ❌ 引用类型属性被共享
dog1.colors.push('brown')
console.log(dog2.colors)  // ['white', 'black', 'brown'] —— 被污染！
```

| 优点 | 缺点 |
|------|------|
| 简单 | 引用类型属性被所有实例共享 |
| 子类可访问父类原型方法 | 创建子类实例时无法向父类构造函数传参 |

### 2. 构造函数继承（借用构造函数）

```javascript
function Animal(type) {
  this.type = type
  this.colors = ['white', 'black']
}
Animal.prototype.getType = function() { return this.type }

function Dog(name, type) {
  Animal.call(this, type)  // 借用父类构造函数
  this.name = name
}

const dog1 = new Dog('旺财', 'dog')
const dog2 = new Dog('富贵', 'dog')

dog1.colors.push('brown')
console.log(dog2.colors)  // ['white', 'black'] ✅ 互不影响

// ❌ 无法访问父类原型上的方法
// dog1.getType()  // TypeError
```

### 3. 组合继承（最常用）

```javascript
function Animal(type) {
  this.type = type
  this.colors = ['white', 'black']
}
Animal.prototype.getType = function() { return this.type }

function Dog(name, type) {
  Animal.call(this, type)         // 第二次调用父类构造函数
  this.name = name
}
Dog.prototype = new Animal()       // 第一次调用父类构造函数
Dog.prototype.constructor = Dog

const dog = new Dog('旺财', 'dog')
dog.getType()   // 'dog' ✅
dog.colors      // ['white', 'black'] ✅
```

| 优点 | 缺点 |
|------|------|
| 既能访问原型方法，又有独立属性 | 父类构造函数被调用了两次 |
| 引用类型不共享 | |

### 4. 寄生组合继承（最优方案）

```javascript
function inheritPrototype(Sub, Super) {
  // 创建父类原型的一个副本
  const prototype = Object.create(Super.prototype)
  prototype.constructor = Sub
  Sub.prototype = prototype
}

function Animal(type) {
  this.type = type
  this.colors = ['white', 'black']
}
Animal.prototype.getType = function() { return this.type }

function Dog(name, type) {
  Animal.call(this, type)   // 只调用一次父类构造函数
  this.name = name
}

inheritPrototype(Dog, Animal)  // 建立原型链
Dog.prototype.bark = function() { return '汪!' }

const dog = new Dog('旺财', 'dog')
dog.getType()  // 'dog' ✅
dog.bark()     // '汪!' ✅
```

### 5. ES6 Class 继承

```javascript
class Animal {
  constructor(type) {
    this.type = type
    this.colors = ['white', 'black']
  }

  getType() {
    return this.type
  }
}

class Dog extends Animal {
  constructor(name, type) {
    super(type)    // 必须先调用 super
    this.name = name
  }

  bark() {
    return '汪!'
  }
}

const dog = new Dog('旺财', 'dog')
dog.getType()  // 'dog' ✅
dog.bark()     // '汪!' ✅
```

> **本质**：ES6 的 `class` 只是语法糖，底层仍然是基于原型链的继承。

### 继承方式对比

| 方式 | 原型方法 | 引用类型独立 | 传参 | 调用次数 | 推荐度 |
|------|---------|------------|------|---------|--------|
| 原型链继承 | ✅ | ❌ | ❌ | 1 | ⭐ |
| 构造函数继承 | ❌ | ✅ | ✅ | 1 | ⭐ |
| 组合继承 | ✅ | ✅ | ✅ | 2 | ⭐⭐⭐ |
| 寄生组合继承 | ✅ | ✅ | ✅ | 1 | ⭐⭐⭐⭐ |
| ES6 Class | ✅ | ✅ | ✅ | 1 | ⭐⭐⭐⭐⭐ |

## 六、面试常见问题

### Q1：`__proto__` 和 `prototype` 的区别？

- **`prototype`**：是**函数**的属性，指向函数的原型对象。它定义了由该函数创建的实例的共享属性和方法。
- **`__proto__`**：是**所有对象**的属性，指向创建该对象的构造函数的 `prototype`。它是实例与原型之间的链接。

```javascript
function Foo() {}
const foo = new Foo()

foo.__proto__ === Foo.prototype  // true
```

### Q2：如何判断对象是某个构造函数的实例？

```javascript
// instanceof —— 检查原型链
tom instanceof Person      // true
tom instanceof Object      // true

// isPrototypeOf —— 检查原型关系
Person.prototype.isPrototypeOf(tom)  // true

// Object.getPrototypeOf —— 获取原型
Object.getPrototypeOf(tom) === Person.prototype  // true
```

### Q3：ES6 的 class 和 function 构造函数有什么区别？

| 维度 | function 构造函数 | ES6 class |
|------|-----------------|-----------|
| 变量提升 | ✅ 提升 | ❌ 不提升（存在 TDZ） |
| 不用 new 调用 | 可以（作为普通函数） | ❌ 报错 |
| 原型方法可枚举 | ✅ 可枚举 | ❌ 不可枚举 |
| 静态方法 | 手动添加到构造函数 | `static` 关键字 |
| 本质 | 函数 | 函数（语法糖） |

### Q4：Object.create(null) 有什么用？

创建一个**没有原型**的纯净对象，适合用作字典/Map：

```javascript
const dict = Object.create(null)
dict['key'] = 'value'
dict.toString   // undefined（没有继承 Object.prototype 的方法）
dict.hasOwnProperty  // undefined
// 适合存储键值对，不用担心与原型属性冲突
```
