## 一、作用域（Scope）

作用域是**变量可访问性的范围**，决定了代码中变量和函数的可见性。

```
┌─ 全局作用域 ──────────────────────────────────────────┐
│  var globalVar = '全局变量'                             │
│                                                        │
│  ┌─ 函数作用域 foo() ────────────────────────────┐     │
│  │  var funcVar = '函数变量'                      │     │
│  │                                                │     │
│  │  ┌─ 块级作用域 { } ───────────────────────┐   │     │
│  │  │  let blockVar = '块级变量'               │   │     │
│  │  │  const x = 1                            │   │     │
│  │  └─────────────────────────────────────────┘   │     │
│  │                                                │     │
│  └────────────────────────────────────────────────┘     │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### 三种作用域

| 类型 | 说明 | 创建时机 |
|------|------|---------|
| **全局作用域** | 最外层，整个脚本可访问 | 脚本加载时 |
| **函数作用域** | 函数内部，外部无法访问 | 函数调用时 |
| **块级作用域**（ES6） | `{ }` 内 `let`/`const` 声明的变量 | 执行到块时 |

```javascript
// 全局作用域
var global = '全局'

function foo() {
  // 函数作用域
  var func = '函数'
  console.log(global)  // ✅ 可访问全局

  if (true) {
    // 块级作用域
    let block = '块级'
    const x = 1
    console.log(func)   // ✅ 可访问外层函数作用域
  }

  // console.log(block)  // ❌ 块级作用域外无法访问
}
```

### var / let / const 对比

| 维度 | `var` | `let` | `const` |
|------|-------|-------|---------|
| 作用域 | 函数作用域 | 块级作用域 | 块级作用域 |
| 变量提升 | ✅ 提升（初始化为 undefined） | ✅ 提升（但存在暂时性死区） | ✅ 提升（但存在暂时性死区） |
| 重复声明 | ✅ 允许 | ❌ 不允许 | ❌ 不允许 |
| 重新赋值 | ✅ 允许 | ✅ 允许 | ❌ 不允许 |
| 全局声明 | 挂载到 `window` | 不挂载到 `window` | 不挂载到 `window` |

```javascript
// 1. var 的变量提升
console.log(a)  // undefined（已提升，但未赋值）
var a = 1

// 2. let/const 的暂时性死区（TDZ）
// console.log(b)  // ReferenceError（在声明之前不可访问）
let b = 2

// 3. var 重复声明
var x = 1
var x = 2   // ✅ 不报错

// 4. let 重复声明
// let y = 1
// let y = 2   // SyntaxError

// 5. 全局声明差异
var g1 = 1
let g2 = 2
console.log(window.g1)  // 1
console.log(window.g2)  // undefined
```

## 二、作用域链（Scope Chain）

### 概念

当访问一个变量时，JavaScript 引擎会沿着**作用域链**从内到外逐层查找，直到找到为止。

```
查找变量 name 的过程：

内层作用域 → 找不到 → 外层作用域 → 找不到 → ... → 全局作用域 → 找不到 → ReferenceError
```

```javascript
let global = '全局'

function outer() {
  let outerVar = '外层'

  function inner() {
    let innerVar = '内层'
    console.log(innerVar)  // ✅ 当前作用域找到
    console.log(outerVar)  // ✅ 外层作用域找到
    console.log(global)    // ✅ 全局作用域找到
    // console.log(unknown) // ❌ ReferenceError
  }

  inner()
}

outer()
```

### 词法作用域（Lexical Scope）

JavaScript 采用**词法作用域**（也叫静态作用域），函数的作用域在**定义时**就确定了，而非调用时。

```javascript
const value = '全局'

function foo() {
  console.log(value)  // '全局'——取决于定义位置，而非调用位置
}

function bar() {
  const value = '局部'
  foo()  // 虽然在 bar 中调用，但 foo 定义时 value 就是全局的
}

bar()  // 输出 '全局'，而非 '局部'
```

> **通俗理解**：词法作用域就像"出生地决定户口"——函数能访问哪些变量在它**出生（定义）**的那一刻就确定了，和它后来被搬到哪里（调用位置）无关。

## 三、闭包（Closure）

### 核心概念

**闭包 = 函数 + 能够访问的外层变量（词法环境）**

当一个函数可以**记住并访问**其定义时的词法作用域，即使该函数在作用域之外执行，就形成了闭包。

```javascript
function createCounter() {
  let count = 0          // 被闭包捕获的变量（自由变量）

  return function() {
    count++               // 访问外层函数的变量
    return count
  }
}

const counter = createCounter()
console.log(counter())  // 1
console.log(counter())  // 2
console.log(counter())  // 3
// count 变量不会被垃圾回收，因为闭包仍然引用着它
```

> **通俗理解**：闭包就像一个"背包"——函数背着它出生时的环境到处跑，无论在哪里被调用，都能从背包里取出当初环境中的变量。

### 闭包的形成条件

```
闭包形成的三个条件：
1. 存在内外两层函数
2. 内层函数访问了外层函数的变量
3. 内层函数被返回到外部并被执行
```

### 闭包的工作原理

```
┌─ createCounter 执行上下文（已弹出调用栈）──┐
│                                            │
│  ┌─ 闭包环境（仍被引用，不会被 GC）───┐   │
│  │  count: 3（持续存在，不会被销毁）   │   │
│  └─────────────────────────────────────┘   │
│                                            │
│  → 返回的匿名函数持有对此环境的引用         │
└────────────────────────────────────────────┘
         │
         ▼
  counter()  →  访问闭包环境中的 count  →  count++  →  返回 3
```

## 四、闭包的经典应用

### 1. 数据私有化 / 封装

```javascript
function createUser(name) {
  let _password = 'default'  // 私有变量，外部无法直接访问

  return {
    getName() { return name },
    setPassword(newPwd) { _password = newPwd },
    checkPassword(pwd) { return pwd === _password }
  }
}

const user = createUser('Tom')
user.getName()              // 'Tom'
user.checkPassword('wrong') // false
user.setPassword('new123')
user.checkPassword('new123')// true
// user._password            // undefined —— 无法直接访问
```

### 2. 函数工厂

```javascript
function multiply(factor) {
  return function(number) {
    return number * factor
  }
}

const double = multiply(2)
const triple = multiply(3)

double(5)   // 10
triple(5)   // 15
```

### 3. 缓存 / 记忆化

```javascript
function memoize(fn) {
  const cache = new Map()
  return function(...args) {
    const key = JSON.stringify(args)
    if (cache.has(key)) {
      console.log('命中缓存')
      return cache.get(key)
    }
    const result = fn.apply(this, args)
    cache.set(key, result)
    return result
  }
}

const expensiveCalc = memoize(function(n) {
  console.log('计算中...')
  return n * n
})

expensiveCalc(5)  // 计算中... → 25
expensiveCalc(5)  // 命中缓存 → 25
```

### 4. 柯里化（Currying）

```javascript
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args)
    }
    return function(...moreArgs) {
      return curried.apply(this, args.concat(moreArgs))
    }
  }
}

const add = curry((a, b, c) => a + b + c)
add(1)(2)(3)     // 6
add(1, 2)(3)     // 6
add(1)(2, 3)     // 6
```

### 5. 防抖与节流

```javascript
// 防抖：触发后等待一段时间再执行，期间再次触发则重新计时
function debounce(fn, delay) {
  let timer = null
  return function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
    }, delay)
  }
}

// 节流：在指定时间内只执行一次
function throttle(fn, interval) {
  let lastTime = 0
  return function(...args) {
    const now = Date.now()
    if (now - lastTime >= interval) {
      lastTime = now
      fn.apply(this, args)
    }
  }
}
```

## 五、闭包的注意事项

### 循环中的经典陷阱

```javascript
// ❌ 经典问题：使用 var
for (var i = 1; i <= 3; i++) {
  setTimeout(() => console.log(i), 100)
}
// 输出：4, 4, 4（共享同一个 i）

// ✅ 解决方案一：使用 let（块级作用域）
for (let i = 1; i <= 3; i++) {
  setTimeout(() => console.log(i), 100)
}
// 输出：1, 2, 3（每次循环创建新的 i）

// ✅ 解决方案二：使用闭包（IIFE）
for (var i = 1; i <= 3; i++) {
  ;(function(j) {
    setTimeout(() => console.log(j), 100)
  })(i)
}
// 输出：1, 2, 3

// ✅ 解决方案三：使用 setTimeout 第三参数
for (var i = 1; i <= 3; i++) {
  setTimeout((j) => console.log(j), 100, i)
}
// 输出：1, 2, 3
```

### 内存泄漏风险

```javascript
function createLeak() {
  const hugeData = new Array(1000000).fill('*')

  return function() {
    // 即使不使用 hugeData，它也不会被回收
    console.log('leak')
  }
}

// ✅ 正确做法：不需要时解除引用
function createSafe() {
  let hugeData = new Array(1000000).fill('*')

  return function() {
    const result = hugeData.length
    hugeData = null  // 使用完后释放
    return result
  }
}
```

## 六、面试常见问题

### Q1：什么是闭包？为什么需要闭包？

**定义**：闭包是一个函数与其词法环境的组合，使得函数可以访问定义时作用域中的变量，即使函数在作用域外执行。

**为什么需要**：
1. **数据封装**：创建私有变量，避免全局污染
2. **状态保持**：函数可以"记住"之前的操作状态
3. **函数工厂**：根据参数生成特定行为的函数
4. **回调与异步**：保持异步操作所需的上下文

### Q2：闭包会导致内存泄漏吗？

**不一定会**。闭包本身是 JavaScript 的正常特性，GC 会正确处理被闭包引用的变量。但如果闭包**无意间**引用了不再需要的大对象，且闭包生命周期很长（如全局变量、DOM 事件监听），那么这些对象就无法被回收，形成内存泄漏。

### Q3：let 和 var 在 for 循环中为什么表现不同？

`var` 是函数作用域，整个循环共享同一个 `i` 变量，循环结束时 `i` 变为最终值。`let` 是块级作用域，**每次循环迭代都创建一个新的 `i` 副本**，每个回调函数捕获的是各自迭代时的 `i` 值。

### Q4：词法作用域和动态作用域的区别？

- **词法作用域**（JavaScript 采用）：函数的作用域在**定义时**确定
- **动态作用域**（如 Bash）：函数的作用域在**调用时**确定

```javascript
// 词法作用域
const x = 'outer'
function foo() { console.log(x) }
function bar() { const x = 'inner'; foo() }
bar()  // 'outer'（取决于 foo 定义的位置）
```
