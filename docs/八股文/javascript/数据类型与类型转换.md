# 数据类型与类型转换

> 官方文档：[MDN - JavaScript 数据类型和数据结构](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Data_structures)

---

## 一、数据类型总览

JavaScript 中数据类型分为**原始类型（Primitive）**和**引用类型（Reference）**两大类：

```
┌─ JavaScript 数据类型 ──────────────────────────────────┐
│                                                         │
│  原始类型（7 种）              引用类型                   │
│  ├── number                    ├── Object               │
│  ├── string                    ├── Array                │
│  ├── boolean                   ├── Function             │
│  ├── undefined                 ├── Date                 │
│  ├── null                      ├── RegExp               │
│  ├── symbol (ES6)              ├── Map / WeakMap         │
│  └── bigint (ES2020)           ├── Set / WeakSet         │
│                                └── 其他内置对象...        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 核心区别

| 维度 | 原始类型 | 引用类型 |
|------|---------|---------|
| 存储内容 | **值本身** | **内存地址（引用）** |
| 存储位置 | 栈（Stack） | 堆（Heap） |
| 赋值行为 | **拷贝值**（互不影响） | **拷贝引用**（指向同一对象） |
| 比较 | 比较值（`===`） | 比较引用地址 |
| 可变性 | 不可变（immutable） | 可变（mutable） |

```javascript
// 原始类型 —— 值拷贝
let a = 1
let b = a
b = 2
console.log(a)  // 1（不受影响）

// 引用类型 —— 引用拷贝
let obj1 = { name: 'Tom' }
let obj2 = obj1
obj2.name = 'Jerry'
console.log(obj1.name)  // 'Jerry'（受影响，指向同一对象）
```

## 二、原始类型详解

### 1. `number`

JavaScript 中所有数字都是**64 位双精度浮点数**（IEEE 754），不区分整数和浮点数。

```javascript
const integer = 42           // 整数
const float = 3.14           // 浮点数
const scientific = 2.5e4     // 科学计数法 → 25000
const hex = 0xff             // 十六进制 → 255
const octal = 0o77           // 八进制 → 63
const binary = 0b1010        // 二进制 → 10

// 特殊值
Number.MAX_SAFE_INTEGER      // 9007199254740991（2^53 - 1）
Number.MIN_SAFE_INTEGER      // -9007199254740991
Number.POSITIVE_INFINITY     // Infinity
Number.NEGATIVE_INFINITY     // -Infinity
NaN                          // Not a Number
```

**精度问题**：

```javascript
0.1 + 0.2 === 0.3   // false（结果是 0.30000000000000004）

// 解决方案
// 方案一：乘以倍数后计算
(0.1 * 10 + 0.2 * 10) / 10 === 0.3  // true

// 方案二：使用 toFixed
parseFloat((0.1 + 0.2).toFixed(10)) === 0.3  // true

// 方案三：使用 Number.EPSILON
Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON  // true
```

### 2. `string`

字符串是**不可变**的字符序列，使用 UTF-16 编码。

```javascript
const str = 'Hello'

// 不可变性
str[0] = 'h'           // ❌ 静默失败
str = 'hello'           // ✅ 重新赋值（创建新字符串）

// 模板字符串（ES6）
const name = 'Tom'
const greeting = `Hello, ${name}!`   // 'Hello, Tom!'

// 常用方法
str.length              // 5
str.charAt(0)           // 'H'
str.includes('ell')     // true
str.startsWith('He')    // true
str.endsWith('lo')      // true
str.slice(1, 3)         // 'el'
str.toUpperCase()       // 'HELLO'
str.split('')           // ['H', 'e', 'l', 'l', 'o']
str.repeat(3)           // 'HelloHelloHello'
str.padStart(8, '*')    // '***Hello'
str.padEnd(8, '*')      // 'Hello***'
```

### 3. `boolean`

```javascript
// 只有 true 和 false 两个值
const isActive = true
const isDeleted = false

// falsy 值（转为 false 的值）
Boolean(0)          // false
Boolean(-0)         // false
Boolean(0n)         // false
Boolean('')         // false
Boolean(null)       // false
Boolean(undefined)  // false
Boolean(NaN)        // false

// truthy 值（除 falsy 以外的所有值）
Boolean(1)          // true
Boolean('hello')    // true
Boolean([])         // true ⚠️ 空数组也是 truthy
Boolean({})         // true ⚠️ 空对象也是 truthy
Boolean('0')        // true ⚠️ 字符串 '0' 是 truthy
```

### 4. `undefined` vs `null`

| 维度 | `undefined` | `null` |
|------|------------|--------|
| 含义 | **未定义**（变量声明但未赋值） | **空值**（ intentional absence） |
| 类型 | `typeof undefined === 'undefined'` | `typeof null === 'object'`（历史 bug） |
| 转数字 | `Number(undefined) → NaN` | `Number(null) → 0` |
| 出现场景 | 变量未赋值、函数无返回值、不存在的属性 | 手动赋值为空、DOM 查询无结果 |
| 相等性 | `undefined == null` → `true` | `undefined === null` → `false` |

```javascript
let a
console.log(a)          // undefined（声明未赋值）

function foo() {}
console.log(foo())      // undefined（函数无返回值）

const obj = {}
console.log(obj.x)      // undefined（属性不存在）

// null 通常表示"有值，但值为空"
let data = null         // 初始为空，后续会赋值
```

### 5. `symbol`（ES6）

Symbol 是**唯一且不可变**的值，主要用作对象属性的键。

```javascript
// 每个 Symbol 都是唯一的
const s1 = Symbol('description')
const s2 = Symbol('description')
s1 === s2               // false（即使描述相同）

// 用作对象属性键（避免命名冲突）
const KEY = Symbol('key')
const obj = { [KEY]: 'value' }
console.log(obj[KEY])   // 'value'

// 全局 Symbol 注册表
const g1 = Symbol.for('app.key')   // 创建或获取
const g2 = Symbol.for('app.key')   // 获取已存在的
g1 === g2             // true
Symbol.keyFor(g1)     // 'app.key'
```

### 6. `bigint`（ES2020）

用于表示**任意精度**的整数，突破 `number` 的安全整数限制。

```javascript
// 定义方式
const big1 = 9007199254740993n     // 后缀 n
const big2 = BigInt(9007199254740993)  // BigInt() 函数

// 运算
100n + 200n           // 300n
100n * 200n           // 20000n
10n ** 3n             // 1000n

// ⚠️ 不能与 number 混合运算
// 100n + 1            // TypeError
Number(100n) + 1      // 101（需要显式转换）
```

## 三、类型判断

### 方法对比

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| `typeof` | 返回类型字符串 | 简单快速 | `null` 返回 `'object'`，无法区分数组/对象 |
| `instanceof` | 检测原型链 | 可检测自定义类 | 不能跨 iframe，不能检测原始类型 |
| `Object.prototype.toString.call()` | 内部 `[[Class]]` 标签 | 最准确 | 写法稍长 |
| `===` | 严格相等 | 简单 | 只能判断 `null` 和 `undefined` |

### typeof 结果表

```javascript
typeof 42              // 'number'
typeof 'hello'         // 'string'
typeof true            // 'boolean'
typeof undefined       // 'undefined'
typeof null            // 'object'      ⚠️ 历史遗留 bug
typeof Symbol()        // 'symbol'
typeof 42n             // 'bigint'
typeof {}              // 'object'
typeof []              // 'object'      ⚠️ 数组也是 object
typeof function(){}    // 'function'
typeof NaN             // 'number'      ⚠️ NaN 也是 number
```

### 推荐的类型判断方式

```javascript
// 1. typeof —— 判断原始类型（除 null）
typeof x === 'string'
typeof x === 'number' && !isNaN(x)  // 安全的数字判断

// 2. Array.isArray() —— 判断数组（推荐）
Array.isArray([1, 2, 3])   // true
Array.isArray('hello')     // false

// 3. Object.prototype.toString.call() —— 最准确
function getType(value) {
  return Object.prototype.toString.call(value).slice(8, -1).toLowerCase()
}

getType(42)           // 'number'
getType('hello')      // 'string'
getType(null)         // 'null'        ✅ 正确
getType([])           // 'array'       ✅ 区分数组
getType({})           // 'object'
getType(new Date())   // 'date'
getType(/regex/)      // 'regexp'
getType(()=>{})       // 'function'

// 4. isNaN 的正确用法
Number.isNaN(NaN)          // true
Number.isNaN('hello')      // false（不强制转换）
isNaN('hello')             // true ⚠️ 会先转换为数字
```

## 四、类型转换

### 隐式转换规则

```
┌─────────── 隐式转换场景 ───────────┐
│                                    │
│  字符串拼接 (+) → 转字符串          │
│  数学运算 (-,*,/,%) → 转数字        │
│  比较运算 (==) → 转相同类型         │
│  逻辑运算 (if, &&, ||, !) → 转布尔  │
│  模板字符串 → 转字符串              │
│                                    │
└────────────────────────────────────┘
```

### `+` 运算符转换

```javascript
// 只要有一侧是字符串，另一侧就会转为字符串
1 + '2'          // '12'（数字 1 转为字符串）
'3' + true       // '3true'
'4' + null       // '4null'
'5' + undefined  // '5undefined'
'6' + [1,2]      // '61,2'
'7' + {}         // '7[object Object]'

// 两侧都是数字 → 数学运算
1 + 2            // 3

// 注意运算顺序
1 + 2 + '3'      // '33'（先 1+2=3，再 3+'3'='33'）
'1' + 2 + 3      // '123'（'1'+2='12'，'12'+3='123'）
```

### `==` 转换规则

```javascript
// null 和 undefined 只与彼此相等
null == undefined    // true
null == 0            // false
null == ''           // false
null == false        // false
undefined == 0       // false

// 数字和字符串比较 → 字符串转数字
'1' == 1             // true
'2' == true          // false（true 转为 1，'2' 转为 2）

// 布尔值参与比较 → 先转为数字
true == 1            // true
false == 0           // true
'' == false          // true
'1' == true          // true

// 对象与原始类型比较 → 对象调用 valueOf() / toString()
[] == false          // true  → [] → '' → 0, false → 0
[] == ![]            // true  → [] → '' → 0, ![] → false → 0
[0] == false         // true  → [0] → '0' → 0, false → 0

// 对象与对象 → 比较引用
{} == {}             // false（不同引用）
[] == []             // false（不同引用）
```

> **最佳实践**：始终使用 `===`（严格相等），避免 `==` 的隐式转换陷阱。

### 显式转换

```javascript
// → String
String(42)              // '42'
(42).toString()         // '42'
String(true)            // 'true'
String(null)            // 'null'
String(undefined)       // 'undefined'
String([1, 2, 3])       // '1,2,3'

// → Number
Number('42')            // 42
Number('3.14')          // 3.14
Number('')              // 0
Number(' ')             // 0
Number(true)            // 1
Number(false)           // 0
Number(null)            // 0
Number(undefined)       // NaN
Number('hello')         // NaN
Number('42px')          // NaN
parseInt('42px', 10)    // 42  ✅ 提取整数部分
parseFloat('3.14abc')   // 3.14 ✅ 提取浮点数

// → Boolean
Boolean(0)              // false
Boolean('')             // false
Boolean(null)           // false
Boolean(undefined)      // false
Boolean(NaN)            // false
Boolean(1)              // true
Boolean('hello')        // true
Boolean([])             // true
Boolean({})             // true
```

## 五、包装类型

原始类型之所以能调用方法，是因为 JavaScript 在访问时会自动创建**包装对象**：

```javascript
const str = 'hello'

str.toUpperCase()  // 'HELLO'
// 执行过程：
// 1. 自动创建 String 包装对象：new String('hello')
// 2. 调用 toUpperCase() 方法
// 3. 销毁包装对象

// 证据：给原始类型添加属性无效
str.customProp = 'test'
console.log(str.customProp)  // undefined（包装对象已销毁）
```

## 六、面试常见问题

### Q1：typeof null 为什么是 'object'？

这是 JavaScript 的**历史遗留 bug**。在 JS 最初的实现中，值以 32 位存储，前 3 位表示类型标记，`000` 代表对象，而 `null` 的所有位都是 0，所以被错误地识别为对象。修复这个 bug 会破坏大量现有代码，因此保留至今。

### Q2：为什么 0.1 + 0.2 !== 0.3？

JavaScript 使用 IEEE 754 双精度浮点数标准，某些十进制小数无法用二进制精确表示（类似十进制无法精确表示 1/3）。0.1 和 0.2 转为二进制后是无限循环小数，相加后产生精度误差。解决方案：使用 `Number.EPSILON` 比较，或乘以倍数后转为整数运算。

### Q3：`[] == ![]` 为什么是 true？

```javascript
[] == ![]
// 1. ![] → false（[] 是 truthy，取反为 false）
// 2. [] == false → 触发隐式转换
// 3. [] → '' → 0（对象先转字符串，再转数字）
// 4. false → 0
// 5. 0 == 0 → true
```

### Q4：如何准确判断一个变量是数组？

```javascript
// 方法一：Array.isArray()（推荐 ✅）
Array.isArray(arr)

// 方法二：Object.prototype.toString.call()
Object.prototype.toString.call(arr) === '[object Array]'

// 方法三：instanceof（注意跨 iframe 限制）
arr instanceof Array

// 方法四：检查构造函数
arr.constructor === Array
```

### Q5：null 和 undefined 的区别是什么？

- `undefined` 表示"**缺失值**"——变量声明了但未赋值、函数没有返回值、对象属性不存在
- `null` 表示"**空值**"——主动设置表示"此处不应有值"，是一个有意义的赋值
- `typeof null === 'object'`（bug），`typeof undefined === 'undefined'`
- `Number(null) === 0`，`Number(undefined) === NaN`
