# TypeScript 类型守卫（Type Guard）

> 📖 [Narrowing —— TypeScript 官方文档](https://www.typescriptlang.org/docs/handbook/2/narrowing.html)
> 📖 [Assertion Functions —— TypeScript 官方文档](https://www.typescriptlang.org/docs/handbook/release-notes/typescript-3-7.html#assertion-functions)

## 一、概述

在 TypeScript 中，一个变量常常是**联合类型**（如 `string | number`）。要安全地访问某种类型独有的属性或方法，就需要先判断它「现在到底是哪一种类型」。

**类型守卫（Type Guard）** 就是用来做这件事的：它是一种**运行时检查**，TypeScript 会根据检查结果，在对应的代码分支里把类型**收窄（narrow）**成更具体的类型，从而允许你访问该类型特有的成员。

```ts
function padLeft(padding: number | string, input: string) {
  if (typeof padding === 'number') {
    // `typeof padding === 'number'` 就是一个类型守卫
    // 在这个分支里，padding 被收窄成 number，可以调用数字方法
    return ' '.repeat(padding) + input; // ✅
  }
  // 这里 padding 被收窄成 string
  return padding + input; // ✅
}
```

> 💡 **通俗理解：** 类型守卫就像门口的「**身份验证**」——通过验证（检查通过）后，TypeScript 才放行你去做该身份专属的操作。

## 二、核心概念：类型守卫与控制流分析

### 2.1 收窄（Narrowing）

把一个较宽的类型（如 `string | number`）在某个分支里**缩小**成较具体的类型（如 `number`），这个过程叫**收窄**。类型守卫是触发收窄的「检查动作」。

### 2.2 控制流分析（Control Flow Analysis）

TypeScript 会分析代码的**执行路径（可达性）**，在每个位置计算出变量「最具体的可能类型」。这种基于可达性的分析叫**控制流分析**。

```ts
function example() {
  let x: string | number | boolean;
  x = Math.random() < 0.5;
  //   ^? boolean —— 赋值后这里 x 是 boolean

  if (Math.random() < 0.5) {
    x = 'hello';
    //   ^? string —— if 分支里是 string
  } else {
    x = 100;
    //   ^? number —— else 分支里是 number
  }
  return x;
  //     ^? string | number —— 合并后是两者联合
}
```

> 💡 **提示：** 控制流分析意味着 TS 会顺着 `if/else`、三元运算符、`switch`、循环、`return`/`throw` 等**所有可能路径**逐段推断类型，不需要你手动写类型断言。

## 三、内置类型守卫

TypeScript 能识别下面几种 JavaScript 原生检查，并把它们当作类型守卫。

### 3.1 `typeof` 类型守卫

`typeof` 适合区分**原始类型**。TypeScript 能识别 `typeof` 返回的这些字符串：`"string"`、`"number"`、`"bigint"`、`"boolean"`、`"symbol"`、`"undefined"`、`"object"`、`"function"`。

```ts
function format(value: string | number) {
  if (typeof value === 'string') {
    return value.toUpperCase(); // ✅ string
  }
  return value.toFixed(2); // ✅ number
}
```

> ⚠️ **注意（经典坑）：** 在 JavaScript 中 `typeof null === "object"`！所以用 `typeof` 判断对象时，**收窄后仍可能是 `null`**：

```ts
function printAll(strs: string | string[] | null) {
  if (typeof strs === 'object') {
    // ⚠️ 这里 strs 是 string[] | null，而不是 string[]
    // 因为 typeof null 也是 "object"
    for (const s of strs) {
      // ❌ 报错：strs 可能为 null
    }
  }
}
```

判空要用 `=== null` 或真值判断（见 3.5）。

### 3.2 `instanceof` 类型守卫

`instanceof` 用来判断一个值是不是某个**类（构造函数）**的实例。它检查对象的原型链，适合区分类实例。

```ts
function logValue(x: Date | string) {
  if (x instanceof Date) {
    console.log(x.toUTCString()); // ✅ Date
  } else {
    console.log(x.toUpperCase()); // ✅ string
  }
}
```

> 💡 **提示：** `instanceof` 只能用于类，对普通对象接口（interface）无效。判断「对象是否有某个属性」用 `in`（见 3.3）。

### 3.3 `in` 类型守卫

`in` 操作符判断对象（及其原型链）**是否存在某个属性**。当左侧是字符串字面量、右侧是联合类型时，TS 会据此收窄。

```ts
type Fish = { swim: () => void };
type Bird = { fly: () => void };

function move(animal: Fish | Bird) {
  if ('swim' in animal) {
    animal.swim(); // ✅ Fish
  } else {
    animal.fly(); // ✅ Bird
  }
}
```

> ⚠️ **注意（可选属性）：** 如果某类型里这个属性是**可选**的（`swim?`），那么它**在 `in` 的两个分支里都可能存在**。例如 `Human` 既可能 `swim` 也可能 `fly`，那么 `Human` 会同时出现在 `in` 检查的两边：

```ts
type Human = { swim?: () => void; fly?: () => void };

function move(animal: Fish | Bird | Human) {
  if ('swim' in animal) {
    animal; // 👉 Fish | Human
  } else {
    animal; // 👉 Bird | Human
  }
}
```

### 3.4 相等性收窄（Equality Narrowing）

用 `===`、`!==`、`==`、`!=` 比较也能收窄。

```ts
function example(x: string | number, y: string | boolean) {
  if (x === y) {
    // 两者相等 → 类型也相等，交集是 string
    x.toUpperCase(); // ✅ string
    y.toLowerCase(); // ✅ string
  }
}
```

> 💡 **提示（== 的特殊能力）：** 用 `== null` 可以**同时**排除 `null` 和 `undefined` 两种情况（因为 `null == undefined` 为真）：

```ts
interface Container {
  value: number | null | undefined;
}

function multiplyValue(container: Container, factor: number) {
  if (container.value != null) {
    // value 被收窄为 number（同时去掉了 null 和 undefined）
    container.value *= factor; // ✅
  }
}
```

### 3.5 真值收窄（Truthiness）

在 `if`、`&&`、`||`、`!` 等条件里，JavaScript 会把值「强制转换」成布尔。下面这些「假值」会被收窄掉：`false`、`0`、`NaN`、`""`、`0n`、`null`、`undefined`。

```ts
function multiplyAll(values: number[] | undefined, factor: number) {
  if (!values) {
    return values; // 👉 undefined
  }
  return values.map((x) => x * factor); // 👉 number[]
}
```

> ⚠️ **注意：** 真值收窄对**原始类型要谨慎**。比如 `if (str)` 不仅排除了 `null`/`undefined`，也排除了**空字符串 `""`**，这可能不符合你的预期。需要精确控制时，用 `=== null` 这种显式判断。

## 四、用户自定义类型守卫（Type Predicate）⭐

当内置守卫不够用（比如要检查对象的多个属性、运行时方法返回值等），可以**自己写类型守卫函数**：让函数的返回类型是一个**类型谓词（type predicate）**。

### 4.1 基本语法：`参数 is 类型`

类型谓词的写法是 `参数名 is 类型`，其中 `参数名` 必须是当前函数签名里的某个参数。

```ts
type Fish = { swim: () => void };
type Bird = { fly: () => void };

// 返回类型是 pet is Fish，这就是类型谓词
function isFish(pet: Fish | Bird): pet is Fish {
  return (pet as Fish).swim !== undefined;
}

let pet: Fish | Bird = getSmallPet();

if (isFish(pet)) {
  pet.swim(); // ✅ TS 知道这里是 Fish
} else {
  pet.fly(); // ✅ else 分支里 pet 是 Bird
}
```

> 💡 **关键：** 类型谓词不仅能让 `if` 分支收窄，**`else` 分支也会自动排除对应类型**（如上例 else 里 `pet` 只能是 `Bird`）。

### 4.2 配合数组过滤（`filter`）

类型守卫函数可以直接传给数组的 `filter`，TS 会据此推断过滤后的数组类型：

```ts
const zoo: (Fish | Bird)[] = [getSmallPet(), getSmallPet()];

// 过滤后类型是 Fish[]，而不是 (Fish | Bird)[]
const underWater: Fish[] = zoo.filter(isFish);
```

> 💡 **提示：** 这是非常实用的技巧——用类型守卫过滤数组，结果数组会自动带上精确的元素类型。

### 4.3 在类中使用 `this is 类型`

类的方法也可以用类型谓词来收窄 `this` 的类型：

```ts
class FileSystemObject {
  isFile(): this is File {
    return this instanceof File;
  }
  isDirectory(): this is Directory {
    return this instanceof Directory;
  }
  // ...
}

const node = getFSObject();
if (node.isFile()) {
  node.read(); // ✅ this 被收窄为 File
}
```

### 4.4 实战：常用类型守卫封装

实际项目里，把常见判断封装成类型守卫，能大幅提升类型安全和代码可读性：

```ts
// 判断非空数组：过滤掉空数组
function isNonEmptyArray<T>(value: T[]): value is [T, ...T[]] {
  return value.length > 0;
}

// 判断 Error（try/catch 里的 e 是 unknown）
function isError(value: unknown): value is Error {
  return value instanceof Error;
}

// 判断 DOM 元素
function isInputElement(el: HTMLElement): el is HTMLInputElement {
  return el.tagName === 'INPUT';
}

// 使用
try {
  doSomething();
} catch (e) {
  if (isError(e)) {
    console.error(e.message); // ✅ Error
  }
}
```

## 五、判别联合（Discriminated Union）

当联合类型的每个成员都带一个**相同的字面量类型字段**（通常叫 `kind` / `type`），TS 就能通过这个字段精确收窄。这是处理「多种形态数据」的**最佳实践**。

### 5.1 定义判别联合

```ts
interface Circle {
  kind: 'circle';   // 👈 判别字段：字面量类型
  radius: number;
}
interface Square {
  kind: 'square';   // 👈 同名字段，不同字面量
  sideLength: number;
}

type Shape = Circle | Square;
```

> ⚠️ **注意：** 判别字段必须是**字面量类型**（如 `'circle'`，而不是 `string`），TS 才能据此区分。如果写成 `kind: string`，就失去了判别能力。

### 5.2 用 `switch` 收窄

```ts
function getArea(shape: Shape) {
  switch (shape.kind) {
    case 'circle':
      return Math.PI * shape.radius ** 2; // ✅ shape 是 Circle
    case 'square':
      return shape.sideLength ** 2;       // ✅ shape 是 Square
  }
}
```

### 5.3 `never` 穷尽检查（Exhaustiveness Checking）

用 `never` 类型可以**强制所有分支都被处理**。将来新增类型却忘了加 `case`，TS 会在编译期报错：

```ts
function getArea(shape: Shape) {
  switch (shape.kind) {
    case 'circle':
      return Math.PI * shape.radius ** 2;
    case 'square':
      return shape.sideLength ** 2;
    default:
      // 所有 case 都处理时，shape 在这里是 never（不报错）
      // 一旦新增 Triangle 却没加 case，shape 就不再是 never → 编译报错
      const _exhaustiveCheck: never = shape;
      return _exhaustiveCheck;
  }
}
```

> 💡 **提示：** `never` 是「不可能出现的类型」。`never` 可以赋给任何类型，但任何类型都不能赋给 `never`（除了 `never` 自身）。利用这一点做穷尽检查，能在新增分支时**自动提醒你补全逻辑**。

## 六、断言函数（Assertion Functions）

断言函数是类型守卫的「**异常版**」：它不返回布尔值，而是**在条件不满足时抛出错误**。TS 3.7+ 支持用 `asserts` 关键字声明。

### 6.1 `asserts 参数 is 类型`

声明函数「执行后，参数就是某类型」，否则抛错。

```ts
// 断言：执行后 val 是 string
function assertIsString(val: any): asserts val is string {
  if (typeof val !== 'string') {
    throw new Error(`期望 string，实际是 ${typeof val}`);
  }
}

function processInput(input: unknown) {
  assertIsString(input); // 调用后...
  input.toUpperCase();   // ✅ input 被收窄为 string（不需要 if 判断）
}
```

### 6.2 `asserts 参数 is NonNullable<T>`（断言非空）

这是断言函数最常用的场景——**收窄掉 `null` / `undefined`**。

```ts
function assertNonNull<T>(val: T | null | undefined): asserts val is NonNullable<T> {
  if (val === null || val === undefined) {
    throw new Error('值为 null 或 undefined');
  }
}

const el = document.getElementById('app'); // HTMLElement | null
assertNonNull(el);
el.innerHTML = 'hello'; // ✅ el 收窄为 HTMLElement
```

> 💡 **提示：** `NonNullable<T>` 是内置工具类型，作用是「从 T 中排除 null 和 undefined」。

### 6.3 `asserts condition`（条件断言）

不带类型参数，只断言「某个条件为真」，执行后 TS 把后面的代码当作「条件成立」。

```ts
function assert(condition: any, msg?: string): asserts condition {
  if (!condition) {
    throw new Error(msg ?? '断言失败');
  }
}

function divide(a: number | null, b: number | null) {
  assert(a !== null && b !== null, 'a 和 b 不能为 null');
  assert(b !== 0, '除数不能为 0');
  return a / b; // ✅ a、b 都是 number
}
```

### 6.4 断言函数 vs 类型守卫函数

两者都能收窄类型，但语义和用法不同：

| 对比项 | 类型守卫函数（`x is T`） | 断言函数（`asserts x is T`） |
| --- | --- | --- |
| 返回值 | 返回 `boolean` | 不返回（无返回值或 `void`） |
| 失败时 | 返回 `false`，走 `else` | **抛出异常**，中断执行 |
| 使用方式 | 配合 `if` 判断 | 直接调用，调用后自动收窄 |
| 典型场景 | 条件分支、数组过滤 | 前置校验、不变式保证 |

```ts
// 类型守卫：失败走 else
if (isString(val)) {
  val.toUpperCase();
} else {
  // 处理非字符串
}

// 断言函数：失败直接抛错
assertIsString(val);
val.toUpperCase(); // 直接用
```

## 七、类型守卫速查表

| 类型守卫 | 写法 | 适用场景 |
| --- | --- | --- |
| `typeof` | `typeof x === 'string'` | 区分原始类型 |
| `instanceof` | `x instanceof Date` | 区分类实例 |
| `in` | `'swim' in animal` | 区分对象的不同属性 |
| 相等性 | `x === 'circle'` / `x != null` | 与字面量/变量比较 |
| 真值 | `if (x)` / `if (!x)` | 排除假值（含 null/undefined） |
| 类型谓词 | `function isX(x): x is T` | 自定义复杂检查 |
| 判别联合 | `switch (x.kind)` | 多形态数据 |
| 断言函数 | `asserts x is T` | 校验失败即抛错 |

## 八、常见问题与注意事项

### 8.1 `typeof null` 是 `"object"`

记住这个 JavaScript 历史遗留：`typeof null === "object"`、`typeof [] === "object"`。判断数组用 `Array.isArray`，判空用 `=== null`。

### 8.2 真值收窄会把「合法值」也收窄掉

`if (value)` 会把 `0`、`""`、`NaN` 这些**有效但为假**的值也排除掉。如果 `0` 或空字符串是合法输入，要避免真值判断，改用 `=== undefined` / `=== null`。

### 8.3 类型守卫函数必须「诚实」

类型谓词只是一种**声明**，TS 不会验证你的函数实现是否真的和谓词一致。如果 `isFish` 内部判断逻辑写错了，TS 仍会按谓词收窄，导致运行时出错。

```ts
// ❌ 实现和谓词不一致：谓词说是 Fish，实现却在判断 fly
function isFish(pet: Fish | Bird): pet is Fish {
  return (pet as Bird).fly !== undefined; // 逻辑写反了
}
```

> 💡 **提示：** 写类型守卫时，务必让「运行时判断」和「谓词声明」保持一致。

### 8.4 收窄在「跨函数」后可能失效

把收窄后的变量传进函数、或被闭包重新赋值，TS 不保证类型仍收窄。复杂场景里，把收窄结果先存到 `const`：

```ts
let val: string | null = getValue();
if (val !== null) {
  // val 收窄为 string
  someAsyncFn(() => {
    console.log(val.length); // ⚠️ 闭包里 TS 仍可能认为 val 是 string | null
  });
}
```

### 8.5 断言函数必须抛错

`asserts` 语义要求：条件不成立时**必须抛出异常**（不能只是 return）。否则 TS 误以为已收窄，但程序继续执行，会埋下隐患。

## 九、总结

- **类型守卫**是触发收窄的运行时检查；**收窄**是「把宽类型缩小成具体类型」的结果；二者靠**控制流分析**自动完成；
- 内置守卫：`typeof`（原始类型）、`instanceof`（类实例）、`in`（属性存在）、相等性、真值；
- 自定义守卫用**类型谓词** `参数 is 类型`，可配合 `filter`、类方法 `this is T`；
- **判别联合**（公共字面量字段 + `switch`）+ **`never` 穷尽检查**是处理多形态数据的最佳实践；
- **断言函数**（`asserts x is T`）是「抛错版」类型守卫，适合前置校验，常用于断言非空（`asserts x is NonNullable<T>`）。

> 💡 **一句话记忆：** 类型守卫 = 一道「检查关卡」，TS 据此在分支里把类型收窄；内置守卫覆盖常见检查，自定义谓词（`is T`）处理复杂场景，断言函数（`asserts`）则用「抛错」来保证类型不变式。

> 📎 相关阅读：实际开发中的收窄场景可参考 [[疑难问题]] 的「联合类型怎么收窄」。
