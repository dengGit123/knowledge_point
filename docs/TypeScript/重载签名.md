# TypeScript 函数重载（重载签名）

> 📖 [More on Functions - Function Overloads —— TypeScript 官方文档](https://www.typescriptlang.org/docs/handbook/2/functions.html#function-overloads)

## 一、概述

**函数重载（Function Overloads）** 是指：为**同一个函数**提供多个**不同的类型签名**，让 TypeScript 能根据传入参数的类型/个数，给出精确的返回类型和参数校验。

```ts
// 同一个函数，不同参数 → 不同返回类型
function parse(input: string): object;   // 传字符串返回对象
function parse(input: string[]): number; // 传数组返回数字
```

> 💡 **通俗理解：** 函数重载就像「**一个岗位多个招聘要求**」——同样是这个岗位（函数），但你按不同的方式应聘（传不同参数），会被分配到不同的工作（不同返回类型）。

> 💡 **为什么不用联合类型参数？** 如果用 `function f(x: string | number): A | B`，调用方拿到的是 `A | B` 的宽泛类型，仍需自己收窄。**重载能让每个调用精确返回对应类型**，类型体验更好（见第八节对比）。

## 二、核心概念：重载签名 vs 实现签名

重载由两部分组成：

| 组成 | 作用 | 是否有函数体 | 对外是否可见 |
| --- | --- | --- | --- |
| **重载签名（overload signatures）** | 描述函数对外支持的多种调用方式 | ❌ 无 | ✅ 对外暴露 |
| **实现签名（implementation signature）** | 提供真正的函数实现 | ✅ 有 | ❌ **不对外暴露** |

> ⚠️ **重要提醒：实现签名本身不对外暴露。** 从外部调用时，TypeScript **只会看到重载签名**，实现签名仅用于内部类型检查和函数体。如果你调用了只有实现签名才支持、但重载签名里没有的参数形式，会报错。

实现签名必须**兼容所有重载签名**：它能接受所有重载声明的参数，并能返回每个重载声明的返回类型。

## 三、函数重载

### 3.1 基本写法

```ts
// 定义重载签名（可写多个，注意：这些行没有函数体，以分号结尾）
function greet(name: string): string;               // 签名1：只接受名字
function greet(name: string, age: number): string;  // 签名2：接受名字和年龄
// 实现签名（函数体）—— 参数用可选参数，返回值兼容所有签名
function greet(name: string, age?: number): string {
  if (age !== undefined) {
    return `Hello ${name}, you are ${age} years old.`;
  }
  return `Hello ${name}`;
}

// 调用
const result1 = greet('Bob');      // ✅ 匹配签名1，返回 string
const result2 = greet('Bob', 25);  // ✅ 匹配签名2，返回 string
```

### 3.2 参数规则

- 实现签名的参数类型，必须是**所有重载签名参数类型的超集**（通常用联合类型 + 可选参数实现）；
- 实现签名的参数个数，**可以少于**重载签名的参数个数（函数体可不依赖所有参数）；
- 实现签名在某个位置的参数，必须兼容所有重载签名**该位置**的参数（可用联合类型或可选参数实现）。

### 3.3 返回值规则

实现签名的返回类型，必须是**所有重载签名返回类型的超类型**（因为实现必须能返回每个重载声明的类型）。严格来说 TS 并不强制一定要写成联合类型，但写成「所有重载返回类型的联合」最安全、最清晰。

## 四、方法重载（在类中使用）

方法重载的语法与函数重载完全一致，区别仅在于它位于**类内部**：

```ts
class ArrayHelper {
  // 重载签名1：通过索引删除
  remove(index: number): number;
  // 重载签名2：通过元素值删除
  remove(element: string): string;
  // 实现签名
  remove(value: number | string): number | string {
    if (typeof value === 'number') {
      // 通过索引删除的逻辑
      return value; // 返回被删除元素的索引
    } else {
      // 通过元素值删除的逻辑
      return value; // 返回被删除的元素
    }
  }
}
```

## 五、构造函数重载

TypeScript 也支持构造函数的重载，所有重载签名共享同一个构造函数实现：

```ts
class Rectangle {
  public width: number;
  public height: number;

  // 重载签名1：分别传入宽高
  constructor(width: number, height: number);
  // 重载签名2：传入配置对象
  constructor(config: { width: number; height: number });
  // 实现签名
  constructor(v1: any, v2?: any) {
    if (typeof v1 === 'object') {
      this.width = v1.width;
      this.height = v1.height;
    } else {
      this.width = v1;
      this.height = v2;
    }
  }

  getArea(): number {
    return this.width * this.height;
  }
}

const rect1 = new Rectangle(10, 20);                      // ✅ 匹配签名1
const rect2 = new Rectangle({ width: 10, height: 20 });   // ✅ 匹配签名2
```

## 六、用 `interface` / `type` 描述重载

除了「多个 function 声明 + 实现」，还可以用 `interface` 或 `type` 给函数类型定义重载，再配合一个实现。常用于**给变量（如箭头函数、从别处导入的函数）补类型**：

```ts
// 用 interface 描述重载的函数类型
interface Greet {
  (name: string): string;
  (name: string, age: number): string;
}

// 实现可以是一个箭头函数（function 声明之外的形式）
const greet: Greet = (name: string, age?: number) => {
  if (age !== undefined) return `Hello ${name}, ${age}`;
  return `Hello ${name}`;
};
```

> 💡 **提示：** 普通的 `function` 重载只能用「多个 function 声明」写。如果要把重载类型赋给一个变量（箭头函数、`const` 函数），就用 `interface` 或 `type` 描述。

## 七、注意事项与陷阱

### 7.1 重载签名的顺序：具体在前

TypeScript 编译器**按签名声明顺序依次匹配**，匹配到第一个就停止。因此应把**更具体的重载放在前面**，更通用的放在后面，否则通用签名会「抢先」匹配。

```ts
// ❌ 顺序错误：string 会先匹配上，永远走不到 string[] 分支
function bad(x: string): string;
function bad(x: string[]): number;
```

### 7.2 实现签名不对外暴露

调用方只能用到**重载签名**里声明过的形式。实现签名里的「更宽松」参数对外不可见：

```ts
function fn(x: string): void;
function fn(x: string, y: number): void;
function fn(x: string, y?: number): void { /* 实现 */ }

fn('a', 2);     // ✅ 重载签名2 允许
fn('a', 2, 3);  // ❌ 报错：没有匹配的重载（实现签名虽然能接受，但不对外）
```

### 7.3 避免返回 `any`

不要为了省事把重载返回类型写成 `any`，这会让重载的**类型安全优势完全丧失**。每个重载签名都应给出精确的返回类型。

### 7.4 组合爆炸：改用配置对象

当函数需要支持大量参数组合时，重载签名会**组合爆炸**。此时应改用**配置对象模式**（参数对象化），既保持类型精确，又大幅简化签名：

```ts
// ❌ 不推荐：过多的重载
function render(width: number, height: number): void;
function render(width: number, height: number, color: string): void;
function render(width: number, height: number, color: string, opacity: number): void;
// ... 更多组合

// ✅ 推荐：配置对象模式
interface RenderOptions {
  width: number;
  height: number;
  color?: string;
  opacity?: number;
}
function render(options: RenderOptions): void { /* 实现... */ }
```

### 7.5 实现签名必须兼容所有重载

```ts
// ❌ 错误：实现签名无法处理 string 输入（签名1 要求接受 string）
function process(input: string): number;
function process(input: number): string;
function process(input: number): string { // ❌ 缺少对 string 的处理
  return input.toFixed(2);
}

// ✅ 正确：实现签名用联合类型，覆盖所有重载
function process(input: string): number;
function process(input: number): string;
function process(input: string | number): string | number {
  if (typeof input === 'string') return input.length;
  return input.toFixed(2);
}
```

## 八、重载 vs 联合类型参数：如何选择

| 场景 | 推荐方式 | 原因 |
| --- | --- | --- |
| 不同参数**对应不同返回类型** | **重载** | 联合参数返回 `A \| B`，调用方还要收窄；重载能精确返回 |
| 参数类型不同但**返回类型一致** | **联合参数** | 无需重载，一个签名搞定，更简洁 |
| 参数个数灵活（可选参数） | **可选参数** | 不必用重载 |
| 大量参数组合 | **配置对象** | 避免组合爆炸 |

```ts
// 重载更优：传 string 返回 number，传 number 返回 string（返回类型不同）
function transform(x: string): number;
function transform(x: number): string;

// 联合参数够用：两种输入都返回同样的类型
function log(msg: string | number): void;
```

## 九、总结

- 函数重载 = **多个对外可见的重载签名** + **一个对内不可见的实现签名**；
- 实现签名必须**兼容所有重载**（参数是超集，返回是超类型）；
- 重载签名**按顺序匹配**，具体在前、通用在后；
- 实现签名**不对外暴露**，调用方只能用重载声明的形式；
- 不同输入对应不同返回类型时用重载；输入不同但返回相同时用联合参数；参数组合多时用配置对象。

> 💡 **一句话记忆：** 重载 = 「**重载签名管对外（按顺序匹配、精确返回），实现签名管对内（兼容所有重载）**」；用它的标准是「**不同参数 → 不同返回类型**」，否则联合参数更省事。
