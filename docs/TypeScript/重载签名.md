### 重载签名
* 重载签名： 同一个函数提供多个不同的类型签名，不包含函数体，仅用于类型检查
* 实现签名: 位于所有重载前面之后，包含实际的函数体。它的类型必须能够兼容所有重载签名
> 💡 重要提醒：实现签名本身不对外暴露。从外部调用时，TypeScript 只会看到重载签名，而实现签名仅用于内部类型检查
### 1. 函数重载：函数声明
* 实现签名的参数类型必须是所有重载签名参数类型的超集（通常使用联合类型 + 可选参数），返回值也必须是所有重载签名返回值的子类型（因为实现必须能返回每一重载声称的返回类型）
#### 1. 参数规则：
  * 不管重载签名有几个参数，参数类型如何，实现签名可以是一个无参函数
  * 实现签名的参数个数可以少于重载签名的参数个数
  * 实现签名的某个位置的参数，必须兼容所有重载签名**该位置**的参数类型(可以通过联合类型或者可选参数实现)
#### 2. 返回值规则
严格来说，TS 并未强制要求实现签名必须用联合类型，理论上下面的写法也是可行的：
```ts
// 定义重载签名（可写多个）
function greet(name: string): string;               // 签名1：只接受名字
function greet(name: string, age: number): string;  // 签名2：接受名字和年龄
// 实现签名（函数体）- 注意参数使用可选成员，返回值使用联合类型
function greet(name: string, age?: number): string {
    if (age !== undefined) {
        return `Hello ${name}, you are ${age} years old.`;
    }
    return `Hello ${name}`;
}

// 调用
const result1 = greet("Bob");        // ✅ 匹配签名1，类型为 string
const result2 = greet("Bob", 25);    // ✅ 匹配签名2，类型为 string
```
### 2. 方法重载：在类中的使用
方法重载的语法与函数重载完全一致，区别仅在于它位于类内部：
```ts
class ArrayHelper {
    // 重载签名1：通过索引删除
    remove(index: number): number;
    // 重载签名2：通过元素值删除
    remove(element: string): string;
    // 实现签名
    remove(value: number | string): number | string {
        if (typeof value === "number") {
            // 通过索引删除逻辑
            return value; // 返回被删除元素的索引
        } else {
            // 通过元素值删除逻辑
            return value; // 返回被删除的元素
        }
    }
}
```
### 3. 构造函数重载
* ypeScript 也支持构造函数的重载，但所有重载签名必须共享同一个构造函数实现
```ts
class Rectangle {
    public width: number;
    public height: number;

    // 重载签名1：分别传入宽高
    constructor(width: number, height: number);
    // 重载签名2：传入配置对象
    constructor(config: { width: number; height: number });
    // 实现签名
    constructor(v1: any, v2?: any) {
        if (typeof v1 === 'object') {
            this.width = v1.width;
            this.height = v1.height;
        } else {
            this.width = v1;
            this.height = v2;
        }
    }

    getArea(): number {
        return this.width * this.height;
    }
}

const rect1 = new Rectangle(10, 20);   // ✅ 匹配签名1
const rect2 = new Rectangle({ width: 10, height: 20 });  // ✅ 匹配签名2
```
### 4. 核心注意事项与常见陷阱
#### 1. 重载签名的顺序：TypeScript 编译器按签名声明顺序依次匹配，因此应将更具体的重载放在前面，更通用的放在后面
#### 2. 返回 any 的类型隐患：不要为了方便而简化返回类型，这会使重载的类型安全优势完全丧失
#### 3. 重载签名过多时的设计良策：当函数需要支持大量参数组合时，重载签名会急剧增加（组合爆炸），导致类型膨胀。此时应改用配置对象模式（即参数对象化），这样既维护了类型的精确性，也大幅简化了函数签名
```ts
// ❌ 不推荐：过多的重载
function render(width: number, height: number): void;
function render(width: number, height: number, color: string): void;
function render(width: number, height: number, color: string, opacity: number): void;
// ... 更多组合

// ✅ 推荐：配置对象模式
interface RenderOptions {
    width: number;
    height: number;
    color?: string;
    opacity?: number;
}
function render(options: RenderOptions): void;
function render(options: RenderOptions): void {
    // 实现...
}
```
#### 4.实现签名不兼容导致的编译错误
```ts
// ❌ 错误示例
function process(input: string): number;
function process(input: number): string;
function process(input: number): string {  // ❌ 实现签名无法处理 string 输入
    return input.toFixed(2);
}

// ✅ 正确：实现签名使用联合类型
function process(input: string): number;
function process(input: number): string;
function process(input: string | number): string | number {
    if (typeof input === 'string') return input.length;
    return input.toFixed(2);
}
```
