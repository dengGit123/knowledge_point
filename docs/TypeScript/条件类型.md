# TypeScript 条件类型（Conditional Types）

> 📖 [Conditional Types —— TypeScript 官方文档](https://www.typescriptlang.org/docs/handbook/2/conditional-types.html)
> 📖 [Conditional Type Inference（infer）—— TypeScript 官方文档](https://www.typescriptlang.org/docs/handbook/2/conditional-types.html#inferring-within-conditional-types)

## 一、概述

**条件类型**是一种「**根据类型关系来选择类型**」的机制，写法很像 JavaScript 的三元运算符。它让我们能像写逻辑一样去「**计算类型**」，是构建高级工具类型的核心。

```ts
// 如果 T 是 string 的子类型，结果就是 'yes'，否则是 'no'
type IsString<T> = T extends string ? 'yes' : 'no';

type A = IsString<'hello'>; // 👉 'yes'
type B = IsString<42>;      // 👉 'no'
```

> 💡 **通俗理解：** 普通类型是「静态描述」，条件类型是「**类型层面的 if-else**」——根据输入类型满足的条件，输出不同的类型。

> 📎 相关：条件类型常和 `extends` 约束（见 [[extends]]）、`infer` 提取（见 [[infer]]）配合使用。

## 二、基本语法

条件类型的形式是 `T extends U ? X : Y`：

- `T extends U`：判断 `T` 是否**可赋值给** `U`（即 `T` 是 `U` 的子类型）；
- 成立 → 结果是 `X`；
- 不成立 → 结果是 `Y`。

```ts
// 根据传入类型选择不同的输出类型
type ResultType<T> = T extends string ? string[] : number[];

type R1 = ResultType<'a'>;   // string[]
type R2 = ResultType<123>;   // number[]
```

> 💡 **提示：** 这里的 `extends` 是「**类型可赋值性**」判断，不是面向对象的「继承」。比如 `42 extends number` 为真，`'a' extends string` 为真。

## 三、分配条件类型（Distributive）⭐

这是条件类型**最重要也最容易踩坑**的特性。

**规则**：当条件类型的 `extends` 左侧是一个「**裸类型参数**」（直接用泛型参数 `T`，没有被包裹），并且传入的 `T` 是**联合类型**时，条件类型会**分配（distribute）**到联合的每个成员上，最后再把结果联合起来。

```ts
type ToArray<T> = T extends any ? T[] : never;

// T = string | number 会分配成：ToArray<string> | ToArray<number>
type R = ToArray<string | number>;
// 👉 string[] | number[]   （不是 (string | number)[] ）
```

**分配的过程**等价于：

```ts
// ToArray<string | number>
// 拆开：
//   ToArray<string>  → string[]
//   ToArray<number>  → number[]
// 合并：string[] | number[]
```

### 3.1 利用分配律：`Exclude` 与 `Extract`

TypeScript 内置的 `Exclude`、`Extract` 正是利用分配律实现的：

```ts
// Exclude：从 T 中排除可赋值给 U 的成员（命中就丢弃 → never）
type MyExclude<T, U> = T extends U ? never : T;

type T1 = MyExclude<'a' | 'b' | 'c', 'a'>;      // 👉 'b' | 'c'
//   'a' → 命中 → never
//   'b' → 不命中 → 'b'
//   'c' → 不命中 → 'c'
// 合并（never 会被吞掉）：'b' | 'c'

// Extract：从 T 中提取可赋值给 U 的成员
type MyExtract<T, U> = T extends U ? T : never;
type T2 = MyExtract<'a' | 'b' | 1, string>;     // 👉 'a' | 'b'
```

> 💡 **提示：** `never` 在联合类型中会被**自动忽略**（`'b' | never | 'c'` 等于 `'b' | 'c'`），这正是 `Exclude` 能「排除」成员的原理。

### 3.2 阻止分配（用方括号包裹）

如果**不想**分配（想整体判断联合类型），就用**方括号**把 `T` 包起来，让它不再是「裸类型参数」：

```ts
// 不分配：整体判断
type ToArrayNonDist<T> = [T] extends [any] ? T[] : never;

type R = ToArrayNonDist<string | number>;
// 👉 (string | number)[]   —— 整体，而不是 string[] | number[]
```

> ⚠️ **注意：** 这是面试和实战的高频考点——**裸类型参数会分配，方括号包裹则不分配**。判断联合「整体」是否可赋值给某类型时，务必用 `[T] extends [U]`。

## 四、`infer`：在条件类型里提取类型

`infer` 关键字用于在条件类型匹配过程中**声明并提取**某个类型变量。它和条件类型是「黄金搭档」。

```ts
// 提取函数的返回类型
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type R = MyReturnType<() => string>; // 👉 string
```

**理解**：`infer R` 的意思是「**先假设 `T` 是一个函数，把这个函数的返回类型捕获到变量 `R` 里**」。如果匹配成功，`R` 就能在这个分支里使用；如果 `T` 根本不是函数，就走 `never` 分支。

### 4.1 常见提取场景

```ts
// 提取函数的参数类型（元组形式）
type MyParameters<T> = T extends (...args: infer P) => any ? P : never;
type P = MyParameters<(a: number, b: string) => void>; // 👉 [number, string]

// 提取 Promise 内部类型
type Unpack<T> = T extends Promise<infer U> ? U : T;
type U = Unpack<Promise<number>>; // 👉 number

// 提取数组的第一个元素
type First<T extends any[]> = T extends [infer F, ...any[]] ? F : never;
type F = First<[1, 2, 3]>; // 👉 1
```

> 💡 **提示：** `infer` 必须写在条件类型的 `extends` 子句里，不能单独使用。详见 [[infer]]。

### 4.2 同一位置多个 `infer` 与约束

```ts
// 取函数第一个参数和返回类型
type FirstParamAndReturn<T> = T extends (a: infer A, ...args: any[]) => infer R
  ? [A, R]
  : never;

// 给 infer 的类型加约束（TS 4.7+）
type GetReturn<T> = T extends (...args: any[]) => infer R extends string ? R : never;
```

## 五、内置工具类型的原理

TypeScript 的很多内置工具类型，本质都是**条件类型 + `infer`**。理解它们的实现，能帮你写出自己的工具类型：

| 工具类型 | 实现（简化） | 作用 |
| --- | --- | --- |
| `Exclude<T, U>` | `T extends U ? never : T` | 从联合 T 中排除可赋值给 U 的成员 |
| `Extract<T, U>` | `T extends U ? T : never` | 从联合 T 中提取可赋值给 U 的成员 |
| `NonNullable<T>` | `T extends null \| undefined ? never : T` | 排除 null 和 undefined |
| `Parameters<T>` | `T extends (...args: infer P) => any ? P : never` | 提取函数参数元组 |
| `ReturnType<T>` | `T extends (...args: any) => infer R ? R : any` | 提取函数返回类型 |
| `ConstructorParameters<T>` | `T extends new (...args: infer P) => any ? P : never` | 提取构造函数参数 |
| `InstanceType<T>` | `T extends new (...args: any) => infer R ? R : any` | 提取构造函数实例类型 |
| `Awaited<T>` | 递归展开 Promise | 提取 await 的最终类型 |

```ts
// NonNullable 的典型应用：从 catch 的 unknown 收窄
type T = NonNullable<string | null | undefined>; // 👉 string
```

## 六、常见实战模式

### 6.1 深度可选 / 深度只读

```ts
// 递归地把所有属性变成可选
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};
```

### 6.2 根据条件选择属性类型

```ts
// 只读地把 string 类型属性转成 number
type StringToNumber<T> = {
  [P in keyof T]: T[P] extends string ? number : T[P];
};
```

### 6.3 提取符合某类型的属性键

```ts
// 挑出值为函数类型的属性名
type FunctionKeys<T> = {
  [K in keyof T]: T[K] extends Function ? K : never;
}[keyof T];

interface Obj { name: string; run: () => void; stop: () => void }
type FK = FunctionKeys<Obj>; // 👉 'run' | 'stop'
```

### 6.4 判断两个类型是否相等（整体判断）

```ts
// 用方括号阻止分配，做整体相等判断
type IsEqual<A, B> = [A] extends [B] ? ([B] extends [A] ? true : false) : false;

type E1 = IsEqual<1 | 2, 1 | 2>; // true
type E2 = IsEqual<1 | 2, 1 | 3>; // false
```

## 七、注意事项与陷阱

| 陷阱 | 说明 | 解决 |
| --- | --- | --- |
| **分配律的意外** | 裸 `T extends U` 对联合会分配，可能不是你想要的 | 用 `[T] extends [U]` 整体判断 |
| **`extends` 是可赋值，不是严格相等** | `1 extends number` 为真，`number extends 1` 为假 | 判等用双向 `extends`（见 6.4） |
| **`infer` 用错位置** | `infer` 只能在条件类型的 `extends` 子句里 | 需要提取得先有条件类型包裹 |
| **递归条件类型深度** | 深度递归（如深度 Partial/readonly）有层数限制 | 避免无限递归，必要时分层 |
| **`any` 的传染性** | `any extends X` 两个分支都会走（结果为 `X \| Y` 的联合） | 处理 `any` 时需特别小心 |

> ⚠️ **注意：** 条件类型对 `boolean` 这种本质是 `true | false` 联合的类型也会分配。`T extends true` 在 `T = boolean` 时行为可能出乎意料，必要时用方括号包裹。

## 八、总结

- 条件类型 = 「**类型层面的 if-else**」，语法 `T extends U ? X : Y`；
- **分配律**：裸类型参数遇到联合会「拆开分别判断再合并」，是 `Exclude`/`Extract` 的原理；想整体判断就用 `[T] extends [U]`；
- **`infer`** 在条件类型里「捕获并提取」类型（函数返回值、参数、Promise 内部值等）；
- 内置工具类型（`ReturnType`/`Parameters`/`NonNullable` 等）大多由「条件类型 + `infer`」实现；
- 实战常用于：深度映射、按值类型筛选属性、类型相等判断。

> 💡 **一句话记忆：** 条件类型是「类型的 if-else」，`extends` 判断、`infer` 提取；记住「**裸参数对联合会分配，方括号包裹就整体判断**」，就能驾驭大多数高级工具类型。
