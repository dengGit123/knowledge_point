### 类型兼容性
* 类型兼容性用于确定一个类型是否能赋值给其他类型
* 类型兼容性是基于结构子类型的，即结构相同的类型被认为是兼容的

### 1.结构化类型系统的基本规则
* 如果 X 可以赋值给 Y，那么 Y 的每个属性在 X 中都有对应的同名属性
* 如果 Y 有可选属性，那么 X 中可以不存在这些属性
* 如果 Y 有只读属性，那么 X 中可以不存在这些属性
```typescript
interface X {
  a: any;
  b?: any;
}
interface Y {
  a: any;
}
let x: X = { a: 1, b: 2 };
let y: Y = x; // OK
```
### 2. 函数类型兼容性
#### 2.1 函数参数
* 当比较函数参数时，源函数的每个参数必须能在目标函数中找到对应的同名参数
```typescript
let x = (a: number) => 0;
let y = (a: number, b: number) => 0;
y = x; // OK
x = y; // Error
```
#### 2.2 函数返回值
* 强制源函数的返回值类型必须是目标函数返回值类型的子类型
```typescript
let x = () => ({ name: "Alice" });
let y = () => ({ name: "Alice", location: "China" });
x = y; // OK
```
### 3. 枚举类型兼容性
- 1. 与**数字类型**兼容
- 2. 不同枚举类型之间不兼容
```typescript
enum Status { Ready, Waiting };
enum Color { Red, Blue, Green };

let status = Status.Ready;
let number = 0;
status = number; // OK
status = Color.Green;  //error
```
### 4. 类的兼容性
#### 4.1 比较两个`类类型`的对象
* 只有实例的成员会被比较，静态成员和构造函数不在比较的范围内
```typescript
class Animal {
    feet: number;
    constructor(name: string, numFeet: number) { }
}

class Size {
    feet: number;
    constructor(numFeet: number) { }
}

let a: Animal;
let s: Size;

a = s;  //OK
s = a;  //OK
```
**代码解释**: 类 Animal 和类 Size 有相同的实例成员 feat 属性，且类型相同，构造函数参数虽然不同，但构造函数不参与两个类类型比较，所以最后两行可以相互赋值
#### 4.2 类的私有成员
* 私有成员会影响兼容性判断. 
- 1. 如果目标类型包含一个私有成员，那么源类型必须包含来自同一个类的这个私有成员
- 2. 允许子类赋值给父类,但是不能赋值给其它有同样类型的类
```typescript
class Animal {
    private name: string;
    constructor(name) { this.name = name; }
}

class Rhino extends Animal {
    constructor() { super("Rhino"); }
}

class Employee {
    private name: string;
    constructor(name) { this.name = name; }
}

let animal = new Animal("Goat");
let rhino = new Rhino();
let employee = new Employee("Bob");

animal = rhino;
animal = employee; // Error: Animal 与 Employee 不兼容
rhino = employee; // Error: Rhino 与 Employee 不兼容
```
