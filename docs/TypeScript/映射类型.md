### 映射类型
* 映射类型可以基于旧类型的属性，创建一个新类型
*  映射修饰符: (如 readonly 和 ?)
  * `+` 表示添加属性: `+readonly [P in keyof T]: T[P]; // 添加只读属性`
  * `-` 表示移除属性: `-readonly [P in keyof T]: T[P]; // 移除只读属性`
```typescript
  // 基本语法：[P in K]: T
// P: 属性名变量
// K: 要遍历的键的联合类型（通常使用 keyof T）
// T: 新属性的类型

type MappedType<T> = {
  [P in keyof T]: T[P];
};
  ```
  ### 二。 基础映射类型
  #### 1. 只读映射类型
  ```typescript
  // 内置 Readonly 的实现
type MyReadonly<T> = {
  readonly [P in keyof T]: T[P];
};

interface User {
  name: string;
  age: number;
}

type ReadonlyUser = MyReadonly<User>;
// 等价于 { readonly name: string; readonly age: number; }
  ```
#### 2. 可选映射
```typescript
// 内置 Partial 的实现
type MyPartial<T> = {
  [P in keyof T]?: T[P];
};

type PartialUser = MyPartial<User>;
// 等价于 { name?: string; age?: number; }
```
#### 3. 必需映射
```typescript
// 内置 Required 的实现
type MyRequired<T> = {
  [P in keyof T]-?: T[P];  // 使用 -? 移除可选性
};

interface OptionalUser {
  name?: string;
  age?: number;
}

type RequiredUser = MyRequired<OptionalUser>;
// 等价于 { name: string; age: number; }
```
#### 4. 移除只读
```typescript
type Mutable<T> = {
  -readonly [P in keyof T]: T[P];  // 使用 -readonly 移除只读
};

type ReadonlyType = { readonly x: number; readonly y: number };
type MutableType = Mutable<ReadonlyType>;
// 等价于 { x: number; y: number; }
```

### 三。 高级映射技巧
#### 1. 使用 as 子句重映射键名
* as 子句允许你在映射类型时重命名**键名**, 例如将属性名从大写转为小写
```typescript
// TypeScript 4.1+ 支持
type Getters<T> = {
  [P in keyof T as `get${Capitalize<string & P>}`]: () => T[P];
};

interface User {
  name: string;
  age: number;
}

type UserGetters = Getters<User>;
// 等价于 { getName: () => string; getAge: () => number; }
```
#### 2. 过滤属性
```typescript
// 只保留特定类型的属性
type FilterMethods<T> = {
  [P in keyof T as T[P] extends Function ? P : never]: T[P];
};

interface Mixed {
  name: string;
  age: number;
  greet: () => void;
  update: () => void;
}

type MethodsOnly = FilterMethods<Mixed>;
// 等价于 { greet: () => void; update: () => void; }
```
#### 3. 基于值的类型映射
```typescript
// 根据属性值类型进行转换
type Stringify<T> = {
  [P in keyof T]: string;
};

type Numberify<T> = {
  [P in keyof T]: number;
};
interface User {
  name: string;
  age: number;
}
type UserStringified = Stringify<User>;
// 等价于 { name: string; age: string; }
```
### 四. 条件映射类型
#### 1. 条件类型与映射结合
```typescript
// 根据条件选择性地映射属性
type ConditionalMap<T> = {
  [P in keyof T]: T[P] extends string ? string : T[P];
};

interface Data {
  id: number;
  name: string;
  timestamp: Date;
  tags: string[];
}

type Modified = ConditionalMap<Data>;
// id: number, name: string, timestamp: Date, tags: string[]
// 实际上没有变化，因为只有 string 类型会被映射为 string
```
#### 2. 深度映射
```typescript
// 递归映射嵌套对象
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface NestedData {
  user: {
    name: string;
    address: {
      city: string;
      street: string;
    };
  };
  settings: {
    theme: string;
    notifications: boolean;
  };
}

type PartialNestedData = DeepPartial<NestedData>;
// 所有层级都变为可选
```
#### 3. 选择性深度映射
```typescript
// 只对特定类型进行深度映射
type DeepReadonlyObject<T> = {
  readonly [P in keyof T]: T[P] extends object 
    ? T[P] extends Function
      ? T[P]  // 函数保持原样
      : DeepReadonlyObject<T[P]>
    : T[P];
};
```
### 五。注意事项
* 映射类型只能用于创建对象类型，不能用于创建基本类型（如 `string`、`number`）
* 映射类型在编译时被完全展开，因此对于大型类型，可能会增加编译时间
