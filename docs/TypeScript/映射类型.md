# TypeScript 映射类型（Mapped Types）

> 📖 [Mapped Types —— TypeScript 官方文档](https://www.typescriptlang.org/docs/handbook/2/mapped-types.html)
> 📖 [Key Remapping via `as` —— TypeScript 官方文档](https://www.typescriptlang.org/docs/handbook/release-notes/typescript-4-1.html#key-remapping-in-mapped-types)

## 一、概述

**映射类型（Mapped Types）** 是一种「**基于一个已有类型，按规则批量变换它的每个属性，生成一个新类型**」的机制。你可以把它理解为「**对类型的属性做一次遍历（map）**」。

```ts
// 把 T 的所有属性都变成可选
type MyPartial<T> = {
  [P in keyof T]?: T[P];
};
```

> 💡 **通俗理解：** 映射类型就像「**给对象的每个属性套上一层规则**」——一键全变只读、一键全变可选、一键重命名，而不必逐个手写。TypeScript 内置的 `Partial`、`Readonly`、`Required`、`Pick` 等工具类型，底层都是映射类型。

> 📎 相关：映射类型常和 `keyof`（见 [[keyof]]）、条件类型（见 [[条件类型]]）、索引查询 `T[P]`（见 [[索引查询]]）配合使用。

## 二、基本语法

```ts
type MappedType<T> = {
  [P in keyof T]: T[P];
};
```

- `P`：属性名变量（每次循环取到的属性名）；
- `in`：遍历关键字；
- `keyof T`：要遍历的键的联合类型（`T` 所有属性名的联合）；
- `T[P]`：原类型在该属性上的值类型（索引查询）。

### 映射修饰符：`readonly` 和 `?`，以及 `+` / `-`

- `+` 表示**添加**修饰符（可省略，默认就是添加）；
- `-` 表示**移除**修饰符。

```ts
type AddReadonly<T> = {
  +readonly [P in keyof T]: T[P];  // 添加只读（+ 可省略）
};
type RemoveReadonly<T> = {
  -readonly [P in keyof T]: T[P];  // 移除只读
};
type AddOptional<T> = {
  [P in keyof T]+?: T[P];          // 添加可选
};
type RemoveOptional<T> = {
  [P in keyof T]-?: T[P];          // 移除可选
};
```

## 三、基础映射类型

内置的 `Readonly`、`Partial`、`Required` 都是最基础的映射类型，我们可以自己实现它们：

### 1. 只读映射 —— `Readonly`

```ts
type MyReadonly<T> = {
  readonly [P in keyof T]: T[P];
};

interface User {
  name: string;
  age: number;
}
type ReadonlyUser = MyReadonly<User>;
// 等价于 { readonly name: string; readonly age: number }
```

### 2. 可选映射 —— `Partial`

```ts
type MyPartial<T> = {
  [P in keyof T]?: T[P];
};
type PartialUser = MyPartial<User>;
// 等价于 { name?: string; age?: number }
```

### 3. 必需映射 —— `Required`

```ts
type MyRequired<T> = {
  [P in keyof T]-?: T[P];  // 用 -? 移除可选性
};
interface OptionalUser {
  name?: string;
  age?: number;
}
type RequiredUser = MyRequired<OptionalUser>;
// 等价于 { name: string; age: number }
```

### 4. 移除只读 —— `Mutable`

```ts
type Mutable<T> = {
  -readonly [P in keyof T]: T[P];  // 用 -readonly 移除只读
};
type ReadonlyType = { readonly x: number; readonly y: number };
type MutableType = Mutable<ReadonlyType>;
// 等价于 { x: number; y: number }
```

## 四、同态映射（Homomorphic）：自动保留修饰符 ⭐

这是映射类型一个**关键且容易被忽视**的特性。

**规则**：当你用 `{ [P in keyof T]: ... }` 这种「遍历 `keyof T`」的形式（称为**同态映射**），映射结果会**自动复制 `T` 原有的 `readonly` 和 `?` 修饰符**，即使你在映射里没写。

```ts
type Example = {
  readonly a: string;
  b?: number;
};

// 同态映射：遍历 keyof T
type Copy<T> = { [P in keyof T]: T[P] };

type R = Copy<Example>;
// 结果保留了修饰符：{ readonly a: string; b?: number }
//   即使 Copy 里没写 readonly / ?
```

> 💡 **提示：** 这正是 `Partial<T>` 为什么能正确工作的原因——它在已有属性基础上加 `?`。而「非同态映射」（遍历的不是 `keyof T`，而是单独的键联合）则**不会**保留修饰符。

对比非同态映射：

```ts
// 遍历一个独立的键联合（不是 keyof T）—— 非同态，不保留修饰符
type Keys = 'a' | 'b';
type NonHomo<T> = { [P in Keys]: T[P] };
```

## 五、高级映射技巧

### 1. 用 `as` 子句重映射键名（TS 4.1+）

`as` 子句允许在映射时**重命名键名**，常配合模板字面量类型使用：

```ts
type Getters<T> = {
  [P in keyof T as `get${Capitalize<string & P>}`]: () => T[P];
};

interface User {
  name: string;
  age: number;
}
type UserGetters = Getters<User>;
// 等价于 { getName: () => string; getAge: () => number }
```

### 2. 过滤属性（配合条件类型 + `as never`）

通过把不要的键重映射为 `never`，实现「只保留/排除某些属性」：

```ts
// 只保留函数类型的属性
type FilterMethods<T> = {
  [P in keyof T as T[P] extends Function ? P : never]: T[P];
};

interface Mixed {
  name: string;
  age: number;
  greet: () => void;
  update: () => void;
}
type MethodsOnly = FilterMethods<Mixed>;
// 等价于 { greet: () => void; update: () => void }
```

> 💡 **提示：** `as never` 会让该键在结果中**消失**（`never` 作为键会被忽略），这是映射类型里做「属性过滤」的标准手法。

### 3. 基于值的类型映射

```ts
type Stringify<T> = {
  [P in keyof T]: string;
};
type UserStringified = Stringify<User>;
// 等价于 { name: string; age: string }（所有值都变成 string）
```

## 六、条件映射类型

### 1. 条件类型与映射结合：按值类型变换

```ts
// 把 string 类型的属性原样保留，其它保持原样
type ConditionalMap<T> = {
  [P in keyof T]: T[P] extends string ? string : T[P];
};
```

### 2. 深度映射 —— `DeepPartial`

```ts
// 递归地把所有层级的属性都变成可选
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface NestedData {
  user: {
    name: string;
    address: { city: string; street: string };
  };
}
type PartialNestedData = DeepPartial<NestedData>;
// 所有层级都变为可选
```

### 3. 选择性深度映射 —— `DeepReadonlyObject`

```ts
// 深度只读，但跳过函数
type DeepReadonlyObject<T> = {
  readonly [P in keyof T]: T[P] extends object
    ? T[P] extends Function
      ? T[P]            // 函数保持原样
      : DeepReadonlyObject<T[P]>
    : T[P];
};
```

## 七、映射类型 vs 索引签名

两者都能描述「键值对」，但定位不同：

| 对比项 | 映射类型 `{ [P in K]: V }` | 索引签名 `{ [key: string]: V }` |
| --- | --- | --- |
| 本质 | **遍历**一组已知的键，生成对象类型 | 声明**任意键**都映射到某值类型 |
| 键的范围 | `K` 是明确的联合（如 `keyof T`） | 任意 `string` / `number` / `symbol` |
| 会保留原属性吗 | 会（同态映射） | 不涉及 |
| 典型用途 | 基于旧类型派生新类型（Partial/Readonly…） | 描述动态键的对象（字典、配置） |

详见 [[索引签名]]。

## 八、注意事项

- 映射类型只能用于创建**对象类型**，不能直接映射原始类型（如 `string`、`number`）。对原始类型做映射会原样返回或需特殊处理；
- 映射类型在编译时被**完全展开**，对于大型/递归类型可能**增加编译时间**，深度递归有层数限制；
- 同态映射（`keyof T`）**保留修饰符**，非同态映射不保留——这是调试时容易困惑的点；
- 用 `as never` 做属性过滤时，`never` 键会被自动忽略，从而「删除」该属性。

## 九、总结

- 映射类型 = 「**遍历一个类型的属性，按规则生成新类型**」，是 `Partial`/`Readonly`/`Required` 等工具类型的底层；
- 修饰符：`readonly`、`?`，用 `+`（可省略）添加、`-` 移除；
- **同态映射**（`keyof T`）自动保留原类型的 `readonly`/`?`，这是关键特性；
- `as` 子句重命名键、`as never` 过滤属性，是高级技巧；
- 配合条件类型可做深度映射（`DeepPartial`、`DeepReadonly` 等）。

> 💡 **一句话记忆：** 映射类型是「**类型层面的 forEach**」——`[P in keyof T]` 遍历属性，修饰符 `+/-` 增删只读/可选，`as` 改键名、`as never` 删属性；记住「同态映射保留修饰符」就不会被它的隐式行为搞晕。
