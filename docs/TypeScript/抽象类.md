### 1. 抽象类 (Abstract Class)
* 抽象类是**不能**被实例化的，只能作为基类供其他子类继承。用于定义公共接口和部分实现
* 抽象类中可以包含抽象方法（没有实现的方法）和具体方法
* 子类继承抽象类时，必须实现所有父类的抽象方法
```typeScript
abstract class Shape {
  // 抽象属性（必须在派生类中实现）
  abstract readonly name: string;
  
  // 抽象方法（必须在派生类中实现）
  abstract calculateArea(): number;
  
  // 具体方法（可以有实现）
  display(): void {
    console.log(`This is a ${this.name} with area ${this.calculateArea()}`);
  }
  
  // 构造函数可以有
  constructor(protected color: string) {}
  
  // 可以包含非抽象属性
  protected createdAt: Date = new Date();
}

class Circle extends Shape {
  readonly name = "Circle";
  
  constructor(color: string, private radius: number) {
    super(color);
  }
  
  calculateArea(): number {
    return Math.PI * this.radius * this.radius;
  }
  
  // 可以新增方法
  getCircumference(): number {
    return 2 * Math.PI * this.radius;
  }
}

class Rectangle extends Shape {
  readonly name = "Rectangle";
  
  constructor(
    color: string, 
    private width: number, 
    private height: number
  ) {
    super(color);
  }
  
  calculateArea(): number {
    return this.width * this.height;
  }
}

// 使用
const circle = new Circle("red", 5);
circle.display();  // "This is a Circle with area 78.53981633974483"
console.log(circle.calculateArea());  // 78.53981633974483

// const shape = new Shape("blue");  // ❌ 错误：无法创建抽象类实例
```
### 2. 抽象类的特点
1. 不能实例化：只能被继承，不能直接创建对象实例
2. 可以包含抽象成员：必须在派生类中实现
3. 可以包含具体实现：可以有具体方法和属性
4. 可以有构造函数：但不能直接调用
5. 支持访问修饰符：`public`、`private`、`protected`
