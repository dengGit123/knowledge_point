## 一、基础概念

### 1. 什么是子类型关系？

在类型系统中，如果类型 `A` 可以赋值给类型 `B`，则称 `A` 是 `B` 的**子类型**（Subtype），记作 `A <: B`。

```typescript
class Animal {
  name: string;
}

class Dog extends Animal {
  bark(): void {
    console.log('汪汪！');
  }
}

class Cat extends Animal {
  meow(): void {
    console.log('喵喵！');
  }
}

// Dog 是 Animal 的子类型
// Dog <: Animal <: any
// any:> Animal:> Dog
```

**类型层次结构：**

```
         any
         ↑
         object
         ↑
       Animal
      /      \
    Dog      Cat
```

### 2. 什么是变体（Variance）？

**变体**描述了复合类型（如 `Array<T>`、`Function<T>`）与其类型参数 `T` 之间的子类型关系。

给定类型构造器 `F<T>`，如果 `A <: B`，那么：

- **协变**：`F<A> <: F<B>` （保持子类型关系）
- **逆变**：`F<B> <: F<A>` （反转子类型关系）
- **不变**：`F<A>` 和 `F<B>` 没有子类型关系
- **双向协变**：`F<A> <: F<B>` 且 `F<B> <: F<A>` （互为子类型）

### 3. 为什么需要理解变体？

理解变体对于：
- ✅ 编写类型安全的泛型代码
- ✅ 避免 TypeScript 类型错误
- ✅ 理解函数类型的赋值规则
- ✅ 设计灵活且安全的 API

## 二、协变（Covariance）

### 1. 定义

**协变**：保持子类型关系的方向不变。

如果 `Dog <: Animal`，那么 `Array<Dog> <: Array<Animal>`

```
Dog <: Animal
↓ 协变
Array<Dog> <: Array<Animal>
```

### 2. 直观理解

**"生产者"（Producer）类型通常是协变的**

当你从对象中**读取**数据时（生产数据），你需要的是类型保证：

```typescript
class Animal {
  name: string;
}

class Dog extends Animal {
  bark(): void {
    console.log('汪汪！');
  }
}

// 协变示例：数组是协变的
const dogs: Dog[] = [new Dog(), new Dog()];
const animals: Animal[] = dogs; // ✅ OK，Dog[] 是 Animal[] 的子类型

// 为什么这是安全的？
// 因为我们从 animals 中读取时，得到的是 Animal 类型
// 而 Dog 肯定是 Animal，所以类型安全
function processAnimals(animals: Animal[]): void {
  animals.forEach(animal => {
    console.log(animal.name); // ✅ 安全，Dog 也有 name 属性
  });
}

processAnimals(dogs); // ✅ OK
```

### 3. 协变的类型

在 TypeScript 中，以下类型是协变的：

#### ① 普通对象类型（只读属性）

```typescript
// 只读属性是协变的
type ReadonlyAnimalBox = {
  readonly get: () => Animal;
};

type ReadonlyDogBox = {
  readonly get: () => Dog;
};

// ReadonlyDogBox <: ReadonlyAnimalBox
const dogBox: ReadonlyDogBox = {
  get: () => new Dog()
};

const animalBox: ReadonlyAnimalBox = dogBox; // ✅ OK
```

#### ② 数组（在 TypeScript 2.6+ 之前）

```typescript
// ⚠️ 注意：TypeScript 2.6+ 中数组被设计为协变的
// 但这在某些情况下可能导致类型不安全（见后面"常见问题"）

const dogs: Dog[] = [new Dog()];
const animals: Animal[] = dogs; // ✅ OK（协变）

animals.push(new Cat()); // ⚠️ 运行时错误！但 TypeScript 允许
// 现在 dogs 数组中有一个 Cat，类型不安全！
```

### 4. 为什么生产者是协变的？

**Liskov 替换原则（LSP）**：

如果 `Dog` 是 `Animal` 的子类型，那么 `Dog` 的实例应该可以替换任何使用 `Animal` 的地方。

```typescript
interface Producer<T> {
  produce(): T;
}

// Producer<Dog> <: Producer<Animal>
const dogProducer: Producer<Dog> = {
  produce: () => new Dog()
};

const animalProducer: Producer<Animal> = dogProducer; // ✅ OK

// 使用 animalProducer 时，期望得到 Animal
// 实际得到的是 Dog，而 Dog 是 Animal，所以类型安全
const animal = animalProducer.produce(); // 类型是 Animal，实际是 Dog
```

## 三、逆变（Contravariance）

### 1. 定义

**逆变**：反转子类型关系的方向。

如果 `Dog <: Animal`，那么 `Consumer<Animal> <: Consumer<Dog>`

```
Dog <: Animal
↓ 逆变
Consumer<Animal> <: Consumer<Dog>
```

### 2. 直观理解

**"消费者"（Consumer）类型是逆变的**

当你向对象中**写入**数据时（消费数据），子类型关系会反转：

```typescript
// 消费者类型：接受参数并处理
type Consumer<T> = (value: T) => void;

// 具体的消费者
function processAnimal(animal: Animal): void {
  console.log(animal.name);
}

function processDog(dog: Dog): void {
  console.log(dog.name);
  dog.bark(); // 只有 Dog 才能调用 bark
}

// ⚠️ 注意这里的反转关系！
const animalConsumer: Consumer<Animal> = processAnimal;
const dogConsumer: Consumer<Dog> = processDog;

// Consumer<Animal> <: Consumer<Dog>
// AnimalConsumer 可以赋值给 dogConsumer
dogConsumer = animalConsumer; // ✅ OK
```

### 3. 为什么消费者是逆变的？

理解逆变的关键：**更严格的函数不能替代更宽松的函数**

```typescript
// 函数类型赋值规则
// 参数类型：逆变的
// 返回类型：协变的

// 类型 A
type FnAnimal = (animal: Animal) => void;

// 类型 B
type FnDog = (dog: Dog) => void;

// FnAnimal <: FnDog
// FnAnimal 可以赋值给 FnDog 类型

let fnAnimal: FnAnimal = (animal: Animal) => {
  console.log(animal.name);
};

let fnDog: FnDog = (dog: Dog) => {
  console.log(dog.name);
  dog.bark(); // Dog 特有的方法
};

// ✅ OK：fnAnimal 可以赋值给 fnDog
// 因为 fnAnimal 可以接受任何 Animal，包括 Dog
fnDog = fnAnimal;

// ❌ Error：fnDog 不能赋值给 fnAnimal
// 因为 fnDog 只能接受 Dog，不能接受 Cat
fnAnimal = fnDog; // Error!
```

### 4. 逆变的直观图解

```
函数参数要求越宽松 → 可以接受的类型越多 → 越通用

       Animal (更宽松)
          ↑
          | 可以接受 Animal 和所有子类型（Dog, Cat, ...）
          |
       Dog (更严格)
          ↑
          | 只能接受 Dog
          |

所以：(animal: Animal) => void 可以赋值给 (dog: Dog) => void
因为前者可以接受后者需要的所有输入（甚至更多）
```

### 5. 函数类型的完整变体规则

```typescript
// 函数类型的变体规则
// 参数位置：逆变
// 返回位置：协变

type Fn1<A, R> = (arg: A) => R;
type Fn2<B, S> = (arg: B) => S;

// 如果 Dog <: Animal，那么：
// Fn1<Animal, Dog> <: Fn2<Dog, Animal>
//    ↑ 逆变           ↑ 协变
```

## 四、双向协变（Bivariance）

### 1. 定义

**双向协变**：两个类型互为子类型。

如果 `Dog <: Animal`，那么 `F<Dog> <: F<Animal>` 且 `F<Animal> <: F<Dog>`

### 2. TypeScript 中的双向协变

在某些情况下，TypeScript 会使用双向协变来提高灵活性，但这可能牺牲类型安全性。

```typescript
// TypeScript 默认对函数参数使用双向协变（在某些场景）
type Fn = (x: Animal) => void;

let fn1: Fn = (animal: Animal) => console.log(animal.name);
let fn2: Fn = (dog: Dog) => console.log(dog.name); // ✅ OK，双向协变
```

### 3. 何时触发双向协变？

#### ① 方法参数

```typescript
interface Animal {
  name: string;
}

interface Dog extends Animal {
  bark(): void;
}

class Keeper {
  // 方法参数是双向协变的
  feed(animal: Animal): void {
    console.log(`Feeding ${animal.name}`);
  }
}

class DogKeeper {
  feed(dog: Dog): void {
    console.log(`Feeding ${dog.name}`);
    dog.bark();
  }
}

const keeper: Keeper = new Keeper();
const dogKeeper: DogKeeper = new DogKeeper();

// ⚠️ TypeScript 允许这样（双向协变）
keeper = dogKeeper; // ✅ OK，但不安全！
keeper.feed(new Cat()); // 运行时可能错误！
```

#### ② 简单函数类型（非严格模式）

```typescript
// tsconfig.json 中 strictFunctionTypes: false 时
type Callback = (x: Animal) => void;

const cb1: Callback = (animal: Animal) => {};
const cb2: Callback = (dog: Dog) => {}; // ✅ OK（双向协变）
```

### 4. 禁用双向协变

在 `tsconfig.json` 中启用严格模式：

```json
{
  "compilerOptions": {
    "strictFunctionTypes": true  // 启用严格函数类型检查
  }
}
```

启用后，函数参数将遵循严格的逆变规则。

## 五、实际应用场景

### 1. 事件处理器（逆变应用）

```typescript
// 事件系统设计
type EventHandler<T> = (event: T) => void;

class EventEmitter<T> {
  private listeners: EventHandler<T>[] = [];

  on(handler: EventHandler<T>): void {
    this.listeners.push(handler);
  }

  emit(event: T): void {
    this.listeners.forEach(handler => handler(event));
  }
}

// 使用
type MouseEvent = { type: 'mouse'; x: number; y: number };
type ClickEvent = { type: 'click'; x: number; y: number; button: number };

// ClickEvent <: MouseEvent
const mouseEvents = new EventEmitter<MouseEvent>();

// 更具体的事件处理器可以赋值给更通用的事件类型
const clickHandler: EventHandler<ClickEvent> = (event) => {
  console.log(`Clicked at ${event.x}, ${event.y} with button ${event.button}`);
};

// ✅ OK：逆变允许这样做
// clickHandler 可以处理 MouseEvent（虽然它期望 ClickEvent）
mouseEvents.on(clickHandler);

// ⚠️ 但发射时需要发送匹配的事件
mouseEvents.emit({ type: 'mouse', x: 10, y: 20 }); // OK
mouseEvents.emit({ type: 'click', x: 10, y: 20, button: 1 }); // OK
```

### 2. Promise 类型（协变应用）

```typescript
// Promise<T> 在返回位置是协变的
// Promise<Dog> <: Promise<Animal>

async function getDog(): Promise<Dog> {
  return new Dog();
}

async function processAnimal(): Promise<Animal> {
  // ✅ OK：Promise<Dog> 可以赋值给 Promise<Animal>
  return getDog();
}
```

### 3. 数组类型（协变 + 逆变）

```typescript
// TypeScript 中数组的特殊处理
// 读取时是协变的
// 写入时是逆变的

function readAnimals(animals: Animal[]): Animal {
  return animals[0]; // 读取：协变，返回 Animal
}

function writeAnimals(animals: Animal[], animal: Animal): void {
  animals.push(animal); // 写入：逆变，需要 Animal
}

// ⚠️ 这就是为什么 TypeScript 2.6+ 中数组协变可能导致问题
const dogs: Dog[] = [new Dog()];
const animals: Animal[] = dogs; // ✅ OK（协变）

animals.push(new Cat()); // ⚠️ 运行时可能不安全
```

### 4. 泛型接口设计

```typescript
// 只读接口（协变）
interface ReadonlyBox<out T> {
  // out 关键字表示 T 只用于输出位置
  get(): T;
}

// 只写接口（逆变）
interface WriteOnlyBox<in T> {
  // in 关键字表示 T 只用于输入位置
  set(value: T): void;
}

// 读写接口（不变）
interface Box<T> {
  get(): T;
  set(value: T): void;
}
```

**类型注解 `in` 和 `out`：**

```typescript
// TypeScript 4.7+ 支持 in/out 注解
interface Producer<out T> {
  // T 只出现在输出位置，所以是协变的
  produce(): T;
}

interface Consumer<in T> {
  // T 只出现在输入位置，所以是逆变的
  consume(value: T): void;
}

interface Transformer<in T, out R> {
  // T 只出现在输入位置（逆变）
  // R 只出现在输出位置（协变）
  transform(value: T): R;
}
```

### 5. 函数组合

```typescript
// 函数组合的类型推断
function compose<A, B, C>(
  f: (b: B) => C,  // B -> C
  g: (a: A) => B   // A -> B
): (a: A) => C {
  return (a: A) => f(g(a));
}

const toString = (x: number) => x.toString();
const toUpperCase = (s: string) => s.toUpperCase();

const composed = compose(toUpperCase, toString);
// 类型: (x: number) => string

console.log(composed(123)); // "123"
```

## 六、常见问题

### 1. 数组协变的类型安全问题

**问题：**

```typescript
const dogs: Dog[] = [new Dog()];
const animals: Animal[] = dogs; // ✅ TypeScript 允许

animals.push(new Cat()); // ✅ TypeScript 允许
// ⚠️ 但现在 dogs 数组中有一个 Cat！
console.log(dogs[0] instanceof Dog); // 可能是 false！
```

**解决方案：**

```typescript
// 方案 1：使用只读数组
const dogs: ReadonlyArray<Dog> = [new Dog()];
const animals: ReadonlyArray<Animal> = dogs; // ✅ 完全安全

// 方案 2：明确指定类型
function processDogs(dogs: Dog[]): void {
  // 不修改原数组
  const processed = dogs.map(dog => {
    console.log(dog.name);
    return dog;
  });
}
```

### 2. 函数参数赋值错误

**问题：**

```typescript
type AnimalHandler = (animal: Animal) => void;
type DogHandler = (dog: Dog) => void;

let handleAnimal: AnimalHandler = (animal) => {};
let handleDog: DogHandler = (dog) => {};

// ❌ 为什么这样不行？
handleAnimal = handleDog; // Error!
```

**理解：**

```typescript
// handleDog 只能处理 Dog
// handleAnimal 需要能处理所有 Animal（包括 Cat）
// 所以 handleDog 不能替代 handleAnimal

// ✅ 正确的方向
handleDog = handleAnimal; // OK
```

### 3. 泛型约束与变体

```typescript
// 泛型约束不直接影响变体
interface Container<T extends Animal> {
  value: T;
}

// Container<Dog> <: Container<Animal>
// 因为 value 属性是协变的
```

### 4. strictFunctionTypes 的影响

```typescript
// tsconfig.json: { "strictFunctionTypes": true }

type Callback<T> = (value: T) => void;

let animalCb: Callback<Animal> = (animal) => {};
let dogCb: Callback<Dog> = (dog) => {};

// 启用 strictFunctionTypes 后：
dogCb = animalCb; // ✅ OK（逆变）
animalCb = dogCb; // ❌ Error（不再是双向协变）
```

### 5. 对象方法的特殊情况

```typescript
interface Animal {
  name: string;
}

interface Dog extends Animal {
  bark(): void;
}

class AnimalKeeper {
  feed(animal: Animal): void {}
}

class DogKeeper {
  feed(dog: Dog): void {}
}

// ⚠️ 方法参数有特殊的双向协变规则
let keeper: AnimalKeeper = new DogKeeper(); // ✅ OK
```

**为什么方法有特殊规则？**

为了兼容常见的面向对象模式，但可能不安全。

## 七、最佳实践

### 1. 启用严格模式

```json
// tsconfig.json
{
  "compilerOptions": {
    "strict": true,                    // 启用所有严格检查
    "strictFunctionTypes": true        // 启用严格函数类型
  }
}
```

### 2. 使用 in/out 注解（TypeScript 4.7+）

```typescript
interface Producer<out T> {
  produce(): T;
}

interface Consumer<in T> {
  consume(value: T): void;
}
```

### 3. 避免可变数组的协变赋值

```typescript
// ❌ 避免
const dogs: Dog[] = getDogs();
const animals: Animal[] = dogs;

// ✅ 推荐
const dogs: ReadonlyArray<Dog> = getDogs();
const animals: ReadonlyArray<Animal> = dogs;
```

### 4. 函数类型设计

```typescript
// ✅ 好：明确的泛型约束
function map<T, U>(array: T[], fn: (item: T) => U): U[] {
  return array.map(fn);
}

// ❌ 避免：过度使用 any
function mapAny(array: any[], fn: (item: any) => any): any[] {
  return array.map(fn);
}
```

### 5. 理解并利用逆变

```typescript
// 事件系统设计
type EventHandler<T> = (event: T) => void;

class EventEmitter<T> {
  // 逆变允许更具体的处理器注册到更通用的事件
  on(handler: EventHandler<T>): void {
    // ...
  }
}
```

## 八、总结

### 变体规则速查表

| 类型构造器 | 变体类型 | 示例 |
|-----------|---------|------|
| `T` | 协变 | `Dog` → `Animal` |
| `Array<T>` | 协变 | `Dog[]` → `Animal[]` |
| `Promise<T>` | 协变 | `Promise<Dog>` → `Promise<Animal>` |
| `(x: T) => void` | 逆变 | `(x: Animal) => void` → `(x: Dog) => void` |
| `() => T` | 协变 | `() => Dog` → `() => Animal` |
| `(x: T) => T` | 不变 | `(x: Dog) => Dog` 与 `(x: Animal) => Animal` |
| `ReadonlyArray<T>` | 协变 | `ReadonlyArray<Dog>` → `ReadonlyArray<Animal>` |

### 关键要点

1. **生产者（输出）是协变的**
   - 读取数据时，保持子类型关系
   - `Producer<Dog>` 可以赋值给 `Producer<Animal>`

2. **消费者（输入）是逆变的**
   - 写入数据时，反转子类型关系
   - `Consumer<Animal>` 可以赋值给 `Consumer<Dog>`

3. **函数类型遵循：参数逆变，返回协变**
   - `(dog: Dog) => Animal` 可以赋值给 `(animal: Animal) => Dog`

4. **启用严格模式以获得最佳类型安全**
   - `strictFunctionTypes: true`

5. **使用 in/out 注解明确意图**
   - `interface Producer<out T>`
   - `interface Consumer<in T>`

### 实用记忆法

```
生产者（Producer）→ 协变（Covariant）
├─ "往外拿" → 保持方向 → 协变
└─ Promise<T>, Array<T>, () => T

消费者（Consumer）→ 逆变（Contravariant）
├─ "往里放" → 反转方向 → 逆变
└─ (x: T) => void

双向协变（Bivariance）
├─ 为了灵活性，牺牲安全性
└─ 方法参数（TypeScript 特殊处理）
```
