# MySQL 函数与存储过程

> 官方文档：[MySQL 8.0 - Functions and Operators](https://dev.mysql.com/doc/refman/8.0/en/functions.html) / [Stored Objects](https://dev.mysql.com/doc/refman/8.0/en/stored-objects.html)

## 这篇讲什么？

本文分两部分：

1. **函数**：MySQL 自带的内置函数（拿来就用）+ 自己写的自定义函数
2. **存储过程**：把一组 SQL 打包成"程序"，可以反复调用

```
内置函数：MySQL 提供好的工具，如 NOW()、COUNT()、SUBSTRING()
自定义函数：你自己写的函数，返回一个值，可在 SELECT 里用
存储过程：你自己写的"批处理程序"，可包含多条 SQL、流程控制，单独调用
```

> 💡 **提示：** 内置函数要熟练（日常查询常用），自定义函数和存储过程了解即可——现代开发中，复杂业务逻辑更倾向于写在**应用层（Java）**而非数据库里。

# 第一部分：内置函数

## 一、字符串函数

```sql
-- 拼接字符串
SELECT CONCAT('姓名:', name, ', 年龄:', age) FROM users;
-- 结果：姓名:张三, 年龄:20

-- 带分隔符拼接（MySQL 推荐用这个，更安全）
SELECT CONCAT_WS('-', '2026', '07', '02');   -- 2026-07-02

-- 长度
SELECT LENGTH(name)      -- 字节数（utf8 下中文算 3 字节）
FROM users;
SELECT CHAR_LENGTH('你好');    -- 2，字符数（中文算 1）

-- 大小写转换
SELECT UPPER('abc');           -- ABC
SELECT LOWER('ABC');           -- abc

-- 截取：SUBSTRING(字符串, 起始位置, 长度) —— 注意从 1 开始
SELECT SUBSTRING('Hello', 1, 3);    -- Hel
SELECT SUBSTRING('Hello', 2);       -- ello（从第2位到末尾）

-- 去空格
SELECT TRIM('  abc  ');        -- abc（去两端）
SELECT LTRIM('  abc');         -- abc（去左）
SELECT RTRIM('abc  ');         -- abc（去右）

-- 替换
SELECT REPLACE('a-b-c', '-', '/');   -- a/b/c

-- 查找位置（找不到返回 0，不是 -1！）
SELECT INSTR('Hello', 'll');          -- 3

-- 填充
SELECT LPAD('5', 3, '0');     -- 005（左填充 0 到 3 位）
SELECT RPAD('5', 3, '0');     -- 500（右填充）
```

### 字符串函数速查表

| 函数 | 作用 | 示例结果 |
|-----|------|---------|
| `CONCAT(a,b)` | 拼接 | `'ab'` |
| `CONCAT_WS(sep,a,b)` | 用分隔符拼接 | `'a-b'` |
| `LENGTH(s)` | 字节数 | `'你好'`→6 |
| `CHAR_LENGTH(s)` | 字符数 | `'你好'`→2 |
| `UPPER(s)` / `LOWER(s)` | 大小写 | `'ABC'` |
| `SUBSTRING(s,开始,长度)` | 截取 | — |
| `TRIM(s)` | 去两端空格 | — |
| `REPLACE(s,旧,新)` | 替换 | — |
| `INSTR(s,子串)` | 查位置 | 找不到为 0 |
| `LPAD(s,len,pad)` / `RPAD` | 填充 | — |

> 💡 **提示：** `CONCAT()` 中任意参数为 NULL，结果就是 NULL。用 `CONCAT_WS()` 或先 `IFNULL()` 处理可避免。

## 二、日期时间函数

```sql
-- 获取当前时间
SELECT NOW();          -- 2026-07-02 12:30:00（日期+时间）
SELECT CURDATE();      -- 2026-07-02（只有日期）
SELECT CURTIME();      -- 12:30:00（只有时间）
SELECT SYSDATE();      -- 当前时间（执行时实时获取）

-- 提取部分
SELECT YEAR(NOW());    -- 2026
SELECT MONTH(NOW());   -- 7
SELECT DAY(NOW());     -- 2
SELECT HOUR(NOW());
SELECT MINUTE(NOW());
SELECT DAYOFWEEK(NOW());  -- 星期几（1=周日）

-- 日期加减
SELECT DATE_ADD(NOW(), INTERVAL 7 DAY);     -- 7 天后
SELECT DATE_SUB(NOW(), INTERVAL 1 MONTH);   -- 1 个月前
SELECT DATE_ADD('2026-07-02', INTERVAL 1 YEAR);  -- 1 年后

-- 日期差
SELECT DATEDIFF('2026-07-02', '2026-01-01');  -- 182（相差天数）
SELECT TIMEDIFF('12:30:00', '10:00:00');      -- 02:30:00

-- 格式化
SELECT DATE_FORMAT(NOW(), '%Y-%m-%d');          -- 2026-07-02
SELECT DATE_FORMAT(NOW(), '%Y年%m月%d日');        -- 2026年07月02日
SELECT DATE_FORMAT(NOW(), '%H:%i:%s');          -- 12:30:00

-- 字符串转日期
SELECT STR_TO_DATE('2026-07-02', '%Y-%m-%d');   -- 2026-07-02
```

### DATE_FORMAT 常用占位符

| 占位符 | 含义 | 示例 |
|-------|------|------|
| `%Y` | 4 位年 | 2026 |
| `%m` | 月（补零） | 07 |
| `%d` | 日（补零） | 02 |
| `%H` | 时（24 小时） | 14 |
| `%i` | 分 | 30 |
| `%s` | 秒 | 00 |
| `%y` | 2 位年 | 26 |

### 实际应用

```sql
-- 查询本月新增用户
SELECT * FROM users
WHERE created_at >= DATE_FORMAT(NOW(), '%Y-%m-01');

-- 按月统计用户注册量
SELECT DATE_FORMAT(created_at, '%Y-%m') AS 月份, COUNT(*) AS 注册量
FROM users
GROUP BY 月份
ORDER BY 月份;

-- 查询注册超过 30 天未登录的用户
SELECT * FROM users
WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY)
  AND last_login IS NULL;
```

## 三、数值函数

```sql
-- 四舍五入
SELECT ROUND(3.1415, 2);    -- 3.14（保留 2 位）
SELECT ROUND(3.5);          -- 4

-- 向上 / 向下取整
SELECT CEIL(3.1);           -- 4（向上）
SELECT FLOOR(3.9);          -- 3（向下）

-- 截断（不四舍五入）
SELECT TRUNCATE(3.1415, 2); -- 3.14

-- 取模 / 绝对值
SELECT MOD(10, 3);          -- 1（10 % 3）
SELECT ABS(-5);             -- 5

-- 随机数
SELECT RAND();              -- 0~1 之间的随机小数
SELECT FLOOR(RAND() * 100); -- 0~99 的随机整数

-- 最值
SELECT GREATEST(1, 2, 3);   -- 3
SELECT LEAST(1, 2, 3);      -- 1
```

```sql
SELECT
    city,
```

## 五、流程控制函数（SQL 里的 if-else）

### 5.1 IF 函数

```sql
-- IF(条件, 真时返回值, 假时返回值)
SELECT name, IF(age >= 18, '成年', '未成年') AS 状态
FROM users;
```

### 5.2 IFNULL 函数（处理 NULL）

```sql
-- IFNULL(值, 为 NULL 时的替换值)
SELECT name, IFNULL(email, '未填写') AS 邮箱
FROM users;
-- email 为 NULL 时显示"未填写"
```

### 5.3 NULLIF 函数

```sql
-- NULLIF(a, b)：a 等于 b 时返回 NULL，否则返回 a
SELECT NULLIF(5, 5);   -- NULL
SELECT NULLIF(5, 3);   -- 5
-- 常用于避免除以 0：a / NULLIF(b, 0)
```

### 5.4 CASE WHEN（最强大）

```sql
-- 方式一：等值判断（类似 switch）
SELECT name,
    CASE city
        WHEN '北京' THEN '北方'
        WHEN '上海' THEN '南方'
        ELSE '其他'
    END AS 地区
FROM users;

-- 方式二：条件判断（类似 if-elseif）
SELECT name,
    CASE
        WHEN age < 18  THEN '未成年'
        WHEN age < 60  THEN '成年'
        ELSE '老年'
    END AS 年龄段
FROM users;
```

### 5.5 CAST 类型转换

```sql
-- CAST(值 AS 类型)
SELECT CAST('123' AS SIGNED);          -- 123（转整数）
SELECT CAST(3.1415 AS DECIMAL(10,2));  -- 3.14
SELECT CAST(NOW() AS DATE);            -- 只取日期部分
```

# 第二部分：自定义函数与存储过程

## 六、自定义函数（FUNCTION）

自定义函数接收参数，**返回一个值**，可以在 `SELECT` 里像内置函数一样使用。

### 6.1 创建函数

```sql
-- 设置分隔符（因为函数体内有分号，会与 SQL 结束符冲突）
DELIMITER $$

CREATE FUNCTION get_user_age(uid INT)
RETURNS INT           -- 声明返回类型
DETERMINISTIC         -- 表示相同输入返回相同结果（可省略，但推荐）
READS SQL DATA        -- 表示会读数据
BEGIN
    DECLARE user_age INT;            -- 声明变量
    SELECT age INTO user_age         -- 把查询结果赋给变量
    FROM users WHERE id = uid;
    RETURN user_age;                 -- 返回值
END $$

DELIMITER ;           -- 恢复默认分隔符
```

### 6.2 使用函数

```sql
-- 像内置函数一样在 SELECT 里用
SELECT get_user_age(1);   -- 返回 id=1 的用户年龄

SELECT name, get_user_age(id) AS age FROM users;
```

### 6.3 删除函数

```sql
DROP FUNCTION IF EXISTS get_user_age;
```

## 七、存储过程（PROCEDURE）

存储过程是**把一组 SQL 语句打包**，存到数据库里，通过名字调用。它可以有**输入/输出参数**，可以包含流程控制，可以执行多条 SQL。

### 7.1 为什么用存储过程

```
✅ 优点：
  - 减少网络往返（一次调用执行多条 SQL）
  - 预编译，执行效率高
  - 复杂逻辑封装，多端复用

❌ 缺点：
  - 难调试、难版本管理
  - 业务逻辑分散在数据库和应用层，维护难
  - 数据库压力大，不易扩展
  - 现代开发趋势：逻辑放应用层（Java），数据库只管存取
```

> ⚠️ **注意：** 互联网公司**很少用存储过程**写核心业务。了解语法即可，实战优先用应用层代码。

### 7.2 创建存储过程（带输入参数）

```sql
DELIMITER $$

CREATE PROCEDURE transfer(IN from_id INT, IN to_id INT, IN amount DECIMAL(10,2))
BEGIN
    -- 扣款
    UPDATE accounts SET balance = balance - amount WHERE id = from_id;
    -- 加款
    UPDATE accounts SET balance = balance + amount WHERE id = to_id;
END $$

DELIMITER ;
```

### 7.3 调用存储过程

```sql
-- 用 CALL 调用
CALL transfer(1, 2, 100);   -- 用户1 转 100 给 用户2
```

### 7.4 带输出参数的存储过程

```sql
DELIMITER $$

CREATE PROCEDURE get_user_count(OUT total INT)
BEGIN
    SELECT COUNT(*) INTO total FROM users;
END $$

DELIMITER ;

-- 调用并接收输出
CALL get_user_count(@count);
SELECT @count;   -- 查看输出变量的值
```

### 7.5 删除存储过程

```sql
DROP PROCEDURE IF EXISTS transfer;
```

### 7.6 查看已有的函数 / 存储过程

```sql
SHOW FUNCTION STATUS WHERE Db = '你的库名';   -- 查看函数
SHOW PROCEDURE STATUS WHERE Db = '你的库名';  -- 查看存储过程

SHOW CREATE FUNCTION get_user_age;            -- 查看函数定义
SHOW CREATE PROCEDURE transfer;               -- 查看存储过程定义
```

## 八、变量与流程控制（函数/存储过程内部）

### 8.1 变量

```sql
-- 局部变量（在 BEGIN...END 内）
DECLARE 变量名 类型 [DEFAULT 默认值];
DECLARE v_count INT DEFAULT 0;
DECLARE v_name VARCHAR(50);

-- 赋值
SET v_count = 10;
-- 或用 SELECT ... INTO
SELECT COUNT(*) INTO v_count FROM users;

-- 会话变量（@ 开头，无需声明，会话内有效）
SET @x = 5;
SELECT @x;
```

### 8.2 IF 条件

```sql
IF age >= 18 THEN
    SET status = '成年';
ELSEIF age >= 60 THEN
    SET status = '老年';
ELSE
    SET status = '未成年';
END IF;
```

### 8.3 CASE

```sql
CASE
    WHEN age < 18 THEN SET group = '少年';
    WHEN age < 60 THEN SET group = '青年';
    ELSE SET group = '老年';
END CASE;
```

### 8.4 WHILE 循环

```sql
DECLARE i INT DEFAULT 1;
WHILE i <= 10 DO
    INSERT INTO log(msg) VALUES(CONCAT('第', i, '条'));
    SET i = i + 1;
END WHILE;
```

### 8.5 游标（CURSOR）

游标用来**逐行处理**查询结果。

```sql
DELIMITER $$

CREATE PROCEDURE process_users()
BEGIN
    DECLARE done INT DEFAULT FALSE;
    DECLARE uid INT;
    DECLARE uname VARCHAR(50);

    -- 声明游标（指向查询结果）
    DECLARE cur CURSOR FOR SELECT id, name FROM users;
    -- 声明结束处理（遍历完置 done 为 true）
    DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = TRUE;

    OPEN cur;   -- 打开游标

    read_loop: LOOP
        FETCH cur INTO uid, uname;   -- 取一行
        IF done THEN
            LEAVE read_loop;          -- 遍历完退出
        END IF;
        -- 处理每一行
        INSERT INTO log(uid, msg) VALUES(uid, CONCAT('处理:', uname));
    END LOOP;

    CLOSE cur;  -- 关闭游标
END $$

DELIMITER ;
```

> 💡 **提示：** 游标性能较差（逐行处理），大数据量处理优先考虑批量 SQL，而非游标逐行。

## 九、函数 vs 存储过程（对比）

| 对比项 | 自定义函数 | 存储过程 |
|-------|-----------|---------|
| 返回值 | **必须返回一个值** | 可返回 0 或多个（用 OUT 参数） |
| 调用方式 | 在 `SELECT` 里用 | 用 `CALL` 单独调用 |
| 能否用在 SELECT | ✅ 能 | ❌ 不能 |
| 能否执行增删改 | 一般只读 | ✅ 可以执行任意 SQL |
| 能否有事务 | ❌ | ✅ 可以 |
| 参数 | 只能 IN | IN / OUT / INOUT |
| 用途 | 计算、转换、返回值 | 批量操作、业务流程 |

```sql
-- 函数：用在 SELECT 里
SELECT name, get_age_group(age) FROM users;

-- 存储过程：单独 CALL
CALL transfer(1, 2, 100);
```

## 十、在 Java 中调用

### 10.1 调用存储过程

```java
// 用 CallableStatement 调用存储过程
String sql = "{call transfer(?, ?, ?)}";

try (CallableStatement cs = conn.prepareCall(sql)) {
    cs.setInt(1, 1);          // from_id
    cs.setInt(2, 2);          // to_id
    cs.setBigDecimal(3, new BigDecimal("100"));  // amount
    cs.execute();
}

// 带输出参数
String sql = "{call get_user_count(?)}";
try (CallableStatement cs = conn.prepareCall(sql)) {
    cs.registerOutParameter(1, Types.INTEGER);  // 注册输出参数
    cs.execute();
    int total = cs.getInt(1);   // 获取输出
    System.out.println("总人数: " + total);
}
```

```xml

</select>
```

## 十一、常见问题与注意事项

### 问题 1：CONCAT 遇到 NULL 怎么办？

```sql
SELECT CONCAT('a', NULL, 'b');   -- 结果是 NULL（整体变 NULL）

-- 解决：用 IFNULL 或 CONCAT_WS
SELECT CONCAT('a', IFNULL(NULL, ''), 'b');  -- 'ab'
SELECT CONCAT_WS(',', 'a', NULL, 'b');       -- 'a,b'
```

### 问题 2：日期格式化怎么记不住占位符？

```
记住大写年 %Y、小写时分秒 %H:%i:%s
  %Y 年、%m 月、%d 日、%H 时、%i 分、%s 秒
  （注意：分是 %i 不是 %m，月才是 %m）
```

### 问题 3：函数和存储过程该不该在项目里大量使用？

```
❌ 不建议大量使用。
  - 业务逻辑分散，难以维护和调试
  - 数据库是单点瓶颈，逻辑放进去会加重负担
  - 不利于迁移（换数据库时存储过程要重写）

✅ 适用场景：
  - 数据迁移、批处理等一次性脚本
  - 性能要求极高的特定计算
  - 与数据库强绑定的简单校验
```

### 问题 4：SUBSTRING 的起始位置从 0 还是 1？

```
MySQL 的 SUBSTRING 从 1 开始（不是 0！）
SUBSTRING('Hello', 1, 2) → 'He'
SUBSTRING('Hello', 2)    → 'ello'
支持负数：SUBSTRING('Hello', -3) → 'llo'（从倒数第3个开始）
```

### 问题 5：创建函数报错 "must be deterministic"

```sql
-- MySQL 默认不允许创建非 DETERMINISTIC 的函数（二进制日志安全考虑）
-- 解决：在函数定义里加 DETERMINISTIC，或开启配置
SET GLOBAL log_bin_trust_function_creators = 1;
```

### 自定义对象语法骨架

```sql
-- 函数
DELIMITER $$
CREATE FUNCTION 名(参数) RETURNS 类型
BEGIN
    ...逻辑...
    RETURN 值;
END $$
DELIMITER ;

-- 存储过程
DELIMITER $$
CREATE PROCEDURE 名(IN/OUT 参数)
BEGIN
    ...多条 SQL + 流程控制...
END $$
DELIMITER ;

-- 调用

CALL 存储过程名(...); -- 存储过程
```

### 选型建议

```
日常查询计算 → 用内置函数（NOW、DATE_FORMAT、CASE WHEN...）
单值计算/转换 → 自定义函数（少用）
批量操作/复杂流程 → 存储过程（少用，优先写 Java）
绝大多数业务逻辑 → 写在应用层（Java/MyBatis），数据库只管存取
```
