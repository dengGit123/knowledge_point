# MySQL 数据库（库的创建与管理）

> 官方文档：[MySQL 8.0 - CREATE DATABASE](https://dev.mysql.com/doc/refman/8.0/en/create-database.html) / [ALTER DATABASE](https://dev.mysql.com/doc/refman/8.0/en/alter-database.html) / [DROP DATABASE](https://dev.mysql.com/doc/refman/8.0/en/drop-database.html)

## 先分清：库（database）vs 表（table）

很多人把"数据库"和"表"混为一谈，先理清层级关系：

```
MySQL 服务器
└── 数据库（database / schema）   ← 本篇讲的就是"库"这一层
    └── 表（table）              ← 表的创建/删除在《表结构(DDL).md》
        └── 数据行（row）        ← 数据的增删改查在《增/删/改/查》
```

| 概念 | 类比 | 操作对象 |
|-----|------|---------|
| **服务器** | 一台电脑 | 整个 MySQL 实例 |
| **数据库（库）** | 一个文件夹 | 装表的"容器" |
| **表** | 文件夹里的 Excel 文件 | 存数据的结构 |
| **数据行** | Excel 里的每一行 | 具体数据 |

> 💡 **通俗理解：** "库"就是文件夹，"表"是文件夹里的文件。本篇讲怎么**新建、删除、切换、修改文件夹**。表（文件）内部怎么建，看 `表结构(DDL).md`。

## 一、创建数据库 CREATE DATABASE

### 1.1 最基本的建库

```sql
-- 创建一个名为 mydb 的数据库
CREATE DATABASE mydb;
```

> ⚠️ **注意：库名不能已存在**，否则报错 `Can't create database 'mydb'; database exists`。

### 1.2 加上 IF NOT EXISTS（推荐）

```sql
-- 如果库已存在，不报错，直接跳过
CREATE DATABASE IF NOT EXISTS mydb;
```

> 💡 **提示：** 写脚本时**永远加 `IF NOT EXISTS`**，避免重复执行脚本报错。

### 1.3 指定字符集和排序规则（推荐）

```sql
-- 建库时就设定好 utf8mb4 字符集（一劳永逸，新建表会继承）
CREATE DATABASE IF NOT EXISTS mydb
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_general_ci;
```

**CHARACTER SET（字符集）和 COLLATE（排序规则）的含义：**

| 配置 | 作用 |
|-----|------|
| `CHARACTER SET utf8mb4` | 数据用什么编码存储（**永远用 utf8mb4**） |
| `COLLATE utf8mb4_general_ci` | 字符怎么比较、排序（`_ci` 不区分大小写） |

> ⚠️ **注意：建库时设好 `utf8mb4`，之后这个库里新建的所有表都会默认继承它**，省得每张表都写一遍。详见 `表结构(DDL).md`。

### 1.4 CREATE SCHEMA 是同义词

```sql
-- MySQL 里 CREATE SCHEMA 和 CREATE DATABASE 完全等价
CREATE SCHEMA mydb;
-- （某些数据库如 PostgreSQL 里两者有区别，MySQL 里没有）
```

## 二、查看数据库

### 2.1 查看所有库

```sql
SHOW DATABASES;
```

**结果示例（会看到系统自带的库）：**

```
+--------------------+
| Database           |
+--------------------+
| information_schema |  ← 系统库（元数据）
| mysql              |  ← 系统库（用户权限）
| mydb               |  ← 你自己建的
| performance_schema |  ← 系统库（性能监控）
| sys                |  ← 系统库（系统视图）
+--------------------+
```

### 2.2 模糊查找

```sql
-- 查找名字以 "my" 开头的库
SHOW DATABASES LIKE 'my%';
```

### 2.3 查看建库语句（含字符集）

```sql
SHOW CREATE DATABASE mydb;
-- 输出完整的 CREATE DATABASE 语句，能看到字符集和排序规则
```

### 2.4 查看当前选中的库

```sql
SELECT DATABASE();
-- 返回当前正在使用的库名，没选则返回 NULL
```

## 三、选择/切换数据库 USE

**`USE` 用来切换到某个库**，之后操作表就不用每次写库名前缀了。

```sql
-- 切换到 mydb 库
USE mydb;

-- 切换后可以直接用表名
SELECT * FROM users;       -- 等价于 mydb.users

-- 没切换的话，要写全限定名
SELECT * FROM mydb.users;  -- 库名.表名
```

> 💡 **提示：** 命令行登录后**第一件事通常是 `USE 库名`**，否则不知道在哪操作。在 JDBC 连接串里通过 URL 指定库，就不用再 USE：`jdbc:mysql://localhost:3306/mydb`

## 四、修改数据库 ALTER DATABASE

`ALTER DATABASE` 只能改**库级别**的选项（字符集、排序规则），**不能改里面的表结构**（改表要用 `ALTER TABLE`，见 `表结构(DDL).md`）。

### 4.1 修改字符集

```sql
-- 把 mydb 的字符集改成 utf8mb4
ALTER DATABASE mydb CHARACTER SET utf8mb4;

-- 同时改字符集和排序规则
ALTER DATABASE mydb
CHARACTER SET utf8mb4
COLLATE utf8mb4_general_ci;
```

> ⚠️ **注意：`ALTER DATABASE` 改的是库的"默认字符集"**，只影响**之后新建的表**，**不会自动转换已有表**的字符集。已有表要逐个 `ALTER TABLE ... CONVERT TO CHARACTER SET`。

### 4.2 修改库本身改不了什么

```sql
-- ❌ 库名不能直接用 ALTER 改
ALTER DATABASE old_name RENAME TO new_name;   -- 不支持！

-- 改库名要绕一下：建新库 → 复制数据 → 删旧库
```

## 五、删除数据库 DROP DATABASE

### 5.1 基本语法

```sql
-- 删除整个数据库（里面的所有表、数据全部消失！）
DROP DATABASE mydb;
```

### 5.2 加上 IF EXISTS

```sql
-- 库不存在时不报错
DROP DATABASE IF EXISTS mydb;
```

### 5.3 ⚠️ 这是不可逆的危险操作

```
DROP DATABASE 会：
  ✅ 删除库里所有表
  ✅ 删除所有数据
  ✅ 删除库本身
  ❌ 无法回滚（DDL 语句，隐式提交）
  ❌ 没有确认提示

执行前务必三思！！！
```

> ⚠️ **最高警告：永远不要在生产环境手敲 `DROP DATABASE`。** 如果必须删，先**备份**，再**二次确认**，最好找人 review。

## 六、字符集与排序规则（建库核心配置）

### 6.1 为什么字符集这么重要

| 字符集 | 字节/字符 | 能存中文吗 | 能存 emoji 吗 |
|-------|----------|:--------:|:------------:|
| `latin1` | 1 字节 | ❌ | ❌ |
| `utf8`（MySQL 版） | 最多 3 字节 | ✅ | ❌ |
| **`utf8mb4`** | 最多 4 字节 | ✅ | ✅ |

> ⚠️ **注意：永远用 `utf8mb4`**。MySQL 的 `utf8` 是残缺版（最多 3 字节），存不下 emoji 和部分生僻字，会导致插入失败或乱码。建库时设好，一劳永逸。

### 6.2 查看可用的字符集和排序规则

```sql
-- 查看所有支持的字符集
SHOW CHARACTER SET;

-- 查看 utf8mb4 支持的排序规则
SHOW COLLATION LIKE 'utf8mb4%';
```

### 6.3 常用排序规则

| 排序规则 | 含义 |
|---------|------|
| `utf8mb4_general_ci` | 不区分大小写，速度快（常用） |
| `utf8mb4_unicode_ci` | 不区分大小写，更符合 Unicode 规范（更准确） |
| `utf8mb4_bin` | 二进制比较，区分大小写 |

```sql
-- _ci：'ABC' = 'abc'（不区分大小写）
-- _bin：'ABC' ≠ 'abc'（区分大小写）
```

## 七、数据库的备份与恢复（实战重点）

误删库、数据丢失时，备份是唯一的救命稻草。

### 7.1 用 mysqldump 导出（备份）

`mysqldump` 是命令行工具（不是 SQL），在**终端**执行：

```bash
# 导出整个数据库（结构 + 数据）
mysqldump -u root -p mydb > mydb_backup.sql

# 导出特定表
mysqldump -u root -p mydb users > users_backup.sql

# 导出多个表
mysqldump -u root -p mydb users orders > multi.sql

# 导出所有数据库
mysqldump -u root -p --all-databases > all_backup.sql

# 只导出表结构（不含数据）
mysqldump -u root -p --no-data mydb > structure.sql

# 只导出数据（不含建表语句）
mysqldump -u root -p --no-create-info mydb > data.sql
```

| 参数 | 作用 |
|-----|------|
| `-u root` | 用户名 |
| `-p` | 提示输入密码（密码不要直接写在命令里，会泄露） |
| `--no-data` | 只导结构 |
| `--no-create-info` | 只导数据 |
| `--all-databases` | 导出所有库 |
| `--databases` | 导出指定多个库 |

### 7.2 恢复数据库

**方式一：命令行导入**

```bash
# 先建好空库，再导入
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS mydb"
mysql -u root -p mydb < mydb_backup.sql
```

**方式二：在 mysql 客户端里用 SOURCE**

```sql
-- 登录 mysql 后
USE mydb;
SOURCE /path/to/mydb_backup.sql;
```

> 💡 **提示：** `SOURCE` 是 mysql 客户端的命令，**不是 SQL 语句**，只能在 mysql 命令行里用，不能在 JDBC/MyBatis 里执行。

### 7.3 备份的最佳实践

```
✅ 定期自动备份（用定时任务 cron 调 mysqldump）
✅ 备份文件存到异地（不要和数据库在同一台机器）
✅ 定期测试恢复（备份文件能恢复才是有效备份）
✅ 大库考虑用 --single-transaction 避免锁表
   mysqldump -u root -p --single-transaction mydb > mydb.sql

❌ 不要只备份不验证
❌ 不要把备份和数据库放同一台服务器（机器挂了一起没）
```

### 7.4 导出/导入数据的其他方式

```sql
-- 导出查询结果到文件（需要 FILE 权限）
SELECT * FROM users INTO OUTFILE '/tmp/users.csv'
FIELDS TERMINATED BY ',';

-- 从文件导入数据（比 INSERT 快得多，适合大数据量）
LOAD DATA INFILE '/tmp/users.csv'
INTO TABLE users
FIELDS TERMINATED BY ',';
```

> 💡 **提示：** `LOAD DATA INFILE` 导入超大文件比逐条 `INSERT` 快几十倍，是大数据量导入的首选。

## 八、MySQL 自带的系统库

`SHOW DATABASES` 看到的几个"系统库"是 MySQL 自带的，**不要随便改、不要删**：

| 系统库 | 作用 | 能删吗 |
|-------|------|:------:|
| `mysql` | 存用户账号、权限、时区等系统信息 | ❌ |
| `information_schema` | 元数据（有哪些表、列、索引），虚拟库 | ❌（虚拟的） |
| `performance_schema` | 性能监控数据（SQL 耗时、锁等） | ❌ |
| `sys` | 基于 performance_schema 的视图，方便人看 | ❌ |

### 实用查询：用 information_schema 查元数据

```sql
-- 查看某个库有哪些表
SELECT table_name FROM information_schema.tables
WHERE table_schema = 'mydb';

-- 查看某张表有哪些列
SELECT column_name, data_type FROM information_schema.columns
WHERE table_schema = 'mydb' AND table_name = 'users';

-- 查看所有库的总大小（数据 + 索引）
SELECT table_schema "库",
       ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) "大小(MB)"
FROM information_schema.tables
GROUP BY table_schema;
```

> 💡 **提示：** `information_schema` 是个**只读的虚拟库**，专门查"数据库自己的信息"（元数据），用起来很方便。

## 九、用户与权限（DCL 简述）

新建库后，通常要给应用分配一个**专用账号**，而不是用 root。

### 9.1 创建用户并授权

```sql
-- 创建用户（限制只能从哪些主机连接：'%' 表示任意主机）
CREATE USER 'appuser'@'%' IDENTIFIED BY '你的密码';

-- 授予 mydb 库所有表的 增删改查 权限
GRANT SELECT, INSERT, UPDATE, DELETE ON mydb.* TO 'appuser'@'%';

-- 刷新权限使其生效
FLUSH PRIVILEGES;
```

### 9.2 权限管理常用语句

```sql
-- 查看某用户的权限
SHOW GRANTS FOR 'appuser'@'%';

-- 撤销某项权限
REVOKE DELETE ON mydb.* FROM 'appuser'@'%';

-- 授予全部权限（慎用）
GRANT ALL PRIVILEGES ON mydb.* TO 'appuser'@'%';

-- 删除用户
DROP USER 'appuser'@'%';

-- 修改密码
ALTER USER 'appuser'@'%' IDENTIFIED BY '新密码';
```

**权限作用范围：**

| 写法 | 范围 |
|-----|------|
| `ON *.*` | 所有库所有表（全局，慎用） |
| `ON mydb.*` | mydb 库的所有表 |
| `ON mydb.users` | 仅 mydb 库的 users 表 |

> ⚠️ **注意：生产环境遵循**最小权限原则**——应用账号只给 `SELECT/INSERT/UPDATE/DELETE`，不要给 `DROP/ALTER/CREATE`，防止程序 bug 误删库表。

## 十、数据库命名规范

```sql
-- ✅ 规范的库名
CREATE DATABASE shop_order;     -- 小写，下划线分词
CREATE DATABASE blog_db;

-- ❌ 不规范的库名
CREATE DATABASE ShopOrder;      -- 大小写混用（Linux 下区分大小写，易出错）
CREATE DATABASE shop-order;     -- 用了连字符（需要反引号，易错）
CREATE DATABASE 123db;          -- 数字开头
CREATE DATABASE order;          -- 用了关键字
```

**命名规则：**

| 规则 | 说明 |
|-----|------|
| 用小写字母 + 下划线 | `user_order` ✅ |
| 不用关键字 | 别用 `order`、`user`、`select` 等 |
| 不以数字开头 | `1db` ❌ |
| 长度适中 | 不超过 64 字符 |
| 见名知意 | `shop`、`blog`、`cms` ✅ |

> 💡 **提示：Linux 下 MySQL 库名**区分大小写**（因为 Linux 文件系统区分），Windows/Mac 默认不区分。用小写命名最安全，避免跨平台迁移出问题。

## 十一、常见问题与注意事项

### 问题 1：建库报错 "database exists"

```sql
-- 原因：库已存在
-- 解决：用 IF NOT EXISTS
CREATE DATABASE IF NOT EXISTS mydb;

-- 或先删再建（确认数据不需要时）
DROP DATABASE IF EXISTS mydb;
CREATE DATABASE mydb;
```

### 问题 2：中文存进去变问号 / 乱码

```
根因：库或表的字符集不是 utf8mb4。
排查：
  SHOW CREATE DATABASE mydb;  -- 看库字符集
  SHOW CREATE TABLE users;    -- 看表字符集
解决：建库/建表时用 utf8mb4，连接串也指定编码
  jdbc:mysql://localhost:3306/mydb?useUnicode=true&characterEncoding=utf8
```

### 问题 3：怎么改库名？

```
MySQL 不支持直接 RENAME DATABASE。方案：
  1. 建新库 → mysqldump 导出旧库 → 导入新库 → 删旧库（最稳）
  2. InnoDB 可以改：先停库，去文件系统改目录名（有风险，不推荐）
  3. 用工具如 RENAME TABLE 逐表迁移
```

### 问题 4：DROP DATABASE 卡住或很慢

```
原因：库里有大量表/数据，删除要逐个删文件。
解决：
  - 大库删除会锁较久，避开业务高峰
  - 删前务必备份
  - 删除后磁盘空间会释放（InnoDB 的表文件被删）
```

### 问题 5：USE 后还是访问不到表

```
检查：
  1. 库名拼对了吗？（SHOW DATABASES;）
  2. 表真的在这个库里吗？（SHOW TABLES;）
  3. 有没有该库的权限？（SHOW GRANTS;）
```

### 问题 6：远程连不上数据库

```sql
-- 1. 用户允许远程连接吗（'%' 表示任意主机，'localhost' 只能本机）
CREATE USER 'appuser'@'%' IDENTIFIED BY 'pwd';

-- 2. MySQL 是否监听了远程端口（my.cnf 里 bind-address）
-- bind-address = 0.0.0.0  （允许远程，默认可能只监听 127.0.0.1）

-- 3. 防火墙是否放行 3306 端口
```

## 十二、快速参考

### 库操作语句总览

```sql
-- 创建
CREATE DATABASE [IF NOT EXISTS] 库名
  [DEFAULT CHARACTER SET utf8mb4]
  [DEFAULT COLLATE utf8mb4_general_ci];

-- 查看
SHOW DATABASES [LIKE 'pattern'];
SHOW CREATE DATABASE 库名;
SELECT DATABASE();              -- 当前库

-- 切换
USE 库名;

-- 修改（只能改字符集等库级选项）
ALTER DATABASE 库名 CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 删除（危险！不可逆）
DROP DATABASE [IF EXISTS] 库名;
```

### 备份与恢复速查

```bash
# 备份
mysqldump -u root -p 库名 > 备份文件.sql              # 单库
mysqldump -u root -p 库名 表名 > 备份文件.sql         # 单表
mysqldump -u root -p --all-databases > 全部.sql       # 全部库
mysqldump -u root -p --single-transaction 库名 > x.sql  # 一致性快照

# 恢复
mysql -u root -p 库名 < 备份文件.sql                   # 命令行导入
# 或在 mysql 内：USE 库名; SOURCE /路径/备份文件.sql;
```

### 用户权限速查

```sql
CREATE USER '用户'@'主机' IDENTIFIED BY '密码';        -- 建用户
GRANT 权限 ON 库名.* TO '用户'@'主机';                  -- 授权
REVOKE 权限 ON 库名.* FROM '用户'@'主机';               -- 撤权
SHOW GRANTS FOR '用户'@'主机';                          -- 查权限
ALTER USER '用户'@'主机' IDENTIFIED BY '新密码';        -- 改密码
DROP USER '用户'@'主机';                                -- 删用户
FLUSH PRIVILEGES;                                       -- 刷新权限
```

### 操作前自检清单

```
□ 建库时设了 utf8mb4 字符集吗？
□ CREATE / DROP 加了 IF NOT EXISTS / IF EXISTS 吗？
□ 删库前备份了吗？（DROP DATABASE 不可逆！）
□ 应用账号用了最小权限吗？（别用 root 连库）
□ 备份文件能成功恢复吗？（定期演练）
□ 库名符合规范吗？（小写 + 下划线）
```
