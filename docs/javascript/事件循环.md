### js是单线程的
* js是单线程的，但是浏览器是多进程的
* js代码执行是在一个单独的线程中，这意味着在同一时间只能执行一个任务

### 事件循环
* 事件循环是浏览器用来处理异步操作的机制
* 当一个任务被添加到队列中时，它会等待当前执行的任务完成后再开始执行
* 当执行栈空闲时，会反复从任务队列中取出任务执行

### 事件循环的执行过程
1. 执行栈（Execution Stack）中有一个全局环境，执行全局代码，同步代码  
2. 遇到异步操作，会在宿主环境中(浏览器)执行，**执行之后**并将回调函数放入**回调队列**中等待。<mark>(即，异步函数**不是立刻**加到任务队列中，而是回调函数达到执行时机后才加入到任务队列中)</mark>
3. 当执行栈空闲时，会反复从任务队列中取出任务执行
4. 一旦主线程上的**所有同步代码都执行完毕**，事件循环就会检查回调队列是否有待处理的回调函数，如果有的话，会将它们加入到执行栈中进行处理
5. 这个过程一直重复进行，直到所有的任务都被处理完为止

### 宏任务
* 由**宿主环境（浏览器）**执行的异步代码，比如setTimeout、setInterval等

|宏任务|执行环境|
|--|--|
|定时器(setTimeout/setInterval)|浏览器环境|
|DOM事件的回调|浏览器环境|
|Ajax请求的回调|浏览器环境|

### 微任务
* 微任务是由**JavaScript引擎**执行的异步代码，比如Promise、MutationObserver等

|微任务|执行环境|
|--|--|
|Promise的**回调函数**(Promise.then/catch/finally)|JavaScript引擎|
|MutationObserver的**回调函数**|JavaScript引擎|

### 宏任务和微任务的执行顺序
1. 执行宏任务之前，会把微任务执行完， 在执行一个宏任务（栈中没有就从宏任务队列中取一个执行）
2. 执行过程遇到异步任务，**执行之后**将回调函数放到任务队列中， 宏任务放到**宏任务队列**，微任务放到**微任务队列**
3. 宏任务执行完毕后，若微任务不为空，就执行**所有**的微任务（依次执行）
4. 继续下一个宏任务，重复上述过程
