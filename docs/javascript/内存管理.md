# JavaScript 内存管理

## 简介

JavaScript 是一门**自动管理内存**的语言，开发者不需要手动分配和释放内存。内存的分配和回收由 JavaScript 引擎自动完成。

## 内存结构图

```
┌─────────────────────────────────────────────────────────┐
│                    JavaScript 内存空间                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌──────────────┐              ┌──────────────────┐    │
│  │              │              │                  │    │
│  │   栈内存     │              │     堆内存        │    │
│  │  (Stack)     │              │     (Heap)       │    │
│  │              │              │                  │    │
│  │  • 原始值    │              │  • 对象          │    │
│  │  • 指针      │              │  • 数组          │    │

│  │              │              │  • 闭包数据      │    │
│  │  先进后出    │              │                  │    │
│  │  空间固定    │              │  动态分配        │    │
```

## 内存生命周期

内存管理分为三个阶段：

```
┌─────────┐      ┌─────────────┐      ┌──────────────┐
│ 分配内存 │  ──> │  使用内存   │  ──> │  释放内存    │
└─────────┘      └─────────────┘      └──────────────┘
                    (读写操作)            (垃圾回收)
```

### 1. 分配内存

```javascript
// 自动分配内存

// 栈内存分配：原始类型
let a = 10;           // Number
let b = "hello";      // String
let c = true;         // Boolean
let d = null;         // Null
let e = undefined;    // Undefined
let f = Symbol();     // Symbol
let g = 10n;          // BigInt

// 堆内存分配：引用类型
let obj = { name: "张三" };        // 对象
let arr = [1, 2, 3];               // 数组
let fn = function() {};            // 函数
let date = new Date();             // Date 对象
```

### 2. 使用内存

```javascript
// 读写操作
obj.name = "李四";      // 写入
console.log(obj.name);  // 读取
arr.push(4);            // 写入
console.log(arr[0]);    // 读取
```

### 3. 释放内存

```javascript
// JavaScript 引擎自动释放
obj = null;  // 解除引用，等待垃圾回收
```

## 栈内存 (Stack)

### 特点

| 特性 | 说明 |
|------|------|
| 存储内容 | 原始类型值、引用类型地址、函数执行上下文 |
| 分配方式 | 编译时确定大小，自动分配释放 |
| 访问速度 | 快速（直接访问） |
| 空间大小 | 固定且有限 |
| 结构 | 线性、有序、LIFO（后进先出） |

### 栈内存结构

```
┌────────────────────────────────────┐
│           栈内存                    │
├────────────────────────────────────┤
│  ┌──────────────────────────────┐  │
│  │   函数执行上下文 (C)          │  │  ← 栈顶
│  │   • 局部变量                  │  │
│  │   • 参数                      │  │
│  │   • 返回地址                  │  │
│  └──────────────────────────────┘  │
│  ┌──────────────────────────────┐  │
│  │   函数执行上下文 (B)          │  │
│  │   • 局部变量                  │  │
│  │   • 参数                      │  │
│  │   • 返回地址                  │  │
│  └──────────────────────────────┘  │
│  ┌──────────────────────────────┐  │
│  │   全局执行上下文 (A)          │  │  ← 栈底
│  │   • 全局变量                  │  │
│  │   • this                      │  │
│  └──────────────────────────────┘  │
└────────────────────────────────────┘
```

### 栈内存示例

```javascript
// 原始类型存储在栈中
let a = 10;
let b = 20;
let c = a;  // 复制值，c = 10

// 栈内存状态
// ┌───┬───┬───┐
// │ a │ b │ c │
// │10 │20 │10 │
// └───┴───┴───┘

b = 30;
// a 仍为 10，c 仍为 10
// ┌───┬───┬───┐
// │ a │ b │ c │
// │10 │30 │10 │
// └───┴───┴───┘
```

### 函数调用栈

```javascript
function A() {
  console.log('A 开始');
  B();
  console.log('A 结束');
}

function B() {
  console.log('B 开始');
  C();
  console.log('B 结束');
}

function C() {
  console.log('C 执行');
}

A();
```

**调用栈变化：**

```
调用 A()：
┌──────────────┐
│ 全局上下文   │ ← 栈底
└──────────────┘

A() 入栈：
┌──────────────┐
│ A 的上下文   │ ← 栈顶
├──────────────┤
│ 全局上下文   │ ← 栈底
└──────────────┘

B() 入栈：
┌──────────────┐
│ B 的上下文   │ ← 栈顶
├──────────────┤
│ A 的上下文   │
├──────────────┤
│ 全局上下文   │ ← 栈底
└──────────────┘

C() 入栈：
┌──────────────┐
│ C 的上下文   │ ← 栈顶
├──────────────┤
│ B 的上下文   │
├──────────────┤
│ A 的上下文   │
├──────────────┤
│ 全局上下文   │ ← 栈底
└──────────────┘

C() 执行完出栈：
┌──────────────┐
│ B 的上下文   │ ← 栈顶
├──────────────┤
│ A 的上下文   │
├──────────────┤
│ 全局上下文   │ ← 栈底
└──────────────┘

B() 执行完出栈：
┌──────────────┐
│ A 的上下文   │ ← 栈顶
├──────────────┤
│ 全局上下文   │ ← 栈底
└──────────────┘

A() 执行完出栈：
┌──────────────┐
│ 全局上下文   │ ← 栈顶 + 栈底
└──────────────┘
```

### 栈溢出 (Stack Overflow)

当调用栈超过最大限制时发生：

```javascript
// 递归没有终止条件会导致栈溢出
function recursive() {
  recursive();
}

recursive();
// Uncaught RangeError: Maximum call stack size exceeded
```

## 堆内存 (Heap)

### 特点

| 特性 | 说明 |
|------|------|
| 存储内容 | 对象、数组、函数等引用类型 |
| 分配方式 | 运行时动态分配 |
| 访问速度 | 较慢（通过引用访问） |
| 空间大小 | 动态、较大 |
| 结构 | 无序、树状结构 |

### 堆内存结构

```
┌─────────────────────────────────────────────────────┐
│                    堆内存                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌───────────────┐      ┌───────────────┐          │
│  │  对象 #1      │      │  对象 #2      │          │
│  │  {name:"A"}  │◄─────┤  {age:20}    │          │
│  │  @0x001      │      │  @0x002       │          │
│  └───────────────┘      └───────────────┘          │
│         │                       ▲                   │
│         │                       │                   │
│         ▼                       │                   │
│  ┌───────────────┐              │                   │
│  │  对象 #3      │              │                   │
│  │  [1,2,3]     │              │                   │
│  │  @0x003       │──────────────┘                   │
│  └───────────────┘                                  │
│                                                     │
│  动态分配，大小不固定，通过引用地址访问              │
└─────────────────────────────────────────────────────┘
```

### 堆内存示例

```javascript
// 引用类型：对象存储在堆中，栈中存储地址引用
let obj1 = { name: "张三" };
let obj2 = obj1;  // 复制引用地址

// 内存布局
// 栈：
// ┌──────────────┐
// │ obj1 │ 0x001 │ ─┐
// ├──────────────┤  │
// │ obj2 │ 0x001 │ ─┼── 都指向同一个堆对象
// └──────────────┘  │
//                   │
// 堆：               ▼
// ┌──────────────────────────────┐
// │ 0x001: { name: "张三" }      │
// └──────────────────────────────┘

obj2.name = "李四";
console.log(obj1.name);  // "李四" - 因为指向同一对象

obj2 = { name: "王五" };
// 栈：
// ┌──────────────┐
// │ obj1 │ 0x001 │ ─┐
// ├──────────────┤  │
// │ obj2 │ 0x002 │ ─┼── 新对象
// └──────────────┘  │
//                   │
// 堆：               ▼
// ┌────────────────────────────────┐
// │ 0x001: { name: "李四" }        │
// │ 0x002: { name: "王五" }        │
// └────────────────────────────────┘

console.log(obj1.name);  // "李四"
console.log(obj2.name);  // "王五"
```

### 数组在堆内存中

```javascript
let arr1 = [1, 2, 3];
let arr2 = arr1;

// 栈：
// ┌──────────────┐
// │ arr1 │ 0x001 │ ─┐
// ├──────────────┤  │
// │ arr2 │ 0x001 │ ─┘
// └──────────────┘
//
// 堆：
// ┌─────────────────────────────┐
// │ 0x001: [1, 2, 3]            │
// └─────────────────────────────┘

arr2.push(4);
console.log(arr1);  // [1, 2, 3, 4]

arr2 = [5, 6];
console.log(arr1);  // [1, 2, 3, 4]
console.log(arr2);  // [5, 6]
```

## 栈与堆的对比

### 详细对比表

| 对比项 | 栈内存 | 堆内存 |
|--------|--------|--------|
| **存储内容** | 原始类型、引用地址 | 对象、数组、函数 |
| **分配时机** | 编译时（静态） | 运行时（动态） |
| **分配方式** | 系统自动 | 动态申请 |
| **释放方式** | 自动（LIFO） | 垃圾回收 |
| **空间大小** | 小（约 1-8MB） | 大（可达 GB 级） |
| **访问速度** | 快（直接访问） | 慢（通过指针） |
| **内存结构** | 线性、连续 | 非线性、离散 |
| **管理成本** | 低 | 高 |
| **生命周期** | 随函数调用 | 取决于引用 |

## 内存分配详解

### 变量声明与内存分配

```javascript
// ========== 原始类型 - 栈分配 ==========
let num = 100;
// 栈：
// ┌────────┐
// │ num │100│
// └────────┘

let str = "hello";
// 栈：
// ┌────────┐  ┌──────────┐
// │ num │100│  │ str │"hello"│
// └────────┘  └──────────┘

let bool = true;
// 栈：
// ┌────────┐  ┌──────────┐  ┌─────────┐
// │ num │100│  │ str │"hello"│ │ bool│true│
// └────────┘  └──────────┘  └─────────┘


// ========== 引用类型 - 堆分配 ==========
let obj = { x: 1, y: 2 };
// 栈：
// ┌────────┐  ┌──────────┐  ┌─────────┐  ┌──────────┐
// │ num │100│  │ str │"hello"│ │ bool│true│  │ obj │0x001│ ─┐
// └────────┘  └──────────┘  └─────────┘  └──────────┘  │
//                                                           │
// 堆：                                                       │
// ┌────────────────────────────────────────────────────────┘
// │ 0x001: { x: 1, y: 2 }                                  │
// └────────────────────────────────────────────────────────┘


let arr = [10, 20, 30];
// 栈：
// ┌────────┐  ┌──────────┐  ┌─────────┐  ┌──────────┐  ┌──────────┐
// │ num │100│  │ str │"hello"│ │ bool│true│  │ obj │0x001│  │ arr │0x002│ ─┐
// └────────┘  └──────────┘  └─────────┘  └──────────┘  └──────────┘  │
//                                                                      │
// 堆：                                                                  │
// ┌──────────────────────────────────────────────────────────────────┐
// │ 0x001: { x: 1, y: 2 }                                            │
// │ 0x002: [10, 20, 30]                                              │
// └──────────────────────────────────────────────────────────────────┘
```

### 函数参数传递

```javascript
// 原始类型：值传递
function modifyPrimitive(x) {
  x = 100;  // 只修改局部变量
}

let a = 10;
modifyPrimitive(a);
console.log(a);  // 10 - 原值不变

// 内存变化：
// 调用前：
// 栈：
// ┌────┐
// │ a │10│
// └────┘

// 调用时（复制值）：
// 栈：
// ┌────────┐
// │ a  │ 10│  ← 全局变量
// ├────────┤
// │ x  │ 10│  ← 函数参数（副本）
// └────────┘

// 函数内修改后：
// 栈：
// ┌────────┐
// │ a  │ 10│  ← 不受影响
// ├────────┤
// │ x  │100│  ← 局部变量被修改
// └────────┘

// 函数返回后：
// 栈：
// ┌────┐
// │ a │10│
// └────┘


// 引用类型：引用传递
function modifyObject(obj) {
  obj.name = "李四";  // 修改堆中的对象
}

let person = { name: "张三" };
modifyObject(person);
console.log(person.name);  // "李四" - 原对象被修改

// 内存变化：
// 调用前：
// 栈：
// ┌──────────────┐
// │person│ 0x001 │ ─┐
// └──────────────┘  │
//                    │
// 堆：                ▼
// ┌──────────────────────────────┐
// │ 0x001: { name: "张三" }      │
// └──────────────────────────────┘

// 调用时（复制引用地址）：
// 栈：
// ┌──────────────┐
// │person│ 0x001 │ ─┐
// ├──────────────┤  │
// │  obj │ 0x001 │ ─┼── 指向同一对象
// └──────────────┘  │
//                    │
// 堆：                ▼
// ┌──────────────────────────────┐
// │ 0x001: { name: "张三" }      │
// └──────────────────────────────┘

// 函数内修改后：
// 堆：
// ┌──────────────────────────────┐
// │ 0x001: { name: "李四" }      │ ← 对象被修改
// └──────────────────────────────┘


// 重新赋值引用
function reassignObject(obj) {
  obj = { name: "王五" };  // 修改的是参数引用
}

let user = { name: "张三" };
reassignObject(user);
console.log(user.name);  // "张三" - 原对象不变

// 内存变化：
// 函数内重新赋值后：
// 栈：
// ┌──────────────┐
// │ user │ 0x001 │ ─┐
// ├──────────────┤  │
// │  obj │ 0x002 │ ─┼── 新对象
// └──────────────┘  │
//                    │
// 堆：                ▼
// ┌────────────────────────────────┐
// │ 0x001: { name: "张三" }        │ ← 原对象不变
// │ 0x002: { name: "王五" }        │ ← 新对象
// └────────────────────────────────┘
```

## 垃圾回收 (Garbage Collection)

JavaScript 引擎自动进行垃圾回收，主要有两种算法：

### 1. 引用计数 (Reference Counting)

**原理**：记录每个对象被引用的次数，次数为 0 时回收。

```javascript
// 引用计数示例
let objA = { value: 1 };  // 对象 { value: 1 } 引用计数 = 1
let objB = objA;          // 引用计数 = 2

objA = null;              // 引用计数 = 1
objB = null;              // 引用计数 = 0 → 可以回收

// 循环引用问题
function circularReference() {
  let obj1 = {};
  let obj2 = {};

  obj1.ref = obj2;  // obj1 引用 obj2
  obj2.ref = obj1;  // obj2 引用 obj1

  // 函数结束后，两个对象仍然互相引用
  // 引用计数都为 1，无法回收 → 内存泄漏
}

circularReference();
```

### 2. 标记清除 (Mark-and-Sweep)

**原理**：从根对象开始遍历，标记可达对象，未标记的清除。

```javascript
// 标记清除过程
let root = {
  name: "root",
  child: {
    name: "child"
  }
};

let orphan = { name: "orphan" };  // 可达对象

// 假设 root 是全局变量（根）
//
// 第一次遍历（标记阶段）：
// 1. root 是根对象 → 标记为可达
// 2. root.child 是 root 的属性 → 标记为可达
// 3. orphan 是全局变量 → 标记为可达
//
// 第二次遍历（清除阶段）：
// 清除所有未被标记的对象

root = null;
// root.child 仍可达吗？不 → 将被回收
orphan = null;
// orphan → 将被回收
```

### V8 引擎的垃圾回收优化

#### 分代回收

V8 将内存分为新生代和老生代：

```
┌─────────────────────────────────────────────────────┐
│                   V8 堆内存                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌───────────────────┐    ┌─────────────────────┐  │
│  │    新生代         │    │     老生代          │  │
│  │  (New Space)     │    │  (Old Space)       │  │
│  │                   │    │                     │  │
│  │  • 小对象         │    │  • 存活久的对象     │  │
│  │  • 短生命周期     │    │  • 大对象           │  │
│  │  • 1-8MB         │    │  • 主要内存区域     │  │
│  │                   │    │                     │  │
│  │  Scavenge 算法   │    │  • 标记清除         │  │
│  │  (复制)          │    │  • 标记整理         │  │
│  └───────────────────┘    └─────────────────────┘  │
│         ▲                        ▲                 │
│         │                        │                 │
│         └────────────────────────┘                 │
│      对象晋升（存活次数多、空间不足）               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

#### Scavenge 算法（新生代）

```javascript
// 新生代使用 Cheney 算法
// 将内存分为 From 和 To 两部分

// 初始状态：
// ┌─────────────┬─────────────┐
// │   From      │     To      │
// │  ┌─────┐    │             │
// │  │ obj1│    │   (空)      │
// │  │ obj2│    │             │
// │  └─────┘    │             │
// └─────────────┴─────────────┘

// 垃圾回收后：
// 1. 复制存活对象到 To
// 2. 清空 From
// 3. 交换 From/To

// ┌─────────────┬─────────────┐
// │   From      │     To      │
// │  (空)       │  ┌─────┐    │
// │             │  │ obj1│    │  ← obj2 被回收
// │             │  └─────┘    │
// └─────────────┴─────────────┘
```

#### 标记整理（老生代）

```javascript
// 整理阶段会将存活对象向一端移动，消除内存碎片

// 整理前：
// ┌──┬──┬──┬──┬──┬──┬──┬──┐
// │A │_ │B │_ │C │_ │_ │D │  │  _ = 碎片空间

// 整理后：
```

## 内存泄漏 (Memory Leak)

### 常见原因及解决方案

#### 1. 意外的全局变量

```javascript
// ❌ 错误：创建意外的全局变量
function leak1() {
  leakyVar = "I'm global now!";  // 没有 let/const/var
}

// 相当于：
// window.leakyVar = "I'm global now!";

// ✅ 正确：使用严格模式
'use strict';
function noLeak() {
  // ReferenceError: leakyVar is not defined
  // leakyVar = "error";
  let localVar = "safe";  // OK
}


// ❌ 错误：this 指向全局
function leak2() {
  this.leaky = "oops";  // 非严格模式下 this 指向 window
}

// ✅ 正确：避免 this 指向全局
function safe() {
  'use strict';
  // this 指向 undefined
}
```

#### 2. 未清理的定时器

```javascript
// ❌ 错误：定时器未清理
function startTimer() {
  setInterval(() => {
    console.log('running...');
    // 如果这里引用了大对象，会造成内存泄漏
  }, 1000);
}

// ✅ 正确：保存引用并清理
let timerId = null;

function startTimer() {
  timerId = setInterval(() => {
    console.log('running...');
  }, 1000);
}

function stopTimer() {
  if (timerId) {
    clearInterval(timerId);
    timerId = null;
  }
}

// 组件中使用（Vue 示例）
export default {
  data() {
    return {
      timerId: null
    };
  },
  mounted() {
    this.timerId = setInterval(() => {
      // 定时任务
    }, 1000);
  },
  beforeUnmount() {  // Vue 2: beforeDestroy
    if (this.timerId) {
      clearInterval(this.timerId);
    }
  }
};
```

#### 3. 闭包持有引用

```javascript
// ❌ 潜在问题：闭包持有大对象
function createHandler() {
  let largeData = new Array(1000000).fill('data');

  return function() {
    // 虽然没有直接使用 largeData
    // 但闭包仍然持有对它的引用
    console.log('handler called');
  };
}

const handler = createHandler();
// largeData 不会被回收

// ✅ 解决方案：手动清除引用
function createHandler() {
  let largeData = new Array(1000000).fill('data');

  return function() {
    console.log('handler called');
    // 使用后清除
    largeData = null;
  };
}


// 实际场景示例
function setupButtons() {
  const hugeData = fetchLargeData();

  document.querySelectorAll('.button').forEach(btn => {
    btn.addEventListener('click', function() {
      // 事件处理函数持有 hugeData 引用
      handleClick(hugeData);
    });
  });
}

// ✅ 解决方案
function setupButtons() {
  document.querySelectorAll('.button').forEach(btn => {
    btn.addEventListener('click', function() {
      // 按需获取数据
      const data = fetchLargeData();
      handleClick(data);
    });
  });
}
```

#### 4. DOM 引用未清理

```javascript
// ❌ 错误：保留已删除 DOM 的引用
let elements = [];

function addElements() {
  for (let i = 0; i < 100; i++) {
    const div = document.createElement('div');
    document.body.appendChild(div);
    elements.push(div);  // 保留引用
  }
}

function removeElements() {
  elements.forEach(el => el.remove());
  // elements 数组仍保留 DOM 引用！
}

// ✅ 正确：同时清除引用
function removeElements() {
  elements.forEach(el => el.remove());
  elements = [];  // 清空数组
}


// ❌ 错误：事件监听器未移除
function attachEvents() {
  const button = document.querySelector('#myButton');
  button.addEventListener('click', handleClick);
}

// ✅ 正确：移除事件监听器
function attachEvents() {
  const button = document.querySelector('#myButton');
  button.addEventListener('click', handleClick);

  // 返回清理函数
  return function cleanup() {
    button.removeEventListener('click', handleClick);
  };
}

const cleanup = attachEvents();
// 组件卸载时调用
cleanup();


// Vue 组件示例
export default {
  mounted() {
    this.handleClick = () => {
      // 处理点击
    };
    document.addEventListener('click', this.handleClick);
  },
  beforeUnmount() {
    document.removeEventListener('click', this.handleClick);
    this.handleClick = null;
  }
};
```

#### 5. 未断开的观察者

```javascript
// ❌ 错误：订阅未取消
class EventEmitter {
  constructor() {
    this.listeners = [];
  }

  on(event, callback) {
    this.listeners.push({ event, callback });
  }

  emit(event, data) {
    this.listeners
      .filter(l => l.event === event)
      .forEach(l => l.callback(data));
  }
}

const emitter = new EventEmitter();
emitter.on('data', handler);
// emitter 不会被清理，handler 也不会被清理

// ✅ 正确：提供取消订阅方法
class EventEmitter {
  constructor() {
    this.listeners = [];
  }

  on(event, callback) {
    const listener = { event, callback };
    this.listeners.push(listener);

    // 返回取消函数
    return () => {
      const index = this.listeners.indexOf(listener);
      if (index > -1) {
        this.listeners.splice(index, 1);
      }
    };
  }

  emit(event, data) {
    this.listeners
      .filter(l => l.event === event)
      .forEach(l => l.callback(data));
  }
}

const emitter = new EventEmitter();
const unsubscribe = emitter.on('data', handler);

// 不需要时取消订阅
unsubscribe();
```

## 内存优化技巧

### 1. 及时释放引用

```javascript
// ✅ 使用完后设为 null
let largeData = fetchData();
processData(largeData);
largeData = null;  // 释放引用
```

### 2. 避免创建不必要的对象

```javascript
// ❌ 每次循环都创建新对象
for (let i = 0; i < 1000; i++) {
  const temp = { value: i };
  process(temp);
}

// ✅ 复用对象
const temp = {};
for (let i = 0; i < 1000; i++) {
  temp.value = i;
  process(temp);
}
```

### 3. 使用 WeakMap/WeakSet

```javascript
// WeakMap 不会阻止垃圾回收
const weakMap = new WeakMap();

let obj = { key: 'value' };
weakMap.set(obj, 'metadata');

obj = null;  // obj 可被回收，WeakMap 中的条目自动移除
```

### 4. 避免过度的闭包

```javascript
// ❌ 每次调用都创建闭包
function createFunctions() {
  const result = [];
  for (let i = 0; i < 1000; i++) {
    result.push(() => i);
  }
  return result;
}

// ✅ 减少闭包作用域
function createFunctions() {
  const result = [];
  for (let i = 0; i < 1000; i++) {
    result.push((function(n) {
      return () => n;
    })(i));
  }
  return result;
}
```

### 5. 虚拟列表（大数据渲染）

```javascript
// 只渲染可见区域的数据
class VirtualList {
  constructor(items, itemHeight, containerHeight) {
    this.items = items;
    this.itemHeight = itemHeight;
    this.containerHeight = containerHeight;
    this.visibleCount = Math.ceil(containerHeight / itemHeight);
  }

  getVisibleItems(scrollTop) {
    const start = Math.floor(scrollTop / this.itemHeight);
    const end = start + this.visibleCount;
    return this.items.slice(start, end);
  }
}
```

## 总结

### 关键概念速查

| 概念 | 说明 |
|------|------|
| **栈内存** | 存储原始类型和引用地址，LIFO，快速 |
| **堆内存** | 存储对象和数组，动态分配，灵活 |
| **值传递** | 原始类型传递值的副本 |
| **引用传递** | 引用类型传递地址的副本 |
| **垃圾回收** | 自动回收不再使用的内存 |
| **内存泄漏** | 内存无法被回收，持续占用 |

### 最佳实践

```javascript
// ✅ DO - 推荐做法
1. 使用 const/let，避免意外的全局变量
2. 及时清理定时器和事件监听器
3. 使用 WeakMap/WeakSet 存储关联数据
4. 大对象使用完设为 null
5. 使用严格模式 'use strict'

// ❌ DON'T - 避免做法
1. 在循环中创建大量对象
2. 保留已删除 DOM 的引用
3. 闭包中持有不需要的大对象
4. 忘记取消订阅和事件监听
5. 创建意外的全局变量
```
