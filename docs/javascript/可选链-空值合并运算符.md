# 可选链与空值合并运算符

## 概述

| 运算符 | 名称 | 描述 | ES 版本 |
|--------|------|------|---------|
| `?.` | 可选链 | 安全访问嵌套属性 | ES2020 |
| `??` | 空值合并 | 为 null/undefined 提供默认值 | ES2020 |

---

## 可选链运算符 `?.`

### 基本语法

```javascript
obj?.prop
obj?.[expr]
arr?.[index]
func?.(args)
```

### 核心作用

安全地访问嵌套对象属性，避免 `Cannot read property 'XXX' of undefined` 错误。

### 使用场景对比

```javascript
const user = {
  name: 'Alice',
  address: {
    city: 'Beijing'
  }
};

// ❌ 传统写法 - 容易报错
const city = user.address.city;  // 如果 address 是 undefined，直接报错

// ⚠️ 短路写法 - 冗长
const city = user && user.address && user.address.city;

// ✅ 可选链 - 简洁安全
const city = user?.address?.city;  // 安全访问
```

### 各种用法示例

```javascript
// 1. 对象属性访问
const street = user?.address?.street;

// 2. 数组元素访问
const firstItem = items?.[0];
const nestedItem = data?.list?.[0]?.name;

// 3. 方法调用
const result = validator?.validate?.(input);
// 只有 validator 存在且 validate 是函数时才调用

// 4. 连续调用
const value = obj?.a?.b?.c?.d;
// 任何一环为 null/undefined 都返回 undefined

// 5. 删除属性
delete user?.address?.street;
```

### 短路求值行为

```javascript
// 如果左侧为 null/undefined，右侧表达式不会执行
const result = obj?.getValue?.();  // obj 为 null 时，getValue() 不会执行

const index = 0;
const item = array?.[index++];     // array 为 null 时，index 不会递增
```

### 与空值合并配合使用

```javascript
const name = user?.profile?.name ?? '匿名用户';
const email = user?.contact?.email ?? '未设置';
```

## 空值合并运算符 `??`

### 基本语法

```javascript
leftExpr ?? rightExpr
```

### 核心作用

仅当左侧为 `null` 或 `undefined` 时，才返回右侧值。

### 真值表

| 左侧值 | `??` 结果 | 说明 |
|--------|----------|------|
| `null` | 右侧值 | 触发 |
| `undefined` | 右侧值 | 触发 |
| `0` | `0` | 不触发 |
| `""` | `""` | 不触发 |
| `false` | `false` | 不触发 |
| `NaN` | `NaN` | 不触发 |

### 与 `||` 运算符对比

```javascript
// || 逻辑或运算符 - 所有假值都会触发
const count = 0 || 10;         // 10
const empty = '' || 'default'; // 'default'
const flag = false || true;    // true
const none = null || 'value';  // 'value'

// ?? 空值合并 - 只有 null/undefined 触发
const count = 0 ?? 10;         // 0
const empty = '' ?? 'default'; // ''
const flag = false ?? true;    // false
const none = null ?? 'value';  // 'value'
```

### 实际应用示例

```javascript
// 1. 函数参数默认值
function getConfig(options) {
  const timeout = options?.timeout ?? 3000;
  const retries = options?.retries ?? 3;
  const debug = options?.debug ?? false;
  // ...
}

// 2. API 响应处理
const username = response?.data?.user?.name ?? 'Guest';
const avatar = response?.data?.user?.avatar ?? '/default.png';

// 3. 表单输入处理
const email = formData?.email ?? '';
const age = formData?.age ?? null;

// 4. 配置文件
const config = {
  port: env.PORT ?? 3000,
  host: env.HOST ?? 'localhost',
  debug: env.DEBUG ?? false
};

// 5. 数组处理
const items = data?.items ?? [];
const tags = post?.tags ?? [];
```

## 组合使用

### 常见组合模式

```javascript
// 模式 1: 可选链 + 空值合并
const title = article?.meta?.title ?? '无标题';
const price = product?.pricing?.amount ?? 0;

// 模式 2: 多层嵌套
const avatarUrl = user?.profile?.avatar?.url ?? '/placeholder.jpg';

// 模式 3: 数组操作
const firstCategory = product?.categories?.[0] ?? '未分类';

// 模式 4: 函数调用结果
const formatted = value?.toString() ?? 'N/A';
```

### 实战示例

```javascript
// 处理复杂的 API 响应
function formatUserInfo(response) {
  return {
    id: response?.data?.id ?? '',
    name: response?.data?.profile?.name ?? '未知用户',
    email: response?.data?.contact?.email ?? '未设置',
    avatar: response?.data?.images?.avatar?.thumbnail ?? '/default.png',
    role: response?.data?.permissions?.role ?? 'guest',
    lastLogin: response?.data?.activity?.lastLogin ?? null
  };
}

// 安全的 DOM 操作
const value = document?.querySelector('#input')?.value ?? '';

// 安全访问 localStorage
const theme = localStorage?.getItem?.('theme') ?? 'light';
```

## 注意事项

### 语法限制

```javascript
// ❌ 不能与 || 或 && 直接连用（需要括号）
null || undefined ?? 'foo';  // SyntaxError
(null || undefined) ?? 'foo'; // ✓ 正确

// ❌ 可选链不能用于赋值
obj?.prop = value;  // SyntaxError
obj?.[key] = value; // SyntaxError

// ❌ 不能用于模板字面量
const text = `Hello ${user?.name ?? 'Guest'}`;  // ✓ 需要括号包裹
```

### 使用建议

```javascript
// ✓ 推荐：明确检查 null/undefined 时使用 ??
const count = inputCount ?? 0;

// ✓ 推荐：处理可能缺失的属性使用 ?.
const city = user?.address?.city;

// ✓ 推荐：两者配合处理深层嵌套
const street = user?.address?.street ?? '未知地址';

// ⚠️ 谨慎：当 0/false/'' 是有效值时，不要用 ||
const discount = product?.discount ?? 0;  // 正确
const discount = product?.discount || 0;  // 错误：discount 为 0 时会被覆盖
```

## 总结

| 场景 | 推荐写法 |
|------|----------|
| 访问可能不存在的属性 | `obj?.prop` |
| 调用可能不存在的方法 | `func?.()` |
| 为 null/undefined 提供默认值 | `value ?? default` |
| 保留 0/false/'' 等假值 | `value ?? default` |
| 任意假值都替换 | `value \|\| default` |

### 最佳实践

```javascript
// 1. 深层属性访问
const value = data?.level1?.level2?.property ?? 'default';

// 2. 数组安全访问
const item = array?.[index] ?? null;

// 3. 函数安全调用
const result = callback?.();

// 4. 配置参数处理
function init(options = {}) {
  const {
    timeout = 3000,
    retries = 3,
    debug = false
  } = options;
  // ...
}

// 5. 表单处理
const form = {
  name: data?.name ?? '',
  age: data?.age ?? null,
  subscribed: data?.subscribed ?? false
};
```
