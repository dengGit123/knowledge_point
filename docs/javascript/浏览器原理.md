### 浏览器
* 浏览器是多进程的
### 一. 浏览器内核
1. Blink内核
  * **代表浏览器**： 谷歌，Edge，Opera
  * **特点**: 由Google开发，基于WebKit分支而来，注重性能和现代网页支持，采用多进程架构提升稳定性和安全性
2. WebKit内核
  * **代表浏览器**： Apple Safar
  * **特点**‌：由苹果公司开发，开源且轻量化，在移动设备上表现优异，深度优化于Apple生态系统
3. Gecko内核
  * **代表浏览器**： Mozilla Firefox(火狐)
  * **特点**‌：由Mozilla开发，开源且跨平台，强调对W3C标准的兼容性和强大的扩展支持
4. Trident内核
  * **代表浏览器**‌：Internet Explorer（已逐步被Edge替代）‌
  * **特点**‌：由微软开发，曾因市场份额高但更新滞后导致兼容性问题，现已不再积极维护‌
### 二. 页面渲染流程
从输入URL到页面展示，主要分为以下步骤‌
1. **导航**‌：用户输入URL后，浏览器发起DNS查询获取服务器IP，建立TCP连接
2. **资源加载**‌：通过HTTP/HTTPS协议请求HTML、CSS、JS等资源
3. **解析与渲染**‌：
   * 解析HTML生成DOM树；解析CSS生成CSSOM树；合并为渲染树
   * 计算布局`（Layout）`和绘制`（Paint）`，最终由`GPU`合成显示
### 三. 核心渲染流程
1. **解析HTML生成DOM树**:浏览器将HTML字符串解析为文档对象模型（DOM树），表示页面的结构‌
2. **解析CSS生成CSSOM树**‌：同时解析CSS样式，构建CSS对象模型（CSSOM树），表示页面的样式规则‌
3. **构建渲染树**‌：将DOM树和CSSOM树合并，生成渲染树（Render Tree），只包含可见节点及其计算后的样式‌
4. **‌布局计算**‌：计算渲染树中每个节点的几何信息（如位置、大小），确定页面布局‌
5. **‌绘制**‌：将布局结果转换为屏幕上的像素信息，完成页面绘制‌
### 四. 资源间的相互影响
   * **CSS与HTML**‌: CSS解析会**阻塞**DOM树的构建,因为渲染树需要cssom来计算样式，浏览器会通过预加载扫描器提前请求CSS资源以减少阻塞‌

   * **javaScript与HTML/CSS**： javascript脚本会**阻塞**HTML解析，直到脚本执行完成。同时，js脚本可能操作dom或修改样式触发`重排`和`重绘`，影响性能

   * **图片等资源**: 非阻塞资源（如图片）的加载不会阻塞HTML解析，但可能影响页面布局和绘制时机‌
### 五.详细阶段解析
1. **DOM树构建**
* HTML解析器将HTML字符串转换为DOM节点，形成树状结构‌.此阶段会处理内联样式和外部CSS的预加载‌
2. **CSSOM树构建**
* CSS解析器将CSS规则转换为样式映射，构建CSSOM树‌.样式会进行标准化处理（如`color: blue`转换为`rgb(0,0,255)`）
3. **渲染树构建**
* 渲染树结合**DOM树和CSSOM树**，**仅包含可见节点**（如`display: none`的元素不包含在内）‌.此阶段会应用计算后的样式到对应元素‌
4. **布局计算**
* 浏览器遍历渲染树，计算每个节点的位置、大小等几何信息，确定页面布局‌。此过程可能因DOM或CSSOM的修改而**重复执行‌**
5. **绘制与优化**
* 布局完成后，浏览器将渲染树转换为像素信息，通过分层、光栅化等步骤绘制到屏幕‌。优化手段包括减少重排、使用硬件加速等‌
### 性能优化建议
* **减少重排**‌：避免频繁修改DOM或样式，使用transform代替top/left等布局属性‌
* **异步加载资源**‌：将非关键CSS/JS标记为async或defer，减少阻塞‌
* **预加载关键资源**‌：利用<link rel="preload">提前加载重要资源‌
