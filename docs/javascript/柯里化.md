# 柯里化（Currying）

> 柯里化是一种将**多参数函数**转化为**一系列单参数函数**的技术。简单来说，就是把一个 `f(a, b, c)` 的调用方式，变成 `f(a)(b)(c)` 的调用方式。

## 核心概念

柯里化（Currying）得名于数学家 **Haskell Curry**。它是一种**函数式编程**的核心技术，核心思想是：

| 概念 | 说明 |
|------|------|
| **定义** | 将一个 N 元函数转换为 N 个一元函数的嵌套调用 |
| **本质** | 利用**闭包**保存已传入的参数，逐步收集直到参数齐备再执行 |
| **结果** | 每次调用只接收一个参数，返回一个新函数等待接收剩余参数 |

```
普通调用：  add(1, 2, 3)       → 6
柯里化调用：add(1)(2)(3)       → 6
也可以：    add(1)(2, 3)       → 6（取决于实现方式）
```

## 直观理解

### 生活类比

想象你去餐厅点餐：

1. **普通方式**：一次性告诉服务员「我要一份宫保鸡丁、不辣、加蛋」
2. **柯里化方式**：
   - 第一步：选择菜品 → `order('宫保鸡丁')`
   - 第二步：选择辣度 → `order('宫保鸡丁')('不辣')`
   - 第三步：选择加料 → `order('宫保鸡丁')('不辣')('加蛋')`

每一步都**记住**了之前的选择，直到信息完整才出餐。

### 代码对比

```javascript
// 普通函数
function add(a, b, c) {
  return a + b + c
}

add(1, 2, 3) // 6

// 柯里化后的函数
function curriedAdd(a) {
  return function (b) {
    return function (c) {
      return a + b + c
    }
  }
}

curriedAdd(1)(2)(3) // 6
```

## 基础实现

### 手写柯里化函数

核心思路：通过递归收集参数，当收集到的参数数量**达到或超过**原函数的参数个数时，执行原函数。

```javascript
function curry(fn) {
  return function curried(...args) {
    // 参数数量足够，直接执行原函数
    if (args.length >= fn.length) {
      return fn.apply(this, args)
    }
    // 参数不够，返回一个新函数继续收集
    return function (...nextArgs) {
      return curried.apply(this, args.concat(nextArgs))
    }
  }
}
```

### 使用示例

```javascript
// 原始函数
function multiply(a, b, c) {
  return a * b * c
}

// 柯里化
const curriedMultiply = curry(multiply)

// 多种调用方式全部支持
curriedMultiply(2)(3)(4)     // 24 — 标准柯里化调用
curriedMultiply(2, 3)(4)     // 24 — 前两个一起传
curriedMultiply(2)(3, 4)     // 24 — 后两个一起传
curriedMultiply(2, 3, 4)     // 24 — 一次性传完，等价于原函数调用
```

### 关键点解析

```
fn.length → 原函数的形参个数（不包含 rest 参数、默认值参数之后的参数）
args      → 当前已收集的参数数组
递归条件  → args.length < fn.length 时继续返回新函数
终止条件  → args.length >= fn.length 时执行原函数
```

## 进阶实现

### 支持 placeholder（占位符）

在实际开发中，有时我们想先传后面的参数，把前面的参数留到后面再填。这时需要**占位符**支持：

```javascript
// 定义一个占位符
curry._ = Symbol('placeholder')

function curry(fn) {
  return function curried(...args) {
    // 参数足够且没有占位符，直接执行
    if (args.length >= fn.length && !args.includes(curry._)) {
      return fn.apply(this, args)
    }

    return function (...nextArgs) {
      // 合并参数：用新参数填充占位符位置
      const merged = args.map(arg =>
        arg === curry._ && nextArgs.length ? nextArgs.shift() : arg
      )
      return curried.apply(this, [...merged, ...nextArgs])
    }
  }
}
```

#### 使用示例

```javascript
function log(level, time, message) {
  return `[${level}] ${time} - ${message}`
}

const curriedLog = curry(log)

// 使用占位符，先传 time，再传 level
curriedLog(curry._)('2024-01-01')('System started')('INFO')
// "[INFO] 2024-01-01 - System started"

// 先固定 message 模板
curriedLog('ERROR', curry._)('Connection failed')
// 后续填充 time
```

### 支持箭头函数与 `this` 绑定

```javascript
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.call(this, ...args)
    }
    return (...nextArgs) => curried.apply(this, args.concat(nextArgs))
  }
}
```

> **注意**：箭头函数没有自己的 `this`，如果需要保持 `this` 上下文，内部返回的函数应使用 `function` 声明或通过 `bind` 绑定。

### TypeScript 类型安全实现

```typescript
// 工具类型：将函数参数逐个拆分为柯里化形式
type Curried<T> = T extends (...args: infer A) => infer R
  ? A extends [infer First, ...infer Rest]
    ? (arg: First) => Curried<(...args: Rest) => R>
    : R
  : never

function curry<T extends (...args: any[]) => any>(fn: T): Curried<T> {
  return function curried(this: any, ...args: any[]): any {
    if (args.length >= fn.length) {
      return fn.call(this, ...args)
    }
    return (...nextArgs: any[]) => curried.call(this, ...args, ...nextArgs)
  } as Curried<T>
}

// 使用 — 完整的类型推导
const add = (a: number, b: number, c: number) => a + b + c
const curriedAdd = curry(add)

curriedAdd(1)(2)(3)       // ✅ 类型推导结果为 number
curriedAdd(1)('2')(3)     // ❌ 类型错误：'2' 不是 number
```

## 柯里化 vs 偏函数应用

两者经常被混淆，但它们是**不同的概念**：

| 维度 | 柯里化（Currying） | 偏函数应用（Partial Application） |
|------|-------------------|-------------------------------|
| **目标** | 将 N 元函数转为 N 个一元函数 | 固定函数的**部分参数**，返回一个新函数 |
| **调用形式** | `f(a)(b)(c)` | `f(a, _, c)` 或 `partial(f, a)` |
| **调用次数** | 每次只传一个参数 | 可以一次固定多个参数 |
| **返回** | 始终返回函数，直到参数齐备 | 返回一个参数更少的新函数 |

```javascript
// 柯里化
const curriedAdd = curry((a, b, c) => a + b + c)
curriedAdd(1)(2)(3) // 6 — 每次传一个

// 偏函数应用
function partial(fn, ...presetArgs) {
  return function (...laterArgs) {
    return fn.apply(this, [...presetArgs, ...laterArgs])
  }
}

const add = (a, b, c) => a + b + c
const addOne = partial(add, 1)    // 固定第一个参数
addOne(2, 3)                      // 6 — 一次传剩余的
```

> **一句话区分**：柯里化是**拆分**参数（每个函数只接收一个），偏函数是**预填**参数（提前固定某些参数值）。

## 实际应用场景

### 1. 参数复用 / 创建专用函数

```javascript
// 通用请求函数
function request(method, url, data) {
  return fetch(url, {
    method,
    headers: { 'Content-Type': 'application/json' },
    body: data ? JSON.stringify(data) : undefined
  })
}

const curriedRequest = curry(request)

// 复用 method，创建专用的请求方法
const get = curriedRequest('GET')
const post = curriedRequest('POST')

// 再复用 url，创建专用的 API 调用
const getUser = get('/api/user')
const getPost = get('/api/post')

getUser()                // GET /api/user
getPost()                // GET /api/post
post('/api/user')(data)  // POST /api/user
```

### 2. 延迟执行

```javascript
// 创建一个日志函数，先注册处理器，再触发
const curriedLog = curry((level, time, message) => {
  console.log(`[${level}] ${time}: ${message}`)
})

// 预设日志级别
const infoLog = curriedLog('INFO')
const errorLog = curriedLog('ERROR')

// 在特定时刻使用
function processTask(task) {
  infoLog(new Date().toISOString())(`开始处理任务: ${task.name}`)
  try {
    task.run()
    infoLog(new Date().toISOString())(`任务完成: ${task.name}`)
  } catch (err) {
    errorLog(new Date().toISOString())(`任务失败: ${err.message}`)
  }
}
```

### 3. 函数组合（Composition）

柯里化是函数组合的天然搭档，让函数可以像**管道**一样串联：

```javascript
// 组合工具函数
const compose = (...fns) =>
  (x) => fns.reduceRight((acc, fn) => fn(acc), x)

const pipe = (...fns) =>
  (x) => fns.reduce((acc, fn) => fn(acc), x)

// 柯里化的数据处理函数
const filter = curry((predicate, arr) => arr.filter(predicate))
const map = curry((fn, arr) => arr.map(fn))
const reduce = curry((fn, init, arr) => arr.reduce(fn, init))
const sort = curry((compareFn, arr) => [...arr].sort(compareFn))

// 组合使用 — 数据处理管道
const getAdultNames = pipe(
  filter(user => user.age >= 18),          // 过滤成年人
  sort((a, b) => a.age - b.age),           // 按年龄排序
  map(user => user.name),                  // 提取名字
  map(name => name.toUpperCase())          // 转大写
)

const users = [
  { name: 'Alice', age: 25 },
  { name: 'Bob', age: 16 },
  { name: 'Charlie', age: 30 },
  { name: 'Diana', age: 20 }
]

getAdultNames(users)
// ['ALICE', 'DIANA', 'CHARLIE']
```

### 4. 事件处理中的参数预设

```javascript
// 柯里化事件处理器
const handleEvent = curry((action, event) => {
  event.preventDefault()
  console.log(`执行动作: ${action}`, event.target)
})

// 在模板中使用
// <button @click="handleEvent('submit')">
// <button @click="handleEvent('cancel')">
```

### 5. 高阶函数简化

```javascript
// 不使用柯里化
const numbers = [1, 2, 3, 4, 5]
const doubled = numbers.map(n => multiply(2, n))

// 使用柯里化 — 更简洁
const curriedMultiply = curry((a, b) => a * b)
const double = curriedMultiply(2) // 固定乘数为 2

const doubled = [1, 2, 3, 4, 5].map(double)
// [2, 4, 6, 8, 10]
```

## 性能考量

### 潜在开销

| 方面 | 影响 |
|------|------|
| **内存** | 每次柯里化调用都创建新的闭包和函数对象，增加内存占用 |
| **调用栈** | 多层嵌套函数调用，调用栈更深 |
| **执行速度** | 相比直接调用，多了一层参数收集的中间层 |

### 优化建议

```javascript
// ❌ 不推荐：在热路径中过度柯里化
function sum(arr) {
  return arr.reduce((acc, n) => acc + n, 0)
}
// 如果只是简单相加，直接用普通函数更高效

// ✅ 推荐：在配置/复用场景使用柯里化
const createFormatter = curry((locale, options, value) =>
  new Intl.NumberFormat(locale, options).format(value)
)

// 创建一次，多处使用
const formatCNY = createFormatter('zh-CN', { style: 'currency', currency: 'CNY' })
const formatUSD = createFormatter('en-US', { style: 'currency', currency: 'USD' })

formatCNY(1234.5)  // "¥1,234.50"
formatUSD(1234.5)  // "$1,234.50"
```

> **原则**：在**高频调用**的核心计算路径中慎用柯里化；在**配置生成**、**函数复用**、**API 封装**等场景中大胆使用。

## 工具库支持

### Lodash

```javascript
import { curry } from 'lodash'

const add = curry((a, b, c) => a + b + c)

add(1)(2)(3)    // 6
add(1, 2)(3)    // 6
add(1)(2, 3)    // 6
```

Lodash 的 `_.curry` 还支持**占位符**：

```javascript
import { curry, _ } from 'lodash'

const log = curry((level, time, msg) => `[${level}] ${time}: ${msg}`)

// 使用占位符跳过某些参数
log(_, '12:00', 'hello')('INFO')
// "[INFO] 12:00: hello"
```

### Ramda

Ramda 的所有函数**默认都是柯里化**的：

```javascript
import R from 'ramda'

// 不需要手动 curry，Ramda 函数天然支持
R.map(n => n * 2, [1, 2, 3])     // [2, 4, 6]
R.map(n => n * 2)([1, 2, 3])     // [2, 4, 6] — 柯里化调用

const doubleAll = R.map(n => n * 2)
doubleAll([1, 2, 3])              // [2, 4, 6]
```

## 总结

| 特性 | 说明 |
|------|------|
| **是什么** | 将多参数函数转化为单参数函数链的技术 |
| **核心原理** | 利用闭包逐步收集参数 |
| **主要价值** | 参数复用、延迟执行、函数组合 |
| **适用场景** | 函数封装、配置生成、数据处理管道 |
| **注意事项** | 高频热路径慎用，注意性能与可读性的平衡 |

### 记忆口诀

> **柯里化 = 拆散参数 + 闭包记忆 + 逐步调用**
