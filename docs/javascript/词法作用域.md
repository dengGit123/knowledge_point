## 什么是作用域

**作用域**（Scope）就是变量和函数的可访问范围，它决定了代码中变量和函数的可见性。

### 作用域的三个层次

```javascript
// ═══════════════════════════════════════
// 1. 全局作用域（Global Scope）
// ═══════════════════════════════════════
var globalVar = "我是全局变量";

function outerFunction() {
    // ═══════════════════════════════════════
    // 2. 函数作用域（Function Scope）
    // ═══════════════════════════════════════
    var outerVar = "我是外层函数变量";

    function innerFunction() {
        // ═══════════════════════════════════════
        // 3. 内层函数作用域
        // ═══════════════════════════════════════
        var innerVar = "我是内层函数变量";

        // 这里可以访问所有三个层次的变量
        console.log(innerVar);  // ✓ 自己的
        console.log(outerVar);  // ✓ 外层的
        console.log(globalVar); // ✓ 全局的
    }

    innerFunction();

    // 这里只能访问外层和全局的
    console.log(outerVar);   // ✓ 自己的
    console.log(globalVar);  // ✓ 全局的
    // console.log(innerVar); // ✗ 报错！
}

outerFunction();

// 这里只能访问全局的
console.log(globalVar);     // ✓ 全局的
// console.log(outerVar);   // ✗ 报错！
```

### ES6 新增：块级作用域

```javascript
// var 没有块级作用域
if (true) {
    var a = 1;
}
console.log(a); // ✓ 1（函数作用域）

// let/const 有块级作用域
if (true) {
    let b = 2;
    const c = 3;
}
console.log(b); // ✗ ReferenceError
console.log(c); // ✗ ReferenceError
```

## 什么是词法作用域

**词法作用域**（Lexical Scope），也叫**静态作用域**（Static Scope），是绝大多数现代编程语言采用的作用域机制。

### 核心定义

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  词法作用域 = 变量的作用域在【代码编写时】确定           │
│                                                         │
│  决定因素 = 函数【定义的位置】，而非【调用的位置】       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 通俗比喻

```
想象一下：

【词法作用域】就像"你的出生地"
- 你出生在北京，你的籍贯就是北京
- 无论你后来去哪里工作、生活
- 你的籍贯（能享受的权利）由出生地决定

【动态作用域】就像"你的当前所在地"
- 你现在在哪里，就按哪里的规则办事
- 每次调用函数都可能产生不同的结果

JavaScript 采用词法作用域，所以函数记住的是它的"出生地"
```

### 经典示例

```javascript
// ===== 示例1：理解词法作用域 =====
var name = "张三";

function sayHello() {
    console.log("你好，我是 " + name);
}

function introduceFriend() {
    var name = "李四";  // 局部变量
    sayHello();         // 调用 sayHello
}

introduceFriend(); // 输出："你好，我是 张三"
                   // 而不是"李四"！

// 原因：sayHello 定义在全局作用域
//      它访问的永远是全局的 name
//      虽然在 introduceFriend 中调用
//      但这不改变它的作用域

// ===== 示例2：嵌套函数的作用域 =====
function outer() {
    var x = 10;

    function inner() {
        // inner 函数定义在 outer 内部
        // 所以它可以访问 outer 的变量
        console.log(x); // 10
    }

    inner();
}

outer();
```

## 作用域链

**作用域链**（Scope Chain）是 JavaScript 引擎查找变量的机制。

### 查找机制图解

```
查找变量时，从内向外逐层查找：

┌────────────────────────────────────────────────────┐
│  当代码访问一个变量时：                             │
│                                                    │
│  1. 当前作用域 → 找到？✓ 使用                      │
│                   ↓ 找不到                         │
│  2. 外层作用域  → 找到？✓ 使用                      │
│                   ↓ 找不到                         │
│  3. 再外层     → 找到？✓ 使用                      │
│                   ↓ 找不到                         │
│  4. 全局作用域 → 找到？✓ 使用                      │
│                   ↓ 找不到                         │
│  5. 抛出 ReferenceError                            │
└────────────────────────────────────────────────────┘
```

### 代码示例

```javascript
var globalColor = "红色";

function level1() {
    var level1Color = "蓝色";

    function level2() {
        var level2Color = "绿色";

        function level3() {
            // 查找 color 变量的过程：
            var level3Color = "黄色";

            // 1. level3 作用域没找到 globalColor
            // 2. level2 作用域没找到 globalColor
            // 3. level1 作用域没找到 globalColor
            // 4. 全局作用域找到了！
            console.log(globalColor);   // "红色"

            // 1. level3 作用域没找到 level1Color
            // 2. level2 作用域没找到 level1Color
            // 3. level1 作用域找到了！
            console.log(level1Color);   // "蓝色"

            // 1. level3 作用域找到了！
            console.log(level3Color);   // "黄色"
        }

        level3();
    }

    level2();
}

level1();
```

### 作用域链的可视化

```javascript
// 代码
var a = 1;
function outer() {
    var b = 2;
    function inner() {
        var c = 3;
        console.log(a + b + c); // 6
    }
    inner();
}
outer();

// 对应的作用域链结构：
/*
inner 的作用域链:
┌─────────────────────┐
│ c = 3               │ ← 当前作用域
├─────────────────────┤
│ b = 2               │ ← outer 的作用域
├─────────────────────┤
│ a = 1, outer, inner │ ← 全局作用域
└─────────────────────┘
*/
```

## 词法作用域 vs 动态作用域

### 对比表格

| 对比项 | 词法作用域 | 动态作用域 |
|:------:|:---------:|:---------:|
| **确定时机** | 代码编写时（静态） | 代码运行时（动态） |
| **决定因素** | 函数定义的位置 | 函数调用的位置 |
| **查找方向** | 向外（词法环境） | 向上（调用栈） |
| **可预测性** | 高，读代码就能确定 | 低，依赖调用链 |
| **代表语言** | JavaScript、Python、Java | Bash、Perl、部分 Lisp |
| **调试难度** | 容易 | 较难 |

### 代码对比示例

```javascript
// 同一段代码，在不同作用域机制下的表现

var x = 10;

function foo() {
    console.log(x);
}

function bar() {
    var x = 20;
    foo();
}

bar();

// ====== 词法作用域（JavaScript） ======
// foo 定义在全局，访问全局的 x
// 输出：10

// ====== 动态作用域（假设） ======
// foo 在 bar 中被调用，访问 bar 的 x
// 输出：20
```

### 为什么 JavaScript 选择词法作用域？

```javascript
// 词法作用域的优势：可预测性

// 场景：写一个工具函数
function createLogger(prefix) {
    return function(message) {
        console.log(prefix + ": " + message);
    };
}

var infoLogger = createLogger("[INFO]");
var errorLogger = createLogger("[ERROR]");

// 词法作用域下：
// 你能清楚地知道每个 logger 的 prefix 是什么
infoLogger("开始处理");    // "[INFO]: 开始处理"
errorLogger("发生错误");   // "[ERROR]: 发生错误"

// 如果是动态作用域：
// prefix 的值会依赖调用时的上下文
// 你无法确定输出的 prefix 是什么
// 这会让代码难以理解和维护
```

## 与执行上下文的关系

### 执行上下文 vs 作用域

```javascript
// 执行上下文（Execution Context）：
// - 函数被调用时创建
// - 包含：变量对象、作用域链、this
// - 运行时概念，存在于调用栈

// 作用域（Scope）：
// - 函数定义时确定（词法作用域）
// - 静态的代码结构
// - 编写时就能确定
```

### 关系图解

```
┌─────────────────────────────────────────────────────┐
│                    执行上下文                        │
│  ┌───────────────────────────────────────────────┐  │
│  │  Variable Object (变量对象/活动对象)          │  │
│  │  - 函数的参数                                  │  │
│  │  - 局部变量                                    │  │
│  └───────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────┐  │
│  │  Scope Chain (作用域链)                       │  │
│  │  - 由词法作用域决定（静态）                    │  │
│  │  - 在函数定义时就确定了                        │  │
│  └───────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────┐  │
│  │  this                                         │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘

关键点：作用域链在函数定义时就确定了（词法作用域）
       但执行上下文在函数调用时才创建
```

### 代码示例

```javascript
var name = "全局";

function Person() {
    var name = "Person";

    return {
        getName: function() {
            // 这里的作用域链在函数定义时就确定了
            // 它永远访问 Person 的 name
            return name;
        }
    };
}

var person1 = Person();
console.log(person1.getName()); // "Person"

var name = "又改了全局";
console.log(person1.getName()); // 仍然是 "Person"
// 作用域链不会因为全局变量改变而改变
```

## 变量遮蔽

### 什么是变量遮蔽

**变量遮蔽**（Variable Shadowing）：内层作用域声明了与外层作用域同名的变量，内层变量会"遮蔽"外层变量。

```javascript
var x = "全局的 x";

function outer() {
    var x = "outer 的 x";  // 遮蔽全局的 x

    function inner() {
        var x = "inner 的 x";  // 遮蔽 outer 的 x

        console.log(x);  // "inner 的 x"
    }

    inner();
    console.log(x);  // "outer 的 x"
}

outer();
console.log(x);  // "全局的 x"
```

### 仍然可以访问全局变量

```javascript
var x = 10;

function foo() {
    var x = 20;  // 遮蔽全局 x

    // 仍然可以通过 window 对象访问全局变量（浏览器环境）
    console.log(x);        // 20（局部）
    console.log(window.x); // 10（全局）
}

foo();
```

### 块级作用域中的遮蔽

```javascript
var x = 1;

function example() {
    var x = 2;

    if (true) {
        let x = 3;  // 遮蔽函数作用域的 x
        const x = 4; // ❌ 语法错误！同一作用域不能重复声明

        console.log(x); // 3
    }

    console.log(x); // 2
}

example();
console.log(x); // 1
```

### 遮蔽的最佳实践

```javascript
// ❌ 不推荐：容易混淆
function process(data) {
    var data = JSON.parse(data);  // 遮蔽参数
    // ...
}

// ✓ 推荐：使用不同的名字
function process(rawData) {
    var data = JSON.parse(rawData);
    // ...
}

// ✓ 推荐：必要时可以遮蔽（如计算中间值）
function calculate(arr) {
    var sum = arr.reduce((a, b) => a + b, 0);

    if (sum > 100) {
        var sum = sum * 1.1;  // 遮蔽，表示调整后的值
        return sum;
    }
    return sum;
}
```

## 实际应用场景

### 1. 闭包（Closure）

闭包是词法作用域最重要的应用之一。

```javascript
// 计数器
function createCounter() {
    var count = 0;  // 私有变量

    return {
        increment: function() {
            count++;
            console.log(count);
        },
        decrement: function() {
            count--;
            console.log(count);
        },
        getCount: function() {
            return count;
        }
    };
}

var counter = createCounter();
counter.increment(); // 1
counter.increment(); // 2
counter.increment(); // 3
counter.decrement(); // 2
// console.log(count); // ReferenceError，无法直接访问
```

### 2. 模块模式（Module Pattern）

```javascript
// 创建私有变量和公共方法
var ShoppingCart = (function() {
    // 私有变量
    var items = [];
    var total = 0;

    // 私有方法
    function calculateTotal() {
        return items.reduce(function(sum, item) {
            return sum + item.price;
        }, 0);
    }

    // 公共接口
    return {
        addItem: function(item) {
            items.push(item);
            total = calculateTotal();
        },
        removeItem: function(index) {
            items.splice(index, 1);
            total = calculateTotal();
        },
        getTotal: function() {
            return total;
        },
        getItemCount: function() {
            return items.length;
        }
    };
})();

// 使用
ShoppingCart.addItem({ name: "苹果", price: 5 });
ShoppingCart.addItem({ name: "香蕉", price: 3 });
console.log(ShoppingCart.getTotal()); // 8
// console.log(ShoppingCart.items);   // undefined，私有
```

### 3. 函数工厂（Factory Functions）

```javascript
// 创建特定功能的函数
function createMultiplier(multiplier) {
    // multiplier 被闭包捕获
    return function(number) {
        return number * multiplier;
    };
}

var double = createMultiplier(2);
var triple = createMultiplier(3);
var quadruple = createMultiplier(4);

console.log(double(5));     // 10
console.log(triple(5));     // 15
console.log(quadruple(5));  // 20
```

### 4. 防抖与节流

```javascript
// 防抖（Debounce）
function debounce(fn, delay) {
    var timer = null;

    return function() {
        var context = this;
        var args = arguments;

        clearTimeout(timer);
        timer = setTimeout(function() {
            fn.apply(context, args);
        }, delay);
    };
}

// 节流（Throttle）
function throttle(fn, delay) {
    var lastTime = 0;

    return function() {
        var now = Date.now();
        var context = this;
        var args = arguments;

        if (now - lastTime >= delay) {
            lastTime = now;
            fn.apply(context, args);
        }
    };
}

// 使用
var handleSearch = debounce(function(keyword) {
    console.log("搜索：" + keyword);
}, 500);

// 快速输入只会执行最后一次
handleSearch("J");
handleSearch("Ja");
handleSearch("Jav");
handleSearch("Java"); // 只有这会触发搜索
```

### 5. 配置管理

```javascript
function createConfig(defaultConfig) {
    var config = Object.assign({}, defaultConfig);

    return {
        get: function(key) {
            return config[key];
        },
        set: function(key, value) {
            config[key] = value;
        },
        getAll: function() {
            return Object.assign({}, config);
        },
        reset: function() {
            config = Object.assign({}, defaultConfig);
        }
    };
}

var appConfig = createConfig({
    theme: "light",
    language: "zh-CN",
    pageSize: 10
});

appConfig.set("theme", "dark");
console.log(appConfig.get("theme")); // "dark"
appConfig.reset();
console.log(appConfig.get("theme")); // "light"
```

### 6. 柯里化（Currying）

```javascript
function curry(fn) {
    return function curried(...args) {
        if (args.length >= fn.length) {
            return fn.apply(this, args);
        } else {
            return function(...moreArgs) {
                return curried.apply(this, args.concat(moreArgs));
            };
        }
    };
}

// 使用
function add(a, b, c) {
    return a + b + c;
}

var curriedAdd = curry(add);
console.log(curriedAdd(1)(2)(3)); // 6
console.log(curriedAdd(1, 2)(3)); // 6
console.log(curriedAdd(1)(2, 3)); // 6
```

## 常见误区与陷阱

### 误区 1：认为函数调用位置决定作用域

```javascript
// ❌ 错误理解
var x = 10;

function foo() {
    console.log(x);
}

function bar() {
    var x = 20;
    foo();  // 期望输出 20
}

bar(); // 实际输出 10

// ✓ 正确理解
// foo 定义在全局作用域，永远访问全局的 x
// 与在哪里调用无关
```

### 误区 2：混淆 var 和 let 的作用域

```javascript
// var：函数作用域
function testVar() {
    if (true) {
        var x = 10;
    }
    console.log(x); // ✓ 10
}

// let：块级作用域
function testLet() {
    if (true) {
        let y = 10;
    }
    console.log(y); // ✗ ReferenceError
}
```

### 误区 3：循环中的闭包问题

```javascript
// ❌ 常见错误
for (var i = 0; i < 3; i++) {
    setTimeout(function() {
        console.log(i);  // 都输出 3
    }, 100);
}

// 问题：var 是函数作用域，三个函数共享同一个 i
// 解决方案 1：使用 let（ES6）
for (let i = 0; i < 3; i++) {
    setTimeout(function() {
        console.log(i);  // 0, 1, 2
    }, 100);
}

// 解决方案 2：立即执行函数
for (var i = 0; i < 3; i++) {
    (function(j) {
        setTimeout(function() {
            console.log(j);  // 0, 1, 2
        }, 100);
    })(i);
}
```

### 误区 4：变量提升

```javascript
// 变量提升示例
console.log(x);  // undefined，不是报错
var x = 10;

// 等价于：
var x;
console.log(x);
x = 10;

// 函数提升
console.log(foo());  // "函数"
function foo() {
    return "函数";
}

// 函数表达式不会提升
console.log(bar());  // TypeError
var bar = function() {
    return "函数表达式";
};
```

### 误区 5：this 与作用域

```javascript
// this 与作用域是两个不同的概念！

var obj = {
    name: "张三",
    greet: function() {
        console.log("你好，我是 " + this.name);

        setTimeout(function() {
            // 这里的 this 不指向 obj！
            console.log("迟到的问候：" + this.name);  // undefined
        }, 100);
    }
};

obj.greet();

// 解决方案 1：保存 this
var obj = {
    name: "张三",
    greet: function() {
        var self = this;
        setTimeout(function() {
            console.log("迟到的问候：" + self.name);
        }, 100);
    }
};

// 解决方案 2：使用箭头函数（ES6）
var obj = {
    name: "张三",
    greet: function() {
        setTimeout(() => {
            // 箭头函数继承外层的 this
            console.log("迟到的问候：" + this.name);
        }, 100);
    }
};
```

## 面试经典题

### 题 1：输出顺序

```javascript
for (var i = 0; i < 3; i++) {
    setTimeout(function() {
        console.log(i);
    }, 0);
}
console.log("结束");

// 输出：
// "结束"
// 3
// 3
// 3

// 解析：
// 1. for 循环同步执行，立即完成
// 2. setTimeout 是异步，放入任务队列
// 3. console.log 先执行
// 4. 循环结束时 i = 3
// 5. 三个 setTimeout 回调都访问同一个 i，输出 3
```

### 题 2：作用域链查找

```javascript
var x = 10;

function foo() {
    var y = 20;

    function bar() {
        var z = 30;
        console.log(x + y + z);
    }

    bar();
}

foo();
console.log(x);

// 输出：
// 60
// 10

// 解析：
// - bar 可以访问：z (自己的)、y (foo 的)、x (全局的)
// - foo 可以访问：y (自己的)、x (全局的)
// - 全局只能访问：x (自己的)
```

### 题 3：闭包计数器

```javascript
function createCounter() {
    var count = 0;
    return {
        increment: function() { count++; },
        decrement: function() { count--; },
        getCount: function() { return count; }
    };
}

var counter1 = createCounter();
var counter2 = createCounter();

counter1.increment();
counter1.increment();
counter2.increment();

console.log(counter1.getCount()); // ?
console.log(counter2.getCount()); // ?

// 输出：
// 2
// 1

// 解析：每次调用 createCounter 都创建一个新的作用域，两个计数器独立
```

### 题 4：变量遮蔽

```javascript
var a = 1;

function test() {
    var a = 2;
    console.log(a);

    function inner() {
        var a = 3;
        console.log(a);
    }

    inner();
    console.log(a);
}

test();
console.log(a);

// 输出：
// 2
// 3
// 2
// 1

// 解析：每个作用域的 a 独立，内层遮蔽外层
```

## 最佳实践

### 1. 减少全局污染

```javascript
// ❌ 不推荐
var name = "张三";
var age = 25;
var city = "北京";

// ✓ 推荐：使用命名空间或模块
var MyApp = {
    config: {
        name: "张三",
        age: 25,
        city: "北京"
    },
    // ...
};

// ✓ 推荐：使用 ES6 模块
// config.js
export const config = {
    name: "张三",
    age: 25
};
```

### 2. 使用 let 和 const

```javascript
// ✓ 推荐：默认使用 const
const API_URL = "https://api.example.com";
const maxRetries = 3;

// ✓ 推荐：需要重新赋值时使用 let
let currentPage = 1;
currentPage++;

// ❌ 避免：不再使用 var（除非有特殊需求）
var data = null;
```

### 3. 函数保持短小

```javascript
// ✓ 推荐：函数短小，作用域清晰
function processData(data) {
    const cleaned = cleanData(data);
    const validated = validateData(cleaned);
    return transformData(validated);
}

// ❌ 避免：函数过长，作用域复杂
function processEverything(data) {
    // 100+ 行代码
    // 多层嵌套
    // 难以追踪变量来源
}
```

### 4. 明确变量声明位置

```javascript
// ✓ 推荐：在函数顶部统一声明（var）
function example() {
    var data = null;
    var result = null;
    var error = null;

    // 函数体
}

// ✓ 推荐：就近声明（let/const）
function example() {
    const config = loadConfig();

    if (config.isValid) {
        const result = process(config);
        return result;
    }
}
```

### 5. 避免深层嵌套

```javascript
// ❌ 不推荐：深层嵌套
function handle(data) {
    if (data) {
        if (data.items) {
            if (data.items.length > 0) {
                // ...
            }
        }
    }
}

// ✓ 推荐：早返回
function handle(data) {
    if (!data) return;
    if (!data.items) return;
    if (data.items.length === 0) return;

    // ...
}
```

### 6. 使用纯函数

```javascript
// ✓ 推荐：纯函数，不依赖外部状态
function add(a, b) {
    return a + b;
}

// ❌ 避免：依赖外部状态
var base = 10;
function add(value) {
    return base + value;  // base 从哪来？不明确
}
```

## 总结

### 核心要点

```
┌─────────────────────────────────────────────────────┐
│                    词法作用域                        │
│                                                     │
│  1. 定义位置决定作用域，而非调用位置                 │
│  2. 内层可以访问外层，外层不能访问内层               │
│  3. 变量查找沿作用域链向外查找                       │
│  4. 闭包是词法作用域的直接应用                       │
│  5. let/const 提供块级作用域                         │
│  6. var 是函数作用域                                 │
└─────────────────────────────────────────────────────┘
```

### 学习路径

```
1. 理解作用域是什么
   ↓
2. 理解词法作用域 vs 动态作用域
   ↓
3. 掌握作用域链的查找机制
   ↓
4. 理解闭包
   ↓
5. 了解执行上下文与作用域的关系
   ↓
6. 实践：模块模式、防抖节流、函数工厂
   ↓
7. 避免常见陷阱
```

闭包记住了，变量不释放
let const 块级域，var 函数域
