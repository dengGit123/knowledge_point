### 一. 全局作用域
* 除了在函数里，块级代码块之外的变量属于全局作用域，可以在代码的任何位置访问
* 使用`var`定义的全局变量会成为`window`对象的属性，而`let`和`const`声明的则不会

### 二. 函数作用域
* 在函数里声明的变量，只能在函数里访问
* 使用var声明的变量具有函数作用域，即使在代码块里声明，也属于函数作用域

### 三. 块级作用域
#### ES6引入的`let`, `const`
* 在`{}`代码块里使用`let`,`const`声明变量，只能在当前代码块里访问

### 作用域提升(只提升到当前作用域前面)
* `var`声明的变量的作用域会提升到当前的作用域前面
* `function`函数的声明也会作用域提升到当前作用域前面
* `let` ,`const`块级作用域<strong>不存在</strong>作用域提升

### 作用域链机制
#### 访问变量的一种机制
* 在访问变量时会沿着<strong>作用域链从当前作用域向外部逐级查找</strong>，直到<strong>全局作用域</strong>

### 作用域链的基本工作流程
#### 当函数被调用时，JavaScript引擎会创建一个执行上下文，该上下文包含一个作用域链。作用域链本质上是一个变量对象的列表，按特定顺序排列:
1. <strong>当前函数的活动对象(AO)</strong>: 包含局部变量、参数、函数声明;
2. <strong>外部函数的活动对象‌</strong>:包含外层函数的变量;
3. <strong>全局对象(GO)</strong>:包含全局变量和函数;

**函数的作用域查找跟<mark>定义的位置</mark>有关系，跟调用位置没关系**
```js
var msg = 'hello,word'
function foo(){
  console.log(msg)
}
function bar(){
  let msg = 'bar'
  foo()
}
//foo执行的父级作用域 是 var msg = 'hello,word' 的作用域
bar() //打印 hello,word
```
