# 元素拖拽（HTML5 Drag and Drop）

## 一、概述

元素拖拽是指用鼠标把一个元素"拖起来"、移动、再"放"到另一个元素上的交互。HTML5 提供了一套原生的 **Drag and Drop API**，只需给元素加上 `draggable` 属性，再监听几个拖拽事件，就能实现拖拽排序、拖拽上传文件、看板任务移动等功能——**完全不需要第三方库**。

整个拖拽过程可以理解成"寄快递"：被拖的元素是**包裹**，拖拽途中要携带的数据（如元素 ID）是**包裹里的物品**，放置区域是**收件人**，而 `DataTransfer` 对象就是那个**包裹本身**。

> 📖 [MDN 官方文档 — HTML 拖拽 API](https://developer.mozilla.org/zh-CN/docs/Web/API/HTML_Drag_and_Drop_API)
>
> 📖 [MDN — 拖拽操作](https://developer.mozilla.org/zh-CN/docs/Web/API/HTML_Drag_and_Drop_API/Drag_operations)
>
> 📖 [WHATWG — Drag and drop 规范](https://html.spec.whatwg.org/multipage/dnd.html)

---

## 二、核心原理

实现拖拽需要两样东西：

1. **`draggable` 属性**：让一个元素变成"可以被拖动"的状态。
2. **拖拽事件**：在拖拽的不同阶段（开始、移动、进入目标、释放、结束）执行对应逻辑。

```
   拖拽源(source)                          拖拽目标(target)
   ┌──────────────┐                       ┌──────────────┐
   │ draggable=true│   ──拖动途中──▶       │  放置区域     │
   │ dragstart     │   携带数据(DataTransfer)│  dragover    │
   │ drag          │   ───────────────▶    │  dragenter   │
   │ dragend       │                       │  dragleave   │
   └──────────────┘                       │  drop        │
                                          └──────────────┘
```

> 💡 **提示：** 默认就能拖动的元素只有 **图片 `<img>`**、**带 href 的链接 `<a>`** 和**选中的文本**。普通的 `<div>`、`<li>` 必须手动加 `draggable="true"` 才能拖。

## 三、七个拖拽事件详解（按触发顺序）

拖拽一共有 **7 个事件**，分属"源元素"和"目标元素"两组。下面按一次完整拖拽的时间顺序讲解。

### 事件总览表

| 事件 | 触发对象 | 触发时机 | 是否必须 `preventDefault` |
|------|:---:|------|:---:|
| `dragstart` | 源元素 | 开始拖拽（按下并移动） | 否 |
| `drag` | 源元素 | 拖拽过程中**持续**触发（类似 mousemove） | 否 |
| `dragend` | 源元素 | 拖拽结束（无论是否放置成功） | 否 |
| `dragenter` | 目标元素 | 拖入元素**进入**目标边界 | 建议阻止 |
| `dragover` | 目标元素 | 拖入元素在目标上**持续**移动 | ✅ **必须阻止** |
| `dragleave` | 目标元素 | 拖入元素**离开**目标边界 | 否 |
| `drop` | 目标元素 | 在目标上**松开鼠标**释放 | ✅ **必须阻止** |

### 完整流程图

```
按下并拖动 source
  │
  ├─▶ ① dragstart    （source：开始，存入数据）
  ├─▶ ② drag          （source：持续触发，可用于状态提示）
  │
  │  进入 target
  ├─▶ ③ dragenter    （target：进入，可加高亮样式）
  ├─▶ ④ dragover     （target：持续，必须 preventDefault！）
  │
  │  ┌─ 离开 target ─▶ ⑤ dragleave（target：移除高亮）
  │  │
  │  └─ 在 target 上松手 ─▶ ⑥ drop（target：处理数据，必须 preventDefault！）
  │
  └─▶ ⑦ dragend      （source：拖拽彻底结束，收尾清理）
```

### 1. dragstart（源元素：开始拖拽）

拖拽开始时触发。**最重要的时机：在这里把要传递的数据存进 `DataTransfer`**。

```js
source.addEventListener('dragstart', (e) => {
  // 存入数据，供 drop 时读取（格式 + 内容）
  e.dataTransfer.setData('text/plain', source.id)
  // 可选：设置允许的放置效果
  e.dataTransfer.effectAllowed = 'move'
})
```

### 2. drag（源元素：拖拽中持续触发）

只要还在拖动，就会**持续不断地**触发（频率类似 `mousemove`）。一般用得少，偶尔用于实时更新位置提示。

```js
source.addEventListener('drag', (e) => {
  // 可用于显示"正在拖拽..."的状态（但通常不必监听）
})
```

### 3. dragend（源元素：拖拽结束）

无论拖拽是成功放下（drop）还是被取消（拖到无效区域松手），**都会触发** `dragend`。适合做收尾清理，比如移除高亮、恢复样式。

```js
source.addEventListener('dragend', (e) => {
  source.classList.remove('dragging')   // 移除拖拽中的样式
})
```

### 4. dragenter（目标元素：进入边界）

被拖元素**刚进入**目标区域时触发一次（类似 `mouseenter`）。常用于给目标加"准备接收"的高亮样式。

```js
target.addEventListener('dragenter', (e) => {
  e.preventDefault()                    // 建议：和 dragover 一起阻止
  target.classList.add('drag-over')     // 高亮目标
})
```

### 5. dragover（目标元素：⚠️ 最关键，必须阻止默认行为）

被拖元素**悬停在目标上方时持续触发**。

> ⚠️ **注意（核心考点 / 最大坑）：** 默认情况下，DOM 元素**不允许被放置**其他元素。**必须**在 `dragover` 中调用 `e.preventDefault()`，目标才会被识别为合法的放置区域。**否则 `drop` 事件根本不会触发**，松手时光标会显示"禁止"图标。

```js
// ❌ 错误：没阻止默认行为，drop 永远不触发
target.addEventListener('dragover', (e) => {
  // 什么都没做
})

// ✅ 正确：阻止默认行为，声明"我可以接收"
target.addEventListener('dragover', (e) => {
  e.preventDefault()                    // ← 关键！允许 drop
  e.dataTransfer.dropEffect = 'move'    // 可选：设置光标效果
})
```

### 6. dragleave（目标元素：离开边界）

被拖元素**离开**目标区域时触发。通常用于移除高亮样式。

```js
target.addEventListener('dragleave', (e) => {
  target.classList.remove('drag-over')
})
```

> ⚠️ **注意：** 当目标区域**内部有子元素**时，拖拽指针移到子元素上，会先触发父元素的 `dragleave`（再触发子元素的 `dragenter`）。这会导致高亮闪烁。解决方案见 [第八节](#1-dragleave-在子元素间穿梭时的闪烁问题)。

### 7. drop（目标元素：释放放置）

在目标上**松开鼠标**时触发。**也必须调用 `e.preventDefault()`**，否则浏览器会执行默认动作（比如拖链接会跳转、拖文件会用浏览器打开文件）。

```js
target.addEventListener('drop', (e) => {
  e.preventDefault()                              // ← 必须阻止！否则触发默认行为
  target.classList.remove('drag-over')

  // 取出 dragstart 时存入的数据
  const id = e.dataTransfer.getData('text/plain')
  const draggedEl = document.getElementById(id)
  target.appendChild(draggedEl)                   // 把元素移到目标区域
})
```

## 四、DataTransfer 对象（拖拽数据的"快递包裹"）

`DataTransfer` 是拖拽过程中**传递数据的载体**，通过 `e.dataTransfer` 访问。它是整套 API 的核心——拖拽的本质就是"带数据走"。

### 1. 存取数据

```js
// 存：在 dragstart 中调用
e.dataTransfer.setData('text/plain', '要传递的数据')

// 取：在 drop 中调用
const data = e.dataTransfer.getData('text/plain')
```

### 2. format（格式）参数

`setData(format, data)` / `getData(format)` 的 `format` 是 **MIME 类型字符串**，常见取值：

| format 值 | 用途 |
|------|------|
| `'text/plain'` | 纯文本（最常用） |
| `'text/html'` | HTML 内容 |
| `'text/uri-list'` | URL 链接 |
| `'application/x-xxx'` | 自定义类型 |

> ⚠️ **注意：** 古老的 IE9 及以下**只认 `'Text'` 和 `'URL'`** 两种格式（且首字母大写）。现代浏览器统一用小写 MIME，如 `'text/plain'`。现在基本不用考虑 IE。

### 3. 完整属性 / 方法表

| 属性 / 方法 | 作用 | 常用时机 |
|------|------|------|
| `setData(format, data)` | 存入数据 | `dragstart` |
| `getData(format)` | 读取数据 | `drop`（其他时机受保护模式限制） |
| `clearData([format])` | 清除指定/全部数据 | `dragstart` / `dragend` |
| `types` | 当前已存数据的格式数组（只读） | 任意时机 |
| `files` | 被拖入的文件列表（`FileList`） | `drop`（文件拖拽） |
| `items` | 数据项列表（含类型和 kind） | `drop` |
| `dropEffect` | 当前的放置效果（影响光标） | `dragover` / `drop` |
| `effectAllowed` | 允许的放置效果 | `dragstart` |

### 4. ⚠️ getData 的"保护模式"

出于安全考虑，`getData()` **不是随时都能读到的**：

- 在 `dragstart`、`drag`、`dragenter`、`dragover`、`dragleave`、`dragend` 中调用 `getData()`，通常返回 **空字符串**（防止目标区域"偷窥"源数据）。
- 只有在 **`drop`** 事件中，才能正常 `getData()`。
- 但 `dataTransfer.types` 在任意时机都可读，可用来判断"拖进来的是不是我要的类型"。

```js
// 用 types 在 dragover 中判断拖入类型，决定是否允许 drop
target.addEventListener('dragover', (e) => {
  if (e.dataTransfer.types.includes('Files')) {
    e.preventDefault()    // 拖的是文件，才允许放置
  }
})
```

## 五、完整示例：两个容器之间拖拽元素

这是一个可直接运行的完整示例：把左边盒子的元素拖到右边盒子，再拖回来。

```html
<style>
  .box {
    min-height: 120px;
    border: 2px dashed #aaa;
    padding: 10px;
    display: inline-block;
    width: 150px;
    vertical-align: top;
  }
  .box.drag-over { border-color: #1890ff; background: #e6f7ff; }   /* 接收高亮 */
  .item { margin: 5px 0; padding: 8px; background: #f0f0f0; cursor: move; }
  .item.dragging { opacity: 0.4; }                                  /* 拖拽中半透明 */
</style>

<div id="box1" class="box">
  <div class="item" draggable="true" id="item1">元素 A</div>
  <div class="item" draggable="true" id="item2">元素 B</div>
</div>
<div id="box2" class="box"></div>

<script>
  const boxes = document.querySelectorAll('.box')
  const items = document.querySelectorAll('.item')

  // ① 源元素：开始拖拽，存入元素 ID
  items.forEach((item) => {
    item.addEventListener('dragstart', (e) => {
      e.dataTransfer.setData('text/plain', e.target.id)   // 存数据
      e.dataTransfer.effectAllowed = 'move'
      e.target.classList.add('dragging')                  // 拖拽中样式
    })
    item.addEventListener('dragend', (e) => {
      e.target.classList.remove('dragging')               // 收尾：移除样式
    })
  })

  // ② 目标容器：处理 dragover / dragenter / dragleave / drop
  boxes.forEach((box) => {
    // 必须阻止 dragover 默认行为，否则 drop 不触发
    box.addEventListener('dragover', (e) => {
      e.preventDefault()
      e.dataTransfer.dropEffect = 'move'
    })

    box.addEventListener('dragenter', (e) => {
      e.preventDefault()
      box.classList.add('drag-over')                      // 进入高亮
    })

    box.addEventListener('dragleave', () => {
      box.classList.remove('drag-over')                   // 离开取消高亮
    })

    box.addEventListener('drop', (e) => {
      e.preventDefault()                                  // 必须阻止！
      box.classList.remove('drag-over')
      const id = e.dataTransfer.getData('text/plain')     // 取出元素 ID
      const draggedEl = document.getElementById(id)
      if (draggedEl) box.appendChild(draggedEl)           // 把元素移到当前容器
    })
  })
</script>
```

## 六、拖拽文件上传

拖拽 API 最实用的场景之一：**把文件从系统拖到网页里上传**。被拖入的文件通过 `e.dataTransfer.files` 获取，它是一个 `FileList`，每个元素是 `File` 对象（配合 [Blob/File API](./ArrayBuffer.md) 使用）。

```html
<div id="dropzone" style="width:300px;height:150px;border:2px dashed #aaa;text-align:center;line-height:150px;">
  把文件拖到这里上传
</div>

<script>
  const dz = document.getElementById('dropzone')

  // 必须 preventDefault 才能成为放置区
  dz.addEventListener('dragover', (e) => {
    e.preventDefault()
    dz.style.borderColor = '#1890ff'
  })

  dz.addEventListener('drop', (e) => {
    e.preventDefault()
    dz.style.borderColor = '#aaa'

    const files = e.dataTransfer.files          // FileList
    if (!files.length) return

    for (const file of files) {
      console.log(`文件名：${file.name}`)        // 文件名
      console.log(`大小：${file.size} 字节`)     // 字节数
      console.log(`类型：${file.type}`)          // MIME 类型

      // 方式一：用 FormData 上传到服务器
      // const formData = new FormData()
      // formData.append('file', file)
      // fetch('/upload', { method: 'POST', body: formData })

      // 方式二：用 FileReader 读取内容（如预览图片）
      if (file.type.startsWith('image/')) {
        const reader = new FileReader()
        reader.onload = (ev) => console.log('图片预览地址：', ev.target.result)
        reader.readAsDataURL(file)              // 转成 base64
      }
    }
  })
</script>
```

> 💡 **提示：** 文件拖拽时，`e.dataTransfer.types` 会包含 `'Files'`，可在 `dragover` 中据此判断"拖的是文件还是元素"，决定是否允许放置。

## 七、设置拖拽的视觉效果（dropEffect / effectAllowed）

这两个属性决定**拖拽时鼠标光标的样式**（复制 ➕、移动 ↔️、链接 🔗、禁止 🚫），让用户直观知道松手后会发生什么。

| 属性 | 在哪设置 | 取值 | 说明 |
|------|:---:|------|------|
| `effectAllowed` | `dragstart`（源元素） | `'none'` `'copy'` `'move'` `'link'` `'copyMove'` `'all'` 等 | 声明源元素**允许**的操作 |
| `dropEffect` | `dragover` / `drop`（目标元素） | `'none'` `'copy'` `'move'` `'link'` | 设置目标**想要的**操作（影响光标） |

```js
// 源元素：声明允许"复制"
source.addEventListener('dragstart', (e) => {
  e.dataTransfer.effectAllowed = 'copy'
})

// 目标元素：声明这里要"复制"，光标会显示 ➕
target.addEventListener('dragover', (e) => {
  e.preventDefault()
  e.dataTransfer.dropEffect = 'copy'
})
```

> 💡 **提示：** 如果 `dropEffect` 设的值不在 `effectAllowed` 允许范围内，光标会显示"禁止"图标。两者需匹配。

## 八、常见问题与注意事项

### 1. dragleave 在子元素间穿梭时的"闪烁"问题

当放置区域**内部有子元素**时，拖拽指针从父元素移到子元素，会触发父元素的 `dragleave`（因为事件目标 `e.target` 变成了子元素），导致高亮被错误移除，来回闪烁。

**解决方案：用计数器**（进入 +1，离开 -1，归零才算真正离开）。

```js
let dragCounter = 0

dz.addEventListener('dragenter', (e) => {
  e.preventDefault()
  dragCounter++                       // 进入（含进入子元素）计数 +1
  dz.classList.add('drag-over')
})

dz.addEventListener('dragleave', () => {
  dragCounter--                       // 离开（含离开到子元素）计数 -1
  if (dragCounter === 0) {            // 真正完全离开容器才取消高亮
    dz.classList.remove('drag-over')
  }
})

dz.addEventListener('drop', (e) => {
  e.preventDefault()
  dragCounter = 0                     // drop 时重置计数器
  dz.classList.remove('drag-over')
})

dz.addEventListener('dragover', (e) => e.preventDefault())   // 允许放置
```

### 2. 忘记 `preventDefault` 导致 drop 失效

这是新手最常遇到的坑：**`dragover` 或 `drop` 里没有 `e.preventDefault()`**，结果拖拽元素无法放下，或松手时浏览器打开了文件/跳转了链接。记住：

- `dragover` 里 `preventDefault()` → 让元素成为合法放置区
- `drop` 里 `preventDefault()` → 阻止浏览器的默认动作

### 3. setData 的数据类型只能是字符串

`setData(format, data)` 的 `data` 会被转成字符串。**不能直接存对象**，需要先 `JSON.stringify`，取出后 `JSON.parse`：

```js
// ✅ 存对象要先序列化
e.dataTransfer.setData('text/plain', JSON.stringify({ id: 1, name: 'Alice' }))

const obj = JSON.parse(e.dataTransfer.getData('text/plain'))
```

### 4. 移动端原生拖拽支持很差

HTML5 拖拽 API 基于**鼠标事件**，在**触摸设备**（手机/平板）上支持极差甚至不工作。如果项目需要支持移动端拖拽，应改用 **Pointer Events / Touch Events** 手动实现，或使用成熟库（如 `Sortable.js` / `@dnd-kit`）。

| 场景 | 推荐方案 |
|------|------|
| 仅桌面端、简单拖拽 | 原生 HTML5 Drag and Drop |
| 需要移动端 / 复杂排序动画 | Pointer Events 自实现 或 Sortable.js 等库 |
| Vue 生态看板拖拽 | `vuedraggable`（基于 Sortable.js） |

### 5. `draggable="true"` 要写在 HTML 上

`draggable` 是 HTML 属性，可以直接写在标签上 `draggable="true"`，也可以用 JS 设置 `el.draggable = true`。注意它的值是字符串，布尔属性。

## 九、Vue 3 中实现拖拽

Vue 3 中直接用原生拖拽 API，配合响应式数据和模板事件绑定即可。下面实现一个**可拖拽排序的列表**。

```vue
<script setup>
import { ref } from 'vue'

const list = ref(['苹果', '香蕉', '橙子', '葡萄'])
let dragIndex = null      // 用普通变量记录拖拽起始索引（不需要响应式）

// 开始拖拽：记录被拖元素的索引
function onDragStart(index) {
  dragIndex = index
}

// dragover 必须阻止默认行为（模板里用 .prevent 修饰符也行）
function onDragOver(e) {
  e.preventDefault()
}

// 放置：把被拖元素插到目标索引位置
function onDrop(index) {
  if (dragIndex === null || dragIndex === index) return
  // 从原位置取出被拖项
  const [moved] = list.value.splice(dragIndex, 1)
  // 插入到新位置
  list.value.splice(index, 0, moved)
  dragIndex = null
}
</script>

<template>
  <ul>
    <li
      v-for="(item, index) in list"
      :key="item"
      draggable="true"
      @dragstart="onDragStart(index)"
      @dragover="onDragOver"
      @drop="onDrop(index)"
    >
      {{ item }}
    </li>
  </ul>
</template>

<style scoped>
li {
  padding: 8px;
  margin: 4px 0;
  background: #f5f5f5;
  cursor: move;
  list-style: none;
}
</style>
```

> 💡 **提示：** Vue 模板里也可以用修饰符简化阻止默认行为：`@dragover.prevent="onDragOver"`，这样 `onDragOver` 里就不用手动 `preventDefault` 了。同理 `@drop.prevent="onDrop(index)"`。

## 十、原生拖拽 vs 鼠标事件实现拖拽（对比）

除了 HTML5 Drag and Drop API，还可以用 `mousedown`/`mousemove`/`mouseup` 手动实现拖拽。两者各有适用场景：

| 对比维度 | HTML5 拖拽 API | 鼠标/指针事件手写 |
|------|:---:|:---:|
| 实现难度 | 简单（声明式 `draggable`） | 较复杂（需手算坐标） |
| 桌面端 | ✅ 良好 | ✅ 良好 |
| 移动端 | ❌ 支持差 | ✅ 用 Pointer Events 支持 |
| 拖拽外观 | 浏览器默认半透明虚影 | 完全自定义 |
| 数据传递 | `DataTransfer`（跨元素） | 自己用变量维护 |
| 典型场景 | 排序、上传文件、容器间移动 | 自定义拖动跟随、自由位置定位 |

> 💡 **提示：** 如果需要"拖动时元素跟随鼠标自由移动"（如自定义弹窗拖拽），HTML5 API 做不到（它显示的是浏览器虚影），必须用鼠标事件手写。

**鼠标事件实现自由拖拽的简化版：**

```js
const box = document.getElementById('box')
let isDragging = false, offsetX = 0, offsetY = 0

box.addEventListener('mousedown', (e) => {
  isDragging = true
  offsetX = e.clientX - box.offsetLeft    // 记录鼠标相对元素的偏移
  offsetY = e.clientY - box.offsetTop
})

document.addEventListener('mousemove', (e) => {
  if (!isDragging) return
  box.style.left = e.clientX - offsetX + 'px'   // 跟随鼠标移动
  box.style.top = e.clientY - offsetY + 'px'
})

document.addEventListener('mouseup', () => {
  isDragging = false                            // 松手停止
})
```

## 十一、总结

| 知识点 | 一句话记忆 |
|------|------|
| **让元素可拖** | 加 `draggable="true"`（`<img>`/`<a>` 默认可拖） |
| **7 个事件** | 源：`dragstart`/`drag`/`dragend`；目标：`dragenter`/`dragover`/`dragleave`/`drop` |
| **最关键的坑** | `dragover` 和 `drop` 必须 `preventDefault()`，否则无法放置 |
| **传数据** | `dragstart` 用 `setData` 存，`drop` 用 `getData` 取；对象需 `JSON` 序列化 |
| **文件上传** | `e.dataTransfer.files` 拿到 `FileList` |
| **视觉效果** | `effectAllowed`（源）+ `dropEffect`（目标）控制光标 |
| **闪烁问题** | 有子元素时 `dragleave` 用计数器解决 |
| **移动端** | 原生 API 支持差，用 Pointer Events 或 Sortable.js |
| **自由拖动** | 用 `mousedown`/`mousemove`/`mouseup` 手写 |

**核心理解**：HTML5 拖拽 = **`draggable` 属性 + 事件监听 + `DataTransfer` 数据传递**。掌握"两个必须 `preventDefault` 的地方"（`dragover` 和 `drop`）和"数据的存取时机"（`dragstart` 存、`drop` 取），就能应对绝大多数拖拽需求；而真正复杂的跨端、动画拖拽，交给 Pointer Events 或专业库。
