# 防抖与节流（Debounce & Throttle）

## 一、概述

**防抖**和**节流**是两种用来**限制函数执行频率**的优化手段。当一个事件会在短时间内被**高频触发**时（比如输入框打字、窗口缩放、鼠标滚动），如果不加控制，绑定的函数会被执行成百上千次，造成页面卡顿、接口被刷爆。防抖和节流就是用来"把执行次数降下来"的两个工具。

一句话区分：

- **防抖（debounce）**：事件**停止触发**一段时间后，才执行一次。`（"你先停，停够了再执行"）`
- **节流（throttle）**：事件触发期间，每隔固定时间**最多执行一次**。`（"固定节奏执行，其余忽略"）`

> 📖 参考：防抖与节流是社区约定俗成的优化模式，本身不属于规范。权威实现参考 [Lodash — debounce](https://lodash.com/docs/4.17.15#debounce) 与 [Lodash — throttle](https://lodash.com/docs/4.17.15#throttle)

---

## 二、为什么需要它们：高频触发的问题

先看一个**没做优化**的反面例子：

```js
// ❌ 每按一个字母就发一次请求，输入 "javascript" 会请求 10 次！
input.addEventListener('input', (e) => {
  fetch('/search?q=' + e.target.value)
})

// ❌ 滚动一屏可能触发几十上百次 scroll，每次都计算布局，卡顿
window.addEventListener('scroll', () => {
  console.log(document.documentElement.scrollTop)
})
```

`input` 每敲一个字符触发一次，`scroll`/`resize`/`mousemove` 滚动时一秒能触发几十上百次。这些场景下我们**并不需要那么频繁地执行**，于是引入防抖 / 节流。

| 问题场景 | 触发频率 | 危害 |
|------|------|------|
| 搜索输入 | 每个字符一次 | 大量无效请求，浪费带宽 |
| 窗口 `resize` | 拖动边框时持续 | 反复重排重绘，卡顿 |
| `scroll` 滚动 | 每像素都可能 | 频繁计算，掉帧 |
| `mousemove` | 鼠标移动持续 | 函数过载 |

## 三、防抖（Debounce）

### 1. 原理与类比

**防抖的核心**：触发事件后，**不立即执行**，而是等待 `delay` 时间；如果在这段时间内又触发了，就**重新计时**。只有"安静"了 `delay` 时间，才真正执行。

> 类比：**电梯门**。有人进电梯，门会等几秒再关；如果这几秒内又有人进来，门就重新等待。直到连续 `delay` 秒没人进来，门才关上。

```
防抖时间轴（delay = 500ms）：

用户输入:  a    b    c                    （停止输入）
           │    │    │
           ▼    ▼    ▼
触发:     ─①───②───③────────────── 等 500ms ──▶ ✅ 执行
计时器:    等→清 等→清 等→清            满 500ms
                                    （只执行 1 次！）

要点：每次新触发都"清空重来"，只有彻底停下来满 delay，才执行。
```

### 2. 基本实现

```js
/**
 * 防抖（非立即执行版）：停止触发 delay 后执行最后一次
 * @param {Function} fn     要防抖的函数
 * @param {number}   delay  等待时间（毫秒）
 */
function debounce(fn, delay) {
  let timer = null                          // 用闭包保存定时器
  return function (...args) {
    if (timer) clearTimeout(timer)          // ① 关键：每次触发都清除上一个定时器（重新计时）
    timer = setTimeout(() => {
      fn.apply(this, args)                  // ② 保留 this 和参数，执行真正的函数
      timer = null                          // 执行后重置
    }, delay)
  }
}
```

使用：

```js
// 搜索框：用户停止输入 500ms 后才请求
input.addEventListener('input', debounce((e) => {
  fetch('/search?q=' + e.target.value)
}, 500))
```

> 💡 **提示：** 闭包变量 `timer` 是关键——它让每次调用都能访问到"上一个定时器"，从而实现"清除并重新计时"。

### 3. 进阶：立即执行版（leading edge）

有时我们希望**第一次触发立即执行**，之后的连续触发被忽略，直到停下来一段时间后才恢复"可立即执行"状态。例如按钮首次点击立即响应、防连击。

```js
/**
 * 防抖（可配置立即执行）
 * @param immediate true 表示首次触发立即执行
 */
function debounce(fn, delay, immediate = false) {
  let timer = null
  return function (...args) {
    if (timer) clearTimeout(timer)
    if (immediate) {
      // 没有 timer 在跑 → 说明这是"新一轮"第一次触发，可以立即执行
      const callNow = !timer
      timer = setTimeout(() => {
        timer = null                        // delay 后解锁，允许下一次立即执行
      }, delay)
      if (callNow) fn.apply(this, args)
    } else {
      timer = setTimeout(() => {
        fn.apply(this, args)
        timer = null
      }, delay)
    }
  }
}

// 首次点击立即提交，500ms 内的连击被忽略
btn.addEventListener('click', debounce(submit, 500, true))
```

### 4. 进阶：带取消功能

有时需要在某些条件下取消等待中的执行（比如组件卸载、用户取消）。

```js
function debounce(fn, delay) {
  let timer = null
  const debounced = function (...args) {
    if (timer) clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
      timer = null
    }, delay)
  }
  // 挂载一个 cancel 方法，可手动取消
  debounced.cancel = function () {
    clearTimeout(timer)
    timer = null
  }
  return debounced
}

const search = debounce(doSearch, 500)
// 用户点了"取消"按钮，取消还没执行的搜索
cancelBtn.addEventListener('click', search.cancel)
```

## 四、节流（Throttle）

### 1. 原理与类比

**节流的核心**：在事件持续触发期间，**每隔 `delay` 时间最多执行一次**，其余的触发全部忽略。它像一个"固定节奏的闸门"，不管你来得多频繁，我都按固定频率放行。

> 类比：**水龙头滴水**。无论你把龙头拧多大，水滴都按固定频率一滴一滴落下。又如**红绿灯**，固定时间间隔切换。

```
节流时间轴（delay = 500ms）：

用户滚动:  ●●●●●●●●●●●●●●●●●● （持续触发）
           │      │      │      │
           ▼      ▼      ▼      ▼
执行:      ●      ●      ●      ●          （每隔 500ms 执行一次）

要点：固定频率放行，触发再频繁，单位时间内也只执行有限次。
```

### 2. 实现①：时间戳版

利用上一次执行的时间戳判断是否达到间隔。

```js
/**
 * 节流 - 时间戳版
 * 特点：① 首次触发立即执行  ② 停止后最后一次不执行
 */
function throttle(fn, delay) {
  let lastTime = 0                          // 上次执行时间，初始 0 保证首次能执行
  return function (...args) {
    const now = Date.now()
    if (now - lastTime >= delay) {          // 距离上次执行已超过 delay
      fn.apply(this, args)
      lastTime = now                        // 更新执行时间
    }
  }
}
```

> 💡 **提示：** 首次触发时 `lastTime = 0`，`now - 0` 几乎一定 `>= delay`，所以**第一次会立即执行**。

### 3. 实现②：定时器版

利用定时器保证每个间隔只执行一次。

```js
/**
 * 节流 - 定时器版
 * 特点：① 首次触发不立即执行（要等 delay）  ② 停止后最后一次会执行
 */
function throttle(fn, delay) {
  let timer = null
  return function (...args) {
    if (!timer) {                           // 没有 timer 在跑 → 开启一个
      timer = setTimeout(() => {
        fn.apply(this, args)
        timer = null                        // 执行完置空，允许下一次
      }, delay)
    }
  }
}
```

### 4. 实现③：时间戳 + 定时器（最完整）

时间戳版"首次立即但末次不执行"，定时器版"末次执行但首次不立即"。两者**组合**，就能做到**首次立即执行 + 末次也执行**，体验最佳。

```js
/**
 * 节流 - 组合版（首次立即执行 + 最后一次也会执行）
 */
function throttle(fn, delay) {
  let lastTime = 0
  let timer = null
  return function (...args) {
    const now = Date.now()
    const remaining = delay - (now - lastTime)   // 距离下次可执行还剩多久

    if (remaining <= 0) {
      // 已超过间隔：立即执行，并清除可能存在的定时器
      if (timer) { clearTimeout(timer); timer = null }
      fn.apply(this, args)
      lastTime = now
    } else if (!timer) {
      // 还没到间隔：设一个定时器，保证最后一次触发也能执行到
      timer = setTimeout(() => {
        fn.apply(this, args)
        lastTime = Date.now()
        timer = null
      }, remaining)
    }
  }
}
```

> 💡 **提示：** 日常使用，**时间戳版**（简单、首次立即）通常就够用了；组合版适合对"末次一定要执行"有要求的场景（如拖拽时记录最终位置）。

## 五、防抖 vs 节流 对比

| 对比维度 | 防抖（debounce） | 节流（throttle） |
|------|------|------|
| **核心思想** | 停止触发后再执行 | 固定时间间隔执行 |
| **执行时机** | 最后一次触发后 delay 执行 | 触发期间每 delay 执行一次 |
| **高频触发的结果** | 只执行最后一次（或第一次） | 按固定频率多次执行 |
| **类比** | 电梯门、弹簧 | 水龙头、红绿灯 |
| **典型场景** | 搜索联想、resize、表单校验 | 滚动监听、mousemove、按钮防连击 |
| **关注点** | "我要最终的值" | "我要过程中的节奏" |

### 同样的高频场景，选哪个？

```
搜索框输入    → 防抖（用户停下来才搜，避免每字一搜）
窗口 resize   → 防抖（停止调整后才重算）
滚动加载更多  → 节流（滚动过程中就要判断是否到底）
鼠标拖拽位置  → 节流（过程中就要更新位置）
按钮防连击    → 节流 或 防抖(立即执行版) 都可
```

## 六、实际应用场景

### 场景 1：搜索框输入联想（防抖）

```js
// 用户停止输入 500ms 后才发请求
const searchInput = document.getElementById('search')
searchInput.addEventListener('input', debounce(function (e) {
  fetch(`/api/search?q=${e.target.value}`)
    .then((r) => r.json())
    .then((list) => renderList(list))
}, 500))
```

### 场景 2：窗口 resize 重算布局（防抖）

```js
// 拖动改变窗口大小，停止 300ms 后才重新计算，避免反复重排
window.addEventListener('resize', debounce(() => {
  const width = document.documentElement.clientWidth
  console.log('窗口宽度：', width)
  // 重新布局 / 重绘图表
}, 300))
```

### 场景 3：滚动监听 / 懒加载（节流）

```js
// 滚动过程中每 200ms 最多判断一次是否到底，性能友好
window.addEventListener('scroll', throttle(() => {
  const { scrollTop, scrollHeight, clientHeight } = document.documentElement
  if (scrollTop + clientHeight >= scrollHeight - 50) {
    loadMore()    // 距底部 50px 时加载更多
  }
}, 200))
```

### 场景 4：按钮防重复提交（节流 / 防抖）

```js
// 立即执行版防抖：首次点击立即提交，1 秒内重复点击忽略
submitBtn.addEventListener('click', debounce(submitForm, 1000, true))
```

## 七、Vue 3 中的使用

### 方式 1：封装成组合式函数（推荐）

把防抖 / 节流封装成可复用的 hook，**并在组件卸载时清除定时器**（避免内存泄漏）。

```js
// useDebounce.js
import { onUnmounted } from 'vue'

export function useDebounce(fn, delay = 300) {
  let timer = null
  const debounced = (...args) => {
    if (timer) clearTimeout(timer)
    timer = setTimeout(() => fn(...args), delay)
  }
  // 组件卸载时清理定时器
  onUnmounted(() => {
    if (timer) clearTimeout(timer)
  })
  return debounced
}

// useThrottle.js（时间戳版）
export function useThrottle(fn, delay = 300) {
  let lastTime = 0
  return (...args) => {
    const now = Date.now()
    if (now - lastTime >= delay) {
      fn(...args)
      lastTime = now
    }
  }
}
```

在组件中使用：

```vue
<script setup>
import { ref } from 'vue'
import { useDebounce } from './useDebounce'

const keyword = ref('')

// 用户停止输入 500ms 后搜索
const handleSearch = useDebounce((val) => {
  console.log('搜索：', val)
  // fetchSearch(val)
}, 500)
</script>

<template>
  <input v-model="keyword" @input="handleSearch(keyword)" />
</template>
```

### 方式 2：使用 VueUse（最省心）

[VueUse](https://vueuse.org/) 提供了开箱即用的 `useDebounceFn` / `useThrottleFn`，生产环境推荐直接用。

```vue
<script setup>
import { ref } from 'vue'
import { useDebounceFn } from '@vueuse/core'

const keyword = ref('')

// VueUse 的防抖函数
const handleSearch = useDebounceFn((val) => {
  console.log('搜索：', val)
}, 500)
</script>

<template>
  <input v-model="keyword" @input="handleSearch(keyword)" />
</template>
```

### 方式 3：使用 Lodash

生产中如果已经引入了 lodash，直接用它功能最全的实现（支持 `leading` / `trailing` / `maxWait` 等选项）：

```js
import { debounce, throttle } from 'lodash-es'

// lodash 节流：首次立即执行(leading)，末次也执行(trailing)
const onScroll = throttle(() => {
  console.log('滚动处理')
}, 200)

// lodash 防抖：支持 maxWait（最大等待时间，防止一直触发永远不执行）
const onInput = debounce(search, 500, { leading: false, trailing: true, maxWait: 1000 })
```

> 💡 **提示：** 推荐优先用 **VueUse**（贴合 Vue 响应式）或 **lodash-es**（功能最全），手写实现主要用于理解原理和面试。

## 八、常见问题与注意事项

### 1. 必须保留 `this` 和参数

防抖/节流返回的是**新函数**，如果原函数依赖 `this`（如对象方法），必须用 `apply` 透传 `this` 和参数，否则行为异常。

```js
// ✅ 正确：fn.apply(this, args)
// ❌ 错误：直接 fn()，丢了 this
```

### 2. 箭头函数与 `this`

注意：返回的函数如果是**普通 `function`**，`this` 才是调用者；如果写成箭头函数，`this` 会被固定为定义时外层的 `this`。所以标准实现里**外层用普通 function**。`setTimeout` 内部则用箭头函数来捕获外层的 `this`。

```js
function debounce(fn, delay) {
  let timer = null
  return function (...args) {          // 普通函数，this = 调用者
    clearTimeout(timer)
    timer = setTimeout(() => {          // 箭头函数，继承上面的 this
      fn.apply(this, args)              // ✅ this 正确
    }, delay)
  }
}
```

### 3. 定时器和事件监听要清理

在 Vue / React 等组件中，组件**卸载时**一定要 `clearTimeout` 并移除事件监听器，否则会**内存泄漏**（组件销毁了，定时器还在跑、还引用着旧数据）。

```js
onUnmounted(() => {
  clearTimeout(timer)
  window.removeEventListener('scroll', handler)
})
```

### 4. 防抖的"永远不执行"风险

如果事件**一直不停触发**（间隔永远小于 delay），非立即执行版防抖会**永远不执行**。需要保证最后能执行的场景，可用 lodash 的 `maxWait` 选项设置最大等待时间，或改用节流。

### 5. 不要对需要"每次都执行"的函数节流

比如表单 `change` 提交、关键日志记录，这些必须每次都执行的逻辑**不能**防抖/节流，否则会丢数据。只对"执行频率高、且可以容忍丢失中间执行"的场景使用。

### 6. 防抖节流的函数是"有状态的"

每次调用 `debounce(fn, delay)` 都会**新建一个闭包**（一个新的 timer 状态）。如果每次渲染都重新创建，状态就不连续了。所以在 Vue 中要确保**复用同一个**防抖函数（如定义在 setup 顶层、或用 `useDebounceFn`）。

```js
// ❌ 错误：每次 input 都新建防抖函数，状态不共享，等于没防抖
input.addEventListener('input', debounce(search, 500))   // 每次都新建

// ✅ 正确：复用同一个防抖函数
const debouncedSearch = debounce(search, 500)
input.addEventListener('input', debouncedSearch)
```

## 九、总结

| 知识点 | 一句话记忆 |
|------|------|
| **防抖** | 停止触发 `delay` 后执行最后一次（电梯门） |
| **节流** | 每隔 `delay` 最多执行一次（水龙头） |
| **防抖核心代码** | `clearTimeout` 旧定时器 → 开新定时器 |
| **节流核心代码** | 时间戳判断 / 定时器占位，控制间隔 |
| **节流三种实现** | 时间戳（首立即）、定时器（末执行）、组合（首+末） |
| **保留 this/参数** | 用 `fn.apply(this, args)` |
| **生产首选** | VueUse 或 lodash-es |
| **组件卸载** | 必须 `clearTimeout` 防止内存泄漏 |

**核心理解**：防抖和节流都是用**闭包 + 定时器/时间戳**来"稀释"高频函数的执行。防抖关注**最终的值**（停下来再说），节流关注**过程中的节奏**（按固定频率放行）。掌握两者的实现原理（尤其闭包保存 `timer`/`lastTime` 状态）和适用场景的区分，就足以应对绝大多数性能优化和面试问题。

> 📖 延伸阅读：[Lodash debounce](https://lodash.com/docs/4.17.15#debounce)、[VueUse — useDebounceFn](https://vueuse.org/core/usedebouncefn/)
