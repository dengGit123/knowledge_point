# JavaScript 闭包完全指南

## 概述

**闭包（Closure）** 是 JavaScript 中最重要的概念之一。简单来说：

> 闭包是一个函数能够**记住并访问**其词法作用域中的变量，即使该函数在其作用域之外执行。

```
┌─────────────────────────────────────────────────────────────┐
│                        闭包的本质                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  函数 + 定义时的词法环境 = 闭包                                │
│                                                              │
│  ┌──────────────────┐         ┌──────────────────┐         │
│  │   内部函数        │  +────▶  │   外部变量环境    │         │
│  │   (可执行代码)    │         │   (持久化存储)    │         │
│  └──────────────────┘         └──────────────────┘         │
│            │                            ▲                   │
│            └────────────────────────────┘                   │
│                    引用关系持续存在                           │
└─────────────────────────────────────────────────────────────┘
```

## 一、闭包的形成机制

### 1.1 三个必要条件

闭包的形成需要同时满足以下三个条件：

| 条件 | 说明 | 示例 |
|------|------|------|
| **函数嵌套** | 一个函数内部定义另一个函数 | `outer()` 内定义 `inner()` |
| **引用外部变量** | 内部函数使用了外部函数的变量 | `inner()` 访问 `outerVar` |
| **导出到外部** | 内部函数被返回/传递/作为回调 | `return inner` |

### 1.2 闭包的创建过程

```javascript
// 步骤1：创建外部函数
function outer() {
    // 步骤2：定义外部变量（词法环境的一部分）
    let outerVar = "我是外部变量";

    // 步骤3：定义内部函数（形成闭包）
    function inner() {
        // 步骤4：内部函数引用外部变量
        console.log(outerVar);
    }

    // 步骤5：将内部函数导出到外部
    return inner;
}

// 步骤6：在外部调用，验证闭包
const closureFunc = outer();  // outer 执行完毕，但 outerVar 仍然存在
closureFunc();  // 输出: "我是外部变量"
```

<div style="background-color: #d1ecf1; border-left: 4px solid #0dcaf0; padding: 10px; margin: 10px 0;">
<strong>💡 关键理解：</strong><code>outer</code> 函数执行完成后，其作用域本应被销毁。但由于返回的 <code>inner</code> 函数仍然引用着 <code>outerVar</code>，这个变量就不会被垃圾回收，形成了闭包。
</div>

### 1.3 闭包的内存模型

```
当 const closureFunc = outer() 执行后：

┌─────────────────────────────────────────────────────────────┐
│                         全局作用域                            │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  closureFunc ────────┐                               │    │
│  └──────────────────────┼───────────────────────────────┘    │
│                         │                                    │
│                         ▼ 引用                                │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              outer 的词法环境（闭包）                  │    │
│  │  ┌────────────────────────────────────────────────┐  │    │
│  │  │  outerVar: "我是外部变量"                       │  │    │
│  │  │  inner: function() { console.log(outerVar); }  │  │    │

└─────────────────────────────────────────────────────────────┘

即使 outer() 已执行完毕，这个词法环境仍然存在于内存中！
```

## 二、闭包与作用域链

### 2.1 词法作用域（静态作用域）

JavaScript 使用的是**词法作用域**，函数的作用域在**定义时**就已确定，而不是在执行时。

```javascript
function outer() {
    let x = 10;

    function inner() {
        console.log(x);  // 这里的 x 在定义时就确定了引用关系
    }

    return inner;
}

const fn = outer();
fn();  // 输出 10，而不是去查找全局变量 x
```

### 2.2 作用域链的查找机制

```javascript
let globalVar = "全局变量";

function level1() {
    let level1Var = "第一层";

    function level2() {
        let level2Var = "第二层";

        function level3() {
            // 查找顺序：level3 → level2 → level1 → 全局
            console.log(globalVar);   // ✅ 找到
            console.log(level1Var);   // ✅ 找到
            console.log(level2Var);   // ✅ 找到
            console.log(level3Var);   // ❌ 报错
        }

        return level3;
    }

    return level2();
}

const closure = level1();
closure();
```

<div style="background-color: #fff3cd; border-left: 4px solid #ffc107; padding: 10px; margin: 10px 0;">
<strong>⚠️ 注意：</strong>作用域链的查找是单向的，只能向上查找，不能向下查找。内部函数可以访问外部变量，但外部函数不能访问内部变量。
</div>

### 2.3 变量的生命周期

```javascript
function createClosure() {
    let secret = 123;  // 这个变量会因为闭包而"长生不老"

    return {
        get: function() { return secret; },
        set: function(value) { secret = value; }
    };
}

const obj = createClosure();
console.log(obj.get());  // 123
obj.set(456);
console.log(obj.get());  // 456

// 只要 obj 存在，secret 就不会被垃圾回收
```

## 三、闭包的实际应用

### 3.1 数据私有化（模块模式）

闭包可以创建真正的私有变量，实现数据封装。

```javascript
function createBankAccount(initialBalance) {
    let balance = initialBalance;  // 私有变量

    return {
        // 公共接口
        deposit: function(amount) {
            if (amount > 0) {
                balance += amount;
                console.log(`存入 ${amount}，余额 ${balance}`);
            }
        },
        withdraw: function(amount) {
            if (amount > 0 && amount <= balance) {
                balance -= amount;
                console.log(`取出 ${amount}，余额 ${balance}`);
            } else {
                console.log('余额不足');
            }
        },
        getBalance: function() {
            return balance;  // 只读访问
        }
    };
}

const myAccount = createBankAccount(1000);
myAccount.deposit(500);    // 存入 500，余额 1500
myAccount.withdraw(200);   // 取出 200，余额 1300
console.log(myAccount.balance);  // undefined - 无法直接访问
console.log(myAccount.getBalance());  // 1300 - 通过方法访问
```

### 3.2 状态保持（计数器）

```javascript
function createCounter(start = 0, step = 1) {
    let count = start;

    return {
        increment: function() {
            count += step;
            return count;
        },
        decrement: function() {
            count -= step;
            return count;
        },
        reset: function() {
            count = start;
            return count;
        },
        getCount: function() {
            return count;
        }
    };
}

const counter = createCounter(0, 2);
console.log(counter.increment());  // 2
console.log(counter.increment());  // 4
console.log(counter.decrement());  // 2
console.log(counter.reset());      // 0

// 创建独立的计数器
const anotherCounter = createCounter(10);
console.log(anotherCounter.increment());  // 11
```

### 3.3 函数柯里化（Currying）

```javascript
function multiply(a) {
    return function(b) {
        return function(c) {
            return a * b * c;
        };
    };
}

// 调用方式1
console.log(multiply(2)(3)(4));  // 24

// 调用方式2：部分应用
const multiplyBy2 = multiply(2);
const multiplyBy2And3 = multiplyBy2(3);
console.log(multiplyBy2And3(4));  // 24

// 实际应用：创建专用函数
const multiplyBy10 = multiply(10);
console.log(multiplyBy10(5)(2));  // 100
```

### 3.4 防抖与节流

```javascript
// 防抖：延迟执行，重复触发会重置计时
function debounce(fn, delay) {
    let timer = null;

    return function(...args) {
        clearTimeout(timer);
        timer = setTimeout(() => {
            fn.apply(this, args);
        }, delay);
    };
}

// 节流：固定时间间隔执行
function throttle(fn, delay) {
    let lastTime = 0;

    return function(...args) {
        const now = Date.now();
        if (now - lastTime >= delay) {
            lastTime = now;
            fn.apply(this, args);
        }
    };
}

// 使用示例
const handleResize = debounce(() => {
    console.log('窗口大小改变了！');
}, 300);

window.addEventListener('resize', handleResize);
```

### 3.5 单例模式

```javascript
const Singleton = (function() {
    let instance;

    function createInstance() {
        return {
            data: [],
            add: function(item) {
                this.data.push(item);
            },
            getData: function() {
                return this.data;
            }
        };
    }

    return {
        getInstance: function() {
            if (!instance) {
                instance = createInstance();
            }
            return instance;
        }
    };
})();

const instance1 = Singleton.getInstance();
const instance2 = Singleton.getInstance();

console.log(instance1 === instance2);  // true - 始终是同一个实例
```

## 四、闭包的常见陷阱

### 4.1 循环中的闭包问题

```javascript
// ❌ 错误示例：经典陷阱
for (var i = 1; i <= 3; i++) {
    setTimeout(function() {
        console.log(i);  // 全部输出 4
    }, 1000);
}
// 原因：var 没有块级作用域，所有回调共享同一个 i
```

```javascript
// ✅ 解决方案 1：使用 let（推荐）
for (let i = 1; i <= 3; i++) {
    setTimeout(function() {
        console.log(i);  // 1, 2, 3
    }, 1000);
}
```

```javascript
// ✅ 解决方案 2：使用 IIFE
for (var i = 1; i <= 3; i++) {
    (function(j) {
        setTimeout(function() {
            console.log(j);  // 1, 2, 3
        }, 1000);
    })(i);
}
```

```javascript
// ✅ 解决方案 3：使用 bind
for (var i = 1; i <= 3; i++) {
    setTimeout(function(j) {
        console.log(j);  // 1, 2, 3
    }.bind(null, i), 1000);
}
```

### 4.2 闭包中的变量共享

```javascript
function createFunctions() {
    let result = [];

    for (var i = 0; i < 3; i++) {
        result[i] = function() {
            return i;
        };
    }

    return result;
}

const funcs = createFunctions();
console.log(funcs[0]());  // 3
console.log(funcs[1]());  // 3
console.log(funcs[2]());  // 3

// 所有函数共享同一个 i！
```

### 4.3 this 指向问题

```javascript
const obj = {
    name: 'Alice',
    friends: ['Bob', 'Charlie'],

    showFriends: function() {
        this.friends.forEach(function(friend) {
            console.log(this.name + ' 的朋友：' + friend);
            // this 指向 window 或 undefined（严格模式）
        });
    }
};

// ✅ 解决方案 1：保存 this
showFriends: function() {
    const self = this;
    this.friends.forEach(function(friend) {
        console.log(self.name + ' 的朋友：' + friend);
    });
}

// ✅ 解决方案 2：使用箭头函数
showFriends: function() {
    this.friends.forEach(friend => {
        console.log(this.name + ' 的朋友：' + friend);
    });
}

// ✅ 解决方案 3：使用 bind
showFriends: function() {
    this.friends.forEach(function(friend) {
        console.log(this.name + ' 的朋友：' + friend);
    }, this);
}
```

## 五、闭包与内存管理

### 5.1 内存泄漏的原因

闭包导致内存泄漏的本质：**闭包保持着对外部变量的引用，使这些变量无法被垃圾回收**。

```javascript
// ❌ 潜在内存泄漏
function setupHandler() {
    const largeData = new Array(1000000).fill('data');

    document.addEventListener('click', function() {
        console.log(largeData.length);  // largeData 永远不会被释放
    });
}

setupHandler();
// 即使不再需要，largeData 也会一直占用内存
```

### 5.2 避免内存泄漏的方法

#### 方法1：及时解除引用

```javascript
// ✅ 正确做法：提供清理方法
function setupHandler() {
    const largeData = new Array(1000000).fill('data');

    function handleClick() {
        console.log(largeData.length);
    }

    document.addEventListener('click', handleClick);

    // 返回清理函数
    return function cleanup() {
        document.removeEventListener('click', handleClick);
        // 此时 largeData 可以被 GC 回收
    };
}

const cleanup = setupHandler();
// 组件卸载时调用
cleanup();
```

#### 方法2：避免在循环中创建大量闭包

```javascript
// ❌ 不推荐：为每个元素创建闭包
document.querySelectorAll('.item').forEach(item => {
    item.addEventListener('click', function() {
        console.log(item.dataset.id);
    });
});

// ✅ 推荐：使用事件委托
document.querySelector('.container').addEventListener('click', function(e) {
    if (e.target.matches('.item')) {
        console.log(e.target.dataset.id);
    }
});
```

#### 方法3：手动置空

```javascript
function createClosure() {
    const data = { /* 大量数据 */ };
    let closure = null;

    closure = function() {
        if (data) {
            console.log(data.value);
        }
    };

    return function() {
        const result = closure();
        data = null;    // 手动断开引用
        closure = null; // 断开闭包自身的引用
        return result;
    };
}
```

### 5.3 性能优化建议

| 建议 | 原因 |
|------|------|
| 避免在热点代码中创建闭包 | 减少内存分配开销 |
| 及时清理不需要的闭包 | 释放内存资源 |
| 使用事件委托替代多个闭包 | 减少闭包数量 |
| 注意闭包中保留的数据量 | 避免无意中保留大量数据 |

## 六、面试常见问题

### Q1：什么是闭包？有什么用途？

**答案**：闭包是函数与其词法环境的组合。用途包括：数据私有化、状态保持、函数柯里化、模块化等。

### Q2：闭包会导致内存泄漏吗？

**答案**：闭包本身不会导致内存泄漏，但不当使用闭包（如未及时清理事件监听器）会导致变量无法被回收，造成内存占用过高。

### Q3：解释一下循环中的闭包问题

**答案**：在循环中使用 `var` 创建闭包时，所有闭包共享同一个变量。解决方法：使用 `let`、IIFE 或 `bind` 为每次迭代创建独立的作用域。

### Q4：闭包中的 this 指向什么？

**答案**：闭包中的 `this` 取决于调用方式，不会自动继承外层函数的 `this`。可以使用箭头函数、`bind` 或保存 `this` 来解决。
