### 一. 重排
* 浏览器重新计算页面**元素布局**，如(位置，尺寸，边距)的过程，之后也会进行重绘
* **触发条件**：
1. 修改元素的**几何**属性(`width`,`height`,`margin`,`padding`)等；
2. `添加`/`删除`dom元素；
3. 修改`display`属性；
4. 窗口大小变化或字体大小调整‌；
5. 读取布局属性，如(`clientHeight`,`offsetWidth`)等会强制同步重排
### 性能影响
1. 重排会触发完整的渲染流水线(布局→分层→绘制),开销较大‌
2. 浏览器对**修改操作‌**采用**异步重排**（批量处理），但对**读取操作**‌会立即**同步**重排‌

### 二.  重绘（Repaint）
*重绘是浏览器重新绘制元素**外观**(如 颜色，背景，边框)的过程，不涉及布局计算
* **触发条件**:
1. 修改 `color`,`background-color`,`border`等非几何属性；
2. 调整`opacity`或`visibility`
### ‌性能影响‌：
1. 重绘**仅触发绘制阶段**，性能开销比重排小‌
2. 频繁重绘（如动画），仍可能导致页面卡顿‌
### 三. 优化策略
1. 减少重排: 
* 批量修改DOM，使用documentFragment或class集中处理样式‌
* 避免频繁读取布局属性 (如 `getBoundingClientRect()`)
2. 利用合成层: 
* 使用`transform`、`opacity`等属性触发GPU加速，跳过重排/重绘‌
* 通过`will-change`: `transform`提前声明动画元素‌
3. 控制动画频率
* 使用`requestAnimationFrame`同步屏幕刷新率‌
* 限制动画元素数量，优先处理关键动效‌
### 读取布局属性会触发强制同步重排
* 主要触发重排的布局属性：
1. offset系列： offsetWidth，offsetHeight,offsetTop,offsetLeft
2. client系列: clientWidth,clientHeight,clientTop,clientLeft
3. scroll系列‌: scrollWidth,scrollHeight,scrollTop,scrollLeft
4. getComputedStyle()‌: 获取元素计算后的样式值
5. getBoundingClientRect()‌: 获取元素相对于视口的位置和尺寸信息
#### 触发机制说明
* 当读取这些属性时，浏览器为了返回精确的布局信息，会强制清空当前的布局队列并**立即进行重排计算**.这种强制同步布局对性能影响较大，因为重排会重新计算元素的几何属性并重构渲染树
