# super 关键字

> 官方文档：[Oracle Java Tutorial - Using the Keyword super](https://docs.oracle.com/javase/tutorial/java/IandI/super.html) ｜ 菜鸟教程：[Java super 关键字](https://www.runoob.com/java/java-super.html)

> 本文是 [继承.md](./继承.md) 中 `super` 关键字一节的**专题深入**，系统讲解 super 的所有用法与原理。

---

## 一、概述

在继承关系中，子类可以重写父类的方法、定义同名的成员变量。但有时子类**需要访问父类的成员**（比如在重写的方法里复用父类逻辑、区分父子同名的变量、初始化时调用父类构造函数），这时就需要用到 **`super` 关键字**。

> 💡 **通俗理解**：`super` 就像"爸爸的电话号码"。
> - 子类重写了方法（自己有了新做法），但偶尔还想用用**爸爸的老办法** → `super.方法()`
> - 子类和父类有同名的变量（容易混淆），想明确指出要**爸爸的那个** → `super.变量`
> - 子类对象创建时，得先让**爸爸的构造函数**跑一遍 → `super()`
>
> 它代表的是**当前对象中"父类特征"的引用**，注意：**不是指一个独立的父类对象**（详见第六节）。

---

## 二、super 的三种用法

| 用法 | 语法 | 作用 |
|------|------|------|
| ① 访问父类的**成员变量** | `super.变量名` | 当父子类有同名变量时，访问父类的那个 |
| ② 调用父类的**成员方法** | `super.方法名(...)` | 在子类（尤其是重写方法）中调用父类的方法 |
| ③ 调用父类的**构造函数** | `super(参数)` | 在子类构造函数中初始化父类部分，**必须在第一行** |

---

## 三、用法一：`super.成员变量`

当**子类定义了与父类同名的成员变量**时，子类的变量会"隐藏（hide）"父类的变量。此时用 `super.变量名` 可以访问到父类的那个变量。

```java
class Father {
    String name = "爸爸的名字";   // 父类的成员变量
}

class Son extends Father {
    String name = "儿子的名字";   // 子类同名变量，隐藏了父类的 name

    public void show() {
        System.out.println(name);        // 默认访问子类的：儿子的名字
        System.out.println(this.name);   // this 也指子类的：儿子的名字
        System.out.println(super.name);  // ✅ super 访问父类的：爸爸的名字
    }
}

// 测试
new Son().show();
// 输出：
// 儿子的名字
// 儿子的名字
// 爸爸的名字
```

> ⚠️ **注意**：成员变量**没有多态性**（编译期就绑定）。即便用 `父类引用指向子类对象`，访问 `name` 也是看声明类型。所以 `super.变量` 始终访问的是直接父类的那个变量。这点和成员方法不同，详见 [多态.md](./多态.md)。

## 四、用法二：`super.方法名()`

这是 **super 最常用的场景**：子类重写了父类的方法，但在新方法里**还想复用父类原有的逻辑**，就用 `super.方法()` 调用。

```java
class Animal {
    public void eat() {
        System.out.println("动物在吃东西");
    }
}

class Dog extends Animal {
    @Override
    public void eat() {
        super.eat();   // ✅ 先调用父类的 eat()：动物在吃东西
        System.out.println("狗在啃骨头");   // 再加上子类自己的逻辑
    }
}

// 测试
new Dog().eat();
// 输出：
// 动物在吃东西
// 狗在啃骨头
```

**典型应用：框架 / 模板方法模式**

很多框架要求子类重写方法时**先调用 `super`**（如 Android 的 `onCreate()`、Servlet 的 `init()`），否则可能破坏父类逻辑：

```java
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);   // ✅ 必须先调用父类，做框架初始化
    // ... 再做自己定制的逻辑
}
```

## 五、用法三：`super(...)` 调用父类构造函数

### 1. 默认的隐式 `super()`

**每个子类构造函数的第一行，默认都有一个隐式的 `super()`**（调用父类的无参构造），即使你不写也存在。

```java
class Animal {
    public Animal() {
        System.out.println("Animal 无参构造");
    }
}

class Dog extends Animal {
    public Dog() {
        // super();  ← 隐式存在，调用 Animal() 无参构造，且必须是第一行
        System.out.println("Dog 无参构造");
    }
}

new Dog();
// 输出：
// Animal 无参构造
// Dog 无参构造
```

### 2. 显式 `super(参数)`

当父类**没有无参构造**时（比如父类只定义了有参构造），子类构造函数**必须显式调用** `super(参数)`，否则编译报错。

```java
class Animal {
    private String type;

    // 父类只有有参构造（因此没有无参构造了）
    public Animal(String type) {
        this.type = type;
        System.out.println("Animal 有参构造：" + type);
    }
}

class Dog extends Animal {
    public Dog() {
        super("犬科");   // ✅ 必须显式调用父类有参构造，且在第一行
        System.out.println("Dog 无参构造");
    }
}

new Dog();
// 输出：
// Animal 有参构造：犬科
// Dog 无参构造
```

```java
// ❌ 如果父类没有无参构造，子类又不显式 super(参数)，编译报错：
class Dog extends Animal {
    public Dog() {
        // 隐式 super() 找不到父类无参构造 → 编译错误！
    }
}
```

### 3. `super()` 和 `this()` 互斥

- `super(...)` 和 `this(...)` **都必须在构造函数的第一行**，所以**不能同时出现**。
- 如果写了 `this(...)`，本构造函数就不会有 `super()` 了——因为被调用的那个构造函数里会负责调用 `super()`。

```java
class Dog extends Animal {
    public Dog() {
        this("默认");   // ✅ 调用 Dog(String)，本行后不再有 super()
    }

    public Dog(String name) {
        super("犬科");  // ✅ 这里调用父类构造
        System.out.println(name);
    }
}
```

### 4. super() 的三条硬性规则

| 规则 | 说明 |
|------|------|
| ① 只能用在**构造函数**中 | 普通方法里不能写 `super(...)` |
| ② 必须是构造函数的**第一行** | 否则编译报错 |
| ③ `super(...)` 与 `this(...)` **二选一** | 不能同时存在 |

## 六、super 到底指向什么？（原理澄清）

这是很多学习者误解的地方：

> ⚠️ **`super` 并不指向一个"独立的父类对象"。Java 中创建子类对象时，并不会额外创建一个父类对象。**

实际内存中只有**一个子类对象**，但它**同时包含了父类部分和子类部分**（继承使得子类对象内部拥有父类声明的字段）。`super` 引用的是这个对象中**"父类特征"的那部分**。

```
        内存中只有一个对象：Dog 实例
   ┌─────────────────────────────────┐
   │  ┌─────────────────────┐        │
   │  │  父类 Animal 部分    │  ← super 指向这部分
   │  │  - type              │
   │  │  - eat() 方法定义     │
   │  └─────────────────────┘        │
   │  ┌─────────────────────┐        │
   │  │  子类 Dog 部分       │  ← this 指向整个对象
   │  │  - name              │
   │  │  - eat() 重写后的定义 │
   │  └─────────────────────┘        │
   └─────────────────────────────────┘
```

- **`this`** 指向**当前整个对象**。
- **`super`** 指向当前对象中**父类特征的那部分**，用于绕过子类的"覆盖/隐藏"，访问父类版本。

> 💡 **结论**：不要把 super 理解成"父类对象的引用"，而要理解为"访问父类特征的引用"。它们指向的是同一个对象的不同视角。

## 七、构造器调用链

由于每个子类构造函数都会调用父类构造函数，会形成一条**从子类一直追溯到 Object 的调用链**：

```
new Dog()
   │
   ▼
Dog 构造函数 ──super()──→ Animal 构造函数 ──super()──→ Object 构造函数
   │                          │                          │
   │                          │                          │
   ▼                          ▼                          ▼
（先执行 Object 构造函数，再 Animal，最后 Dog）
```

执行顺序是**先父后子**：

```
① Object 构造函数
② Animal 构造函数
③ Dog 构造函数
```

> 💡 **记忆**：构造时**"先造爹再造儿"**——父类构造函数先执行完毕，保证子类能继承的父类部分已初始化好，然后才轮到子类自己的构造函数。这与 `super()` 必须在第一行相呼应：先让父类初始化完成，子类才能安全使用从父类继承来的成员。

## 八、super vs this 对比

| 对比维度 | `super` | `this` |
|---------|---------|--------|
| **含义** | 当前对象中**父类特征**的引用 | **当前对象**本身的引用 |
| **访问成员变量** | `super.变量`（父类的） | `this.变量`（本类的） |
| **调用方法** | `super.方法()`（父类的） | `this.方法()`（本类的，常省略） |
| **调用构造函数** | `super(...)`（父类构造，第一行） | `this(...)`（本类其他构造，第一行） |
| **能否单独使用** | ❌ 不能 `return super;` | ✅ 可以 `return this;`（链式编程） |
| **能否用在 static 方法** | ❌ | ❌（static 中无 this/super） |
| **指向的对象** | 同一个对象（父类视角） | 同一个对象（整体视角） |

## 九、使用场景总结

| 场景 | 用法 |
|------|------|
| 子类重写方法后，复用父类原逻辑 | `super.方法()` |
| 父子类同名变量，需访问父类的 | `super.变量` |
| 子类构造时初始化父类部分 | `super(参数)` |
| 父类无无参构造，子类必须初始化父类 | 显式 `super(参数)` |
| 框架/模板方法要求先调父类 | `super.方法()` 放在方法首行 |

## 十、注意事项与陷阱

1. **`super()` 只能在构造函数第一行** —— 普通方法中不能调用父类构造函数。
2. **`super(...)` 和 `this(...)` 互斥** —— 都要求第一行，只能存在一个。
3. **`super` 不是父类对象** —— 内存中没有独立的父类对象，super 是"父类特征的引用"。
4. **static 方法中不能用 super（也不能用 this）** —— static 属于类，不存在当前对象。
5. **父类 private 成员不能用 super 访问** —— private 对子类不可见，`super.privateMethod()` 编译报错。
6. **构造函数调用链是先父后子** —— 父类构造函数先执行完，才执行子类构造函数体。
7. **不要在构造函数中调用可被子类重写的方法** —— 此时子类尚未初始化完成，可能引发错误。

```java
class Base {
    public Base() {
        init();   // ⚠️ 危险：如果子类重写了 init()，会调用子类的 init()，
                  //         但此时子类构造函数还没执行，子类字段都是默认值！
    }
    public void init() {}
}
```

## 十一、面试常见问题

### Q1：super 关键字有哪几种用法？

三种：① `super.成员变量` 访问父类变量（区分父子同名）；② `super.方法()` 调用父类方法（常用于重写后复用父类逻辑）；③ `super(参数)` 在子类构造函数中调用父类构造函数，必须在第一行。

### Q2：super 和 this 有什么区别？

`this` 指向**当前对象本身**，`super` 指向当前对象中**父类特征的那部分**（不是独立的父类对象）。`this` 可以单独使用（如 `return this`），`super` 不能。`this(...)` 调用本类其他构造，`super(...)` 调用父类构造，两者都必须在构造函数第一行、且不能共存。

### Q3：为什么子类构造函数默认会调用父类的无参构造？

因为子类对象在初始化时，必须保证从父类继承来的那部分也先被正确初始化。所以子类构造函数第一行隐式调用 `super()`。如果父类没有无参构造，子类就必须显式 `super(参数)`。

### Q4：super() 和 this() 能同时出现吗？

**不能**。它们都要求出现在构造函数的**第一行**，所以一个构造函数里只能有一个。如果写了 `this(...)`，本构造函数就不含 `super()` 了，由被 `this()` 调用的那个构造函数负责调用 `super()`。

### Q5：super 指向的是一个父类对象吗？

**不是**。创建子类对象时，内存中只有一个对象（包含父类部分和子类部分），不会单独创建父类对象。`super` 是访问该对象中"父类特征"的引用，用于绕过子类的覆盖/隐藏。`this` 和 `super` 指向的是同一个对象的不同视角。

### Q6：父类的 private 方法能用 super 调用吗？

**不能**。private 方法对子类不可见，子类无法重写、也无法用 `super` 访问，`super.privateMethod()` 会编译报错。

### Q7：构造函数的执行顺序是怎样的？

**先父后子**：从子类构造函数出发，沿 `super()` 调用链一直追溯到 Object，然后**自顶向下**依次执行各层构造函数（Object → 父类 → 子类）。这保证子类构造时，父类部分已初始化完毕。

### Q8：子类构造函数中 super() 不在第一行会怎样？

**编译报错**。Java 语法强制要求 `super(...)` 和 `this(...)` 必须是构造函数的第一条语句，目的是保证父类先于子类初始化。

## 十二、总结

| 知识点 | 核心要点 |
|--------|---------|
| 三种用法 | `super.变量` / `super.方法()` / `super(参数)` |
| super() 规则 | 只在构造函数、必须第一行、与 this() 互斥 |
| 本质 | 父类特征的引用，非父类对象 |
| 调用链 | 先父后子：Object → ... → 父类 → 子类 |
| 与 this 对比 | this 指整体，super 指父类部分；this 可单独用，super 不可 |
| static 中 | 都不能用 super / this |

**记住三句话**：

1. **`super.变量 / super.方法()` 访问父类成员，`super(参数)` 调用父类构造（必须第一行）。**
2. **构造顺序"先父后子"，super() 让父类先初始化完，子类才安全。**
3. **super 不是父类对象，而是当前对象中"父类特征"的引用。**
