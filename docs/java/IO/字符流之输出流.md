## 一、概述

上一篇我们学习了**字符输入流（Reader）**，解决了字节流读中文乱码的问题。与之对称，**字符输出流（Writer）** 负责以**字符**为单位把数据写到文本文件，并在底层按指定编码把字符转换成字节，天然适合写**纯文本**（尤其是含中文的文本）。

- **字符输入流（Reader）**：读文本，自动解码（字节 → 字符）。
- **字符输出流（Writer）**：写文本，自动编码（字符 → 字节）。

> 💡 **提示：** 写入文本时，**编码一定要和将来读取时的编码一致**（推荐统一 UTF-8），否则跨平台或跨程序读取会出现乱码。

---

## 二、核心概念与体系结构

### 1. Writer 是所有字符输出流的抽象基类

`java.io.Writer` 是抽象类，定义了字符输出流最核心的 `write()`、`append()`、`flush()` 方法。所有以字符为单位「写」数据的流都继承自它。

```
                          ┌─────────┐
            抽象基类        │ Writer  │   (定义 write / append / flush / close)
                          └────┬────┘
                               │
        ┌──────────────────────┼────────────────────────┐
        │                      │                        │
┌───────────────────┐  ┌──────────────────┐   ┌──────────────────┐
│OutputStreamWriter │  │  BufferedWriter  │   │   PrintWriter    │
│(字符→字节的桥梁)  │  │ (缓冲 + newLine) │   │(按行打印各种类型)│
└────────┬──────────┘  └────────▲─────────┘   └────────▲─────────┘
         │                      │                      │
         │ 子类(便捷类)         │ (装饰任意 Writer)     │ (可包装 Writer)
         │                      │                      │
┌────────┴──────────┐   BufferedWriter(       PrintWriter(
│    FileWriter     │     new FileWriter(f))    new FileWriter(f))
│ (使用默认编码)    │
└───────────────────┘
```

常见子类对比：

| 流类 | 作用 | 能否指定编码 |
| --- | --- | --- |
| `OutputStreamWriter` | **字符流 → 字节流的桥梁**，核心类 | ✅ 可指定 |
| `FileWriter` | `OutputStreamWriter` 的便捷子类 | ❌ 用默认编码 |
| `BufferedWriter` | 装饰器，提供缓冲 + `newLine()` 跨平台换行 | 取决于被包装的流 |
| `PrintWriter` | 按 `print/println` 打印各种类型，支持自动刷新 | ✅ 可指定 |

### 2. 字节流 vs 字符流输出对比

| 对比项 | 字节输出流 | 字符输出流 |
| --- | --- | --- |
| 最小单位 | 字节（byte） | 字符（char） |
| 抽象基类 | `OutputStream` | `Writer` |
| 写中文 | 需手动 `getBytes(编码)` | ✅ 自动按编码转换 |
| 核心方法 | `write(int/byte[])` | `write(int/char[]/String)`、`append` |
| 适用文件 | 任意文件 | **仅文本文件** |

## 三、FileWriter 便捷类

`FileWriter` 是 `OutputStreamWriter` 的子类，**使用系统默认编码**写入文本文件。构造简单，适合快速写「编码与系统默认一致」的文本。

### 1. 构造方法

| 构造方法 | 写入模式 |
| --- | --- |
| `FileWriter(String fileName)` | **覆盖** |
| `FileWriter(File file)` | **覆盖** |
| `FileWriter(String fileName, boolean append)` | `append=true` 追加 |
| `FileWriter(File file, boolean append)` | `append=true` 追加 |

### 2. 使用示例

```java
import java.io.FileWriter;
import java.io.IOException;

public class FileWriterDemo {
    public static void main(String[] args) {
        // FileWriter 使用系统默认编码；第二个参数 true 表示追加
        try (FileWriter fw = new FileWriter("/Users/demo/a.txt", true)) {
            fw.write("你好，字符流！"); // 直接写字符串，无需手动编码
            fw.write("\n");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> ⚠️ **注意：** 与 `FileReader` 一样，`FileWriter` **不能指定编码**，始终用 JVM 默认编码。跨平台写中文时，推荐用 `OutputStreamWriter` 显式指定编码。

## 四、OutputStreamWriter：字符流到字节流的桥梁（重点）

`OutputStreamWriter` 是字符输出流体系中最关键的类——它把一个**字符输出流**「翻译」成**字节输出流**写到文件，并在构造时**指定编码**。你写的每一个字符，都会按指定编码转换成字节落盘。

### 1. 工作原理

```
程序写字符串 "你好"
      │
      ▼
OutputStreamWriter (指定编码)
      │ 按 UTF-8 把字符编码成字节
      ▼
字节流 (FileOutputStream) ──► 文件
  [E4 BD A0] [E5 A5 BD]   ← "你好" 的 UTF-8 字节序列
```

### 2. 构造方法

| 构造方法 | 说明 |
| --- | --- |
| `OutputStreamWriter(OutputStream out)` | 使用默认编码 |
| `OutputStreamWriter(OutputStream out, String charsetName)` | **指定编码名称**（如 `"GBK"`、`"UTF-8"`） |
| `OutputStreamWriter(OutputStream out, Charset cs)` | 指定 `Charset` 对象（推荐） |

### 3. 使用示例（指定 GBK 编码写入）

```java
import java.io.*;
import java.nio.charset.Charset;

public class OutputStreamWriterDemo {
    public static void main(String[] args) {
        // 先建字节流，再用 OutputStreamWriter 包装并指定 GBK 编码
        try (OutputStreamWriter osw = new OutputStreamWriter(
                new FileOutputStream("/Users/demo/gbk.txt"),
                Charset.forName("GBK"))) { // ✅ 指定 GBK 编码

            osw.write("你好，GBK 编码的文本！");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> 💡 **提示：** 写入时指定的编码，必须与将来**读取时使用的编码一致**。如果用 GBK 写，读取时也必须用 GBK（如 `new InputStreamReader(fis, "GBK")`），否则乱码。日常推荐统一用 **UTF-8**。

## 五、BufferedWriter：缓冲 + 跨平台换行

`BufferedWriter` 是 `Writer` 的装饰器，提供两大能力：
1. **缓冲**：内部维护缓冲区，减少磁盘写入次数，提升写入效率。
2. **`newLine()`**：写入一个**跨平台的换行符**（Windows 是 `\r\n`，Linux/Mac 是 `\n`），由 JVM 根据系统自动选择。

### 1. 核心方法

| 方法 | 说明 |
| --- | --- |
| `void newLine()` | 写入一个跨平台换行符 |
| `void write(int c)` | 写入单个字符 |
| `void write(String s, int off, int len)` | 写入字符串的一部分 |
| `void flush()` | 刷新缓冲区 |

> 💡 **提示：** 写换行时优先用 `newLine()` 而非硬编码 `"\n"`，这样代码在任何操作系统上都能正确换行。

### 2. 使用示例：按行写入 UTF-8 文本

```java
import java.io.*;
import java.nio.charset.StandardCharsets;

public class BufferedWriterDemo {
    public static void main(String[] args) {
        // 经典组合：字节流 + OutputStreamWriter(指定编码) + BufferedWriter(缓冲)
        try (BufferedWriter bw = new BufferedWriter(
                new OutputStreamWriter(
                        new FileOutputStream("/Users/demo/output.txt"),
                        StandardCharsets.UTF_8))) {

            bw.write("第一行：学习 Java 字符流");
            bw.newLine(); // 跨平台换行
            bw.write("第二行：写入 UTF-8 文本");
            bw.newLine();
            bw.write("第三行：结束");
            // close() 前会自动 flush
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 六、PrintWriter：按行打印各种类型

`PrintWriter` 提供类似 `System.out` 的 `print` / `println` 方法，可以方便地打印**各种类型**（String、int、double、Object 等），并且支持**自动刷新**（auto flush）。

### 1. 常用方法

| 方法 | 说明 |
| --- | --- |
| `void print(String s)` | 打印字符串（不换行） |
| `void println(String s)` | 打印字符串并换行 |
| `void println(int / double / Object x)` | 打印各种类型并换行 |
| `void printf(String format, Object... args)` | 格式化打印 |

### 2. 使用示例

```java
import java.io.*;
import java.nio.charset.StandardCharsets;

public class PrintWriterDemo {
    public static void main(String[] args) {
        // PrintWriter 第二个参数 true 表示开启自动 flush（println 时刷新）
        try (PrintWriter pw = new PrintWriter(
                new OutputStreamWriter(
                        new FileOutputStream("/Users/demo/log.txt"),
                        StandardCharsets.UTF_8),
                true)) { // autoFlush = true

            pw.println("姓名：张三");
            pw.println("年龄：" + 20);
            pw.printf("成绩：%.2f\n", 95.5);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> 💡 **提示：** `PrintWriter` 的方法**不抛 `IOException`**（内部吞掉异常，可用 `checkError()` 检查是否出错），写日志、生成报告时很方便。但缺点是异常被隐藏，关键业务场景要注意。

## 七、核心方法与 flush() 的重要性

### 1. Writer 常用方法汇总

| 方法签名 | 说明 |
| --- | --- |
| `void write(int c)` | 写入**一个字符**（低 16 位） |
| `void write(char[] cbuf)` | 写入整个字符数组 |
| `void write(char[] cbuf, int off, int len)` | 写入字符数组的一部分 |
| `void write(String str)` | 写入字符串 |
| `void write(String str, int off, int len)` | 写入字符串的一部分 |
| `Writer append(CharSequence csq)` | 追加字符序列（返回 this，可链式调用） |
| `void flush()` | 刷新缓冲区，把数据真正写出 |
| `void close()` | 先 flush 再关闭 |

### 2. flush() 的重要性

与字节输出流一样，`BufferedWriter` 等带缓冲的字符流，`write()` 先写到内存缓冲区，**只有缓冲区满、调用 `flush()` 或 `close()` 时才真正落盘**。如果程序异常退出且没有 flush/close，数据会**丢失**。

> ⚠️ **注意：** 本系列所有示例都用 try-with-resources，`close()` 会自动触发 `flush()`，保证数据落盘。如果需要在关闭前就让数据可见（如日志实时写入），应显式调用 `flush()`。

## 八、编码问题详解

### 1. 写入与读取编码必须一致

```
写入：字符 "你" ──UTF-8 编码──► [E4 BD A0] 三字节 ──► 文件
读取：文件 ──► [E4 BD A0] ──UTF-8 解码──► 字符 "你"   ✅ 一致，正常
读取：文件 ──► [E4 BD A0] ──GBK  解码──► 乱码 "浣犲ソ"  ❌ 不一致，乱码
```

### 2. 推荐做法

| 做法 | 说明 |
| --- | --- |
| 统一使用 **UTF-8** | 业界标准，跨平台、跨语言通用 |
| 用 `StandardCharsets.UTF_8` 常量 | 编译期检查，避免拼写错误 |
| 显式指定，不依赖默认编码 | `FileWriter` / `FileReader` 默认编码受系统影响，慎用 |

```java
// ❌ 依赖系统默认编码，跨平台易乱码
new FileWriter("a.txt");

// ✅ 显式指定 UTF-8 编码
new OutputStreamWriter(new FileOutputStream("a.txt"), StandardCharsets.UTF_8);
```

## 九、完整示例：逐行写入文本文件

```java
import java.io.*;
import java.nio.charset.StandardCharsets;
import java.util.Arrays;
import java.util.List;

public class WriteLinesDemo {
    public static void main(String[] args) {
        List<String> lines = Arrays.asList(
                "Java IO 学习笔记",
                "1. 字节流：InputStream / OutputStream",
                "2. 字符流：Reader / Writer",
                "3. 资源关闭：try-with-resources"
        );

        // 标准组合：OutputStreamWriter(UTF-8) + BufferedWriter
        try (BufferedWriter bw = new BufferedWriter(
                new OutputStreamWriter(
                        new FileOutputStream("/Users/demo/notes.txt"),
                        StandardCharsets.UTF_8))) {

            for (String line : lines) {
                bw.write(line); // 写一行
                bw.newLine();   // 跨平台换行
            }
            bw.flush(); // 显式刷新（可选，close 时也会刷新）
            System.out.println("写入完成");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 十、注意事项与最佳实践

1. **文本用字符流，二进制用字节流**：图片、视频等绝不能用字符流。
2. **优先 `OutputStreamWriter` 而非 `FileWriter`**：前者可指定编码，跨平台可靠。
3. **写换行用 `newLine()`**：跨平台兼容，不要硬编码 `"\n"` 或 `"\r\n"`。
4. **flush/close 必不可少**：缓冲流场景下不关闭数据会丢失。
5. **统一 try-with-resources** 关闭资源。

```java
// ✅ 写入文本文件的标准模板
try (BufferedWriter bw = new BufferedWriter(
        new OutputStreamWriter(
                new FileOutputStream("目标文件", true), // true=追加
                StandardCharsets.UTF_8))) {
    bw.write("要写入的内容");
    bw.newLine();
} catch (IOException e) {
    e.printStackTrace();
}
```

## 十一、面试常见问题

### Q1：`FileWriter` 和 `OutputStreamWriter` 有什么区别？

`FileWriter` 是 `OutputStreamWriter` 的便捷子类，**只能用系统默认编码**，不能指定编码；`OutputStreamWriter` 可以在构造时指定任意编码（如 UTF-8、GBK），跨平台更可靠。**生产环境推荐 `OutputStreamWriter`**。

### Q2：`BufferedWriter` 的 `newLine()` 有什么用？

`newLine()` 写入一个**跨平台换行符**——Windows 上是 `\r\n`，Linux/Mac 上是 `\n`，由 JVM 根据当前操作系统自动选择。这样代码在任何系统上都能正确换行，比硬编码换行符更可靠。

### Q3：写完文本不调用 `flush()` 或 `close()` 会怎样？

如果是 `BufferedWriter` 等带缓冲的流，数据只写在内存缓冲区，**没有真正落盘**，程序结束后这部分数据会**丢失**。`close()` 内部会自动 `flush()`，所以用 try-with-resources 正常关闭即可保证数据落盘。

### Q4：`PrintWriter` 和 `BufferedWriter` 怎么选？

- `BufferedWriter`：专注于**缓冲 + 换行**，方法抛 `IOException`，适合严谨的文件写入。
- `PrintWriter`：提供 `print/println/printf` 打印**各种类型**，方法不抛 `IOException`，支持自动刷新，适合**写日志、生成报告**等便捷场景。

### Q5：写入中文出现乱码怎么解决？

确保**写入编码与读取编码一致**。最佳实践是用 `OutputStreamWriter` 显式指定编码（推荐 `StandardCharsets.UTF_8`），不要用依赖默认编码的 `FileWriter`，也不要混用不同编码。

### Q6：字符流能写图片吗？

**不能**。字符流在「字符 → 字节」编码转换时会改变数据，而图片是二进制数据，转换后内容损坏无法还原。写二进制文件必须用**字节流**（`FileOutputStream`）。

---

## 十二、总结

- **字符输出流**为写文本而生，自动按编码把字符转换成字节。
- **`Writer`** 是所有字符输出流的抽象基类。
- **`OutputStreamWriter`** 是「字符流 → 字节流」的桥梁，**可指定编码**，是核心类。
- **`FileWriter`** 是便捷类，但**不能指定编码**，跨平台慎用。
- **`BufferedWriter`** 提供缓冲 + `newLine()` 跨平台换行，是写文本的**性能首选**。
- **`PrintWriter`** 提供 `print/println` 打印各种类型，适合写日志。
- **编码读写一致**：推荐统一 UTF-8，用 `StandardCharsets.UTF_8` 常量。
- **资源关闭统一使用 try-with-resources**，与整个 IO 流系列保持一致。
