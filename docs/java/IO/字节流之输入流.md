## 一、概述

在 Java 中，程序与外部数据（文件、网络、键盘等）之间的数据传输称为 **输入 / 输出（Input / Output，简称 IO）**。Java 用「流（Stream）」这个抽象概念来描述一串连续流动的数据。

- **输入流（InputStream / Reader）**：数据从外部「流入」程序，即**读（read）**操作。
- **输出流（OutputStream / Writer）**：数据从程序「流出」到外部，即**写（write）**操作。

按照**处理数据的最小单位**，流又分为两大类：

| 类型 | 最小单位 | 抽象基类 | 适用场景 |
| --- | --- | --- | --- |
| 字节流 | 1 个字节（byte，8 bit） | `InputStream` / `OutputStream` | 任意文件（图片、视频、音频、压缩包等二进制文件） |
| 字符流 | 1 个字符（char，Java 中占 2 字节） | `Reader` / `Writer` | 纯文本文件（txt、java、html 等） |

> 💡 **提示：** 字节流是「万能流」，能读写任意文件；字符流专门为「文本」设计，能正确处理编码、避免中文乱码。

本文聚焦 **字节输入流（InputStream）**：从外部以「字节」为单位读取数据到程序中。

## 二、核心概念与体系结构

### 1. InputStream 是所有字节输入流的抽象基类

`java.io.InputStream` 是一个抽象类，定义了字节输入流最核心的 `read()` 方法。所有以字节为单位「读」数据的流都继承自它。

```
                        ┌──────────────┐
            抽象基类      │ InputStream  │   (定义 read / available / close)
                        └──────┬───────┘
                               │
        ┌──────────────────────┼───────────────────────────────┐
        │                      │                               │
┌────────────────┐   ┌──────────────────┐            ┌────────────────────┐
│ FileInputStream│   │BufferedInputStream│           │ ObjectInputStream │
│ (从文件读字节) │   │ (带缓冲的装饰器)  │            │ (读对象-反序列化) │
└────────────────┘   └──────────────────┘            └────────────────────┘
        │                      │
        │  (作为被装饰对象)     │ (装饰任意 InputStream)
        │                      │
        └────────────► BufferedInputStream(new FileInputStream(file))
                               │
                     ┌─────────┴──────────┐
                     │ByteArrayInputStream │  (从内存中的 byte[] 读)
                     └────────────────────┘
```

常见子类对比：

| 流类 | 数据来源 | 特点 |
| --- | --- | --- |
| `FileInputStream` | 磁盘文件 | 最常用，按字节从文件读取 |
| `BufferedInputStream` | 装饰其它输入流 | 自带 8KB 缓冲区，大幅减少磁盘 IO |
| `ObjectInputStream` | 任意输入流 | 反序列化，把字节还原成 Java 对象 |
| `ByteArrayInputStream` | 内存中的 `byte[]` | 把一个数组当流来读，常用于测试 |

### 2. 装饰器模式（Decorator Pattern）

IO 流体系大量使用了**装饰器模式**：`BufferedInputStream` 本身也是一个 `InputStream`，但它「包裹」住另一个 `InputStream`（如 `FileInputStream`），在不改变原类的前提下为其增加「缓冲」能力。

```java
// FileInputStream 负责真正从磁盘读字节
// BufferedInputStream 负责在内存中做缓冲，减少磁盘访问次数
InputStream in = new BufferedInputStream(new FileInputStream("a.txt"));
```

> 💡 **提示：** 装饰器模式让你可以「自由组合」功能，例如 `BufferedInputStream + FileInputStream + 其它流` 层层嵌套，灵活强大。

## 三、FileInputStream 详解

`FileInputStream` 是最基础的字节文件输入流，从文件系统中某个文件按字节读取数据。

### 1. 构造方法

| 构造方法 | 说明 |
| --- | --- |
| `FileInputStream(String name)` | 通过文件**路径字符串**构造 |
| `FileInputStream(File file)` | 通过 `File` 对象构造 |
| `FileInputStream(FileDescriptor fdObj)` | 通过文件描述符构造（少用） |

```java
// 方式一：直接传路径字符串
FileInputStream fis1 = new FileInputStream("/Users/demo/a.txt");

// 方式二：传 File 对象（推荐，可先判断文件是否存在）
File file = new File("/Users/demo/a.txt");
if (file.exists()) {
    FileInputStream fis2 = new FileInputStream(file);
}
```

> ⚠️ **注意：** 如果文件不存在，构造方法会抛出 `FileNotFoundException`（`IOException` 的子类），必须处理（`try-catch` 或 `throws`）。

### 2. 核心方法

| 方法签名 | 说明 |
| --- | --- |
| `int read()` | 读取**一个字节**，返回 0~255 的 int；到达文件末尾返回 **-1** |
| `int read(byte[] b)` | 读取若干字节存入数组 `b`，返回**实际读取的字节数**；末尾返回 -1 |
| `int read(byte[] b, int off, int len)` | 读取最多 `len` 个字节，存入 `b` 从 `off` 开始的位置 |
| `int available()` | 返回「在不阻塞的情况下」还能读到的近似字节数 |
| `void close()` | 关闭流并释放系统资源（**必须调用**） |

> 💡 **关键理解：** `read()` 返回 `int` 而不是 `byte`。原因是 `byte` 是有符号的（-128~127），无法表示 0~255 全部值，也无法表示「文件末尾 -1」。用 `int` 后，0~255 表示真正的字节值，-1 表示「读完了」。

## 四、读取文件的标准流程

读取文件的核心套路是「**打开 → 循环读 → 处理 → 关闭**」：

```
  打开流 (new FileInputStream)  ──►  循环 read()
                                          │
                            ┌─────────────┴─────────────┐
                            │                           │
                       返回 -1？                     返回数据
                            │                           │
                       关闭 close()                处理数据，继续读
```

### 1. 一次读一个字节（低效，仅演示原理）

```java
import java.io.FileInputStream;
import java.io.IOException;

public class ReadOneByteDemo {
    public static void main(String[] args) {
        // try-with-resources：自动关闭实现了 AutoCloseable 的流（Java 7+）
        try (FileInputStream fis = new FileInputStream("/Users/demo/a.txt")) {
            int b; // 存放每次读取到的字节
            // read() 返回 -1 表示读到文件末尾
            while ((b = fis.read()) != -1) {
                System.out.print((char) b); // 强转为字符输出（仅对英文可靠）
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> ⚠️ **注意：** 每读 1 个字节就调用一次 `read()`，意味着每次都可能发生一次磁盘 IO，性能很差。**生产环境绝不这样用**，下面用缓冲数组优化。

### 2. 一次读一批字节（推荐写法）

```java
import java.io.FileInputStream;
import java.io.IOException;

public class ReadBatchDemo {
    public static void main(String[] args) {
        // 定义缓冲数组（通常 1024 或 8192 字节）
        byte[] buffer = new byte[1024];
        try (FileInputStream fis = new FileInputStream("/Users/demo/a.txt")) {
            int len; // 每次实际读到的字节数
            // read(buffer) 一次性读入最多 buffer.length 个字节，返回实际读取数
            while ((len = fis.read(buffer)) != -1) {
                // 只转换实际读取的 len 个字节，避免末尾脏数据
                String text = new String(buffer, 0, len);
                System.out.print(text);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> 💡 **提示：** 用 `new String(buffer, 0, len)` 而不是 `new String(buffer)`，因为最后一次读取往往不足 1024 字节，多余位置是旧数据（脏数据）。

### 3. 性能对比

| 方式 | 每次读取 | 1MB 文件大约磁盘 IO 次数 | 性能 |
| --- | --- | --- | --- |
| `read()` 逐字节 | 1 字节 | 约 100 万次 | 极慢 |
| `read(byte[])` 批量 | 1024 字节 | 约 1000 次 | 快 |
| `BufferedInputStream` | 8192 字节缓冲 | 极少 | 最快 |

## 五、中文乱码问题

字节流按「字节」读写，而**中文在 UTF-8 编码下占 3 个字节**（GBK 占 2 个字节）。如果用字节流逐字节读取，一个中文字符会被拆成多个字节，强行 `char` 转换必然乱码。

```java
// ❌ 用字节流读中文文本，极容易乱码
try (FileInputStream fis = new FileInputStream("中文.txt")) {
    int b;
    while ((b = fis.read()) != -1) {
        System.out.print((char) b); // 中文字符被拆开，输出乱码
    }
}
```

> ⚠️ **注意：** 凡是**纯文本文件（尤其是含中文）**，请使用 **字符流（Reader / Writer）**，它能按「字符 + 编码」正确解析。字节流只适合二进制文件（图片、视频、音频、压缩包）。

## 六、BufferedInputStream 缓冲字节流

`BufferedInputStream` 是 `InputStream` 的装饰器，内部维护一个**默认 8192 字节（8KB）的缓冲区**。读取时一次性从磁盘读一大块数据到缓冲区，后续 `read()` 直接从内存缓冲区取，极大减少磁盘 IO 次数。

### 1. 工作原理

```
磁盘文件 ──一次性读 8KB──►  BufferedInputStream 内部缓冲区
                                       │
                            后续 read() 从内存取（极快）
                                       │
                                   返回给程序
```

### 2. 使用示例

```java
import java.io.*;

public class BufferedInputDemo {
    public static void main(String[] args) {
        // 用 BufferedInputStream 包装 FileInputStream
        try (BufferedInputStream bis = new BufferedInputStream(
                new FileInputStream("/Users/demo/a.txt"))) {

            byte[] buffer = new byte[1024];
            int len;
            while ((len = bis.read(buffer)) != -1) {
                System.out.print(new String(buffer, 0, len));
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> 💡 **提示：** `BufferedInputStream` 实现了 `AutoCloseable`，关闭外层 `bis` 时会自动级联关闭内部的 `FileInputStream`，因此 try-with-resources 中只需声明最外层流。

### 3. 何时用字节输入流

| 场景 | 推荐 |
| --- | --- |
| 读取图片 / 视频 / 音频 / 压缩包等二进制文件 | ✅ 字节流（`FileInputStream` + `BufferedInputStream`） |
| 读取纯文本（txt / 代码 / 配置文件） | ❌ 用字符流（`BufferedReader`） |
| 网络传输原始字节 | ✅ `InputStream`（Socket 返回的流） |
| 复制任意文件 | ✅ 字节流（字节流是万能流） |

## 七、注意事项与最佳实践

1. **必须关闭资源**：流占用操作系统文件句柄，不关闭会导致句柄泄漏、文件被锁定。统一使用 **try-with-resources**。
2. **不要忽略返回值 -1**：循环条件用 `!= -1` 判断文件末尾，避免死循环。
3. **批量读取优于单字节**：始终用 `read(byte[])`，配合缓冲数组。
4. **文本用字符流**：含中文的文本不要用字节流。
5. **缓冲流优先**：文件 IO 优先 `BufferedInputStream` 包装。

```java
// ✅ 标准模板：字节输入流 + 缓冲 + try-with-resources
try (BufferedInputStream bis = new BufferedInputStream(
        new FileInputStream("目标文件"))) {
    byte[] buffer = new byte[8192];
    int len;
    while ((len = bis.read(buffer)) != -1) {
        // 处理 buffer 的 0~len 字节
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

## 八、面试常见问题

### Q1：`read()` 方法为什么返回 `int` 而不是 `byte`？

`byte` 是有符号的（范围 -128~127），既无法表示 0~255 的全部字节值，也无法用一个特殊值表示「文件末尾」。用 `int` 返回，0~255 表示真实字节，**-1 表示读到末尾**，避免了二义性。

### Q2：`FileInputStream` 和 `BufferedInputStream` 有什么区别？

`FileInputStream` 每次调用 `read()` 都可能直接访问磁盘，效率低；`BufferedInputStream` 用装饰器模式在内部维护 8KB 缓冲区，先一次性读一大块到内存，再从内存逐字节返回，大幅减少磁盘 IO。**实际开发中文件读取几乎都用 `BufferedInputStream` 包装。**

### Q3：字节流能读中文吗？

技术上能读（读到的是字节），但**按字节逐个处理中文会乱码**，因为一个中文在 UTF-8 下占 3 个字节。正确做法是使用**字符流**（`Reader`），它内部自动按编码把字节组合成字符。

### Q4：什么是装饰器模式？IO 流中怎么体现？

装饰器模式指在不改变原有类、且不使用继承的前提下，通过「包装」一个对象来扩展功能。`BufferedInputStream`「包装」`FileInputStream`，为其增加缓冲能力，但对外仍是一个 `InputStream`，调用方式完全一致，这就是装饰器模式的典型应用。

### Q5：`available()` 方法可靠吗？

`available()` 返回的是「当前不阻塞还能读到的近似字节数」。对普通文件较可靠，但对网络流等不一定准确，**不要用它来判断流是否结束**，判断结束只能用 `read() == -1`。

## 九、总结

- **`InputStream`** 是所有字节输入流的抽象基类，核心方法是 `read()`。
- **`FileInputStream`** 从文件按字节读取，常用 `read(byte[])` 批量读取提升效率。
- **`BufferedInputStream`** 是装饰器，提供 8KB 缓冲，是文件读取的性能首选。
- **读文件标准流程**：打开流 → 循环 `read`（判断 -1）→ 处理 → 关闭流。
- **资源关闭**：统一使用 **try-with-resources**（Java 7+），避免泄漏。
- **文本（尤其含中文）请用字符流**，字节流只适合二进制文件。
