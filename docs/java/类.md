# 类（Class）

> 官方文档：[Classes (Java SE 21)](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Object.html) ｜ 菜鸟教程：[Java 类和对象](https://www.runoob.com/java/java-object-classes.html)

## 一、概述

在面向对象编程（OOP）中，**类（Class）** 是 Java 语言最核心的基本单元。可以把类理解为一张"设计图纸"或"模具"，它定义了某一类事物共同具备的**特征（属性）** 和**行为（方法）**。而根据这张图纸实际"生产"出来的具体个体，就是**对象（Object）**。

举个通俗的类比：
- **类** 就像是"汽车的设计图纸"，它规定了每辆汽车都应该有颜色、品牌、速度等属性，以及启动、刹车、加速等行为，但图纸本身不是一辆真正的车。
- **对象** 就是根据图纸造出来的"一辆具体的车"，比如"一辆红色的特斯拉 Model 3"，它有具体的属性值。

Java 中一切代码都必须写在类里面（这一点和 C 语言、JavaScript 不同），这也体现了 Java "万物皆对象"的设计哲学。

## 二、核心概念

### 2.1 类与对象的关系

```
┌──────────────────────────┐         new          ┌──────────────────────────┐
│        类 (Class)         │  ──────────────────▶ │      对象 (Object)        │
│   "模板 / 蓝图 / 图纸"    │                      │   "实例 / 具体的实体"     │
├──────────────────────────┤                      ├──────────────────────────┤
│  属性的定义（类型 名字）  │                      │  属性的具体值             │
│  方法的定义（无具体值）   │                      │  方法可被调用             │
└──────────────────────────┘                      └──────────────────────────┘
        仅有一份                                          可创建无数个
```

- **类是抽象的**：描述"有什么"，但不存储具体数据。
- **对象是具体的**：每个对象在内存中独立存在，拥有自己的属性值。
- **一个类可以创建多个对象**：这些对象彼此独立，互不影响。

### 2.2 对象在内存中的位置

- **类本身**（的字节码）存放在 **方法区（Method Area）** 中。
- **对象实例** 存放在 **堆内存（Heap）** 中。
- **引用变量**（指向对象的变量）存放在 **栈内存（Stack）** 中。

```
┌──────── Stack（栈） ────────┐        ┌──────── Heap（堆） ──────────┐
│  Student s = ───────────────┼───────▶│  ┌──────────────────────┐   │
│            (引用地址 0x01)   │        │  │  Student 对象 0x01    │   │
└─────────────────────────────┘        │  │   name = "张三"        │   │
                                       │  │   age  = 18           │   │
                                       │  └──────────────────────┘   │
                                       └──────────────────────────────┘
```

> 💡 **提示：** `s` 这个变量本身只是一个"遥控器"，它保存的是对象在堆内存中的地址。真正存数据的是堆里的对象。

## 三、类的定义语法

### 3.1 基本语法

```java
[修饰符] class 类名 {
    // 1. 成员变量（属性 / 字段）
    // 2. 构造方法（构造器）
    // 3. 成员方法（行为）
    // 4. 代码块
    // 5. 内部类
}
```

### 3.2 类的五大组成部分

| 组成部分 | 作用 | 示例 |
|---------|------|------|
| **成员变量（Field）** | 描述对象的属性/状态 | `String name;` |
| **成员方法（Method）** | 描述对象的行为/功能 | `void study() { ... }` |
| **构造方法（Constructor）** | 创建对象时初始化对象 | `Student(...) { ... }` |
| **代码块（Block）** | 每次创建对象时执行的初始化逻辑 | `{ ... }` |
| **内部类（Inner Class）** | 定义在类内部的类 | `class Inner { ... }` |

下面用一个最简单的类来展示：

```java
// 定义一个学生类
public class Student {

    // 1. 成员变量（属性）
    String name;
    int age;

    // 2. 构造方法（无参）
    public Student() {
        System.out.println("无参构造器被调用");
    }

    // 2. 构造方法（有参，重载）
    public Student(String name, int age) {
        this.name = name;   // this.name 表示成员变量，name 表示参数
        this.age = age;
    }

    // 3. 成员方法
    public void study() {
        System.out.println(name + " 正在学习");
    }
}
```

### 3.3 创建对象（new 关键字）

```java
public class Test {
    public static void main(String[] args) {
        // 使用 new 关键字创建对象
        Student s1 = new Student();               // 调用无参构造
        s1.name = "张三";
        s1.age = 18;
        s1.study();                               // 输出：张三 正在学习

        // 创建另一个对象，互不影响
        Student s2 = new Student("李四", 20);     // 调用有参构造
        s2.study();                               // 输出：李四 正在学习
    }
}
```

`new` 关键字做了三件事：
1. 在堆内存中开辟空间，创建对象。
2. 调用对应的构造方法，初始化对象。
3. 返回对象在堆内存中的地址，赋值给引用变量。

## 四、this 关键字

`this` 代表**当前对象的引用**（谁调用该方法，`this` 就指向谁）。它主要有三种用法。

### 4.1 区分成员变量和局部变量

当方法的参数名与成员变量同名时，必须用 `this` 来指明成员变量，否则会发生"就近原则"导致赋值失败。

```java
public class Student {
    String name;
    int age;

    // ❌ 错误写法：参数名和成员变量同名，赋值给了自己
    public void setName(String name) {
        name = name;   // 这里的 name 全是参数，成员变量没被赋值
    }

    // ✅ 正确写法：用 this 指明成员变量
    public void setName2(String name) {
        this.name = name;   // this.name 是成员变量，name 是参数
    }
}
```

### 4.2 this.方法名() 调用本类其他方法

```java
public void study() {
    this.rest();     // 调用本类的 rest() 方法，this 可省略
}
public void rest() {
    System.out.println("休息一下");
}
```

### 4.3 this() 调用本类其他构造方法

`this()` 必须放在构造方法的第一行，用于构造器之间的复用。

```java
public class Student {
    String name;
    int age;

    public Student() {
        this("默认姓名", 18);   // 调用下面的有参构造，必须放第一行
        System.out.println("无参构造");
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

> ⚠️ **注意：** `this()` 和 `super()` 不能同时出现在一个构造方法中，因为它们都要求必须放在第一行。

## 五、构造方法（Constructor）

构造方法也叫构造器，是**专门用来创建对象并初始化对象**的特殊方法。

### 5.1 构造方法的特点

| 特性 | 说明 |
|------|------|
| **方法名** | 必须与类名**完全相同**（包括大小写） |
| **返回值** | **没有返回值类型**，连 `void` 都不能写 |
| **调用时机** | 在 `new` 创建对象时由 JVM 自动调用，且只调用一次 |
| **修饰符** | 可以是 public、protected、默认、private |

### 5.2 构造方法与普通方法的区别

| 对比项 | 构造方法 | 普通方法 |
|--------|---------|---------|
| 方法名 | 必须与类名相同 | 自定义 |
| 返回值 | 无（不能写 void） | 必须声明返回值类型（含 void） |
| 调用方式 | `new` 时自动调用 | 对象.方法名() 手动调用 |
| 调用次数 | 每个对象仅调用一次 | 可多次调用 |
| 作用 | 创建并初始化对象 | 描述对象的行为 |

### 5.3 默认构造器的生成规则

> 💡 **提示：** 如果一个类中**没有显式定义任何构造方法**，编译器会自动生成一个 **public 的无参构造方法**。但**只要你写了任何一个构造方法**，编译器就**不再**自动生成无参构造器了。

```java
public class Student {
    String name;
    // 此时编译器会自动生成：public Student() {}
}

public class Student2 {
    String name;
    public Student2(String name) {   // 显式定义了有参构造
        this.name = name;
    }
    // 此时编译器不会生成无参构造！
    // 如果还想用 new Student2()，必须自己手动写
}
```

### 5.4 构造方法重载

构造方法可以重载（同名不同参数），用于提供多种创建对象的方式。

```java
public class Student {
    String name;
    int age;
    String gender;

    public Student() {
    }

    public Student(String name) {
        this.name = name;
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public Student(String name, int age, String gender) {
        this(name, age);       // 复用上面的两参构造
        this.gender = gender;
    }
}
```

## 六、封装（Encapsulation）

封装是面向对象的三大特性之一。它的核心思想是：**隐藏对象的属性和实现细节，仅对外提供公共的访问方式**。

### 6.1 为什么要封装

如果不封装，外部代码可以直接修改对象的内部数据，可能写入非法值：

```java
// ❌ 不安全：外部可以随意赋值
Student s = new Student();
s.age = -1000;     // 年龄怎么可能为负数？不合理！
```

### 6.2 封装的标准做法

1. 成员变量用 `private` 修饰，禁止外部直接访问。
2. 提供 `public` 的 `getXxx()` 和 `setXxx()` 方法，在这些方法中加入逻辑校验。

```java
public class Student {
    private String name;
    private int age;

    // getter：提供读取
    public String getName() {
        return name;
    }

    // setter：提供设置，并加入校验
    public void setAge(int age) {
        if (age < 0 || age > 150) {
            System.out.println("年龄不合法");
            return;
        }
        this.age = age;     // 只有合法的值才会被赋给成员变量
    }

    public int getAge() {
        return age;
    }
}
```

> 💡 **提示：** 封装的好处是：(1) 提高代码的安全性（防止非法赋值）；(2) 提高代码的复用性和可维护性；(3) 隐藏实现细节，降低使用者的学习成本。

## 七、完整示例：Student 类

下面是一个符合 JavaBean 规范的完整 Student 类：

```java
public class Student {

    // 1. 私有属性（成员变量）
    private String name;
    private int age;
    private String gender;

    // 2. 无参构造（JavaBean 规范要求必须有无参构造）
    public Student() {
    }

    // 3. 全参构造
    public Student(String name, int age, String gender) {
        this.name = name;
        this.age = age;
        this.gender = gender;
    }

    // 4. getter / setter
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public String getGender() {
        return gender;
    }

    public void setGender(String gender) {
        this.gender = gender;
    }

    // 5. 普通方法
    public void study() {
        System.out.println(name + " 正在学习，今年 " + age + " 岁");
    }

    // 6. 重写 toString，方便打印对象
    @Override
    public String toString() {
        return "Student{name='" + name + "', age=" + age + ", gender='" + gender + "'}";
    }
}
```

使用方式：

```java
public class Test {
    public static void main(String[] args) {
        Student s1 = new Student("张三", 18, "男");
        s1.study();
        System.out.println(s1);        // 直接打印对象，自动调用 toString()

        Student s2 = new Student();    // 无参构造
        s2.setName("李四");
        s2.setAge(20);
        System.out.println(s2.getName());
    }
}
```

## 八、JavaBean 规范与成员变量默认值

### 8.1 JavaBean 规范

JavaBean 是 Java 中一种标准的组件写法，许多框架（如 Spring、MyBatis）都依赖这种规范。一个标准的 JavaBean 应该满足：

| 规范要求 | 说明 |
|---------|------|
| 类是 public 的 | 外部可以访问 |
| 有无参构造方法 | 框架反射创建对象时需要 |
| 属性是 private 的 | 通过 getter/setter 访问 |
| 提供 public 的 getter/setter | 方法名遵循 `getXxx()` / `setXxx()` 命名 |
| 实现 Serializable（可选） | 用于序列化 |

### 8.2 getter/setter 命名规则

| 成员变量 | getter 方法 | setter 方法 |
|---------|------------|------------|
| `String name` | `getName()` | `setName(String)` |
| `boolean flag` | `isFlag()`（布尔类型用 is） | `setFlag(boolean)` |
| `int age` | `getAge()` | `setAge(int)` |

### 8.3 成员变量的默认值

成员变量（在堆中）在创建对象时会被自动初始化为默认值，**不需要手动赋值**。注意：**局部变量没有默认值，使用前必须初始化**。

| 数据类型 | 默认值 |
|---------|--------|
| `byte` / `short` / `int` | `0` |
| `long` | `0L` |
| `float` | `0.0f` |
| `double` | `0.0` |
| `char` | `' '`（空字符） |
| `boolean` | `false` |
| 引用类型（String、数组、对象等） | `null` |

```java
public class Demo {
    int a;             // 默认 0
    boolean b;         // 默认 false
    String s;          // 默认 null

    public void test() {
        int x;
        // System.out.println(x);  // ❌ 编译错误：局部变量 x 未初始化
        System.out.println(a);      // ✅ 输出 0，成员变量有默认值
    }
}
```

## 九、一个 Java 文件中的类定义规则

> ⚠️ **注意：** 一个 `.java` 源文件中：
> 1. **只能有一个** `public` 修饰的类，且**文件名必须与该 public 类名完全一致**。
> 2. 可以有**多个**非 public（默认访问级别）的类。
> 3. 一个 `.java` 文件编译后会生成与每个类对应的 `.class` 文件。

```java
// 文件名必须是 Student.java（因为 public 类叫 Student）
public class Student {        // ✅ 只能有一个 public 类
    // ...
}

class Teacher {               // ✅ 默认访问级别的类可以有多个
    // ...
}

class Headmaster {
    // ...
}
// 编译后会生成：Student.class、Teacher.class、Headmaster.class
```

## 十、面试常见问题

### Q1：类和对象是什么关系？

类是对象的模板/蓝图，对象是类的实例。类是抽象的，描述"有什么"；对象是具体的，存储实际数据。一个类可以创建多个对象，对象之间相互独立。类比：类是"汽车图纸"，对象是"一辆真实的车"。

### Q2：成员变量和局部变量有什么区别？

| 对比项 | 成员变量 | 局部变量 |
|--------|---------|---------|
| 定义位置 | 类中、方法外 | 方法中、方法参数、代码块中 |
| 内存位置 | 堆（随对象） | 栈（随方法调用） |
| 默认值 | 有默认值（int 为 0，引用为 null） | 无默认值，必须先初始化 |
| 作用域 | 整个类 | 从定义处到方法/代码块结束 |
| 生命周期 | 随对象的创建而存在，随对象的回收而消失 | 随方法的调用而存在，随方法结束而消失 |

### Q3：this 关键字的作用是什么？能用在 static 方法中吗？

`this` 代表当前对象的引用，主要用于：(1) 区分成员变量和局部变量；(2) 在构造方法中用 `this()` 调用本类其他构造方法；(3) 用 `this.方法名()` 调用本类其他方法。`this` **不能**用在 `static` 方法中，因为 static 方法属于类，不依赖对象，调用时可能还没有任何对象。

### Q4：一个类如果没有写构造方法，能创建对象吗？

能。因为编译器会自动为该类生成一个 public 的无参构造方法。但**一旦你显式定义了任何构造方法**，编译器就不再自动生成无参构造。如果此时还想用 `new 类名()` 创建对象，就必须手动再写一个无参构造，否则会编译错误。

### Q5：为什么构造方法没有返回值，连 void 都不能写？

构造方法的返回值是"隐式"的——它返回的就是新创建的对象本身（通过 `new` 表达式接收）。如果写成 `void`，Java 编译器会把它当作一个**普通方法**（恰好与类同名），而不是构造方法，那么 `new 类名()` 时就找不到对应的无参构造了。

### Q6：什么是 JavaBean？为什么需要无参构造？

JavaBean 是一种标准的 Java 类写法规范：public 类、private 属性、public 的 getter/setter、必须有无参构造。无参构造非常重要，因为很多框架（如 Spring、MyBatis、各种 JSON 序列化库）在底层通过**反射**创建对象时，默认调用无参构造方法，如果没有无参构造会导致反射创建失败。

## 十一、总结

| 知识点 | 核心要点 |
|--------|---------|
| 类的本质 | 模板/蓝图，定义属性和行为 |
| 类的组成 | 成员变量、构造方法、成员方法、代码块、内部类 |
| 对象创建 | `new` 关键字，对象在堆内存，引用在栈内存 |
| this 关键字 | 当前对象引用，区分变量、构造器复用、调用本类方法 |
| 构造方法 | 与类同名、无返回值、`new` 时调用、可重载、可被自动生成 |
| 封装 | private 属性 + public getter/setter，提高安全性和可维护性 |
| JavaBean | public 类、无参构造、private 属性、标准 getter/setter |
| 成员变量默认值 | 基本类型为 0/false，引用类型为 null |
| 文件规则 | 一个 .java 只能有一个 public 类，文件名与 public 类同名 |

掌握类是学习面向对象的基础。把类的定义、对象的创建、`this` 的使用、构造方法和封装彻底搞懂，后续的继承、多态、接口等高级特性才能轻松上手。
