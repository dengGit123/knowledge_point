# 泛型（Generics）

> 官方文档：[Java Generics Trail](https://docs.oracle.com/javase/tutorial/java/generics/)、[菜鸟教程 Java 泛型](https://www.runoob.com/java/java-generics.html)

## 一、概述

**泛型（Generics）** 是 Java 5 引入的特性，它的本质是**参数化类型**——把"类型"当作参数传给类、接口或方法，让一份代码能适用于多种数据类型，同时在**编译期**进行类型检查。

> 💡 **提示：** 类比：泛型就像"万能容器"或"模板"。`List<String>` 表示"装字符串的列表"，`List<Integer>` 表示"装整数的列表"，同一个 `List` 类的代码可以装不同类型，但一旦确定了类型，编译器就会保证你只能往里放这种类型的东西。

### 为什么需要泛型？

```java
// ❌ 泛型出现之前：集合只能存 Object，取出时强转，运行时可能 ClassCastException
List list = new ArrayList();
list.add("hello");
list.add(123);            // 编译能过，因为都是 Object
String s = (String) list.get(1); // 运行时报错：ClassCastException

// ✅ 泛型出现之后：编译期就检查类型，取出来不需要强转
List<String> list2 = new ArrayList<>();
list2.add("hello");
list2.add(123);           // ❌ 编译报错，类型不匹配
String s2 = list2.get(0); // ✅ 不用强转
```

泛型的三大价值：
1. **编译期类型检查**：把类型错误从运行期提前到编译期。
2. **消除强制类型转换**：存取无需手动强转。
3. **代码复用**：一份泛型代码适用于多种类型。

## 二、核心概念：泛型类、泛型方法、泛型接口

### 1. 泛型类

在类名后用 `<T>` 声明类型参数，`T` 在整个类中可当作类型使用。

```java
// 定义泛型类 Box<T>，T 是类型参数
public class Box<T> {
    private T item;

    public void set(T item) {
        this.item = item;
    }

    public T get() {
        return item;
    }

    public static void main(String[] args) {
        Box<String> stringBox = new Box<>(); // 钻石操作符 <>
        stringBox.set("hello");
        String s = stringBox.get(); // ✅ 无需强转

        Box<Integer> intBox = new Box<>();
        intBox.set(100);
        Integer i = intBox.get();
    }
}
```

### 2. 泛型方法

泛型方法的类型参数声明 `<T>` 写在**返回值之前**，与方法所在类是否是泛型类无关。

```java
public class GenericMethod {
    // <T> 声明类型参数，写在返回值 T 之前
    public static <T> T getFirst(T[] array) {
        if (array == null || array.length == 0) {
            return null;
        }
        return array[0];
    }

    // 多个类型参数
    public static <K, V> void print(K key, V value) {
        System.out.println(key + " = " + value);
    }

    public static void main(String[] args) {
        String[] names = {"Alice", "Bob"};
        String first = getFirst(names); // ✅ 编译器自动推断 T=String
        Integer[] nums = {1, 2, 3};
        Integer n = getFirst(nums);
        print("age", 18); // K=String, V=Integer
    }
}
```

### 3. 泛型接口

```java
// 定义泛型接口
public interface Repository<T> {
    void save(T t);
    T findById(int id);
}

// 实现方式一：实现时指定具体类型
public class UserRepository implements Repository<User> {
    public void save(User u) { /* ... */ }
    public User findById(int id) { /* ... */ }
}

// 实现方式二：实现时继续保留泛型
public class BaseRepository<T> implements Repository<T> {
    public void save(T t) { /* ... */ }
    public T findById(int id) { return null; }
}
```

## 三、类型参数命名约定

| 类型参数 | 含义 | 常见场景 |
|---------|------|---------|
| `T` | Type（类型） | 通用类型 |
| `E` | Element（元素） | 集合中的元素，如 `List<E>` |
| `K` | Key（键） | Map 的键，如 `Map<K, V>` |
| `V` | Value（值） | Map 的值 |
| `R` | Result（结果） | 函数式接口的返回值，如 `Function<T, R>` |
| `N` | Number（数字） | 数字类型 |

## 四、钻石操作符（Diamond Operator，Java 7+）

在构造对象时，右侧的泛型类型可以省略，由编译器根据左侧推断，称为**菱形语法**。

```java
// Java 7 之前：必须两边都写
Map<String, List<Integer>> map1 = new HashMap<String, List<Integer>>();

// Java 7+：右侧用 <> 即可
Map<String, List<Integer>> map2 = new HashMap<>();
```

## 五、类型擦除（Type Erasure）

**这是 Java 泛型最关键的底层原理。** Java 的泛型只在**编译期**存在，编译后会进行"类型擦除"，运行时并不保留泛型信息。

```text
源码                          编译后（运行时）
Box<String>      ──擦除──►   Box（内部用 Object）
List<Integer>    ──擦除──►   List（内部用 Object）
T extends Number ──擦除──►   Number（用上界替换）
```

### 擦除带来的"反直觉"现象

```java
public class ErasureDemo {
    public static void main(String[] args) {
        List<String> a = new ArrayList<>();
        List<Integer> b = new ArrayList<>();
        // ❌ 运行时它们的 Class 是同一个！泛型被擦除了
        System.out.println(a.getClass() == b.getClass()); // true

        // ❌ 运行时拿不到泛型类型
        // System.out.println(new ArrayList<String>().getClass()); 只能拿到 ArrayList
    }
}
```

> ⚠️ **注意：** 由于类型擦除，以下操作无法进行（见下方限制表格）。

## 六、通配符与 PECS 原则

### 1. 三种通配符

| 通配符 | 名称 | 含义 | 能否写入 | 能否读取 |
|--------|------|------|---------|---------|
| `<?>` | 无界通配符 | 未知类型 | 不能（除 null） | 只能读成 Object |
| `<? extends T>` | 上界通配符（协变） | T 或 T 的子类 | 不能（除 null） | 能读成 T |
| `<? super T>` | 下界通配符（逆变） | T 或 T 的父类 | 能写 T 及子类 | 只能读成 Object |

### 2. 上界通配符 `<? extends T>`：只能读不能写

```java
// 表示"某种 Number 子类的列表"，具体是 Integer 还是 Double 不知道
List<? extends Number> list = new ArrayList<Integer>();

Number n = list.get(0);  // ✅ 读出来一定是 Number
// list.add(1);          // ❌ 不能写：编译器无法保证添加的类型匹配
```

### 3. 下界通配符 `<? super T>`：只能写不能（精确）读

```java
// 表示"Integer 或 Integer 父类的列表"
List<? super Integer> list = new ArrayList<Number>();

list.add(1);             // ✅ 可以写 Integer（及其子类）
list.add(2);
Object o = list.get(0);  // 只能读成 Object（因为不知道确切父类）
```

### 4. PECS 原则（Producer Extends, Consumer Super）

> 💡 **提示：** 记忆口诀——**"频繁读用 extends，频繁写用 super"**。
> - 如果集合是一个**生产者**（你要从中读取数据），用 `<? extends T>`。
> - 如果集合是一个**消费者**（你要往里写数据），用 `<? super T>`。

```java
// 经典示例：把 src 的元素拷贝到 dest
public static <T> void copy(List<? super T> dest,        // 消费者：写入，用 super
                            List<? extends T> src) {      // 生产者：读取，用 extends
    for (T item : src) {
        dest.add(item);
    }
}
```

## 七、泛型的限制

| 限制 | 说明 | 示例 |
|------|------|------|
| 不能用基本类型 | 必须用对应的包装类 | `List<int>` ❌ → `List<Integer>` ✅ |
| 不能 `new T()` | 类型擦除后 T 变成 Object，无法 new | `new T()` ❌ |
| 不能用于静态字段/方法 | 静态成员在类加载时存在，而泛型类型是创建对象时才确定 | `static T x;` ❌ |
| 不能 `catch` 泛型异常类 | 异常匹配在运行时，泛型已擦除 | `catch (MyException<T> e)` ❌ |
| 不能创建泛型数组 | 数组有运行时类型检查，与擦除冲突 | `new List<String>[10]` ❌ |
| 运行时 instanceof 不能用泛型 | 擦除后无泛型信息 | `x instanceof List<String>` ❌ → `x instanceof List` ✅ |
| 不能用同一泛型的不同参数类型重载 | 擦除后签名相同 | `void f(List<String>)` 和 `void f(List<Integer>)` 冲突 ❌ |

```java
public class GenericLimit<T> {
    // ❌ private T obj = new T();     不能 new T()
    // ❌ private static T x;          静态字段不能用类的类型参数
    // ❌ private T[] arr = new T[10]; 不能创建泛型数组

    // ✅ 创建泛型数组只能这样（需强制转换，编译器警告）
    @SuppressWarnings("unchecked")
    public T[] createArray(Class<T> clazz, int size) {
        return (T[]) java.lang.reflect.Array.newInstance(clazz, size);
    }
}
```

## 八、注意事项与陷阱

> ⚠️ **注意：**
> 1. 泛型**不支持基本类型**，要用包装类（`int` → `Integer`），否则会有自动装箱开销。
> 2. 泛型类型**不协变**：`List<Integer>` 不是 `List<Number>` 的子类（虽然 Integer 是 Number 的子类）。需要协变时用通配符。
> 3. 运行时通过反射向泛型集合插入"错误类型"是可能的（绕过编译检查），但这是不安全的做法。
> 4. 注意 `<? extends T>` 只读、`<? super T>` 只写的特性，合理应用 PECS。

```java
// ❌ 泛型不协变
// List<Number> numbers = new ArrayList<Integer>(); // 编译报错

// ✅ 用通配符实现协变
List<? extends Number> numbers = new ArrayList<Integer>(); // OK
```

## 九、面试常见问题

### Q1：什么是泛型的类型擦除？

Java 的泛型只在编译期存在，编译后会擦除类型参数：无界擦除为 `Object`，有上界擦除为上界。运行时 `List<String>` 和 `List<Integer>` 是同一个类。所以运行时无法通过反射或 instanceof 获取具体的泛型参数类型。

### Q2：PECS 原则是什么？

**Producer Extends, Consumer Super**。如果要从集合读数据（生产者），用 `<? extends T>`；如果要往集合写数据（消费者），用 `<? super T>`。简记"读 extends、写 super"。

### Q3：泛型中 `<? extends T>` 和 `<? super T>` 的区别？

`<? extends T>` 是上界通配符，表示 T 或其子类，只能读不能写；`<? super T>` 是下界通配符，表示 T 或其父类，只能写不能精确读。它们分别用于协变读取和逆变写入场景。

### Q4：为什么 Java 泛型不能用基本类型？

因为类型擦除后类型参数会变成 Object（或上界），而基本类型不是 Object 的子类。所以必须用包装类（如 `Integer`、`Double`），由自动装箱/拆箱完成转换。

### Q5：泛型类和泛型方法有什么区别？

泛型类的类型参数作用于整个类；泛型方法的类型参数声明 `<T>` 写在返回值前，只作用于该方法，且与所在类是否是泛型无关。调用泛型方法时类型一般由编译器推断。

## 十、总结

- 泛型是**参数化类型**，在编译期做类型检查，消除强转，提升复用性。
- 三种形式：泛型类 `class Box<T>`、泛型方法 `<T> T foo(T t)`、泛型接口 `interface Repo<T>`。
- **类型擦除**是 Java 泛型底层原理，运行时不保留泛型信息，带来一系列限制。
- 通配符三兄弟：`<?>`（任意）、`<? extends T>`（上界只读）、`<? super T>`（下界只写）。
- 遵循 **PECS 原则**（Producer Extends, Consumer Super）选择通配符。
- 基本类型要用包装类，注意泛型不协变、不能 new T()、不能创建泛型数组等限制。
