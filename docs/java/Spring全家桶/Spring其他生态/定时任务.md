### 一、概述

> 📖 [Spring - Task Execution and Scheduling](https://docs.spring.io/spring-framework/reference/integration/scheduling.html)

**Spring Task** 是 Spring 内置的**轻量定时任务**方案，开箱即用——加两个注解（`@EnableScheduling` + `@Scheduled`）就能让方法定时执行，**无需引入 Quartz 等重型框架**。

大白话：想让某个方法「每天凌晨跑一次」「每分钟跑一次」，不用自己写定时器线程、不用装 Quartz，Spring Task 给你两个注解搞定。它像是 Spring 送的「**定时闹钟**」。

| 你将学到 | 说明 |
| --- | --- |
| 基本用法 | `@Scheduled` 三种触发方式 |
| cron 表达式 | 怎么精确控制时间 |
| 多线程坑 | 默认单线程阻塞问题 |
| 分布式坑 | 多实例重复执行问题 |

> 📌 本文是 Spring其他生态子目录的第一篇，后续 [[缓存抽象]]、[[异步与事件]]。

### 二、基本用法：两步走

#### 1. 开启定时任务支持

启动类加 `@EnableScheduling`：

```java
@SpringBootApplication
@EnableScheduling          // ✅ 开启定时任务支持
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

#### 2. 标记定时方法

```java
@Component
public class ReportTask {

    @Scheduled(cron = "0 0 2 * * ?")    // 每天凌晨 2 点执行
    public void dailyReport() {
        generateDailyReport();           // 你的定时逻辑
    }
}
```

> 💡 **提示：** 标了 `@Scheduled` 的方法所在的类，**必须是 Spring 管理的 Bean**（加 `@Component`），否则不生效——这和 [[../Spring框架核心/AOP面向切面]] 的要求一致（定时任务底层也是代理）。

### 三、三种触发方式

`@Scheduled` 提供三种方式，按场景选用：

| 方式 | 注解属性 | 含义 | 示例 |
| --- | --- | --- | --- |
| **cron 表达式** | `cron` | 按 cron 规则（最灵活） | `cron = "0 0 2 * * ?"` |
| **固定频率** | `fixedRate` | 从**上次开始**算，间隔 N 毫秒 | `fixedRate = 5000` |
| **固定延迟** | `fixedDelay` | 从**上次结束**算，间隔 N 毫秒 | `fixedDelay = 5000` |

```java
@Scheduled(fixedRate = 5000)         // 每 5 秒一次（不管上次有没有跑完）
public void task1() { }

@Scheduled(fixedDelay = 5000)        // 上次跑完后，再等 5 秒
public void task2() { }

@Scheduled(cron = "0 */1 * * * ?")   // 每分钟一次
public void task3() { }
```

#### fixedRate vs fixedDelay（易混淆）

```
fixedRate（固定频率，从开始算）：
  任务1开始 ──(5s)──► 任务2开始 ──(5s)──► 任务3开始
  （若任务执行超过 5s，会等当前跑完再立刻执行下一个）

fixedDelay（固定延迟，从结束算）：
  任务1开始 ──执行──► 任务1结束 ──(5s)──► 任务2开始 ──执行──► ...
  （任务再慢也保证两次之间间隔 5s）
```

> 💡 **提示：** `fixedRate`/`fixedDelay` 的单位默认毫秒，可用 `timeUnit = TimeUnit.SECONDS` 改成秒。也可用 `fixedRateString = "${task.interval:5000}"` 从配置读。

### 四、cron 表达式详解

cron 是最灵活的触发方式，由 6~7 个字段组成：

```
秒  分  时  日  月  周  [年]
0   0   2   *   *   ?
│   │   │   │   │   │
│   │   │   │   │   └─ 星期（0/1=周日，1-7）
│   │   │   │   └───── 月（1-12）
│   │   │   └───────── 日（1-31）
│   │   └───────────── 时（0-23）
│   └─────────────── 分（0-59）
└─────────────────── 秒（0-59）
```

#### 特殊字符

| 字符 | 含义 | 示例 |
| --- | --- | --- |
| `*` | 任意值 | `*` 表示「每」 |
| `,` | 枚举 | `1,3,5` 表示 1、3、5 |
| `-` | 范围 | `1-5` 表示 1 到 5 |
| `/` | 步长 | `0/5` 表示从 0 开始每 5 一次 |
| `?` | 不指定（只能用在**日和周**，互斥） | 日用了 `*`，周必须用 `?` |
| `L` | 最后（Last） | `0 0 0 L * ?` = 每月最后一天 |
| `W` | 最近的工作日 | `15W` = 离 15 号最近的工作日 |

#### 常用示例

| cron | 含义 |
| --- | --- |
| `0 0 2 * * ?` | 每天凌晨 2 点 |
| `0 */5 * * * ?` | 每 5 分钟 |
| `0 0 * * * ?` | 每小时整点 |
| `0 0 9 ? * MON-FRI` | 工作日早上 9 点 |
| `0 0 0 1 * ?` | 每月 1 号零点 |
| `0 0 10,14,16 * * ?` | 每天 10、14、16 点 |

> 💡 **提示：** cron 字段里 **「日」和「周」二选一**——指定了日就不能再指定周，另一个用 `?` 表示「不限制」。这是最常踩的坑。

> ⚠️ **注意：** Spring 的 cron 是 **6 位**（秒 分 时 日 月 周），而 Linux crontab 是 **5 位**（没有秒）。两者格式不同，别混。

### 五、坑一：默认单线程串行执行

> ⚠️ **重大陷阱：** Spring Task 默认用**单线程**执行所有定时任务。如果一个任务执行慢，**会阻塞其他定时任务**。

```java
@Scheduled(fixedRate = 1000)
public void slowTask() {
    Thread.sleep(10000);   // 慢任务，执行 10 秒
}

@Scheduled(fixedRate = 1000)
public void fastTask() {
    System.out.println("快任务");   // ❌ 被慢任务阻塞，每 10 秒才执行一次！
}
```

#### 解决：配置线程池

```java
@Configuration
public class SchedulerConfig implements SchedulingConfigurer {
    @Override
    public void configureTasks(ScheduledTaskRegistrar registrar) {
        // 给定时任务配一个线程池（多线程并行）
        registrar.setScheduler(Executors.newScheduledThreadPool(5));
    }
}
```

> 💡 **提示：** 配 5 个线程，意味着最多 5 个定时任务能并行。按任务数量设大小。

### 六、坑二：分布式环境重复执行

> ⚠️ **重大陷阱：** 微服务下你的应用会部署**多个实例**，每个实例都会触发定时任务 → **同一个任务被执行多次**（如报表生成 3 份、短信发 3 条）。

```
3 个实例都部署了 @Scheduled 任务
实例1 ─► 凌晨 2 点触发 → 执行一次
实例2 ─► 凌晨 2 点触发 → 执行一次
实例3 ─► 凌晨 2 点触发 → 执行一次
        同一个任务被跑了 3 遍！
```

#### 解决方案

| 方案 | 说明 |
| --- | --- |
| **ShedLock** | 基于数据库/Redis 的分布式锁，保证同一时刻只有一个实例执行 |
| **数据库锁** | 任务执行前去某张表「抢占」一条记录，抢到的才执行 |
| **集中式调度（推荐）** | 用 **XXL-JOB / PowerJob** 等专业调度中心，把定时任务从应用里剥离 |

> 💡 **提示：** 单机用 Spring Task 足够；**一旦上了多实例（集群），务必加分布式锁或换集中式调度**，否则一定会重复执行。

### 七、实际应用场景

1. **定时报表**：每天凌晨生成日报、每月生成月报。
2. **数据同步**：定时把数据从一个系统同步到另一个。
3. **缓存预热**：启动后定时刷新热点缓存。
4. **清理任务**：定时删除过期数据、临时文件。
5. **心跳/对账**：定时和第三方系统对账。

---

### 八、常见问题与注意事项

> ⚠️ **注意：**
> - **不要吞异常**：定时任务方法里抛异常，下次仍会执行；但如果用 try-catch 吞了且没记日志，问题会被掩盖。建议加日志 + 监控告警。
> - **任务别执行太久**：长时间任务占着线程，影响其他任务（单线程时尤甚）。
> - **自调用失效**：`@Scheduled` 方法被同类内部直接调用会失效（和 AOP 一样的代理问题）。
> - **`@Scheduled` 不要手动调用**：它是给调度器用的，手动 `task.run()` 会绕过调度逻辑。

> 💡 **提示：** Spring Task 适合**轻量、单机**的定时需求。复杂场景（任务编排、失败重试、可视化、集群调度）应该上 **XXL-JOB / PowerJob** 这类专业调度框架。

---

### 九、总结

- **Spring Task = 内置轻量定时任务**，`@EnableScheduling` + `@Scheduled` 两步上手。
- **三种触发**：`fixedRate`（从开始算）、`fixedDelay`（从结束算）、`cron`（最灵活）。
- **cron 6 字段**：秒 分 时 日 月 周；日和周二选一，另一个用 `?`。
- **坑一**：默认单线程，慢任务会阻塞其他 → 配线程池。
- **坑二**：多实例重复执行 → 加分布式锁（ShedLock）或换集中式调度（XXL-JOB）。
