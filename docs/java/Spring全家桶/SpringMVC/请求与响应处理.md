### 二、接收参数的注解

#### 1. `@RequestParam` —— 查询参数 / 表单参数

接收 URL 里 `?key=value` 的查询参数：

```java
@GetMapping("/users")
public List<User> list(
    @RequestParam(defaultValue = "1") int page,        // 默认值
    @RequestParam(defaultValue = "10") int size,
    @RequestParam(required = false) String name        // 非必填
) {
    // GET /users?page=2&size=20&name=张三
    return userService.list(page, size, name);
}
```

| 属性 | 作用 |
| --- | --- |
| `value`/`name` | 参数名（默认和方法参数名一致） |
| `required` | 是否必填（默认 `true`，缺失报 400） |
| `defaultValue` | 默认值（设了之后 `required` 自动变 `false`） |

#### 2. `@PathVariable` —— 路径变量

从 URL 路径里取参数（RESTful 风格）：

```java
@GetMapping("/users/{id}")
public User get(@PathVariable Long id) {              // 取 /users/123 里的 123
    return userService.findById(id);
}

@GetMapping("/users/{userId}/orders/{orderId}")       // 多个路径变量
public Order get(@PathVariable Long userId, @PathVariable Long orderId) {
    return orderService.find(userId, orderId);
}
```

#### 3. `@RequestBody` —— 请求体（JSON）

接收 POST/PUT 的 JSON 请求体，自动反序列化成对象：

```java
@PostMapping("/users")
public User create(@RequestBody User user) {          // 前端传 JSON {"name":"张三","age":20}
    return userService.save(user);
}
```

> ⚠️ **注意：** `@RequestBody` 一个方法**只能用一次**；且只能用于 **POST/PUT**（带 body 的请求），GET 没有 body。

#### 4. `@RequestHeader` 与 `@CookieValue`

```java
@GetMapping("/info")
public String info(
    @RequestHeader("Authorization") String token,     // 取请求头
    @CookieValue("sessionId") String sessionId        // 取 Cookie
) {
    return token + ", " + sessionId;
}
```

#### 四个注解对比

| 注解 | 数据来源 | 典型场景 |
| --- | --- | --- |
| `@RequestParam` | 查询串 `?x=1` / 表单 | 筛选条件、分页 |
| `@PathVariable` | URL 路径 `/users/{id}` | RESTful 资源定位 |
| `@RequestBody` | 请求体 JSON | 新增、修改提交的数据 |
| `@RequestHeader` | 请求头 | token、自定义头 |
| `@CookieValue` | Cookie | 会话标识 |

### 三、自动绑定到对象（POJO）

参数多时，逐个写太啰嗦。Spring 能把请求参数自动绑定到一个对象：

```java
// 查询对象
public class UserQuery {
    private String name;
    private Integer age;
    private Integer page;
    // getter/setter...
}

@GetMapping("/users/search")
public List<User> search(UserQuery query) {            // 不用注解，自动绑定
    // GET /users/search?name=张三&age=20&page=1
    // query.name="张三", query.age=20, query.page=1
    return userService.search(query);
}
```

> 💡 **提示：** 绑定要求**参数名和对象属性名一致**。复杂对象（如内嵌列表）也能绑定，配合 `@ModelAttribute` 更可控。

### 四、日期类型处理

日期参数默认无法自动转换，需要指定格式：

```java
@GetMapping("/orders")
public List<Order> list(
    @RequestParam @DateTimeFormat(pattern = "yyyy-MM-dd") LocalDate start,  // 查询参数里的日期
    @RequestParam @DateTimeFormat(pattern = "yyyy-MM-dd") LocalDate end
) {
    return orderService.listBetween(start, end);
}
```

返回 JSON 时的日期格式，用 Jackson 的 `@JsonFormat`：

```java
public class Order {
    @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
    private Date createTime;       // 序列化成 "2026-07-06 10:30:00"
}
```

> ⚠️ **注意：** 不加 `@JsonFormat`，日期会序列化成时间戳或 ISO 格式，前端不好用。**时区**也要指定（`GMT+8`），否则可能差 8 小时。

### 五、返回数据：`@ResponseBody`

`@RestController` 等价于类上每个方法都加了 `@ResponseBody`，返回值会被 **Jackson** 序列化成 JSON：

```java
@RestController
@RequestMapping("/users")
public class UserController {

    @GetMapping("/{id}")
    public User get(@PathVariable Long id) {           // 返回 User 对象 → 自动转 JSON
        return userService.findById(id);
    }
}
```

通常配合**统一返回结构**，让所有接口返回格式一致：

```java
// 统一返回包装类
public class Result<T> {
    private int code;          // 状态码
    private String message;    // 提示信息
    private T data;            // 业务数据

    public static <T> Result<T> success(T data) {
        return new Result<>(200, "success", data);
    }
    public static <T> Result<T> error(int code, String msg) {
        return new Result<>(code, msg, null);
    }
    // getter/setter...
}

@GetMapping("/{id}")
public Result<User> get(@PathVariable Long id) {
    return Result.success(userService.findById(id));
    // 返回 {"code":200,"message":"success","data":{...}}
}
```

> 💡 **提示：** 统一返回结构是接口设计的**最佳实践**，前端解析逻辑可以统一处理（成功取 data，失败按 code 报错）。配合 [[全局异常处理]]，异常也能返回统一格式。

### 六、完整 CRUD 示例

把上面的组合起来，写一套用户增删改查：

```java
@RestController
@RequestMapping("/users")
public class UserController {

    @Autowired
    private UserService userService;

    @GetMapping                      // GET    /users          查询列表
    public Result<List<User>> list() {
        return Result.success(userService.findAll());
    }

    @GetMapping("/{id}")             // GET    /users/123      查询单个
    public Result<User> get(@PathVariable Long id) {
        return Result.success(userService.findById(id));
    }

    @PostMapping                     // POST   /users          新增
    public Result<User> create(@RequestBody User user) {
        return Result.success(userService.save(user));
    }

    @PutMapping("/{id}")             // PUT    /users/123      修改
    public Result<User> update(@PathVariable Long id, @RequestBody User user) {
        user.setId(id);
        return Result.success(userService.update(user));
    }

    @DeleteMapping("/{id}")          // DELETE /users/123      删除
    public Result<Void> delete(@PathVariable Long id) {
        userService.deleteById(id);
        return Result.success(null);
    }
}
```

### 七、文件上传

用 `MultipartFile` 接收上传的文件：

```java
@PostMapping("/upload")
public String upload(@RequestParam("file") MultipartFile file) throws IOException {
    if (file.isEmpty()) {
        throw new RuntimeException("文件不能为空");
    }
    String originalName = file.getOriginalFilename();   // 原始文件名
    String destPath = "/uploads/" + originalName;
    file.transferTo(new File(destPath));                // 保存到磁盘
    return "上传成功: " + originalName;
}
```

配置上传限制（`application.yml`）：

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB        # 单个文件大小上限
      max-request-size: 100MB    # 整个请求大小上限
```

### 八、常见问题与注意事项

> ⚠️ **注意：**
> - **参数类型转换失败**（如 `?id=abc` 但接收 `Long id`）会返回 **400**。
> - **`@RequestBody` 缺了 `Content-Type: application/json`** 会报 **415**（不支持的媒体类型）。
> - **必填参数没传**，`@RequestParam(required=true)` 报 **400**。
> - **GET 请求不要用 `@RequestBody`**，GET 没有 body。
> - 大文件上传注意配 `max-file-size`，否则报 `MaxUploadSizeExceededException`。

> 💡 **提示：** 想清楚每个参数「**从哪来**」（URL 路径？查询串？请求体？请求头？），就能选对注解，不用死记。

---

### 九、总结

- **接收参数**：查询串用 `@RequestParam`，路径用 `@PathVariable`，JSON 体用 `@RequestBody`。
- **对象绑定**：参数名与属性名一致即可自动绑定到 POJO。
- **日期**：入参用 `@DateTimeFormat`，出参用 `@JsonFormat`。
- **返回数据**：`@RestController` + 统一 `Result<T>` 包装是最佳实践。
- **文件上传**：`MultipartFile`，注意大小限制。
