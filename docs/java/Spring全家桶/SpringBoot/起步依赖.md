### 一、概述

> 📖 [Spring Boot - Starters](https://docs.spring.io/spring-boot/docs/current/reference/html/using.html#using.build-systems.starters)

**起步依赖（Starter）** 是 Spring Boot 的两大核心特性之一（另一个是自动配置）。它把某个场景需要的**所有依赖**打包成**一个坐标**，让你不用再一个个找依赖、调版本。

大白话：以前做 Web 项目，得自己引入 `spring-web`、`spring-mvc`、`tomcat`、`jackson`……还要担心版本冲突。Starter 就像「**Web 开发全家桶套餐**」——点一份 `spring-boot-starter-web`，相关的一切都端上来了，而且版本都帮你配好。

| 你将学到 | 说明 |
| --- | --- |
| starter 是什么 | 一站式场景依赖包 |
| starter-parent | 统一管理版本的「母依赖」 |
| 常用 starter | 官方提供的常用套餐 |
| starter 原理 | 它内部是怎么实现的 |
| 自定义 starter | 自己封装一个场景包 |

> 📌 配合 [[SpringBoot入门]]、[[自动配置原理]] 阅读。

### 二、`spring-boot-starter-parent`：版本大管家

每个 Spring Boot 项目都建议继承这个 parent：

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
</parent>
```

它干了三件事：

| 职责 | 说明 |
| --- | --- |
| **统一版本** | 预定义了几百个常用库的版本（`spring.version`、`jackson.version` 等），你写依赖时**不用写 version** |
| **编译配置** | 默认 JDK 版本、UTF-8 编码、编译参数 |
| **插件管理** | 预配置 `spring-boot-maven-plugin`（打包用） |

```xml
<!-- 继承 parent 后，引入依赖不用写 version，版本由 parent 统一管控 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <!-- 没有 <version>！ -->
</dependency>
```

> 💡 **提示：** 这就是「依赖版本冲突」问题被解决的关键——所有相关库的版本由 parent 统一协调，保证彼此兼容。

### 三、常用官方 Starter

官方 starter 都以 `spring-boot-starter-xxx` 命名：

| Starter | 用途 | 一句话说明 |
| --- | --- | --- |
| `spring-boot-starter` | 核心基础 | 自动配置、日志、YAML 支持（不含 Web） |
| `spring-boot-starter-web` | Web 开发 | Spring MVC + 内嵌 Tomcat + Jackson |
| `spring-boot-starter-data-jpa` | 数据库 ORM | Spring Data JPA + Hibernate |
| `spring-boot-starter-data-redis` | Redis | Lettuce 客户端 + Spring Data Redis |
| `spring-boot-starter-security` | 安全 | Spring Security |
| `spring-boot-starter-test` | 测试 | JUnit 5 + Mockito + AssertJ |
| `spring-boot-starter-actuator` | 生产监控 | 健康检查、指标、审计端点 |
| `spring-boot-starter-validation` | 参数校验 | Hibernate Validator（`@Valid`） |
| `spring-boot-starter-amqp` | 消息队列 | RabbitMQ |
| `spring-boot-starter-aop` | AOP | AspectJ 支持 |

> 💡 **记忆规律：** 想用什么功能，找 `spring-boot-starter-{功能名}` 就对了。

### 四、Starter 的原理：本质是一个 pom

starter 没有任何「魔法代码」，它**就是一个 pom 文件**，里面聚合了一堆传递依赖。以 `spring-boot-starter-web` 为例，它大致引入了：

```
spring-boot-starter-web
  ├─ spring-boot-starter          （核心：自动配置、日志）
  │    ├─ spring-boot
  │    ├─ spring-boot-autoconfigure
  │    └─ spring-core
  ├─ spring-boot-starter-json     （JSON：Jackson）
  ├─ spring-boot-starter-tomcat   （内嵌 Tomcat）
  ├─ spring-web                   （Web 基础）
  └─ spring-webmvc                （MVC）
```

所以引入一个 `starter-web`，就等于把这些都引入了，且**版本已经由 parent 协调好**。

> 💡 **提示：** 在 IDEA 里展开 `spring-boot-starter-web` 的依赖树，能清楚看到它「打包」了哪些库。这就是它省事的本质。

### 五、如何自定义 Starter

当你的团队有多个项目都要用同一套配置（如统一的 Redis 客户端、统一的日志切面），可以封装成自己的 starter，实现「**引入即生效**」。

#### 命名规范

> ⚠️ **注意：** 官方 starter 叫 `spring-boot-starter-xxx`。**第三方 starter 必须叫 `xxx-spring-boot-starter`**（名字在前），避免和官方混淆。

#### 三步走

1. **创建普通 Maven 模块** `my-feature-spring-boot-starter`，依赖 `spring-boot-autoconfigure`。
2. **写自动配置类**（带条件装配）：

   ```java
   @AutoConfiguration
   @ConditionalOnClass(MyFeature.class)        // classpath 有 MyFeature 才生效
   @EnableConfigurationProperties(MyFeatureProperties.class)
   public class MyFeatureAutoConfiguration {

       @Bean
       @ConditionalOnMissingBean               // 用户没自定义时才创建
       public MyFeature myFeature(MyFeatureProperties props) {
           return new MyFeature(props.getPrefix());
       }
   }
   ```

3. **注册自动配置**：在 `src/main/resources/META-INF/spring/` 下创建文件 `org.springframework.boot.autoconfigure.AutoConfiguration.imports`，内容：

   ```
   com.example.MyFeatureAutoConfiguration
   ```

之后任何项目引入这个 starter，`MyFeature` 就自动装配好了——和用官方 starter 体验一致。详见 [[自动配置原理]]。

### 六、常见问题与注意事项

> ⚠️ **注意：**
> - **版本要匹配**：Spring Boot 2.x 和 3.x 的 starter 不通用（3.x 基于 JDK 17 + Jakarta EE）。
> - **不要混用官方和第三方命名**：第三方 starter 必须是 `xxx-spring-boot-starter`。
> - starter 会引入「**可能用不到**」的依赖（比如 `starter-web` 引入了 Tomcat，哪怕你想换 Jetty）。可用 `<exclusions>` 排除不需要的部分。

> 💡 **提示：** 排查「某个 Bean 从哪来」时，看引入了哪些 starter，再结合 [[自动配置原理]] 的 `--debug` 启动日志，能快速定位。

### 七、总结

- **Starter = 场景化的依赖套餐**，一个坐标引入一整套协调好的依赖。
- **`starter-parent` 统一管版本**，引入依赖不用写 `version`。
- **想用什么找 `spring-boot-starter-{功能}`**：web、data-jpa、redis、security……
- **本质是 pom 聚合**，配合 [[自动配置原理]] 实现「引入即生效」。
- 自定义 starter 命名为 `xxx-spring-boot-starter`。
