# 静态属性（static 变量）

> 官方文档：[Java Language Specification - Fields](https://docs.oracle.com/javase/specs/jls/se17/html/jls-8.html#jls-8.3)、[菜鸟教程 Java 静态变量](https://www.runoob.com/java/java-static.html)

## 一、概述

在 Java 中，用 `static` 关键字修饰的成员变量称为**静态变量**（Static Variable），也叫**类变量**（Class Variable）。与之相对的是普通的**实例变量**（Instance Variable）。

> 💡 **提示：** 一个简单的类比：把一个班看成"类"，每个学生看成"对象"。实例变量就像每个学生自己的姓名、学号（每个对象一份）；静态变量就像班级名、班主任姓名（全班所有学生共享同一份）。

静态变量的核心特征是：**它属于类本身，而不属于任何一个对象**。无论创建多少个对象，静态变量在内存中只有一份，被所有对象共享。

## 二、核心概念与原理

### 1. 声明语法

```java
class Student {
    // 实例变量：每个对象各有一份
    String name;
    int age;

    // 静态变量：所有对象共享一份
    static String schoolName;
    static int count; // 用于统计创建了多少个 Student 对象
}
```

### 2. 内存图解

```text
┌──────────────────────────────────────────────────────────┐
│                      JVM 内存结构                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   堆（Heap）                                              │
│   ┌─────────────────────┐                               │
│   │  Student 对象 s1     │  ─── name="张三", age=18      │
│   ├─────────────────────┤                               │
│   │  Student 对象 s2     │  ─── name="李四", age=19      │
│   └─────────────────────┘                               │
│                                                          │
│   方法区/元空间（Metaspace，JDK 8+）                       │
│   ┌────────────────────────────────────────────────────┐ │
│   │  Student 类信息                                      │ │
│   │   ├── schoolName = "清华附中"  ← 静态变量（仅一份）   │ │
│   │   ├── count = 2                ← 静态变量（仅一份）   │ │
│   │   └── 方法字节码                                    │ │
│   └────────────────────────────────────────────────────┘ │
│           ▲              ▲                                │
│           │              │  所有对象都指向同一份静态变量     │
│        s1 ───────────────┴──────────────                  │
│        s2 ──────────────────────────────                  │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

> 💡 **提示：** 在 JDK 8 之前，静态变量存储在"永久代（PermGen）"中；从 **JDK 8 开始**，永久代被移除，类的元数据（包括静态变量关联的 Class 对象）存储在**元空间（Metaspace）**，而静态变量本身存储在与 Class 对象关联的堆内存区域。简言之：**静态变量跟随 Class 对象，生命周期与类一致**。

## 三、详细语法与用法

### 1. 访问方式

```java
public class Demo {
    public static void main(String[] args) {
        // ✅ 推荐：使用「类名.属性名」访问
        Student.schoolName = "清华附中";

        // ⚠️ 不推荐但合法：通过「对象.属性名」也能访问（容易让人误以为是实例变量）
        Student s1 = new Student();
        s1.schoolName = "北大附中"; // 语法合法，但不推荐

        System.out.println(Student.schoolName); // 北大附中
    }
}
```

### 2. 静态变量的初始化时机

静态变量在**类加载阶段**就被分配内存并初始化，整个过程只发生一次，顺序如下：

```text
类加载 ──► 1. 为静态变量分配内存，赋「默认值」（如 int → 0, 引用 → null）
         │
         ├──► 2. 执行静态变量的「显式赋值」和「静态代码块」（按代码书写顺序）
         │
         └──► 3. 初始化完成，静态变量可被使用
```

```java
class Config {
    static int a;              // 步骤 1：默认值 0
    static int b = 10;         // 步骤 2：显式赋值为 10
    static int c = initC();    // 步骤 2：调用方法赋值

    static {
        System.out.println("静态代码块执行，此时 a=" + a + ", b=" + b);
        a = 100;               // 静态代码块中再次赋值
    }

    static int initC() {
        return b + 5;
    }
}
```

## 四、代码示例：统计创建的对象数量

```java
public class Counter {
    private String name;
    // 静态变量：记录总共创建了多少个 Counter 对象
    private static int total = 0;

    public Counter(String name) {
        this.name = name;
        total++; // 每创建一个对象，静态变量 +1（所有对象共享）
        System.out.println("创建了对象：" + name + "，当前总数：" + total);
    }

    public static int getTotal() {
        return total;
    }

    public static void main(String[] args) {
        new Counter("对象A");
        new Counter("对象B");
        new Counter("对象C");
        // 直接通过类名访问静态方法
        System.out.println("最终创建的对象总数：" + Counter.getTotal());
    }
}
```

运行结果：

```text
创建了对象：对象A，当前总数：1
创建了对象：对象B，当前总数：2
创建了对象：对象C，当前总数：3
最终创建的对象总数：3
```

## 五、静态变量 vs 实例变量

| 对比维度 | 静态变量（static） | 实例变量（非 static） |
|---------|------------------|---------------------|
| 归属 | 属于类（Class） | 属于对象（Object） |
| 内存份数 | 全类共享一份 | 每个对象各一份 |
| 内存位置 | 元空间/与 Class 对象关联的堆区域 | 堆内存中对象内部 |
| 生命周期 | 类加载时创建，类卸载时销毁 | 对象创建时分配，对象被 GC 回收时销毁 |
| 访问方式 | `类名.变量名`（推荐） | `对象.变量名` |
| 默认值 | 有（int → 0, boolean → false, 引用 → null） | 有（同左） |
| 是否共享 | 共享 | 不共享 |
| 能否在静态方法中直接访问 | 能 | 不能（需要通过对象） |

## 六、应用场景

1. **对象计数器**：统计类的实例创建次数（见上方示例）。
2. **全局配置常量**：配合 `final` 定义全局不变的配置。
3. **工具类的共享数据**：如缓存、连接池配置。
4. **唯一标识生成**：自增 ID。

```java
public class Constants {
    // 配合 final 定义全局常量（静态常量）
    public static final double PI = 3.141592653589793;
    public static final String APP_NAME = "MyApplication";
    public static final int MAX_RETRY = 3;
}

public class User {
    private static int nextId = 1000; // 自增 ID 起点
    private int id;

    public User() {
        this.id = nextId++; // 每个用户分配唯一 ID
    }
}
```

## 七、注意事项与陷阱

> ⚠️ **注意：**
> 1. **不要用静态变量保存对象特有的状态**。静态变量是全局共享的，多线程环境下并发修改会有线程安全问题。
> 2. **通过对象访问静态变量不推荐**，因为会让人误以为它是对象的属性，且当引用为 `null` 时也不会抛 `NullPointerException`（因为编译期已绑定到类）。
> 3. **静态变量的修改会影响所有对象**，一处修改全局生效。

```java
Student s = null;
// ✅ 不会抛异常，因为静态变量访问不依赖对象
System.out.println(s.schoolName);
```

## 八、面试常见问题

### Q1：静态变量存放在内存的什么位置？

JDK 7 及之前存放在方法区（永久代）中；JDK 8 开始，永久代被移除，类的元数据存放在元空间（Metaspace，使用本地内存），而静态变量则跟随 Class 对象存储在堆内存的相关区域。总之静态变量的生命周期与**类**绑定，而非对象。

### Q2：静态变量和实例变量的区别？

见上方对比表格。核心区别：归属（类 vs 对象）、内存份数（一份 vs 多份）、生命周期（类加载到类卸载 vs 对象创建到对象回收）。

### Q3：在多线程环境下静态变量安全吗？

不安全。静态变量被所有线程共享，多个线程同时读写时会出现竞态条件。若需要线程安全，要使用 `synchronized`、`volatile`、`Atomic` 类或并发容器来保护。

### Q4：静态变量什么时候初始化？

在类加载阶段初始化，先赋予默认值，再按代码书写顺序执行显式赋值和静态代码块，整个过程只执行一次。

## 九、总结

- `static` 修饰的变量属于**类**，所有对象共享同一份数据。
- 推荐用 `类名.变量名` 访问，它在**类加载时初始化**，生命周期与类一致。
- 典型用途：对象计数器、全局常量、共享配置。
- 静态变量是"全局共享"的，要警惕多线程安全问题，不要用它保存对象特有的状态。
