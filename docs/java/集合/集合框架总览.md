# Java 集合框架总览

> 📖 官方文档：[Collections (Oracle)](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/package-summary.html)、[菜鸟教程 Java 集合](https://www.runoob.com/java/java-collections.html)

## 一、什么是集合框架

**集合（Collection）就是"用来装一堆对象的容器"**。当需要存储多个对象时，数组不够用（长度固定、类型单一、功能少），于是 Java 提供了一套丰富的集合类。

```
数组的局限：
  - 长度固定，不能自动扩容
  - 只能存同一类型
  - 增删元素麻烦（要手动移动）
  - 没有现成的查找、排序等方法

集合的优势：
  - 长度可变（自动扩容）
  - 只能存对象（基本类型用包装类）
  - 提供丰富的增删改查、排序、去重等方法
  - 有多种结构可选（数组、链表、树、哈希表）
```

> 💡 **类比：** 数组像"固定格子的书架"，集合像"各种功能的收纳工具箱"——有按顺序排的、有不重复的、有按名字快速找的，按需选用。

## 二、两大根接口：Collection 和 Map

Java 集合框架分**两大体系**，这是最重要的认知：

```
        ┌─────────────────────────────────────────┐
        │            集合框架两大体系               │
        ├──────────────────┬──────────────────────┤
        │   Collection     │       Map            │
        │  (单列数据)       │    (双列数据，键值对) │
        │  装一个个元素     │   装 key→value 映射   │
        └────────┬─────────┴──────────┬───────────┘
                 │                    │
        ┌────────┼────────┐    ┌──────┴──────┐
        ▼        ▼        ▼    ▼             ▼
      List      Set     Queue  HashMap     TreeMap
                                  ...
```

| 对比 | Collection 体系 | Map 体系 |
|-----|----------------|---------|
| 存储形式 | 单列（一个一个元素） | 双列（键值对 key-value） |
| 根接口 | `Collection<E>` | `Map<K,V>` |
| 子接口 | List、Set、Queue | 无子接口，直接是各种实现类 |
| 示例 | `[a, b, c]` | `{name=张三, age=20}` |

> ⚠️ **注意：Map 不继承 Collection**，它是一个独立的根接口。这点很多人搞混。

## 三、Collection 体系结构图

```
              ┌──────────────┐
              │  Iterable    │  ← 可迭代（支持 for-each）
              └──────┬───────┘
                     │ extends
              ┌──────▼───────┐
              │  Collection  │  ← 单列集合根接口
              └──┬─────┬───┬─┘
                 │     │   │ extends
        ┌────────┘     │   └────────────┐
        ▼ extends      ▼ extends        ▼ extends
   ┌─────────┐    ┌─────────┐     ┌──────────┐
   │  List   │    │   Set   │     │  Queue   │
   │有序可重复│    │无序不重复│     │ 队列结构  │
   └────┬────┘    └────┬────┘     └────┬─────┘
        │ implements    │ implements    │ implements
   ┌────┴────┐     ┌────┴──────┐   ┌───┴──────┐
   │         │     │           │   │          │
ArrayList  LinkedList  HashSet   │  ArrayDeque PriorityQueue
                    LinkedHashSet│
                    TreeSet      │ (Deque 继承 Queue)
```

### Collection 三大子接口

| 接口 | 特点 | 常用实现 |
|-----|------|---------|
| **List** | 有序、可重复、有索引 | ArrayList、LinkedList、Vector |
| **Set** | 无序（部分有序）、不可重复 | HashSet、LinkedHashSet、TreeSet |
| **Queue** | 队列（先进先出）/栈（后进先出） | ArrayDeque、LinkedList、PriorityQueue |

## 四、Map 体系结构图

```
              ┌──────────┐
              │   Map    │  ← 双列集合根接口（键值对）
              └─┬──┬──┬──┘
        ┌───────┘  │  └───────┐
        ▼          ▼          ▼
   ┌─────────┐ ┌─────────┐ ┌────────────────┐
   │ HashMap │ │ TreeMap │ │ConcurrentHashMap│
   └────┬────┘ └─────────┘ └────────────────┘
        │ extends
   ┌────▼─────────┐
   │ LinkedHashMap│
   └──────────────┘

   （还有 Hashtable：老的线程安全实现，已基本不用）
```

### Map 常用实现

| 实现类 | 特点 | 适用 |
|-------|------|------|
| **HashMap** | 无序、允许 null、非线程安全 | **最常用** |
| **LinkedHashMap** | 维护插入顺序 | 需要按插入顺序遍历 |
| **TreeMap** | 按 key 排序（红黑树） | 需要排序 |
| **ConcurrentHashMap** | 线程安全 | 多线程并发 |
| **Hashtable** | 线程安全（遗留） | 不推荐，用 ConcurrentHashMap |

## 五、Collection 接口的通用方法

凡是 `Collection` 体系下的集合（List/Set/Queue），都有这些方法：

```java
Collection<String> c = new ArrayList<>();

c.add("张三");              // 添加元素
c.add("李四");
c.remove("张三");           // 删除元素
c.contains("李四");         // 是否包含
c.size();                   // 元素个数
c.isEmpty();                // 是否为空
c.clear();                  // 清空

// 批量操作
c.addAll(otherCollection);          // 批量添加
c.removeAll(otherCollection);       // 批量删除（交集）
c.retainAll(otherCollection);       // 只保留交集
c.containsAll(otherCollection);     // 是否包含全部

// 转数组
Object[] arr = c.toArray();
String[] arr2 = c.toArray(new String[0]);

// 遍历
Iterator<String> it = c.iterator(); // 迭代器
```

### 遍历集合的四种方式

```java
List<String> list = new ArrayList<>(List.of("a", "b", "c"));

// 1. 增强 for 循环（for-each，最常用）
for (String s : list) {
    System.out.println(s);
}

// 2. 迭代器（能边遍历边删除）
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if (s.equals("b")) it.remove();   // ✅ 迭代器删除，安全
}

// 3. Lambda + forEach（Java 8+，最简洁）
list.forEach(s -> System.out.println(s));
list.forEach(System.out::println);   // 方法引用

// 4. 普通 for（仅 List 有索引）
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}
```

> ⚠️ **注意：** 用 for-each 遍历时**不能直接 `list.remove()`**，会抛 `ConcurrentModificationException`。要边遍历边删除，**必须用迭代器的 `it.remove()`**。

## 六、Map 接口的常用方法

```java
Map<String, Integer> map = new HashMap<>();

map.put("张三", 20);              // 添加/修改键值对
map.get("张三");                 // 根据 key 取 value（没有返回 null）
map.getOrDefault("王五", 0);     // 取不到时给默认值
map.remove("张三");              // 删除
map.containsKey("张三");         // 是否包含 key
map.containsValue(20);           // 是否包含 value
map.size();                      // 键值对个数
map.isEmpty();                   // 是否为空
map.clear();                     // 清空

// 遍历（三种）
for (String key : map.keySet()) { ... }              // 遍历 key
for (Integer v : map.values()) { ... }               // 遍历 value
for (Map.Entry<String, Integer> e : map.entrySet()) { // 遍历键值对（推荐）
    e.getKey();
    e.getValue();
}

// Java 8+ 简洁遍历
map.forEach((k, v) -> System.out.println(k + "=" + v));
```

## 七、如何选择集合（决策树）

```
要存的是"键值对"吗？
├─ 是 → Map
│      ├─ 要线程安全？→ ConcurrentHashMap
│      ├─ 要按 key 排序？→ TreeMap
│      ├─ 要按插入顺序？→ LinkedHashMap
│      └─ 都不需要 → HashMap（首选）
│
└─ 否 → Collection（单列）
         ├─ 元素唯一（去重）？→ Set
         │   ├─ 要排序？→ TreeSet
         │   ├─ 要插入顺序？→ LinkedHashSet
         │   └─ 都不需要 → HashSet（首选）
         │
         ├─ 是队列/栈结构？→ Queue/Deque
         │   ├─ 普通队列/栈 → ArrayDeque
         │   └─ 按优先级出队 → PriorityQueue
         │
         └─ 允许重复、按索引访问？→ List
              ├─ 查询多、增删尾部多 → ArrayList（首选）
              └─ 频繁头尾增删/当队列 → LinkedList
```

### 快速决策表

| 我想要... | 选择 |
|----------|------|
| 长度可变的"数组" | ArrayList |
| 频繁头尾增删 | LinkedList / ArrayDeque |
| 元素不重复 | HashSet |
| 元素不重复且排序 | TreeSet |
| 元素不重复且保持顺序 | LinkedHashSet |
| 键值对映射 | HashMap |
| 键值对 + 排序 | TreeMap |
| 键值对 + 顺序 | LinkedHashMap |
| 多线程下的键值对 | ConcurrentHashMap |

## 八、集合只能存对象，不能存基本类型

```java
// ❌ 集合不能直接存基本类型
List<int> list = new ArrayList<>();   // 编译错误

// ✅ 用包装类
List<Integer> list = new ArrayList<>();
list.add(1);            // 自动装箱 int → Integer
int n = list.get(0);    // 自动拆箱 Integer → int
```

| 基本类型 | 包装类 |
|---------|--------|
| int | Integer |
| long | Long |
| double | Double |
| boolean | Boolean |
| char | Character |
| byte/short/float | Byte/Short/Float |

> ⚠️ **注意：自动装箱拆箱有性能开销**，大量操作基本类型时，装箱拆箱会消耗内存和时间。

## 九、常见问题

### Q1：Collection 和 Collections 的区别？

```
Collection：接口，集合的根接口（单列集合）。
Collections：工具类，提供排序、查找、线程安全包装等静态方法。
（一个首字母大写接口，一个加 s 的工具类）
```

### Q2：数组和集合怎么互相转换？

```java
// 集合 → 数组
String[] arr = list.toArray(new String[0]);

// 数组 → 集合
List<String> list = Arrays.asList(arr);   // ⚠️ 返回的是固定大小，不能增删
List<String> list = new ArrayList<>(Arrays.asList(arr));  // ✅ 可变
List<String> list = List.of(arr);          // Java 9+，不可变
```

> ⚠️ **注意：** `Arrays.asList()` 返回的 List **长度固定，不能 add/remove**，否则报错。要可变就用 `new ArrayList<>(...)` 包装。

### Q3：HashMap 和 Hashtable 的区别？

```
HashMap：非线程安全，允许 null key/value，性能好（用这个）
Hashtable：线程安全（方法都 synchronized），不允许 null，已淘汰
（大小写不同：HashMap 大写 M，Hashtable 小写 t，注意拼写）
```

### Q4：fail-fast 是什么？

```
集合在遍历时，如果同时被修改（结构变化），会快速失败抛
ConcurrentModificationException，避免数据不一致。
多线程下要用 CopyOnWriteArrayList / ConcurrentHashMap。
```
