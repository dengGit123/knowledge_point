# 多态（Polymorphism）

> 官方文档：[Polymorphism (Oracle Java Tutorials)](https://docs.oracle.com/javase/tutorial/java/IandI/polymorphism.html) ｜ 菜鸟教程：[Java 多态](https://www.runoob.com/java/java-polymorphism.html)

## 一、概述

**多态（Polymorphism）** 是面向对象三大特性中最难理解、也是最精妙的一个。字面意思是"多种形态"，它指的是：**同一个对象，在不同时刻表现出不同的形态**。或者更准确地说：**父类引用指向子类对象，调用同一个方法时，会执行子类各自重写后的逻辑**。

通俗的类比：
- 你（父类引用）喊一句"叫一声"。
- 狗（子类对象 1）回答"汪汪"。
- 猫（子类对象 2）回答"喵喵"。
- 鸭子（子类对象 3）回答"嘎嘎"。

同样的指令、同样的调用代码，因为接收对象的不同，产生了不同的行为——这就是多态。

多态的意义在于：**让代码更灵活、更易扩展**。新增一种子类时，原有代码几乎不用修改。

## 二、多态的三个前提（缺一不可）

要发生多态，必须同时满足以下三个条件：

| 序号 | 前提条件 | 说明 |
|------|---------|------|
| ① | **继承或实现关系** | 子类继承父类，或类实现接口 |
| ② | **方法重写** | 子类必须重写父类（接口）的方法 |
| ③ | **父类引用指向子类对象** | `父类型 变量 = new 子类();`（向上转型） |

```java
// ① 继承关系
class Animal {
    // 父类方法
    public void speak() { System.out.println("动物叫"); }
}
class Dog extends Animal {
    // ② 方法重写
    @Override
    public void speak() { System.out.println("汪汪"); }
}
class Cat extends Animal {
    @Override
    public void speak() { System.out.println("喵喵"); }
}

public class Test {
    public static void main(String[] args) {
        // ③ 父类引用指向子类对象（向上转型）
        Animal a1 = new Dog();      // Dog 表现为 Animal
        Animal a2 = new Cat();      // Cat 表现为 Animal
        a1.speak();                 // 输出：汪汪（多态！）
        a2.speak();                 // 输出：喵喵（多态！）
    }
}
```

## 三、编译时类型 vs 运行时类型（核心原理）

理解多态的关键，是区分对象的**两种类型**：

| 类型 | 别名 | 决定时机 | 含义 |
|------|------|---------|------|
| **编译时类型** | 声明类型 / 静态类型 | 编译期 | 由 `=` 左边声明变量的类型决定 |
| **运行时类型** | 实际类型 / 动态类型 | 运行期 | 由 `=` 右边 `new` 出来的真实对象类型决定 |

```java
Animal a = new Dog();
//  ↑         ↑
// 编译时类型   运行时类型
// (Animal)   (Dog)
```

- **编译时类型** 决定了编译器"认为"这个变量能调用哪些方法（决定了**能调什么**）。
- **运行时类型** 决定了运行时实际执行的是哪个类的代码（决定了**执行什么**）。

```
           编译期（看左边 Animal）          运行期（看右边 Dog）
           ─────────────────────          ─────────────────────
   a.speak()  通过编译吗？  ─────────▶  实际执行 Dog.speak()
   a.bark()   编译报错！                （因为 Dog 重写了 speak）
   （Animal 里没有 bark）
```

## 四、向上转型与向下转型

### 4.1 向上转型（自动）

**子类对象赋值给父类引用**，这是自动完成的，不需要强制转换。这正是多态发生的前提。

```java
Animal a = new Dog();     // 向上转型，自动
Animal a2 = new Cat();    // 向上转型，自动
```

向上转型的"代价"：**父类引用只能调用父类中声明过的方法，不能调用子类独有的方法**。

```java
Animal a = new Dog();
a.speak();             // ✅ Animal 有 speak()
// a.bark();           // ❌ 编译错误：Animal 中没有 bark()，尽管实际对象是 Dog
```

### 4.2 向下转型（强制）

把**父类引用转回子类类型**，必须**显式强转**。常用于多态后想调用子类独有方法的场景。

```java
Animal a = new Dog();
Dog d = (Dog) a;        // 向下转型，强制
d.bark();               // ✅ 现在可以调用 Dog 独有的 bark() 了
```

> ⚠️ **注意：** 向下转型**有风险**！如果实际对象和强转的目标类型不匹配，运行时会抛出 `ClassCastException`（类型转换异常）：

```java
Animal a = new Dog();
Cat c = (Cat) a;        // ❌ 运行时抛出 ClassCastException：Dog 不能转成 Cat
```

### 4.3 instanceof 检查（安全转型）

向下转型前，**务必先用 `instanceof` 判断**对象的实际类型，避免转换异常：

```java
Animal a = new Dog();
if (a instanceof Dog) {
    Dog d = (Dog) a;
    d.bark();
}
```

**Java 16+ 模式匹配 instanceof（Pattern Matching）**：可以让代码更简洁，转型同时绑定变量：

```java
// Java 16+ 语法：instanceof 同时完成判断和绑定，无需再手动强转
Object obj = "hello";
if (obj instanceof String s) {     // s 自动是 String 类型
    System.out.println(s.length());
}
```

## 五、动态绑定（虚方法调用）

多态的底层机制叫**动态绑定（Dynamic Binding）**，也叫**虚方法调用（Virtual Method Call）**。

- **静态绑定**：在**编译期**就确定调用哪个方法。典型是 `static` 方法、`private` 方法、`final` 方法（这些方法不会发生多态）。
- **动态绑定**：在**运行期**，根据对象的**实际类型**决定调用哪个类的重写方法。

```
   编译：Animal a = new Dog();
        a.speak();   ──▶ 编译期只校验 Animal 有没有 speak（静态绑定不了，是动态绑定）

   运行：JVM 查 a 指向的实际对象是 Dog
        ──▶ 调用 Dog 的 speak()，而不是 Animal 的 speak()
```

> 💡 **提示：** Java 中所有**非 final、非 static、非 private 的实例方法**默认都是虚方法，都支持动态绑定，这也是 Java 多态几乎"免费"的原因（C++ 需要显式 `virtual` 关键字）。

## 六、多态的成员访问特点（★★★ 重点）

这是多态最容易考、也最容易踩坑的地方。用一句口诀概括：

> **成员变量：看左边（编译时类型）**
> **成员方法：看右边（运行时类型）**
> **静态方法：看左边（编译时类型）**

### 6.1 完整对比示例

```java
class Father {
    int num = 10;                       // 父类成员变量
    public void method() {              // 父类普通方法（可被子类重写）
        System.out.println("父类 method");
    }
    public static void staticMethod() { // 父类静态方法
        System.out.println("父类 staticMethod");
    }
}

class Son extends Father {
    int num = 20;                       // 子类成员变量（同名）
    @Override
    public void method() {              // 重写父类方法
        System.out.println("子类 method");
    }
    public static void staticMethod() { // 隐藏父类静态方法（不是重写）
        System.out.println("子类 staticMethod");
    }
}

public class Test {
    public static void main(String[] args) {
        Father f = new Son();           // 父类引用指向子类对象
        System.out.println(f.num);      // ① 10（成员变量看左边）
        f.method();                     // ② 子类 method（方法看右边）
        f.staticMethod();               // ③ 父类 staticMethod（静态方法看左边）
    }
}
```

### 6.2 成员访问特点对照表

| 成员类型 | 看哪边 | 原因 | 示例结果 |
|---------|--------|------|---------|
| **成员变量** | 看左边（编译时类型 Father） | 成员变量不具备多态，编译期就绑定 | `f.num` → **10** |
| **非静态方法** | 看右边（运行时类型 Son） | 动态绑定，运行时根据实际对象决定 | `f.method()` → **子类 method** |
| **静态方法** | 看左边（编译时类型 Father） | 静态方法属于类，是静态绑定 | `f.staticMethod()` → **父类 staticMethod** |

> ⚠️ **注意：** 很多人误以为"父类引用指向子类对象后，访问 `num` 会得到 20"。这是错的！成员变量**没有多态性**，编译期就由声明类型决定了。

### 6.3 为什么成员变量和静态方法不支持多态？

- **成员变量**：编译时就根据变量的**声明类型**决定访问哪个类的字段。对象里虽然同时存有父类和子类的 num，但编译期已确定引用 Father 的 num。
- **静态方法**：属于**类**，不属于对象，调用时根本不依赖对象实例，所以谈不上"运行时根据对象类型决定"，自然是静态绑定（看左边）。

## 七、多态的应用场景

多态在实际开发中应用极其广泛，主要体现在三个方面。

### 7.1 方法参数多态（最常用）

方法形参声明为父类类型，可以接收任何子类对象，极大提高方法的**通用性**和**扩展性**。

```java
class Animal { public void speak() {} }
class Dog extends Animal { public void speak() { System.out.println("汪"); } }
class Cat extends Animal { public void speak() { System.out.println("喵"); } }

public class Vet {
    // 形参用父类 Animal，无论传入 Dog、Cat 还是将来新增的 Pig，都能处理
    public void treat(Animal a) {
        System.out.print("医生让动物叫：");
        a.speak();        // 多态：不同动物不同叫声
    }
}

// 调用
Vet vet = new Vet();
vet.treat(new Dog());     // 输出：医生让动物叫：汪
vet.treat(new Cat());     // 输出：医生让动物叫：喵
```

### 7.2 数组 / 集合存储父类类型

用一个父类类型的数组，统一管理各种子类对象：

```java
Animal[] zoo = new Animal[3];
zoo[0] = new Dog();
zoo[1] = new Cat();
zoo[2] = new Pig();        // 假设有 Pig 类

for (Animal a : zoo) {
    a.speak();             // 多态：每个动物叫出自己的声音
}
```

### 7.3 返回值多态

方法返回父类类型，实际可以返回任意子类对象（典型的工厂模式）：

```java
public Animal createAnimal(String type) {
    if ("dog".equals(type)) return new Dog();
    if ("cat".equals(type)) return new Cat();
    return new Animal();
}
```

## 八、多态的好处

| 好处 | 说明 |
|------|------|
| **提高扩展性** | 新增子类时，无需修改使用父类的已有代码（符合开闭原则） |
| **提高灵活性** | 同一段代码可处理多种子类对象 |
| **提高代码复用** | 父类引用统一管理，减少重复代码 |
| **解耦** | 调用者只依赖父类/接口，不依赖具体子类，降低耦合 |

> 💡 **提示：** 多态的本质是 **"面向抽象编程，而不是面向具体编程"**。这正是设计模式（如策略模式、工厂模式、模板方法模式）的基础。

## 九、注意事项与陷阱

1. **多态不适用于成员变量和静态方法**（见第六节），只适用于非静态的实例方法。
2. **多态后不能调用子类独有方法**，除非向下转型。
3. **向下转型前必须 `instanceof` 判断**，避免 `ClassCastException`。
4. **`private`、`static`、`final` 方法不参与多态**：
   - `private` 子类不可见，谈不上重写；
   - `static` 属于类，静态绑定；
   - `final` 禁止重写。
5. **不要为了多态而多态**：只有在需要"统一处理多种子类"的场景才用，否则徒增复杂度。

```java
class Father {
    public final void finalMethod() { }        // final 方法，不可重写，无多态
    public static void staticMethod() { }      // static 方法，静态绑定，无多态
    private void privateMethod() { }           // private，子类不可见，无多态
}
class Son extends Father {
    // 无法重写上面三个方法
}
```

## 十、面试常见问题

### Q1：多态的前提条件是什么？

三个：① 存在继承或实现关系；② 子类重写了父类（接口）的方法；③ 父类引用指向子类对象（向上转型）。三者缺一不可，缺少任何一个都不会发生多态。

### Q2：多态中成员变量、普通方法、静态方法分别访问哪个类的？（最常考）

- **成员变量**：看左边（编译时类型），编译期绑定，无多态。
- **普通方法**：看右边（运行时类型），动态绑定，这才是多态的体现。
- **静态方法**：看左边（编译时类型），静态方法属于类，静态绑定，无多态。

口诀："变量看左、方法看右、静态看左"。

### Q3：Java 多态的实现原理是什么？什么是动态绑定？

多态底层靠**动态绑定（虚方法调用）**实现。编译时，编译器只根据声明类型校验方法是否存在，但**不绑定**到具体实现；运行时，JVM 根据对象的**实际类型**，在方法表中查找并调用对应子类重写后的方法。这就是为什么"方法看右边"。

### Q4：向下转型为什么需要 instanceof？

因为向下转型有 `ClassCastException` 风险——如果对象的实际类型和目标类型不匹配（比如 Dog 强转成 Cat），运行时就会抛异常。用 `instanceof` 先判断实际类型，可以安全地进行转型。Java 16+ 还支持 `instanceof X x` 模式匹配，判断和转型一步到位。

### Q5：多态后能不能调用子类特有的方法？

不能直接调用。父类引用只能看到父类中声明的方法，调用子类独有方法会编译报错。必须**向下转型**为子类类型后才能调用。这就是为什么需要向下转型。

### Q6：静态方法、final 方法、private 方法能实现多态吗？

都不能。
- **静态方法**属于类、不属于对象，是静态绑定，看左边。
- **final 方法**禁止被重写，没有"子类版本"可言。
- **private 方法**对子类不可见，无法被重写。
只有**非 final、非 static、非 private 的实例方法**才能多态。

### Q7：多态的好处是什么？

提高代码的扩展性、灵活性和复用性，降低耦合。新增一种子类时，处理父类引用的旧代码几乎不用修改，符合"开闭原则"（对扩展开放，对修改关闭）。这也是各种设计模式的基石。

## 十一、总结

| 知识点 | 核心要点 |
|--------|---------|
| 多态三前提 | 继承/实现 + 方法重写 + 父类引用指向子类对象 |
| 两种类型 | 编译时类型看左边（声明），运行时类型看右边（实际） |
| 向上转型 | 子→父，自动；只能调父类方法 |
| 向下转型 | 父→子，强转；必须先 instanceof 检查 |
| 动态绑定 | 运行时根据实际对象类型决定调用哪个重写方法 |
| 成员变量 | 看左边（编译时类型） |
| 普通方法 | 看右边（运行时类型）—— 多态的核心 |
| 静态方法 | 看左边（编译时类型） |
| 应用场景 | 方法参数、数组/集合、返回值多态 |
| 设计意义 | 面向抽象编程，提高扩展性、灵活性、解耦 |

多态是面向对象的灵魂，它让代码"面向抽象而不是面向具体"。彻底理解"成员变量看左、方法看右、静态看左"以及动态绑定机制，你就真正掌握了 Java 多态。
