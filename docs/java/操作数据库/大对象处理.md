### 一、概述

> 📖 [Blob（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.sql/java/sql/Blob.html) ｜ [Clob（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.sql/java/sql/Clob.html)

数据库除了存普通文本和数字，还能存「**大对象**」——大段文本或二进制文件。JDBC 用 **Clob** 和 **Blob** 来处理它们。

大白话：
- **Clob**（Character Large Object）= **大文本**（一篇文章、一个 JSON、几 MB 的小说）
- **Blob**（Binary Large Object）= **大二进制**（图片、视频、PDF、压缩包）

| 类型 | 全称 | 存什么 | 数据库类型 |
| --- | --- | --- | --- |
| **Clob** | Character Large Object | **大段文本** | TEXT / CLOB / LONGTEXT |
| **Blob** | Binary Large Object | **二进制文件** | BLOB / LONGBLOB |

> 💡 **提示：** 小文本直接用 VARCHAR，**超过普通文本长度**（如几万字文章、大 JSON）才用 Clob；图片文件用 Blob。

### 二、Blob：存取二进制（图片/文件）

#### 1. 存储 Blob（上传图片到数据库）

用 `setBinaryStream` 把文件的字节流写入 Blob 字段：

```java
import java.io.*;
import java.sql.*;

public class BlobWriteDemo {
    public static void main(String[] args) throws Exception {
        try (Connection conn = getConnection();
             PreparedStatement ps = conn.prepareStatement(
                 "INSERT INTO avatar(user_id, image) VALUES(?, ?)")) {

            File file = new File("avatar.png");

            ps.setInt(1, 1001);

            // ✅ 方式1：用流写入（推荐）
            try (FileInputStream fis = new FileInputStream(file)) {
                ps.setBinaryStream(2, fis, (int) file.length());
                ps.executeUpdate();
            }

            // 方式2：用 setBlob
            // try (InputStream in = new FileInputStream(file)) {
            //     ps.setBlob(2, in);
            // }
        }
    }
}
```

#### 2. 读取 Blob（从数据库取图片）

用 `getBinaryStream` 把 Blob 字段读成流，写入文件：

```java
try (Connection conn = getConnection();
     PreparedStatement ps = conn.prepareStatement(
         "SELECT image FROM avatar WHERE user_id = ?")) {

    ps.setInt(1, 1001);
    try (ResultSet rs = ps.executeQuery()) {
        if (rs.next()) {
            // ✅ 读取 Blob 为流，写入文件
            try (InputStream in = rs.getBinaryStream("image");
                 FileOutputStream out = new FileOutputStream("download.png")) {
                byte[] buffer = new byte[4096];
                int len;
                while ((len = in.read(buffer)) != -1) {
                    out.write(buffer, 0, len);
                }
            }
            System.out.println("图片已保存");
        }
    }
}
```

> 💡 **提示：** Blob 操作的本质是**字节流读写**（参考 [[../流/字节流之输入流]]、[[../流/缓冲流]]）。用 `setBinaryStream` / `getBinaryStream` 最直接。

### 三、Clob：存取大文本

#### 1. 存储 Clob

```java
try (Connection conn = getConnection();
     PreparedStatement ps = conn.prepareStatement(
         "INSERT INTO article(title, content) VALUES(?, ?)")) {

    ps.setString(1, "我的文章");
    String longText = "这里是几万字的长文章内容...";

    // ✅ 用 setCharacterStream 写入字符流
    try (Reader reader = new StringReader(longText)) {
        ps.setCharacterStream(2, reader, longText.length());
        ps.executeUpdate();
    }

    // 或直接 setString（文本不太大时）
    // ps.setString(2, longText);
}
```

#### 2. 读取 Clob

```java
try (PreparedStatement ps = conn.prepareStatement(
        "SELECT content FROM article WHERE id = ?")) {
    ps.setInt(1, 1);
    try (ResultSet rs = ps.executeQuery()) {
        if (rs.next()) {
            // ✅ 方式1：直接读成字符串
            String content = rs.getString("content");

            // 方式2：用 Reader 读字符流（大文本推荐）
            // try (Reader reader = rs.getCharacterStream("content")) {
            //     // 读取处理
            // }
        }
    }
}
```

> 💡 **提示：** 文本不是特别大时，Clob 可以直接用 `setString` / `getString`，比流方式简单。

### 四、setBlob/setClob vs setStream

写入大对象有几种方式，按推荐度：

| 写法 | 说明 |
| --- | --- |
| `setBinaryStream(i, stream)` | ✅ 最通用，字节流写 Blob |
| `setBlob(i, stream)` | 显式用 Blob，效果同上 |
| `setBytes(i, bytes)` | 小文件可全量读成 byte[] 写入 |
| `setCharacterStream(i, reader)` | ✅ 字符流写 Clob |
| `setString(i, text)` | 小文本直接写 |

读取同理：`getBinaryStream` / `getCharacterStream` / `getString` / `getBytes`。

> 💡 **提示：** **大文件优先用流（Stream）**，避免一次性全读进内存导致 OOM。

### 五、内存与流式读取

处理大 Blob/Clob 时，**务必用流式读写**，分块处理：

```java
// ✅ 大文件：用缓冲区分块读写，不占大内存
try (InputStream in = rs.getBinaryStream("file");
     FileOutputStream out = new FileOutputStream("big.zip")) {
    byte[] buffer = new byte[8192];
    int len;
    while ((len = in.read(buffer)) != -1) {
        out.write(buffer, 0, len);
    }
}

// ❌ 小心：超大文件一次读完会 OOM
// byte[] all = rs.getBytes("file");   // 文件大就炸
```

详见 [[../流/缓冲流]]。

### 六、实际应用场景

| 场景 | 类型 |
| --- | --- |
| 用户头像/证件照 | Blob |
| 文章/富文本内容 | Clob |
| 附件上传（PDF/Word） | Blob |
| 存大 JSON / XML | Clob |
| 视频/音频 | Blob（一般不建议存数据库） |

> ⚠️ **注意：** **大文件（视频、大图）一般不建议直接存数据库**——会让数据库膨胀、备份困难、性能下降。常见做法是：**文件存文件系统/对象存储（OSS），数据库只存文件路径/URL**。只有小图片、必要的小附件才考虑存数据库。

### 七、数据库类型对照

不同数据库的大对象类型：

| 数据库 | 大文本 | 大二进制 |
| --- | --- | --- |
| MySQL | TEXT / MEDIUMTEXT / LONGTEXT | BLOB / MEDIUMBLOB / LONGBLOB |
| Oracle | CLOB | BLOB |
| PostgreSQL | TEXT | BYTEA |
| SQL Server | NVARCHAR(MAX) | VARBINARY(MAX) |

#### MySQL 各 TEXT/BLOB 容量

| 类型 | 最大长度 |
| --- | --- |
| TINYTEXT / TINYBLOB | 255 字节 |
| TEXT / BLOB | 64 KB |
| MEDIUMTEXT / MEDIUMBLOB | 16 MB |
| LONGTEXT / LONGBLOB | 4 GB |

### 八、常见问题与注意事项

#### 1. 内存溢出

一次性 `getBytes` 读取超大 Blob 会 OOM。**用流式读取**。

#### 2. 数据库表膨胀

大量大文件入数据库，数据库体积暴涨，备份/查询变慢。**大文件存文件系统**，DB 存路径。

#### 3. 字符编码

Clob 读写涉及编码，注意指定 UTF-8（参考 [[../流/字符编码]]）。

#### 4. 连接超时

传超大文件耗时长，可能连接超时。调大数据库和连接池的超时配置。

#### 5. 事务回滚段

大对象在事务里写入，回滚段会很大。大文件操作要单独事务、及时提交。

---

### 九、总结

| 要点 | 说明 |
| --- | --- |
| Clob | 大文本（TEXT/CLOB） |
| Blob | 大二进制（图片/文件，BLOB） |
| 写入 | `setBinaryStream` / `setCharacterStream`（流式） |
| 读取 | `getBinaryStream` / `getCharacterStream` / `getString` |
| 大文件 | 用流式分块读写，避免 OOM |
| 最佳实践 | 大文件存文件系统，DB 存路径 |

一句话：**小文件/必要附件可用 Blob/Clob 存数据库；大文件存文件系统，DB 只存 URL**。操作时用流式读写。
