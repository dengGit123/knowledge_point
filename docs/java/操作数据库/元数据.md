### 一、概述

> 📖 [DatabaseMetaData（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.sql/java/sql/DatabaseMetaData.html) ｜ [ResultSetMetaData（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.sql/java/sql/ResultSetMetaData.html)

「元数据」（MetaData）就是「**描述数据的数据**」。在 JDBC 里，它指**描述数据库结构的信息**——有哪些表、表有哪些列、列是什么类型等。

大白话：数据是「**用户表里有张三**」，元数据是「**有个叫 user 的表，它有 id、name、age 三列，id 是 INT 类型**」。元数据是「数据的说明书」。

JDBC 提供三种元数据接口：

| 元数据 | 描述什么 | 获取方式 |
| --- | --- | --- |
| `DatabaseMetaData` | **数据库**的信息（库名、版本、有哪些表） | `conn.getMetaData()` |
| `ResultSetMetaData` | **结果集**的信息（几列、列名、列类型） | `rs.getMetaData()` |
| `ParameterMetaData` | **PreparedStatement 参数**的信息 | `ps.getParameterMetaData()` |

> 💡 **提示：** 元数据用得不如 CRUD 频繁，但在「动态处理未知结构的表」「写数据库工具/框架」时必不可少。MyBatis、Hibernate 底层都大量用元数据。

---

### 二、DatabaseMetaData（数据库元数据）

`DatabaseMetaData` 描述**数据库本身**的信息。

#### 获取方式

```java
DatabaseMetaData dbmd = conn.getMetaData();
```

#### 常用功能

```java
// ① 数据库基本信息
System.out.println("数据库名: " + dbmd.getDatabaseProductName());   // MySQL
System.out.println("数据库版本: " + dbmd.getDatabaseProductVersion());
System.out.println("驱动名: " + dbmd.getDriverName());
System.out.println("用户名: " + dbmd.getUserName());

// ② 支持的特性
System.out.println("支持事务: " + dbmd.supportsTransactions());
System.out.println("支持批处理: " + dbmd.supportsBatchUpdates());
System.out.println("支持存储过程: " + dbmd.supportsStoredProcedures());
```

#### 获取所有表

```java
// ✅ 查询当前库的所有表
ResultSet tables = dbmd.getTables(
    null, null, "%", new String[]{"TABLE"});

while (tables.next()) {
    String tableName = tables.getString("TABLE_NAME");
    System.out.println("表: " + tableName);
}
```

#### 获取表的列信息

```java
// ✅ 查询某张表的所有列
ResultSet columns = dbmd.getColumns(null, null, "user", "%");
while (columns.next()) {
    String colName = columns.getString("COLUMN_NAME");
    String colType = columns.getString("TYPE_NAME");
    String remark  = columns.getString("REMARKS");   // 列注释
    System.out.println(colName + " : " + colType + " (" + remark + ")");
}
```

> 💡 **提示：** `getTables` / `getColumns` 用通配符 `%` 匹配。这在「数据库表结构对比」「代码生成器」里很有用。

### 三、ResultSetMetaData（结果集元数据）⭐

`ResultSetMetaData` 描述**查询结果集的结构**——最常用的元数据。

#### 获取方式

```java
ResultSet rs = ps.executeQuery();
ResultSetMetaData rsmd = rs.getMetaData();
```

#### 常用功能

```java
// ① 结果集有几列
int columnCount = rsmd.getColumnCount();
System.out.println("列数: " + columnCount);

// ② 遍历每列的信息
for (int i = 1; i <= columnCount; i++) {     // ✅ 列号从 1 开始
    String name = rsmd.getColumnLabel(i);     // 列名/别名
    String type = rsmd.getColumnTypeName(i);  // 列类型名（VARCHAR/INT）
    int sqlType = rsmd.getColumnType(i);      // 类型常量（java.sql.Types.VARCHAR）
    System.out.println(name + " : " + type);
}
```

#### 用元数据动态遍历结果集（通用查询）

```java
// ✅ 不知道有哪些列，用元数据动态读取（万能查询）
public List<Map<String, Object>> queryToList(String sql) throws SQLException {
    List<Map<String, Object>> result = new ArrayList<>();
    try (PreparedStatement ps = conn.prepareStatement(sql);
         ResultSet rs = ps.executeQuery()) {

        ResultSetMetaData md = rs.getMetaData();
        int count = md.getColumnCount();

        while (rs.next()) {
            Map<String, Object> row = new LinkedHashMap<>();
            for (int i = 1; i <= count; i++) {
                row.put(md.getColumnLabel(i), rs.getObject(i));   // 列名 → 值
            }
            result.add(row);
        }
    }
    return result;
}
```

> 💡 **提示：** 上面这个「ResultSet → List<Map>」的通用转换，就是很多简易 ORM/工具类的基础。它的核心就是用 `ResultSetMetaData` 动态获取列信息。MyBatis 底层也是这么干的。

#### getColumnLabel vs getColumnName

| 方法 | 说明 |
| --- | --- |
| `getColumnName(i)` | 列的**真实名字** |
| `getColumnLabel(i)` | 列的**别名**（推荐用） |

```sql
SELECT name AS username FROM user
```

```java
md.getColumnName(1);   // name（真实名）
md.getColumnLabel(1);  // username（别名）
```

> ⚠️ **注意：** 推荐 `getColumnLabel`，因为它返回别名，更贴合「我查询时怎么命名的」。

### 四、ParameterMetaData（参数元数据）

`ParameterMetaData` 描述 **PreparedStatement 的参数**信息。

```java
PreparedStatement ps = conn.prepareStatement("INSERT INTO user(name, age) VALUES(?, ?)");
ParameterMetaData pmd = ps.getParameterMetaData();

int paramCount = pmd.getParameterCount();   // 2 个参数
System.out.println("参数个数: " + paramCount);
```

> 💡 **提示：** ParameterMetaData 用得少（因为参数都是我们自己 set 的，已经知道类型）。MySQL 驱动对它的支持也有限，了解即可。

### 五、元数据的应用场景

| 场景 | 用什么 |
| --- | --- |
| 写通用查询工具 | ResultSetMetaData（动态列） |
| ORM 框架底层 | ResultSetMetaData（对象映射） |
| 数据库表结构浏览工具 | DatabaseMetaData |
| 代码生成器（根据表生成实体类） | DatabaseMetaData |
| 数据库迁移/对比工具 | DatabaseMetaData |
| 动态报表 | ResultSetMetaData |

#### 实战：根据表生成实体类（代码生成器原理）

```java
// 根据 user 表的字段，动态生成 Java 类代码
DatabaseMetaData dbmd = conn.getMetaData();
ResultSet columns = dbmd.getColumns(null, null, "user", "%");

StringBuilder clazz = new StringBuilder("public class User {\n");
while (columns.next()) {
    String colName = columns.getString("COLUMN_NAME");
    String colType = columns.getString("TYPE_NAME");
    String javaField = toCamelCase(colName);          // user_name → userName
    String javaType  = sqlTypeToJava(colType);         // VARCHAR → String
    clazz.append("    private ").append(javaType)
         .append(" ").append(javaField).append(";\n");
}
clazz.append("}");
// 输出生成的实体类代码
```

这就是 MyBatis Generator、JPA 实体生成等工具的底层原理。

### 六、java.sql.Types（类型常量）

`getColumnType` 返回的是 `java.sql.Types` 里的常量：

| 常量 | 对应 Java 类型 |
| --- | --- |
| `Types.INTEGER` | int / Integer |
| `Types.BIGINT` | long / Long |
| `Types.VARCHAR` / `CHAR` | String |
| `Types.DATE` | java.sql.Date |
| `Types.TIMESTAMP` | Timestamp |
| `Types.DOUBLE` / `FLOAT` | double |
| `Types.BOOLEAN` | boolean |
| `Types.BLOB` | byte[] |

```java
int type = rsmd.getColumnType(i);
if (type == Types.VARCHAR) { /* 字符串 */ }
```

### 七、常见问题与注意事项

#### 1. 列号从 1 开始

ResultSetMetaData 和 ParameterMetaData 的索引**都从 1 开始**（和 ResultSet 一致）。

#### 2. 元数据查询有性能开销

`getTables` / `getColumns` 会查数据库的系统表，频繁调用较慢。结果可**缓存**复用。

#### 3. 不同数据库差异

不同数据库的元数据返回有差异（类型名、是否支持某些方法）。写跨数据库工具要测试。

#### 4. getColumnLabel 更可靠

用 `getColumnLabel` 而非 `getColumnName`，避免别名/重名导致的混淆。

#### 5. 不要滥用元数据

能直接知道列名就别用元数据动态获取。元数据主要用于「**处理未知结构**」的通用场景。

---

### 八、总结

| 元数据 | 描述 | 获取 | 主要用途 |
| --- | --- | --- | --- |
| **DatabaseMetaData** | 数据库信息 | `conn.getMetaData()` | 查表/列、库信息 |
| **ResultSetMetaData** | 结果集结构 | `rs.getMetaData()` | 动态遍历、ORM |
| **ParameterMetaData** | 参数信息 | `ps.getParameterMetaData()` | 少用 |

一句话：**要动态处理「不知道结构」的数据，用元数据**。它是工具类、ORM 框架、代码生成器的基石。
