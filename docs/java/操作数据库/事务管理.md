### 一、概述

> 📖 [JDBC 事务（Oracle 教程）](https://docs.oracle.com/javase/tutorial/jdbc/basics/transactions.html)

事务（Transaction）是数据库操作的**基本执行单位**，保证「**要么全部成功，要么全部失败**」。JDBC 提供了完整的事务控制 API。

大白话：转账时，A 扣钱和 B 加钱必须**同时成功或同时失败**——A 扣了钱但 B 没加上，钱就消失了。事务就是把这「两步」**打包成一个不可分割的整体**，任何一步失败，全部回滚。

| 你将学到 | 说明 |
| --- | --- |
| 事务的 ACID 特性 | 原子性、一致性、隔离性、持久性 |
| JDBC 事务 API | setAutoCommit / commit / rollback |
| 隔离级别 | 4 个级别、各解决什么并发问题 |
| Savepoint | 部分回滚 |

> 💡 **提示：** JDBC 默认每条 SQL 自动提交（一条一事务）。要开启「多语句一个事务」，得手动 `setAutoCommit(false)`。

---

### 二、事务的 ACID 特性

| 特性 | 英文 | 含义 |
| --- | --- | --- |
| **原子性** | Atomicity | 事务内操作**要么全做，要么全不做** |
| **一致性** | Consistency | 事务前后，数据**保持一致**（如转账总金额不变） |
| **隔离性** | Isolation | 多个事务**并发互不干扰** |
| **持久性** | Durability | 事务提交后，**永久保存**（断电不丢） |

记忆：**ACID**（酸）。

---

### 三、JDBC 事务 API

#### 核心 API

| 方法 | 作用 |
| --- | --- |
| `setAutoCommit(false)` | **关闭自动提交**，开启手动事务 |
| `commit()` | **提交**事务（全部成功时） |
| `rollback()` | **回滚**事务（出错时撤销全部） |
| `setSavepoint()` | 设置保存点（部分回滚） |

#### 默认行为：自动提交

```java
// 默认 autoCommit = true，每条 SQL 执行完立即提交
stmt.executeUpdate("UPDATE account SET money = money - 100 WHERE id = 1");  // 立即提交
stmt.executeUpdate("UPDATE account SET money = money + 100 WHERE id = 2");  // 立即提交
// 两条 SQL 各自独立提交，不是同一个事务
```

### 四、标准事务流程（转账示例）

```java
import java.sql.*;

public class TransferDemo {
    public static void main(String[] args) {
        try (Connection conn = DriverManager.getConnection(url, user, pwd)) {
            // ① ✅ 开启事务：关闭自动提交
            conn.setAutoCommit(false);

            try (PreparedStatement ps = conn.prepareStatement(
                    "UPDATE account SET money = money + ? WHERE id = ?")) {

                // A 扣 100
                ps.setInt(1, -100);
                ps.setInt(2, 1);
                ps.executeUpdate();

                // 模拟异常
                if (true) throw new RuntimeException("转账中途出错");

                // B 加 100
                ps.setInt(1, 100);
                ps.setInt(2, 2);
                ps.executeUpdate();

                // ② 全部成功，提交
                conn.commit();
                System.out.println("转账成功");
            } catch (Exception e) {
                // ③ 出错，回滚（A 扣的钱会退回）
                conn.rollback();
                System.out.println("转账失败，已回滚");
                e.printStackTrace();
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
```

#### 事务的标准套路

```
1. setAutoCommit(false)   ← 开启事务
2. 执行多条 SQL
3. 全部成功 → commit()
   出现异常 → rollback()
4. （可选）finally 里恢复 autoCommit = true
```

> 💡 **提示：** 事务的**提交/回滚是 Connection 级别**的。一个 Connection 内的所有操作属于同一个事务。

### 五、并发问题与隔离级别

多个事务同时操作同一数据时，会产生并发问题。隔离级别越高，越安全，但性能越低。

#### 1. 四种并发问题

| 问题 | 说明 |
| --- | --- |
| **脏读** | 读到别的事务**未提交**的数据（对方回滚了，你读到的是脏数据） |
| **不可重复读** | 同一事务内，两次读同一行，**结果不同**（被别的事务修改/删除了） |
| **幻读** | 同一事务内，两次查询，**行数不同**（被别的事务插入/删除了） |
| 丢失更新 | 两个事务同时改，后提交的覆盖前者 |

#### 2. 四种隔离级别

从低到高：

| 隔离级别 | 脏读 | 不可重复读 | 幻读 | 性能 |
| --- | --- | --- | --- | --- |
| **READ_UNCOMMITTED** 读未提交 | ❌ 可能 | ❌ 可能 | ❌ 可能 | 最高 |
| **READ_COMMITTED** 读已提交 | ✅ 防止 | ❌ 可能 | ❌ 可能 | 高 |
| **REPEATABLE_READ** 可重复读 | ✅ 防止 | ✅ 防止 | ❌ 可能 | 中 |
| **SERIALIZABLE** 串行化 | ✅ 防止 | ✅ 防止 | ✅ 防止 | 最低 |

> 💡 **提示：** MySQL 默认隔离级别是 **REPEATABLE_READ**（可重复读），且用 MVCC + 间隙锁额外解决了幻读。Oracle 默认 READ_COMMITTED。

#### 3. JDBC 设置隔离级别

```java
import java.sql.Connection;

// 设置隔离级别（在事务开始前设置）
conn.setTransactionIsolation(Connection.TRANSACTION_REPEATABLE_READ);

// JDBC 提供的常量
Connection.TRANSACTION_READ_UNCOMMITTED;   // 1
Connection.TRANSACTION_READ_COMMITTED;     // 2
Connection.TRANSACTION_REPEATABLE_READ;    // 4
Connection.TRANSACTION_SERIALIZABLE;       // 8
```

### 六、Savepoint（部分回滚）

普通 `rollback()` 撤销整个事务。`Savepoint` 让你**回滚到某个保存点**，保留之前的操作：

```java
conn.setAutoCommit(false);
try {
    stmt.executeUpdate("INSERT ...");   // 操作1

    Savepoint sp = conn.setSavepoint("POINT_A");   // ✅ 设置保存点

    stmt.executeUpdate("UPDATE ...");   // 操作2

    // 操作2 出错了，只想回滚操作2，保留操作1
    conn.rollback(sp);                  // ✅ 回滚到保存点，操作1 保留
    // 之后还能继续执行并 commit
    conn.commit();
}
```

> 💡 **提示：** Savepoint 用于「**部分操作失败时，保留前面的成功部分**」的场景，用得较少但偶尔有用。

### 七、常见问题与注意事项

#### 1. 忘记 commit 导致数据没保存

```java
conn.setAutoCommit(false);
stmt.executeUpdate("INSERT ...");
// ❌ 没 commit 也没 rollback，连接关闭后数据丢失
```

**铁律**：开启了事务（autoCommit=false），就必须 `commit()` 或 `rollback()`。

#### 2. 异常没回滚

```java
try {
    conn.setAutoCommit(false);
    // 操作
    conn.commit();
} catch (Exception e) {
    e.printStackTrace();
    // ❌ 漏了 rollback，事务挂起可能锁表
}
// ✅ catch 里必须 rollback
conn.rollback();
```

#### 3. rollback 要在 catch 里

```java
try {
    conn.setAutoCommit(false);
    // 业务
    conn.commit();
} catch (Exception e) {
    if (conn != null) conn.rollback();   // ✅
    throw e;
}
```

#### 4. 事务范围要尽量小

事务持有数据库锁，**时间越长越影响并发**。不要在事务里做耗时操作（远程调用、文件 IO），事务只包必要的数据库操作。

#### 5. Spring 的事务管理

实际 Spring 项目用 `@Transactional` 注解管理事务，底层仍是这套 JDBC 事务 API。Spring 帮你处理了 commit/rollback 的样板代码。

### 八、实际应用场景

| 场景 | 用法 |
| --- | --- |
| 银行转账 | 扣款 + 加款同一事务 |
| 订单创建 | 订单 + 订单明细 + 库存扣减同一事务 |
| 批量导入 | 多条插入同一事务，失败整体回滚 |
| 转账 + 日志 | 主业务 + 日志记录（日志可用 Savepoint 隔离失败） |

```java
// 订单创建示例
conn.setAutoCommit(false);
try {
    insertOrder(conn, order);          // 插订单
    insertOrderItems(conn, items);     // 插明细
    updateStock(conn, items);          // 扣库存
    conn.commit();                     // 全部成功才提交
} catch (Exception e) {
    conn.rollback();                   // 任一失败，全部撤销
}
```

### 九、总结

| 要点 | 说明 |
| --- | --- |
| ACID | 原子、一致、隔离、持久 |
| 开启事务 | `setAutoCommit(false)` |
| 提交/回滚 | `commit()` / `rollback()` |
| 隔离级别 | READ_UNCOMMITTED → SERIALIZABLE（越高越安全越慢） |
| MySQL 默认 | REPEATABLE_READ |
| Savepoint | 部分回滚 |
| 铁律 | 开了事务必须 commit 或 rollback |

事务是保证数据一致性的核心机制。掌握「**setAutoCommit(false) + try-commit-catch-rollback**」这套模板，就能处理绝大多数事务场景。
