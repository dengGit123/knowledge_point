### 一、概述

> 📖 [JDBC 批处理（Oracle 教程）](https://docs.oracle.com/javase/tutorial/jdbc/basics/retrieving.html) ｜ [PreparedStatement.executeBatch](https://docs.oracle.com/en/java/javase/17/docs/api/java.sql/java/sql/Statement.html#executeBatch())

批处理（Batch）是指**一次性提交多条 SQL 执行**，而不是一条一条发。它能大幅提升大量插入/更新的性能，是大数据量操作的必备技巧。

大白话：要插入 10000 条数据，一条一条执行就像「**跑 10000 趟快递，每趟送 1 件**」；批处理像「**装一车，一趟送 10000 件**」——网络往返少了，速度快几十倍。

| 你将学到 | 说明 |
| --- | --- |
| 为什么用批处理 | 逐条执行的性能问题 |
| addBatch / executeBatch | 批处理 API |
| 优化技巧 | rewriteBatchedStatements |
| 与事务结合 | 批处理 + 事务最快 |

> 💡 **提示：** 批处理主要用于 **INSERT/UPDATE/DELETE**（批量增删改），不用于 SELECT。

---

### 二、为什么需要批处理

#### 逐条执行的问题

```java
// ❌ 逐条插入 10000 条
for (int i = 0; i < 10000; i++) {
    ps.setString(1, "user" + i);
    ps.executeUpdate();   // 每条都和数据库通信一次
}
```

每次 `executeUpdate` 都要：
- 把 SQL 发到数据库（网络往返）
- 数据库解析执行
- 返回结果

**10000 条 = 10000 次网络往返**，极慢。

#### 批处理的优化

```java
// ✅ 批处理：攒一批一起发
for (int i = 0; i < 10000; i++) {
    ps.setString(1, "user" + i);
    ps.addBatch();           // 先攒着
    if (i % 1000 == 0) {
        ps.executeBatch();   // 每 1000 条执行一次
        ps.clearBatch();
    }
}
ps.executeBatch();           // 剩余的
```

**10000 条可能只需几十次往返**，快几十倍。

### 三、批处理 API

#### 核心 API

| 方法 | 作用 |
| --- | --- |
| `addBatch()` | 把当前参数配置加入批处理 |
| `executeBatch()` | 执行所有批处理，返回每条影响行数 |
| `clearBatch()` | 清空批处理缓存 |

#### 标准流程

```java
PreparedStatement ps = conn.prepareStatement("INSERT INTO user(name) VALUES(?)");

for (String name : names) {
    ps.setString(1, name);
    ps.addBatch();            // ① 加入批处理
}

int[] results = ps.executeBatch();  // ② 执行，返回每条的影响行数
// results[i] 表示第 i 条 SQL 影响的行数
```

#### executeBatch 返回值

返回 `int[]`，每个元素对应一条 SQL 的影响行数：

| 返回值 | 含义 |
| --- | --- |
| `≥ 0` | 影响的行数 |
| `SUCCESS_NO_INFO` (-2) | 成功，但不知道具体行数 |
| `EXECUTE_FAILED` (-3) | 执行失败 |

### 四、分批执行（重要技巧）

一次性 `addBatch` 几十万条会**撑爆内存**。正确做法是**分批**：

```java
int batchSize = 1000;   // 每批 1000 条
for (int i = 0; i < names.size(); i++) {
    ps.setString(1, names.get(i));
    ps.addBatch();

    if ((i + 1) % batchSize == 0) {
        ps.executeBatch();   // 每 1000 条执行一次
        ps.clearBatch();     // 清空，准备下一批
    }
}
// 执行剩余不足一批的
if (names.size() % batchSize != 0) {
    ps.executeBatch();
}
```

> 💡 **提示：** **batchSize 一般 500~1000** 比较合适。太小没发挥批处理优势，太大内存压力大且单次执行时间过长。

### 五、MySQL 的关键优化：rewriteBatchedStatements

MySQL 默认**不会真正合并批处理**——它只是逐条发送。要开启真正的批处理优化，URL 加参数：

```
jdbc:mysql://localhost:3306/test?rewriteBatchedStatements=true
```

```java
// ✅ 加上 rewriteBatchedStatements=true，性能提升 10 倍以上
String url = "jdbc:mysql://localhost:3306/test?rewriteBatchedStatements=true&useSSL=false&serverTimezone=UTC";
```

> ⚠️ **注意：** **MySQL 必须加 `rewriteBatchedStatements=true`** 才能享受批处理的真正加速！否则 `addBatch` 只是「伪批处理」，和逐条差不多慢。这是 MySQL 批处理的头号优化点。

### 六、批处理 + 事务（最快组合）

把批处理放在**一个事务**里，减少提交次数，性能最高：

```java
conn.setAutoCommit(false);   // ✅ 开启事务
try (PreparedStatement ps = conn.prepareStatement(
        "INSERT INTO user(name, age) VALUES(?, ?)")) {

    int batchSize = 1000;
    for (int i = 0; i < list.size(); i++) {
        User u = list.get(i);
        ps.setString(1, u.getName());
        ps.setInt(2, u.getAge());
        ps.addBatch();

        if ((i + 1) % batchSize == 0) {
            ps.executeBatch();
            ps.clearBatch();
        }
    }
    ps.executeBatch();
    conn.commit();            // ✅ 一次性提交
} catch (Exception e) {
    conn.rollback();          // 失败全部回滚
    throw e;
} finally {
    conn.setAutoCommit(true);
}
```

#### 三层优化叠加效果

```
逐条 + 自动提交           ：最慢（基准）
批处理 + 自动提交          ：快几倍
批处理 + 事务             ：快几十倍
批处理 + 事务 + rewriteBatchedStatements：快上百倍 ✅
```

详见 [[事务管理]]。

### 七、批处理的两种方式

#### 1. PreparedStatement 批处理（推荐）

参数化、防注入、可预编译，**首选**。

#### 2. Statement 批处理

```java
Statement stmt = conn.createStatement();
stmt.addBatch("INSERT INTO user(name) VALUES('A')");
stmt.addBatch("INSERT INTO user(name) VALUES('B')");
stmt.addBatch("UPDATE user SET age = 20");
stmt.executeBatch();
```

适用于「**每条 SQL 都不同**」的场景，但有 SQL 注入风险（拼接 SQL），少用。

> 💡 **提示：** 绝大多数批处理用 PreparedStatement（批量插入结构相同的数据）。

### 八、常见问题与注意事项

#### 1. 内存溢出（OOM）

```java
// ❌ 一次性 addBatch 百万条，内存爆了
for (int i = 0; i < 1000000; i++) {
    ps.addBatch();
}
ps.executeBatch();
```

**分批 executeBatch**，每批几百到一千条。

#### 2. MySQL 没加速

忘加 `rewriteBatchedStatements=true`。这是 MySQL 独有的坑，务必检查 URL。

#### 3. 一条失败全部失败？

批处理默认某些驱动下，一条失败可能影响整批。配合事务（rollback 全部），保证数据一致。

#### 4. 批量插入的极致优化

对于「纯插入」，还可以用 SQL 多值语法（比 PreparedStatement 批处理还快）：

```java
// ✅ 拼 VALUES (...),(...),(...) 多值，一条 SQL 插入多行
// INSERT INTO user(name) VALUES('A'),('B'),('C')
// MyBatis 的批量插入就是用这个
```

#### 5. 批处理期间不要做重活

批处理最好单独、快速执行，不要在循环里混入慢操作（远程调用、文件 IO）。

### 九、实际应用场景

| 场景 | 用法 |
| --- | --- |
| 大数据导入 | 几百万条数据入库 |
| 日志批量写入 | 攒一批日志一起写 |
| 数据迁移 | 从旧表批量搬到新表 |
| 批量更新 | 批量修改状态 |
| Excel 导入 | 解析 Excel 批量插入 |

```java
// Excel 批量导入用户（实战示例）
public void importUsers(List<User> users) throws SQLException {
    conn.setAutoCommit(false);
    try (PreparedStatement ps = conn.prepareStatement(
            "INSERT INTO user(name, age) VALUES(?, ?)")) {
        for (int i = 0; i < users.size(); i++) {
            ps.setString(1, users.get(i).getName());
            ps.setInt(2, users.get(i).getAge());
            ps.addBatch();
            if ((i + 1) % 1000 == 0) {
                ps.executeBatch();
                ps.clearBatch();
            }
        }
        ps.executeBatch();
        conn.commit();
    } catch (Exception e) {
        conn.rollback();
        throw e;
    }
}
```

### 十、总结

| 要点 | 说明 |
| --- | --- |
| 核心价值 | 减少网络往返，大批量提速 |
| API | `addBatch` → `executeBatch` → `clearBatch` |
| 分批 | 每批 500~1000 条，防 OOM |
| MySQL 优化 | URL 加 `rewriteBatchedStatements=true` |
| 最快组合 | 批处理 + 事务 + rewriteBatchedStatements |
| 适用 | INSERT/UPDATE/DELETE 批量操作 |

一句话：**大批量增删改，用 PreparedStatement 批处理 + 事务 +（MySQL）rewriteBatchedStatements**，能比逐条快上百倍。
