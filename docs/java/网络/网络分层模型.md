### 一、概述

> 📖 [OSI 模型（维基百科）](https://zh.wikipedia.org/wiki/OSI%E6%A8%A1%E5%9E%8B) ｜ [TCP/IP 协议族（维基百科）](https://zh.wikipedia.org/wiki/TCP/IP%E5%8D%8F%E8%AE%AE%E6%97%8F)

网络分层模型是理解整个网络体系的**地图**。它把复杂的网络通信拆成一层层的职责，每一层只管自己的事，并给上一层提供服务。

大白话：寄快递时，你只管「把东西装进盒子、写地址」（应用层），至于快递公司怎么分拣、怎么走高速、用什么车送（下面几层），你完全不用操心——**每一层都封装了下面的复杂度**。网络分层就是同样的道理。

两大主流模型：

| 模型 | 层数 | 说明 |
| --- | --- | --- |
| **OSI 七层模型** | 7 层 | 理论标准，用来教学和理解 |
| **TCP/IP 四层模型** | 4 层 | 实际互联网使用的模型 |

---

### 二、为什么要分层

不分层的话，整个通信逻辑会混成一团，没法维护。分层带来三大好处：

1. **各层各司其职**：每层专注解决一类问题，互不干扰
2. **灵活替换**：换底层（比如从 WiFi 换成有线），上层应用代码不用改
3. **降低复杂度**：每层只面对相邻层的接口，像搭积木

> 💡 **提示：** 记住一句话——**每一层都为上一层提供服务，并使用下一层的服务**。

### 三、OSI 七层模型

```
┌────────────────┐
│  7. 应用层       │  ← 你写的程序在这里（HTTP、FTP、DNS、SMTP）
├────────────────┤
│  6. 表示层       │  ← 数据格式转换、加密、压缩
├────────────────┤
│  5. 会话层       │  ← 建立/管理/终止会话
├────────────────┤
│  4. 传输层       │  ← 端到端通信（TCP、UDP），带端口
├────────────────┤
│  3. 网络层       │  ← 跨网络寻址路由（IP）
├────────────────┤
│  2. 数据链路层    │  ← 相邻节点传输（MAC、帧、交换机）
├────────────────┤
│  1. 物理层       │  ← 比特流传输（网线、光纤、电信号）
└────────────────┘
```

#### 各层详解

| 层 | 名称 | 职责 | 典型协议/设备 | 数据单元(PDU) |
| --- | --- | --- | --- | --- |
| 7 | 应用层 | 为应用程序提供网络服务接口 | HTTP、HTTPS、DNS、FTP、SMTP | 报文（Message） |
| 6 | 表示层 | 数据格式化、加密、压缩 | SSL/TLS、JPEG、ASCII | 报文 |
| 5 | 会话层 | 建立、管理、终止会话 | RPC、NetBIOS | 报文 |
| 4 | 传输层 | 端到端的可靠/不可靠传输，端口 | **TCP、UDP** | 段（Segment）/ 数据报（Datagram） |
| 3 | 网络层 | 路由选择、逻辑寻址 | **IP**、ICMP；路由器 | 包（Packet） |
| 2 | 数据链路层 | 物理寻址、帧封装、差错检测 | Ethernet、MAC；交换机 | 帧（Frame） |
| 1 | 物理层 | 传输原始比特流 | 网线、光纤、集线器 | 比特（Bit） |

> 💡 **提示：** PDU（Protocol Data Unit，协议数据单元）是每层处理的数据单位名称。应用层叫「报文」，传输层叫「段/数据报」，网络层叫「包」，链路层叫「帧」，物理层叫「比特」。

### 四、TCP/IP 四层模型（实际使用）

实际开发中，OSI 的 5/6/7 层常合并，于是有了更实用的 **TCP/IP 四层模型**：

```
┌──────────────┐        OSI 对应        典型协议
│  应用层       │ ◄── 应用+表示+会话层    HTTP、DNS、FTP、SMTP
├──────────────┤
│  传输层       │ ◄── 传输层              TCP、UDP
├──────────────┤
│  网际层       │ ◄── 网络层              IP、ICMP
├──────────────┤
│  网络接口层    │ ◄── 数据链路+物理层     以太网、WiFi
└──────────────┘
```

| TCP/IP 层 | 对应 OSI | 主要协议 | 作用 |
| --- | --- | --- | --- |
| **应用层** | 应用、表示、会话层 | HTTP、DNS、FTP、SMTP | 直接面向用户程序 |
| **传输层** | 传输层 | TCP、UDP | 进程间通信，端口寻址 |
| **网际层（Internet）** | 网络层 | IP、ICMP | 主机寻址、路由 |
| **网络接口层** | 数据链路 + 物理层 | Ethernet、Wi-Fi | 物理传输、帧封装 |

> 💡 **提示：** TCP/IP 模型因「互联网协议 TCP/IP」得名，是当前互联网**事实上的标准**。日常说的「TCP/IP 协议族」就是指这一整套协议。

### 五、数据封装与解封装（核心！）

这是理解「数据怎么在网络里传输」的关键。

#### 1. 发送方：逐层封装（加头）

应用程序产生数据后，每往下走一层，就被**套上一层「头部」**：

```
应用层：   [ 应用数据 ]
传输层：   [ TCP头 | 应用数据 ]          ← 加 TCP 头（含源端口、目的端口）
网络层：   [ IP头 | TCP头 | 应用数据 ]   ← 加 IP 头（含源 IP、目的 IP）
链路层：   [ 帧头 | IP头 | TCP头 | 应用数据 | 帧尾 ]  ← 加帧头帧尾（含 MAC）
```

类比：像一封信层层套信封——信纸（应用数据）装进信封写上「给某程序」（TCP 头），再装进大信封写上「给某地址」（IP 头），再装进快递袋写上「给某门牌」（帧头），最后交给快递员。

#### 2. 接收方：逐层解封装（拆头）

到了接收方，过程反过来，**每往上一层就拆掉一层头**，最后把应用数据交给程序：

```
链路层：   [ 帧头 | IP头 | TCP头 | 应用数据 | 帧尾 ]  → 拆帧头帧尾
网络层：   [ IP头 | TCP头 | 应用数据 ]               → 拆 IP 头
传输层：   [ TCP头 | 应用数据 ]                       → 拆 TCP 头
应用层：   [ 应用数据 ]                               → 交给程序
```

> 💡 **提示：** 每一层只看懂、处理自己的头，不会去管别的层的内容——这就是分层解耦的精髓。

### 六、一次网页请求在分层模型里的流动

打开浏览器访问 `https://www.example.com`，数据在各层是这样流动的：

```
你的电脑（发送）                                  服务器（接收）
┌─────────────────────┐                ┌─────────────────────┐
│ 应用层：发出 HTTP 请求 │                │ 应用层：收到 HTTP 请求 │
│ 传输层：套 TCP 头      │ ── 封装 ──►   │ 传输层：拆 TCP 头      │
│ 网际层：套 IP 头       │   逐层         │ 网际层：拆 IP 头       │
│ 接口层：套帧头成帧     │   向下         │ 接口层：拆帧头         │
└─────────────────────┘                └─────────────────────┘
              │ 经过多个路由器逐跳转发 ▲
```

途中每经过一个路由器，路由器会拆到**网络层**看 IP 地址，决定下一跳怎么走，再重新封装成帧继续传。

### 七、面试常考：各层协议对应

把常用协议归到对应层，是高频考点：

| 层 | 协议 |
| --- | --- |
| 应用层 | **HTTP**、**HTTPS**、**DNS**、FTP、SMTP、SSH、Telnet |
| 传输层 | **TCP**、UDP |
| 网际层 | **IP**、ICMP（ping）、ARP（IP→MAC） |
| 网络接口层 | Ethernet、Wi-Fi（802.11）、PPP |

记忆口诀：
- **应用层**：上网用的（HTTP/DNS）
- **传输层**：传数据的方式（TCP 可靠 / UDP 快）
- **网际层**：找路用的（IP）
- **接口层**：物理链路（以太网/WiFi）

### 八、常见问题与注意事项

#### 1. OSI 和 TCP/IP 该记哪个？

**两个都要知道**。OSI 是理论框架（面试爱问七层），TCP/IP 是实际使用（开发中真正打交道）。背的时候先背 OSI 七层，再对应到 TCP/IP 四层。

#### 2. 表示层、会话层好像没什么存在感？

确实，现代协议（如 HTTP）常把表示、会话功能揉进应用层，所以 TCP/IP 把它们合并了。考试会单独考，开发中很少单独接触。

#### 3. 传输层的「端口」和网络层的「IP」配合定位

- IP 决定数据送到**哪台机器**
- 端口决定交给那台机器的**哪个程序**
- 两者结合（`IP:端口`）才能精确定位一个通信端点

#### 4. 路由器 vs 交换机（从分层角度）

| 设备 | 工作层 | 处理的地址 |
| --- | --- | --- |
| 交换机 | 数据链路层（TCP/IP 接口层） | MAC 地址 |
| 路由器 | 网络层（TCP/IP 网际层） | IP 地址 |

详见 [[网络基础]]。

### 九、总结

| 要点 | 说明 |
| --- | --- |
| 分层目的 | 解耦、各司其职、灵活替换 |
| OSI 七层 | 物理→链路→网络→传输→会话→表示→应用 |
| TCP/IP 四层 | 接口→网际→传输→应用（实际标准） |
| 封装 | 发送时逐层加头 |
| 解封装 | 接收时逐层拆头 |
| 每层数据单元 | 比特→帧→包→段→报文 |

学完分层模型，下一步按层深入：[[IP协议]]（网络层）→ [[TCP协议]]/[[UDP协议]]（传输层）→ [[HTTP协议]]（应用层）。
