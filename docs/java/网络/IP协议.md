### 一、概述

> 📖 [IP（维基百科）](https://zh.wikipedia.org/wiki/%E7%BD%91%E9%99%85%E5%8D%8F%E8%AE%AE) ｜ [RFC 791（IPv4 规范）](https://datatracker.ietf.org/doc/html/rfc791)

IP（Internet Protocol，网际协议）是网络层的核心协议，负责**给每台设备分配一个地址，并决定数据怎么在网络间路由转发**。

大白话：IP 地址是网络世界的「收货地址」，IP 协议则是「邮政系统」——它规定地址怎么写、信件（数据包）怎么从寄件人一步步转送到收件人手里。

本文讲清楚：IP 地址体系、子网划分、公网私网、网关路由、ARP。这是理解一切网络通信的根基。

| 子主题 | 说明 |
| --- | --- |
| IPv4 / IPv6 | 两代地址 |
| 子网掩码 / CIDR | 如何划分网络 |
| 公网 / 私网 IP | 哪些能直接上网 |
| 网关 / 路由 | 数据怎么跨网络走 |
| ARP | IP 地址怎么找到 MAC 地址 |

### 二、IP 地址基础

#### 1. IPv4 地址

IPv4 地址是 **32 位**二进制，通常写成**点分十进制**——4 段、每段 0~255：

```
二进制： 11000000 10101000 00000001 00000001
点分十进制： 192 . 168 . 1 . 1
```

> 💡 **提示：** IPv4 总共只有约 **43 亿**个地址，早就不够用了，所以才有了 IPv6。

#### 2. IPv4 地址的组成

一个 IP 地址由两部分组成：

```
[ 网络号 | 主机号 ]
  前面部分    后面部分
```

- **网络号**：标识属于哪个网络（类比：哪个小区）
- **主机号**：标识该网络内的哪台主机（类比：小区里几号楼几号）

要切分这两部分，靠的是**子网掩码**（见下文）。

#### 3. IPv6 地址

IPv6 是 **128 位**，写成 8 组十六进制，用冒号分隔：

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

- 地址数量多到「地球上每粒沙子都能分一个」
- 连续的 0 可缩写为 `::`（如 `2001:db8::1`）
- 解决了 IPv4 地址枯竭问题，是未来方向

### 三、IP 地址的分类（A/B/C/D/E）

早期按网络规模，IPv4 分成 5 类：

| 类别 | 地址范围 | 网络号位数 | 适用场景 |
| --- | --- | --- | --- |
| **A 类** | `1.0.0.0` ~ `126.255.255.255` | 8 位 | 超大型网络 |
| **B 类** | `128.0.0.0` ~ `191.255.255.255` | 16 位 | 中型网络 |
| **C 类** | `192.0.0.0` ~ `223.255.255.255` | 24 位 | 小型网络（最常见） |
| **D 类** | `224.0.0.0` ~ `239.255.255.255` | - | 多播地址 |
| **E 类** | `240.0.0.0` ~ `255.255.255.255` | - | 保留（科研用） |

> 💡 **提示：** 日常接触最多的是 **C 类**（`192.168.x.x` 就是 C 类私网）。这种固定分类方式太死板，现在普遍被 **CIDR** 取代（见下文）。

#### 特殊地址

| 地址 | 含义 |
| --- | --- |
| `127.0.0.1` | 本机回环地址（localhost） |
| `0.0.0.0` | 本机所有网卡 / 默认路由 |
| `255.255.255.255` | 广播地址（发给本网所有机器） |

### 四、子网掩码与 CIDR

#### 1. 子网掩码

子网掩码用来区分 IP 里的「网络号」和「主机号」。掩码中**为 1 的位对应网络号，为 0 的位对应主机号**。

以 `192.168.1.100` 子网掩码 `255.255.255.0` 为例：

```
IP:    192 . 168 . 1 . 100      11000000.10101000.00000001.01100100
掩码:  255 . 255 . 255 . 0       11111111.11111111.11111111.00000000
                                └──── 网络号 ────┘└─ 主机号 ─┘
```

- 网络号（IP & 掩码）：`192.168.1.0`
- 主机号：`0.0.0.100`

> 💡 **提示：** 「按位与（&）」IP 和掩码，得到的就是**网络地址**，它代表整个子网。

#### 2. CIDR（无类别域间路由）

子网掩码太长不好写，CIDR 用**斜线记法**简洁表示：

```
192.168.1.100/24
            └─ 表示前 24 位是网络号，等价于掩码 255.255.255.0
```

| CIDR | 掩码 | 主机位 | 可用主机数 |
| --- | --- | --- | --- |
| `/8` | 255.0.0.0 | 24 位 | 约 1677 万 |
| `/16` | 255.255.0.0 | 16 位 | 65534 |
| `/24` | 255.255.255.0 | 8 位 | 254 |
| `/30` | 255.255.255.252 | 2 位 | 2 |

> ⚠️ **注意：** 一个子网里，主机号**全 0 是网络地址**（代表整个子网），**全 1 是广播地址**，都不能分配给主机，所以可用主机数 = `2^主机位 - 2`。

### 五、公网 IP 与私网 IP

#### 1. 私网地址范围

不是所有 IP 都能直接上网。下面三段是**私有地址**，只在局域网内部使用，不能出现在公网：

| 类别 | 私网范围 | 常见 |
| --- | --- | --- |
| A | `10.0.0.0` ~ `10.255.255.255` | 大企业、云服务器内网 |
| B | `172.16.0.0` ~ `172.31.255.255` | Docker 默认网段 |
| C | `192.168.0.0` ~ `192.168.255.255` | 家庭/公司路由器 |

> 💡 **提示：** 你电脑上 `ipconfig`/`ifconfig` 看到的 `192.168.x.x` 是**私网 IP**，全世界千千万万台电脑都可能用同一个，它们在公网上并不唯一。

#### 2. NAT（网络地址转换）

既然私网 IP 不能上公网，那家里一堆设备怎么上网？靠 **NAT**——路由器把你私网 IP 转换成它的**公网 IP** 再发出去。

```
你的电脑 192.168.1.10  ──►  路由器(NAT)  ──►  公网 113.96.x.x  ──►  服务器
                          把源地址改成公网IP
```

NAT 带来的现象：
- 多台设备共享一个公网 IP 上网
- 外部无法主动访问内部设备（保护了内网安全）
- 这也是为什么 P2P、内网穿透会有难点

### 六、网关与路由

#### 1. 网关（Gateway）

网关是一个网络连接其他网络的「出口」。当数据要发给**别的网络**时，就交给网关转发。家庭网络里，**路由器的 IP（如 `192.168.1.1`）通常就是默认网关**。

#### 2. 路由（Routing）

数据跨网络传输时，每经过一个路由器，路由器查**路由表**决定「下一跳」往哪走，就像问路一样一站站接力。

```
你的数据(发往 8.8.8.8)
   │
   ▼ ① 先发给网关 192.168.1.1
[路由器A] ──查路由表──► 下一跳是路由器B
[路由器B] ──查路由表──► 下一跳是路由器C
   ... 经过多个路由器接力 ...
[目标服务器 8.8.8.8]
```

> 💡 **提示：** 用 `tracert www.baidu.com`（Windows）或 `traceroute`（Mac/Linux）能看到数据经过的每一跳路由器。

### 七、ARP 协议：IP 怎么找到 MAC

数据在局域网内传输，最终靠的是 **MAC 地址**（数据链路层只认 MAC）。但程序只知道 IP，怎么得到对应 MAC？靠 **ARP（Address Resolution Protocol，地址解析协议）**。

#### ARP 工作过程

```
主机A 要发数据给 192.168.1.5（同局域网），但不知道它的 MAC：

① A 广播 ARP 请求：「谁是 192.168.1.5？把你的 MAC 告诉我！」
       ↓（局域网所有机器都收到）
② 主机B(192.168.1.5) 单播回应：「我是，我的 MAC 是 xx:xx:xx:xx:xx:xx」
③ A 把这个 IP→MAC 映射缓存进 ARP 表，下次直接用
```

> 💡 **提示：** 跨网络通信时，ARP 解析的是**网关的 MAC**（数据先交给网关），而不是目标机器的 MAC。

#### 查看 ARP 表

```bash
# Mac/Linux
arp -a

# Windows
arp -a
```

### 八、ICMP 协议

ICMP（Internet Control Message Protocol，互联网控制报文协议）是 IP 的「辅助协议」，用来报告错误和探测网络。

最常见的两个应用：

| 工具 | 作用 |
| --- | --- |
| **ping** | 测试目标是否可达、测算延迟（发 ICMP 回显请求） |
| **traceroute** | 探测数据经过的路由路径 |

```bash
# 测试网络连通性
ping www.baidu.com

# 查看到目标经过的路径
traceroute www.baidu.com
```

### 十、常见问题与注意事项

#### 1. 怎么判断两个 IP 是否在同一网段？

把它们分别和子网掩码「按位与」，结果相同就在同一网段：

```
192.168.1.10/24  →  192.168.1.0
192.168.1.20/24  →  192.168.1.0   ✅ 同网段
192.168.2.10/24  →  192.168.2.0   ❌ 不同网段
```

#### 2. `127.0.0.1` vs `0.0.0.0` vs 本机真实 IP

- `127.0.0.1`：仅本机自己能访问
- `0.0.0.0`：监听本机**所有网卡**，外部也能访问（服务绑定时常用）
- 真实 IP（如 `192.168.1.10`）：局域网其他机器可访问

> ⚠️ **注意：** 服务器绑定 `127.0.0.1` 时外部访问不了，要对外提供服务应绑定 `0.0.0.0` 或具体网卡 IP。

#### 3. IPv6 为什么普及慢？

IPv6 虽好，但需要设备、运营商、应用全面升级，改造成本高。目前通过 NAT 缓解了 IPv4 紧缺，所以 IPv4 仍占主流，两者长期共存。

| 要点 | 说明 |
| --- | --- |
| IP 地址 | 网络层逻辑地址，IPv4 32 位 / IPv6 128 位 |
