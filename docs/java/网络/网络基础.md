### 一、概述

> 📖 [计算机网络（维基百科）](https://zh.wikipedia.org/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C)

计算机网络，就是把多台计算机用通信线路连接起来，让它们能**互相交换数据、共享资源**。

大白话：你家里有多台电脑、手机，连上同一个 WiFi，它们就组成了一张小网络。全世界所有的网络再连在一起，就成了「互联网」（Internet）。

理解网络基础，是学习一切通信协议、网络编程的前提。本文先讲清楚网络的**分类、结构、设备**，为后续协议篇打底。

---

### 二、网络的分类（按覆盖范围）

按照覆盖的地理范围，网络通常分几类：

| 类型 | 全称 | 覆盖范围 | 典型例子 |
| --- | --- | --- | --- |
| **PAN** | 个人局域网 | 几米内 | 蓝牙耳机连手机 |
| **LAN** | 局域网（Local Area Network） | 几百米～几公里 | 公司办公网、家庭 WiFi、机房 |
| **MAN** | 城域网（Metropolitan Area Network） | 一座城市 | 城市有线电视网 |
| **WAN** | 广域网（Wide Area Network） | 跨城市、跨国家 | 互联网骨干、运营商网络 |

> 💡 **提示：** 局域网（LAN）是你最容易接触到的，家里的 WiFi 路由器就把所有设备组成了一个 LAN。互联网本质是「无数个 LAN/WAN 互联」的结果。

### 三、网络拓扑结构

「拓扑」指的是网络中设备是怎么连在一起的。常见几种结构：

```
① 星型拓扑            ② 总线型拓扑         ③ 环型拓扑
    [节点]               [节点]              [节点]
      │                    │                   │
   [中心]──[节点]      ═════════════         [节点]
      │                    │                ╱      ╲
   [节点]               [节点]           [节点]    [节点]
（中心挂了全挂）      （一根线连所有）      （首尾相连成环）
```

| 拓扑 | 特点 | 缺点 |
| --- | --- | --- |
| **星型** | 最常见，所有设备连到一个中心（交换机/路由器） | 中心坏了全瘫痪 |
| **总线型** | 一根总线串所有设备，早期用 | 总线断了全瘫，性能差 |
| **环型** | 设备首尾相连成环 | 任一节点故障影响全局 |
| **网状** | 设备间多条路径互联，互联网骨干就是这种 | 布线复杂、成本高 |

> 💡 **提示：** 现代家庭/公司局域网几乎都是**星型拓扑**——所有设备插到交换机/路由器上。

### 四、常见网络设备

理解设备的作用，能帮你搞清楚数据在网络里是怎么「一跳一跳」传过去的。

#### 1. 网卡（NIC）

每台联网设备的「身份证明 + 收发器」。网卡有唯一的 **MAC 地址**（物理地址，烧在硬件里，理论上全球唯一）。

#### 2. 集线器（Hub）——已淘汰

把信号**广播**给所有端口（所有设备都能收到，靠设备自己判断要不要）。共享带宽、效率低，现已被交换机取代。

#### 3. 交换机（Switch）

工作在**数据链路层**，根据 **MAC 地址**转发数据。

- 像「聪明的快递分拣员」：记住每个端口连着哪个 MAC，精准投递，不广播
- 用于**局域网内部**设备互联

> 💡 **提示：** 交换机认 **MAC 地址**（物理地址，管「同一局域网内的哪台机器」）。

#### 4. 路由器（Router）

工作在**网络层**，根据 **IP 地址**转发数据，连接**不同的网络**。

- 像「跨城转运中心」：决定数据从 A 网络怎么走到 B 网络
- 家里的「WiFi 路由器」其实集成了路由器 + 交换机 + 无线接入点

> 💡 **提示：** 路由器认 **IP 地址**（逻辑地址，管「跨网络怎么走」）。

#### 5. 网关（Gateway）

一个网络的「出口/入口」。你要访问别的网络（比如上网），数据要先交给网关。家庭网络里，**路由器的 IP 通常就是网关**（如 `192.168.1.1`）。

#### 设备对照表

| 设备 | 工作层级 | 依据地址 | 作用 |
| --- | --- | --- | --- |
| 集线器 Hub | 物理层 | 无 | 广播信号（淘汰） |
| 交换机 Switch | 数据链路层 | MAC 地址 | 局域网内精准转发 |
| 路由器 Router | 网络层 | IP 地址 | 跨网络路由转发 |
| 网关 Gateway | 应用层 | - | 不同网络/协议转换的出入口 |

### 五、MAC 地址 vs IP 地址

这是两个最容易混淆的概念，务必分清：

| 对比项 | MAC 地址 | IP 地址 |
| --- | --- | --- |
| 所属层 | 数据链路层 | 网络层 |
| 性质 | 物理/硬件地址 | 逻辑地址 |
| 长度 | 48 位（如 `08:00:27:71:29:c1`） | IPv4 32 位 / IPv6 128 位 |
| 是否可变 | 烧在网卡里，基本不变 | 随网络环境变化（DHCP 分配） |
| 作用范围 | 同一局域网内 | 跨网络全局 |

**类比：**
- **MAC 地址**像你的**身份证号**——一辈子不变，但只用来确认「是你这个人」
- **IP 地址**像你的**收货地址**——搬家就变，但快递（数据）靠它找到你

数据跨网络传输时：**IP 地址决定终点，每经过一个网络，MAC 地址会逐跳变化**（数据包像接力棒，每跳换一次「下一棒的 MAC」）。

### 六、带宽、速率、延迟

衡量网络质量的几个关键指标：

| 指标 | 含义 | 例子 |
| --- | --- | --- |
| **带宽（Bandwidth）** | 信道能传输的最大数据速率 | 「100M 宽带」指 100Mbps（约 12.5MB/s） |
| **吞吐量（Throughput）** | 实际单位时间传输的数据量 | 受带宽、延迟、丢包影响，通常小于带宽 |
| **延迟（Latency）** | 数据从 A 到 B 的时间 | 打游戏看「ping」值，越低越流畅 |
| **丢包率** | 传输中丢失数据包的比例 | 高丢包导致卡顿、重传 |

> ⚠️ **注意：** 带宽单位是 **bps**（bit per second，比特/秒），而文件大小单位是 **Byte**（字节）。1 Byte = 8 bit。所以「100M 宽带」理论下载速度是 `100Mbps ÷ 8 = 12.5MB/s`，不是 100MB/s。

### 七、IP、端口、协议再回顾

这三者是网络通信定位数据的核心要素，已在 [[通信基础]] 详述，这里简要回顾：

- **IP 地址**：定位「哪台机器」
- **端口（Port）**：定位「机器上的哪个程序」
- **协议**：双方约定好的「对话规则」

三者结合，才能完成一次精确的通信。

### 八、常见网络缩写速查

网络领域缩写极多，先记住下面这批高频的：

| 缩写 | 全称 | 含义 |
| --- | --- | --- |
| LAN / WAN | Local / Wide Area Network | 局域网 / 广域网 |
| NIC | Network Interface Card | 网卡 |
| MAC | Media Access Control | 物理地址 |
| IP | Internet Protocol | 网络协议 |
| TCP / UDP | Transmission Control / User Datagram Protocol | 传输控制 / 用户数据报协议 |
| DNS | Domain Name System | 域名系统 |
| DHCP | Dynamic Host Configuration Protocol | 动态分配 IP |
| NAT | Network Address Translation | 网络地址转换 |
| MTU | Maximum Transmission Unit | 最大传输单元 |

### 九、总结

- 网络按范围分 **LAN（局域网）/ WAN（广域网）** 等，互联网是它们的互联
- 局域网常用**星型拓扑**，核心是交换机
- **交换机**认 MAC（链路层），**路由器**认 IP（网络层）
- **MAC 地址**是硬件地址不变，**IP 地址**是逻辑地址会变
- 带宽单位是 bit/s，注意和字节换算（÷8）
