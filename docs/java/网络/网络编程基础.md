### 一、概述

> 📖 [Java 网络编程（Oracle）](https://docs.oracle.com/javase/tutorial/networking/) ｜ [All About Sockets（Oracle）](https://docs.oracle.com/javase/tutorial/networking/sockets/)

网络编程，就是**用代码让两个程序通过网络进行通信**。它把抽象的网络协议（TCP/UDP）变成程序员能调用的 API，让你能写出「服务端 + 客户端」互相收发数据的程序。

大白话：协议原理（[[TCP协议]]）是「规则」，网络编程是「按规则动手干活」。你写代码建个服务端监听端口、写个客户端连过去、互发消息——这就是网络编程。

本文是网络编程系列的入门，先把**网络编程的整体模型、编程步骤、几个关键概念**讲清楚。具体的 Socket API 用法见 [[Socket通信]]，高性能 IO 模型见 [[IO模型]]。

### 二、C/S 与 B/S 架构

网络程序按客户端形态分两大类：

#### 1. C/S（Client/Server，客户端/服务端）

需要**安装专门的客户端程序**，客户端和服务端通信。

```
[专用客户端 App] ◄══ 网络 ══► [服务端]
```

- 例：QQ、微信 PC 版、网游客户端、数据库连接工具
- 优点：性能好、功能强、能用 TCP/UDP 自定义协议
- 缺点：需要安装升级、跨平台麻烦

#### 2. B/S（Browser/Server，浏览器/服务端）

客户端就是**浏览器**，服务端是 Web 服务。

```
[浏览器] ◄══ HTTP/HTTPS ══► [Web 服务端]
```

- 例：网站、网页应用、后台管理系统
- 优点：无需安装、跨平台、易升级
- 缺点：受浏览器限制、强依赖网络

> 💡 **提示：** C/S 常用自定义 TCP 协议或长连接（如 [[Socket通信]]、[[WebSocket通信]]）；B/S 用 HTTP 协议（见 [[HTTP协议]]）。现代前端项目多是 B/S 架构。

### 三、网络编程三要素

要完成一次网络通信，必须确定三件事：

| 要素 | 作用 | Java 类 |
| --- | --- | --- |
| **IP 地址** | 找到「哪台机器」 | `InetAddress` |
| **端口** | 找到机器上「哪个程序」 | `int port` |
| **协议** | 约定「怎么对话」 | TCP: `Socket` / UDP: `DatagramSocket` |

三者缺一不可。例如 `192.168.1.10:8080` 走 TCP，就精确定位了一个通信端点。

### 四、服务端 / 客户端编程步骤（TCP）

TCP 网络编程有套路化的步骤，记住这个框架，后续所有 Socket 代码都是它的扩展。

#### 服务端四步

```
① 创建 ServerSocket，绑定端口
② accept() 等待客户端连接（阻塞）
③ 通过输入流/输出流 读写数据
④ close() 关闭连接
```

```java
import java.io.*;
import java.net.ServerSocket;
import java.net.Socket;

public class ServerStep {
    public static void main(String[] args) throws IOException {
        // ① 绑定 8080 端口
        try (ServerSocket server = new ServerSocket(8080);
             // ② 等待连接（阻塞，直到有客户端连进来）
             Socket client = server.accept()) {

            System.out.println("客户端连入: " + client.getInetAddress());

            // ③ 读写数据
            BufferedReader in = new BufferedReader(
                new InputStreamReader(client.getInputStream(), "UTF-8"));
            PrintWriter out = new PrintWriter(
                new OutputStreamWriter(client.getOutputStream(), "UTF-8"), true);

            String msg = in.readLine();      // 读
            System.out.println("收到: " + msg);
            out.println("已收到: " + msg);   // 写

            // ④ try-with-resources 自动 close
        }
    }
}
```

#### 客户端三步

```
① 创建 Socket，指定服务端 IP + 端口（发起连接）
② 通过输入流/输出流 读写数据
③ close() 关闭连接
```

```java
import java.io.*;
import java.net.Socket;

public class ClientStep {
    public static void main(String[] args) throws IOException {
        // ① 连接服务端
        try (Socket socket = new Socket("127.0.0.1", 8080);
             // ② 读写数据
             PrintWriter out = new PrintWriter(
                 new OutputStreamWriter(socket.getOutputStream(), "UTF-8"), true);
             BufferedReader in = new BufferedReader(
                 new InputStreamReader(socket.getInputStream(), "UTF-8"))) {

            out.println("你好服务端");      // 写
            System.out.println("回复: " + in.readLine());  // 读
            // ③ 自动 close
        }
    }
}
```

> 💡 **提示：** 完整的 Socket 用法（多线程、文件传输、超时）见 [[Socket通信]]。这里只看框架。

### 五、UDP 网络编程步骤

UDP 没有「连接」概念，双方都用 `DatagramSocket`，靠 `DatagramPacket` 收发：

```
接收端：① 创建 DatagramSocket 绑定端口 → ② receive() 接收 → ③ close
发送端：① 创建 DatagramSocket → ② send() 发包 → ③ close
```

详细用法见 [[UDP通信]]。

### 六、易混淆概念：阻塞 / 非阻塞 / 同步 / 异步 ⭐

这组概念是理解后续 [[IO模型]] 的前提，务必分清。

#### 1. 阻塞 vs 非阻塞

指的是**调用方在等待结果时，能不能干别的事**：

- **阻塞（Blocking）**：调用后**线程被挂起**，直到有结果才继续。例：`accept()` 没连接时一直卡住
- **非阻塞（Non-blocking）**：调用后**立即返回**，没有结果就返回「暂时没有」，线程可以去干别的

#### 2. 同步 vs 异步

指的是**调用方怎么拿到结果**：

- **同步（Synchronous）**：调用方**自己主动去等/去问**结果
- **异步（Asynchronous）**：被调用方做完后**主动通知**调用方（回调 / 事件）

#### 3. 组合关系

| 组合 | 说明 | 例子 |
| --- | --- | --- |
| 同步阻塞 | 等 + 自己拿 | BIO（传统 Socket，accept 阻塞） |
| 同步非阻塞 | 不干等 + 自己拿 | NIO（轮询就绪的 Channel） |
| 异步非阻塞 | 不等 + 被通知 | AIO（Future / 回调） |

> 💡 **提示：** 网络编程里常说的「BIO/NIO/AIO」就是基于这套概念划分的三种 IO 模型，详见 [[IO模型]]。

### 七、网络编程的关键设计问题

#### 1. 一个服务端怎么同时服务多个客户端？

这是网络编程的核心难题。三种经典方案（对应三种 IO 模型）：

| 方案 | 做法 | 缺点 |
| --- | --- | --- |
| **多线程 BIO** | 每个客户端开一个线程 | 线程太多会爆 |
| **线程池 BIO** | 用线程池限制数量 | 连接数仍受线程数限制 |
| **NIO 多路复用** | 一个线程管很多连接 | 编程复杂，但能扛海量连接 |

详见 [[IO模型]] 和 [[IO多路复用]]。

#### 2. 通信协议怎么设计？

TCP 是字节流无边界（见 [[TCP协议]] 的粘包问题），应用层要自定义消息边界：

- 定长消息
- 特殊分隔符（如换行）
- **长度前缀 + 内容**（最常用，如 `消息总长度(4字节) + 消息体`）

#### 3. 字符编码

网络传输是字节，文本必须指定编码（推荐 UTF-8）。

#### 4. 资源关闭

Socket、流必须关闭，用 try-with-resources。

### 八、Java 网络编程 API 体系

Java 提供了从低到高的网络编程 API：

| 层次 | API | 特点 |
| --- | --- | --- |
| 地址 | `InetAddress` | IP 解析 |
| 阻塞 IO（BIO） | `ServerSocket` / `Socket` | 同步阻塞，简单 |
| 非阻塞 IO（NIO） | `Channel` / `Selector` / `ByteBuffer` | 同步非阻塞，多路复用 |
| 异步 IO（AIO） | `AsynchronousSocketChannel` | 异步非阻塞，回调通知 |
| HTTP 客户端 | `HttpClient` | 封装好的 HTTP 调用 |
| 框架 | **Netty** | 工业级 NIO 框架 |

> 💡 **提示：** 学习路线建议：先掌握 BIO（[[Socket通信]]）→ 理解 NIO（[[IO模型]]）→ 实战用 Netty（[[Netty框架]]）。

### 九、常见问题与注意事项

#### 1. accept / read 一直卡住怎么办？

BIO 的 `accept()`、`read()` 默认无限阻塞。生产环境必须设超时：

```java
socket.setSoTimeout(5000);  // 读超时 5 秒
```

#### 2. 服务端连不上的常见原因

- 端口被占用（`BindException`）
- 防火墙拦截（云服务器安全组没放行）
- 服务端绑定了 `127.0.0.1` 而非 `0.0.0.0`（外部访问不了，见 [[IP协议]]）

#### 3. 多客户端阻塞问题

BIO 下，如果服务端单线程处理一个客户端时阻塞了，其他客户端就得排队——这就是为什么 BIO 要多线程/线程池，也催生了 NIO。

#### 4. 中文乱码

读写文本务必指定编码：`new InputStreamReader(in, "UTF-8")`。
