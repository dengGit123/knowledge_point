# 类中的执行顺序及次数

> 官方文档：[JVM Specification - Initialization](https://docs.oracle.com/javase/specs/jvms/se21/html/jvms-5.html#jvms-5.5) ｜ [菜鸟教程：Java 初始化顺序](https://www.runoob.com/java/java-initialization-order.html)

> 本文是**综合性专题**，整合了 [静态属性.md](./静态属性.md)、[静态方法.md](./静态方法.md)、[静态代码块.md](./静态代码块.md)、[构造函数.md](./构造函数.md)、[super.md](./super.md) 中关于"执行顺序"的内容，系统讲解使用类时各种成员的执行顺序和执行次数。

---

## 一、概述

当我们**使用一个类**（比如 `new` 创建对象、访问它的静态成员）时，类内部的各个成员并不是随机执行的，而是有严格的**执行顺序**和固定的**执行次数**。理解这一点，是看懂 Java 初始化机制、写出正确的构造逻辑、以及应对面试中"打印执行顺序"类题目的关键。

一句话总结两类初始化：

| 类型 | 触发时机 | 执行频率 |
|------|---------|---------|
| **类初始化** | 类第一次被使用（类加载）时 | **整个 JVM 生命周期内只执行 1 次** |
| **实例初始化** | 每次 `new` 创建对象时 | **每创建一个对象执行 1 次** |

> 💡 **通俗理解**：
> - **类初始化**像"工厂开机"——一个工厂（类）开机自检、装填默认配置，**这辈子只做一次**。
> - **实例初始化**像"生产一件产品"——每生产一件产品（对象），都要走一遍流水线（实例变量赋值 → 实例代码块 → 构造函数），**生产多少件就跑多少遍**。

## 二、先认识类中的各类成员

先把要讨论的成员梳理清楚，它们分属"静态"和"实例"两大阵营：

| 成员 | 归属 | 关键字 / 语法 | 自动执行吗 |
|------|------|--------------|:---------:|
| 静态变量 | 类（所有对象共享） | `static` 变量 | 类加载时初始化 |
| 静态代码块 | 类 | `static { }` | 类加载时执行 |
| 静态方法 | 类 | `static` 方法 | ❌ 不自动，需调用 |
| 实例变量 | 对象 | 普通成员变量 | new 时初始化 |
| 实例代码块（构造代码块） | 对象 | `{ }` | new 时执行 |
| 构造函数 | 对象 | `类名()` | new 时执行 |

> ⚠️ **重点区分**：静态方法**不会自动执行**——它只在你**显式调用**时才运行。所以它不参与"初始化顺序"，它的执行次数等于你调用它的次数。

---

## 三、两条核心规则

### 规则一：类初始化（静态部分）——只执行一次

**类第一次被使用时**，会执行**类初始化**：

```
① 父类的静态变量 + 静态代码块（按源码书写顺序）
        │
        ▼
② 子类的静态变量 + 静态代码块（按源码书写顺序）
```

要点：
- **父类先于子类**初始化（与继承的 `super()` 思想一致）。
- **静态变量赋值**与**静态代码块**按**源码中出现的先后顺序**执行（谁写在前面谁先执行，不存在"变量一定先于代码块"的规则）。
- **只执行一次**，之后无论再 `new` 多少个对象，静态部分都**不再执行**。

### 规则二：实例初始化（实例部分）——每次 new 都执行

**每次 `new` 一个对象**时，会执行**实例初始化**：

```
① 父类的实例变量 + 实例代码块（按源码书写顺序）
        │
        ▼
② 父类的构造函数
        │
        ▼
③ 子类的实例变量 + 实例代码块（按源码书写顺序）
        │
        ▼
④ 子类的构造函数
```

要点：
- **先父后子**，父类部分（实例变量/代码块/构造函数）全部完成后，才轮到子类。
- 实例变量赋值与实例代码块按**源码书写顺序**执行，**然后**才执行构造函数体。

## 四、完整执行顺序总览（核心图解）

把两条规则合起来，使用一个子类时（含继承），完整的执行顺序如下：

```
【第一次使用 Child 类时，触发类初始化（只 1 次）】
   1.  Parent 静态变量 / 静态代码块（按源码顺序）
   2.  Child 静态变量 / 静态代码块（按源码顺序）

【每次 new Child() 时，触发实例初始化（每 new 一次一遍）】
   3.  Parent 实例变量 / 实例代码块（按源码顺序）
   4.  Parent 构造函数
   5.  Child 实例变量 / 实例代码块（按源码顺序）
   6.  Child 构造函数
```

简记口诀：

> **父静 → 子静 → 父实例块 → 父构造 → 子实例块 → 子构造**
> **（静态一次定终身，实例每次都重来）**

## 五、执行次数详解（重点）

这是本主题最核心、最常被考查的部分：

| 成员 | 执行时机 | 执行次数 | 说明 |
|------|---------|:--------:|------|
| **静态变量初始化** | 类加载时 | **1 次** | 类初始化阶段执行，之后不再执行 |
| **静态代码块** | 类加载时 | **1 次** | 同上，常用于一次性初始化 |
| **静态方法** | 被显式调用时 | **= 调用次数** | 不自动执行，调用一次执行一次 |
| **实例变量初始化** | 每次 `new` | **每 new 一次 1 次** | 每个对象都有自己的实例变量 |
| **实例代码块** | 每次 `new` | **每 new 一次 1 次** | 每个对象创建时都执行 |
| **构造函数** | 每次 `new` | **每 new 一次 1 次** | 每个对象创建时都执行 |

> 💡 **一句话区分**：
> - 带有 `static` 的**变量和代码块**：**只执行 1 次**（类加载时）。
> - **静态方法**：**不自动执行**，调用几次执行几次。
> - **实例相关的所有成员**（变量/代码块/构造函数）：**每 new 一次执行一次**。

### 次数演示

```java
class Demo {
    static int staticCount = 0;   // 静态变量，记录静态代码块执行次数
    int instanceCount = 0;        // 实例变量，记录实例代码块执行次数

    static {
        staticCount++;
        System.out.println("静态代码块执行，第 " + staticCount + " 次");
    }

    {
        System.out.println("实例代码块执行（属于当前这个新对象）");
    }

    public Demo() {
        System.out.println("构造函数执行（属于当前这个新对象）");
    }
}

public class Test {
    public static void main(String[] args) {
        System.out.println("--- 准备创建第 1 个对象 ---");
        new Demo();
        System.out.println("--- 准备创建第 2 个对象 ---");
        new Demo();
        System.out.println("--- 准备创建第 3 个对象 ---");
        new Demo();
    }
}
```

输出：

```
静态代码块执行，第 1 次          ← 类初始化，只执行这 1 次
--- 准备创建第 1 个对象 ---
实例代码块执行（属于当前这个新对象）
构造函数执行（属于当前这个新对象）
--- 准备创建第 2 个对象 ---
实例代码块执行（属于当前这个新对象）   ← 静态代码块不再执行！
构造函数执行（属于当前这个新对象）
--- 准备创建第 3 个对象 ---
实例代码块执行（属于当前这个新对象）   ← 静态代码块依然不执行！
构造函数执行（属于当前这个新对象）
```

**结论**：静态代码块只打印了 **1 次**，实例代码块和构造函数打印了 **3 次**（每 new 一次一次）。

## 六、经典完整示例（含继承 + 全部成员）

下面这个例子包含**静态变量、静态代码块、实例变量、实例代码块、构造函数**，并有**父子继承**，是面试最典型的考法。建议照着在 IDE 里运行一遍，观察输出顺序。

```java
class Parent {
    // ① 静态变量（类加载时）
    static String pStatic = print("① Parent 静态变量");

    // ② 静态代码块（类加载时）
    static {
        print("② Parent 静态代码块");
    }

    // ③ 实例变量（new 时）
    String pInstance = print("④ Parent 实例变量");

    // ④ 实例代码块（new 时）
    {
        print("⑤ Parent 实例代码块");
    }

    // ⑤ 构造函数（new 时）
    public Parent() {
        print("⑥ Parent 构造函数");
    }

    // 工具方法：打印并返回
    static String print(String msg) {
        System.out.println(msg);
        return msg;
    }
}

class Child extends Parent {
    // 静态变量
    static String cStatic = print("③ Child 静态变量");

    // 静态代码块
    static {
        print("(Child 静态代码块)");
    }

    // 实例变量
    String cInstance = print("⑦ Child 实例变量");

    // 实例代码块
    {
        print("⑧ Child 实例代码块");
    }

    // 构造函数
    public Child() {
        // 隐式 super() 已在第一行，先完成父类初始化
        print("⑨ Child 构造函数");
    }

    // 重写 print 以便子类自己的调用走子类版本（演示用）
    // 这里为了演示清晰，复用父类的 print
}

public class OrderTest {
    public static void main(String[] args) {
        System.out.println("====== 第一次 new Child() ======");
        new Child();
        System.out.println("====== 第二次 new Child() ======");
        new Child();
    }
}
```

输出顺序：

```
① Parent 静态变量          ┐
② Parent 静态代码块         │  类初始化（只 1 次）
③ Child 静态变量           │  父静 → 子静
(Child 静态代码块)          ┘
====== 第一次 new Child() ======
④ Parent 实例变量          ┐
⑤ Parent 实例代码块         │
⑥ Parent 构造函数          ┘  父实例部分
⑦ Child 实例变量           ┐
⑧ Child 实例代码块          │  子实例部分
⑨ Child 构造函数           ┘
====== 第二次 new Child() ======
④ Parent 实例变量          ┐
⑤ Parent 实例代码块         │  实例初始化再来一遍
⑥ Parent 构造函数          │  （静态部分不再执行）
⑦ Child 实例变量           │
⑧ Child 实例代码块          │
⑨ Child 构造函数           ┘
```

**逐行解读**：
1. `main` 第一次遇到 `new Child()`，触发 `Child` 类初始化 → 先初始化父类 `Parent` 的静态部分（①②），再初始化子类的静态部分（③）。
2. 类初始化完成后，才执行实例初始化：父类实例变量/代码块（④⑤）→ 父类构造函数（⑥）→ 子类实例变量/代码块（⑦⑧）→ 子类构造函数（⑨）。
3. 第二次 `new Child()` 时，**类初始化不再执行**（①②③不再出现），只重复实例初始化（④~⑨）。

## 七、触发类初始化的场景（什么时候"执行那 1 次"）

类初始化**只在类第一次被"主动使用"时触发**，之后不再触发。主动使用的场景：

| 觺发场景 | 示例 |
|---------|------|
| ① 创建实例 | `new Child()` |
| ② 访问静态字段（非 final 常量） | `int x = Child.cStatic;` |
| ③ 调用静态方法 | `Child.print();` |
| ④ 反射加载 | `Class.forName("Child")` |
| ⑤ 初始化子类时，父类未初始化 | 初始化 Child 会先触发 Parent 初始化 |
| ⑥ JVM 启动的主类 | 含 `main` 方法的类会被初始化 |

### 不会触发类初始化的情况（重要）

```java
class Config {
    static {
        System.out.println("Config 类初始化了");   // 这行会执行吗？
    }
    // 编译期常量（基本类型 + 字面量，static final）
    public static final int MAX = 100;
}

public class Test {
    public static void main(String[] args) {
        // ❌ 不会触发 Config 类初始化！
        // 因为 MAX 是编译期常量，在编译时已被直接替换为 100，无需加载 Config 类
        System.out.println(Config.MAX);
        // 输出只有：100（没有 "Config 类初始化了"）
    }
}
```

> ⚠️ **注意（经典考点）**：访问 `static final` 的**编译期常量**（基本类型 / String 字面量）**不会触发类初始化**，因为它的值在编译期就确定了，被直接内联到使用处。但如果 `final` 变量的值**需要运行时才能确定**（如 `static final int X = new Random().nextInt();`），访问它**会**触发类初始化。

## 八、源码书写顺序的影响（容易混淆的点）

**静态变量赋值**与**静态代码块**之间、**实例变量赋值**与**实例代码块**之间，**严格按源码中出现的先后顺序执行**——不是"变量一定先、代码块一定后"。

```java
class Order {
    static int a = init("a = 1");    // 先执行

    static {
        System.out.println("静态代码块");
    }

    static int b = init("b = 2");    // 后执行（在静态代码块之后）

    static int init(String msg) {
        System.out.println(msg);
        return 0;
    }
}
```

输出：

```
a = 1
静态代码块
b = 2     ← b 的初始化在静态代码块之后，因为源码里 b 写在后面
```

> 💡 **结论**：同一"层级"（都静态 或 都实例）的变量赋值与代码块，**谁写在前面谁先执行**。这点很容易被误以为"变量总是先于代码块"，要注意。

## 九、静态方法为什么不参与初始化顺序

静态方法**只定义、不自动执行**。它何时执行、执行几次，完全取决于你**调用它几次**：

```java
class Tool {
    static {
        System.out.println("静态代码块（类加载时执行 1 次）");
    }

    public static void doSomething() {
        System.out.println("静态方法被调用了");
    }
}

public class Test {
    public static void main(String[] args) {
        Tool.doSomething();   // 第 1 次调用：先触发类初始化（执行静态代码块），再执行方法
        Tool.doSomething();   // 第 2 次调用：类已加载，直接执行方法，静态代码块不再执行
        Tool.doSomething();   // 第 3 次调用：同上
    }
}
```

输出：

```
静态代码块（类加载时执行 1 次）
静态方法被调用了
静态方法被调用了
静态方法被调用了
```

**结论**：
- **第一次调用静态方法**时，会**顺带触发类初始化**（因为这是"主动使用"类），所以静态代码块执行一次。
- 之后调用静态方法，类初始化不再执行，只有方法体执行。

## 十、常见陷阱与注意事项

### 陷阱 1：在静态代码块中使用尚未初始化的静态变量

由于按源码顺序执行，**前面**的静态代码块**不能使用后面才声明的静态变量**（此时它还是默认值）。

```java
class Bad {
    static {
        // ❌ 此时 x 还没被赋值（x 在下面才声明），打印的是默认值 0
        System.out.println(x);   // 输出 0
    }
    static int x = 10;   // x 在静态代码块之后才初始化
}
```

> ⚠️ 注意：编译器**不允许**在静态代码块中"向前引用"**字面量赋值**的 static 变量（会编译报错 illegal forward reference），但通过方法间接访问可能绕过，得到默认值 0。总之，**初始化顺序敏感，避免循环依赖**。

### 陷阱 2：构造函数中调用可被子类重写的方法

构造函数执行时（尤其是父类构造函数），如果调用了被子类重写的方法，由于多态，实际执行的是**子类**的方法——但此时子类的实例变量**还没初始化**，可能引发错误。

```java
class Base {
    public Base() {
        init();   // ⚠️ 若子类重写 init()，此时子类字段还未初始化！
    }
    public void init() {}
}

class Sub extends Base {
    int value = 100;          // 这行要到 Sub 实例初始化阶段才执行
    @Override
    public void init() {
        System.out.println(value);   // 此时 value 还是默认值 0（不是 100）
    }
}
new Sub();   // 输出 0，而非 100
```

> 💡 **最佳实践**：**构造函数中不要调用可被子类重写的方法**（应设为 `private` 或 `final`）。

### 陷阱 3：main 方法所在类本身也会被初始化

```java
public class Main {
    static {
        System.out.println("Main 类初始化（main 方法所在类，JVM 启动时）");
    }
    public static void main(String[] args) {
        System.out.println("main 方法执行");
    }
}
// 输出：
// Main 类初始化（main 方法所在类，JVM 启动时）
// main 方法执行
```

含 `main` 的主类在 JVM 启动时会被初始化，所以它的静态代码块**在 `main` 执行之前**就执行了。

## 十一、面试常见问题

### Q1：一个类的初始化顺序是怎样的？

完整顺序：**父类静态 → 子类静态 → 父类实例变量/代码块 → 父类构造函数 → 子类实例变量/代码块 → 子类构造函数**。其中静态变量赋值和静态代码块（以及实例变量和实例代码块）都按**源码书写顺序**执行。

### Q2：静态代码块执行几次？构造函数执行几次？

- **静态代码块**：只执行 **1 次**（类加载时）。
- **构造函数**：每 `new` 一个对象执行 **1 次**，创建多少个对象就执行多少次。

### Q3：静态方法会自动执行吗？

**不会**。静态方法只在被**显式调用**时才执行，执行次数等于调用次数。但**第一次调用**某个类的静态方法会**触发该类的初始化**（执行静态代码块），之后再调用就不再触发了。

### Q4：什么时候会触发类初始化？

主动使用类的 6 种场景：创建实例、访问静态字段（非 final 常量）、调用静态方法、反射 `Class.forName`、初始化子类时父类先初始化、JVM 启动主类。

### Q5：访问 `static final` 常量会触发类初始化吗？

如果是**编译期常量**（`static final` 修饰的**基本类型**或 **String 字面量**，值在编译期可确定），**不会触发**类初始化，因为值被直接内联到使用处。如果 final 变量的值需要运行时计算（如 `static final int X = new Random().nextInt();`），则**会触发**。

### Q6：静态变量和静态代码块谁先执行？

**按源码中出现的先后顺序执行**，谁写在前面谁先执行。不存在"变量一定先于代码块"的固定规则。实例变量和实例代码块同理。

### Q7：子类实例化时，父类构造函数和子类实例代码块谁先执行？

**父类构造函数先执行**。因为实例初始化顺序是：父类实例变量/代码块 → 父类构造函数 → 子类实例变量/代码块 → 子类构造函数。父类构造函数（通过 `super()`）一定先于子类的实例代码块和构造函数。

### Q8：`new` 一个对象时，构造函数是最后执行的吗？

是的。对**单个类**而言，实例初始化顺序是：实例变量赋值 + 实例代码块（按源码顺序）→ 构造函数体。所以构造函数体在实例变量和实例代码块**之后**执行（构造函数首行的 `super()` 除外，它先调用父类构造）。这意味着构造函数可以"覆盖"前面实例变量赋的初值。

---

## 十二、总结

| 知识点 | 核心要点 |
|--------|---------|
| 完整顺序 | 父静 → 子静 → 父实例块 → 父构造 → 子实例块 → 子构造 |
| 静态变量 / 静态代码块 | 类加载时执行，**只 1 次** |
| 静态方法 | 不自动执行，调用几次执行几次 |
| 实例变量 / 实例代码块 / 构造函数 | 每次 `new` 执行，**每 new 一次 1 次** |
| 同层顺序 | 变量赋值与代码块**按源码书写顺序**执行 |
| 类初始化触发 | new、访问静态成员、反射、初始化子类、主类启动 |
| 编译期常量 | `static final` 基本类型/String 字面量**不触发**类初始化 |

**记住三句话**：

1. **顺序口诀：「父静、子静、父实例块、父构造、子实例块、子构造」。**
2. **次数口诀：「静态变量和静态代码块只 1 次，实例部分每 new 一次一次，静态方法调用几次几次」。**
3. **同层的变量和代码块按源码顺序，构造函数在实例变量和实例代码块之后执行（父类先于子类）。**
