## 一、概述

前面两篇我们学习了**字节流**（InputStream / OutputStream），它能读写任意文件，是「万能流」。但字节流有一个痛点：**读写中文文本容易乱码**。

原因是字节流按**字节**处理数据，而中文在不同编码下占多个字节（UTF-8 占 **3 字节**，GBK 占 **2 字节**）。如果用字节流逐字节读取，一个中文字符会被拆开，无法还原成正确的字符。

```
中文字符 "你"
    │ UTF-8 编码
    ▼
[字节1] [字节2] [字节3]   ← 字节流可能在这里截断 → 乱码
```

为了优雅地处理文本，Java 提供了**字符流（Reader / Writer）**：

- **字符流 = 字节流 + 编码**，以**字符（char）**为单位读写。
- 它在底层自动完成「字节 ↔ 字符」的编解码转换，天然适合**纯文本文件**。

本文聚焦 **字符输入流（Reader）**：以字符为单位从文本文件读取数据。

---

## 二、核心概念与体系结构

### 1. Reader 是所有字符输入流的抽象基类

`java.io.Reader` 是抽象类，定义了字符输入流最核心的 `read()` 方法。所有以字符为单位「读」数据的流都继承自它。

```
                          ┌─────────┐
            抽象基类        │ Reader  │   (定义 read / close)
                          └────┬────┘
                               │
                    ┌──────────┴──────────┐
                    │                     │
          ┌──────────────────┐   ┌──────────────────┐
          │ InputStreamReader│   │   BufferedReader  │
          │ (字节→字符的桥梁)│   │  (缓冲 + readLine)│
          └────────┬─────────┘   └────────▲───────────┘
                   │                      │
                   │ 子类(便捷类)          │ (装饰任意 Reader)
                   │                      │
          ┌────────┴─────────┐    BufferedReader(
          │    FileReader    │       new FileReader(f))
          │ (使用默认编码)   │
          └──────────────────┘
```

常见子类对比：

| 流类 | 作用 | 能否指定编码 |
| --- | --- | --- |
| `InputStreamReader` | **字节流 → 字符流的桥梁**，核心类 | ✅ 可指定 |
| `FileReader` | `InputStreamReader` 的便捷子类 | ❌ 用默认编码 |
| `BufferedReader` | 装饰器，提供缓冲 + `readLine()` 按行读 | 取决于被包装的流 |

### 2. 字节流 vs 字符流对比

| 对比项 | 字节流 | 字符流 |
| --- | --- | --- |
| 最小单位 | 字节（byte，8 bit） | 字符（char，Java 中 2 字节） |
| 抽象基类 | `InputStream` / `OutputStream` | `Reader` / `Writer` |
| 是否处理编码 | 否，原样读写字节 | 是，按编码做「字节 ↔ 字符」转换 |
| 适用文件 | 任意文件（二进制、文本都行） | **仅文本文件** |
| 读写中文 | 易乱码 | ✅ 正常 |
| 出现版本 | Java 1.0 | Java 1.1（为弥补字节流文本处理的不足） |

> 💡 **提示：** 记住一句话——**「文本用字符流，二进制用字节流」**。

## 三、FileReader 便捷类

`FileReader` 是 `InputStreamReader` 的子类，**使用系统默认编码**读取文本文件。它的优势是构造简单，适合快速读取「编码与系统默认一致」的文本。

### 1. 构造方法

| 构造方法 | 说明 |
| --- | --- |
| `FileReader(String fileName)` | 通过文件路径构造 |
| `FileReader(File file)` | 通过 `File` 对象构造 |

### 2. 使用示例

```java
import java.io.FileReader;
import java.io.IOException;

public class FileReaderDemo {
    public static void main(String[] args) {
        // FileReader 使用系统默认编码
        try (FileReader fr = new FileReader("/Users/demo/a.txt")) {
            int ch;
            // read() 返回一个字符（0~65535），末尾返回 -1
            while ((ch = fr.read()) != -1) {
                System.out.print((char) ch);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> ⚠️ **注意：** `FileReader` **不能指定编码**！它始终用 JVM 的默认编码（受操作系统影响，Windows 中文版常是 GBK，Linux/Mac 常是 UTF-8）。如果文件编码与默认编码不一致，照样乱码。**生产环境推荐用 `InputStreamReader` 显式指定编码。**

## 四、InputStreamReader：字节流到字符流的桥梁（重点）

`InputStreamReader` 是字符流体系中最关键的类之一——它把一个**字节输入流**「包装」成**字符输入流**，并在构造时**指定编码**，从而彻底解决跨平台编码问题。

### 1. 工作原理

```
字节流 (FileInputStream)        InputStreamReader (指定编码)        程序
   │                                 │                              │
   │  原始字节                        │  按 UTF-8 把字节组合成字符     │
   │  [E4 BD A0] ──────────────────► │  ──►  字符 '你'(0x4F60) ───► │ read() 返回 '你'
```

### 2. 构造方法

| 构造方法 | 说明 |
| --- | --- |
| `InputStreamReader(InputStream in)` | 使用默认编码 |
| `InputStreamReader(InputStream in, String charsetName)` | **指定编码名称**（如 `"UTF-8"`、`"GBK"`） |
| `InputStreamReader(InputStream in, Charset cs)` | 指定 `Charset` 对象（推荐，编译期检查） |

### 3. 使用示例（指定 UTF-8 编码读取）

```java
import java.io.*;
import java.nio.charset.StandardCharsets;

public class InputStreamReaderDemo {
    public static void main(String[] args) {
        // 先建字节流，再用 InputStreamReader 包装并指定 UTF-8 编码
        try (InputStreamReader isr = new InputStreamReader(
                new FileInputStream("/Users/demo/中文.txt"),
                StandardCharsets.UTF_8)) { // ✅ 推荐：用常量，避免拼写错误

            int ch;
            while ((ch = isr.read()) != -1) {
                System.out.print((char) ch); // 正确输出中文
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> 💡 **提示：** 推荐用 `StandardCharsets.UTF_8` 常量而不是字符串 `"UTF-8"`：
> - 常量是 `Charset` 类型，编译期就能检查，避免拼写错误（如 `"UTF_8"`）。
> - 字符串形式若拼写错误，运行时才抛 `UnsupportedEncodingException`。

> ⚠️ **注意：** **读写的编码必须一致**。文件是 UTF-8 编码就用 UTF-8 读，是 GBK 就用 GBK 读，否则依然乱码。

## 五、BufferedReader：缓冲 + 按行读取

`BufferedReader` 是 `Reader` 的装饰器，提供两大能力：
1. **缓冲**：内部维护缓冲区，减少底层 IO 次数，提升读取效率。
2. **`readLine()` 按行读取**：一次返回一行文本（`String`），到末尾返回 `null`，处理文本非常方便。

### 1. 核心方法

| 方法 | 说明 |
| --- | --- |
| `String readLine()` | 读取一行文本，**返回内容不含换行符**；到达末尾返回 `null` |
| `int read()` | 读取单个字符 |
| `int read(char[] cbuf, int off, int len)` | 读入字符数组 |

> 💡 **关键理解：** `readLine()` 返回的字符串**不包含换行符**（`\n` 或 `\r\n`）。如果需要换行，写入时要自己加。

### 2. 使用示例：按行读取 UTF-8 文本

```java
import java.io.*;
import java.nio.charset.StandardCharsets;

public class BufferedReaderDemo {
    public static void main(String[] args) {
        // 经典组合：字节流 + InputStreamReader(指定编码) + BufferedReader(按行读)
        try (BufferedReader br = new BufferedReader(
                new InputStreamReader(
                        new FileInputStream("/Users/demo/中文.txt"),
                        StandardCharsets.UTF_8))) {

            String line;
            // readLine() 返回 null 表示读到文件末尾
            while ((line = br.readLine()) != null) {
                System.out.println(line); // 每行输出
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> 💡 **提示：** 这是**读取文本文件的最佳实践组合**：
> - `FileInputStream`：真正读字节。
> - `InputStreamReader(UTF-8)`：把字节按 UTF-8 解码成字符。
> - `BufferedReader`：缓冲 + 按行读，性能好、代码简洁。

## 六、完整示例：逐行统计行数

```java
import java.io.*;
import java.nio.charset.StandardCharsets;

public class LineCountDemo {
    public static void main(String[] args) {
        int count = 0;
        try (BufferedReader br = new BufferedReader(
                new InputStreamReader(
                        new FileInputStream("/Users/demo/a.txt"),
                        StandardCharsets.UTF_8))) {

            String line;
            while ((line = br.readLine()) != null) {
                count++;
                System.out.println("第 " + count + " 行：" + line);
            }
            System.out.println("文件总行数：" + count);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 七、注意事项与最佳实践

1. **文本用字符流，二进制用字节流**：图片、视频等绝不能用字符流，否则数据损坏。
2. **优先 `InputStreamReader` 而非 `FileReader`**：前者可指定编码，跨平台更可靠。
3. **用 `BufferedReader` 提升性能**：尤其配合 `readLine()` 处理文本。
4. **编码读写一致**：写入用什么编码，读取就用什么编码。
5. **统一 try-with-resources** 关闭资源。

```java
// ✅ 读取文本文件的标准模板
try (BufferedReader br = new BufferedReader(
        new InputStreamReader(
                new FileInputStream("目标文件"),
                StandardCharsets.UTF_8))) {
    String line;
    while ((line = br.readLine()) != null) {
        // 处理每一行
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

## 八、面试常见问题

### Q1：为什么要有字符流？字节流不够用吗？

字节流按字节处理，读写中文时一个字符会被拆成多个字节，容易乱码。字符流 = 字节流 + 编码，以字符为单位读写，自动完成编解码，专门解决文本（尤其含中文）的处理问题。

### Q2：`FileReader` 和 `InputStreamReader` 有什么区别？

`FileReader` 是 `InputStreamReader` 的便捷子类，**只能用系统默认编码**，不能指定编码；`InputStreamReader` 可以在构造时指定任意编码（如 UTF-8、GBK），跨平台更可靠。**生产环境推荐 `InputStreamReader`**。

### Q3：`readLine()` 返回的字符串包含换行符吗？

**不包含**。`readLine()` 会读取一整行内容，但把结尾的换行符（`\n`、`\r`、`\r\n`）去掉。写入时如果需要换行，要自己补上（或用 `BufferedWriter.newLine()`）。

### Q4：字符流能用来复制图片吗？

**不能**。字符流在「字节 ↔ 字符」转换时依赖编解码，而图片是二进制数据，转换会损坏内容。复制二进制文件必须用**字节流**。

### Q5：读取文本时如何避免乱码？

保证**读取编码与文件实际编码一致**。最佳实践是用 `InputStreamReader` 显式指定编码（推荐 `StandardCharsets.UTF_8`），不要依赖系统默认编码（`FileReader` 的默认行为）。

### Q6：`BufferedReader` 的 `readLine()` 怎么判断读完？

当读到文件末尾时，`readLine()` 返回 `null`。所以循环条件是 `while ((line = br.readLine()) != null)`。

---

## 九、总结

- **字符流**为文本而生，自动处理编码，避免中文乱码。
- **`Reader`** 是所有字符输入流的抽象基类。
- **`InputStreamReader`** 是「字节流 → 字符流」的桥梁，**可指定编码**，是核心类。
- **`FileReader`** 是便捷类，但**不能指定编码**，跨平台慎用。
- **`BufferedReader`** 提供缓冲 + `readLine()` 按行读取，是读取文本的**性能首选**。
- **标准组合**：`FileInputStream` + `InputStreamReader(UTF-8)` + `BufferedReader`。
- **资源关闭**统一使用 **try-with-resources**，与整个 IO 流系列保持一致。
