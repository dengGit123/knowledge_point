# 线程相关概念

> 📖 官方教程：[Java Concurrency (Oracle Java Tutorials)](https://docs.oracle.com/javase/tutorial/essential/concurrency/)
> 📖 官方 API：[Thread (Java Platform SE 21)](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Thread.html)

## 一、概述

学习多线程之前，必须先把一组容易混淆的基础概念理清楚：程序、进程、线程是什么关系？并发和并行到底差在哪？同步异步、阻塞非阻塞又怎么区分？这些概念是理解后续所有线程知识的地基。

本文专门梳理这些"前置概念"，不涉及具体代码写法，重在把"是什么、为什么"讲明白。

---

## 二、程序、进程、线程

三者是从"静态到动态"、从"大到小"层层递进的关系：

| 概念 | 英文 | 本质 | 状态 |
| --- | --- | --- | --- |
| 程序 | Program | 磁盘上的代码文件 | 静态 |
| 进程 | Process | 程序运行后的实例 | 动态 |
| 线程 | Thread | 进程中的执行单元 | 动态 |

- **程序**：躺在硬盘里的代码，没运行时只是个文件。例如 `WeChat.exe`，双击之前它什么都不会做。
- **进程**：程序被操作系统"启动"后，就变成了一个进程。操作系统会给它分配内存、文件句柄等资源。打开一次微信就是一个进程，打开三次就是三个进程。
- **线程**：进程内部真正干活的执行单元。一个进程至少有一个线程（主线程），也可以有多个。**线程是 CPU 调度的最小单位**——CPU 实际执行的是线程，而不是进程。

> 💡 **提示：** 一句话记忆——**进程是资源分配的最小单位，线程是 CPU 调度的最小单位**。

---

## 三、进程与线程的关系

用一个**公司**来类比：

- **进程 = 一家公司**：有独立的办公楼和仓库（独立内存空间），不同公司之间互不干涉。
- **线程 = 公司里的员工**：员工共用本公司的办公楼和仓库（共享进程内存），但每人手上的活可以不同。
- 多招几个员工（多线程），公司就能同时推进多项工作。
- 因为员工共用仓库，可能会**抢同一份资料**（线程安全问题），需要排队或上锁。

### 进程 vs 线程 对比

| 对比项 | 进程 | 线程 |
| --- | --- | --- |
| 定义 | 程序运行后的实例 | 进程内的执行单元 |
| 资源 | 操作系统**分配资源**的基本单位 | **CPU 调度**的基本单位 |
| 内存 | 拥有**独立**内存空间 | **共享**所在进程的内存 |
| 数量 | 一个程序可对应多个进程 | 一个进程可含多个线程（至少 1 个） |
| 通信 | 进程间通信（IPC），复杂且慢 | 直接读写共享内存，简单快 |
| 切换开销 | 大（要切换内存空间、页表等） | 小（内存不变，只切换执行上下文） |
| 健壮性 | 一个进程崩溃通常不影响其他进程 | 一个线程崩溃可能导致整个进程挂掉 |

> ⚠️ **注意：** 线程"共享内存"是双刃剑——好处是通信方便，坏处是多个线程同时修改同一份数据会出错，这正是"线程安全"要解决的核心问题。

---

## 四、并发（Concurrency）与并行（Parallelism）

这两个词极易混淆，核心区别只有一句：

> **并发是"同时处理"多件事，并行是"同时做"多件事。**

### 1. 并发

多个任务在**同一时间段内**都有进展，但**不一定是同一时刻**在执行。强调"能同时应对多个任务的能力"。

- 在**单核 CPU** 上，并发靠**时间片快速切换**实现：CPU 在多个任务间飞速来回切换，由于切换极快，人感觉它们"同时"在跑，其实任意一刻只有一个任务在执行。
- 类比：**一个厨师同时张罗几道菜**——炖汤的间隙去切菜，切菜的间隙去摆盘。

### 2. 并行

多个任务在**同一时刻**真正同时执行。强调"真正同时执行的能力"。

- 需要**多核 CPU** 才能实现，每个核跑一个任务，物理上同时进行。
- 类比：**几个厨师各做各的菜**，每人一口锅，同一瞬间真的在同时炒。

```
并发 Concurrency —— 单核靠"快速切换"实现（任意时刻只跑 1 个）
   ┌──T1──┐┌──T2──┐┌──T1──┐┌──T2──┐   交替推进，"看起来"同时

并行 Parallelism —— 多核"真正同时"（同一时刻跑多个）
   核1: ┌──────────T1──────────┐
   核2: ┌──────────T2──────────┐
   核3: ┌──────────T3──────────┘
```

### 3. 对比

| 对比项 | 并发 | 并行 |
| --- | --- | --- |
| 本质 | 时间段内交替推进 | 同一时刻同时执行 |
| 硬件要求 | 单核也能并发 | 必须多核/多处理器 |
| 侧重 | **结构**：程序能处理多任务 | **能力**：硬件真能同时算 |

二者并非对立：并发是一种**程序结构**（把任务设计成可独立推进），并行是一种**执行能力**（有多核真正同时跑）。一个设计良好的并发程序，跑在多核 CPU 上时，任务会被分配到不同核心**并行**执行。

## 五、同步与异步、阻塞与非阻塞

这两组词常被放在一起，但描述的维度不同。

### 1. 同步（Synchronous）与异步（Asynchronous）

描述的是**调用方要不要等待结果**：

- **同步**：调用后必须**一直等到拿到结果**才能继续往下走。
- **异步**：调用后**立即返回**，不等结果，结果好了通过回调、通知等方式再告知。

```java
// 同步：必须等 calc() 算完才能继续
int result = calc();

// 异步：立即返回，结果以后再说
doAsync(() -> { /* 结果就绪后回调 */ });
```

### 2. 阻塞（Blocking）与非阻塞（Non-blocking）

描述的是**线程在等待时是否被挂起**：

- **阻塞**：线程停下来等待，期间什么也干不了（CPU 让给别人）。
- **非阻塞**：线程不停下，能立即得到一个状态（比如"还没好"），可以继续干别的。

> 💡 **提示：** 同步/异步说的是"等不等结果"，阻塞/非阻塞说的是"等的时候线程挂不挂起"。两者可以组合，例如"同步阻塞""异步非阻塞"等。

---

## 六、串行、并发、并行

这三个词描述的是任务执行的**不同方式**：

| 方式 | 说明 | 类比 |
| --- | --- | --- |
| 串行 | 多个任务**一个接一个**执行，前一个完事才轮到下一个 | 一个窗口排队办事 |
| 并发 | 多个任务**交替**推进，看起来同时 | 一个窗口多个业务轮流处理 |
| 并行 | 多个任务**真正同时**执行 | 开多个窗口同时办事 |

---

## 七、上下文切换（Context Switch）

CPU 从执行一个线程切换到另一个线程时，需要保存当前线程的执行状态（寄存器、程序计数器等），再恢复另一个线程的状态，这个过程叫**上下文切换**。

- 切换需要时间，所以线程**不是越多越好**——太多线程会频繁切换，反而拖慢整体性能。
- 进程切换比线程切换开销更大，因为还要切换内存空间（页表、TLB 等）。

> ⚠️ **注意：** 减少"无谓的上下文切换"是多线程性能优化的重要方向，例如合理设置线程池大小、减少锁竞争。

---

## 八、用户线程与守护线程（Daemon）

Java 线程按用途可分为两类：

| 类型 | 说明 | 典型例子 |
| --- | --- | --- |
| 用户线程 | 程序的主干任务线程 | main 主线程、业务线程 |
| 守护线程 | 为用户线程服务的后台线程 | GC 垃圾回收线程 |

**关键规则**：当**所有用户线程都结束时，守护线程会随之被强制结束**，JVM 也就退出了。也就是说，守护线程不会阻止 JVM 退出。

> ⚠️ **注意：** `setDaemon(true)` 必须在 `start()` 之前调用，否则抛 `IllegalThreadStateException`。守护线程里不要写必须收尾的逻辑（如关闭文件、刷盘），因为它可能在任意时刻被中断。

---

## 九、Java 线程与操作系统线程

- 从 **JDK 1.2** 起，Java 线程采用 **1:1 模型**：每个 Java 线程直接对应一个操作系统线程，调度交给操作系统。
- **JDK 21** 起正式引入**虚拟线程（Virtual Thread）**：大量轻量级线程映射到少量平台线程上（M:N 模型），适合超高并发、IO 密集场景，开销远小于传统线程。

---

## 十、总结

| 概念 | 一句话记忆 |
| --- | --- |
| 程序/进程/线程 | 程序是静态代码，进程是运行实例，线程是执行单元 |
| 进程 vs 线程 | 进程是资源单位（独立内存），线程是调度单位（共享内存） |
| 并发 vs 并行 | 并发是"同时处理"（单核也行），并行是"同时做"（要多核） |
| 同步 vs 异步 | 等不等结果 |
| 阻塞 vs 非阻塞 | 等的时候线程挂不挂起 |
| 上下文切换 | 线程切换的保存/恢复开销，线程非越多越好 |
| 用户/守护线程 | 用户线程结束，守护线程被强制终止 |

> 💡 **提示：** 这些概念是理解多线程后续内容的基础，建议先彻底搞懂"进程/线程""并发/并行"两组概念，再进入具体的线程编程。
