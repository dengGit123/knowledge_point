# 线程的休眠

> 📖 官方教程：[Pause Execution with Sleep (Oracle)](https://docs.oracle.com/javase/tutorial/essential/concurrency/sleep.html)
> 📖 官方 API：[Thread.sleep (Java Platform SE 21)](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Thread.html#sleep(long))

## 一、概述

"让线程歇一会"是多线程编程里最常见的操作之一，核心方法就是 `Thread.sleep()`。它能让**当前线程**暂停一段时间，常用于**模拟耗时、定时、轮询间隔**等场景。

本文聚焦 `Thread.sleep()` 的用法、特点、注意事项，以及它与 `wait()` 的本质区别（完整对比见 [sleep和wait](./sleep和wait.md)）。

---

## 二、基本用法

`Thread.sleep(long millis)` 让**当前线程**休眠指定毫秒数。1 秒 = 1000 毫秒。

```java
// 当前线程睡 2 秒
try {
    Thread.sleep(2000); // 2000 毫秒 = 2 秒
} catch (InterruptedException e) {
    // 睡眠期间若被 interrupt()，会抛此异常
    Thread.currentThread().interrupt(); // 建议重新设置中断标志
}
System.out.println("睡醒了");
```

也可以指定到纳秒级（实际精度取决于操作系统）：

```java
Thread.sleep(2, 500_000); // 睡 2 毫秒 + 500000 纳秒（= 0.5 毫秒）
```

---

## 三、更可读的写法：TimeUnit

直接写 `2000` 容易让人误以为是"2 秒"还是"2 毫秒"。`java.util.concurrent.TimeUnit` 枚举提供了更直观的写法：

```java
import java.util.concurrent.TimeUnit;

TimeUnit.SECONDS.sleep(2);   // 睡 2 秒
TimeUnit.MINUTES.sleep(1);   // 睡 1 分钟
TimeUnit.MILLISECONDS.sleep(500); // 睡 500 毫秒
```

> 💡 **提示：** `TimeUnit.X.sleep(n)` 内部其实就是调用 `Thread.sleep()`，但可读性远好于裸数字，**实际开发推荐用 `TimeUnit`**。

---

## 四、四个关键特点

### 1. 休眠的是"当前线程"

`Thread.sleep()` 是**静态方法**，它**总是让当前正在执行的线程休眠**，和"哪个对象调用它"无关。

```java
Thread t = new Thread(() -> {
    Thread.sleep(1000); // 休眠的是 t 这个新线程，不是 main
});
// 在 main 线程里写 t.sleep(1000) 其实休眠的也是 main！
// 因为 sleep 是静态方法，实际等价于 Thread.sleep(1000)
```

> ⚠️ **注意：** 千万别写成 `someThread.sleep(1000)` 以为能让"某个线程"睡——它休眠的是**调用处所在的当前线程**。规范写法是 `Thread.sleep(1000)`。

### 2. 不释放锁

这是 `sleep()` 和 `wait()` **最核心的区别**。`sleep()` 休眠期间**会一直持有它已获得的锁**，其他等待该锁的线程只能继续等。

```java
synchronized (lock) {
    Thread.sleep(5000); // 睡 5 秒，期间一直占着 lock，别的线程拿不到
}
```

### 3. 时间到自动唤醒

`sleep()` 不需要别人来叫醒，时间一到自己醒来，重新进入 `RUNNABLE`（就绪）状态等待 CPU。

### 4. 可被中断

如果线程在 `sleep()` 期间被别的线程 `interrupt()`，会立刻抛出 `InterruptedException` 并提前醒来，同时**中断标志会被清除**。

---

## 五、休眠期间的状态

调用 `Thread.sleep(ms)` 后，线程进入 **`TIMED_WAITING`**（限时等待）状态，**不是** `RUNNABLE`，也不是 `BLOCKED`。

```
RUNNABLE ──sleep(ms)──▶ TIMED_WAITING ──时间到/被中断──▶ RUNNABLE
```

> ⚠️ **注意：** 有人说"sleep 让线程进入阻塞"是**不准确**的。在 Java 的 6 种状态里，`BLOCKED` 特指**等 `synchronized` 锁**；`sleep()` 进入的是 `TIMED_WAITING`。详见 [线程的生命周期(状态)](./线程的生命周期(状态).md)。

## 六、典型应用场景

### 1. 模拟耗时任务（演示、测试）

```java
new Thread(() -> {
    try { TimeUnit.SECONDS.sleep(1); } catch (InterruptedException e) {}
    System.out.println("耗时任务完成");
}).start();
```

### 2. 轮询 / 定时检查间隔

```java
while (!done) {
    doCheck();
    try { TimeUnit.MILLISECONDS.sleep(500); } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        break;
    }
}
```

> 💡 **提示：** 生产环境做"定时"更推荐用 `ScheduledExecutorService`（见 [线程池](./线程池.md)）或定时框架，而不是 `while + sleep`，后者不够精确、也不优雅。

### 3. 简易倒计时

```java
for (int i = 3; i > 0; i--) {
    System.out.println(i);
    TimeUnit.SECONDS.sleep(1);
}
System.out.println("开始！");
```

## 七、常见问题与注意事项

1. **必须处理 `InterruptedException`**：`sleep()` 抛受检异常，要么 `try-catch`，要么向上声明 `throws`。捕获后建议重置中断标志。
2. **精度不保证**：`sleep(1000)` 不代表"精确睡满 1000 毫秒"，取决于 OS 调度和系统时钟，通常会略长。
3. **不释放锁**：在 `synchronized` 块内 `sleep` 会一直占锁，要小心影响并发度。
4. **别用 `t.sleep()`**：它是静态方法，休眠的是当前线程，写成 `t.sleep()` 极具误导性。
5. **不要用 `sleep` 做精确同步**：靠"睡一会儿"来等别的线程完成是不可靠的（执行顺序不可预测，见 [线程的调度](./线程的调度.md)），应该用 `join()` 或 `CountDownLatch`。

---

## 八、总结

| 特点 | 说明 |
| --- | --- |
| 作用对象 | **当前线程**（静态方法） |
| 是否释放锁 | **不释放** |
| 唤醒方式 | 时间到自动唤醒，或被 `interrupt()` 提前唤醒 |
| 所处状态 | `TIMED_WAITING` |
| 推荐写法 | `TimeUnit.SECONDS.sleep(n)` |

> 💡 **提示：** 记住 `sleep()` 的两大记忆点：**"睡的是当前线程"**、**"睡的时候不撒手（不释放锁）"**。这正好和 `wait()` 相反（`wait()` 释放锁），详见 [sleep和wait](./sleep和wait.md)。
