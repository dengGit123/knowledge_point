# 线程的生命周期（状态）

> 📖 官方 API：[Thread.State (Java Platform SE 21)](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Thread.State.html)
> 📖 官方教程：[Thread Information (Oracle)](https://docs.oracle.com/javase/tutorial/essential/concurrency/threadinfo.html)

## 一、概述

一个线程从"创建"到"死亡"，会经历多个**状态**。Java 把线程状态定义在枚举 `java.lang.Thread.State` 里，**共 6 种**。搞懂这 6 种状态以及它们之间如何转换，是理解线程行为、排查线程问题（比如卡住、阻塞）的基础。

> 💡 **提示：** 线程状态是**程序视角的快照**，用 `Thread.getState()` 查看；而线程"什么时候执行、执行多久"是操作系统决定的调度问题，详见 [线程的调度](./线程的调度.md)。

---

## 二、六种状态一览

| 状态 | 含义 | 典型触发条件 |
| --- | --- | --- |
| `NEW` | 已创建但**未启动** | `new Thread()` 后、未调 `start()` |
| `RUNNABLE` | **可运行**（含 OS 层的就绪和运行） | 调用了 `start()` |
| `BLOCKED` | 阻塞，等待 `synchronized` 的锁 | 进入同步块/方法但没抢到锁 |
| `WAITING` | **无限等待**，需被显式唤醒 | `wait()`、`join()`、`LockSupport.park()` |
| `TIMED_WAITING` | **限时等待**，时间到自动唤醒 | `sleep(ms)`、`wait(ms)`、`join(ms)` |
| `TERMINATED` | 终止 | `run()` 执行完或抛出未捕获异常 |

> ⚠️ **注意：** Java **没有单独的"运行中（Running）"状态**。`RUNNABLE` 已经包含了操作系统层面的"就绪（Ready）"和"运行（Running）"两种情况——只要线程"有可能正在 CPU 上执行或在等待 CPU"，Java 都视为 `RUNNABLE`。


## 四、各状态详解

### 1. NEW（新建）

线程对象已创建，但还没调用 `start()`。此时它只是一个普通 Java 对象，**还没有真正的操作系统线程**。

```java
Thread t = new Thread(() -> {});
System.out.println(t.getState()); // NEW
```

### 2. RUNNABLE（可运行）

调用 `start()` 后进入此状态，表示线程"准备好随时被 CPU 执行"。它**包含了两种子状态**：

- **就绪（Ready）**：等着操作系统分配 CPU 时间片。
- **运行（Running）**：正在 CPU 上执行。

Java 把这两者合并为 `RUNNABLE`，因为程序无法区分，也没必要区分。

```java
Thread t = new Thread(() -> {});
t.start();
System.out.println(t.getState()); // RUNNABLE（很可能）
```

### 3. BLOCKED（阻塞）

线程在等待一把 **`synchronized` 的 monitor 锁**。比如它要进入一个 `synchronized` 方法/块，但锁正被别的线程持有，它就会被阻塞。

```java
Object lock = new Object();
Thread t1 = new Thread(() -> { synchronized (lock) { sleepLong(); } });
Thread t2 = new Thread(() -> { synchronized (lock) { /* 要等 t1 释放 */ } });
t1.start(); t2.start();
// t2 很可能是 BLOCKED
```

### 4. WAITING（无限等待）

线程进入**无限期等待**，必须由**其他线程显式唤醒**，否则会一直等下去。常见触发方法：

| 方法 | 唤醒方式 |
| --- | --- |
| `Object.wait()`（无参） | `notify()` / `notifyAll()` |
| `Thread.join()`（无参） | 目标线程执行结束 |
| `LockSupport.park()` | `LockSupport.unpark()` |

### 5. TIMED_WAITING（限时等待）

和 `WAITING` 类似，但有**时间上限**，时间到了会自动唤醒。常见触发方法：

| 方法 | 自动唤醒条件 |
| --- | --- |
| `Thread.sleep(ms)` | 睡眠时间到 |
| `Object.wait(ms)` | 超时或被唤醒 |
| `Thread.join(ms)` | 超时或目标线程结束 |

### 6. TERMINATED（终止）

线程的 `run()` 方法执行完毕（正常返回或抛出未捕获异常），线程进入终止状态。

> ⚠️ **注意：** 终止后的线程**不能再次 `start()`**，会抛 `IllegalThreadStateException`。要再执行任务，必须新建一个线程对象。

## 五、状态转换的关键规律

- **`start()` 之前**：`NEW`；调用后进入 `RUNNABLE`。
- **抢 `synchronized` 锁失败**：进入 `BLOCKED`；抢到锁回到 `RUNNABLE`。
- **`Lock`（如 `ReentrantLock`）抢锁失败**：进入的是 `WAITING`（底层用 `LockSupport.park()`），**而不是** `BLOCKED`——这是 `synchronized` 和 `Lock` 在状态上的一个重要区别。
- **`sleep` / `wait(ms)` / `join(ms)`**：进入 `TIMED_WAITING`。
- **`wait()` / `join()`**：进入 `WAITING`。
- **传统阻塞式 IO（如 `Socket.read()`）**：在操作系统层面线程被挂起，但 **JVM 感知不到**这种挂起，所以 Java 层面仍显示 `RUNNABLE`。
- **`run()` 结束**：进入 `TERMINATED`，**不可重新 `start()`**。

## 六、用代码观察状态

```java
public class StateDemo {
    public static void main(String[] args) throws InterruptedException {
        Object lock = new Object();

        Thread t = new Thread(() -> {
            synchronized (lock) {
                try {
                    lock.wait(); // 进入 WAITING
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });

        System.out.println("1: " + t.getState()); // NEW

        t.start();
        Thread.sleep(100); // 等子线程真的进入 wait
        System.out.println("2: " + t.getState()); // WAITING

        synchronized (lock) {
            lock.notify(); // 唤醒 t
        }
        Thread.sleep(100); // 等 t 执行完
        System.out.println("3: " + t.getState()); // TERMINATED
    }
}
```

## 七、常见误区与注意事项

1. **`sleep()` 不是 `RUNNABLE`**：`Thread.sleep()` 会让线程进入 **`TIMED_WAITING`**，不要误以为它还在"可运行"。
2. **阻塞式 IO 在 Java 层显示 `RUNNABLE`**：所以看到一个线程是 `RUNNABLE`，不代表它真的在用 CPU——它可能在等网络/磁盘 IO。
3. **`BLOCKED` 只和 `synchronized` 有关**：用 `ReentrantLock` 等待锁时是 `WAITING`，不是 `BLOCKED`。
4. **不要用 `getState()` 做同步控制**：状态是某一瞬间的快照，下一刻可能就变了，不能用来替代真正的同步机制。
5. **终结线程无法复用**：`TERMINATED` 后只能新建线程，不能 `start()` 复活。

---

## 八、总结

| 状态 | 触发要点 |
| --- | --- |
| `NEW` | 创建未启动 |
| `RUNNABLE` | `start()` 后（含就绪+运行） |
| `BLOCKED` | 等 `synchronized` 锁 |
| `WAITING` | `wait()`/`join()`/`park()` 无限等 |
| `TIMED_WAITING` | `sleep(ms)`/`wait(ms)`/`join(ms)` 限时等 |
| `TERMINATED` | `run()` 结束，不可复活 |

> 💡 **提示：** 记住三组关键转换：**`start()` 进 RUNNABLE**、**抢 synchronized 失败进 BLOCKED**、**sleep/wait 进 TIMED_WAITING/WAITING**。掌握这些，就能看懂 `jstack` 里的线程堆栈状态了。
