## 一、概述

学完概念后，第一个要落地的就是：**在 Java 里怎么把一个线程创建出来、启动它、并做日常控制**。本文覆盖线程的四种创建方式、启动方法、命名、中断、优先级、守护线程等"创建之后的使用"内容。

> 💡 **延伸：** 线程的休眠、合并、调度、生命周期等有专门文档，本文聚焦"创建"与"基本使用"。

## 二、线程与任务的关系（核心思维）

> 💡 这是理解 Java 线程编程最重要的一组概念：**搞清楚"线程"和"任务"是两个不同的东西。** 后面所有的创建方式，本质都是围绕这对关系展开的。

### 1. 线程是"执行者"，任务是"要做的事"

- **线程（Thread）**：真正干活的**执行者**，是 CPU 调度的单位。
- **任务（Runnable / Callable）**：一段**要做的事**的描述，本质是一段代码/逻辑。

一句话类比：**线程是工人，任务是工单。** 工人按工单干活，工单本身不会自己跑起来——必须有人（线程）去执行它。

### 2. 线程是被任务"驱动"的

打开 `Thread` 源码，会发现它内部持有一个 `Runnable target` 字段：

```java
// Thread 简化示意（说明原理，非完整源码）
class Thread implements Runnable {
    private Runnable target; // 线程启动后会回调 target.run()

    public Thread(Runnable target) { this.target = target; }

    @Override
    public void run() {
        if (target != null) target.run(); // 线程跑起来后，执行的就是 target 这段任务
    }
}
```

这揭示了本质：**线程本身没有"固定要做的事"，它只是执行你塞进去的那个 `Runnable` 任务。** 如果什么都不传，线程启动后什么都不做就结束了。

> 💡 **关键结论：** 在 Java 中，**真正启动一个新线程的代码只有一条——`new Thread().start()`**。所谓的"几种创建方式"，本质都是**把"要执行的任务"用不同形式交给 `Thread`**（继承重写 `run()`、传 `Runnable`、传 `FutureTask`、交给线程池……）。**"几种方式"写的是任务，"开线程"靠的是 `Thread`。**

### 3. 任务和线程可以灵活组合

正因为任务和线程是分开的，它们能灵活搭配，远不是"一个任务配一个线程"那么死板：

| 组合关系 | 例子 | 说明 |
| --- | --- | --- |
| **一个任务 → 多个线程** | 3 个线程跑同一个卖票 `Runnable` | **共享任务**，常用于资源共享场景 |
| **一个线程 → 多个任务** | 线程池里一个线程依次跑 N 个任务 | **复用线程**，避免反复创建销毁 |
| **多个任务 → 线程池** | 往线程池提交一堆任务 | 任务排队，由池内线程轮流执行 |

```java
// 一个任务 → 多个线程（任务共享）
Runnable ticketTask = new Ticket();
new Thread(ticketTask, "窗口1").start();
new Thread(ticketTask, "窗口2").start(); // 同一个任务，交给两个线程跑

// 多个任务 → 线程池（线程复用）
ExecutorService pool = Executors.newFixedThreadPool(2); // 示例，生产用 ThreadPoolExecutor
pool.execute(taskA); // 池里 2 个线程轮流执行 taskA/B/C...
pool.execute(taskB);
pool.execute(taskC);
```

> ⚠️ **注意：** "一个任务被多个线程共享"时，如果任务内部操作了共享数据（如 `ticket--`），就会产生**线程安全问题**，必须加同步，详见 [线程的安全](./线程的安全.md)。

### 4. 为什么要分离任务和线程

| 好处 | 说明 |
| --- | --- |
| **解耦** | 任务只描述"做什么"，不关心"谁来跑、怎么跑" |
| **复用** | 同一任务可交给任意执行器（`Thread` / 线程池 / `ForkJoinPool`） |
| **资源共享** | 一个任务实例可被多个线程共享 |
| **灵活切换** | 想换"执行策略"时，任务代码不用改 |

### 5. Executor 框架：把分离做到极致

`java.util.concurrent` 的 **Executor 框架**把"提交任务"和"执行任务"彻底分离：

```java
ExecutorService pool = ...;         // "执行者"：谁来跑、用几个线程
pool.execute(() -> doSomething());  // "任务"：做什么
```

你只管"**提交任务**"，"用几个线程、什么时候跑、怎么复用"全交给执行器决定。这就是为什么**实际开发推荐用线程池**——它最彻底地践行了"任务与线程分离"的思想，详见 [线程池](./线程池.md)。

### 6. 常见误区

- ❌ **"`Runnable` 就是一个线程"** —— 错。`Runnable` 只是一个任务接口（带 `run()` 方法），它本身**不会产生新线程**，必须交给 `Thread` 才会被执行。
- ❌ **"一个任务只能对应一个线程"** —— 错。同一个任务可以被多个线程执行（共享）。
- ❌ **"线程池里的任务和线程是一一对应的"** —— 错。少数线程会复用执行大量任务。
- ❌ **"继承 Thread 写的就不是任务"** —— 不准确。继承 `Thread` 重写的 `run()` 本质也是任务代码，只不过把"任务"和"线程"绑死在了一起，所以扩展性差、不推荐。

> 💡 **一句话总结：** **"任务"用 `Runnable`/`Callable` 描述（做什么），"线程"用 `Thread`/线程池承载（谁来做），启动靠 `start()`（开干）。** 理解这组关系，后面所有"创建方式"都只是"换个写法传任务"而已。

## 三、四种创建方式

### 方式一：继承 Thread 类

`Thread` 本身是个类，继承它并重写 `run()`，任务写在 `run()` 里。

```java
// 方式一：继承 Thread，重写 run()
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " 执行中");
    }
}

public class Demo {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        t1.start(); // start() 才会真正启动新线程
        t2.start();
    }
}
```

> ⚠️ **注意：** Java 是**单继承**，继承了 `Thread` 就不能再继承别的类，扩展性差，**不推荐**。

### 方式二：实现 Runnable 接口（推荐）

实现 `Runnable` 的 `run()` 方法，再用 `Thread` 包装启动。把"任务"和"线程"解耦。

```java
// 方式二：实现 Runnable（推荐）
class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " 执行中");
    }
}

public class Demo {
    public static void main(String[] args) {
        Runnable r = new MyRunnable();
        new Thread(r, "线程-A").start(); // 把任务 r 交给线程
        new Thread(r, "线程-B").start(); // 同一个 r 可被多个线程共享

        // Runnable 是函数式接口，可用 Lambda 简化
        new Thread(() -> System.out.println("Lambda 线程"), "线程-C").start();
    }
}
```

**推荐理由**：

- **避免单继承局限**：实现接口不影响继承其他类。
- **任务与线程分离**：`Runnable` 只描述"做什么"，由谁执行可灵活搭配。
- **资源共享**：同一个 `Runnable` 实例可被多个线程共享（如卖票场景）。

### 方式三：实现 Callable + FutureTask（需要返回值时）

`Runnable` 的 `run()` **没有返回值**、**不能抛出受检异常**。如果任务需要返回结果，用 `Callable`。

> ⚠️ **注意：** `Callable` **不能直接交给 `Thread`**（`Thread` 只认 `Runnable`），必须先用 `FutureTask` 包装一层——`FutureTask` 同时实现了 `Runnable` 和 `Future`，这样就能交给 `Thread`。

```java
import java.util.concurrent.Callable;
import java.util.concurrent.FutureTask;
import java.util.concurrent.ExecutionException;

// 方式三：Callable + FutureTask
class MyCallable implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {   // call() 有返回值，可抛受检异常
        int sum = 0;
        for (int i = 1; i <= 100; i++) sum += i;
        return sum;                            // 返回计算结果
    }
}

public class Demo {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        // 1. 用 FutureTask 包装 Callable
        FutureTask<Integer> future = new FutureTask<>(new MyCallable());
        // 2. 把 FutureTask（它实现了 Runnable）交给 Thread
        new Thread(future, "计算线程").start();

        // 3. get() 会阻塞，直到任务执行完并拿到结果
        Integer result = future.get();
        System.out.println("结果：" + result); // 5050
    }
}
```

### 方式四：线程池（实际开发最常用）

实际项目里**几乎不手动 `new Thread()`**，而是用**线程池**统一创建和管理线程。线程池既能执行 `Runnable`，也能执行 `Callable`。

```java
import java.util.concurrent.*;

// 示例：生产环境应直接用 new ThreadPoolExecutor(...)，详见《线程池》
ExecutorService pool = Executors.newFixedThreadPool(3);

pool.execute(() -> System.out.println("Runnable 任务，无返回值"));   // 提交 Runnable
Future<String> f = pool.submit(() -> "Callable 任务，有返回值");    // 提交 Callable
System.out.println(f.get());                                        // 拿返回值

pool.shutdown();
```

> ⚠️ **注意：** `Executors` 工具方法存在 OOM 风险，生产环境应直接用 `new ThreadPoolExecutor(...)`，详见 [线程池](./线程池.md)。

### 四种方式对比

| 对比项 | 继承 Thread | 实现 Runnable | 实现 Callable | 线程池 |
| --- | --- | --- | --- | --- |
| 返回值 | 无 (`void run()`) | 无 | 有 (`V call()`) | 有/无均可 |
| 受检异常 | 不能抛 | 不能抛 | 可抛 | 取决于任务 |
| 继承局限 | 受单继承限制 | 无 | 无 | 无 |
| 资源共享 | 不便 | 方便 | 方便 | 方便 |
| 线程管理 | 手动创建销毁 | 手动创建销毁 | 手动创建销毁 | 复用、统一管理 |
| 推荐度 | 不推荐 | 推荐 | 推荐（要返回值） | 强烈推荐 |

## 四、启动线程：start() vs run()（高频考点）

这是面试必问、也是新手最常踩的坑。

| 对比项 | `start()` | `run()` |
| --- | --- | --- |
| 作用 | 启动**新线程**，由它回调 `run()` | 就是**普通方法调用** |
| 是否开新线程 | 是 | 否（在调用者线程里执行） |
| 调用次数 | 一个 `Thread` 只能调一次 | 可多次调用 |
| 重复调用 | 抛 `IllegalThreadStateException` | 正常 |

```java
Thread t = new Thread(() ->
    System.out.println("执行线程：" + Thread.currentThread().getName()));

// 错误：直接调 run()，没开新线程，代码在 main 线程里同步执行
t.run();   // 输出：执行线程：main

// 正确：调 start()，启动新线程，由新线程回调 run()
t.start(); // 输出：执行线程：Thread-0

// 错误：同一个 Thread 不能重复 start()
t.start(); // 抛 IllegalThreadStateException
```

> 💡 **提示：** `start()` 底层会调用**本地方法（native）**，由 JVM 在操作系统层面创建新线程，再由新线程回调 `run()`。日常**只调 `start()`**，绝不手动调 `run()`。

## 五、线程命名与获取当前线程

```java
// 创建时通过构造方法命名
Thread t = new Thread(() -> {}, "我的线程");

// 或用 setName() 设置
t.setName("worker-1");

// 获取当前正在执行的线程（任意代码里都能用）
Thread current = Thread.currentThread();
String name = current.getName();   // 如 "main"、"worker-1"
```

> 💡 **提示：** 给线程起有意义的名字，**排查问题时非常重要**——堆栈、日志、`jstack` 都会显示线程名。线程池里建议通过 `ThreadFactory` 自定义命名。

## 六、中断线程：interrupt()（重要且易错）

Java **没有"强制停止线程"的安全 API**（已废弃的 `stop()` 极不安全）。正确做法是**协作式中断**：一个线程通过 `interrupt()` 发出"请停止"的请求，目标线程**自己检查并决定何时退出**。

```java
Thread t = new Thread(() -> {
    // 推荐模式：在循环里检查中断标志
    while (!Thread.currentThread().isInterrupted()) {
        // 干活...
    }
    System.out.println("收到中断，正常收尾后退出");
}, "worker");
t.start();

t.interrupt(); // 请求中断：把 t 的中断标志设为 true
```

相关方法：

| 方法 | 作用 |
| --- | --- |
| `interrupt()` | 把目标线程的**中断标志**设为 true |
| `isInterrupted()` | 检查中断标志（不清除） |
| `Thread.interrupted()` | 检查并**清除**当前线程的中断标志（静态方法） |

> ⚠️ **注意（经典坑）：** 如果线程正卡在 `sleep()` / `wait()` / `join()` 中被 `interrupt()`，会立刻抛出 `InterruptedException`，**并且中断标志会被自动清除**（变回 false）。所以捕获异常后通常要**重新设置中断标志**或向上抛，否则上层无法感知中断：

```java
try {
    Thread.sleep(5000);
} catch (InterruptedException e) {
    // 异常抛出时，中断标志已被清除
    Thread.currentThread().interrupt(); // 重新设置，让上层能检测到
}
```

## 七、优先级

```java
Thread t = new Thread(() -> {});
t.setPriority(Thread.MAX_PRIORITY); // 10，最大
// 范围：Thread.MIN_PRIORITY(1) ~ Thread.MAX_PRIORITY(10)，默认 NORM_PRIORITY(5)
```

> ⚠️ **注意：** 优先级**只是给调度器一个提示**，**不保证**高优先级线程一定先执行或执行更多。不能依赖优先级来保证执行顺序，详见 [线程的调度](./线程的调度.md)。

## 八、守护线程（Daemon）

为**用户线程**服务的后台线程。**当所有用户线程都结束时，守护线程会被强制结束**（最典型的是 GC 线程）。

```java
Thread t = new Thread(() -> {
    while (true) {
        // 后台任务，如心跳、监控
    }
});
t.setDaemon(true); // 必须在 start() 之前设置
t.start();
```

> ⚠️ **注意：** `setDaemon(true)` **必须在 `start()` 之前调用**，否则抛 `IllegalThreadStateException`。守护线程中不要做必须完成的收尾工作（如关闭文件、刷盘），因为它可能在任意时刻被中断。

## 十、常见问题与注意事项

1. **不要调 `run()`，要调 `start()`**：调 `run()` 是普通方法调用，不会开新线程。
2. **同一个 `Thread` 不能 `start()` 两次**：会抛 `IllegalThreadStateException`，要再跑得新建线程。
3. **优先 `Runnable`/`Callable` 而非继承 `Thread`**：解耦、可共享、不受单继承限制。
4. **生产环境用线程池**，而不是手动 `new Thread()`：便于复用、限流、监控。
5. **不要用 `stop()`/`suspend()`/`resume()`**：已废弃，不安全，用 `interrupt()` 协作式中断。
6. **线程命名要规范**：排查问题（日志、`jstack`）全靠它。

## 十一、总结

| 知识点 | 关键内容 |
| --- | --- |
| 线程 vs 任务 | 线程是执行者，任务（`Runnable`/`Callable`）是要做的事；真正开线程只有 `new Thread().start()`，其余都是"传任务" |
| 四种方式 | 继承 Thread / 实现 Runnable / 实现 Callable+FutureTask / 线程池 |
| start vs run | start 开新线程，run 是普通调用 |
| 中断 | 协作式 `interrupt()`，遇阻塞抛异常后要重置标志 |
| 守护线程 | 必须在 start 前设置，用户线程结束它就被终止 |
| 优先级 | 只是提示，不保证顺序 |

> 💡 **提示：** 记住一句话——**"任务"用接口描述（Runnable/Callable），"线程"用 Thread/线程池承载，启动靠 start()**。这是 Java 线程编程的核心思维。
