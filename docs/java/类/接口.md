# 接口（Interface）

> 官方文档：[Java Language Specification: Interfaces](https://docs.oracle.com/javase/specs/jls/se17/html/jls-9.html) | [菜鸟教程：Java 接口](https://www.runoob.com/java/java-interfaces.html)

## 一、概述

接口（Interface）是 Java 中一种引用类型，它是方法的集合，用于定义一组规范（契约）。你可以把接口想象成一份「合同」或「插座标准」：

- **插座标准**规定了插头的形状、电压、电流参数，但不管你插的是什么电器（电视、电饭煲、电脑），只要符合标准就能通电；
- **接口**规定了一组方法签名（做什么），但不关心具体实现（怎么做），只要某个类声明实现了这个接口，就必须履行这份"合同"。

接口的核心价值在于**抽象、多态和解耦**：它让程序面向"能力"编程，而不是面向"具体实现"编程。

### 1.1 接口定义的基本语法

```java
public interface 接口名 {
    // 常量
    // 抽象方法
    // 默认方法（Java 8+）
    // 静态方法（Java 8+）
    // 私有方法（Java 9+）
}
```

```java
// 定义接口
public interface Flyable {
    // 接口中的抽象方法（自动是 public abstract，可省略）
    void fly();
}

// 实现接口
public class Bird implements Flyable {
    @Override
    public void fly() {                // 实现接口的方法
        System.out.println("鸟儿飞翔");
    }
}
```

> 💡 **提示：** `interface` 是定义接口的关键字，`implements` 是实现接口的关键字。一个类可以实现多个接口，接口之间用逗号分隔。

## 二、核心概念

### 2.1 接口的基本特点

| 特点 | 说明 |
| --- | --- |
| 支持多实现 | 一个类可以 `implements` 多个接口，用逗号分隔 |
| 不能实例化 | `new Flyable()` 是错误的，必须通过实现类对象使用 |
| 类型向上转型 | 实现类对象可以赋值给接口类型引用（多态） |
| 实现类必须重写 | 实现类必须重写接口中所有抽象方法（除非该类是抽象类） |
| 无构造器 | 接口没有构造方法 |

```java
public interface Swimable {
    void swim();
}

// 一个类可以同时实现多个接口
public class Duck implements Flyable, Swimable {
    @Override
    public void fly() {
        System.out.println("鸭子扑腾着翅膀飞");
    }

    @Override
    public void swim() {
        System.out.println("鸭子在水里游");
    }
}
```

```java
public class InterfaceDemo {
    public static void main(String[] args) {
        // ❌ 接口不能直接实例化
        // Flyable f = new Flyable();

        // ✅ 通过实现类对象，向上转型为接口类型（多态）
        Flyable flyable = new Duck();
        flyable.fly();                     // 输出: 鸭子扑腾着翅膀飞

        Swimable swimable = new Duck();
        swimable.swim();                   // 输出: 鸭子在水里游
    }
}
```

### 2.2 接口的"能力"语义

接口表达的是 **can-do（能做什么）/ has-a（拥有某种能力）** 关系，而非继承的 is-a 关系：

```
┌─────────────────────────────────────────────┐
│   Flyable 接口  (can-do: 能飞)              │
│   └─ void fly();                            │
└─────────────────────────────────────────────┘
              ▲ implements
              │
┌─────────────────────────────────────────────┐
│   Bird 类 (本质: 是一只鸟)                   │
│   同时具备 fly() 的能力                       │
└─────────────────────────────────────────────┘
```

一个对象可以拥有多种"能力"（实现多个接口），但它的"本质"（继承自哪个父类）只有一条线。

## 三、接口的成员及各版本演进

接口里的成员随着 Java 版本不断丰富，从最初的"纯抽象"逐渐演变成更强大的工具。

### 3.1 成员总览表

| 成员类型 | 引入版本 | 默认修饰符（可省略） | 是否有方法体 | 作用 |
| --- | --- | --- | --- | --- |
| 常量 | Java 1.0 | `public static final` | — | 定义全局常量 |
| 抽象方法 | Java 1.0 | `public abstract` | 无 | 定义契约 |
| 默认方法 | Java 8 | `public default` | 有 | 接口升级，提供默认实现 |
| 静态方法 | Java 8 | `public static` | 有 | 工具方法，接口直接调用 |
| 私有方法 | Java 9 | `private` / `private static` | 有 | 接口内部代码复用 |

> ⚠️ **注意：** 接口中的成员访问权限**默认都是 public**（除私有方法外），即使你不写 public，它也是 public 的，不能写成 protected 或默认权限。

### 3.2 常量

接口中所有的变量都会自动成为 `public static final` 常量，必须在声明时赋值。

```java
public interface Constants {
    // 下面两种写法完全等价
    int MAX_SPEED = 300;                          // 简写
    public static final int MIN_SPEED = 0;        // 完整写法

    String NAME = "交通工具";
}
```

```java
public class ConstantTest {
    public static void main(String[] args) {
        System.out.println(Constants.MAX_SPEED);  // 300
        // Constants.MAX_SPEED = 200;             // ❌ final 常量不可重新赋值
    }
}
```

### 3.3 抽象方法

抽象方法只有方法声明，没有方法体，由实现类去实现。

```java
public interface Comparable<T> {
    int compareTo(T o);                           // 自动是 public abstract
}
```

### 3.4 默认方法（Java 8+）

默认方法用 `default` 关键字修饰，**有方法体**，主要解决「接口升级」问题：

> 想象接口是一个已经发布的标准，已经有很多类实现了它。如果往接口里加一个新抽象方法，所有实现类都得改，会"牵一发动全身"。默认方法提供了向后兼容的方案——新增方法带默认实现，旧实现类无需改动。

```java
public interface Vehicle {
    void start();

    // 默认方法：新增的、带默认实现的方法
    default void honk() {
        System.out.println("嘀嘀嘀~鸣笛");
    }
}

// 旧实现类无需实现 honk()，直接继承默认实现
public class Car implements Vehicle {
    @Override
    public void start() {
        System.out.println("汽车启动");
    }
}

// 也可以重写默认方法
public class ElectricCar implements Vehicle {
    @Override
    public void start() {
        System.out.println("电车静音启动");
    }

    @Override
    public void honk() {
        System.out.println("嗡嗡嗡~电车喇叭");
    }
}
```

### 3.5 静态方法（Java 8+）

接口中的静态方法属于接口本身，用 `接口名.方法名()` 调用，常作为接口相关的工具方法。

```java
public interface MathOperation {
    static int add(int a, int b) {
        return a + b;
    }

    static int square(int n) {
        return n * n;
    }
}

// 使用：直接用接口名调用，不需要对象
class StaticMethodTest {
    public static void main(String[] args) {
        int sum = MathOperation.add(3, 5);        // 8
        int sq  = MathOperation.square(4);        // 16
    }
}
```

### 3.6 私有方法（Java 9+）

当接口中有多个默认方法/静态方法存在重复逻辑时，可以提取成私有方法供内部复用，避免代码重复。

```java
public interface DataProcessor {

    default void processString(String s) {
        checkNotNull(s);                          // 调用私有方法
        System.out.println("处理字符串: " + s);
    }

    default void processNumber(int n) {
        checkNotNull(n);                          // 复用私有方法
        System.out.println("处理数字: " + n);
    }

    // Java 9+ 私有方法，仅供接口内部使用
    private void checkNotNull(Object obj) {
        if (obj == null) {
            throw new IllegalArgumentException("参数不能为 null");
        }
    }

    // 私有静态方法，供静态方法复用
    private static void log(String msg) {
        System.out.println("[LOG] " + msg);
    }
}
```

## 四、接口之间的多继承

接口可以使用 `extends` 继承**多个**父接口（注意：类只能单继承，但接口可以多继承），子接口会继承所有父接口的方法。

```
┌──────────────┐    ┌──────────────┐
│  InterfaceA  │    │  InterfaceB  │
└──────┬───────┘    └──────┬───────┘
       │ extends            │ extends
       └─────────┬──────────┘
                 ▼
         ┌──────────────┐
         │  InterfaceC  │   (继承多个父接口)
         └──────────────┘
```

```java
public interface A {
    void methodA();
}

public interface B {
    void methodB();
}

// 接口可以继承多个接口
public interface C extends A, B {
    void methodC();
}

// 实现 C 的类必须实现 A、B、C 的所有抽象方法
public class CImpl implements C {
    @Override public void methodA() {}
    @Override public void methodB() {}
    @Override public void methodC() {}
}
```

## 五、函数式接口

函数式接口（Functional Interface）是指**有且仅有一个抽象方法**的接口，它是 Lambda 表达式的基础。

```java
@FunctionalInterface        // 这个注解让编译器帮你校验：是否只有一个抽象方法
public interface Calculator {
    int calc(int a, int b);     // 唯一的抽象方法
}
```

```java
public class FunctionalDemo {
    public static void main(String[] args) {
        // 匿名内部类写法
        Calculator c1 = new Calculator() {
            @Override
            public int calc(int a, int b) {
                return a + b;
            }
        };

        // Lambda 写法（更简洁）
        Calculator c2 = (a, b) -> a + b;
        Calculator c3 = (a, b) -> a * b;

        System.out.println(c2.calc(3, 4));        // 7
        System.out.println(c3.calc(3, 4));        // 12
    }
}
```

> 💡 **提示：** 默认方法、静态方法、`Object` 类的方法（如 `equals`）不计入函数式接口的"唯一抽象方法"计数。JDK 内置了很多函数式接口，如 `Runnable`、`Comparator`、`Function`、`Predicate` 等。

## 六、默认方法的冲突问题

当一个类同时实现了两个含有**相同签名默认方法**的接口时，会产生冲突，编译器不知道用哪一个。解决办法是：**在实现类中重写该方法，手动指定逻辑**。

```java
public interface Interface1 {
    default void hello() {
        System.out.println("Hello from Interface1");
    }
}

public interface Interface2 {
    default void hello() {
        System.out.println("Hello from Interface2");
    }
}

// ❌ 不重写会编译错误：两个接口有同名默认方法，产生冲突
// public class Conflict implements Interface1, Interface2 { }

// ✅ 必须重写，解决冲突
public class Conflict implements Interface1, Interface2 {
    @Override
    public void hello() {
        System.out.println("我自己实现的 hello");
    }
}

// ✅ 或者明确调用某个父接口的默认方法
public class Conflict2 implements Interface1, Interface2 {
    @Override
    public void hello() {
        Interface1.super.hello();                 // 调用 Interface1 的默认方法
    }
}
```

> ⚠️ **注意：** 当父类的方法和接口的默认方法冲突时，**父类的方法优先**（类优先原则）。

## 七、接口的应用场景

### 7.1 策略模式

定义一系列算法，把它们各自封装，并且使它们可互相替换。

```java
// 策略接口
public interface SortStrategy {
    void sort(int[] arr);
}

// 具体策略：冒泡
public class BubbleSort implements SortStrategy {
    @Override
    public void sort(int[] arr) {
        System.out.println("使用冒泡排序");
        // ...省略排序实现
    }
}

// 具体策略：快速
public class QuickSort implements SortStrategy {
    @Override
    public void sort(int[] arr) {
        System.out.println("使用快速排序");
        // ...省略排序实现
    }
}

// 上下文：依赖接口，运行时切换策略
public class Sorter {
    private SortStrategy strategy;
    public void setStrategy(SortStrategy strategy) {
        this.strategy = strategy;
    }
    public void doSort(int[] arr) {
        strategy.sort(arr);
    }
}
```

### 7.2 回调

```java
public interface Callback {
    void onComplete(String result);
}

public class AsyncTask {
    public void execute(Callback callback) {
        // ...执行耗时操作
        callback.onComplete("任务完成");          // 完成后回调通知
    }
}
```

### 7.3 依赖倒置（解耦）

高层模块不应依赖低层模块，二者都应依赖抽象（接口）。

```java
public interface MessageSender {
    void send(String msg);
}

public class EmailSender implements MessageSender {
    @Override public void send(String msg) { System.out.println("邮件发送: " + msg); }
}

public class SmsSender implements MessageSender {
    @Override public void send(String msg) { System.out.println("短信发送: " + msg); }
}

// 业务类依赖接口（抽象），而不是具体实现，便于替换
public class OrderService {
    private final MessageSender sender;            // 依赖接口
    public OrderService(MessageSender sender) {
        this.sender = sender;
    }
    public void notifyUser() {
        sender.send("您的订单已发货");
    }
}
```

## 八、接口 vs 抽象类对比

| 对比维度 | 接口（interface） | 抽象类（abstract class） |
| --- | --- | --- |
| 设计语义 | can-do / has-a，表示「能力」「契约」 | is-a，表示「本质」「分类」 |
| 关键字 | `interface` 定义，`implements` 实现 | `abstract class` 定义，`extends` 继承 |
| 多继承 | 支持多实现、接口间多继承 | 类只能单继承 |
| 成员变量 | 只能是 `public static final` 常量 | 可以有各种修饰符的普通成员变量 |
| 构造器 | 没有 | 有（供子类 super 调用） |
| 方法 | 抽象/默认/静态/私有方法 | 抽象方法 + 普通方法均可 |
| 实例化 | 都不能直接实例化 | 都不能直接实例化 |
| 使用倾向 | 定义跨类型家族的公共行为 | 共享代码 + 部分实现 |

> 💡 **提示：** 经验法则——如果一组类有共同的「字段或核心代码」要复用，用抽象类；如果只是要表达一组类都「具备某种能力」，用接口。Java 8 之后接口也可以有默认方法，使二者的界限模糊，但语义层面的区分依然重要。

## 九、面试常见问题

### Q1：接口能实例化吗？为什么？

不能直接实例化。接口是抽象的规范，没有构造器，也没有具体实现。但可以通过实现类对象进行向上转型，例如 `Flyable f = new Bird();`，这本质是创建实现类对象、再用接口类型引用它（多态）。也可以用匿名内部类/Lambda 创建实现接口的"匿名对象"。

### Q2：接口里的变量都是什么修饰符？

都是 `public static final`，即全局静态常量，必须初始化且不可修改。即使不写这些修饰符，编译器也会自动补全。

### Q3：Java 8 的默认方法解决了什么问题？它和抽象方法有什么区别？

默认方法用 `default` 修饰，**有方法体**，主要用于接口升级——向已发布的接口添加新方法而不破坏旧的实现类（旧类不强制重写，直接使用默认实现）。抽象方法没有方法体，实现类必须重写。

### Q4：一个类实现了两个有相同默认方法的接口会怎样？

会产生「默认方法冲突」编译错误。解决办法是在实现类中重写该方法，逻辑自己写，或用 `接口名.super.方法名()` 明确调用某个父接口的默认实现。

### Q5：接口和抽象类应该怎么选？

- 表达「能力/契约」、需要跨不同继承体系复用行为、需要多重继承时，选接口；
- 需要共享成员变量或核心代码、子类间有强 is-a 关系时，选抽象类。
- 实际项目中，常常"接口定义契约 + 抽象类提供骨架实现"组合使用，如 JDK 的 `List`（接口）与 `AbstractList`（抽象类）。

### Q6：函数式接口和 Lambda 有什么关系？

函数式接口是只有一个抽象方法的接口，Lambda 表达式的类型本质上就是函数式接口的实例。Lambda 让我们用更简洁的方式创建函数式接口的匿名实现，省去了匿名内部类的样板代码。

## 十、总结

接口是 Java 实现抽象、多态、解耦的核心机制，它定义「能做什么」的契约。从 Java 1.0 的纯抽象方法，到 Java 8 的默认方法和静态方法，再到 Java 9 的私有方法，接口的能力不断增强。掌握接口的关键在于：

1. **理解语义**：接口表达能力（can-do），抽象类表达本质（is-a）；
2. **熟悉成员**：常量、抽象方法、默认方法、静态方法、私有方法各自的用途；
3. **善用多态与解耦**：面向接口编程，提升代码扩展性和可维护性；
4. **注意陷阱**：默认方法冲突、接口不能实例化、变量都是常量等。
