# 静态代码块（static block）

> 官方文档：[Java Language Specification - Static Initializers](https://docs.oracle.com/javase/specs/jls/se17/html/jls-8.html#jls-8.7)、[菜鸟教程 Java 静态代码块](https://www.runoob.com/java/java-static.html)

## 一、概述

**静态代码块**是用 `static` 关键字修饰、用大括号 `{}` 包裹的代码块，也叫**静态初始化块**。它的作用是在**类加载时**执行一次性的初始化操作。

```java
static {
    // 这里面的代码在类加载时执行，且只执行一次
}
```

> 💡 **提示：** 类比：静态代码块就像"新店开业典礼"——只在开店（类加载）那天举行一次，之后无论多少顾客（创建多少对象）进来，都不会再举行。而实例代码块和构造器则像"每接待一位顾客的迎宾流程"，每次都要做。

## 二、核心概念与原理

### 1. 执行时机

- 静态代码块在**类加载阶段**执行，**且只执行一次**。
- 当第一次主动使用类时（如创建对象、调用静态方法、访问静态变量、通过反射加载类等），类会被加载，静态代码块随之执行。

### 2. 作用

- 初始化复杂的静态变量（无法用一行赋值完成的初始化）。
- 加载只需要做一次的资源：读取配置文件、加载驱动（如 JDBC 的 `Class.forName`）、初始化日志、注册服务等。

```java
public class AppConfig {
    static Properties props = new Properties();

    // 静态代码块：类加载时读取配置文件，只读一次
    static {
        try (InputStream in = AppConfig.class
                .getResourceAsStream("config.properties")) {
            if (in != null) {
                props.load(in);
            }
        } catch (IOException e) {
            throw new RuntimeException("加载配置文件失败", e);
        }
    }

    public static String get(String key) {
        return props.getProperty(key);
    }
}
```

### 3. 多个静态代码块的执行顺序

如果类中有多个静态代码块，以及静态变量的显式赋值，它们会**按在源码中书写的先后顺序**依次执行。

```java
public class OrderDemo {
    static int a = init("第 1 步：静态变量 a 赋值");

    static {
        System.out.println("第 2 步：第一个静态代码块");
    }

    static int b = init("第 3 步：静态变量 b 赋值");

    static {
        System.out.println("第 4 步：第二个静态代码块");
    }

    static int init(String msg) {
        System.out.println(msg);
        return 1;
    }
}
```

## 三、重点：完整的初始化顺序（含继承）

这是面试高频考点。创建子类对象时，整个初始化顺序如下：

```text
┌─────────────────────────────────────────────────────────┐
│                  类加载阶段（仅一次）                       │
│   1. 父类静态变量赋默认值 → 显式赋值 + 父类静态代码块        │
│   2. 子类静态变量赋默认值 → 显式赋值 + 子类静态代码块        │
├─────────────────────────────────────────────────────────┤
│                  对象创建阶段（每次 new 都执行）             │
│   3. 父类实例变量赋默认值                                   │
│   4. 父类实例代码块（构造代码块）                           │
│   5. 父类构造器                                            │
│   6. 子类实例变量赋默认值                                   │
│   7. 子类实例代码块（构造代码块）                           │
│   8. 子类构造器                                            │
└─────────────────────────────────────────────────────────┘
```

简化口诀：**父静 → 子静 → 父实例块 → 父构造 → 子实例块 → 子构造**。

### 完整可运行示例

```java
class Father {
    static {
        System.out.println("1. 父类静态代码块");
    }

    {
        System.out.println("3. 父类实例代码块");
    }

    Father() {
        System.out.println("4. 父类构造器");
    }
}

class Son extends Father {
    static {
        System.out.println("2. 子类静态代码块");
    }

    {
        System.out.println("5. 子类实例代码块");
    }

    Son() {
        System.out.println("6. 子类构造器");
    }
}

public class InitOrderDemo {
    public static void main(String[] args) {
        System.out.println("---- 第一次 new Son() ----");
        new Son();
        System.out.println("---- 第二次 new Son() ----");
        new Son();
    }
}
```

运行结果（注意静态代码块只执行一次）：

```text
---- 第一次 new Son() ----
1. 父类静态代码块
2. 子类静态代码块
3. 父类实例代码块
4. 父类构造器
5. 子类实例代码块
6. 子类构造器
---- 第二次 new Son() ----
3. 父类实例代码块
4. 父类构造器
5. 子类实例代码块
6. 子类构造器
```

## 四、静态代码块 vs 实例代码块 vs 构造器

| 对比维度 | 静态代码块 `static {}` | 实例代码块（构造代码块）`{}` | 构造器 `类名()` |
|---------|----------------------|---------------------------|----------------|
| 修饰 | 用 `static` 修饰 | 无修饰 | 与类同名的方法 |
| 执行时机 | 类加载时 | 每次 `new` 对象时，先于构造器 | 每次 `new` 对象时 |
| 执行次数 | **只执行一次** | 每创建一个对象执行一次 | 每创建一个对象执行一次 |
| 用途 | 初始化静态变量、加载一次性的资源 | 初始化实例变量的通用逻辑 | 初始化对象 |
| 能否访问实例成员 | **不能**（没有对象） | 能 | 能 |
| 能否使用 `this` | **不能** | 能 | 能 |
| 执行顺序（继承） | 最先（父静→子静） | 中间（父实例块→子实例块） | 最后（父构造→子构造） |

## 五、注意事项与陷阱

> ⚠️ **注意：**
> 1. 静态代码块**只能访问静态成员**，不能访问实例变量/实例方法，不能使用 `this`。
> 2. 静态代码块中抛出的异常若不被捕获，会包装为 `ExceptionInInitializerError`，并导致类初始化失败（此后再用该类会抛 `NoClassDefFoundError`）。
> 3. 静态代码块**只执行一次**，不要在里面放依赖每次创建对象的逻辑。
> 4. 多个静态代码块和静态变量显式赋值，严格按**书写顺序**执行。

```java
public class TrapDemo {
    static int x = 5;

    static {
        // ❌ 错误写法：如果前面没给 y 赋值，这里读到的是默认值 0
        System.out.println("x = " + x); // 5
    }

    static int y = getValue();

    static int getValue() {
        // ✅ 静态代码块可以调用静态方法
        return x * 2;
    }
}
```

## 六、应用场景

1. **加载配置文件**：应用启动时读取 properties、yaml 配置。
2. **加载驱动**：如 JDBC 注册驱动、日志框架初始化。
3. **复杂静态变量初始化**：例如构建一个静态查表用的 Map。
4. **一次性环境准备**：连接池预热、缓存预热。

```java
public class JdbcUtil {
    static {
        try {
            // 类加载时注册 JDBC 驱动，只注册一次
            Class.forName("com.mysql.cj.jdbc.Driver");
        } catch (ClassNotFoundException e) {
            throw new RuntimeException("驱动加载失败", e);
        }
    }
}
```

## 七、面试常见问题

### Q1：静态代码块什么时候执行？执行几次？

在类加载阶段执行，且**只执行一次**。无论创建多少个对象，或者访问多少次静态成员，静态代码块都只会在类第一次被加载时执行。

### Q2：构造代码块、静态代码块、构造器的执行顺序？

对于单个对象创建：**静态代码块 → 实例代码块 → 构造器**（静态代码块只在第一次类加载时执行）。在有继承的情况下：父静 → 子静 → 父实例块 → 父构造 → 子实例块 → 子构造。

### Q3：静态代码块中能使用 this 吗？能访问实例变量吗？

都不能。静态代码块在类加载时执行，此时还没有任何对象，因此不能使用 `this`，也不能直接访问实例成员，只能访问静态成员。

### Q4：静态代码块和静态变量显式赋值的执行顺序？

按它们在源码中**书写的先后顺序**执行。即静态变量赋值和静态代码块是从上到下依次执行的。

## 八、总结

- 静态代码块用 `static { }` 定义，在**类加载时执行且只执行一次**。
- 主要用途：初始化静态变量、加载一次性资源（配置文件、驱动、缓存预热）。
- 完整初始化顺序口诀：**父静 → 子静 → 父实例块 → 父构造 → 子实例块 → 子构造**。
- 静态代码块只能访问静态成员，不能使用 `this`；多个静态代码块按书写顺序执行。
- 与实例代码块、构造器相比，它的执行时机最早、次数最少（一次）。
