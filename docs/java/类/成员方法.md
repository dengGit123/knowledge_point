# 成员方法

> 官方文档：[Java Language Specification: Methods](https://docs.oracle.com/javase/specs/jls/se17/html/jls-8.html#jls-8.4) | [菜鸟教程：Java 方法](https://www.runoob.com/java/java-methods.html)

## 一、概述

方法（Method）是一段用来完成特定功能的代码块，它描述了对象「能做什么」。如果说成员变量是对象的「属性」，那么成员方法就是对象的「行为/技能」。

把方法理解为一个「加工机器」：你把原材料（参数）投进去，机器按既定流程（方法体）加工，最后产出成品（返回值）。这样做的好处是：一次定义，反复调用，避免代码重复。

```java
public class Calculator {
    // 成员方法：两个整数相加
    public int add(int a, int b) {
        return a + b;
    }
}
```

## 二、方法的完整定义语法

### 2.1 语法格式

```
修饰符  返回值类型  方法名(参数列表) [throws 异常] {
    方法体;
    return 返回值;          // void 方法可省略 return
}
```

完整示例：

```java
public class MethodSyntax {
    //     ①          ②        ③        ④
    public static   int     add      (int a, int b) {
        // ⑤ 方法体
        int sum = a + b;
        // ⑥ 返回值
        return sum;
    }
}
```

| 组成部分 | 说明 | 示例 |
| --- | --- | --- |
| ① 修饰符 | 访问权限（public/protected/private/默认）和特征（static/final/abstract 等） | `public static` |
| ② 返回值类型 | 方法产出结果的数据类型；无返回值用 `void` | `int` / `void` |
| ③ 方法名 | 标识符，遵循小驼峰命名（如 `getName`） | `add` |
| ④ 参数列表 | 形参，可以为空；多个用逗号分隔 | `(int a, int b)` |
| ⑤ 方法体 | 具体的执行逻辑 | `int sum = a + b;` |
| ⑥ return | 返回结果并结束方法；void 方法可写 `return;` 或省略 | `return sum;` |

> 💡 **提示：** 方法名推荐使用「动词」开头，如 `get`、`set`、`is`、`calculate`、`print`，符合"做什么"的语义。

## 三、方法的调用

### 3.1 三种调用方式

```java
public class Calculator {
    // 1. 实例方法
    public int add(int a, int b) {
        return a + b;
    }

    // 2. 静态方法
    public static int multiply(int a, int b) {
        return a * b;
    }

    // 同类中调用其他方法
    public void demo() {
        int r1 = add(1, 2);                      // ① 同类中直接调用（实例方法）
        int r2 = multiply(3, 4);                 // ② 同类中直接调用（静态方法）
        System.out.println(r1 + ", " + r2);
    }
}

class CallDemo {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        int r1 = calc.add(1, 2);                 // ③ 跨类通过对象调用实例方法

        int r2 = Calculator.multiply(3, 4);      // ④ 静态方法通过"类名.方法名"调用
        // 也能通过对象调用，但不推荐
    }
}
```

| 调用场景 | 调用方式 |
| --- | --- |
| 同类中调用实例方法 | 直接 `方法名(参数)` |
| 同类中调用静态方法 | 直接 `方法名(参数)` |
| 跨类调用实例方法 | `对象.方法名(参数)` |
| 调用静态方法（推荐） | `类名.方法名(参数)` |

### 3.2 形参与实参

| 概念 | 说明 |
| --- | --- |
| 形式参数（形参） | 方法**定义**时括号里的变量，相当于「占位符」，用于接收数据 |
| 实际参数（实参） | 方法**调用**时传入的具体值，会被赋给形参 |

```java
public int add(int a, int b) { return a + b; }  // a, b 是形参

// 调用
add(3, 5);                                      // 3, 5 是实参
```

## 四、重点：Java 是值传递

这是 Java 方法调用最关键、最容易踩坑的特性。**Java 中所有方法的参数传递都是「值传递」**——传递的是实参的「副本」，而不是实参本身。

但「副本」的内容因数据类型而异，理解时要分两种情况：

### 4.1 基本类型——传递值的副本

基本类型传递的是数据值的一份拷贝，方法内修改形参**不影响**原变量。

```java
public class PrimitivePass {
    public static void change(int x) {
        x = 100;                                // 只改了副本
    }

    public static void main(String[] args) {
        int a = 10;
        change(a);
        System.out.println(a);                   // 10（不变）
    }
}
```

内存示意：

```
  调用前：           调用 change(a) 时（复制一份）：
 ┌───────┐         ┌───────┐     ┌───────┐
 │  a=10 │  ───►   │  a=10 │     │  x=10 │   (x 是 a 的副本)
 └───────┘         └───────┘     └───┬───┘
                                     │ 方法内 x=100
                                     ▼
                                 ┌───────┐
                                 │ x=100 │   副本被改，原 a 不变
                                 └───────┘
```

### 4.2 引用类型——传递引用（地址）的副本

引用类型（对象、数组）传递的是「引用（地址）的副本」。形参和实参指向**同一个对象**，所以通过形参修改对象**内部内容**会影响原对象；但若在方法内**重新赋值**形参（让它指向新对象），则不影响原变量。

#### 情况 A：通过引用修改对象内部 —— 影响原对象

```java
public class RefPassA {
    static class Person {
        String name;
        Person(String name) { this.name = name; }
    }

    public static void changeName(Person p) {
        p.name = "李四";                         // 通过引用修改对象内部
    }

    public static void main(String[] args) {
        Person person = new Person("张三");
        changeName(person);
        System.out.println(person.name);         // 李四（被改变了！）
    }
}
```

#### 情况 B：重新赋值引用 —— 不影响原变量

```java
public class RefPassB {
    static class Person {
        String name;
        Person(String name) { this.name = name; }
    }

    public static void reassign(Person p) {
        p = new Person("王五");                  // 只让"副本引用"指向新对象
    }

    public static void main(String[] args) {
        Person person = new Person("张三");
        reassign(person);
        System.out.println(person.name);         // 张三（不变）
    }
}
```

内存示意：

```
  调用 reassign(person)：
 ┌──────────┐         ┌──────────────┐
 │ person ──┼──────►  │ Person("张三")│  ← 实参 person 和形参 p 都指向它
 └──────────┘         └──────────────┘
        ▲                      ▲
        │                      │
   person                  p (副本)
                            │
   方法内 p = new Person("王五")
                            ▼
                      ┌──────────────┐
                      │ Person("王五")│  ← 副本 p 指向新对象，原 person 不变
                      └──────────────┘
```

> ⚠️ **注意：** 很多人误以为「引用类型是引用传递」，这是错误的。Java 是**值传递**：对于引用类型，传递的是「引用的值（地址）的副本」。引用传递（C++的 &）可以让方法内重新指向影响外部，而 Java 做不到。

## 五、方法重载（Overload）

### 5.1 什么是重载

方法重载（Overloading）是指**在同一个类中**，方法名相同、但**参数列表不同**的多个方法。它让同一个"动作"可以适配不同的参数形式，类似于「一个名字多种用法」。

```java
public class OverloadDemo {
    // 三个同名方法，参数列表不同 → 重载
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

### 5.2 重载的判定规则

| 判定要素 | 是否影响重载 |
| --- | --- |
| 方法名 | 必须**相同** |
| 参数个数 | 不同则可重载 |
| 参数类型 | 不同则可重载 |
| 参数顺序（类型不同时） | 不同则可重载 |
| **返回值类型** | **无关**（仅返回值不同不构成重载，会编译错误） |
| 参数名 | **无关**（`add(int a)` 和 `add(int b)` 是同一个方法） |
| 访问修饰符 | 无关 |

> ⚠️ **注意：** 只靠「返回值类型不同」是无法重载的。例如同时定义 `int add(int a)` 和 `double add(int a)`，编译器无法根据调用方区分要用哪一个，会报错。

### 5.3 重载（overload）vs 重写（override）

这是面试高频对比，务必分清：

| 对比维度 | 重载 Overload | 重写 Override |
| --- | --- | --- |
| 发生位置 | 同一个类中 | 父子类之间 |
| 方法名 | 相同 | 相同 |
| 参数列表 | **必须不同** | **必须相同** |
| 返回值 | 无关 | 相同（或子类型，协变返回） |
| 访问修饰符 | 无关 | 子类不能比父类更严格 |
| 抛出异常 | 无关 | 子类不能抛出更宽的受检异常 |
| 关系 | 编译时多态（静态分派） | 运行时多态（动态绑定） |
| 触发关键字 | 无 | 通常加 `@Override` |

## 六、可变参数（Varargs）

### 6.1 语法与本质

可变参数用 `类型... 参数名` 表示，允许调用时传入任意数量（包括 0 个）的参数。**可变参数的本质是数组**，在方法体内当作数组使用。

```java
public class VarargsDemo {
    // 可变参数：本质是 int[]
    public static int sum(int... nums) {
        int total = 0;
        for (int n : nums) {                     // 当数组遍历
            total += n;
        }
        return total;
    }

    public static void main(String[] args) {
        System.out.println(sum());               // 0
        System.out.println(sum(1, 2));           // 3
        System.out.println(sum(1, 2, 3, 4, 5));  // 15
        System.out.println(sum(new int[]{1,2,3}));// 6（也可直接传数组）
    }
}
```

### 6.2 可变参数的规则

| 规则 | 说明 |
| --- | --- |
| 一个方法最多一个可变参数 | 不能同时有 `int... a` 和 `String... b` |
| 必须放在参数列表最后 | `void f(int a, int... nums)` ✅；`void f(int... nums, int a)` ❌ |
| 本质是数组 | 方法体内可当数组用，也可传数组实参 |
| 与重载配合 | 可变参数的方法与固定参数的方法重载时，优先匹配固定参数方法 |

> ⚠️ **注意：** 可变参数是 Java 5 引入的特性。JDK 中常见于 `System.out.printf`、`String.format`、`List.of` 等。

## 七、return 关键字

`return` 有两个作用：**返回结果**和**结束方法**。

| 场景 | 用法 |
| --- | --- |
| 有返回值的方法 | `return 值;` 返回结果并结束方法 |
| void 方法 | `return;` 仅用于提前结束方法（可省略） |

```java
public class ReturnDemo {
    // 有返回值
    public int max(int a, int b) {
        if (a > b) {
            return a;                            // 返回并结束
        }
        return b;
    }

    // void 方法中用 return 提前结束
    public void printPositive(int n) {
        if (n <= 0) {
            System.out.println("非正数，提前返回");
            return;                              // 提前结束方法
        }
        System.out.println("正数: " + n);
    }
}
```

> ⚠️ **注意：** 有返回值的方法，所有可能的执行路径都必须 return 一个值，否则编译报错。

## 八、方法的递归

递归（Recursion）是指方法**直接或间接调用自身**。它适合解决「大问题可以拆解为同类型的更小问题」的场景，核心要素是：**递归出口（终止条件）+ 递归体（向出口逼近）**。

### 8.1 递归执行流程（阶乘）

```java
public class RecursionDemo {
    // 计算 n 的阶乘 n!
    public static int factorial(int n) {
        if (n == 1) {                            // ① 递归出口（终止条件）
            return 1;
        }
        return n * factorial(n - 1);             // ② 递归体（问题规模缩小）
    }

    public static void main(String[] args) {
        System.out.println(factorial(5));        // 120
    }
}
```

递归调用栈（以 `factorial(5)` 为例）：

```
factorial(5)
= 5 * factorial(4)
= 5 * 4 * factorial(3)
= 5 * 4 * 3 * factorial(2)
= 5 * 4 * 3 * 2 * factorial(1)
= 5 * 4 * 3 * 2 * 1          ← 触达出口，开始回溯
= 120
```

### 8.2 斐波那契数列

```java
// 斐波那契：1, 1, 2, 3, 5, 8, 13, ...
// 规律：fib(n) = fib(n-1) + fib(n-2)
public static int fib(int n) {
    if (n == 1 || n == 2) {                      // 出口
        return 1;
    }
    return fib(n - 1) + fib(n - 2);              // 递归体
}
```

### 8.3 递归注意事项

> ⚠️ **注意：**
> 1. **必须有终止条件**，否则会无限递归，导致 `StackOverflowError`（栈溢出）。
> 2. 递归层次不宜过深，每调用一层都消耗栈帧，默认栈深度有限。
> 3. 存在重复计算的递归（如朴素斐波那契）效率低，可用「记忆化（缓存）」或改为循环优化。

## 九、注意事项与陷阱

### 9.1 方法不能嵌套定义

Java 中方法不能定义在另一个方法内部（Java 不支持局部方法，8 之前不支持方法内部的方法定义）。

```java
public class Wrong {
    public void outer() {
        // public void inner() { }    // ❌ 方法不能嵌套定义
    }
}
```

### 9.2 空指针：调用前判断对象是否为 null

```java
String s = null;
// s.length();                         // ❌ NullPointerException
if (s != null) {
    System.out.println(s.length());
}
```

### 9.3 void 方法也能用 return 提前结束

```java
public void process(int n) {
    if (n < 0) return;                          // 提前结束，不带值
    // 正常逻辑...
}
```

## 十、面试常见问题

### Q1：Java 是值传递还是引用传递？

Java **只有值传递**。对于基本类型，传递的是数据值的副本；对于引用类型，传递的是引用（地址）的副本。所以基本类型修改形参不影响原变量；引用类型通过形参修改对象内部会影响原对象，但重新赋值形参（指向新对象）不影响原变量。Java 不支持 C++ 那样的引用传递。

### Q2：方法重载的判定条件是什么？

方法名相同，且参数列表不同（参数个数、类型、顺序至少一个不同）。返回值类型、参数名、访问修饰符都与重载无关。仅返回值不同不能构成重载。

### Q3：重载和重写有什么区别？

重载（Overload）发生在**同一个类中**，方法名相同但参数列表不同，是编译时的静态分派；重写（Override）发生在**父子类之间**，方法签名完全相同（参数列表必须相同），是运行时的动态绑定，体现多态。

### Q4：可变参数的本质是什么？有什么限制？

可变参数的本质是数组。限制：一个方法只能有一个可变参数，且必须放在参数列表的最后。调用时可以传 0 到多个值，也可以直接传一个数组。

### Q5：void 方法中能用 return 吗？

可以，但只能用 `return;`（不带值），用于提前结束方法。带值的 `return 值;` 只能用在有返回值类型的方法中。

### Q6：递归必须注意什么？

必须有**终止条件（递归出口）**，否则会无限递归导致 `StackOverflowError`。同时要注意递归深度和重复计算问题：过深的递归消耗大量栈空间，重复计算的递归可用记忆化或改为迭代优化。

### Q7：基本类型和引用类型作为参数，方法内修改对外部有什么影响？

基本类型：传值的副本，方法内修改不影响外部原变量。引用类型：传引用（地址）的副本，方法内通过该引用修改对象内部会影响外部（因为指向同一个对象），但若在方法内让形参指向新对象（重新赋值引用），则不影响外部原变量。

## 十一、总结

成员方法是对象「行为」的封装，是 Java 程序逻辑的基本单元。掌握它的要点：

1. **语法结构**：修饰符 + 返回值类型 + 方法名 + 参数列表 + 方法体；
2. **调用规则**：同类直接调、跨类用对象、静态用类名；
3. **值传递**：Java 永远是值传递，区分基本类型和引用类型的副本内容是关键；
4. **重载 vs 重写**：重载看参数列表，发生在同类；重写看父子类，方法签名一致；
5. **可变参数**：本质是数组，一个方法只能一个且必须放最后；
6. **递归**：必须设终止条件，注意栈深度与效率。
