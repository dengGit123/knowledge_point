# 访问修饰符（Access Modifiers）

> 官方文档：[Controlling Access to Members of a Class (Oracle Java Tutorials)](https://docs.oracle.com/javase/tutorial/java/javaOO/accesscontrol.html) ｜ 菜鸟教程：[Java 修饰符](https://www.runoob.com/java/java-modifier-types.html)

## 一、概述

**访问修饰符（Access Modifier）** 是 Java 用来控制类、成员变量、方法、构造方法**可见范围（访问权限）** 的关键字。简单说，它决定了"谁可以访问这个东西"。

为什么需要访问控制？想象一栋大楼：
- **大厅**（public）：所有人都能进。
- **员工办公区**（protected）：本公司员工、以及有合作关系的人能进。
- **内部走廊**（默认/包级）：只有同一栋楼里的人能进。
- **老板办公室**（private）：只有老板自己能进。

访问修饰符就是 Java 实现**封装（Encapsulation）** 的核心工具，它能有效保护数据安全、隐藏实现细节、降低模块间的耦合。

## 二、四种访问级别

Java 提供了**四种**访问级别，权限从大到小依次是：

| 访问级别 | 关键字 | 说明 |
|---------|--------|------|
| **公共** | `public` | 所有类都能访问 |
| **受保护** | `protected` | 同包 + 不同包的子类可访问 |
| **默认（包级私有）** | （不写修饰符） | 只有**同一个包**内的类可访问 |
| **私有** | `private` | 只有**本类**内部可访问 |

> ⚠️ **注意：** "默认"访问级别**没有关键字**，是"什么都不写"时的访问级别，也叫 **package-private（包级私有）**。很多初学者误以为"不写修饰符=任意可访问"，这是**错误**的。

## 三、访问范围对比表（★★★ 核心）

这是访问修饰符最重要的表格，必须牢记：

| 访问级别 | 本类 | 同一个包（非子类） | 同包子类 | 不同包子类 | 不同包（非子类） |
|---------|------|-----------------|---------|-----------|---------------|
| **public** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **protected** | ✅ | ✅ | ✅ | ✅ ⚠️ | ❌ |
| **默认（default）** | ✅ | ✅ | ✅ | ❌ | ❌ |
| **private** | ✅ | ❌ | ❌ | ❌ | ❌ |

**权限大小排序**：`public` > `protected` > `默认` > `private`

记忆要点：
- **private** 只在自己家（本类）能进。
- **默认** 在自己小区（同一个包）能进。
- **protected** 在自己小区能进，**外加** 不在小区但和自己有血缘关系的人（不同包子类）也能进（有特殊限制，见下文）。
- **public** 全世界都能进。

## 四、protected 的特殊规则（难点）

`protected` 是最容易让人迷惑的访问级别。它包含两层含义：

1. **同包内**：无论是否是子类，都可以访问（和默认访问级别一样）。
2. **不同包**：**只有子类**能访问，而且**只能通过子类自己的引用**访问，**不能通过父类引用**访问。

```java
// ============ 包 a ============
package a;
public class Father {
    protected void hi() {                       // protected 方法
        System.out.println("Father hi");
    }
}

// ============ 包 b（不同包） ============
package b;
import a.Father;

public class Son extends Father {
    public void test() {
        // ✅ 通过子类自己的引用（this）访问继承来的 protected 方法
        Son son = new Son();
        son.hi();                               // 合法

        // ✅ 直接调用（相当于 this.hi()）
        hi();                                   // 合法

        // ❌ 不能通过父类引用访问（不同包下 protected 的特殊限制）
        // Father f = new Father();
        // f.hi();                             // 编译错误！

        // ❌ 不能通过另一个无关子类的引用访问
        // 假设 OtherSon 也继承 Father，但不与 Son 同包：
        // OtherSon other = new OtherSon();
        // other.hi();                        // 编译错误（类型不匹配的引用）
    }
}
```

> 💡 **提示：** 不同包子类访问 protected 成员时，JVM 要求访问的**引用类型必须是当前子类（或其子类）本身**，这是为了防止"借子类之名，访问父类的 protected 成员"，保持封装的严密性。

## 五、不同元素的选择建议

### 5.1 成员变量（字段）

| 建议 | 原因 |
|------|------|
| **默认全部 private** | 封装原则，禁止外部直接修改，通过 getter/setter 暴露 |

```java
public class Student {
    private String name;        // ✅ 属性私有
    private int age;

    public String getName() { return name; }       // public 方法对外提供访问
    public void setName(String name) { this.name = name; }
}
```

### 5.2 方法

| 建议 | 原因 |
|------|------|
| 对外功能用 **public** | 方便调用 |
| 仅供本类内部用用 **private** | 隐藏实现细节（工具方法） |
| 仅供子类重写/扩展用 **protected** | 模板方法模式中常见 |

### 5.3 构造方法

| 建议 | 适用场景 |
|------|---------|
| **public** | 普通类，允许任意地方 new |
| **默认** | 只允许同包内 new |
| **protected** | 只允许同包或子类（如抽象类常用） |
| **private** | 单例模式、工具类（禁止 new，强制用静态方法） |

```java
// 单例模式：构造器私有，禁止外部 new
public class Singleton {
    private static final Singleton INSTANCE = new Singleton();
    private Singleton() { }                 // private 构造器
    public static Singleton getInstance() { return INSTANCE; }
}

// 工具类：构造器私有，只用静态方法
public class MathUtil {
    private MathUtil() { }                  // 禁止实例化
    public static int add(int a, int b) { return a + b; }
}
```

### 5.4 类（顶层类）

> ⚠️ **注意：** **顶层类（外部类）只能用 `public` 或默认（包级）** 两种访问级别，**不能用 `private` 或 `protected`**。因为顶层类本身就是"包"的成员，没有比它更外层的"本类"概念。

```java
public class OuterA { }    // ✅ public，任意包可见
class OuterB { }           // ✅ 默认，同包可见
// private class OuterC { }   // ❌ 编译错误：顶层类不能用 private
// protected class OuterD { } // ❌ 编译错误：顶层类不能用 protected
```

但 **内部类**（定义在类里面的类）可以使用所有四种访问级别，因为它被视为外部类的一个成员：

```java
public class Outer {
    public    class Inner1 { }    // ✅ 内部类可用四种修饰符
    protected class Inner2 { }
              class Inner3 { }
    private  class Inner4 { }
}
```

## 六、封装原则

访问修饰符是实现封装的关键。封装的核心原则：

> **属性私有（private），对外提供公共（public）的访问方法（getter/setter）。**

```
┌─────────────── 外部代码 ───────────────┐
│                                       │
│    student.setAge(-5)   ──┐           │
│                           ▼           │
│  ┌──────────── Student 类 ──────────┐ │
│  │  private int age;                │ │
│  │                                  │ │
│  │  public void setAge(int age) {   │ │
│  │      if (age < 0) {              │ │
│  │          throw new ...;          │ │  ← 校验逻辑在这里把关
│  │      }                           │ │
│  │      this.age = age;             │ │
│  │  }                               │ │
│  └──────────────────────────────────┘ │
└───────────────────────────────────────┘
```

封装的好处：
1. **安全性**：外部不能绕过校验直接修改数据。
2. **可维护性**：内部实现可以随时修改，只要方法签名不变，外部无感知。
3. **降低耦合**：使用者不需要知道内部细节，只需调用公开方法。

## 七、常见误区辨析

### 误区 1：不写修饰符 = 谁都能访问

❌ 错误。不写修饰符是**默认（包级私有）** 访问级别，**只有同包类**能访问，其他包即使不是子类也访问不了。

```java
// 包 a
package a;
class A { void hi() {} }        // 默认访问级别，仅包 a 内可访问

// 包 b
package b;
// new A().hi();               // ❌ 不同包，无法访问
```

### 误区 2：protected 只能被同包子类访问

❌ 错误。`protected` **不仅同包可访问，不同包的子类也能访问**（但有"必须通过子类引用"的限制，见第四节）。它比默认访问级别更宽，多了"不同包子类"这一项。

### 误区 3：protected 在不同包下随便用

❌ 错误。不同包下，protected 成员**只能通过子类自身的引用访问**，不能通过父类引用或无关类引用访问。这是为了防止封装被绕过。

### 误区 4：顶层类可以用 private

❌ 错误。顶层类（直接定义在 .java 文件中的类）**只能用 public 或默认**。private/protected 只能用于类的成员（成员变量、方法、内部类等）。

### 误区 5：private 成员子类完全访问不到

部分正确，但不完整。子类**不能直接访问** private 成员（编译报错），但 private 成员在子类对象的内存中**是存在的**，子类可以通过父类提供的 `public`/`protected` 方法**间接访问**。

```java
class Father {
    private int secret = 100;
    public int getSecret() { return secret; }   // 暴露 public 方法
}
class Son extends Father {
    public void test() {
        // System.out.println(secret);    // ❌ private 不可直接访问
        System.out.println(getSecret());  // ✅ 通过 public 方法间接访问 → 100
    }
}
```

## 八、与相关概念对比

### 8.1 访问修饰符 vs 非访问修饰符

Java 的修饰符分为两大类，不要混淆：

| 类别 | 关键字 | 作用 |
|------|--------|------|
| **访问修饰符** | `public` `protected` `默认` `private` | 控制**可见范围**（谁能访问） |
| **非访问修饰符** | `static` `final` `abstract` `synchronized` `native` 等 | 控制**其他特性**（静态、常量、抽象、同步等） |

### 8.2 同一个成员可同时使用多个修饰符

```java
public static final int MAX = 100;          // public + static + final
protected abstract void draw();             // protected + abstract
private synchronized void update() { }      // private + synchronized
```

> 💡 **提示：** 修饰符之间的顺序不影响语义，习惯写法是：**访问修饰符在前，其他修饰符在后**（如 `public static final`）。

### 8.3 各修饰符可用的元素对照表

| 修饰符 | 顶层类 | 成员变量 | 方法 | 构造方法 | 内部类 |
|--------|:------:|:------:|:----:|:------:|:-----:|
| `public` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `protected` | ❌ | ✅ | ✅ | ✅ | ✅ |
| `默认` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `private` | ❌ | ✅ | ✅ | ✅ | ✅ |

## 九、完整示例：封装一个银行账户类

下面用一个贴近实际的例子，展示访问修饰符如何配合实现良好的封装：

```java
public class BankAccount {

    // 1. 私有属性：账户余额，外部绝对不能直接修改
    private double balance;
    private String password;

    // 2. public 构造方法
    public BankAccount(String password, double initialBalance) {
        this.password = password;
        if (initialBalance >= 0) {
            this.balance = initialBalance;
        }
    }

    // 3. public 方法：对外提供的"取钱"功能（带校验）
    public boolean withdraw(double amount, String pwd) {
        if (!checkPassword(pwd)) {
            System.out.println("密码错误");
            return false;
        }
        if (amount <= 0 || amount > balance) {
            System.out.println("金额非法或余额不足");
            return false;
        }
        balance -= amount;
        return true;
    }

    // 4. public 方法：对外提供余额查询（只读，不暴露修改入口）
    public double getBalance(String pwd) {
        if (checkPassword(pwd)) {
            return balance;
        }
        return -1;            // 密码错误返回 -1
    }

    // 5. private 方法：内部密码校验逻辑，外部无需知晓
    private boolean checkPassword(String pwd) {
        return this.password.equals(pwd);
    }
}
```

```java
public class Test {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount("123456", 1000);
        // acc.balance = 99999999;         // ❌ 编译错误：private 不可访问
        acc.withdraw(500, "123456");       // ✅ 通过 public 方法操作
        System.out.println(acc.getBalance("123456"));   // 500.0
    }
}
```

## 十、面试常见问题

### Q1：Java 有几种访问修饰符？它们的访问范围分别是？

四种：`public`（全部可访问）、`protected`（同包 + 不同包子类）、默认/包级私有（仅同包）、`private`（仅本类）。权限从大到小：public > protected > 默认 > private。

### Q2：默认（default）访问级别是什么？是不是谁都能访问？

默认访问级别是**不写任何修饰符**时的访问级别，也叫 package-private（包级私有）。它**只有同一个包内的类**可以访问，**不是**谁都能访问。这是一个非常常见的考点误区。

### Q3：protected 和默认访问级别有什么区别？

默认只能同包访问；protected 在同包访问的基础上，**额外允许不同包的子类访问**。但不同包子类访问 protected 时有特殊限制：**必须通过子类自身的引用**，不能通过父类引用或其他类型引用访问。

### Q4：顶层类可以用 private 或 protected 修饰吗？

不能。顶层类（直接定义在 .java 文件中的类）只能用 `public` 或默认两种访问级别。`private` 和 `protected` 只能用于类的**成员**（成员变量、方法、构造方法、内部类）。

### Q5：private 成员能被子类继承吗？子类能访问吗？

private 成员在子类对象的内存中**存在**，但子类**不能直接访问**（编译报错）。严格地说，private 成员**不算被子类继承**（不可见）。子类只能通过父类提供的 public/protected 方法**间接访问**。这也是封装的体现——子类也无法绕过父类的设计直接操作其私有数据。

### Q6：如何用访问修饰符实现单例模式？

把构造方法设为 `private`，禁止外部 `new`；在类内部自己持有一个唯一实例，通过 `public static` 方法对外提供访问入口。这样从访问控制层面保证了"全局只有一个实例"。

```java
public class Singleton {
    private static final Singleton INSTANCE = new Singleton();
    private Singleton() { }                            // private 构造器
    public static Singleton getInstance() { return INSTANCE; }
}
```

### Q7：为什么属性一般都要设为 private？

为了封装。如果属性公开，外部可以绕过校验随意赋值（如 age = -1000），数据安全性无法保证。设为 private 后，所有访问都必须经过 getter/setter，可以在方法中加入校验、日志、懒加载等逻辑，既保证数据安全，又保留未来修改实现的灵活性（只要方法签名不变，内部怎么改都不影响外部）。

## 十一、总结

| 知识点 | 核心要点 |
|--------|---------|
| 四种级别 | public > protected > 默认 > private |
| public | 任意可访问 |
| protected | 同包 + 不同包子类（需子类引用） |
| 默认（package-private） | 仅同包 |
| private | 仅本类 |
| 顶层类 | 只能用 public 或默认，不能用 private/protected |
| 内部类 | 四种修饰符都可以用 |
| 封装原则 | 属性 private，方法 public（getter/setter） |
| 单例/工具类 | 构造方法设 private |

访问修饰符看似简单，却是 Java 封装特性的基石。牢记那张"四种访问级别 × 五种场景"的对比表，理解 protected 在不同包下的特殊限制，并坚持"属性私有、方法公共"的封装原则，你的代码就会更安全、更易维护、更专业。
