# 抽象类（Abstract Class）

> 官方文档：[Java Language Specification: Classes - Abstract Classes](https://docs.oracle.com/javase/specs/jls/se17/html/jls-8.html#jls-8.1.1.1) | [菜鸟教程：Java 抽象类](https://www.runoob.com/java/java-abstraction.html)

## 一、概述

抽象类（Abstract Class）是用 `abstract` 关键字修饰的类，它是一种「半成品」类——可以包含已实现的方法，也可以包含未实现（抽象）的方法。你可以把抽象类理解为一个**模板**或**未完工的图纸**：

- 图纸上画好了框架和部分细节（普通方法、成员变量），但有些关键部分留白（抽象方法）；
- 这些留白的部分必须由「施工队」（子类）根据图纸去完成；
- 留白没填完之前，这份图纸不能直接"盖房子"（不能实例化）。

抽象类主要用于：**复用代码 + 强制子类实现某些关键方法**，是面向对象中"继承体系"承上启下的关键。

### 1.1 基本语法

```java
// 用 abstract 修饰类
public abstract class 类名 {
    // 普通成员变量
    // 普通方法（有方法体）
    // 抽象方法（无方法体，用 abstract 修饰）
}
```

```java
// 抽象类
public abstract class Animal {
    String name;                                // 普通成员变量

    // 普通方法：所有子类共享
    public void breathe() {
        System.out.println(name + " 在呼吸");
    }

    // 抽象方法：没有方法体，强制子类实现
    public abstract void sound();
}

// 子类继承抽象类，必须实现所有抽象方法
public class Dog extends Animal {
    public Dog(String name) {
        this.name = name;
    }

    @Override
    public void sound() {                       // 必须实现
        System.out.println(name + " 汪汪叫");
    }
}
```

> 💡 **提示：** `abstract` 关键字既可以修饰类（抽象类），也可以修饰方法（抽象方法）。抽象方法必须位于抽象类中（接口里的抽象方法是另一回事）。

## 二、核心概念与规则

### 2.1 抽象类不能实例化

```java
public class AbstractDemo {
    public static void main(String[] args) {
        // ❌ 抽象类不能直接 new
        // Animal a = new Animal();

        // ✅ 通过具体子类对象，向上转型为抽象类类型（多态）
        Animal a = new Dog("旺财");
        a.sound();                              // 输出: 旺财 汪汪叫
        a.breathe();                            // 输出: 旺财 在呼吸
    }
}
```

### 2.2 抽象类有构造器

虽然抽象类不能实例化，但它**有构造器**，作用是供子类 `super()` 调用，完成父类部分（成员变量）的初始化。

```java
public abstract class Shape {
    protected String color;

    // 抽象类的构造器
    public Shape(String color) {
        this.color = color;
        System.out.println("Shape 构造器被调用，color=" + color);
    }

    public abstract double area();              // 抽象方法
}

public class Circle extends Shape {
    private double radius;

    public Circle(String color, double radius) {
        super(color);                           // 子类构造器必须调用父类构造器
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}
```

### 2.3 抽象方法的规则

| 规则 | 说明 |
| --- | --- |
| 只有声明，无方法体 | `public abstract void run();` 后面是分号，不是大括号 |
| 子类必须重写 | 除非子类本身也是抽象类 |
| 不能用 `private` 修饰 | 因为 private 方法子类访问不到，更无法重写，没有意义 |
| 不能用 `static` 修饰 | 静态方法属于类，可以被直接调用，而抽象方法无方法体无意义 |
| 不能用 `final` 修饰 | final 方法禁止重写，与"要求子类重写"矛盾 |

### 2.4 抽象类可以「混合」各种成员

抽象类并不要求一定有抽象方法（即使没有抽象方法也可以是抽象类，纯粹为了禁止实例化）；它可以同时拥有：普通成员变量、静态变量、常量、普通方法、静态方法、抽象方法、构造器、代码块等。

```java
public abstract class Vehicle {
    public static final int WHEELS_DEFAULT = 4; // 常量
    protected String brand;                     // 普通成员变量
    private static int count;                   // 静态变量

    static {                                    // 静态代码块
        count = 0;
    }

    public Vehicle(String brand) {              // 构造器
        this.brand = brand;
        count++;
    }

    public void info() {                        // 普通方法
        System.out.println("品牌: " + brand);
    }

    public abstract void drive();               // 抽象方法
}
```

## 三、abstract 不能与 final、static、private 共用的原因

这是面试高频考点，理解背后的"逻辑矛盾"比死记更重要：

| 组合 | 是否合法 | 原因 |
| --- | --- | --- |
| `abstract + final` | ❌ 非法 | `final` 禁止重写，而 `abstract` 要求子类重写，二者矛盾 |
| `abstract + static` | ❌ 非法 | `static` 方法属于类，可直接用类名调用；而抽象方法无方法体，调用无意义 |
| `abstract + private` | ❌ 非法 | `private` 对子类不可见，子类无法重写，与 abstract 要求重写矛盾 |

```java
public abstract class Wrong {
    // private abstract void a();      // ❌ 非法
    // static abstract void b();       // ❌ 非法
    // final abstract void c();        // ❌ 非法
}
```

## 四、典型应用：模板方法模式（Template Method）

抽象类最经典的应用就是**模板方法模式**：在父类（抽象类）中定义算法的「骨架」（用普通方法固定执行步骤），把可变的具体步骤延迟到子类（抽象方法）实现。

```
┌──────────────────────────────────────────────┐
│  抽象类 Game (模板)                           │
│  ┌────────────────────────────────────────┐  │
│  │  play() {            ← 模板方法(final) │  │
│  │      initialize();   ← 步骤1           │  │
│  │      startPlay();    ← 步骤2           │  │
│  │      endPlay();      ← 步骤3           │  │
│  │  }                                     │  │
│  │  abstract void initialize();           │  │
│  │  abstract void startPlay();            │  │
│  │  abstract void endPlay();              │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
            ▲ extends                ▲ extends
┌────────────────────┐    ┌────────────────────┐
│  Football          │    │  Basketball        │
│  实现三个抽象方法    │    │  实现三个抽象方法    │
└────────────────────┘    └────────────────────┘
```

### 完整示例

```java
// 抽象模板类
public abstract class Game {
    // 模板方法：定义执行步骤骨架，用 final 锁定，防止子类破坏流程
    public final void play() {
        initialize();
        startPlay();
        endPlay();
    }

    // 抽象方法：具体实现交给子类
    protected abstract void initialize();
    protected abstract void startPlay();
    protected abstract void endPlay();
}

// 具体子类：足球
public class Football extends Game {
    @Override
    protected void initialize() {
        System.out.println("足球：准备足球场地，球员入场");
    }

    @Override
    protected void startPlay() {
        System.out.println("足球：开球，比赛开始");
    }

    @Override
    protected void endPlay() {
        System.out.println("足球：吹哨，比赛结束");
    }
}

// 具体子类：篮球
public class Basketball extends Game {
    @Override
    protected void initialize() {
        System.out.println("篮球：准备篮球场地，队员热身");
    }

    @Override
    protected void startPlay() {
        System.out.println("篮球：跳球，比赛开始");
    }

    @Override
    protected void endPlay() {
        System.out.println("篮球：终场哨响，比赛结束");
    }
}

// 测试
public class TemplateDemo {
    public static void main(String[] args) {
        Game football = new Football();
        football.play();                        // 调用同样的 play()，执行不同的具体步骤

        Game basketball = new Basketball();
        basketball.play();
    }
}
```

> 💡 **提示：** 模板方法通常声明为 `final`，防止子类修改算法结构；而可变的"步骤"声明为 `abstract`，由子类填补。这就是"复用骨架 + 强制定制细节"。

## 五、抽象类 vs 接口 vs 普通类三方对比

| 对比维度 | 抽象类（abstract class） | 接口（interface） | 普通类（class） |
| --- | --- | --- | --- |
| 关键字 | `abstract class` / `extends` | `interface` / `implements` | `class` |
| 设计语义 | is-a（本质、分类） | can-do（能力、契约） | 具体实体 |
| 能否实例化 | 不能 | 不能 | 能 |
| 是否有构造器 | 有 | 没有 | 有 |
| 继承数量 | 单继承 | 多实现/多继承 | 单继承 |
| 成员变量 | 任意修饰符的变量 | 仅 `public static final` 常量 | 任意修饰符的变量 |
| 方法 | 抽象方法 + 普通方法 + 静态方法 | 抽象/默认/静态/私有方法 | 普通方法 + 静态方法 |
| 是否必须有抽象方法 | 不必须（可有可无） | Java 8 前必须有 | 不允许有抽象方法 |
| 使用场景 | 共享字段和代码、定义家族骨架 | 定义跨家族的能力契约 | 具体的业务实现 |

## 六、抽象类的意义

1. **强制子类实现关键方法**：把必须由子类定制的方法声明为抽象方法，编译器会帮你检查是否实现，避免遗漏。
2. **复用代码**：把通用逻辑（普通方法、成员变量）放在抽象类中，子类直接继承，减少重复。
3. **统一接口与部分实现**：抽象类既能定义规范（抽象方法），又能提供部分实现（普通方法），是接口和具体类之间的"中间层"。
4. **多态的基石**：通过抽象类引用指向子类对象，实现运行时多态。

```java
// 多态：抽象类引用指向子类对象
public class PolymorphismDemo {
    public static void main(String[] args) {
        Shape s1 = new Circle("红", 2.0);
        Shape s2 = new Rectangle("蓝", 3.0, 4.0);
        printArea(s1);                           // 调用 Circle 的 area()
        printArea(s2);                           // 调用 Rectangle 的 area()
    }

    // 参数类型用抽象类，可以接收任何子类对象
    public static void printArea(Shape s) {
        System.out.println(s.color + " 的面积 = " + s.area());
    }
}

abstract class Shape {
    protected String color;
    public Shape(String color) { this.color = color; }
    public abstract double area();
}

class Rectangle extends Shape {
    private double w, h;
    public Rectangle(String c, double w, double h) { super(c); this.w = w; this.h = h; }
    @Override public double area() { return w * h; }
}
```

## 七、注意事项与陷阱

### 7.1 抽象类不一定有抽象方法

```java
// 合法：没有抽象方法的抽象类（纯粹为了禁止实例化）
public abstract class Utility {
    public static void helper() {
        System.out.println("我是一个工具方法");
    }
}
// Utility u = new Utility();   // ❌ 仍然不能实例化
```

### 7.2 有抽象方法的类必须声明为抽象类

```java
// ❌ 编译错误：包含抽象方法，类必须加 abstract
// public class Bad {
//     public abstract void run();
// }
```

### 7.3 子类没实现全部抽象方法，则子类也必须是抽象类

```java
public abstract class A {
    public abstract void m1();
    public abstract void m2();
}

// 子类只实现了 m1，没实现 m2，所以必须也是抽象类
public abstract class B extends A {
    @Override public void m1() {}
    // m2 未实现，OK，因为 B 是抽象类
}
```

### 7.4 抽象类不能用 final 修饰

`final` 类不能被继承，而抽象类就是设计来被继承的，二者矛盾。

```java
// ❌ 非法
// public final abstract class X { }
```

## 八、面试常见问题

### Q1：抽象类能实例化吗？为什么还有构造器？

抽象类**不能直接实例化**（因为它含有未实现的部分）。但抽象类**有构造器**，其作用是供子类 `super()` 调用，用于初始化从父类继承下来的成员变量。在创建子类对象时，会先调用父类（抽象类）的构造器。

### Q2：抽象类和接口有什么区别？

- **设计语义**：抽象类是 is-a（本质），接口是 can-do（能力）；
- **继承关系**：类单继承抽象类，但可多实现接口；
- **成员**：抽象类可有各种成员变量和构造器；接口只能有常量（Java 9 前无私有方法）；
- **使用倾向**：抽象类适合共享代码和字段；接口适合定义跨类型的公共行为契约。
- 详细对比见本文「五、三方对比」表。

### Q3：abstract 为什么不能和 final / static / private 一起用？

- `final` 禁止重写，而 `abstract` 要求重写，矛盾；
- `static` 方法可直接用类名调用，抽象方法无方法体无意义；
- `private` 方法对子类不可见，子类无法重写，与 abstract 要求重写矛盾。

### Q4：抽象类一定要有抽象方法吗？没有抽象方法的抽象类有什么用？

不一定要有。没有抽象方法的抽象类通常是单纯为了**禁止该类被直接实例化**，强制使用者通过其子类（具体类）来使用，起到保护性设计的作用。

### Q5：子类继承抽象类后必须做什么？

子类必须实现抽象类中的**所有抽象方法**（一一重写），否则子类自身必须用 `abstract` 声明为抽象类，把"未完成"状态继续传递给下一层子类。

### Q6：抽象类如何体现"代码复用"？

抽象类把通用的成员变量、普通方法、静态方法等放在父类中，子类无需重复编写即可继承使用。子类只需关注实现自己的抽象方法（定制化部分），从而实现「骨架复用 + 细节定制」。典型体现就是模板方法模式。

## 九、总结

抽象类是 Java 面向对象体系中承上启下的关键结构，它兼具「定义规范」和「提供部分实现」的能力。掌握抽象类的要点：

1. **理解定位**：抽象类是"半成品模板"，用 `abstract` 修饰，不能实例化但可以被继承；
2. **遵守规则**：抽象方法无方法体、子类必须实现、abstract 不能与 final/static/private 共用；
3. **善用构造器**：抽象类有构造器供子类 super 调用，完成成员初始化；
4. **活用模板方法模式**：把算法骨架用 final 普通方法固定，把可变步骤用抽象方法延迟到子类；
5. **对比选择**：抽象类重在共享代码与本质关系，接口重在能力契约，二者常配合使用。
