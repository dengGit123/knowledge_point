# 静态方法（static 方法）

> 官方文档：[Java Language Specification - Methods](https://docs.oracle.com/javase/specs/jls/se17/html/jls-8.html#jls-8.4)、[菜鸟教程 Java 静态方法](https://www.runoob.com/java/java-static.html)

## 一、概述

用 `static` 关键字修饰的方法称为**静态方法**（Static Method），也叫**类方法**（Class Method）。它属于**类本身**，调用时**不需要创建对象**，直接用 `类名.方法名()` 即可。

> 💡 **提示：** 类比：实例方法像"请电工上门修电灯"，必须先有一个电工（对象）才能干活；静态方法像"打 114 查号台"，只要你拨号（用类名调用）就能用，不需要谁上门。

典型的静态方法：`Math.abs()`、`Arrays.sort()`、`Collections.emptyList()`，以及所有程序的入口 `main` 方法。

## 二、核心概念与原理

### 1. 声明与调用

```java
public class MathUtils {
    // 静态方法：用 static 修饰
    public static int square(int x) {
        return x * x;
    }

    public static void main(String[] args) {
        // ✅ 推荐：类名.方法名() 调用，无需创建对象
        int result = MathUtils.square(5);
        System.out.println(result); // 25
    }
}
```

### 2. 核心限制：静态方法中只能直接访问静态成员

这是静态方法最重要的规则。**静态方法没有 `this`**（因为它不属于任何对象），所以无法直接访问实例变量和实例方法。

```java
public class User {
    String name;          // 实例变量
    static int count;     // 静态变量

    public void sayHi() { }         // 实例方法
    public static void hello() { }  // 静态方法

    public static void staticMethod() {
        // ✅ 可以直接访问静态成员
        count++;
        hello();

        // ❌ 不能直接访问实例成员（没有 this，不知道操作哪个对象）
        // name = "Tom";
        // sayHi();

        // ✅ 但可以通过"对象引用"间接访问
        User u = new User();
        u.name = "Tom";
        u.sayHi();
    }
}
```

### 3. 为什么 main 方法必须是 public static void

```java
public static void main(String[] args)
```

| 关键字 | 原因 |
|--------|------|
| `public` | JVM 在类外部调用 main，必须是公共的才能被访问 |
| `static` | JVM 启动时还没有创建任何对象，必须能通过类名直接调用 |
| `void` | main 是程序入口，不需要返回值给任何调用者 |
| `String[] args` | 接收命令行参数 |

## 三、详细用法

### 1. 工具类设计

工具类（如 `java.lang.Math`、`java.util.Arrays`）的方法基本都是静态的，调用方便、无需实例化。

```java
public class ArrayUtils {
    // 私有构造器：禁止实例化（工具类常见做法）
    private ArrayUtils() { }

    public static int max(int[] arr) {
        int m = arr[0];
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] > m) m = arr[i];
        }
        return m;
    }

    public static void main(String[] args) {
        int[] data = {3, 7, 2, 9, 5};
        System.out.println(ArrayUtils.max(data)); // 9
    }
}
```

### 2. 静态导入（import static）

从 Java 5 开始，可以静态导入某个类的静态成员，调用时省略类名。

```java
import static java.lang.Math.*;       // 导入 Math 类的所有静态成员
import static java.lang.System.out;   // 导入 System.out

public class StaticImportDemo {
    public static void main(String[] args) {
        // 直接使用 sqrt / PI / out，不用写类名前缀
        double r = sqrt(16);            // Math.sqrt(16)
        double area = PI * r * r;       // Math.PI
        out.println(area);              // System.out.println
    }
}
```

> ⚠️ **注意：** 静态导入虽然方便，但过度使用会让代码的可读性变差（看不清某个方法来自哪个类），建议只在数学计算等少数场景谨慎使用，避免使用通配符 `*` 导入。

## 四、静态方法 vs 实例方法

| 对比维度 | 静态方法（static） | 实例方法（非 static） |
|---------|------------------|---------------------|
| 归属 | 属于类 | 属于对象 |
| 调用方式 | `类名.方法名()`（推荐） | `对象.方法名()` |
| 能否访问实例成员 | 不能直接访问（需通过对象） | 能直接访问 |
| 能否访问静态成员 | 能 | 能 |
| 能否使用 `this` / `super` | 不能 | 能 |
| 调用时是否需要对象 | 不需要 | 需要 |
| 是否支持多态（重写） | 不能被重写，只能被"隐藏" | 支持重写（多态） |
| 绑定时机 | 编译期（静态绑定） | 运行期（动态绑定） |

## 五、静态方法能被继承但不能被重写（方法隐藏 hide）

子类可以继承父类的静态方法，但如果子类定义了**相同签名**的静态方法，这叫**方法隐藏（Hiding）**，不是**重写（Override）**。两者的区别在于绑定时机：重写是运行期动态绑定，隐藏是编译期静态绑定。

```java
class Father {
    public static void hi() {
        System.out.println("Father 的静态方法");
    }
    public void hello() {
        System.out.println("Father 的实例方法");
    }
}

class Son extends Father {
    // 隐藏父类的静态方法（不是重写）
    public static void hi() {
        System.out.println("Son 的静态方法");
    }
    // 重写父类的实例方法（真正的多态）
    @Override
    public void hello() {
        System.out.println("Son 的实例方法");
    }
}

public class HideDemo {
    public static void main(String[] args) {
        Father f = new Son();
        f.hello(); // Son 的实例方法（运行期绑定到子类）
        f.hi();    // Father 的静态方法（编译期绑定到引用类型 Father）
    }
}
```

> ⚠️ **注意：** 静态方法不能用 `@Override` 注解（编译报错）。子类"覆盖"父类静态方法是隐藏行为，不参与多态。

## 六、注意事项与陷阱

> ⚠️ **注意：**
> 1. 静态方法中**不能使用 `this` 和 `super`**，因为没有当前对象。
> 2. 静态方法**只能直接访问静态成员**；访问实例成员必须先创建对象。
> 3. 静态方法**不能被重写**，只能被隐藏，不体现多态。
> 4. **不要滥用静态方法**：把所有方法都写成 static 会破坏面向对象设计，使代码难以测试和扩展。

## 七、面试常见问题

### Q1：静态方法中能使用 this 吗？

不能。`this` 代表当前对象的引用，而静态方法调用时不依赖任何对象（没有当前对象），所以在静态方法中使用 `this` 会导致编译错误。

### Q2：静态方法和实例方法的区别？

见上方对比表格。核心区别：归属（类 vs 对象）、调用方式、能否访问实例成员、能否使用 this、是否支持多态。

### Q3：静态方法能被重写吗？

不能被重写（Override），但能被隐藏（Hide）。子类定义同名同参的静态方法属于隐藏，调用时看引用类型而非实际对象类型，不体现多态。

### Q4：为什么 main 方法是静态的？

因为 JVM 启动程序时还未创建任何对象，必须能通过类名直接调用 main 方法作为入口，所以 main 必须是 `public static void`。

### Q5：静态方法中如何访问实例变量？

通过显式创建对象，再用 `对象.实例变量` 的方式访问。

## 八、总结

- 静态方法属于**类**，用 `类名.方法名()` 调用，无需创建对象。
- 静态方法**只能直接访问静态成员**，不能使用 `this`/`super`，不能访问实例成员。
- 典型用途：工具类方法、程序入口 `main`、不需要依赖对象状态的计算逻辑。
- 静态方法能被继承，但只能被**隐藏**而非重写，不参与多态。
- 配合私有构造器，可以设计出禁止实例化的工具类。
