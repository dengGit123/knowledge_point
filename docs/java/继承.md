# 继承（Inheritance）

> 官方文档：[Object 类 (Java SE 21)](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Object.html) ｜ 菜鸟教程：[Java 继承](https://www.runoob.com/java/java-inheritance.html)

## 一、概述

**继承（Inheritance）** 是面向对象三大特性之一（封装、继承、多态）。它描述的是一种 **"is-a"（是一个）** 的关系：当多个类存在相同的属性和方法时，可以把这些公共部分抽取到一个**父类（超类 / 基类，superclass）** 中，然后让**子类（派生类，subclass）** 通过继承来复用这些代码。

通俗的类比：
- "动物"是一个父类，它有"吃、睡"等通用行为。
- "狗"和"猫"是子类，它们继承了动物的所有特性，同时各自还能扩展出"汪汪叫""喵喵叫"等独有的行为。

继承的核心价值是 **代码复用** 和 **建立类之间的层次结构**。

## 二、核心概念

### 2.1 继承的语法（extends 关键字）

Java 中使用 `extends` 关键字表示继承关系：

```java
// 父类
class Animal {
    String name;
    public void eat() {
        System.out.println(name + " 在吃东西");
    }
}

// 子类使用 extends 继承父类
class Dog extends Animal {
    public void bark() {
        System.out.println(name + " 汪汪叫");   // 可以直接使用父类的 name
    }
}
```

```java
Dog d = new Dog();
d.name = "旺财";      // name 来自父类 Animal
d.eat();             // eat() 来自父类 Animal
d.bark();            // bark() 是子类自己的
```

### 2.2 继承的内存结构图

子类对象在堆内存中，**既包含子类自己的成员，也包含从父类继承来的成员**：

```
┌──────────────── Dog 对象（堆内存） ─────────────────┐
│  ┌──────────────────────────────┐                  │
│  │  来自父类 Animal 的部分        │   name           │
│  │                              │   eat()          │
│  ├──────────────────────────────┤                  │
│  │  子类 Dog 自己的部分           │   bark()         │
│  └──────────────────────────────┘                  │
└────────────────────────────────────────────────────┘
```

### 2.3 Java 是单继承

> ⚠️ **注意：** Java 中的类**只支持单继承**，即一个类只能 `extends` **一个**直接父类，不能同时继承多个类。

```java
class A {}
class B {}
// class C extends A, B {}   // ❌ 编译错误：Java 不支持多重继承
class C extends A { }        // ✅ 只能继承一个
```

为什么 Java 不支持多继承？主要是为了避免**菱形继承问题（钻石问题）**：如果两个父类有同名方法，子类调用时会产生二义性。Java 用 **接口（interface）的多实现** 来弥补这一限制（一个类可以实现多个接口）。

不过，继承是可以**多层传递**的：

```
class A {}
class B extends A {}      // B 继承 A
class C extends B {}      // C 继承 B，间接继承了 A

// 继承链：C → B → A → Object
```

## 三、子类能继承父类的哪些成员

这是继承中最容易混淆的知识点。**不是父类的所有成员都能被子类继承**。

| 父类成员类型 | 子类能否继承/访问 | 说明 |
|------------|------------------|------|
| `public` 成员变量 | ✅ 能继承 | 可直接使用 |
| 默认（包级）成员变量 | ✅ 能继承（同包） | 不同包子类不可见 |
| `protected` 成员变量 | ✅ 能继承 | 不同包子类也可访问 |
| `private` 成员变量 | ❌ 不能直接访问 | 内存中存在，但子类无法直接访问，需通过父类 public 方法 |
| `public` / `protected` 方法 | ✅ 能继承 | 可直接调用 |
| `private` 方法 | ❌ 不能继承 | 子类完全不可见 |
| **构造方法** | ❌ **不能继承** | 构造方法名必须与类名相同，子类不可能"继承"父类的构造器 |
| 静态成员（static） | ⚠️ 严格说不叫继承，是"共享访问" | 可以用 `子类名.静态成员` 访问，但不属于继承范畴 |

```java
class Father {
    public int a = 1;          // 子类可继承
    protected int b = 2;       // 子类可继承
    int c = 3;                 // 同包子类可继承
    private int d = 4;         // 子类不能直接访问

    public Father() { }        // 构造方法，不能被子类继承
}

class Son extends Father {
    public void test() {
        System.out.println(a);   // ✅ 1
        System.out.println(b);   // ✅ 2
        System.out.println(c);   // ✅ 3（同包情况下）
        // System.out.println(d);  // ❌ 编译错误：private 不可访问
    }
}
```

> 💡 **提示：** 虽然 private 成员不能被子类直接访问，但它在子类对象的内存中是**存在的**，子类可以通过父类提供的 `public` getter/setter 间接操作。

## 四、方法重写（Override）

**方法重写（Override）** 指：子类中出现了和父类**方法名、参数列表完全相同**的方法，子类用这个新方法**覆盖**了父类的旧方法。这是实现**多态**的关键。

### 4.1 基本示例

```java
class Animal {
    public void speak() {
        System.out.println("动物发出声音");
    }
}

class Dog extends Animal {
    @Override                    // 重写注解，建议始终加上
    public void speak() {        // 方法签名与父类完全相同
        System.out.println("汪汪汪");
    }
}

Animal a = new Dog();
a.speak();                       // 输出：汪汪汪（调用的是子类重写后的方法）
```

`@Override` 注解的作用是让编译器帮你检查：这个方法是不是真的正确重写了父类的方法。如果拼写错误或参数不一致，编译器会报错，避免"假重写"。

### 4.2 方法重写的规则（重点）

| 规则 | 说明 |
|------|------|
| ① 方法名 | 必须与父类**相同** |
| ② 参数列表 | 必须与父类**相同**（个数、类型、顺序） |
| ③ 返回值类型 | 必须与父类**相同**，或者是父类返回值类型的**子类型**（协变返回类型，Java 5+） |
| ④ 访问权限 | **不能比父类更严格**（父 public → 子只能 public；父默认 → 子可默认/public） |
| ⑤ 抛出异常 | 子类重写方法抛出的**受检异常**不能比父类更大（可以是父类异常的子类或不抛） |
| ⑥ 不能重写的方法 | `private`（不可见）、`static`（属于类）、`final`（禁止重写） |
| ⑦ static 方法 | 不能被重写，但可以"隐藏"（不构成多态） |

访问权限从严到宽的顺序：`private` < 默认 < `protected` < `public`

```java
class Father {
    public Number test() throws Exception { return 1; }
}

class Son extends Father {
    // ✅ 合法重写：返回值 Number 的子类 Integer、权限 public 不变、异常是 Exception 的子类或更小
    @Override
    public Integer test() throws RuntimeException { return 2; }
}
```

```java
class Father2 {
    public void run() { }
    private void secret() { }
    public static void hello() { }
}

class Son2 extends Father2 {
    // @Override
    // private void secret() { }      // ❌ 不是重写，private 不可见；这只是一个新方法
    // @Override
    // public static void hello() { } // ❌ static 方法不能被 @Override 重写
}
```

> ⚠️ **注意：** 子类重写的方法访问权限**不能更严**。例如父类是 `public`，子类重写时不能改成 `protected` 或默认，否则编译报错。

### 4.3 重写（Override）与重载（Overload）的区别

| 对比项 | 重写（Override） | 重载（Overload） |
|--------|-----------------|-----------------|
| 发生位置 | 父子类之间 | **同一个类**中 |
| 方法名 | 相同 | 相同 |
| 参数列表 | **必须相同** | **必须不同** |
| 返回值 | 相同或子类型 | 无要求 |
| 访问权限 | 不能更严 | 无要求 |
| 目的 | 子类改变父类行为 | 提供同名的多种参数版本 |

## 五、super 关键字

`super` 代表**父类的引用**（准确说是用来访问父类的成员），与 `this`（当前对象）相对。它有三种用法。

### 5.1 super.成员变量

当子类和父类有同名变量时，用 `super.变量名` 访问父类的变量：

```java
class Father {
    String name = "父亲";
}
class Son extends Father {
    String name = "儿子";
    public void show() {
        System.out.println(name);        // 儿子（本类的）
        System.out.println(super.name);  // 父亲（父类的）
    }
}
```

### 5.2 super.方法名()

在子类重写了父类方法后，如果想调用父类**被覆盖的原始版本**，用 `super.方法名()`：

```java
class Animal {
    public void eat() { System.out.println("动物吃东西"); }
}
class Dog extends Animal {
    @Override
    public void eat() {
        super.eat();                     // 先调用父类的 eat()
        System.out.println("狗啃骨头");   // 再添加子类特有的逻辑
    }
}
// new Dog().eat(); 输出：
// 动物吃东西
// 狗啃骨头
```

### 5.3 super() 调用父类构造方法

子类的构造方法**第一行**默认会隐式调用父类的无参构造 `super()`。也可以手动用 `super(参数)` 调用父类的有参构造。

```java
class Father {
    public Father() {
        System.out.println("父类无参构造");
    }
    public Father(String name) {
        System.out.println("父类有参构造：" + name);
    }
}
class Son extends Father {
    public Son() {
        super();                  // 默认就有，可省略；调用父类无参构造
        System.out.println("子类无参构造");
    }
    public Son(String name) {
        super(name);              // 显式调用父类有参构造，必须在第一行
        System.out.println("子类有参构造");
    }
}
// new Son();      输出：父类无参构造 → 子类无参构造
// new Son("小明"); 输出：父类有参构造：小明 → 子类有参构造
```

## 六、构造器调用链（重点）

> ⚠️ **注意：** 子类构造方法的第一行，**必须**是 `super(...)` 或 `this(...)`。如果你什么都不写，编译器会自动补一个 `super();`（调用父类无参构造）。

这意味着：**创建子类对象时，一定会先创建并初始化父类部分**。

```
new Son()
   │
   ▼
Son 构造方法第一行  ──▶  super()  ──▶  Father 构造方法
   │                                       │
   │                                       ▼
   │                                  (父类成员初始化完毕)
   ▼
(子类自己的初始化逻辑)
   │
   ▼
对象创建完成
```

**经典陷阱**：如果父类**没有无参构造**（因为你写了有参构造），而子类构造里又没显式 `super(参数)`，会编译报错：

```java
class Father {
    public Father(String name) { }    // 只有无参构造被"顶替"了
}
class Son extends Father {
    public Son() {
        // 编译器在这里隐式插入 super();  但父类没有无参构造！❌ 编译错误
    }
    // ✅ 解决：手动调用父类的有参构造
    // public Son() { super("默认"); }
}
```

## 七、Object 类：所有类的根父类

Java 中，**如果一个类没有显式继承任何父类，那么它默认继承 `java.lang.Object`**。因此 `Object` 是所有类的**最终根父类（祖先类）**，所有对象（包括数组）都"是一个 Object"。

```
        Object
          ▲
          │ extends
     ┌────┴────┐
     │         │
   Animal    String
     ▲
     │
    Dog
```

Object 类中定义了若干所有对象都具备的方法，最常用的三个：

| 方法 | 作用 | 默认行为 | 建议重写 |
|------|------|---------|---------|
| `toString()` | 返回对象的字符串表示 | 返回 `类名@哈希值`，无意义 | ✅ 建议重写，返回有意义的属性信息 |
| `equals(Object)` | 判断两个对象是否"相等" | 默认比较地址（`==`） | ✅ 建议重写，按内容比较 |
| `hashCode()` | 返回对象的哈希码 | 基于地址计算 | ✅ 重写 equals 时通常一并重写 |

```java
public class Student {
    String name;
    int age;

    @Override
    public String toString() {
        return "Student{name='" + name + "', age=" + age + "}";
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;          // 同一个对象
        if (o == null || getClass() != o.getClass()) return false;
        Student s = (Student) o;
        return age == s.age && name.equals(s.name);
    }

    @Override
    public int hashCode() {
        return name.hashCode() * 31 + age;
    }
}
```

> 💡 **提示：** `equals` 和 `hashCode` 有契约：两个 `equals` 相等的对象，它们的 `hashCode` 必须相等。所以重写 `equals` 时一定要同时重写 `hashCode`。

## 八、final 关键字

`final` 有"最终、不可变"的含义，可以修饰**类、方法、变量**，含义各不相同。

| 修饰对象 | 含义 | 示例 |
|---------|------|------|
| **final 修饰类** | 该类**不能被继承**（没有子类） | `public final class String { }` |
| **final 修饰方法** | 该方法**不能被子类重写** | `public final void run() { }` |
| **final 修饰变量** | 该变量只能被赋值一次，成为**常量** | `final int MAX = 100;` |

### 8.1 final 修饰类

```java
public final class String { }     // String 是 final 类，不能被继承
// class MyString extends String { }  // ❌ 编译错误：final 类不可继承
```

常见 final 类：`String`、`Integer`、`Math` 等。设计为 final 是为了保证其行为不被篡改（如 String 的不可变性对线程安全、字符串常量池至关重要）。

### 8.2 final 修饰方法

```java
class Father {
    public final void safeMethod() { }   // 这个方法子类不能重写
}
class Son extends Father {
    // public void safeMethod() { }      // ❌ 编译错误：final 方法不可重写
}
```

### 8.3 final 修饰变量

```java
final double PI = 3.14159;     // 常量，只能赋值一次
// PI = 3.14;                  // ❌ 编译错误：不能再赋值
```

> ⚠️ **注意：** `final` 修饰引用类型变量时，指的是**引用地址不可变**，但对象内部的内容仍然可以修改：
> ```java
> final int[] arr = {1, 2, 3};
> arr[0] = 100;          // ✅ 合法，修改的是数组内部内容
> // arr = new int[5];   // ❌ 非法，arr 的引用不能再指向新数组
> ```

## 九、继承的优缺点与"组合优于继承"

### 9.1 继承的优缺点

| 优点 | 缺点 |
|------|------|
| 提高代码复用性 | 增强了类之间的耦合度（父类改动会影响所有子类） |
| 建立类层次，便于多态 | 破坏封装：子类依赖父类的实现细节 |
| 减少重复代码 | 继承层次过深时，代码难以理解和维护 |

### 9.2 组合优于继承（Effective Java 核心原则）

**组合（Composition）** 指：在一个类中**持有另一个类的对象作为成员变量**，而不是去继承它。这是著名的设计原则——**"优先使用组合而不是继承"（Favor composition over inheritance）**。

```java
// ❌ 继承方式：HashSet 本来不该暴露 addAll 的实现细节，但继承强制暴露了
class MySet<E> extends java.util.HashSet<E> {
    public int addCount = 0;
    @Override public boolean add(E e) { addCount++; return super.add(e); }
    @Override public boolean addAll(java.util.Collection<? extends E> c) {
        addCount += c.size();
        return super.addAll(c);   // ⚠️ 父类 addAll 内部会调用 add，导致计数翻倍！
    }
}

// ✅ 组合方式：内部持有 HashSet，只暴露需要的方法，不依赖父类实现
class MySet2<E> {
    private final java.util.HashSet<E> set = new java.util.HashSet<>();
    public boolean add(E e) { return set.add(e); }
    // ... 只暴露需要的能力，彻底解耦
}
```

> 💡 **提示：** 判断该用继承还是组合的简单标准——如果是 **"is-a"（是一个）** 关系用继承（如"狗是一个动物"）；如果是 **"has-a"（有一个）** 关系用组合（如"汽车有一个引擎"）。大部分业务场景下，组合比继承更灵活、更安全。

## 十、面试常见问题

### Q1：Java 为什么只支持单继承，不支持多继承？

为了避免**菱形继承（钻石问题）**。假设类 A 和类 B 都有同名方法 `run()`，如果类 C 同时继承 A 和 B，调用 `c.run()` 时会产生二义性。Java 通过"类单继承 + 接口多实现"的设计来兼顾复用性和明确性。

### Q2：子类能继承父类的构造方法吗？

**不能**。构造方法必须与类名相同，子类不可能拥有父类名字的构造方法，所以构造方法不能被继承。子类对象创建时，是通过 `super(...)` 来**调用**父类的构造方法完成父类部分的初始化，而不是"继承"它。

### Q3：方法重写有哪些规则？@Override 有什么用？

重写规则：方法名相同、参数列表相同、返回值相同（或子类型）、访问权限不能更严、抛出的受检异常不能更大、不能重写 `private`/`static`/`final` 方法。`@Override` 注解强制编译器检查该方法是否真正重写了父类方法，避免因拼写或签名不一致导致的"假重写"，强烈建议每次重写都加上。

### Q4：super() 和 this() 能同时出现在构造方法中吗？

**不能**。因为 `super()` 和 `this()` 都要求**必须放在构造方法的第一行**，两者不可能同时第一。实际上，如果构造方法第一行是 `this(...)`，那么被调用的那个构造方法里一定会有 `super(...)`，最终保证每个构造链顶端都会调用父类构造。

### Q5：为什么子类构造方法一定会调用父类构造？

因为子类对象在内存中**包含**父类的成员（包括 private 的），必须先初始化父类部分才能保证对象的完整性。Java 强制规定：子类构造第一行必须先初始化父类，要么显式 `super(...)`，要么隐式 `super()`。这也是"先有父才有子"的体现。

### Q6：final 修饰类、方法、变量分别是什么效果？

final 类不能被继承（如 String）；final 方法不能被子类重写（防止行为被篡改）；final 变量只能赋值一次成为常量（引用类型时地址不变，但内容可变）。

### Q7：equals 和 hashCode 有什么关系？

Java 规定：两个对象 `equals` 相等，它们的 `hashCode` 必须相等（反过来不强求）。因此**重写 equals 时必须同时重写 hashCode**，否则在 HashMap、HashSet 等基于哈希的容器中会出现逻辑错误——两个"相等"的对象却被当作不同对象。

## 十一、总结

| 知识点 | 核心要点 |
|--------|---------|
| extends 关键字 | 表示继承，Java 单继承，可多层传递 |
| 可继承的成员 | 非 private 的成员变量和方法；**构造方法不能继承** |
| 方法重写 | 方法名、参数相同；返回值兼容；权限不能更严；@Override 校验 |
| super 关键字 | super.变量、super.方法()、super() 调父类构造 |
| 构造器链 | 子类构造第一行必为 super() 或 this()，先初始化父类 |
| Object 类 | 所有类的根父类；常用 toString/equals/hashCode |
| final 关键字 | 修饰类不可继承、方法不可重写、变量为常量 |
| 设计原则 | "组合优于继承"，is-a 用继承，has-a 用组合 |

继承是面向对象的基石之一，掌握它能让你写出层次清晰、复用性强的代码；同时要牢记"组合优于继承"，避免滥用继承造成的高耦合陷阱。
