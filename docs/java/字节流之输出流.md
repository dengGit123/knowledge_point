## 一、概述

上一篇我们学习了**字节输入流**（把数据从外部「读」进程序）。与之对称，**字节输出流（OutputStream）** 负责把数据从程序「写」到外部目的地（文件、网络、控制台等）。

- **输入流（InputStream）**：读 —— 数据从外部流入程序。
- **输出流（OutputStream）**：写 —— 数据从程序流出到外部。

字节输出流以**字节（byte）**为最小单位，适合写**任意二进制数据**（图片、视频、音频、压缩包，以及不含编码要求的原始字节）。它与字节输入流一起，构成 Java IO 字节流体系。

> 💡 **提示：** 在 Java IO 中，「输入 / 输出」永远以**程序本身**为参照系：数据进程序叫输入，出程序叫输出。

---

## 二、核心概念与体系结构

### 1. OutputStream 是所有字节输出流的抽象基类

`java.io.OutputStream` 是抽象类，定义了字节输出流最核心的 `write()` 和 `flush()` 方法。所有以字节为单位「写」数据的流都继承自它。

```
                         ┌───────────────┐
            抽象基类       │ OutputStream  │   (定义 write / flush / close)
                         └───────┬───────┘
                                 │
        ┌────────────────────────┼───────────────────────────────┐
        │                        │                               │
┌─────────────────┐   ┌───────────────────┐           ┌─────────────────────┐
│ FileOutputStream│   │BufferedOutputStream│          │ ObjectOutputStream │
│ (写字节到文件)  │   │ (带缓冲的装饰器)   │          │ (写对象-序列化)     │
└─────────────────┘   └───────────────────┘           └─────────────────────┘
        │                        │
        │  (作为被装饰对象)       │ (装饰任意 OutputStream)
        │                        │
        └─────────────► BufferedOutputStream(new FileOutputStream(file))
                                 │
                     ┌───────────┴───────────┐
                     │ByteArrayOutputStream │  (写到内存中的 byte[])
                     └──────────────────────┘
```

常见子类对比：

| 流类 | 数据去向 | 特点 |
| --- | --- | --- |
| `FileOutputStream` | 磁盘文件 | 最常用，按字节写入文件 |
| `BufferedOutputStream` | 装饰其它输出流 | 自带 8KB 缓冲，减少磁盘写入次数 |
| `ObjectOutputStream` | 任意输出流 | 序列化，把 Java 对象写成字节 |
| `ByteArrayOutputStream` | 内存中的 `byte[]` | 写到内存缓冲区，常用于测试、拼接数据 |

> 💡 **提示：** 与输入流一样，输出流体系也使用了**装饰器模式**：`BufferedOutputStream`「包装」`FileOutputStream` 为其增加缓冲能力。

## 三、FileOutputStream 详解

`FileOutputStream` 把字节写入文件系统中的某个文件。

### 1. 构造方法（覆盖 vs 追加）

| 构造方法 | 写入模式 | 说明 |
| --- | --- | --- |
| `FileOutputStream(String name)` | **覆盖** | 文件不存在则创建；存在则**清空原内容** |
| `FileOutputStream(File file)` | **覆盖** | 同上 |
| `FileOutputStream(String name, boolean append)` | 可选 | `append=true` 为**追加**模式 |
| `FileOutputStream(File file, boolean append)` | 可选 | `append=true` 为**追加**模式 |

> ⚠️ **注意：** 默认是**覆盖模式**！如果文件已存在，原内容会被全部清空。需要保留原内容、在末尾续写时，务必传 `append=true`。

```java
// ❌ 覆盖模式：每次运行都会清空 a.txt 原有内容
FileOutputStream fos1 = new FileOutputStream("a.txt");

// ✅ 追加模式：在文件末尾继续写，保留原内容
FileOutputStream fos2 = new FileOutputStream("a.txt", true);
```

### 2. 核心方法

| 方法签名 | 说明 |
| --- | --- |
| `void write(int b)` | 写入**一个字节**（只写 int 的**低 8 位**） |
| `void write(byte[] b)` | 把整个字节数组写入 |
| `void write(byte[] b, int off, int len)` | 写入数组从 `off` 开始的 `len` 个字节 |
| `void flush()` | 把缓冲区中的数据**真正写入**目的地 |
| `void close()` | 关闭流并释放资源（关闭前会自动 flush） |

> 💡 **关键理解：** `write(int b)` 虽然接收 `int`，但只写入它的**低 8 位**（即 `b & 0xFF`）。例如 `write(300)` 实际写入的是 `300` 的低 8 位 `44`。所以传参时要确保数值在 0~255 之间。

## 四、写入文件示例

### 1. 基本写入

```java
import java.io.FileOutputStream;
import java.io.IOException;

public class WriteDemo {
    public static void main(String[] args) {
        // try-with-resources：自动关闭流（Java 7+）
        try (FileOutputStream fos = new FileOutputStream("/Users/demo/out.txt")) {
            fos.write(72);  // 写入字节 'H'
            fos.write(105); // 写入字节 'i'
            // 写入一个字节数组（"Java" 的 UTF-8 字节）
            fos.write("Java".getBytes());
            // 写入数组的一部分
            byte[] data = "HelloWorld".getBytes();
            fos.write(data, 5, 5); // 只写 "World"
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### 2. 追加写入

```java
import java.io.FileOutputStream;
import java.io.IOException;

public class AppendDemo {
    public static void main(String[] args) {
        // 第二个参数 true 表示追加模式
        try (FileOutputStream fos = new FileOutputStream("/Users/demo/log.txt", true)) {
            fos.write("这一行会被追加到文件末尾\n".getBytes());
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 五、flush() 的作用

`flush()` 的作用是把流**内部缓冲区中还没写出去的数据**「冲刷」到真正的目的地。

```
程序写入的数据 ──► [流的内部缓冲区] ──flush()──► 真正的文件/网络
                          │
                  缓冲区满了也会自动写出
```

- 对于 `FileOutputStream` 这种**无缓冲**的流，`write()` 基本直接写入文件，`flush()` 意义不大。
- 对于 `BufferedOutputStream` 这种**有缓冲**的流，`write()` 先写到内存缓冲区，**只有缓冲区满、调用 `flush()` 或 `close()` 时才真正落盘**。

> ⚠️ **注意：** 使用缓冲输出流时，如果程序中途崩溃且**没有 flush/close**，缓冲区里的数据会**丢失**！务必通过 `close()`（推荐 try-with-resources）或显式 `flush()` 保证数据落盘。

## 六、BufferedOutputStream 缓冲字节流

`BufferedOutputStream` 是 `OutputStream` 的装饰器，内部维护**默认 8192 字节（8KB）的缓冲区**。写入时先攒到内存缓冲区，攒满后再一次性写入磁盘，大幅减少磁盘写入次数。

### 1. 使用示例

```java
import java.io.*;

public class BufferedOutputDemo {
    public static void main(String[] args) {
        // 用 BufferedOutputStream 包装 FileOutputStream
        try (BufferedOutputStream bos = new BufferedOutputStream(
                new FileOutputStream("/Users/demo/out.bin"))) {

            for (int i = 0; i < 1000; i++) {
                bos.write(i); // 先写入内存缓冲区（极快）
            }
            // close() 前会自动 flush，缓冲区数据真正写入磁盘
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> 💡 **提示：** `close()` 内部会自动调用 `flush()`，所以正常关闭流时数据不会丢失。但如果你需要在**关闭前**就让数据立刻可见（例如日志、网络场景），应显式调用 `flush()`。

## 七、综合示例：文件复制

文件复制是字节流最经典的应用：用 `FileInputStream` 读取源文件，用 `FileOutputStream` 写入目标文件，配合 `byte[]` 缓冲一次读写一批。由于按字节复制，**任何类型的文件（图片、视频、压缩包等）都能正确复制**。

```java
import java.io.*;

public class FileCopyDemo {
    public static void main(String[] args) {
        File src = new File("/Users/demo/src.jpg");
        File dest = new File("/Users/demo/dest.jpg");

        // try-with-resources 同时管理输入流和输出流，用分号分隔
        try (FileInputStream fis = new FileInputStream(src);
             FileOutputStream fos = new FileOutputStream(dest)) {

            byte[] buffer = new byte[8192]; // 8KB 缓冲区
            int len;
            // 边读边写：read 返回实际读到的字节数，-1 表示读完
            while ((len = fis.read(buffer)) != -1) {
                // 只写实际读到的 len 个字节，避免末尾脏数据
                fos.write(buffer, 0, len);
            }
            System.out.println("复制完成");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

> 💡 **进阶：** 加上缓冲流后性能更好：
> ```java
> try (BufferedInputStream bis = new BufferedInputStream(new FileInputStream(src));
>      BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream(dest))) {
>     byte[] buffer = new byte[8192];
>     int len;
>     while ((len = bis.read(buffer)) != -1) {
>         bos.write(buffer, 0, len);
>     }
> } catch (IOException e) {
>     e.printStackTrace();
> }
> ```

## 八、资源关闭：try-with-resources（重点）

### 1. 为什么必须关闭流

流对象会占用**操作系统文件句柄**等稀缺资源。如果不关闭：
- 文件句柄泄漏，最终导致程序无法再打开新文件（`Too many open files`）。
- 文件可能被锁定，其他程序无法访问。
- 缓冲流中的数据可能没有 flush 而**丢失**。

### 2. 传统方式：try-finally 手动关闭（繁琐、易错）

```java
// ❌ 传统写法：代码冗长，关闭顺序、异常处理都容易出错
FileInputStream fis = null;
try {
    fis = new FileInputStream("a.txt");
    // 使用 fis ...
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (fis != null) {
        try {
            fis.close(); // 关闭时还可能抛异常，需要再包一层
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### 3. 最佳实践：try-with-resources（Java 7+）

只要流实现了 `AutoCloseable` 接口（所有标准 IO 流都实现了），就可以放在 try 的「资源声明」括号里，**无论正常结束还是抛异常，JVM 都会自动按声明逆序关闭它们**。

```java
// ✅ try-with-resources：简洁、安全、自动关闭
try (FileInputStream fis = new FileInputStream("a.txt");
     FileOutputStream fos = new FileOutputStream("b.txt")) {
    // 使用 fis、fos ...
} catch (IOException e) {
    e.printStackTrace();
}
// 此处 fis、fos 已被自动关闭，无需手动 close
```

> 💡 **提示：** 本系列所有 IO 示例**统一使用 try-with-resources** 关闭资源。这是 Java 7+ 的最佳实践，既避免了资源泄漏，又让代码简洁可读。

### 4. 两种方式对比

| 对比项 | try-finally | try-with-resources |
| --- | --- | --- |
| 代码量 | 多，冗长 | 少，简洁 |
| 自动关闭 | 否，需手动 close | 是，JVM 自动关闭 |
| 异常处理 | 关闭异常需额外处理 | 自动处理，抑制异常不丢失 |
| 出错风险 | 高（忘记关闭、顺序错误） | 低 |
| 推荐度 | ❌ 旧代码 | ✅ 最佳实践 |

## 九、注意事项与最佳实践

1. **默认覆盖模式**：`new FileOutputStream(file)` 会清空原文件；需追加传 `true`。
2. **写完必须 flush/close**：缓冲流场景下不关闭，数据会丢失。
3. **`write(int)` 只写低 8 位**：传参值大于 255 时只保留最低字节。
4. **只写实际长度**：`write(buffer, 0, len)` 避免写入末尾脏数据。
5. **统一 try-with-resources**：自动关闭，杜绝资源泄漏。

```java
// ✅ 字节输出流标准模板
try (BufferedOutputStream bos = new BufferedOutputStream(
        new FileOutputStream("目标文件", true))) { // true=追加
    bos.write("要写入的内容".getBytes());
    bos.flush(); // 需要立即落盘时显式调用
} catch (IOException e) {
    e.printStackTrace();
}
```

## 十、面试常见问题

### Q1：`FileOutputStream` 的覆盖和追加有什么区别？

默认构造方法（不传 `append` 参数）是**覆盖模式**，会清空原文件内容；传入 `append=true` 则是**追加模式**，新内容写到文件末尾，原内容保留。

### Q2：`flush()` 和 `close()` 的区别？不调用会怎样？

`flush()` 只是把缓冲区数据冲刷到目的地，**流仍可继续使用**；`close()` 先 flush 再关闭流，**之后流不可再用**。对于缓冲输出流，如果既不 flush 也不 close 就结束程序，缓冲区里的数据会**丢失**。

### Q3：`write(int b)` 传入 300 会写入什么？

只写入 `int` 的**低 8 位**。`300` 的二进制低 8 位是 `00101100`，即十进制 `44`（字符 `,`）。所以实际写入的是 44 这个字节。

### Q4：为什么推荐 try-with-resources？

它要求资源实现 `AutoCloseable`，JVM 保证无论正常返回还是抛异常都会**自动关闭**资源，避免了 try-finally 手动关闭时容易出现的「忘记关闭」「关闭顺序错误」「关闭异常被吞」等问题，是 Java 7+ 处理资源的最佳实践。

### Q5：字节流能写中文吗？

能写，但要把字符串按某种编码转成字节数组（`"中文".getBytes("UTF-8")`）。如果编码处理不当（写时编码与读时不一致），会导致乱码。**写中文文本建议直接用字符流**（`Writer`），它自动处理编码。

### Q6：文件复制为什么要用字节流？

字节流是「万能流」，按字节逐位复制，能完整保留任意二进制文件的内容（图片、视频、压缩包等）。如果用字符流复制二进制文件，会因为「字节 ↔ 字符」的编解码转换导致数据损坏。

## 十一、总结

- **`OutputStream`** 是所有字节输出流的抽象基类，核心方法是 `write()` 和 `flush()`。
- **`FileOutputStream`** 按字节写入文件，注意区分**覆盖**（默认）和**追加**（`append=true`）。
- **`BufferedOutputStream`** 是装饰器，提供 8KB 缓冲，大幅提升写入效率。
- **`flush()` 很重要**：缓冲流场景必须 flush 或 close 才能真正落盘。
- **文件复制** = `FileInputStream` 读 + `FileOutputStream` 写 + `byte[]` 缓冲，可复制任意文件。
- **资源关闭统一用 try-with-resources**（Java 7+），这是整个 IO 流系列的最佳实践。
