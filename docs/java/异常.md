# 异常（Exception）

> 官方文档：[Java Exceptions Trail](https://docs.oracle.com/javase/tutorial/essential/exceptions/)、[菜鸟教程 Java 异常处理](https://www.runoob.com/java/java-exceptions.html)

## 一、概述

**异常（Exception）** 是程序运行过程中出现的非正常情况。Java 提供了一套完善的异常处理机制（`try-catch-finally`、`throw`、`throws`），让程序在遇到错误时能够**优雅地处理**，而不是直接崩溃。

> 💡 **提示：** 类比：异常处理就像开车系安全带——平时用不上，但出事故（异常）时能保护你。`try` 是"可能出事的路段"，`catch` 是"事故应对预案"，`finally` 是"不管出没出事都要做的事"（如熄火、锁车）。

## 二、核心概念：异常体系结构

Java 中所有异常和错误的根类是 `Throwable`，它有两个分支：`Error` 和 `Exception`。

```text
                        Throwable（所有错误/异常的根）
                         │
            ┌────────────┴────────────┐
          Error                    Exception
            │                         │
   （JVM 严重错误，         ┌──────────┴───────────────────┐
    程序不应处理）      受检异常 Checked          非受检异常 RuntimeException
                        （编译器强制处理）        （编译器不强制处理）
                            │                         │
                   IOException              NullPointerException
                   SQLException             ArrayIndexOutOfBoundsException
                   ClassNotFoundException  ClassCastException
                   FileNotFoundException    ArithmeticException
                                            IllegalArgumentException
```

### 三大分类

| 类别 | 说明 | 是否必须处理 | 典型例子 |
|------|------|------------|---------|
| `Error` | JVM 严重错误，程序通常无法恢复 | 否（也不建议处理） | `OutOfMemoryError`、`StackOverflowError` |
| 受检异常（Checked Exception） | `Exception` 的非 `RuntimeException` 子类，编译器强制处理 | **是**，否则编译不通过 | `IOException`、`SQLException` |
| 非受检异常（RuntimeException） | 运行时异常，编译器不强制处理 | 否（可以处理，也可以不处理） | `NullPointerException`、`ArithmeticException` |

```java
// 受检异常：编译器强制你处理（要么 try-catch，要么 throws）
public void readFile() {
    // ❌ 编译报错：未处理的异常 IOException
    // FileInputStream fis = new FileInputStream("a.txt");
}

// 非受检异常：编译器不强制
public int divide(int a, int b) {
    return a / b; // 可能抛 ArithmeticException，但编译能过
}
```

## 三、try-catch-finally

### 1. 基本执行流程

```java
try {
    // 可能抛出异常的代码
} catch (具体异常类型 e) {
    // 处理该类型异常
} finally {
    // 无论是否发生异常，都会执行（用于资源清理）
}
```

```java
public class TryCatchDemo {
    public static void main(String[] args) {
        try {
            int[] arr = {1, 2, 3};
            System.out.println(arr[5]); // 越界，抛 ArrayIndexOutOfBoundsException
            System.out.println("这行不会执行"); // 异常后的代码不执行
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("捕获到越界异常：" + e.getMessage());
        } finally {
            System.out.println("finally 一定执行");
        }
        System.out.println("程序继续运行");
    }
}
```

### 2. 多个 catch 的顺序：子类在前

当有多个 `catch` 时，**必须把子类异常写在父类异常前面**，否则编译报错（因为父类会先捕获，后面的子类永远到不了）。

```java
try {
    // ...
} catch (FileNotFoundException e) {   // ✅ 子类在前
    System.out.println("文件未找到");
} catch (IOException e) {              // ✅ 父类在后
    System.out.println("IO 异常");
}
// catch (FileNotFoundException e) {}  ❌ 若放这里编译报错：已被前面的 IOException 捕获
```

### 3. 多异常捕获（Java 7+ multi-catch）

```java
try {
    // ...
} catch (IOException | SQLException e) {   // 用 | 连接多个异常类型
    // e 的类型是这些异常的共同父类，且隐式 final
    e.printStackTrace();
}
```

### 4. finally 一定会执行

`finally` 块**无论是否发生异常、是否 return、catch 是否捕获**，都会执行（唯一例外：`System.exit()` 或 JVM 崩溃）。

```java
public static int demo() {
    try {
        return 1;
    } finally {
        System.out.println("finally 仍会执行，即便 try 里 return 了");
    }
}
```

## 四、throw 与 throws

| 关键字 | 位置 | 作用 | 后面跟什么 |
|--------|------|------|-----------|
| `throw` | 方法体内 | **手动抛出**一个异常对象 | 一个异常对象（`new XxxException()`） |
| `throws` | 方法签名上 | **声明**该方法可能抛出的异常类型 | 异常类名（可多个，逗号分隔） |

```java
public class ThrowThrowsDemo {
    // throws：声明该方法可能抛出 IllegalArgumentException（非受检，可省略）和 IOException（受检，必须声明）
    public void checkAge(int age) throws IllegalArgumentException {
        if (age < 0) {
            // throw：手动抛出一个异常对象
            throw new IllegalArgumentException("年龄不能为负数：" + age);
        }
        System.out.println("年龄合法：" + age);
    }

    public static void main(String[] args) {
        try {
            new ThrowThrowsDemo().checkAge(-5);
        } catch (IllegalArgumentException e) {
            System.out.println("捕获：" + e.getMessage());
        }
    }
}
```

## 五、自定义异常

业务开发中经常需要自定义异常类。继承 `Exception` 表示**受检异常**，继承 `RuntimeException` 表示**非受检异常**。通常要提供带消息和带原因的构造器。

```java
// 自定义受检异常
public class BusinessException extends Exception {
    private int code;

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }

    public BusinessException(int code, String message, Throwable cause) {
        super(message, cause); // 保留原因（异常链）
        this.code = code;
    }

    public int getCode() {
        return code;
    }
}

// 使用
public class UserService {
    public void login(String username, String password) throws BusinessException {
        if (username == null || username.isEmpty()) {
            throw new BusinessException(4001, "用户名不能为空");
        }
        // ...
    }
}
```

## 六、try-with-resources（Java 7+）

对于实现了 `AutoCloseable` 接口的资源（文件流、数据库连接等），用 `try-with-resources` 可以**自动关闭**，无需手写 `finally`，避免资源泄漏。

```java
import java.io.*;

public class TryWithResourcesDemo {
    public static void main(String[] args) {
        // ✅ 在 try 后的括号里声明的资源会自动关闭（即使发生异常）
        try (BufferedReader br = new BufferedReader(new FileReader("a.txt"))) {
            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        // br 在此处已自动关闭，无需 finally
    }
}
```

> 💡 **提示：** Java 9 起，`try` 括号里可以引用外部已声明的 `final`（或 effectively final）变量，写法更简洁。

## 七、异常链（Exception Chaining）

在 `catch` 中抛出新异常时，应把原异常作为 `cause` 传入，保留完整的错误信息，方便排查问题。

```java
public Data loadData() throws BusinessException {
    try {
        // 读取文件
        return readFromFile();
    } catch (IOException e) {
        // 把底层 IOException 包装成业务异常，但保留原因
        throw new BusinessException(5001, "数据加载失败", e);  // e 作为 cause
    }
}
```

调用 `e.printStackTrace()` 会打印完整的异常链（`Caused by: ...`）。

## 八、陷阱：finally 与 return

> ⚠️ **注意：** `finally` 中的 `return` 会**覆盖** `try`/`catch` 中的 `return`，还会"吞掉"异常，强烈不推荐在 `finally` 中 `return`。

```java
public static int trap() {
    try {
        return 1;
    } finally {
        return 2; // ❌ 会覆盖 try 的 return 1，最终返回 2
    }
}

public static int trap2() {
    try {
        throw new RuntimeException("出错了");
    } catch (RuntimeException e) {
        return 1;
    } finally {
        return 2;       // ❌ 不仅覆盖了 return 1，还吞掉了异常（异常丢失）
    }
}
```

> 💡 **提示：** `finally` 中改变基本类型返回值也不会生效（因为 `return` 的值在进入 finally 前已确定），但修改对象内部状态会生效。最佳实践：**finally 只做资源清理，不要写 return**。

## 九、异常处理最佳实践

> ⚠️ **注意：**
> 1. **捕获具体的异常**，不要图省事 `catch (Exception e)` 吞掉所有异常，会掩盖真正的错误。
> 2. **不要捕获后什么都不做**（空 catch），至少要记录日志。
> 3. **能处理的异常才捕获**，处理不了的向上抛出，让上层统一处理。
> 4. **finally 一定要关闭资源**，或优先用 try-with-resources。
> 5. **自定义异常要有意义**，携带错误码和清晰的提示信息。
> 6. **保留异常链**，包装异常时传入 `cause`。
> 7. **受检异常慎用**，现代 Java（尤其是框架中）更倾向用非受检异常（RuntimeException）。

```java
// ❌ 反例：捕获太宽 + 空 catch
try {
    // ... 很多代码
} catch (Exception e) {
    // 什么都不做，问题被掩盖
}

// ✅ 正例：捕获具体异常 + 记录日志 + 合理处理
try {
    readFile();
} catch (FileNotFoundException e) {
    log.error("文件不存在：{}", path, e);
    throw new BusinessException(4004, "文件不存在", e);
}
```

## 十、受检异常 vs 非受检异常

| 对比维度 | 受检异常（Checked） | 非受检异常（RuntimeException） |
|---------|--------------------|-----------------------------|
| 继承关系 | `Exception` 的非 `RuntimeException` 子类 | `RuntimeException` 及其子类 |
| 编译检查 | **强制**处理（try-catch 或 throws） | 不强制 |
| 性质 | 外部环境导致的错误（文件、网络、数据库） | 程序逻辑错误（空指针、越界） |
| 典型例子 | `IOException`、`SQLException` | `NullPointerException`、`ArithmeticException` |
| 是否可恢复 | 通常可恢复（重试、提示用户） | 通常是代码 bug，应修改代码 |

## 十一、面试常见问题

### Q1：Error 和 Exception 的区别？

`Error` 是 JVM 严重错误（如 `OutOfMemoryError`、`StackOverflowError`），程序通常无法也不应该处理；`Exception` 是程序可处理的异常。两者都继承自 `Throwable`。

### Q2：运行时异常和受检异常的区别？

受检异常（`Exception` 的非 `RuntimeException` 子类）编译器强制处理，代表外部环境错误（如 IO、数据库）；运行时异常（`RuntimeException`）编译器不强制处理，代表程序逻辑错误（如空指针、数组越界）。

### Q3：throw 和 throws 的区别？

`throw` 在方法体内，用于**手动抛出**一个异常对象；`throws` 在方法签名上，用于**声明**该方法可能抛出的异常类型（可多个）。前者跟对象，后者跟类名。

### Q4：finally 一定会执行吗？

在正常情况下一定会执行。唯一例外：`System.exit()` 被调用、JVM 崩溃、或线程被杀死。此外，如果 try/finally 在守护线程中且主线程已退出，finally 也可能不执行。

### Q5：finally 中的 return 会发生什么？

`finally` 中的 `return` 会覆盖 try/catch 中的 `return`，并且会吞掉异常（异常被丢弃）。这是严重的不良实践，应避免在 finally 中写 return。

### Q6：什么是 try-with-resources？

Java 7 引入的语法，用于自动关闭实现了 `AutoCloseable` 的资源。在 `try` 后的括号中声明资源，无论是否异常都会自动调用 `close()`，比手写 finally 更简洁、更安全，能避免资源泄漏。

## 十二、总结

- Java 异常体系的根是 `Throwable`，分 `Error`（不处理）和 `Exception`（可处理）。
- `Exception` 分受检异常（强制处理）和非受检异常（不强制）。
- 异常处理三件套：`try`（尝试）、`catch`（捕获）、`finally`（收尾一定执行）。
- `throw` 抛异常对象，`throws` 声明异常类型；自定义异常继承 `Exception` 或 `RuntimeException`。
- 优先用 **try-with-resources** 关闭资源；包装异常时保留**异常链**；遵循"捕获具体、不吞异常、finally 不 return"的最佳实践。
