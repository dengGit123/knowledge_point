### 一、概述

> 📖 [仓库简介](https://maven.apache.org/guides/introduction/introduction-to-repositories.html) ｜ [镜像配置指南](https://maven.apache.org/guides/mini/guide-mirror-settings.html)

Maven 的「**仓库**」就是**存放 jar 包的地方**。你声明依赖后，Maven 会按「**本地 → 私服 → 中央**」的顺序去找 jar，找到就缓存到本地。

大白话：**仓库就是「jar 包的快递驿站」**。本地仓库是你家的储物间（最快），中央仓库是总仓库（最全但远），私服是你公司的中转站。每次要 jar，先翻家里储物间，没有就去公司中转站，再没有就去总仓。

| 你将学到 | 说明 |
| --- | --- |
| 三种仓库 | 本地仓库 / 私服 / 中央仓库 |
| 仓库查找顺序 | Maven 找 jar 的优先级 |
| 镜像（mirror） | 怎么加速下载（配阿里云镜像） |
| 私服 | Nexus 搭建与使用 |

---

### 二、三种仓库

#### 1. 本地仓库（Local Repository）

你电脑上的目录，默认 `~/.m2/repository`（Windows 是 `C:\Users\用户名\.m2\repository`）。

- 第一次下载的 jar 会缓存到这里
- 以后再用同一个 jar，**直接从本地读，不联网**
- 可以自定义位置（在 `settings.xml` 配 `localRepository`）

```xml
<!-- settings.xml -->
<settings>
    <localRepository>D:/maven-repo</localRepository>
</settings>
```

> 💡 **提示：** 本地仓库可以随时整个删除，下次构建会重新下载所需的 jar。当遇到「jar 损坏、版本错乱」的玄学问题，删掉本地仓库里对应目录重新下，是终极解决方案。

#### 2. 中央仓库（Central Repository）

Maven **默认**的远程公共仓库，地址 `https://repo1.maven.org/maven2/`。

- 全球共享，jar 最全
- 在**国外**，国内访问慢
- Maven 自带的「**超级 POM（Super POM）**」里配置了它，所以默认就能用

去 [Maven Central](https://central.sonatype.com/) 或 [MVNRepository](https://mvnrepository.com/) 搜索 jar 坐标。

#### 3. 私服（Private Repository）

公司或团队**内部搭建**的仓库，常用 **Nexus** 或 **JFrog Artifactory**。

作用：

- **加速**：私服缓存中央仓库的 jar，团队只下载一次，内部网络快
- **私有 jar**：公司内部开发的 jar 包，不上传公网，只在私服共享
- **统一管理**：控制谁能发布、谁能下载

```
开发者的电脑
    │ 找 jar
    ▼
本地仓库 (.m2) ──没有──► 私服 (Nexus) ──没有──► 中央仓库
                              │                      │
                              └──── 缓存 ◄──────────┘
```

### 三、仓库查找顺序（重要）

Maven 找一个 jar 时的优先级：

```
1. 本地仓库          → 有就用，没有下一步
2. settings.xml 里的镜像（mirror）→ 配了阿里云就走阿里云
3. pom.xml 里声明的仓库（repositories）→ 私服等
4. 中央仓库          → 最后兜底
```

> 💡 **提示：** 理解这个顺序很关键。配了镜像后，**所有对被镜像仓库的访问都会被拦截重定向**到镜像地址，这就是镜像能加速的原理。

### 四、镜像（Mirror）配置（加速必备）

#### 1. 为什么用镜像

中央仓库在国外，国内下载经常超时。用国内镜像（阿里云、华为云）可以**大幅提速**。

#### 2. mirror 的工作原理

mirror 不是「多加一个仓库」，而是「**替换**」。`mirrorOf=central` 表示：**凡是本来要访问 central 中央仓库的请求，全部改道去镜像**。

```
你要访问 central ──镜像拦截──► 改去 aliyun
```

#### 3. 配置阿里云镜像

编辑 `settings.xml`（`~/.m2/settings.xml` 或 `MAVEN_HOME/conf/settings.xml`）：

```xml
<settings>
    ...
    <mirrors>
        <mirror>
            <id>aliyun-public</id>
            <mirrorOf>*</mirrorOf>   <!-- 拦截所有仓库 -->
            <name>阿里云公共仓库</name>
            <url>https://maven.aliyun.com/repository/public</url>
        </mirror>
    </mirrors>
</settings>
```

#### 4. `mirrorOf` 的取值（关键）

| `mirrorOf` 值 | 含义 |
| --- | --- |
| `*` | 拦截**所有**仓库请求（最常用、最省心） |
| `central` | 只拦截中央仓库 |
| `external:http:*` | 拦截所有非本地、用 http 的仓库 |
| `repo1,repo2` | 拦截 id 为 repo1 和 repo2 的仓库 |
| `!repo1,*` | 除了 repo1，其余都拦截 |

> ⚠️ **注意：** 公司里有私服时，`mirrorOf=*` 会**连私服请求也拦截**，导致拉不到公司私有 jar。这种情况要写 `mirrorOf=central` 或 `mirrorOf=external:*`，**避开私服**。

#### 5. 常用国内镜像地址

| 镜像 | 地址 |
| --- | --- |
| 阿里云公共 | `https://maven.aliyun.com/repository/public` |
| 阿里云中央 | `https://maven.aliyun.com/repository/central` |
| 华为云 | `https://mirrors.huaweicloud.com/repository/maven/public` |
| 腾讯云 | `https://mirrors.cloud.tencent.com/nexus/repository/maven-public/` |

### 五、在 pom.xml 中声明仓库

除了用 settings.xml 的镜像，也可以在单个项目的 `pom.xml` 里声明额外的仓库：

```xml
<repositories>
    <repository>
        <id>my-repo</id>
        <name>公司私服</name>
        <url>http://nexus.company.com/repository/maven-public/</url>
        <releases>
            <enabled>true</enabled>             <!-- 允许下载 release 版 -->
        </releases>
        <snapshots>
            <enabled>true</enabled>             <!-- 允许下载 snapshot 版 -->
            <updatePolicy>always</updatePolicy> <!-- 每次都检查更新 -->
        </snapshots>
    </repository>
</repositories>
```

| 配置项 | 说明 |
| --- | --- |
| `id` | 仓库唯一标识 |
| `url` | 仓库地址 |
| `releases.enabled` | 是否下载 release 版 |
| `snapshots.enabled` | 是否下载 snapshot 版 |
| `snapshots.updatePolicy` | snapshot 更新策略：`always`/`daily`/`never` |

详见 [[pom配置/repositories配置]]。

> 💡 **提示：** 仓库声明**优先在 settings.xml 或父 pom 里配**，避免每个子项目重复写。

### 六、发布到远程仓库（deploy）

打好的 jar 要发到私服让团队用，用 `mvn deploy`，配合 `distributionManagement`：

```xml
<distributionManagement>
    <!-- release 版发到这个仓库 -->
    <repository>
        <id>company-releases</id>
        <url>http://nexus.company.com/repository/maven-releases/</url>
    </repository>
    <!-- snapshot 版发到这个仓库 -->
    <snapshotRepository>
        <id>company-snapshots</id>
        <url>http://nexus.company.com/repository/maven-snapshots/</url>
    </snapshotRepository>
</distributionManagement>
```

私服需要认证，在 `settings.xml` 配账号密码（`id` 要和上面对应）：

```xml
<servers>
    <server>
        <id>company-releases</id>      <!-- 必须和 pom 里的 id 一致 -->
        <username>admin</username>
        <password>123456</password>
    </server>
    <server>
        <id>company-snapshots</id>
        <username>admin</username>
        <password>123456</password>
    </server>
</servers>
```

详见 [[pom配置/repositories配置]] 和 [[settings配置]]。

### 七、私服（Nexus）简介

#### 1. Nexus 的几种仓库类型

| 类型 | 说明 |
| --- | --- |
| **hosted** | 宿主仓库，存放公司自己上传的 jar |
| **proxy** | 代理仓库，代理中央仓库等外部仓库，自动缓存 |
| **group** | 组仓库，把多个仓库组合成一个地址对外提供 |

> 💡 **提示：** 实际开发中，团队一般访问一个 **group 仓库**（里面包含了 hosted + proxy），这样一个地址既能下公司私有 jar，又能下中央仓库的 jar。

#### 2. 常用 Nexus 操作

- **上传 jar**：`mvn deploy`（通过 `distributionManagement`）
- **手动上传**：Nexus 网页界面可手动上传第三方 jar
- **下载 jar**：在 `pom.xml` 或 `settings.xml` 配置 group 仓库地址

### 八、常见问题与注意事项

#### 1. 下载卡住 / 连接超时

- 配国内镜像（阿里云）
- 检查网络、代理设置
- 公司网络可能有防火墙，需配代理（`<proxies>`）

#### 2. 找不到 jar（`Could not resolve dependencies`）

- 坐标写错（groupId / artifactId / version）
- 该版本真的不存在，去 [MVNRepository](https://mvnrepository.com/) 核对
- 私服里没有，且没配对应的仓库

#### 3. 镜像配了还是慢

- 检查 `settings.xml` 是不是 IDEA 实际读取的那个（`User settings file`）
- `mirrorOf` 写错（比如配成 `central`，但你项目用的是别的仓库 id）
- 加 `-U` 强制更新

#### 4. SNAPSHOT 不更新

Maven 默认**每天**检查一次 SNAPSHOT 更新。想立即更新，加 `-U`：

```bash
mvn clean install -U   # 强制检查所有 SNAPSHOT 更新
```

#### 5. 本地仓库 jar 损坏

报 `invalid LOC header` 之类错误，说明 jar 下载不完整。删除本地仓库里该 jar 的目录，重新下载。

#### 6. 公司私服 jar 拉不到

检查 `mirrorOf` 是不是写成了 `*` 把私服也拦截了。改成 `central` 或排除私服。

### 九、总结

| 要点 | 说明 |
| --- | --- |
| 三种仓库 | 本地（`.m2`）/ 私服（Nexus）/ 中央仓库 |
| 查找顺序 | 本地 → 镜像 → pom 仓库 → 中央仓库 |
| 镜像 mirror | 「替换」而非「新增」，`mirrorOf=*` 拦截所有 |
| 加速下载 | 配阿里云镜像 |
| 私服作用 | 加速、存私有 jar、统一管理 |
| 发布到私服 | `distributionManagement` + `mvn deploy` |
