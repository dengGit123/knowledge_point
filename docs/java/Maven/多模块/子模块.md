### 一、概述

> 📖 [Project Inheritance（项目继承）](https://maven.apache.org/guides/introduction/introduction-to-the-pom.html#Project_Inheritance) ｜ [The parent elements](https://maven.apache.org/ref/current/maven-model/maven.mdo.html#class_parent)

**子模块**通过 `<parent>` 标签声明「**我继承自哪个父 pom**」，从而**自动复用**父 pom 的所有共享配置（版本号、依赖标准、插件配置等）。本篇讲子模块怎么写、继承了什么、`relativePath` 的作用。

大白话：子模块用 `<parent>` **认父亲**——「我爹是 my-app-parent」。认了之后，爹的「家训」（版本号、配置）自动算我头上，我不用重复写。

| 你将学到 | 说明 |
| --- | --- |
| `<parent>` 标签 | 怎么声明父项目 |
| `relativePath` | 本地查找父 pom 的路径 |
| 子模块继承了什么 | properties/dependencyManagement/pluginManagement 等 |
| 子模块能省略什么 | groupId/version（继承父） |

---

### 二、子模块的 `<parent>` 标签

```xml
<!-- 子模块 pom.xml -->
<project>
    <modelVersion>4.0.0</modelVersion>

    <!-- ① 声明父项目 -->
    <parent>
        <groupId>com.example</groupId>
        <artifactId>my-app-parent</artifactId>
        <version>1.0.0</version>
        <relativePath>../pom.xml</relativePath>   <!-- 父 pom 的相对路径 -->
    </parent>

    <!-- ② 子模块自己的坐标（groupId、version 可继承，省略） -->
    <artifactId>my-app-common</artifactId>
    <packaging>jar</packaging>

    <!-- ③ 子模块自己的依赖、配置 -->
</project>
```

#### `<parent>` 的子标签

| 子标签 | 含义 | 是否必须 |
| --- | --- | --- |
| `groupId` | 父项目的 groupId | ✅ 必须 |
| `artifactId` | 父项目的 artifactId | ✅ 必须 |
| `version` | 父项目的 version | ✅ 必须 |
| `relativePath` | 父 pom 在**本地文件系统**的相对路径 | ❌ 可选（但建议写） |

> 💡 **提示：** `parent` 标签里写的是**父项目**的 GAV（不是自己的），这和写依赖很像——本质是「这个 pom 继承自哪个 pom」。

### 三、`relativePath`：本地查找父 pom

`relativePath` 告诉 Maven「**父 pom 在本地文件系统的哪个位置**」，这样不用去仓库下载就能找到。

```xml
<parent>
    <groupId>com.example</groupId>
    <artifactId>my-app-parent</artifactId>
    <version>1.0.0</version>
    <relativePath>../pom.xml</relativePath>   <!-- 父 pom 在上一级目录 -->
</parent>
```

#### `relativePath` 的查找逻辑

```
Maven 找父 pom：
  1. 先看 relativePath 指向的本地路径（如 ../pom.xml）
  2. 找不到，再去本地仓库（~/.m2）找 com/example/my-app-parent/1.0.0
  3. 还找不到，去远程仓库下载
```

> 💡 **提示：** 父 pom 和子模块在**同一个项目目录树**里（聚合项目），`relativePath` 指向本地文件，**构建时不用先 install 父 pom**。

#### `relativePath` 不写或写空

```xml
<relativePath/>   <!-- 空值：不在本地找，直接去仓库找 -->
```

不写 `relativePath` 时，默认值是 `../pom.xml`（Maven 会默认找上一级）。

#### 何时不用 `relativePath`

当父 pom 是**远程**的（如 `spring-boot-starter-parent`），父 pom 不在你本地目录里，靠仓库下载：

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
    <relativePath/>   <!-- 空值，从仓库找 -->
</parent>
```

### 四、子模块继承了父 pom 的什么

子模块**自动继承**父 pom 里这些配置（无需重复写）：

| 继承的内容 | 说明 |
| --- | --- |
| `groupId` | 子模块可省略自己的 groupId |
| `version` | 子模块可省略自己的 version |
| `properties` | 子模块可直接用 `${变量}` |
| `dependencyManagement` | 子模块的依赖可省略 version |
| `pluginManagement` | 子模块的插件可省略 version 和配置 |
| `repositories` | 子模块自动用父配置的仓库 |
| `distributionManagement` | 子模块 deploy 时用父的发布配置 |
| `dependencies` | **也会继承**（所以父 pom 别乱堆依赖） |
| `build`（部分） | 插件、资源等配置 |

> ⚠️ **注意：** 父 pom 的 `dependencies`（不是 `dependencyManagement`）**会被子模块直接继承引入**。所以父 pom 要慎用直接 `dependencies`，避免所有子模块被迫引入一堆不需要的 jar。版本标准应该用 `dependencyManagement`。

### 五、子模块能省略的配置

继承父 pom 后，子模块能省：

#### 1. 省略 `groupId` 和 `version`（继承父）

```xml
<project>
    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>com.example</groupId>
        <artifactId>my-app-parent</artifactId>
        <version>1.0.0</version>          <!-- version 继承 -->
    </parent>

    <artifactId>my-app-common</artifactId>  <!-- 只写 artifactId -->
    <!-- groupId、version 都省略，继承父的 -->
</project>
```

> 💡 **提示：** `artifactId` **不能省**——每个子模块要有自己的唯一名字。但 `groupId`、`version` 可以省（继承父）。

#### 2. 依赖省略 `version`（继承 dependencyManagement）

```xml
<!-- 父 pom 的 dependencyManagement 已定义 spring-core 的版本 -->
<!-- 子模块引入时省略 version -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-core</artifactId>
    <!-- 不写 version，继承父 pom dependencyManagement 的 -->
</dependency>
```

#### 3. 插件省略 `version`（继承 pluginManagement）

```xml
<!-- 父 pom 的 pluginManagement 已定义 compiler 版本 -->
<!-- 子模块激活时省略 version 和 configuration -->
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <!-- 不写 version 和 configuration，继承父的 -->
        </plugin>
    </plugins>
</build>
```

### 六、子模块的完整示例

```xml
<!-- my-app-service/pom.xml：一个典型的子模块 -->
<project>
    <modelVersion>4.0.0</modelVersion>

    <!-- 继承父 pom -->
    <parent>
        <groupId>com.example</groupId>
        <artifactId>my-app-parent</artifactId>
        <version>1.0.0</version>
        <relativePath>../pom.xml</relativePath>
    </parent>

    <!-- 本模块坐标（省略 groupId、version） -->
    <artifactId>my-app-service</artifactId>
    <packaging>jar</packaging>

    <!-- 本模块依赖（依赖 common 模块 + 第三方） -->
    <dependencies>
        <!-- 依赖同项目的 common 模块 -->
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>my-app-common</artifactId>
            <!-- version 继承父 dependencyManagement -->
        </dependency>

        <!-- 第三方依赖（version 继承父） -->
        <dependency>
            <groupId>org.springframework</groupId>
            <artifactId>spring-context</artifactId>
        </dependency>
    </dependencies>
</project>
```

### 七、模块间依赖（子模块互相引用）

多模块项目里，子模块之间通过**坐标**互相依赖（和依赖第三方 jar 一样）：

```xml
<!-- my-app-service 依赖 my-app-common -->
<dependency>
    <groupId>com.example</groupId>
    <artifactId>my-app-common</artifactId>
    <version>${project.version}</version>   <!-- 版本随父走 -->
</dependency>
```

> 💡 **提示：** 模块间依赖的 version，最佳实践是在父 pom 的 `dependencyManagement` 里声明 `${project.version}`，子模块依赖时省略 version。这样整个项目版本统一升降。

```xml
<!-- 父 pom -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>my-app-common</artifactId>
            <version>${project.version}</version>   <!-- 所有模块统一版本 -->
        </dependency>
    </dependencies>
</dependencyManagement>
```

### 八、子模块是否必须配置

| 场景 | 是否要写 `<parent>` |
| --- | --- |
| 多模块项目的子模块 | ✅ 必须（继承共享配置） |
| 单模块独立项目 | ❌ 不写（或继承 spring-boot-starter-parent） |
| Spring Boot 项目 | ✅ 继承 `spring-boot-starter-parent` |

### 九、常见问题与注意事项

#### 1. 找不到父 pom（`Non-resolvable parent POM`）

- `relativePath` 路径写错（核对相对父 pom 的位置）
- 父 pom 没 `mvn install` 到本地仓库（远程继承时）
- 父 pom 坐标写错

#### 2. 子模块 `groupId`/`version` 省略后报错

确认 `<parent>` 配置正确。省略的前提是能从父 pom 继承。

#### 3. 子模块继承了一堆不需要的依赖

父 pom 用了直接 `dependencies`（而非 `dependencyManagement`）。把版本标准移到 `dependencyManagement`，子模块按需引入。

#### 4. `relativePath` 该不该写

- 父 pom 在**本地目录**（聚合项目）：写，指向 `../pom.xml`
- 父 pom 在**远程仓库**（如 Spring Boot parent）：写空 `<relativePath/>`

#### 5. 子模块覆盖父的属性/版本

子模块重写同名配置即可（子优先于父）。但不推荐——破坏统一管理。

### 十、总结

| 要点 | 说明 |
| --- | --- |
| 子模块 | 通过 `<parent>` 继承父 pom |
| `<parent>` 内容 | 父项目 GAV + relativePath |
| `relativePath` | 父 pom 本地路径，本地有就写，远程就空 |
| 继承什么 | groupId/version/properties/dependencyManagement/pluginManagement 等 |
| 可省略 | groupId、version、依赖和插件的 version |
| 模块间依赖 | 用坐标互相引用，version 走父 dependencyManagement |
