前面几篇分别讲了父模块、子模块、聚合与继承。本篇是**实战收尾**——从零搭建一个完整的「**聚合 + 继承**」多模块项目，给出每一份 pom.xml 的完整内容，并列出所有注意事项。

大白话：把前面学的零件**组装起来**，造一个真正能跑的多模块项目。以经典的「**common（公共）→ service（业务）→ web（启动）**」三层结构为例。

### 二、项目结构设计

#### 目标：一个分层架构的多模块项目

```
my-app/                          ← 根目录（父项目）
├── pom.xml                      ← 父 pom（聚合 + 继承）
│
├── my-app-common/               ← 公共模块（工具类、实体、常量）
│   ├── pom.xml                  ← jar
│   └── src/main/java/...
│
├── my-app-service/              ← 业务模块（依赖 common）
│   ├── pom.xml                  ← jar
│   └── src/main/java/...
│
└── my-app-web/                  ← 启动模块（依赖 service，对外提供接口）
    ├── pom.xml                  ← jar（或 war）
    └── src/main/java/...
```

#### 模块依赖关系

```
my-app-web  ──依赖──►  my-app-service  ──依赖──►  my-app-common
（启动入口）            （业务逻辑）              （公共工具/实体）
```

> 💡 **提示：** 依赖方向是「**上层依赖下层**」。`common` 是最底层的公共模块，谁都能依赖它，但它不依赖别人。

### 三、第一步：父 pom（`my-app/pom.xml`）

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- ① 父项目坐标 -->
    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0.0</version>

    <!-- ② 父项目必须 pom -->
    <packaging>pom</packaging>

    <!-- ③ 聚合：声明子模块 -->
    <modules>
        <module>my-app-common</module>
        <module>my-app-service</module>
        <module>my-app-web</module>
    </modules>

    <!-- ④ 共享属性 -->
    <properties>
        <maven.compiler.source>17</maven.compiler.source>
        <maven.compiler.target>17</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>

        <!-- 第三方版本号集中管理 -->
        <spring.version>6.1.5</spring.version>
        <junit.version>4.13.2</junit.version>
        <lombok.version>1.18.30</lombok.version>
    </properties>

    <!-- ⑤ 依赖版本管理（子模块省略 version） -->
    <dependencyManagement>
        <dependencies>
            <!-- 模块间依赖：版本随项目走 -->
            <dependency>
                <groupId>com.example</groupId>
                <artifactId>my-app-common</artifactId>
                <version>${project.version}</version>
            </dependency>
            <dependency>
                <groupId>com.example</groupId>
                <artifactId>my-app-service</artifactId>
                <version>${project.version}</version>
            </dependency>

            <!-- 第三方依赖版本标准 -->
            <dependency>
                <groupId>org.springframework</groupId>
                <artifactId>spring-context</artifactId>
                <version>${spring.version}</version>
            </dependency>
            <dependency>
                <groupId>org.projectlombok</groupId>
                <artifactId>lombok</artifactId>
                <version>${lombok.version}</version>
                <scope>provided</scope>
            </dependency>
            <dependency>
                <groupId>junit</groupId>
                <artifactId>junit</artifactId>
                <version>${junit.version}</version>
                <scope>test</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>

    <!-- ⑥ 插件版本管理 -->
    <build>
        <pluginManagement>
            <plugins>
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-compiler-plugin</artifactId>
                    <version>3.13.0</version>
                    <configuration>
                        <source>17</source>
                        <target>17</target>
                        <encoding>UTF-8</encoding>
                    </configuration>
                </plugin>
            </plugins>
        </pluginManagement>
    </build>

</project>
```

#### 父 pom 要点

- `packaging=pom`
- `modules` 聚合三个子模块
- `properties` 集中版本号
- `dependencyManagement` 声明所有依赖版本（含模块间依赖用 `${project.version}`）
- `pluginManagement` 统一插件版本

### 四、第二步：common 子模块（`my-app/my-app-common/pom.xml`）

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- 继承父 pom -->
    <parent>
        <groupId>com.example</groupId>
        <artifactId>my-app</artifactId>
        <version>1.0.0</version>
        <relativePath>../pom.xml</relativePath>
    </parent>

    <!-- 本模块坐标（省略 groupId、version，继承父） -->
    <artifactId>my-app-common</artifactId>
    <packaging>jar</packaging>

    <!-- 本模块依赖（version 继承父 dependencyManagement） -->
    <dependencies>
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <!-- version、scope 继承父 -->
        </dependency>

        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <!-- version、scope 继承父 -->
        </dependency>
    </dependencies>

</project>
```

### 五、第三步：service 子模块（依赖 common）

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- 继承父 pom -->
    <parent>
        <groupId>com.example</groupId>
        <artifactId>my-app</artifactId>
        <version>1.0.0</version>
        <relativePath>../pom.xml</relativePath>
    </parent>

    <artifactId>my-app-service</artifactId>
    <packaging>jar</packaging>

    <dependencies>
        <!-- 依赖同项目的 common 模块（version 继承父） -->
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>my-app-common</artifactId>
        </dependency>

        <!-- 第三方依赖 -->
        <dependency>
            <groupId>org.springframework</groupId>
            <artifactId>spring-context</artifactId>
            <!-- version 继承父 -->
        </dependency>

        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
        </dependency>

        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
        </dependency>
    </dependencies>

</project>
```

> 💡 **提示：** `my-app-service` 依赖 `my-app-common`，这样 service 就能用 common 里的工具类和实体了。

### 六、第四步：web 子模块（依赖 service，启动入口）

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- 继承父 pom -->
    <parent>
        <groupId>com.example</groupId>
        <artifactId>my-app</artifactId>
        <version>1.0.0</version>
        <relativePath>../pom.xml</relativePath>
    </parent>

    <artifactId>my-app-web</artifactId>
    <packaging>jar</packaging>

    <dependencies>
        <!-- 依赖 service 模块（会传递依赖 common） -->
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>my-app-service</artifactId>
        </dependency>

        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
        </dependency>
    </dependencies>

    <build>
        <!-- 打成可执行 jar：指定主类 -->
        <finalName>my-app</finalName>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-shade-plugin</artifactId>
                <version>3.5.2</version>
                <executions>
                    <execution>
                        <phase>package</phase>
                        <goals><goal>shade</goal></goals>
                        <configuration>
                            <transformers>
                                <transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">
                                    <mainClass>com.example.web.Application</mainClass>
                                </transformer>
                            </transformers>
                        </configuration>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>

</project>
```

### 七、构建与运行

#### 在根目录一键构建

```bash
cd my-app                # 进入父项目根目录
mvn clean install        # 一键构建所有模块（反应堆自动按依赖排序）
```

构建顺序（反应堆自动决定）：

```
1. my-app (父 pom)
2. my-app-common   ← 最底层，先建
3. my-app-service  ← 依赖 common
4. my-app-web      ← 依赖 service，最后建
```

#### 部分构建

```bash
# 只构建 service 模块
mvn clean install -pl my-app-service

# 构建 service 及它依赖的 common（-am = also make）
mvn clean install -pl my-app-service -am

# 构建 web 及依赖它的所有模块（-amd = also make dependents）
mvn clean install -pl my-app-common -amd
```

#### 运行 web 模块

```bash
java -jar my-app-web/target/my-app.jar
```

### 九、注意事项大汇总（重点）

#### 1. 父 pom 必须 `packaging=pom`

父项目只做配置和聚合，不产出 jar。忘记配会报错。

#### 2. 父 pom 别堆 `dependencies`（用 `dependencyManagement`）

父 pom 的 `dependencies` 会被所有子模块**强制继承**。版本标准用 `dependencyManagement`，子模块按需引入。

#### 3. 模块间依赖用 `${project.version}`

在父 pom 的 `dependencyManagement` 里声明模块间依赖版本为 `${project.version}`，保证所有模块版本统一。升级时改父 pom 的 version 一处即可。

#### 4. 注意依赖方向，避免循环

```
✅ web → service → common（单向，正确）
❌ common → service → common（循环，Maven 报错）
```

#### 5. 子模块省略 groupId/version 的前提

能继承父 pom 时才能省。`artifactId` 永远不能省（每个子模块要有唯一名字）。

#### 6. `relativePath` 要写对

聚合项目里子模块的 `<parent><relativePath>` 指向本地父 pom（通常是 `../pom.xml`），这样构建时不用先 install 父 pom。

#### 7. 改了父 pom 要刷新

父 pom 改动后，IDEA 里 `Reload All Maven Projects`，否则子模块可能用的是旧配置。

#### 8. 构建顺序交给反应堆

不用手动指定先 build 哪个模块。反应堆自动按依赖关系排序。如果顺序「不对」，通常是依赖声明有问题。

#### 9. 只改一个模块别全量构建

用 `-pl` + `-am` 精准构建，节省时间：

```bash
mvn install -pl my-app-service -am   # 只构建 service 和它依赖的 common
```

#### 10. SNAPSHOT 联调，Release 发布

- 团队内部联调：所有模块版本用 `1.0.0-SNAPSHOT`
- 正式发布：改成 `1.0.0`（Release），发布到 release 仓库

### 十、常见问题排查

| 报错 | 原因 | 解决 |
| --- | --- | --- |
| `Non-resolvable parent POM` | 子模块找不到父 pom | 检查 `<parent>` 坐标和 `relativePath` |
| `Could not find artifact com.example:my-app-common` | service 依赖 common，但 common 没先构建 | 在根目录用 `mvn install`，或加 `-am` |
| `Cycle in the dependency graph` | 模块循环依赖 | 重新设计依赖方向 |
| 父 pom 配置子模块没生效 | 没 Reload | IDEA 刷新 Maven |
| `mvn package` 报「packaging=pom 不能打包」 | 把父 pom 当成普通项目构建 | 在根目录构建，或子模块单独构建 |

### 十一、总结

| 要点 | 说明 |
| --- | --- |
| 结构 | 父 pom（聚合+继承） + 多个子模块 |
| 父 pom 核心 | `packaging=pom` + `modules` + `properties` + `dependencyManagement` + `pluginManagement` |
| 子 pom 核心 | `<parent>` + 自己的 `artifactId` + `dependencies` |
| 模块间依赖 | 用坐标引用，version 走父 `dependencyManagement` |
| 构建 | 根目录 `mvn clean install` 一键全建 |
| 依赖方向 | 单向，上层依赖下层，避免循环 |
