# 反射（Reflection）

> 官方文档：[Java Reflection API (Oracle)](https://docs.oracle.com/javase/tutorial/reflect/)、[菜鸟教程 Java 反射](https://www.runoob.com/java/java-reflection.html)

## 一、概述

**反射（Reflection）** 是 Java 提供的一种机制，它允许程序在**运行期间**动态地获取任意一个类的内部信息（类名、属性、方法、构造器、注解等），并且能够在运行时动态地创建对象、调用方法、读写字段。

正常情况下，Java 程序在**编译期**就已经确定了要操作哪个类、调用哪个方法。而反射打破了这个限制——它让你在程序运行起来之后，依然可以"临时决定"要操作哪个类、调用哪个方法。

> 💡 **提示：** 类比：反射就像医院的"X 光透视"。平时你只能看到一个对象"外表"（公开的方法和属性），而反射能让你看清对象内部的"骨骼结构"（包括 private 私有的部分），甚至还能"动手"去操作这些隐藏的结构。

### 为什么需要反射？

```java
// ❌ 不用反射：编译期就写死了要用哪个类，换实现必须改代码重新编译
User user = new User();
user.sayHello();

// ✅ 用反射：运行时根据字符串/配置文件决定加载哪个类，无需改代码
String className = config.getProperty("user.class"); // 例如 "com.example.User"
Class<?> clazz = Class.forName(className);
Object obj = clazz.getDeclaredConstructor().newInstance();
```

反射的核心价值：
1. **动态性**：运行时才决定加载哪个类、调用哪个方法，编译期无需写死。
2. **解耦**：调用方和被调用方之间通过"字符串/配置"关联，互不依赖，实现松耦合。
3. **框架的基石**：Spring（IoC 创建 Bean）、MyBatis（SQL 结果映射）、Jackson（JSON 序列化）、JUnit（动态调用测试方法）等几乎所有主流框架底层都重度依赖反射。

### 反射的整体结构

```
┌─────────────────────────────────────────────────────────────┐
│                        运行中的 JVM                          │
│                                                             │
│   方法区 / 元空间 中存放着每个类对应的 Class 对象              │
│                                                             │
│   ┌──────────────┐   1个类 = 1个 Class对象（全局唯一）        │
│   │ Class 对象   │ ◄──────── 反射操作的"入口"                │
│   └──────┬───────┘                                          │
│          │ 包含该类的全部元信息                               │
│          ▼                                                  │
│   ┌────────────────────────────────────────────────────┐   │
│   │  Constructor[]  构造器集合                          │   │
│   │  Field[]        成员变量集合                        │   │
│   │  Method[]       方法集合                           │   │
│   │  Annotation[]   注解集合                           │   │
│   └────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

## 二、获取 Class 对象的三种方式（重点）

反射的第一步永远是先拿到 `Class` 对象。`Class` 类是反射的"总入口"，每一个被加载的类在 JVM 中都有且只有一个对应的 `Class` 对象。获取 `Class` 对象有三种常用方式：

### 1. `类名.class`

当你在**编译期就知道**要操作哪个类时使用。这种方式最简洁，也不会触发类的初始化（只是拿到 Class 对象）。

```java
Class<String> clazz = String.class;      // String 类的 Class 对象
Class<Integer> intClazz = int.class;     // 基本类型也有 Class 对象
Class<int[]> arrayClazz = int[].class;   // 数组类型也有
```

### 2. `对象.getClass()`

当你**已经有一个对象**，但想获取它的真实类型时使用。`getClass()` 是 `Object` 类的方法，所有对象都有。

```java
String s = "hello";
Class<?> clazz = s.getClass();   // 返回 String.class
```

### 3. `Class.forName("全限定类名")`

当类的名字在**配置文件 / 字符串**里，需要动态加载时使用。这是最灵活也是最常用的方式，但**会触发类的初始化**（执行 static 代码块和静态变量的赋值）。

```java
// 全限定类名 = 包名 + 类名
Class<?> clazz = Class.forName("java.lang.String");
```

### 三种方式对比

| 方式 | 语法 | 何时使用 | 是否需要类已加载 | 是否触发初始化 | 适用场景 |
|------|------|---------|----------------|---------------|---------|
| `类名.class` | `String.class` | 编译期已知类型 | 否 | 否 | 知道具体类型，常作参数传递 |
| `对象.getClass()` | `obj.getClass()` | 已有对象实例 | 是（对象已创建） | 否（对象创建时已初始化） | 运行时拿到对象后想了解其类型 |
| `Class.forName(...)` | `Class.forName("java.lang.String")` | 只有类名字符串 | 否 | **是** | 配置文件、动态加载、框架核心 |

> ⚠️ **注意：** `Class.forName` 会抛出受检异常 `ClassNotFoundException`，必须 try-catch 或 throws。它触发初始化这一特性，也是 JDBC 中加载驱动 `Class.forName("com.mysql.cj.jdbc.Driver")` 的原理——驱动类的 static 块会把自己注册到 DriverManager。

### 验证：同一个类的 Class 对象全局唯一

无论用哪种方式获取，同一个类加载器加载的同一个类，得到的 `Class` 对象是**同一个**。

```java
public class ClassSingleton {
    public static void main(String[] args) throws ClassNotFoundException {
        Class<?> c1 = String.class;
        Class<?> c2 = "hello".getClass();
        Class<?> c3 = Class.forName("java.lang.String");

        System.out.println(c1 == c2); // true
        System.out.println(c2 == c3); // true —— 全局唯一，地址相同
    }
}
```

## 三、通过反射获取类的结构

拿到 `Class` 对象后，可以查询类的各种内部结构。下面用一个目标类 `User` 作为示例贯穿本章。

```java
package com.example;

public class User {
    public String name;          // public 字段
    private int age;             // private 字段
    private String secret = "原始密钥"; // private 字段，带初值

    // 无参构造器
    public User() {}

    // 有参构造器
    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // public 方法
    public void publicHello() {
        System.out.println("public hello, name=" + name);
    }

    // private 方法
    private String privateHello(String msg) {
        return "private hello: " + msg;
    }
}
```

### 1. 获取类名

| 方法 | 返回内容 | 示例（com.example.User） |
|------|---------|----------------------|
| `getName()` | 全限定类名 | `com.example.User` |
| `getSimpleName()` | 简单类名 | `User` |
| `getCanonicalName()` | 规范名（对数组和内部类更友好） | `com.example.User` |

```java
Class<?> clazz = Class.forName("com.example.User");
System.out.println(clazz.getName());        // com.example.User
System.out.println(clazz.getSimpleName()); // User
```

### 2. 获取构造器

| 方法 | 含义 |
|------|------|
| `getConstructors()` | 所有 **public** 构造器 |
| `getDeclaredConstructors()` | 所有声明的构造器（含 private/protected/默认） |
| `getConstructor(Class<?>... paramTypes)` | 指定参数的 public 构造器 |
| `getDeclaredConstructor(Class<?>... paramTypes)` | 指定参数的构造器（含 private） |

### 3. 获取成员变量（Field）

| 方法 | 含义 |
|------|------|
| `getFields()` | 所有 **public** 字段（**含继承自父类的 public 字段**） |
| `getDeclaredFields()` | 本类声明的所有字段（含 private，**不含继承**） |
| `getField(String name)` | 指定名称的 public 字段 |
| `getDeclaredField(String name)` | 本类声明的指定字段（含 private） |

### 4. 获取方法（Method）

| 方法 | 含义 |
|------|------|
| `getMethods()` | 所有 **public** 方法（**含继承自父类/接口的 public 方法**） |
| `getDeclaredMethods()` | 本类声明的所有方法（含 private，**不含继承**） |
| `getMethod(String name, Class<?>... paramTypes)` | 指定名称和参数的 public 方法 |
| `getDeclaredMethod(String name, Class<?>... paramTypes)` | 本类声明的指定方法（含 private） |

### 5. 重点：`getXxx` vs `getDeclaredXxx` 的区别

这是反射面试最高频考点之一，务必牢记：

| 对比项 | `getXxx`（如 getMethods） | `getDeclaredXxx`（如 getDeclaredMethods） |
|--------|-------------------------|------------------------------------------|
| 访问范围 | 只含 **public** | 含 **所有**（public/protected/默认/private） |
| 是否包含继承成员 | **包含**父类/接口的 public 成员 | **不包含**，只有本类自己声明的 |
| 访问 private | 不能直接拿到 | 能拿到，但调用前需 `setAccessible(true)` |

> 💡 **提示：** 一句话记忆——`getDeclared` 拿的是"**本类声明的全部**（含私有不含继承）"；不带 Declared 拿的是"**所有能见的 public**（含继承）"。

```java
// 演示 getMethods 包含继承来的方法（Object 的 toString/hashCode 等）
Class<?> clazz = User.class;
Method[] methods = clazz.getMethods();
// methods 中除了 publicHello，还有从 Object 继承的 toString、hashCode、equals、getClass 等

Method[] declared = clazz.getDeclaredMethods();
// declared 中只有 publicHello 和 privateHello（本类声明的）
```

## 四、反射操作（重点）

获取结构只是"读"，反射更强大的地方在于"写"——动态创建对象、调用方法、修改字段。

### 1. 创建对象

```java
import java.lang.reflect.Constructor;

Class<?> clazz = Class.forName("com.example.User");

// ✅ 方式一：用无参构造器（Java 9+ 推荐写法）
Object obj1 = clazz.getDeclaredConstructor().newInstance();

// ✅ 方式二：用有参构造器
Constructor<?> con = clazz.getDeclaredConstructor(String.class, int.class);
Object obj2 = con.newInstance("张三", 20);

// ❌ 方式三（已废弃）：clazz.newInstance()
// Object obj3 = clazz.newInstance(); // Java 9 起 @Deprecated，只能调用无参 public 构造器
```

> ⚠️ **注意：** `clazz.newInstance()` 从 Java 9 开始被废弃，因为它绕过了编译时的异常检查，且只能调用 public 无参构造器。官方推荐用 `getDeclaredConstructor().newInstance()`，它更明确、支持任意构造器、错误处理更清晰。

### 2. 调用方法

`Method.invoke(Object obj, Object... args)` 用于调用方法：第一个参数是目标对象（静态方法传 `null`），后面是实参。

```java
import java.lang.reflect.Method;

Class<?> clazz = Class.forName("com.example.User");
Object obj = clazz.getDeclaredConstructor().newInstance();

// 调用 public 方法
Method m1 = clazz.getMethod("publicHello");
m1.invoke(obj); // 输出: public hello, name=null

// 调用 private 方法：必须先 setAccessible(true)
Method m2 = clazz.getDeclaredMethod("privateHello", String.class);
m2.setAccessible(true); // ✅ 突破 private 访问检查
String result = (String) m2.invoke(obj, "反射调用");
System.out.println(result); // 输出: private hello: 反射调用
```

### 3. 访问/修改成员变量

```java
import java.lang.reflect.Field;

Class<?> clazz = Class.forName("com.example.User");
Object obj = clazz.getDeclaredConstructor(String.class, int.class)
                  .newInstance("李四", 30);

// 访问 public 字段
Field nameField = clazz.getField("name");
System.out.println(nameField.get(obj)); // 李四

// 访问并修改 private 字段
Field ageField = clazz.getDeclaredField("age");
ageField.setAccessible(true);              // ✅ 突破 private
System.out.println(ageField.getInt(obj));  // 30
ageField.set(obj, 99);                     // 修改为 99
System.out.println(ageField.getInt(obj));  // 99
```

### 4. `setAccessible(true)` 突破访问检查

`setAccessible(true)` 是反射中非常重要的方法，它**关闭 Java 的访问权限检查**，让 private 成员也能被访问。`Field`、`Method`、`Constructor` 都继承自 `AccessibleObject`，都有此方法。

```
AccessibleObject（抽象基类，提供 setAccessible）
   ▲
   ├── Field
   ├── Method
   └── Constructor
```

## 五、完整综合示例

下面是一个完整的可运行示例，串起"获取结构 → 创建对象 → 调用私有方法 → 修改私有字段"全过程。

```java
package com.example;

import java.lang.reflect.*;

public class ReflectionDemo {
    public static void main(String[] args) throws Exception {
        // 1. 获取 Class 对象
        Class<?> clazz = Class.forName("com.example.User");

        // 2. 打印类的所有字段（含 private）
        System.out.println("===== 字段列表 =====");
        for (Field f : clazz.getDeclaredFields()) {
            System.out.println(Modifier.toString(f.getModifiers()) + " "
                    + f.getType().getSimpleName() + " " + f.getName());
        }

        // 3. 打印类的所有方法
        System.out.println("===== 方法列表 =====");
        for (Method m : clazz.getDeclaredMethods()) {
            System.out.println(Modifier.toString(m.getModifiers()) + " "
                    + m.getReturnType().getSimpleName() + " " + m.getName());
        }

        // 4. 用无参构造器创建对象
        Object user = clazz.getDeclaredConstructor().newInstance();

        // 5. 修改 private 字段 secret
        Field secretField = clazz.getDeclaredField("secret");
        secretField.setAccessible(true);                       // ✅ 突破 private
        System.out.println("修改前: " + secretField.get(user)); // 原始密钥
        secretField.set(user, "已被反射篡改");
        System.out.println("修改后: " + secretField.get(user)); // 已被反射篡改

        // 6. 调用 private 方法 privateHello
        Method m = clazz.getDeclaredMethod("privateHello", String.class);
        m.setAccessible(true);
        String ret = (String) m.invoke(user, "综合示例");
        System.out.println("调用结果: " + ret); // private hello: 综合示例
    }
}
```

运行输出（示意）：

```
===== 字段列表 =====
public String name
private int age
private String secret
===== 方法列表 =====
public void publicHello
private String privateHello
修改前: 原始密钥
修改后: 已被反射篡改
调用结果: private hello: 综合示例
```

## 六、反射的应用场景

### 1. Spring IoC（根据配置创建 Bean）

Spring 容器读取 XML / 注解配置，根据类名字符串用反射创建对象并注入依赖，从而实现"控制反转"。

```java
// 框架内部的简化逻辑
String className = "com.example.UserService";
Class<?> clazz = Class.forName(className);
Object bean = clazz.getDeclaredConstructor().newInstance(); // 反射创建 Bean
```

### 2. 动态代理（`Proxy.newProxyInstance`）

JDK 动态代理基于反射在运行时生成代理类，是 AOP、MyBatis Mapper、Feign 等技术的底层实现。

```java
import java.lang.reflect.*;

// InvocationHandler：方法调用被转发到这里
Object proxy = Proxy.newProxyInstance(
        target.getClass().getClassLoader(),
        target.getClass().getInterfaces(),
        (proxyObj, method, args) -> {
            System.out.println("前置增强");
            Object result = method.invoke(target, args); // 反射调用真实方法
            System.out.println("后置增强");
            return result;
        });
```

### 3. 注解处理（读取运行时注解）

`@Retention(RetentionPolicy.RUNTIME)` 的注解可通过反射在运行时读取，这是 Spring MVC、JUnit、Lombok 等框架的基础。

```java
// 读取方法上的注解
Method method = clazz.getDeclaredMethod("doSomething");
if (method.isAnnotationPresent(MyAnnotation.class)) {
    MyAnnotation anno = method.getAnnotation(MyAnnotation.class);
    System.out.println(anno.value());
}
```

### 4. ORM / JDBC（结果集映射到对象）

MyBatis、Hibernate 把数据库查询结果（ResultSet）通过反射映射成 Java 对象：根据列名找字段、调用 setter。

### 5. JSON 序列化框架

Jackson、Fastjson、Gson 通过反射遍历对象的字段，读取值并转成 JSON 字符串；反序列化时再通过反射创建对象并填充字段。

| 应用场景 | 反射用来做什么 |
|---------|-------------|
| Spring IoC | 根据配置创建 Bean、注入依赖 |
| 动态代理 / AOP | 运行时生成代理类、转发方法调用 |
| 注解处理 | 运行时读取注解元信息 |
| ORM（MyBatis/Hibernate） | 结果集映射成对象 |
| JSON 框架（Jackson/Gson） | 序列化/反序列化时读写字段 |
| JUnit | 劫标 `@Test` 的方法并调用 |

## 七、反射的优缺点

### 优点

1. **灵活、动态**：运行时决定操作哪个类，编译期无需写死。
2. **解耦**：调用方与被调用方通过字符串/配置解耦，是框架实现可插拔的基础。
3. **能力强**：能突破访问权限，访问 private 成员。

### 缺点

1. **性能损耗**：反射调用比直接调用慢（涉及方法查找、参数装箱、安全检查等）。
2. **破坏封装**：能随意访问 private，破坏了面向对象的封装设计。
3. **编译期检查丢失**：方法名写错、参数类型不匹配等，编译期发现不了，要等运行时才报错。
4. **代码可读性差**：反射代码晦涩，不利于维护。

### 性能对比与优化

| 调用方式 | 相对耗时（数量级） | 备注 |
|---------|-----------------|------|
| 直接调用 | 基准（最快） | 编译期确定，JIT 可优化 |
| 反射调用（无缓存） | 慢 10 ~ 50 倍 | 每次都要查找 Method、安全检查 |
| 反射调用（缓存 + setAccessible） | 慢 2 ~ 5 倍 | 已大幅优化 |

性能优化建议：
1. **缓存反射对象**：把 `Class`、`Method`、`Field` 缓存起来复用，避免重复查找。
2. **关闭访问检查**：`setAccessible(true)` 会跳过权限校验，能提升速度。
3. **高频场景用 MethodHandle / Lambda**：Java 7+ 的 `MethodHandle` 或通过 `LambdaMetafactory` 生成 `Function`，性能接近直接调用。

```java
// 优化示例：缓存 Method 对象
private static final Method HELLO;
static {
    try {
        HELLO = User.class.getDeclaredMethod("publicHello");
        HELLO.setAccessible(true); // 关闭访问检查，提速
    } catch (NoSuchMethodException e) {
        throw new RuntimeException(e);
    }
}
// 业务中直接复用 HELLO，不再重复查找
```

## 八、注意事项与陷阱

### 1. `setAccessible` 与安全管理器

`setAccessible(true)` 在默认的 Java 环境中通常能正常工作，但在启用 `SecurityManager`（安全管理器）或模块系统（Java 9+）时，对其他模块的非公开成员调用 `setAccessible` 可能抛出 `InaccessibleObjectException`。

> ⚠️ **注意：** Java 9 引入模块系统后，反射访问 JDK 内部 API（如 `sun.misc.*`）受限。运行时可能需要加 `--add-opens` 参数打开指定包。

### 2. 反射修改 final 字段的局限

`final` 字段并非绝对不能改，但有两种情况改了"也没用"：

- **编译期常量**：如 `static final int X = 10;`，编译器会在用到的地方直接**内联**成字面量，反射改了字段值，但调用处依然是旧值。
- **JIT 内联优化**：`private final` 的简单字段可能被 JIT 内联，反射修改后读到的可能是旧值。

```java
public class FinalTrap {
    private final int value = 100; // 反射可改，但要注意可能的内联

    public static void main(String[] args) throws Exception {
        FinalTrap t = new FinalTrap();
        Field f = FinalTrap.class.getDeclaredField("value");
        f.setAccessible(true);
        f.set(t, 200);
        System.out.println(f.get(t));   // 200（字段值确实变了）
        // 但若 value 在别处被内联，读到仍可能是 100
    }
}
```

> 💡 **提示：** 不要在生产代码中依赖"反射改 final"来做事，它不可靠、不可移植，且语义混乱。

### 3. 反射的性能问题

在**热点路径**（如每秒调用上万次的循环）中尽量避免反射，改用直接调用或预编译的 `Function`/`Supplier`。反射更适合"低频、一次性的初始化"场景。

### 4. 异常处理

反射相关的 API 会抛出多种异常，注意区分：

| 异常 | 触发场景 |
|------|---------|
| `ClassNotFoundException` | `Class.forName` 找不到类 |
| `NoSuchMethodException` | 找不到指定方法 |
| `NoSuchFieldException` | 找不到指定字段 |
| `IllegalAccessException` | 访问 private 未 `setAccessible` |
| `InvocationTargetException` | 被调方法内部抛了异常（真实异常藏在 `getCause()` 里） |
| `InstantiationException` | 抽象类/接口/无无参构造器时创建对象失败 |

> ⚠️ **注意：** 反射调用方法时，被调方法抛出的异常会被包装成 `InvocationTargetException`，必须用 `e.getCause()` 才能拿到原始异常，否则会丢失根因。

## 九、与相关概念对比

### 反射 vs 普通 new 调用

| 对比项 | 普通 `new` / 直接调用 | 反射 |
|--------|---------------------|------|
| 类型确定时机 | 编译期 | 运行期 |
| 性能 | 快 | 较慢 |
| 访问 private | 不能 | 能（`setAccessible`） |
| 编译期检查 | 有（类型错误编译失败） | 无（运行时才报错） |
| 适用场景 | 业务代码日常开发 | 框架、工具、底层基础设施 |

### 反射 vs 动态代理

反射是"操作单个类成员"的底层能力；动态代理是建立在反射之上的"批量拦截方法调用"的高级技术，二者常配合使用。

## 十、面试常见问题

### Q1：什么是 Java 反射？

反射是 Java 在**运行期间**动态获取类信息（属性、方法、构造器、注解等）并操作它们的能力。它把"在编译期写死类和方法"的限制打开，让程序运行时也能临时决定要加载哪个类、调用哪个方法。它是一切框架（Spring、MyBatis 等）的底层基石。

### Q2：获取 Class 对象有几种方式？区别是什么？

三种：
1. `类名.class`：编译期已知，不触发初始化。
2. `对象.getClass()`：已有对象时用，不触发初始化。
3. `Class.forName("全限定类名")`：动态加载，**会触发初始化**（执行 static 块），常用于框架和配置加载。

### Q3：`getDeclaredXxx` 和 `getXxx` 有什么区别？

- `getXxx`：只返回 **public** 成员，**包含**从父类/接口继承来的。
- `getDeclaredXxx`：返回本类**声明的所有**成员（含 private/protected/默认），但**不包含**继承的。
- 访问 `getDeclaredXxx` 拿到的 private 成员时，需先 `setAccessible(true)`。

### Q4：反射创建对象有哪些方式？

- **推荐**：`clazz.getDeclaredConstructor().newInstance()`（Java 9+），支持任意构造器，错误处理清晰。
- **有参构造**：`clazz.getDeclaredConstructor(参数类型...).newInstance(参数...)`。
- **已废弃**：`clazz.newInstance()`（只能调 public 无参构造，Java 9 起 deprecated）。

### Q5：反射有哪些典型应用场景？

1. 框架 IoC（Spring 根据 XML/注解创建 Bean）
2. 动态代理 / AOP（`Proxy.newProxyInstance`）
3. 运行时注解处理（Spring MVC、JUnit）
4. ORM 结果集映射（MyBatis、Hibernate）
5. JSON 序列化（Jackson、Gson、Fastjson）

### Q6：反射的缺点是什么？如何优化？

缺点：性能损耗、破坏封装、丢失编译期检查、代码可读性差。
优化：缓存 `Class/Method/Field` 对象、用 `setAccessible(true)` 关闭访问检查、热点路径改用 `MethodHandle` 或 `LambdaMetafactory` 生成函数式接口。

### Q7：`setAccessible(true)` 的作用是什么？

它关闭 Java 的访问权限检查，让程序能访问 `private`、`protected`、包级私有的成员。在 Java 9+ 模块系统下，跨模块访问非公开成员可能受限，会抛 `InaccessibleObjectException`。

### Q8：反射调用方法抛出的异常为什么是 `InvocationTargetException`？

反射不会把被调方法内部的异常原样抛出，而是包装成 `InvocationTargetException`，真实异常通过 `getCause()` 获取。这样设计是为了和反射调用本身的错误（如参数不匹配）区分开。

### Q9：反射能修改 final 字段吗？

技术上可以（用 `setAccessible(true)`），但不可靠：编译期常量会被内联成字面量，`private final` 简单字段也可能被 JIT 内联，导致"改了字段但读到旧值"。生产代码不应依赖此行为。

## 十一、总结

- 反射是 Java 在**运行时**动态获取和操作类信息的能力，是框架实现动态性、解耦的基石。
- 获取 `Class` 对象有三种方式：`类名.class`、`对象.getClass()`、`Class.forName(...)`，其中 `Class.forName` 会触发初始化。
- `getXxx` 只拿 public 且含继承；`getDeclaredXxx` 拿本类全部含 private 不含继承——这是高频考点。
- 反射能创建对象、调用方法、读写字段，访问私有成员需 `setAccessible(true)`。
- 优点是灵活、解耦、动态；缺点是慢、破坏封装、丢失编译期检查，应通过**缓存反射对象**和**关闭访问检查**优化。
- 实战中遵循"业务代码少用反射，框架底层善用反射"的原则。
