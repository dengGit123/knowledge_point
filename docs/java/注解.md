# 注解（Annotation）

> 官方文档：[Annotations](https://docs.oracle.com/javase/tutorial/java/annotations/) ｜ 菜鸟教程：[Java 注解](https://www.runoob.com/java/java-annotation.html)

## 一、概述

### 1.1 什么是注解

注解（Annotation）是 Java 5 引入的一种**元数据（metadata）**机制，用于给程序元素（类、方法、字段、参数、包等）添加说明信息。

> 💡 **提示：** 注解本身**不会直接改变程序的执行逻辑**，它只是给程序"贴标签"。真正让标签生效的，是**读取这些标签的工具或框架**（如编译器、Spring、JUnit、Lombok 等）。

### 1.2 注解的本质

- 定义注解使用关键字 `@interface`
- 所有注解都隐式继承自 `java.lang.annotation.Annotation` 接口
- 注解在编译后本质上是一个接口

```java
// 自定义注解
public @interface MyAnnotation {
    String value();
}
```

编译后，`MyAnnotation` 实际上等价于：

```java
public interface MyAnnotation extends java.lang.annotation.Annotation {
    String value();
}
```

### 1.3 通俗类比：注解就像"贴标签"

想象你在超市里给商品贴标签：

```
┌────────────────────────────────────────────────┐
│  商品（代码元素）                                  │
│  ┌──────┐  ┌──────┐  ┌──────────┐              │
│  │ 促销 │  │ 进口 │  │ 限时打折  │  ← 标签（注解）│
│  └──────┘  └──────┘  └──────────┘              │
└────────────────────────────────────────────────┘
                  ↓
┌────────────────────────────────────────────────┐
│  收银员 / 理货员（工具 / 框架）                     │
│  看到标签 → 执行对应操作（打折、上架、促销展示）       │
└────────────────────────────────────────────────┘
```

- 标签（注解）只是标记，自己不会让商品变成促销品
- 是收银员（框架/工具）识别标签后，才执行了打折动作

## 二、元注解（修饰注解的注解）

> 元注解（Meta-Annotation）就是**用来注解其他注解的注解**，相当于"标签的说明书"，告诉编译器和框架：这个标签能贴在哪里、保留多久、能否被继承等。

Java 提供了 5 个标准元注解，定义在 `java.lang.annotation` 包下。

### 2.1 元注解总览

| 元注解 | 作用 | 取值 / 说明 | 是否常用 |
|--------|------|-------------|----------|
| `@Target` | 注解能用在哪些元素上 | `ElementType` 枚举 | ⭐⭐⭐⭐⭐ |
| `@Retention` | 注解保留到哪个阶段 | `RetentionPolicy` 枚举 | ⭐⭐⭐⭐⭐ |
| `@Documented` | 是否出现在 Javadoc 中 | 无参数（标记注解） | ⭐⭐⭐ |
| `@Inherited` | 子类是否自动继承该注解 | 无参数（标记注解） | ⭐⭐⭐ |
| `@Repeatable` | 同一位置是否可重复使用（Java 8+） | 容器注解的 Class | ⭐⭐ |

### 2.2 `@Target`：注解能用在哪里

`@Target` 指定注解可以应用的程序元素类型，取值来自 `java.lang.annotation.ElementType` 枚举：

| `ElementType` 值 | 可标注的目标元素 |
|------------------|------------------|
| `TYPE` | 类、接口、注解、枚举 |
| `FIELD` | 字段（含枚举常量） |
| `METHOD` | 方法 |
| `PARAMETER` | 方法参数 |
| `CONSTRUCTOR` | 构造方法 |
| `LOCAL_VARIABLE` | 局部变量 |
| `ANNOTATION_TYPE` | 注解类型（元注解） |
| `PACKAGE` | 包 |
| `TYPE_PARAMETER`（Java 8+） | 类型参数，如 `<T>` |
| `TYPE_USE`（Java 8+） | 类型使用处，如 `List<@NonNull String>` |
| `MODULE`（Java 9+） | 模块 |

```java
import java.lang.annotation.ElementType;
import java.lang.annotation.Target;

// 只能用在方法和字段上
@Target({ElementType.METHOD, ElementType.FIELD})
public @interface MyAnnotation {
    String value();
}
```

### 2.3 `@Retention`：注解保留到什么时候 ⭐ 重点

`@Retention` 决定注解的**生命周期**，取值来自 `java.lang.annotation.RetentionPolicy` 枚举：

| 保留策略 | 保留阶段 | 是否写入 class 文件 | 能否反射读取 | 典型应用 |
|----------|----------|---------------------|--------------|----------|
| `SOURCE` | 仅源码 | ❌ 否 | ❌ 否 | `@Override`、`@SuppressWarnings` |
| `CLASS` | 源码 + class 文件 | ✅ 是 | ❌ 否（默认值，但很少显式用） | 字节码工具处理 |
| `RUNTIME` | 源码 + class + 运行时 | ✅ 是 | ✅ 是 | Spring、JUnit 等框架 |

```
源码 (.java)
   │  编译
   ▼
字节码 (.class)   ←  SOURCE 注解到这里已经消失
   │  类加载
   ▼
运行时 (JVM 内存)  ←  CLASS 注解到这里消失，RUNTIME 注解还在
   │  反射
   ▼
框架读取注解执行逻辑
```

> ⚠️ **注意：** 想要通过反射读取注解，**必须**把保留策略设为 `RUNTIME`，否则 `getAnnotation()` 永远返回 `null`。这是新手最常踩的坑。

```java
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;

// ✅ 运行时可被反射读取
@Retention(RetentionPolicy.RUNTIME)
public @interface RuntimeAnnotation {
    String value();
}

// ❌ 反射读取不到（默认或显式 CLASS/SOURCE）
@Retention(RetentionPolicy.SOURCE)
public @interface SourceAnnotation {
}
```

### 2.4 `@Documented`：是否包含在 Javadoc 中

被 `@Documented` 修饰的注解，在生成 Javadoc 时会出现在文档中。

```java
import java.lang.annotation.Documented;

@Documented
public @interface Author {
    String name();
}
```

### 2.5 `@Inherited`：子类是否继承该注解

`@Inherited` 表示如果一个类使用了被该元注解修饰的注解，那么其**子类会自动继承**这个注解。

> ⚠️ **注意：** `@Inherited` **只对类有效**，对接口、方法、字段都无效。

```java
import java.lang.annotation.Inherited;

@Inherited
public @interface InheritedAnnotation {
    String value();
}

@InheritedAnnotation("父类")
public class Parent {}

// Child 也拥有 @InheritedAnnotation("父类")
public class Child extends Parent {}
```

### 2.6 `@Repeatable`：可重复注解（Java 8+）

允许在同一位置多次使用同一个注解，需要定义一个**容器注解**。

```java
import java.lang.annotation.Repeatable;

// 容器注解
public @interface Schedules {
    Schedule[] value();
}

// 可重复注解
@Repeatable(Schedules.class)
public @interface Schedule {
    String time();
    String task();
}

// 使用：同一方法上重复标注
@Schedule(time = "09:00", task = "晨会")
@Schedule(time = "18:00", task = "复盘")
public void dailyRoutine() {}
```

## 三、内置注解

Java 在 `java.lang` 包下提供了几个常用内置注解：

| 注解 | 作用 | 保留策略 | 典型场景 |
|------|------|----------|----------|
| `@Override` | 检查方法是否正确重写父类方法 | SOURCE | 重写方法时 |
| `@Deprecated` | 标记已过时 | RUNTIME | 提醒不再推荐使用 |
| `@SuppressWarnings` | 抑制编译器警告 | SOURCE | 关闭 unchecked 等警告 |
| `@FunctionalInterface` | 标记函数式接口 | RUNTIME | 配合 Lambda |
| `@SafeVarargs` | 抑制泛型可变参数警告 | RUNTIME | 泛型 varargs 方法 |

### 3.1 `@Override`

强制编译器检查该方法是否真的重写了父类（或实现了接口）方法，防止拼写错误。

```java
public class Animal {
    public void eat() {}
}

public class Dog extends Animal {
    @Override  // ✅ 编译器会校验是否真的重写
    public void eat() {
        System.out.println("狗在吃骨头");
    }

    // @Override
    // public void aet() {}  // ❌ 编译报错：没有重写任何方法
}
```

### 3.2 `@Deprecated`

标记某个程序元素已过时，使用时编译器会给出警告。

```java
@Deprecated
public class OldUtil {
    @Deprecated
    public static void oldMethod() {}
}
```

### 3.3 `@SuppressWarnings`

抑制编译器产生的警告。

```java
import java.util.ArrayList;
import java.util.List;

@SuppressWarnings("unchecked")
public void demo() {
    // 没有泛型，正常会报警告，这里被抑制
    List list = new ArrayList();
    list.add("hello");
}

@SuppressWarnings({"unchecked", "rawtypes", "unused"})
public void multi() {}
```

常见抑制值：`unchecked`、`rawtypes`、`unused`、`deprecation`、`all`。

### 3.4 `@FunctionalInterface`

确保接口有且只有一个抽象方法，可作为 Lambda 目标。

```java
@FunctionalInterface
public interface Calculator {
    int calc(int a, int b);
}

Calculator add = (a, b) -> a + b;
```

### 3.5 `@SafeVarargs`

修饰 `static` 或 `final` 的泛型可变参数方法，抑制堆污染警告（Java 7+）。

```java
@SafeVarargs
public static <T> List<T> of(T... elements) {
    List<T> list = new ArrayList<>();
    for (T e : elements) {
        list.add(e);
    }
    return list;
}
```

## 四、自定义注解

### 4.1 定义语法

使用 `@interface` 关键字定义注解：

```java
[元注解]
public @interface 注解名 {
    数据类型 属性名() [default 默认值];
}
```

### 4.2 注解的属性

注解中的"属性"看起来像方法，实际上是注解的配置项：

```java
public @interface MyAnnotation {
    String name();              // 必须赋值
    int age() default 18;       // 有默认值，可省略
    String[] tags() default {}; // 数组属性
}
```

### 4.3 属性类型限制 ⚠️ 重点

注解属性**只能**使用以下类型，否则编译报错：

| ✅ 允许的类型 | ❌ 不允许的类型 |
|----------------|------------------|
| 8 种基本类型（int、long、char…） | `Object` |
| `String` | 包装类之外的其他引用类型（如自定义类） |
| 枚举（Enum） | `Integer` 等包装类以外的对象 |
| `Class` | 集合（List、Map 等） |
| 注解类型 | 数组列表中的非法元素 |
| 以上类型的一维数组 | 多维数组 |

```java
public @interface ValidAnnotation {
    int id();                 // ✅ 基本类型
    String name();            // ✅ String
    Class<?> clazz();         // ✅ Class
    RetentionPolicy policy(); // ✅ 枚举
    Deprecated dep();         // ✅ 注解类型
    String[] alias() default {};// ✅ 数组

    // Object obj();   // ❌ 编译错误
    // List list();    // ❌ 编译错误
}
```

### 4.4 使用注解并赋值

```java
@MyAnnotation(name = "张三", age = 25, tags = {"java", "注解"})
public class User {}
```

### 4.5 `value` 属性的特殊性

当注解**只有一个属性且名为 `value`** 时，使用时可省略 `value =`：

```java
public @interface Name {
    String value();
}

// ✅ 简写
@Name("hello")
public class A {}

// 等价于
@Name(value = "hello")
public class B {}
```

> 💡 **提示：** 当存在多个属性且都已有默认值，只给 `value` 赋值时，仍可省略 `value =`。

## 五、完整示例：自定义注解 + 反射读取

下面实现一个**简易的测试框架**：自定义 `@Test` 注解标记测试方法，通过反射扫描并执行。

### 5.1 定义注解

```java
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

/**
 * 测试方法注解
 * - 只能用于方法
 * - 运行时保留，可被反射读取
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Test {
    String description() default "";  // 测试描述
}
```

### 5.2 使用注解

```java
public class CalculatorTest {

    @Test(description = "测试加法")
    public void testAdd() {
        System.out.println("执行 testAdd...");
        assert 1 + 1 == 2;
    }

    @Test
    public void testSub() {
        System.out.println("执行 testSub...");
        assert 5 - 3 == 2;
    }

    // 没有 @Test，不会被框架执行
    public void helper() {
        System.out.println("辅助方法");
    }
}
```

### 5.3 通过反射读取并执行

```java
import java.lang.reflect.Method;

public class TestRunner {

    public static void run(Class<?> testClass) throws Exception {
        Object instance = testClass.getDeclaredConstructor().newInstance();
        Method[] methods = testClass.getDeclaredMethods();

        int passed = 0;
        int failed = 0;

        for (Method method : methods) {
            // 判断方法上是否有 @Test 注解
            if (method.isAnnotationPresent(Test.class)) {
                try {
                    // 获取注解实例并读取属性
                    Test testAnno = method.getAnnotation(Test.class);
                    System.out.println("运行测试: " + method.getName()
                            + "（" + testAnno.description() + "）");
                    method.invoke(instance);
                    passed++;
                } catch (Exception e) {
                    System.out.println("❌ 测试失败: " + method.getName());
                    failed++;
                }
            }
        }

        System.out.println("通过: " + passed + "，失败: " + failed);
    }

    public static void main(String[] args) throws Exception {
        run(CalculatorTest.class);
    }
}
```

### 5.4 反射 API 速查

| 方法 | 作用 |
|------|------|
| `Class.isAnnotationPresent(Class)` | 类上是否存在某注解 |
| `Method.isAnnotationPresent(Class)` | 方法上是否存在某注解 |
| `Field.isAnnotationPresent(Class)` | 字段上是否存在某注解 |
| `getAnnotation(Class)` | 获取单个注解（需 RUNTIME） |
| `getAnnotations()` | 获取所有注解数组 |
| `getDeclaredAnnotations()` | 获取本元素直接声明的注解 |

```
┌──────────────────────────────────────────────────────┐
│  反射读取注解流程                                       │
├──────────────────────────────────────────────────────┤
│  Class<?> clz = ...                                   │
│        │                                              │
│        ▼                                              │
│  clz.isAnnotationPresent(MyAnno.class)                │
│        │ true                                         │
│        ▼                                              │
│  MyAnno anno = clz.getAnnotation(MyAnno.class)        │
│        │                                              │
│        ▼                                              │
│  anno.value()  → 读取属性值                            │
└──────────────────────────────────────────────────────┘
```

## 六、注解的处理时机

注解"贴上去"之后，谁来读取并执行逻辑？主要有三种处理时机：

### 6.1 三种处理时机对比

| 处理时机 | 工具 / 机制 | 典型代表 | 是否需要 RUNTIME |
|----------|-------------|----------|------------------|
| **编译时** | 注解处理器（APT，Annotation Processing Tool） | Lombok、ButterKnife、MapStruct | 否（SOURCE 即可） |
| **运行时** | 反射（Reflection） | Spring、JUnit、MyBatis | 是（必须 RUNTIME） |
| **字节码期** | ASM、Javassist 等字节码工具 | 各类字节码增强框架 | 否（CLASS） |

```
┌─────────────┐    编译    ┌─────────────┐   类加载   ┌─────────────┐
│  源码 .java  │ ────────▶ │ 字节码 .class │ ────────▶ │  运行时 JVM  │
└─────────────┘           └─────────────┘           └─────────────┘
       │                        │                         │
       ▼                        ▼                         ▼
  ┌──────────┐           ┌────────────┐            ┌────────────┐
  │  APT     │           │ ASM 等工具  │            │  反射       │
  │ 注解处理器│           │ 修改字节码   │            │ 读取注解    │
  │(Lombok)  │           │            │            │(Spring)    │
  └──────────┘           └────────────┘            └────────────┘
```

### 6.2 编译时处理（APT）

Java 编译器在编译期会调用注解处理器（实现 `javax.annotation.processing.Processor` 接口）。处理器可以读取注解、生成新的源文件或修改现有代码。

- **Lombok** 的 `@Data`、`@Getter` 并非通过反射，而是在编译期通过 APT 直接向 class 中注入 getter/setter/toString 等方法
- **MapStruct** 在编译期根据 `@Mapper` 注解生成转换实现类

### 6.3 运行时处理（反射）

保留策略为 `RUNTIME` 的注解会进入 JVM，框架在运行时通过反射读取并执行逻辑：

- **Spring**：扫描 `@Component` 注册 Bean，`@Autowired` 注入依赖
- **JUnit**：扫描 `@Test` 标记的方法并执行
- **MyBatis**：读取 `@Select`、`@Insert` 上的 SQL 并执行

### 6.4 字节码处理

通过 ASM、Javassist、ByteBuddy 等工具直接操作 `.class` 文件，常用于 AOP、动态代理、性能监控等。

## 七、注解的典型应用

| 框架 / 工具 | 典型注解 | 处理时机 | 作用 |
|-------------|----------|----------|------|
| Lombok | `@Data`、`@Getter`、`@Slf4j` | 编译时（APT） | 生成 getter/setter/toString 等 |
| Spring | `@Component`、`@Autowired`、`@RequestMapping`、`@Service` | 运行时（反射） | Bean 注册、依赖注入、路由映射 |
| Spring Boot | `@SpringBootApplication`、`@ConfigurationProperties` | 运行时 | 自动配置、属性绑定 |
| JUnit | `@Test`、`@BeforeEach`、`@ParameterizedTest` | 运行时 | 标记测试方法 |
| MyBatis | `@Select`、`@Insert`、`@Param` | 运行时 | 绑定 SQL |
| JPA / Hibernate | `@Entity`、`@Table`、`@Id`、`@Column` | 运行时 | ORM 映射 |
| ButterKnife | `@BindView` | 编译时（APT） | View 绑定（Android） |
| Jackson | `@JsonProperty`、`@JsonIgnore` | 运行时 | JSON 序列化控制 |

### 7.1 Lombok 编译期生成示例

```java
import lombok.Data;

@Data  // 编译期自动生成 getter/setter/toString/equals/hashCode
public class User {
    private Long id;
    private String name;
}
```

### 7.2 Spring 运行时反射示例

```java
import org.springframework.stereotype.Component;
import org.springframework.beans.factory.annotation.Autowired;

@Component          // 运行时被 Spring 扫描，注册为 Bean
public class UserService {

    @Autowired      // 运行时由 Spring 注入依赖
    private UserMapper userMapper;

    public User findById(Long id) {
        return userMapper.selectById(id);
    }
}
```

## 八、注意事项与陷阱

> ⚠️ **注意：** 自定义注解如果只定义不读取，等于啥也没干。**注解必须搭配反射或 APT 才有意义**。

### 8.1 常见陷阱清单

| 陷阱 | 说明 | 正确做法 |
|------|------|----------|
| 反射读不到注解 | 保留策略不是 `RUNTIME` | 显式声明 `@Retention(RUNTIME)` |
| 属性类型非法 | 使用了 `Object`、`List` 等 | 只用允许的 6 类类型 |
| 注解能继承类 | 注解隐式继承 `Annotation` 接口 | 注解不能再 `extends` 其他类 |
| 属性没默认值又不赋值 | 必填属性未赋值编译报错 | 加 `default` 或使用时显式赋值 |
| `@Inherited` 误用 | 期望接口实现类也继承 | 只对类继承有效，接口无效 |
| 注解属性值为 `null` | 注解属性**不能**赋值为 `null` | 用空字符串或默认值代替 |
| `@Target` 限制 | 注解用在了不允许的位置编译报错 | 按需配置 `@Target` |

### 8.2 注解属性不能为 `null`

```java
public @interface Demo {
    String name();
}

// @Demo(name = null)  // ❌ 编译错误：注解属性值不能为 null
@Demo(name = "")        // ✅ 用空字符串代替
public class A {}
```

### 8.3 注解可以注解在注解上

注解本身也可以被注解修饰（这就是元注解）：

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
//       ↑ 这两个就是注解在注解上的注解（元注解）
public @interface MyAnnotation {
    String value();
}
```

## 九、注解 vs 注释 vs 关键字 对比

| 对比项 | 注解（Annotation） | 注释（Comment） | 关键字（Keyword） |
|--------|---------------------|------------------|-------------------|
| 语法 | `@Annotation` | `//`、`/* */` | `class`、`public` 等 |
| 编译后是否保留 | 取决于保留策略 | ❌ 全部丢弃 | 保留为字节码指令 |
| 程序能否读取 | RUNTIME 可被反射读 | ❌ 不能 | ❌ 不能 |
| 作用 | 给工具/框架提供元数据 | 给人看说明 | 定义语法结构 |

## 十、面试常见问题

### Q1：什么是注解？它的本质是什么？

注解是 Java 5 引入的一种**元数据机制**，用于给程序元素添加说明信息。它的本质是用 `@interface` 定义的特殊接口，隐式继承 `java.lang.annotation.Annotation` 接口。注解本身不改变程序逻辑，需要工具或框架读取后才发挥作用。

### Q2：元注解有哪些？分别什么作用？

Java 提供 5 个元注解：

- `@Target`：注解能用在哪些元素上（ElementType）
- `@Retention`：注解保留到哪个阶段（RetentionPolicy）
- `@Documented`：是否出现在 Javadoc 中
- `@Inherited`：子类是否继承该注解
- `@Repeatable`：是否可在同一位置重复使用（Java 8+）

### Q3：`@Retention` 的三种策略有什么区别？

| 策略 | 保留阶段 | 反射可读 | 典型代表 |
|------|----------|----------|----------|
| `SOURCE` | 仅源码 | ❌ | `@Override` |
| `CLASS` | 到 class 文件 | ❌ | 默认值，字节码工具用 |
| `RUNTIME` | 到运行时 | ✅ | Spring 的 `@Component` |

只有 `RUNTIME` 才能通过反射读取，这是自定义注解最常需指定的策略。

### Q4：如何自定义一个注解？属性有哪些限制？

用 `@interface` 定义，语法为 `类型 属性名() [default 默认值]`。属性类型只能用：基本类型、`String`、枚举、`Class`、注解类型、以及以上类型的一维数组。属性值不能为 `null`，没有默认值的属性在使用时必须显式赋值。当只有一个属性且名为 `value` 时可省略 `value =`。

### Q5：注解是如何被处理的？

主要有三种处理时机：

1. **编译时**：通过注解处理器（APT），如 Lombok 生成 getter/setter
2. **运行时**：通过反射读取（需 `RUNTIME` 保留策略），如 Spring 注册 Bean
3. **字节码期**：通过 ASM/Javassist 等工具修改字节码

### Q6：为什么通过反射读不到自定义注解？

最常见原因是**保留策略不是 `RUNTIME`**。如果注解默认保留策略或设为 `SOURCE`/`CLASS`，运行时反射 `getAnnotation()` 会返回 `null`。解决方法是显式声明：

```java
@Retention(RetentionPolicy.RUNTIME)  // ✅ 关键
public @interface MyAnnotation {}
```

### Q7：注解能继承类吗？`@Inherited` 的作用范围是什么？

注解**不能显式继承**其他类，因为它已经隐式继承了 `Annotation` 接口。`@Inherited` 元注解表示子类会自动继承父类上的该注解，但它**只对类的继承有效**，对接口实现、方法、字段都无效。

### Q8：Lombok 和 Spring 使用注解的原理一样吗？

不一样。Lombok 在**编译期**通过注解处理器（APT）直接修改/生成字节码，不需要运行时反射，所以效率高、无运行时开销。Spring 则在**运行时**通过反射扫描 `@Component` 等注解完成 Bean 注册和依赖注入。

## 十一、总结

1. **注解是元数据**：给程序元素"贴标签"，本身不改变逻辑，由工具/框架读取后发挥作用。
2. **本质是接口**：用 `@interface` 定义，隐式继承 `Annotation`。
3. **元注解是关键**：`@Target` 控制位置、`@Retention` 控制生命周期、`@Documented`/`@Inherited`/`@Repeatable` 控制行为。
4. **保留策略要记牢**：`SOURCE` → `CLASS` → `RUNTIME`，只有 `RUNTIME` 才能反射读取。
5. **自定义注解限制**：属性类型只能是 6 类合法类型，值不能为 `null`。
6. **处理时机三种**：编译时（APT）、运行时（反射）、字节码（ASM）。
7. **实践要点**：自定义注解必须搭配反射或 APT 才有意义，否则形同虚设。

掌握注解是理解 Spring、MyBatis、JUnit、Lombok 等主流框架的基石，也是 Java 中高级开发者的必备技能。
