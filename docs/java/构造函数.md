# 构造函数（Constructor）

> 官方文档：[Oracle Java Tutorial - Providing Constructors](https://docs.oracle.com/javase/tutorial/java/javaOO/constructors.html) ｜ 菜鸟教程：[Java 构造方法](https://www.runoob.com/java/java-constructor.html)

> 本文是 [类.md](./类.md) 中"构造方法"一节的**专题深入**，更系统地讲解构造函数的所有知识点。

---

## 一、概述

**构造函数（Constructor）**，也叫**构造方法**，是一种**特殊的方法**，它的作用是：**在创建对象（new）时，完成对象的初始化**（给成员变量赋初始值、做必要的准备工作）。

```java
// new Person("张三") 时，JVM 会自动调用 Person 类对应的构造函数
Person p = new Person("张三");
```

> 💡 **通俗理解**：构造函数就像"新员工的入职手续"。
> - `new Person()` 就像公司招了一个新员工"对象"。
> - 构造函数就是这个员工办理入职的过程——**分配工号、领工牌、填写基本信息**。
> - 入职手续（构造函数）只在"入职那一刻（new）"自动执行一次，之后不会再跑。

注意：构造函数是**初始化对象**，而**不是创建对象**。对象的内存分配（在堆中开辟空间）是 `new` 关键字完成的，构造函数负责的是把这块内存里的成员变量填上合理的值。

## 二、构造函数的定义与语法

```java
修饰符 类名(参数列表) {
    // 方法体：初始化代码
}
```

一个完整的构造函数示例：

```java
public class Student {
    private String name;
    private int age;

    // 构造函数：方法名与类名相同，无返回值类型
    public Student(String name, int age) {
        this.name = name;   // 用 this 区分成员变量和参数
        this.age = age;
    }
}

// 使用
Student s = new Student("张三", 18);
```

## 三、构造函数的四大特点

| 特点 | 说明 | 示例 |
|------|------|------|
| ① 方法名与类名**完全相同** | 大小写也要一致 | 类叫 `Student`，构造函数也叫 `Student` |
| ② **没有返回值类型** | 连 `void` 都不能写 | `public Student()` 而非 `public void Student()` |
| ③ 不需要 `return` | 不返回任何值（虽然可以写 `return;` 提前结束） | —— |
| ④ 由 `new` 自动调用 | 不能像普通方法那样手动调用 | `new Student()` 触发 |

```java
public class Student {
    // ✅ 正确：无返回值类型
    public Student() {
        System.out.println("无参构造");
    }

    // ❌ 错误：这不是构造函数！这是普通的实例方法（有 void）
    public void Student() {
        System.out.println("这是一个普通方法，不是构造函数");
    }
}
```

> ⚠️ **注意（易错点）**：如果不小心给"构造函数"加了 `void`，它就变成了一个**普通的实例方法**，`new` 时**不会调用它**，对象就不会被正确初始化！这是新手常踩的坑。

### 构造函数的修饰符

| 修饰符 | 能否修饰构造函数 | 说明 |
|--------|:---------------:|------|
| `public` / `protected` / 默认 / `private` | ✅ | 控制谁能在哪里创建对象 |
| `static` | ❌ | 构造函数用于初始化对象，static 与"对象"无关 |
| `final` | ❌ | final 修饰方法是为了禁止重写，而构造函数本来就不能被重写 |
| `abstract` | ❌ | abstract 用于要求子类实现，构造函数不存在"被实现"的概念 |
| `synchronized` | ✅（少用） | 可以，但一般无意义 |

## 四、默认构造函数（Default Constructor）

这是构造函数最重要的规则之一：

> **如果类中没有显式定义任何构造函数，编译器会自动生成一个无参、空方法体的构造函数（默认构造函数）。**
> **一旦你手动定义了任何一个构造函数（无论有参无参），编译器就不再提供默认构造函数。**

```java
public class A {
    // 没有写任何构造函数
}
// 编译后，A 类实际等价于：
public class A {
    public A() {   // 编译器自动添加的默认无参构造
    }
}

public class B {
    private int x;
    // 手动定义了一个有参构造
    public B(int x) {
        this.x = x;
    }
    // ❌ 此时编译器不会再提供无参构造！
}
// new B(); // 编译报错，因为 B 没有无参构造了
```

> 💡 **重要后果**：如果一个类本来有无参构造，后来你加了一个有参构造，原本依赖无参构造的代码（如反射、框架的 `newInstance()`、JSON 反序列化）就会**编译/运行报错**。所以**定义了有参构造时，通常建议显式补上无参构造**。

```java
public class User {
    private String name;

    public User() {}          // ✅ 显式保留无参构造（推荐）

    public User(String name) {
        this.name = name;
    }
}
```

## 五、构造函数的重载（Overloading）

构造函数可以重载：**同一个类中可以有多个构造函数，只要它们的参数列表不同**（参数个数、类型、顺序不同）。这样可以用不同方式创建对象。

```java
public class Student {
    private String name;
    private int age;
    private String school;

    // 构造函数 1：无参
    public Student() {
        System.out.println("无参构造");
    }

    // 构造函数 2：一个参数（重载）
    public Student(String name) {
        this.name = name;
    }

    // 构造函数 3：两个参数（重载）
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 构造函数 4：三个参数（重载）
    public Student(String name, int age, String school) {
        this.name = name;
        this.age = age;
        this.school = school;
    }
}

// 使用：根据参数自动匹配对应构造函数
new Student();
new Student("张三");
new Student("李四", 20);
new Student("王五", 22, "清华大学");
```

## 六、`this()` 调用本类的其他构造函数

在一个构造函数中，可以用 `this(参数)` 调用**本类的另一个构造函数**，避免重复写初始化代码。

```java
public class Student {
    private String name;
    private int age;
    private String school;

    // 主构造函数：做全部初始化
    public Student(String name, int age, String school) {
        this.name = name;
        this.age = age;
        this.school = school;
    }

    // 用 this() 复用上面的构造函数，避免重复代码
    public Student(String name, int age) {
        this(name, age, "未知学校");   // ✅ 调用三个参数的构造函数
    }

    public Student() {
        this("无名", 0);               // ✅ 调用两个参数的构造函数（链式调用）
    }
}
```

**`this()` 的两条硬性规则**：

| 规则 | 说明 |
|------|------|
| ① 必须在构造函数的**第一行** | `this(...)` 只能出现在构造函数中，且必须是第一条语句 |
| ② 不能形成**循环调用** | 构造函数间不能互相递归调用 `this()`，否则编译报错 |

```java
public class Wrong {
    public Wrong(int a) {
        System.out.println("do something");
        this();  // ❌ 编译错误！this() 必须在第一行
    }

    public Wrong() {
        this(1); // 调用 Wrong(int)
    }
    public Wrong(int a) {
        this();  // ❌ 编译错误！Wrong() 和 Wrong(int) 互相调用，形成死循环
    }
}
```

> ⚠️ **`this(...)` 和 `super(...)` 不能同时出现**：它们都要求在第一行，所以一个构造函数里**只能有一个**。如果写了 `this(...)`，就不会有默认的 `super()` 了（因为被调用的那个构造函数里会调用 `super()`）。详见 [super.md](./super.md)。

## 七、构造函数与初始化顺序

对象创建时，成员变量的初始化和构造函数有固定的执行顺序：

```
① 成员变量的默认值
   （int=0、引用=null 等，由 JVM 分配内存时零值初始化）
        │
        ▼
② 成员变量的显式赋值 / 实例代码块 { }
   （按源码中书写的先后顺序执行）
        │
        ▼
③ 构造函数
   （最后执行，可以覆盖前面的赋值）
```

```java
public class Order {
    int a = 10;        // ② 声明处赋值
    int b;             // 只有默认值 0

    {                  // ② 实例代码块
        b = 20;
        System.out.println("实例代码块执行，b = " + b);
    }

    public Order() {   // ③ 构造函数
        a = 100;
        b = 200;
        System.out.println("构造函数执行，a = " + a + ", b = " + b);
    }

    public static void main(String[] args) {
        Order o = new Order();
        // 输出：
        // 实例代码块执行，b = 20
        // 构造函数执行，a = 100, b = 200
        System.out.println("最终 a = " + o.a + ", b = " + o.b);
        // 输出：最终 a = 100, b = 200（被构造函数覆盖）
    }
}
```

> 💡 **记忆口诀**：**"默认值 → 赋值 / 代码块 → 构造函数"**。构造函数是初始化的"最后一关"，可以覆盖前面所有的赋值。

## 八、private 构造函数

构造函数可以被 `private` 修饰，这样**类外部就无法 `new` 出对象**。这有两个经典应用：

### 1. 单例模式（Singleton）

保证一个类全局只有一个实例：

```java
public class Singleton {
    // 1. 私有构造函数：外部无法 new
    private Singleton() {}

    // 2. 类内部持有一个唯一实例
    private static final Singleton INSTANCE = new Singleton();

    // 3. 提供全局访问点
    public static Singleton getInstance() {
        return INSTANCE;
    }
}
```

### 2. 工具类（防止实例化）

工具类（如 `java.lang.Math`、`java.util.Arrays`）只有静态方法，不需要、也不应该被实例化：

```java
public class MathUtils {
    // 私有构造函数：防止外部 new（工具类不需要对象）
    private MathUtils() {
        throw new AssertionError(); // 防止通过反射在类内部调用
    }

    public static int add(int a, int b) {
        return a + b;
    }
}
// new MathUtils(); // ❌ 编译错误，构造函数不可见
```

## 九、构造函数 vs 普通方法

| 对比维度 | 构造函数 | 普通方法 |
|---------|---------|---------|
| **方法名** | 必须与类名相同 | 自定义（符合标识符规范） |
| **返回值类型** | 无（连 void 都不能写） | 必须声明（void 或具体类型） |
| **调用方式** | 由 `new` 自动调用 | 通过对象名 / 类名手动调用 |
| **调用时机** | 创建对象时调用一次 | 可多次调用 |
| **作用** | 初始化对象 | 实现某个功能 |
| **能否被继承** | ❌ 不能 | ✅ 能（非 private） |
| **能否被重写** | ❌ 不能（但能重载） | ✅ 能 |
| **能否被 abstract / final / static 修饰** | ❌ | 部分可以 |

## 十、子类构造函数与父类构造函数

子类构造函数**第一行默认会调用父类的无参构造**（隐式 `super()`）。关于这部分，详见 [super.md](./super.md) 和 [继承.md](./继承.md)。这里给一个直观示例：

```java
class Animal {
    public Animal() {
        System.out.println("Animal 的无参构造");
    }
}

class Dog extends Animal {
    public Dog() {
        super();  // 隐式调用，即使不写也存在，且必须在第一行
        System.out.println("Dog 的无参构造");
    }
}

new Dog();
// 输出：
// Animal 的无参构造
// Dog 的无参构造
```

> ⚠️ **注意**：如果父类**没有无参构造**（比如父类只定义了有参构造），子类构造函数就必须**显式调用** `super(参数)`，否则编译报错。

## 十一、注意事项与陷阱

1. **别给构造函数加 `void`** —— 否则它就变成了普通方法，`new` 时不会调用，对象不会被初始化。
2. **定义了有参构造后，记得补无参构造** —— 很多框架（Spring、MyBatis、JSON 反序列化）依赖无参构造，缺少会导致运行时报错。
3. **构造函数不能被继承** —— 子类无法"继承"父类的构造函数，但能通过 `super()` 调用。
4. **构造函数不能被重写** —— 但可以重载（一个类多个构造函数）。
5. **不要在构造函数中做耗时操作** —— 构造函数应只做初始化，耗时逻辑（如 IO、网络）应放到独立方法中。
6. **不要在构造函数中调用可被重写的方法** —— 此时子类还没初始化完成，可能引发空指针或状态错误。

## 十二、面试常见问题

### Q1：构造函数和普通方法有什么区别？

构造函数方法名与类名相同、没有返回值类型（连 void 都没有）、由 `new` 自动调用、用于初始化对象、不能被继承和重写（可重载）。普通方法则相反：自定义名称、必须声明返回值、手动调用、实现功能逻辑、可被继承和重写。

### Q2：什么是默认构造函数？

当类中没有显式定义任何构造函数时，编译器会自动生成一个 `public` 无参、空方法体的构造函数。一旦手动定义了任意构造函数，编译器就不再提供。所以定义有参构造后通常要显式补上无参构造。

### Q3：构造函数可以被重写吗？

**不能**。构造函数不属于"可继承的成员"，所以谈不上重写。但构造函数**可以重载**——一个类可以有多个参数列表不同的构造函数。

### Q4：this() 和 super() 有什么要求？

两者都必须在构造函数的**第一行**，因此一个构造函数里**只能有一个**（不能同时存在）。`this()` 调用本类其他构造函数，`super()` 调用父类构造函数。它们都不能形成循环调用。

### Q5：一个类如果没有写构造函数，能 new 对象吗？

能。因为编译器会自动提供一个无参的默认构造函数。但如果手动定义了有参构造而忘了写无参构造，再用 `new 类名()` 就会编译报错。

### Q6：构造函数为什么不能被 abstract、static、final 修饰？

- **abstract**：abstract 用于要求子类实现，但构造函数不能被继承/重写，谈不上"被实现"。
- **static**：static 属于类不属于对象，而构造函数专门用于初始化对象，矛盾。
- **final**：final 用于禁止重写，但构造函数本来就不能被重写，加 final 无意义。

### Q7：为什么定义了有参构造后建议补无参构造？

因为很多框架（Spring 依赖注入、JPA/Hibernate 实体映射、Jackson/Gson JSON 反序列化、反射 `newInstance()`）都**依赖无参构造**来创建对象。缺少无参构造会导致这些框架运行时抛出异常。

### Q8：私有构造函数有什么用？

主要有两类用途：① **单例模式**——控制全局只有一个实例；② **工具类**——如 `Math`、`Arrays`，只有静态方法，防止被实例化。

## 十三、总结

| 知识点 | 核心要点 |
|--------|---------|
| 作用 | 在 `new` 时初始化对象（不是创建对象） |
| 四大特点 | 同名、无返回值、无 return、由 new 自动调用 |
| 默认构造 | 无任何构造时编译器自动加；一旦自定义则不再加 |
| 重载 | 可有多个，参数列表不同 |
| this() | 调用本类其他构造，必须在第一行，不能循环 |
| 初始化顺序 | 默认值 → 赋值/实例块 → 构造函数 |
| private 构造 | 单例模式、工具类 |
| 子类构造 | 首行默认 super()，详见 super.md |

**记住三句话**：

1. **构造函数 = 同名 + 无返回值 + new 自动调用，用于初始化（别加 void！）。**
2. **没写构造有默认，写了构造（有参）就没了无参——记得补。**
3. **初始化顺序是"默认值 → 赋值/代码块 → 构造函数"，this() 和 super() 都必须在第一行。**
