### 一、概述

> 📖 [System（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/lang/System.html) ｜ [Scanner（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/util/Scanner.html)

标准流是 Java 程序与外部（控制台）交互的**三个预定义流**，封装在 `System` 类里：

| 标准流 | 类型 | 方向 | 用途 |
| --- | --- | --- | --- |
| `System.in` | `InputStream` | 输入 | 读取**键盘输入** |
| `System.out` | `PrintStream` | 输出 | 输出到**控制台**（正常信息） |
| `System.err` | `PrintStream` | 输出 | 输出到**控制台**（错误信息，红色） |

大白话：`System.out` 是程序的「**嘴巴**」（往控制台说话），`System.in` 是「**耳朵**」（听键盘输入），`System.err` 是「**警报器**」（喊错误）。

> 💡 **提示：** 你写的第一个 Java 程序 `System.out.println("Hello World")`，用的就是标准流里的 `out`。

---

### 二、System.out（标准输出）

`System.out` 是一个 `PrintStream`，负责把信息打印到控制台。

```java
System.out.println("打印并换行");
System.out.print("打印不换行");
System.out.printf("格式化：%s = %d%n", "张三", 20);

// System.out 就是 PrintStream，可以赋值给变量
PrintStream out = System.out;
out.println("等价写法");
```

详见 [[打印流]]——`System.out` 本质就是 `PrintStream`。

---

### 三、System.err（标准错误）

`System.err` 也是 `PrintStream`，专门输出**错误/警告信息**，在控制台通常显示为**红色**。

```java
System.err.println("发生错误！");
System.err.println("文件不存在");
```

#### out 和 err 的区别

| 对比 | System.out | System.err |
| --- | --- | --- |
| 用途 | 正常输出 | 错误/警告 |
| 颜色 | 默认色 | 红色 |
| 缓冲 | 行缓冲 | （可能）无缓冲 |
| 重定向 | 可分开重定向 | 可分开重定向 |

> 💡 **提示：** `out` 和 `err` 是**两个独立**的流，可以分别重定向到不同地方（见下文）。日志框架（如 logback）的 ERROR 级别通常走 `err`。

#### out 和 err 顺序混乱问题

由于 `out` 有缓冲、`err` 无缓冲，混用时**输出顺序可能错乱**：

```java
System.out.println("第一");
System.err.println("第二");   // 可能在"第一"之前显示
```

调试时如需严格顺序，统一用一个流，或手动 flush。

### 四、System.in（标准输入）

`System.in` 是一个 `InputStream`，读取键盘输入。但它只读字节，**实际使用要包装**：

```java
import java.io.*;

public class InputDemo {
    public static void main(String[] args) throws IOException {
        // ❌ 直接用 System.in 读字节，处理输入很麻烦
        // int b = System.in.read();

        // ✅ 包装成字符流 + 缓冲流，才能方便地读取
        BufferedReader reader = new BufferedReader(
            new InputStreamReader(System.in, "UTF-8"));

        System.out.print("请输入名字：");
        String name = reader.readLine();   // 读一行
        System.out.println("你好，" + name);
    }
}
```

> 💡 **提示：** `System.in` 是字节流，要读文本必须配合 [[转换流]]（`InputStreamReader`）转成字符流。

### 五、Scanner：更方便的输入工具

`java.util.Scanner` 是处理标准输入**最常用、最方便**的工具，能按行、按单词、按类型读取：

```java
import java.util.Scanner;

public class ScannerDemo {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);   // 包装 System.in

        System.out.print("输入姓名：");
        String name = scanner.nextLine();           // 读一行

        System.out.print("输入年龄：");
        int age = scanner.nextInt();                // ✅ 直接读 int

        System.out.print("输入身高：");
        double height = scanner.nextDouble();       // ✅ 直接读 double

        System.out.printf("姓名：%s，年龄：%d，身高：%.1f%n", name, age, height);

        scanner.close();   // 用完关闭
    }
}
```

#### Scanner 常用方法

| 方法 | 作用 |
| --- | --- |
| `nextLine()` | 读一整行 |
| `next()` | 读一个单词（空格分隔） |
| `nextInt()` | 读一个 int |
| `nextDouble()` | 读一个 double |
| `nextBoolean()` | 读一个 boolean |
| `hasNext()` | 是否还有输入 |

> 💡 **提示：** `Scanner` 比直接包装 `System.in` 方便得多，**日常控制台输入首选 Scanner**。

#### nextLine 和 next 的坑

```java
// nextInt 后接 nextLine 会跳过的问题
int age = scanner.nextInt();        // 读数字，但留下换行符
String line = scanner.nextLine();   // ❌ 直接读到空行（吃掉了残留换行符）

// 解决：多调一次 nextLine 消耗换行符，或都用 nextXxx 配对
```

### 六、重定向（setIn / setOut / setErr）

标准流的「目标」可以**重定向**——让 `out` 不再打印到控制台，而是写到文件。

#### 1. 输出重定向到文件

```java
import java.io.*;

public class RedirectDemo {
    public static void main(String[] args) throws Exception {
        // ✅ 把 System.out 重定向到文件
        PrintStream fileOut = new PrintStream("console.log");
        System.setOut(fileOut);

        System.out.println("这行会写入 console.log，而不是控制台");
        System.out.println("看不见我～");
    }
}
```

#### 2. 输入重定向（从文件读，代替键盘）

```java
// ✅ 把 System.in 重定向为读文件
System.setIn(new FileInputStream("input.txt"));

Scanner scanner = new Scanner(System.in);   // 现在读的是文件内容
```

#### 3. 错误重定向

```java
System.setErr(new PrintStream("error.log"));
```

| 方法 | 作用 |
| --- | --- |
| `System.setIn(InputStream)` | 重定向标准输入 |
| `System.setOut(PrintStream)` | 重定向标准输出 |
| `System.setErr(PrintStream)` | 重定向标准错误 |

> 💡 **提示：** 重定向常用于：**把控制台日志存到文件**、**自动化测试用文件模拟键盘输入**。

### 七、标准流的本质

`System.in/out/err` 是 `System` 类里的 `static final` 变量，JVM 启动时初始化：

```java
public final class System {
    public final static InputStream in = null;
    public final static PrintStream out = null;
    public final static PrintStream err = null;
    // ... setIn/setOut/setErr 方法
}
```

虽然声明为 `final`，但 `setIn/setOut/setErr` 能修改它们（这是 native 方法，绕过了 final 限制）。

### 八、实际应用场景

| 场景 | 用法 |
| --- | --- |
| 控制台输出 | `System.out.println` |
| 错误/日志输出 | `System.err.println` |
| 命令行程序读取输入 | `Scanner(System.in)` |
| 把程序输出存到文件 | `System.setOut(new PrintStream("log"))` |
| 测试时模拟输入 | `System.setIn(new ByteArrayInputStream(...))` |
| 日志框架底层 | 许多日志库最终输出到 System.out/err |

```java
// 测试场景：模拟用户输入，不用真敲键盘
String simulatedInput = "张三\n20\n";
System.setIn(new ByteArrayInputStream(simulatedInput.getBytes()));
Scanner sc = new Scanner(System.in);
// 现在扫描到的是 "张三" 和 "20"
```

### 九、常见问题与注意事项

#### 1. System.in 要包装才能用

直接 `System.in.read()` 读字节，处理文本极麻烦。**用 `Scanner` 或 `BufferedReader(InputStreamReader(...))` 包装**。

#### 2. Scanner 忘记关闭

`Scanner` 包装了 `System.in`，关闭 Scanner 会**连带关闭 System.in**。在某些场景（如 IDE 测试）关闭后再读会出错。注意生命周期。

#### 3. println 中文乱码

控制台编码和 JVM 编码不一致时，`System.out.println` 打印中文乱码。解决：统一用 UTF-8（IDE 设置 + `-Dfile.encoding=UTF-8`）。

#### 4. err 和 out 混用顺序乱

见上文，调试时统一用一个流或手动 flush。

#### 5. 生产环境别用 System.out 打日志

`System.out.println` 性能差、不可控、没法分级。**生产用日志框架**（SLF4J + Logback 等）。`System.out` 只用于调试或简单工具。

---

### 十、总结

| 要点 | 说明 |
| --- | --- |
| 三大标准流 | `System.in`（输入）、`System.out`（输出）、`System.err`（错误） |
| out/err 本质 | 都是 `PrintStream`（见 [[打印流]]） |
| in 本质 | `InputStream`，要包装成字符流/Scanner 用 |
| Scanner | 控制台输入最方便的工具 |
| 重定向 | `setIn/setOut/setErr` 改变流的去向 |

标准流是 Java 程序与外界交互的最基础通道。日常调试、命令行工具、测试都离不开它。
