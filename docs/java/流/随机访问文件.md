### 一、概述

> 📖 [RandomAccessFile（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/RandomAccessFile.html)

`RandomAccessFile`（随机访问文件）是 Java 中一个特殊的类，它能**在文件的任意位置读写**，而不像普通流只能从头到尾顺序读写。

大白话：普通流像「**磁带**」——只能顺序快进；`RandomAccessFile` 像「**黑胶唱片**」——唱针能直接放到任意位置（随机访问）。

> 💡 **提示：** 「随机」在这里不是「随机数」的意思，而是「**任意位置**」（random access，随机存取）之意。内存能随机访问（数组下标直接取），磁盘顺序访问是流，能随机访问的就是 `RandomAccessFile`。

| 特点 | 说明 |
| --- | --- |
| 任意位置读写 | 通过 `seek()` 移动指针 |
| 可读可写 | 一个对象既能读又能写 |
| 独立于流体系 | 不继承 InputStream/OutputStream |
| 适合 | 多线程下载、断点续传、固定格式文件 |

---

### 二、为什么需要它

普通流（`FileInputStream`/`FileOutputStream`）是**单向顺序**的：
- 输入流只能读、不能写
- 输出流只能写、不能读
- 都只能从头到尾，**不能跳着读写**

但很多场景需要「**跳到中间某处读写**」：

- **断点续传**：下载到一半断了，从断开的位置继续
- **多线程下载**：多个线程各下载文件的不同区间
- **修改文件中间某段**：只改某条记录，不重写整个文件
- **大文件分块**：随机读某一块数据

这些用普通流做不到，`RandomAccessFile` 登场。

### 三、基本用法

#### 1. 创建（指定读写模式）

```java
import java.io.RandomAccessFile;
import java.io.IOException;

public class RAFDemo {
    public static void main(String[] args) throws IOException {
        // ✅ 模式："r" 只读 / "rw" 读写 / "rws"/"rwd" 读写+同步刷盘
        RandomAccessFile raf = new RandomAccessFile("data.txt", "rw");
        // ...
    }
}
```

| 模式 | 含义 |
| --- | --- |
| `"r"` | 只读，文件不存在会报错 |
| `"rw"` | 读写，文件不存在会创建 |
| `"rws"` | 读写，每次写入同步刷盘（内容+元数据） |
| `"rwd"` | 读写，每次写入同步刷盘（仅内容） |

> 💡 **提示：** `"rws"`/`"rwd"` 能保证数据立即落盘（断电不丢），但性能差，**仅关键数据用**。

#### 2. 写入与读取

```java
RandomAccessFile raf = new RandomAccessFile("data.txt", "rw");

// 写（从文件指针当前位置写）
raf.writeUTF("你好");      // 写字符串
raf.writeInt(100);         // 写 int
raf.writeDouble(3.14);     // 写 double

// 读取前要先 seek 到开头
raf.seek(0);               // ✅ 移动文件指针到开头
String s = raf.readUTF();  // 你好
int i = raf.readInt();     // 100
double d = raf.readDouble();// 3.14

raf.close();
```

> 💡 **提示：** `RandomAccessFile` 的读写方法名和数据流（[[数据流]]）很像（`writeInt`/`readInt`），因为它也实现了 `DataInput`/`DataOutput` 接口。

### 四、核心方法：文件指针

`RandomAccessFile` 内部维护一个**文件指针（file pointer）**，表示「下一次读写的位置」，类似书签。

| 方法 | 作用 |
| --- | --- |
| `getFilePointer()` | 获取当前指针位置 |
| `seek(long pos)` | **跳到指定位置**（核心！） |
| `length()` | 文件总长度 |
| `skipBytes(int n)` | 向前跳 n 字节 |

```java
RandomAccessFile raf = new RandomAccessFile("data.txt", "r");

long total = raf.length();          // 文件总字节
raf.seek(total / 2);                // ✅ 跳到文件中间
System.out.println(raf.readLine()); // 从中间读一行

System.out.println("当前位置: " + raf.getFilePointer());
```

### 五、经典应用：多线程下载 / 断点续传 ⭐

这是 `RandomAccessFile` 最经典的应用。原理：**先建一个和目标文件一样大的空文件，多个线程各自 `seek` 到自己的区间写入**。

```java
public class MultiThreadDownload {
    public static void main(String[] args) throws IOException {
        long fileSize = 100_000_000L;   // 假设文件 100MB
        int threadCount = 4;            // 4 个线程
        long blockSize = fileSize / threadCount;

        // ✅ 1. 先创建一个指定大小的空文件
        try (RandomAccessFile raf = new RandomAccessFile("movie.mp4", "rw")) {
            raf.setLength(fileSize);    // 预分配文件大小
        }

        // ✅ 2. 每个线程负责一段区间
        for (int i = 0; i < threadCount; i++) {
            long startPos = i * blockSize;
            long endPos = (i == threadCount - 1) ? fileSize : startPos + blockSize;

            new Thread(() -> {
                try (RandomAccessFile raf = new RandomAccessFile("movie.mp4", "rw")) {
                    raf.seek(startPos);   // ✅ 跳到自己的起始位置
                    // 从网络读取这一段数据，写入 raf
                    // ... 下载逻辑 ...
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}
```

```
文件：[========线程1========|========线程2========|========线程3========|========线程4========]
       seek(0)              seek(25M)            seek(50M)            seek(75M)
```

> 💡 **提示：** 这就是迅雷、IDM 等下载工具「多线程加速」的底层原理。

#### 断点续传

下载中断后，记录每个线程已下载到的位置，下次 `seek` 到该位置继续，**不用从头下**。

### 六、固定记录大小的随机读写

如果文件里每条记录大小固定（如每条 100 字节），可以精准定位第 N 条：

```java
// 第 N 条记录的起始位置 = N * 记录大小
raf.seek(recordIndex * RECORD_SIZE);

// 直接修改第 5 条记录（不用重写整个文件）
raf.seek(5 * RECORD_SIZE);
raf.write(...);
```

> 💡 **提示：** 变长记录无法精确定位（不知道第 N 条在哪），随机访问适合**定长记录**。

---

### 七、RandomAccessFile vs 普通流

| 对比 | RandomAccessFile | 普通流（FileInputStream/OutputStream） |
| --- | --- | --- |
| 读写位置 | **任意位置**（seek） | 只能顺序 |
| 读写 | 一个对象既读又写 | 输入/输出分开 |
| 继承体系 | 独立（实现 DataInput/DataOutput） | 继承 InputStream/OutputStream |
| 性能 | 单次随机访问较慢 | 顺序读快（可配缓冲流） |
| 适用 | 任意定位、多线程下载 | 顺序读写 |

### 八、常见问题与注意事项

#### 1. seek 超出文件长度

`seek` 到超过文件大小的位置，再写入，中间会**用 0 填充**（文件被「撑大」）：

```java
raf.seek(1000);   // 超出文件长度
raf.write(65);    // 文件被扩展到 1001 字节，前面全是 0
```

#### 2. 性能问题

`RandomAccessFile` **没有内置缓冲**，频繁小量读写性能差。需要时可自己加缓冲或用 NIO 的 `FileChannel`。

#### 3. 多线程并发

多个线程**各自打开**自己的 `RandomAccessFile` 操作**不同区间**是安全的。但同一区间并发写要加锁，否则数据错乱。

#### 4. 读写顺序

`RandomAccessFile` 的 `readInt`/`writeInt` 等仍要**按类型顺序**读写（和数据流一样）。`seek` 改变的是**起始位置**，不是跳过类型。

#### 5. 是否需要 close

需要！它持有文件句柄，用完要 close（try-with-resources）。

---

### 九、总结

| 要点 | 说明 |
| --- | --- |
| 核心能力 | 文件**任意位置**读写（seek） |
| 模式 | `r` 只读、`rw` 读写 |
| 文件指针 | `seek` 跳转、`getFilePointer` 查位置 |
| 读写方法 | 类似数据流（`writeInt`/`readInt`） |
| 经典应用 | 多线程下载、断点续传、定长记录随机改 |
| 注意 | 无内置缓冲；变长记录难定位；多线程写要分区 |

`RandomAccessFile` 是处理「**需要跳着读写**」场景的利器，普通流做不到的事它来做。
