### 一、概述

> 📖 [BufferedReader（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/BufferedReader.html) ｜ [BufferedInputStream（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/BufferedInputStream.html)

缓冲流（Buffered Stream）是套在普通流外面的**处理流**，作用是**通过内存缓冲区大幅提升读写性能**。它是 [[流概述]] 里讲的**装饰器模式**最典型的应用。

大白话：普通流像「一次搬一块砖」，慢得要命；缓冲流像「用推车一次搬一车砖」——先把砖攒够一车（读到缓冲区），再一次性处理，效率几十倍地提升。

四个缓冲流：

| 字节缓冲流 | 字符缓冲流 |
| --- | --- |
| `BufferedInputStream` | `BufferedReader` |
| `BufferedOutputStream` | `BufferedWriter` |

> 💡 **提示：** 实际开发中，**几乎不会直接用节点流**，而是默认套一层缓冲流。这是性能优化的基本操作。

---

### 二、为什么缓冲流快

#### 1. 节点流的低效

`FileInputStream.read()` 每读 1 个字节，就要做一次磁盘 IO：

```
读 1024 个字节 = 1024 次磁盘访问（极慢！）
```

#### 2. 缓冲流的优化

`BufferedInputStream` 内部维护一个**字节数组缓冲区（默认 8KB）**：

```
① 一次性从磁盘读 8192 字节到缓冲区（1 次磁盘访问）
② 之后 read() 直接从内存缓冲区取（快如闪电）
③ 缓冲区取完了，再批量读下一批
```

```
读 1024 个字节：缓冲流只触发 1 次磁盘访问（一次读 8192 够用了）
```

> 💡 **提示：** 磁盘 IO 比内存慢几万倍，**减少 IO 次数**就是缓冲流提速的本质。

#### 3. 写入的缓冲

写数据时同理：`BufferedOutputStream.write()` 先写到内存缓冲区，攒满 8KB 才真正写磁盘（`flush`）。

```
write() 一个字节 → 先进缓冲区（不落盘）
缓冲区满 / 调用 flush() / 关闭流 → 才一次性写入磁盘
```

> ⚠️ **注意：** 因为写入是先缓冲、后落盘，**不及时 flush/close 可能丢数据**（程序崩溃时缓冲区内容没写出去）。务必正确关闭。

### 三、基本用法

#### 1. 字节缓冲流（读写二进制/文件）

```java
import java.io.*;

public class BufferedByteDemo {
    public static void main(String[] args) throws IOException {
        // ✅ 复制文件：节点流套缓冲流
        try (BufferedInputStream bis = new BufferedInputStream(
                new FileInputStream("source.jpg"));
             BufferedOutputStream bos = new BufferedOutputStream(
                new FileOutputStream("target.jpg"))) {

            byte[] buffer = new byte[8192];
            int len;
            while ((len = bis.read(buffer)) != -1) {
                bos.write(buffer, 0, len);
            }
        }   // 自动关闭（会触发 flush + close）
    }
}
```

#### 2. 字符缓冲流（读写文本）

```java
import java.io.*;

public class BufferedCharDemo {
    public static void main(String[] args) throws IOException {
        // 读
        try (BufferedReader reader = new BufferedReader(
                new FileReader("a.txt"))) {
            String line;
            while ((line = reader.readLine()) != null) {   // ✅ 按行读
                System.out.println(line);
            }
        }

        // 写
        try (BufferedWriter writer = new BufferedWriter(
                new FileWriter("b.txt"))) {
            writer.write("第一行");
            writer.newLine();              // ✅ 写跨平台换行符
            writer.write("第二行");
        }
    }
}
```

#### 3. 指定缓冲区大小

```java
// 默认 8192 字节，可自定义（一般用默认即可）
new BufferedInputStream(new FileInputStream("a.bin"), 16384);
```

### 四、缓冲流特有方法

#### BufferedReader 特有

| 方法 | 作用 |
| --- | --- |
| `readLine()` | **读取一整行**（不含换行符），到末尾返回 `null` |
| `lines()` | 返回 `Stream<String>`，可用流式 API 处理（Java 8+） |

```java
// 用 lines() 流式处理大文件（Java 8+）
try (BufferedReader reader = new BufferedReader(new FileReader("big.log"))) {
    reader.lines()
          .filter(line -> line.contains("ERROR"))
          .forEach(System.out::println);
}
```

#### BufferedWriter 特有

| 方法 | 作用 |
| --- | --- |
| `newLine()` | 写一个**跨平台的换行符**（Windows 是 `\r\n`，Linux 是 `\n`） |

> 💡 **提示：** 写文本换行，**优先用 `newLine()` 而不是写死 `\n`**，保证跨平台一致。

### 五、自定义缓冲区 vs 缓冲流

很多人会问：我自己用 `read(byte[])` 指定数组，是不是就不需要缓冲流了？

```java
// 方式A：节点流 + 自己的数组
FileInputStream fis = new FileInputStream("a");
byte[] buf = new byte[8192];
fis.read(buf);

// 方式B：节点流套缓冲流
BufferedInputStream bis = new BufferedInputStream(new FileInputStream("a"));
bis.read();   // 单字节读，但有内部缓冲
```

两者性能接近，但：

| 对比 | 自定义数组 | 缓冲流 |
| --- | --- | --- |
| 单字节 `read()` 性能 | 慢（每字节一次 IO） | 快（内部缓冲） |
| 代码简洁度 | 要自己管数组 | 透明，自动缓冲 |
| 灵活性 | 高 | 封装好 |
| 推荐度 | 批量读写够用 | **单字节读写场景必用** |

> 💡 **提示：** 一般场景**「节点流 + 自定义数组」和「缓冲流」二选一**即可，没必要 `BufferedInputStream` 里再套自定义大数组（重复缓冲意义不大）。

### 六、flush 的作用

`flush()` 强制把缓冲区的数据**真正写入目的地**（落盘/发出）。

```java
BufferedWriter writer = new BufferedWriter(new FileWriter("log.txt"));
writer.write("重要数据");
// 此时数据还在缓冲区，文件里没有！

writer.flush();   // ✅ 强制落盘
// 或 writer.close() 也会自动 flush
```

什么场景必须手动 flush？
- **网络流**：写完想立即让对方收到（不 flush 对方收不到）
- **日志实时写入**：希望立即看到日志
- **关闭前确保数据落盘**

> ⚠️ **注意：** `BufferedOutputStream`/`PrintWriter` 等带 autoflush 的，`close()` 时会自动 flush。但**网络通信场景**仍需主动 flush（见 [[Socket通信]]）。

| 场景 | 用法 |
| --- | --- |
| 文件复制 | BufferedInputStream + BufferedOutputStream |
| 读大文本日志 | BufferedReader + readLine / lines() |
| CSV/配置文件处理 | BufferedReader 逐行解析 |
| 网络通信包装 | Socket 流套 BufferedReader/Writer |
| 写日志 | BufferedWriter（配合 flush） |

### 八、常见问题与注意事项

#### 1. 忘记 flush/close 导致数据丢失

```java
// ❌ 没 close，缓冲区数据没落盘
BufferedWriter writer = new BufferedWriter(new FileWriter("a.txt"));
writer.write("数据");
// 程序结束，文件可能是空的！

// ✅ try-with-resources 自动 close（含 flush）
try (BufferedWriter writer = new BufferedWriter(new FileWriter("a.txt"))) {
    writer.write("数据");
}
```

#### 2. readLine 读取卡住

`readLine()` 遇到**换行符**才返回一行。如果数据源没用换行符结尾（如网络流没发 `\n`），会一直阻塞。详见 [[Socket通信]] 的注意事项。

#### 3. 缓冲流套缓冲流

```java
// ❌ 没必要的双重缓冲
new BufferedInputStream(new BufferedInputStream(new FileInputStream("a")));

// ✅ 一层就够
new BufferedInputStream(new FileInputStream("a"));
```

#### 4. 缓冲区大小怎么选

默认 8KB（8192）是经验最优值，**绝大多数情况不用改**。极特殊场景（如大文件流式拷贝）可调到 16KB~64KB。

#### 5. BufferedReader 不处理编码

`new BufferedReader(new FileReader("a.txt"))` 用系统默认编码读，跨平台可能乱码。要指定编码：

```java
// ✅ FileReader（Java 11+）支持指定编码
new BufferedReader(new FileReader("a.txt", StandardCharsets.UTF_8));
// 或用转换流（见 [[转换流]]）
new BufferedReader(new InputStreamReader(new FileInputStream("a.txt"), "UTF-8"));
```

### 九、总结

| 要点 | 说明 |
| --- | --- |
| 作用 | 用内存缓冲区减少 IO 次数，大幅提速 |
| 本质 | 装饰器模式，套在节点流外 |
| 字节缓冲 | `BufferedInputStream` / `BufferedOutputStream` |
| 字符缓冲 | `BufferedReader` / `BufferedWriter` |
| 特有方法 | `readLine()`、`newLine()`、`lines()` |
| flush | 强制把缓冲区数据写出去 |
| 默认缓冲区 | 8KB |

一句话：**读写文件时，默认就套一层缓冲流**，这是性能优化的最低成本手段。
