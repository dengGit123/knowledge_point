### 一、概述

> 📖 [ObjectInputStream（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/ObjectInputStream.html) ｜ [ObjectOutputStream（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/ObjectOutputStream.html) ｜ [Java 序列化（Oracle）](https://docs.oracle.com/javase/tutorial/jndi/objects/serial.html)

对象流用于**直接读写 Java 对象**。它能把一个对象「拍扁」成一串字节写出去（序列化），也能把字节「还原」成对象（反序列化）。

大白话：你想把一个 `User` 对象存到文件里、或通过网络发给另一台机器，怎么办？对象流帮你把对象「**打包成字节流**」传过去，对方再「**拆包还原**」成 `User` 对象。

| 流 | 作用 |
| --- | --- |
| `ObjectOutputStream` | 把对象**序列化**写入（对象 → 字节） |
| `ObjectInputStream` | 把字节**反序列化**读出（字节 → 对象） |

> 💡 **提示：** 对象流是 Java 序列化机制的 IO 入口。关于序列化的完整原理（版本号、安全性等）见 [[序列化和反序列化]]。本篇聚焦「怎么用对象流」。

---

### 二、前提：对象必须可序列化

要用对象流读写，对象所属的类**必须实现 `Serializable` 接口**：

```java
import java.io.Serializable;

// ✅ 实现 Serializable 接口（标记接口，无需实现任何方法）
public class User implements Serializable {
    private String name;
    private int age;

    // 推荐显式声明 serialVersionUID
    private static final long serialVersionUID = 1L;

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public String toString() {
        return "User{name='" + name + "', age=" + age + "}";
    }
}
```

> ⚠️ **注意：** 不实现 `Serializable` 直接用对象流，会抛 `NotSerializableException`。

### 三、基本用法

#### 1. 序列化（写出对象）

```java
import java.io.*;

public class SerializeDemo {
    public static void main(String[] args) throws IOException {
        User user = new User("张三", 20);

        // ✅ 用 ObjectOutputStream 把对象写入文件
        try (ObjectOutputStream oos = new ObjectOutputStream(
                new FileOutputStream("user.dat"))) {
            oos.writeObject(user);   // 序列化
        }
        System.out.println("序列化完成");
    }
}
```

#### 2. 反序列化（读回对象）

```java
import java.io.*;

public class DeserializeDemo {
    public static void main(String[] args) throws Exception {
        // ✅ 用 ObjectInputStream 把字节读回对象
        try (ObjectInputStream ois = new ObjectInputStream(
                new FileInputStream("user.dat"))) {
            User user = (User) ois.readObject();   // 反序列化（要强转）
            System.out.println("读回: " + user);    // User{name='张三', age=20}
        }
    }
}
```

#### 3. 序列化多个对象

```java
// 写多个对象
try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("list.dat"))) {
    oos.writeObject(user1);
    oos.writeObject(user2);
}

// 读多个：读到末尾会抛 EOFException，用它判断结束
try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream("list.dat"))) {
    while (true) {
        try {
            User u = (User) ois.readObject();
            System.out.println(u);
        } catch (EOFException e) {
            break;   // 读完了
        }
    }
}
```

> 💡 **提示：** 也可以把多个对象放进 `List` / 数组，序列化整个集合，读回时一次拿到。

### 四、transient 关键字

用 `transient` 修饰的字段**不会被序列化**（敏感数据、临时数据）。

```java
public class User implements Serializable {
    private String name;
    private transient String password;   // ✅ 密码不序列化
    // 反序列化后 password = null
}
```

| 何时用 transient | 例子 |
| --- | --- |
| 敏感信息 | 密码、token |
| 不可序列化的字段 | 数据库连接、线程对象 |
| 临时计算字段 | 缓存、临时状态 |

---

### 五、serialVersionUID（版本号）

序列化时，类有个版本号。**反序列化时，类的版本号必须和序列化时一致**，否则抛 `InvalidClassException`。

```java
// ✅ 推荐显式声明，避免自动生成导致不兼容
private static final long serialVersionUID = 1L;
```

| 做法 | 风险 |
| --- | --- |
| **显式声明** | 可控，新增字段也算兼容（推荐） |
| 不声明（编译器自动算） | 改一点代码，版本号就变，旧的序列化数据全读不回来 |

> ⚠️ **注意：** 凡是要序列化的类，**务必显式声明 `serialVersionUID`**，否则一次无关的代码改动就可能让历史数据反序列化失败。

### 六、序列化的注意事项

#### 1. 引用类型字段也要可序列化

```java
public class Order implements Serializable {
    private User user;   // ✅ User 也必须实现 Serializable
    private List<Item> items;  // 集合里的元素类型也要可序列化
}
```

对象图里所有被引用的对象，**必须都可序列化**（`transient` 的除外）。

#### 2. static 字段不参与序列化

`static` 属于类，不属于对象，**序列化只存对象的实例数据**。

#### 3. 循环引用

对象流能正确处理**循环引用**（A 引用 B，B 引用 A），不会死循环——序列化会记录对象引用，重复对象只存一份。

### 七、对象流 vs 数据流

| 对比 | 对象流 | 数据流 |
| --- | --- | --- |
| 读写单位 | 整个对象 | 基本类型（int/double/...） |
| 类要求 | 实现 Serializable | 无 |
| 用途 | 存对象、网络传对象 | 存基本类型数据 |
| 灵活性 | 高（存复杂对象） | 低（只存基本类型） |

要存基本类型，也可以用 [[数据流]]；要存整个对象，用对象流。详见 [[数据流]]。

### 八、实际应用场景

| 场景 | 用法 |
| --- | --- |
| 对象持久化 | 把对象存到文件，下次启动读回 |
| 网络传输对象 | Socket 传对象（见 [[Socket通信]]） |
| 深拷贝 | 序列化再反序列化实现深拷贝 |
| 缓存 | 把对象序列化存 Redis/磁盘 |
| Session 持久化 | Web 容器钝化 Session |

```java
// 用对象流做深拷贝（巧妙用法）
public static <T> T deepCopy(T obj) throws Exception {
    ByteArrayOutputStream baos = new ByteArrayOutputStream();
    try (ObjectOutputStream oos = new ObjectOutputStream(baos)) {
        oos.writeObject(obj);
    }
    try (ObjectInputStream ois = new ObjectInputStream(
            new ByteArrayInputStream(baos.toByteArray()))) {
        return (T) ois.readObject();   // 完全独立的新对象
    }
}
```

### 九、Java 原生序列化的局限（重要）

Java 原生序列化（`ObjectOutputStream`）有**严重缺陷**，现代项目慎用：

| 缺陷 | 说明 |
| --- | --- |
| 不安全 | 反序列化可触发任意代码（**反序列化漏洞**，曾导致无数事故） |
| 跨语言差 | 只有 Java 能读，不能和别的语言互通 |
| 体积大 | 带类信息，冗余多 |
| 性能一般 | 序列化/反序列化慢 |

> ⚠️ **注意：** Oracle 官方已建议**弃用 Java 原生序列化**。生产环境推荐用 **JSON（Jackson、Gson）** 或 **Protobuf、Hessian** 等替代方案。

详见 [[序列化和反序列化]]。

### 十、常见问题与注意事项

#### 1. NotSerializableException

某个字段或其引用的对象没实现 `Serializable`。检查整个对象图。

#### 2. InvalidClassException

反序列化时类变了（加了字段、改了结构）且没声明 `serialVersionUID`。**显式声明版本号**。

#### 3. 反序列化漏洞（安全）

绝不要反序列化**不可信来源**的数据（如用户传来的字节流），可能执行恶意代码。生产用 JSON。

#### 4. transient 后是默认值

`transient` 字段反序列化后是**类型的默认值**（int=0，对象=null），不会保留原值。

#### 5. 序列化顺序 = 反序列化顺序

写多个对象时，读取必须**按相同顺序**读回。

### 十一、总结

| 要点 | 说明 |
| --- | --- |
| 作用 | 直接读写 Java 对象 |
| 前提 | 类实现 `Serializable` |
| 核心方法 | `writeObject` / `readObject` |
| transient | 标记字段不序列化 |
| serialVersionUID | 版本号，必须显式声明 |
| 安全警示 | 原生序列化有漏洞，生产用 JSON/Protobuf |

对象流让对象的存储和传输变得简单，但要注意**安全性和跨语言**问题。深入学习序列化原理见 [[序列化和反序列化]]。
