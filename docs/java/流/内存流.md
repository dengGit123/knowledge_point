### 一、概述

> 📖 [ByteArrayInputStream（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/ByteArrayInputStream.html) ｜ [ByteArrayOutputStream（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/ByteArrayOutputStream.html)

内存流是**以内存（字节数组 / 字符串）为数据源**的流，不涉及磁盘文件或网络。它的数据全部在内存里读写，速度极快。

大白话：普通流是「水龙头接水管」（连着文件/网络）；内存流是「**拿个桶在屋里接水**」——数据从头到尾都在内存里打转，不用碰磁盘。

| 流 | 数据源 | 说明 |
| --- | --- | --- |
| `ByteArrayInputStream` | 字节数组 | 从内存数组读 |
| `ByteArrayOutputStream` | 内存缓冲区 | 写到内存数组 |
| `StringReader` | 字符串 | 从字符串读字符 |
| `StringWriter` | 内存缓冲区 | 写成字符串 |
| `CharArrayReader/Writer` | 字符数组 | 从 char[] 读写 |

> 💡 **提示：** 内存流常用作「**中间桥梁**」——比如先在内存里组装数据，再统一发出去；或把字节流数据转成字符串处理。

### 二、ByteArrayInputStream / ByteArrayOutputStream（字节内存流）

#### 1. ByteArrayOutputStream（写到内存）

```java
import java.io.*;

public class BAOSDemo {
    public static void main(String[] args) throws IOException {
        ByteArrayOutputStream baos = new ByteArrayOutputStream();

        baos.write(65);                      // 写一个字节
        baos.write("Hello".getBytes());      // 写字节数组
        baos.write(new byte[]{66, 67});      // 写 B, C

        // 取出全部数据
        byte[] all = baos.toByteArray();     // [65, H, e, l, l, o, 66, 67]
        System.out.println(new String(all)); // AHelloBC

        // 或转成字符串
        System.out.println(baos.toString()); // AHelloBC
    }
}
```

`ByteArrayOutputStream` 内部维护一个**自动扩容**的字节数组，写入多少都行，最后一次性 `toByteArray()` 取出。

#### 2. ByteArrayInputStream（从内存读）

```java
import java.io.*;

public class BAISDemo {
    public static void main(String[] args) throws IOException {
        byte[] data = "Hello".getBytes();
        ByteArrayInputStream bais = new ByteArrayInputStream(data);

        int b;
        while ((b = bais.read()) != -1) {
            System.out.print((char) b);   // Hello
        }
        // ✅ ByteArrayInputStream 不需要 close（它没有底层资源）
    }
}
```

### 三、核心应用场景

#### 1. 数据组装后再统一输出

```java
// 先在内存组装，再一次性写文件/网络
ByteArrayOutputStream buffer = new ByteArrayOutputStream();
buffer.write("头部".getBytes());
buffer.write("正文".getBytes());
buffer.write("尾部".getBytes());

// 一次性写出
byte[] result = buffer.toByteArray();
```

#### 2. 配合对象流做深拷贝 / 内存序列化

```java
// 对象 → 字节 → 对象（内存中完成，常用于深拷贝）
ByteArrayOutputStream baos = new ByteArrayOutputStream();
try (ObjectOutputStream oos = new ObjectOutputStream(baos)) {
    oos.writeObject(user);
}

try (ObjectInputStream ois = new ObjectInputStream(
        new ByteArrayInputStream(baos.toByteArray()))) {
    User copy = (User) ois.readObject();   // 深拷贝结果
}
```

详见 [[对象流]]。

#### 3. 字节流 ↔ 字符串转换

```java
// 把一段字节解码成字符串（指定编码）
ByteArrayOutputStream baos = new ByteArrayOutputStream();
baos.write(bytes);
String text = baos.toString("UTF-8");
```

#### 4. 处理不定长输入

不知道数据有多长时，先全写进 `ByteArrayOutputStream`，最后统一处理（如读取 HTTP 响应体）。

### 四、StringReader / StringWriter（字符内存流）

字符版的内存流，数据源是字符串：

```java
import java.io.*;

public class StringStreamDemo {
    public static void main(String[] args) throws IOException {
        // StringReader：把字符串当流读
        StringReader reader = new StringReader("第一行\n第二行");
        BufferedReader br = new BufferedReader(reader);
        System.out.println(br.readLine());   // 第一行

        // StringWriter：写到内存字符串
        StringWriter writer = new StringWriter();
        writer.write("Hello");
        writer.write(" World");
        String result = writer.toString();   // Hello World
    }
}
```

> 💡 **提示：** `StringWriter` 常用于「**把异常堆栈转成字符串**」：

```java
// 经典用法：把异常堆栈转字符串
StringWriter sw = new StringWriter();
exception.printStackTrace(new PrintWriter(sw));
String stackTrace = sw.toString();   // 拿到堆栈字符串
```

### 五、内存流的特点

| 特点 | 说明 |
| --- | --- |
| 极快 | 全在内存，无 IO |
| 无需关闭 | ByteArrayInputStream/StringReader 等不持有系统资源，close 是空操作 |
| 容量限制 | 受内存大小限制，**别用它存超大数据**（OOM 风险） |
| 支持reset | ByteArrayInputStream 可 `reset()` 回到起点重新读 |

> 💡 **提示：** 内存流**不需要关闭**（关了也无害），但关了也无妨。习惯上仍可放在 try-with-resources 里。

### 六、内存流 vs 节点流

| 对比 | 内存流 | 文件流（节点流） |
| --- | --- | --- |
| 数据源 | 内存数组/字符串 | 磁盘文件 |
| 速度 | 极快 | 受磁盘 IO 限制 |
| 持久化 | ❌ 程序结束就没了 | ✅ 落盘 |
| 容量 | 受内存限制 | 受磁盘限制 |
| 关闭 | 不强制 | 必须关闭 |

### 七、实际应用场景

| 场景 | 用法 |
| --- | --- |
| 深拷贝对象 | ByteArrayOutputStream + ObjectInputStream |
| 数据缓冲组装 | ByteArrayOutputStream 拼接后再输出 |
| 异常堆栈转字符串 | printStackTrace(StringWriter) |
| 字节↔字符串转换 | ByteArrayOutputStream.toString(编码) |
| 测试 | 用内存流代替文件流，方便单元测试 |
| 图片/文件处理 | 在内存中处理字节后再落盘 |

```java
// 经典：把 InputStream（如网络响应）完整读成字符串
public static String readAll(InputStream in) throws IOException {
    ByteArrayOutputStream baos = new ByteArrayOutputStream();
    byte[] buf = new byte[8192];
    int len;
    while ((len = in.read(buf)) != -1) {
        baos.write(buf, 0, len);
    }
    return baos.toString("UTF-8");
}
```

### 八、常见问题与注意事项

#### 1. 别用内存流存大数据

`ByteArrayOutputStream` 把所有数据放内存，处理几个 GB 的文件会直接 OOM。大文件用文件流 + 缓冲流。

#### 2. toString 默认编码

`ByteArrayOutputStream.toString()` 用默认编码，可能乱码。指定：`baos.toString("UTF-8")` 或 `baos.toString(StandardCharsets.UTF_8)`。

#### 3. ByteArrayInputStream 可重复读

```java
ByteArrayInputStream bais = new ByteArrayInputStream(data);
// 读一遍后
bais.reset();   // ✅ 回到起点，可以再读一遍
```

普通文件流读完不能回头，内存流可以。

#### 4. 线程安全

内存流**不是线程安全**的，多线程共享要注意同步。

### 九、总结

| 要点 | 说明 |
| --- | --- |
| 作用 | 以内存数组/字符串为数据源，无磁盘 IO |
| 字节版 | `ByteArrayInputStream` / `ByteArrayOutputStream` |
| 字符版 | `StringReader` / `StringWriter` / `CharArrayReader` |
| 特点 | 极快、无需关闭、受内存限制 |
| 经典用法 | 深拷贝、数据组装、异常转字符串、字节↔字符串 |

内存流是「**内存里的数据搬运工**」，常作为中间缓冲桥梁使用，配合对象流、转换流能实现很多巧妙功能。
