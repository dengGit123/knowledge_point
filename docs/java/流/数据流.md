### 一、概述

> 📖 [DataInputStream（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/DataInput.html) ｜ [DataOutputStream（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/DataOutput.html)

数据流用于**读写 Java 的基本数据类型**（int、double、boolean、UTF 字符串等）。它能保证「写出去的 int，读回来还是 int」，类型不会丢失。

大白话：普通字节流读写的是「裸字节」，你自己也不知道一段字节是 int 还是 double；数据流像一个「**贴标签的打包机**」——按 Java 的格式把基本类型精确地写出去，读回来时也按类型精确读取，**读写顺序一致就能完美还原**。

| 流 | 作用 |
| --- | --- |
| `DataOutputStream` | 写基本类型（int → 4 字节等） |
| `DataInputStream` | 读基本类型 |

> 💡 **提示：** 数据流读写的是**二进制格式**（不是文本），适合存结构化的数值数据。要存整个对象用 [[对象流]]。

---

### 二、为什么需要数据流

#### 1. 字节流读数字的麻烦

```java
// 数字 658467 用字节流写，是 3 个字节，读回来要自己拼接还原
// ❌ 手动拼字节计算，极易出错

// ✅ 数据流一行搞定
dataOutputStream.writeInt(658467);
int n = dataInputStream.readInt();   // 658467，类型完整
```

#### 2. 保证类型不丢失

数据流按 Java 规范的二进制格式存储，**跨平台读写一致**（解决了字节序问题）：

| 类型 | write 方法 | 占用字节 | read 方法 |
| --- | --- | --- | --- |
| int | `writeInt` | 4 | `readInt` |
| long | `writeLong` | 8 | `readLong` |
| double | `writeDouble` | 8 | `readDouble` |
| boolean | `writeBoolean` | 1 | `readBoolean` |
| char | `writeChar` | 2 | `readChar` |
| UTF 字符串 | `writeUTF` | 变长 | `readUTF` |

---

### 三、基本用法

#### 1. 写数据

```java
import java.io.*;

public class DataWriteDemo {
    public static void main(String[] args) throws IOException {
        try (DataOutputStream dos = new DataOutputStream(
                new FileOutputStream("data.bin"))) {
            dos.writeInt(100);              // 写 int
            dos.writeDouble(3.14);          // 写 double
            dos.writeBoolean(true);         // 写 boolean
            dos.writeChar('A');             // 写 char
            dos.writeUTF("你好");           // 写 UTF 字符串
        }
        System.out.println("写入完成");
    }
}
```

#### 2. 读数据（顺序必须一致！）

```java
import java.io.*;

public class DataReadDemo {
    public static void main(String[] args) throws IOException {
        try (DataInputStream dis = new DataInputStream(
                new FileInputStream("data.bin"))) {

            int i = dis.readInt();            // 100
            double d = dis.readDouble();      // 3.14
            boolean b = dis.readBoolean();    // true
            char c = dis.readChar();          // A
            String s = dis.readUTF();         // 你好

            System.out.println(i + ", " + d + ", " + b + ", " + c + ", " + s);
        }
    }
}
```

> ⚠️ **注意：** 读取的**顺序必须和写入顺序完全一致**！写了 int、double、boolean，就必须按 int、double、boolean 读。顺序错了，数据全乱。

### 四、writeUTF / readUTF 的特殊性

`writeUTF` 写的不是标准 UTF-8，而是 Java 自己的「**Modified UTF-8**」格式：

- 开头有 2 字节表示字符串长度
- 适合用 `readUTF` 配对读取

```java
dos.writeUTF("标准用法");
String s = dis.readUTF();   // ✅ 配对使用
```

> 💡 **提示：** `writeUTF` 写出的数据**只能用 `readUTF` 读**，混用普通字符流读会乱。

### 五、数据流 vs 对象流 vs 普通字节流

| 对比 | 普通字节流 | 数据流 | 对象流 |
| --- | --- | --- | --- |
| 读写单位 | 字节 | 基本类型 | 整个对象 |
| 类型保留 | ❌ | ✅ | ✅ |
| 要实现接口 | 否 | 否 | `Serializable` |
| 适用 | 文件复制、二进制 | 存数值记录 | 存对象 |
| 体积 | 最小 | 较小 | 较大（带类信息） |

**选型：**
- 只存一堆**基本类型数值** → 数据流
- 存**完整对象** → 对象流
- 复制**任意文件** → 普通字节流（见 `../IO/字节流之输入流`）

### 六、实际应用场景

| 场景 | 用法 |
| --- | --- |
| 保存游戏数据 | 角色等级、金币、坐标（一堆 int/double） |
| 二进制日志 | 记录结构化的数值事件 |
| 自定义二进制协议 | 按固定格式收发数据（配合 [[Socket通信]]） |
| 缓存基本数据 | 存计算结果到文件 |

```java
// 游戏存档示例：存一堆基本类型
try (DataOutputStream dos = new DataOutputStream(new FileOutputStream("save.dat"))) {
    dos.writeUTF(playerName);
    dos.writeInt(level);
    dos.writeInt(gold);
    dos.writeDouble(x);
    dos.writeDouble(y);
}
```

### 七、常见问题与注意事项

#### 1. 读到末尾抛 EOFException

数据流读完继续读，会抛 `EOFException`（而不是像普通流返回 -1）。处理方式：

```java
try {
    while (true) {
        int n = dis.readInt();   // 读到末尾抛 EOFException
        // 处理
    }
} catch (EOFException e) {
    // 读完了，正常结束
}
```

> 💡 **提示：** `EOFException` 在数据流里是**正常**的结束信号，不是错误，catch 后结束即可。

#### 2. 读写顺序必须一致

写了什么类型，就必须按相同顺序读。**类型或顺序错一个，全部乱套**。

#### 3. readUTF 读不了 writeUTF 以外写的数据

`readUTF` 期望开头是 2 字节长度。如果用普通字符流写的 UTF-8，用 `readUTF` 读会出错。

#### 4. 不适合存大文本

数据流是为**结构化数值**设计的，存大段文本不如用字符流（[[转换流]]）。

### 八、总结

| 要点 | 说明 |
| --- | --- |
| 作用 | 精确读写 Java 基本类型 |
| 核心 | `writeXxx` / `readXxx` 成对使用，顺序一致 |
| writeUTF | 特殊的 Modified UTF-8，必须配对 readUTF |
| 结束标志 | 抛 `EOFException`（非 -1） |
| 适用 | 存数值记录、自定义二进制协议 |

数据流解决了「字节流读写基本类型」的麻烦，是处理**结构化二进制数据**的利器。
