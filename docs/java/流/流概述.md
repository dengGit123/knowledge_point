### 一、概述

> 📖 [Java IO（Oracle 教程）](https://docs.oracle.com/javase/tutorial/essential/io/streams.html) ｜ [java.io 包](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/package-summary.html)

「流」（Stream）是 Java 处理**数据输入输出**的核心抽象。它把数据的传输想象成「水流」——数据像水一样从一端（源）**流向**另一端（目的地），程序只需要在流的管道上读取或写入。

大白话：流就是一根「**数据管道**」。你想从文件读数据，就接一根「输入流」管道，数据从文件流向你的程序；你想往文件写数据，就接一根「输出流」管道，数据从你的程序流向文件。

Java 的 IO 流体系庞大（光 `java.io` 就有几十个流类），但只要抓住**三个分类维度**和**一个设计模式（装饰器）**，就能理清全部。

| 本文讲什么 | 说明 |
| --- | --- |
| 流的分类 | 按方向、按数据单位、按功能 |
| 四大抽象基类 | InputStream/OutputStream/Reader/Writer |
| 装饰器模式 | 理解流为什么能层层嵌套 |
| 流的使用与关闭 | try-with-resources、关闭顺序 |

> 💡 **提示：** 本文是「流」系列的总览。基础字节流/字符流的用法见 `../IO/` 目录下的 [[字节流之输入流]] 等；各类功能流见本目录其他文档。

### 二、流的分类

#### 1. 按方向：输入流 vs 输出流

| 类型 | 方向 | 基类 | 动作 |
| --- | --- | --- | --- |
| **输入流** | 数据 → 程序（读） | `InputStream` / `Reader` | `read()` |
| **输出流** | 程序 → 数据（写） | `OutputStream` / `Writer` | `write()` |

> 💡 **提示：** 「输入/输出」永远是**站在程序的角度**看：数据流进程序就是输入，流出程序就是输出。

#### 2. 按数据单位：字节流 vs 字符流

| 类型 | 单位 | 基类 | 适用场景 |
| --- | --- | --- | --- |
| **字节流** | 字节（byte，1 字节） | `InputStream` / `OutputStream` | 一切数据（图片、视频、二进制文件） |
| **字符流** | 字符（char） | `Reader` / `Writer` | 文本（自动处理编码） |

> 💡 **提示：** 记住一句话——**「字节流万能，字符流专攻文本」**。读图片、视频必须用字节流；读文本用字符流更方便（自动处理编码，避免乱码）。

#### 3. 按功能：节点流 vs 处理流（装饰器）

| 类型 | 说明 | 例子 |
| --- | --- | --- |
| **节点流** | 直接连接数据源（文件、内存、网络） | `FileInputStream`、`FileReader` |
| **处理流** | 包装在节点流之上，增强功能 | `BufferedReader`、`InputStreamReader` |

处理流**不能单独存在**，必须套在节点流外面：

```java
// 节点流（直接接文件）被处理流（缓冲、转换）层层包装
BufferedReader reader = new BufferedReader(
    new InputStreamReader(
        new FileInputStream("a.txt"), "UTF-8"));
```

### 三、四大抽象基类

Java 所有流类的「祖宗」是这四个抽象基类：

```
           字节流                   字符流
输入   InputStream              Reader
输出   OutputStream             Writer
```

| 基类 | 核心方法 |
| --- | --- |
| `InputStream` | `int read()` 读一个字节 |
| `OutputStream` | `void write(int b)` 写一个字节 |
| `Reader` | `int read()` 读一个字符 |
| `Writer` | `void write(String str)` 写字符串 |

记忆口诀：**「字节 Input/Output，字符 Reader/Writer」**。

#### 常见流类继承关系（节选）

```
InputStream (字节输入根)
├── FileInputStream          ← 节点流，读文件
├── ByteArrayInputStream     ← 节点流，读内存
├── FilterInputStream        ← 处理流基类
│   ├── BufferedInputStream  ← 缓冲流
│   └── DataInputStream      ← 数据流
└── ObjectInputStream        ← 对象流

Reader (字符输入根)
├── FileReader               ← 节点流，读文件
├── BufferedReader           ← 缓冲流
└── InputStreamReader        ← 转换流（字节→字符）
```

### 四、装饰器设计模式（核心！）

Java IO 流最让人困惑的是：**为什么 new 一个流要套好几层？** 这正是**装饰器模式（Decorator）**。

#### 1. 装饰器的思想

装饰器 = **不改变原对象，动态地给它增加功能**。就像给手机「套手机壳」「贴膜」——手机还是那个手机，但功能（防护）增强了。

```
FileInputStream          ← 能读文件（基础功能）
  └─ 套上 BufferedInputStream → 现在还能「缓冲」（增强性能）
      └─ 套上 DataInputStream  → 现在还能「读基本类型」（增强功能）
```

#### 2. 装饰器在 IO 中的体现

所有 `FilterXxx` 和处理流都是装饰器：

```java
// 想要「能读文件 + 有缓冲 + 能读 Java 类型」的流，就层层套：
DataInputStream dis = new DataInputStream(
    new BufferedInputStream(
        new FileInputStream("data.bin")));

int n = dis.readInt();   // 享受所有增强功能
```

#### 3. 为什么用装饰器而不是继承？

如果用继承，要为每种组合都建一个类（FileBufferedDataStreamInputStream、FileBufferedInputStream...），类爆炸。装饰器让你**像搭积木一样自由组合**功能。

> 💡 **提示：** 理解装饰器，就看懂了 Java IO 流的全部设计。详见设计模式相关文档。

### 五、流的标准用法

#### 1. 基本步骤：创建 → 读写 → 关闭

```java
// 字符流读文件（最经典的写法）
try (FileReader reader = new FileReader("a.txt")) {  // ① 创建（try-with-resources）
    int ch;
    while ((ch = reader.read()) != -1) {             // ② 读写（读到 -1 表示结束）
        System.out.print((char) ch);
    }
}                                                    // ③ 自动关闭
```

#### 2. 包装流：层层嵌套

```java
// 高效读文本（缓冲 + 指定编码）
try (BufferedReader reader = new BufferedReader(
        new InputStreamReader(
            new FileInputStream("a.txt"), "UTF-8"))) {
    String line;
    while ((line = reader.readLine()) != null) {     // 按行读
        System.out.println(line);
    }
}
```

> 💡 **提示：** 嵌套顺序：**最内层是节点流（接数据源），向外逐层套处理流（增强功能）**。

### 六、流的关闭（重要！）

#### 1. 为什么必须关闭

流持有操作系统资源（文件句柄、网络连接）。**不关闭会导致资源泄漏**，最终报 `Too many open files`，程序崩溃。

#### 2. try-with-resources（推荐）

```java
// ✅ JDK 7+，自动关闭（流实现了 AutoCloseable）
try (FileInputStream fis = new FileInputStream("a.txt")) {
    // 使用 fis
}   // 这里自动调用 fis.close()
```

#### 3. 关闭顺序

嵌套的流，**只需关闭最外层**——关闭外层会自动级联关闭内层：

```java
try (BufferedReader reader = new BufferedReader(
        new FileReader("a.txt"))) {  // 只关闭外层 reader 即可
    // ...
}
```

> ⚠️ **注意：** 如果内层流可能关闭失败而外层不级联，或需要精细控制，再单独处理。一般场景关最外层就够。

#### 4. JDK 9+ 增强： effectively final 变量

```java
// JDK 9+ 可直接用已存在的流变量
BufferedReader reader = Files.newBufferedReader(path);
try (reader) {   // 直接用变量
    // ...
}
```

### 八、常见问题与注意事项

#### 1. 字节流和字符流怎么选？

- 读**非文本**（图片、视频、压缩包）→ 字节流
- 读**文本**（.txt、.java、.html）→ 字符流（自动处理编码）
- 不确定 → 字节流（万能，但文本要自己处理编码）

#### 2. 为什么要套缓冲流？

直接 `FileInputStream.read()` 每读一个字节就访问一次磁盘，极慢。套上 `BufferedInputStream` 后，会**一次读一大块到内存缓冲区**，后续从内存读，快几十倍。详见 [[缓冲流]]。

#### 3. 字符流读文本乱码？

字符流用的是**系统默认编码**，和文件实际编码不一致就乱码。要用 [[转换流]] 显式指定编码。

#### 4. 流能不能重复读？

流是「单向、一次性」的，读完就没了。要重读需重新创建，或用 `mark()/reset()`（少数流支持）。

#### 5. 读完判断结束

- 字节流/字符流 `read()` 返回 **-1** 表示结束
- `BufferedReader.readLine()` 返回 **null** 表示结束

| 要点 | 说明 |
| --- | --- |
| 三个分类 | 方向（输入/输出）、单位（字节/字符）、功能（节点/处理） |
