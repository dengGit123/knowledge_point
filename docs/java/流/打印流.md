### 一、概述

> 📖 [PrintStream（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/PrintStream.html) ｜ [PrintWriter（Java）](https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/io/PrintWriter.html)

打印流是**最方便的输出流**。它提供了 `print()` / `println()` / `printf()` 等方法，能**直接打印各种类型的数据**，不用自己转成字节或字符串。

大白话：普通输出流写数据很麻烦，要先转成 byte/char；打印流像 `System.out.println()` 一样，**想打印什么就打印什么**（数字、布尔、对象都行），自动帮你转换。

```
// 普通字节流：写个数字要手动转
outputStream.write("123".getBytes());

// 打印流：直接打印
printStream.println(123);
printStream.println("你好");
printStream.println(3.14);
```

| 打印流 | 类型 | 典型代表 |
| --- | --- | --- |
| `PrintStream` | 字节打印流 | **`System.out` 就是它** |
| `PrintWriter` | 字符打印流 | Web 开发 `response.getWriter()` |

> 💡 **提示：** 你天天用的 `System.out.println()`，底层就是 `PrintStream`！打印流是日常接触最多的流。

### 二、打印流的特点

#### 1. 只输出，不输入

打印流**只有输出**（`print/write`），没有读取方法，是纯粹的「写」流。

#### 2. 方便地打印各种类型

```java
printStream.println(100);          // int
printStream.println(3.14);         // double
printStream.println(true);         // boolean
printStream.println('A');          // char
printStream.println("字符串");      // String
printStream.println(new Object()); // 对象（调用 toString）
printStream.printf("名字：%s，年龄：%d", "张三", 20);  // 格式化
```

#### 3. 自动 flush（可配置）

```java
// 第二个参数 true = 开启自动刷新（每次 println 自动 flush）
new PrintWriter(new FileWriter("a.txt"), true);
```

#### 4. 不抛 IOException（重要特征）

打印流的 `print` 方法**不抛出 `IOException`**，而是把异常吞掉，存到 `checkError()` 里：

```java
PrintWriter writer = new PrintWriter("a.txt");
writer.println("hello");   // ❌ 不抛异常（即使底层出错）
if (writer.checkError()) { // ✅ 要主动检查是否有错
    System.out.println("写入出错");
}
```

> ⚠️ **注意：** 打印流「不抛异常」既是优点（代码简洁）也是陷阱——**错误被静默吞掉**，生产环境记得用 `checkError()` 检查。

### 三、基本用法

#### 1. PrintStream（字节打印流）

```java
import java.io.*;

public class PrintStreamDemo {
    public static void main(String[] args) throws IOException {
        // 打印到文件
        try (PrintStream ps = new PrintStream("log.txt", "UTF-8")) {
            ps.println("===== 日志开始 =====");
            ps.println("时间: " + System.currentTimeMillis());
            ps.printf("用户 %s 登录，状态：%b%n", "admin", true);
            ps.print("数字：");
            ps.println(42);
        }

        // System.out 就是 PrintStream
        PrintStream out = System.out;
        out.println("打印到控制台");
    }
}
```

#### 2. PrintWriter（字符打印流，更常用）

```java
import java.io.*;

public class PrintWriterDemo {
    public static void main(String[] args) throws IOException {
        // ✅ 字符打印流，可指定编码，autoflush
        try (PrintWriter pw = new PrintWriter(
                new OutputStreamWriter(
                    new FileOutputStream("b.txt"), "UTF-8"), true)) {

            pw.println("第一行");
            pw.println("第二行");
            pw.printf("格式化：%d-%d-%d%n", 2026, 7, 2);
        }

        // Web 场景：response.getWriter() 返回的就是 PrintWriter
        // response.getWriter().println("<h1>Hello</h1>");
    }
}
```

### 四、格式化输出 printf / format

打印流支持 C 语言风格的格式化：

```java
PrintStream out = System.out;

// printf 和 format 完全等价
out.printf("我叫 %s，今年 %d 岁%n", "张三", 20);
out.format("圆周率约 %.2f%n", 3.14159);
```

#### 常用格式符

| 格式符 | 含义 | 示例 |
| --- | --- | --- |
| `%s` | 字符串 | `"张三"` |
| `%d` | 十进制整数 | `20` |
| `%f` | 浮点数 | `3.14` |
| `%.2f` | 保留 2 位小数 | `3.14` |
| `%b` | 布尔 | `true` |
| `%c` | 字符 | `A` |
| `%n` | 换行（跨平台） | 换行 |
| `%x` | 十六进制 | `ff` |

```java
System.out.printf("%-10s|%5d%n", "姓名", "分数");  // 左对齐/宽度
// 输出：姓名      |  分数
```

> 💡 **提示：** 格式化输出推荐用 `%n` 而非 `\n`，保证跨平台换行正确。

### 五、PrintStream vs PrintWriter

两者功能几乎一样，区别：

| 对比项 | PrintStream | PrintWriter |
| --- | --- | --- |
| 单位 | 字节 | 字符 |
| 编码 | JDK 5+ 也能指定编码 | 天然支持指定编码 |
| 代表 | `System.out` | Web `response.getWriter()` |
| 自动 flush | 按 println 触发 | 构造时可指定 autoflush |
| 推荐度 | 控制台输出 | **文本文件/Web 推荐** |

```
PrintStream  ←─继承─ OutputStream（字节）
PrintWriter  ←─继承─ Writer（字符）
```

> 💡 **提示：** 写**文本文件**优先 `PrintWriter`（字符流，编码友好）；用 `System.out` 时它本身就是 `PrintStream`，不必纠结。

### 六、System.out 的真相

`System.out` 是一个 `PrintStream` 类型的静态常量：

```java
// System 类的源码
public final static PrintStream out = null;  // 初始化为指向控制台

// 所以你可以：
PrintStream out = System.out;
out.println("hello");

// 甚至替换它（重定向，见 [[标准流]]）
System.setOut(new PrintStream("log.txt"));
```

详见 [[标准流]]。

### 七、实际应用场景

| 场景 | 用法 |
| --- | --- |
| 控制台输出 | `System.out.println`（PrintStream） |
| 写日志文件 | `PrintWriter`（配合 autoflush + 编码） |
| Web 响应输出 | `response.getWriter()`（PrintWriter） |
| 生成报表/模板文本 | `printf` 格式化 |
| 调试输出 | `System.err.println`（红色错误） |

```java
// 写日志（实战示例）
try (PrintWriter logger = new PrintWriter(
        new FileWriter("app.log", true), true)) {  // true=追加, true=autoflush
    logger.printf("[%s] %s%n", "INFO", "服务启动成功");
}
```

### 八、常见问题与注意事项

#### 1. 不抛异常导致错误被忽略

```java
// ❌ 文件路径错误、磁盘满，println 都不报错，数据悄悄丢失
PrintWriter pw = new PrintWriter("/不存在/的/路径/a.txt");
pw.println("重要数据");
pw.close();  // 没异常！

// ✅ 关键场景用 checkError 检查
if (pw.checkError()) {
    throw new RuntimeException("写入失败");
}
```

#### 2. autoflush 默认关闭

```java
// ❌ 不开 autoflush，println 后数据在缓冲区，文件可能为空
new PrintWriter(new FileWriter("a.txt"));

// ✅ 构造时传 true 开启 autoflush
new PrintWriter(new FileWriter("a.txt"), true);
```

#### 3. PrintStream 直接传 File/String 构造（JDK 10+ 编码坑）

```java
// PrintStream(String) 老版本用默认编码，可能乱码
new PrintStream("a.txt");   // 默认编码

// ✅ 显式指定编码
new PrintStream("a.txt", "UTF-8");
```

#### 4. 性能：打印流内部自带缓冲

`PrintStream`/`PrintWriter` 写入时会经过缓冲，但**和 BufferedXxx 的缓冲不完全等同**。高频写入仍建议套缓冲流或开 autoflush。

#### 5. println 自动加换行

`println` 会自动在末尾加平台换行符；`print` 不会。注意区分。

### 九、总结

| 要点 | 说明 |
| --- | --- |
| 作用 | 最方便的输出流，直接 print 各种类型 |
| 两个类 | `PrintStream`（字节，System.out）、`PrintWriter`（字符） |
| 核心方法 | `print` / `println` / `printf` / `format` |
| 特点 | 不抛异常（要 checkError）、可 autoflush |
| 格式化 | `%s` `%d` `%f` `%n` 等格式符 |

一句话：**要方便地输出各种数据，用打印流；要写文本文件，用 PrintWriter 并指定编码**。
